## <a name="example"></a><span data-ttu-id="2b825-128">Пример</span><span class="sxs-lookup"><span data-stu-id="2b825-128">Example</span></span>

<span data-ttu-id="2b825-129">Следующий фрагмент кода содержит абстрактный тип, нарушающий это правило.</span><span class="sxs-lookup"><span data-stu-id="2b825-129">The following code snippet contains an abstract type that violates this rule.</span></span>

:::code language="vb" source="snippets/vb/all-rules/ca1012-abstract-types-should-not-have-constructors_1.vb" id="snippet1":::

```csharp
// Violates this rule
public abstract class Book
{
    public Book()
    {
    }
}
```

<span data-ttu-id="2b825-130">Следующий фрагмент кода устраняет предыдущее нарушение, изменяя доступность конструктора с `public` на `protected` .</span><span class="sxs-lookup"><span data-stu-id="2b825-130">The following code snippet fixes the previous violation by changing the accessibility of the constructor from `public` to `protected`.</span></span>

```csharp
// Does not violate this rule
public abstract class Book
{
    protected Book()
    {
    }
}
```
