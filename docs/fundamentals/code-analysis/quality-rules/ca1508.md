## <a name="rule-description"></a><span data-ttu-id="b0385-115">Описание правила</span><span class="sxs-lookup"><span data-stu-id="b0385-115">Rule description</span></span>

<span data-ttu-id="b0385-116">Методы могут иметь условный код, например операторы If, двоичные выражения ( `==` , `!=` , `<` , `>` ), проверки значений NULL и т. д. Например, рассмотрим следующий код:</span><span class="sxs-lookup"><span data-stu-id="b0385-116">Methods can have conditional code, such as if statements, binary expressions (`==`, `!=`, `<`, `>`), null checks, etc. For example, consider the following code:</span></span>

```csharp
public void M(int i, int j)
{
    if (i != 0)
    {
        return;
    }

    if (j != 0)
    {
        return;
    }

    // Below condition will always evaluate to 'false' as 'i' and 'j' are both '0' here.
    if (i != j)
    {
        // Code in this 'if' branch is dead code.
        // It can either be removed or refactored.
        ...
    }
}
```

.</span><span class="sxs-lookup"><span data-stu-id="b0385-117">C# and VB compilers perform analysis of conditional checks involving compile-time _constant values_ that always evaluate to `true` or _неконстантные значения_.</span><span class="sxs-lookup"><span data-stu-id="b0385-118">This analyzer performs data flow analysis of non-constant variables to determine redundant conditional checks involving _non-constant values_.</span></span> <span data-ttu-id="b0385-119">В приведенном выше коде анализатор определяет, что `i` и `j` оба являются `0` для всех ветвей кода, доступных для `i != j` проверки.</span><span class="sxs-lookup"><span data-stu-id="b0385-119">In the preceding code, the analyzer determines that `i` and `j` are both `0` for all code paths that reach `i != j` data-ttu-id="b0385-122">Аналогичным образом анализатор отслеживает значения NULL переменных и сообщает о избыточных проверках значений NULL.</span><span class="sxs-lookup"><span
