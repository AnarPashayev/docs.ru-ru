# <a name="net-runtime-contention-events"></a>События состязания времени выполнения .NET

Эти события времени выполнения захватывают сведения о конфликтах блокировки монитора, таких как WITH `Monitor.Enter` или ключевое слово C# lock. Дополнительные сведения об использовании этих событий в целях диагностики см. в разделе [ведение журнала и трассировка приложений .NET](../../core/diagnostics/logging-tracing.md) .

## <a name="contentionstart_v1-event"></a>ContentionStart_V1 событие

Это событие создается в начале конфликта блокировки монитора.

|Ключевое слово для вызова события|Level|
|-----------------------------------|-----------|
|`ContentionKeyword` (0x4000)|Информационный (4)|

 В таблице ниже представлены сведения о событии.

|Событие|Идентификатор события|Условие вызова|
|-----------|--------------|-----------------|
|`ContentionStart_V1`|81|Начинается состязание за блокировки монитора.|

|Имя поля|Тип данных|Описание|
|----------------|---------------|-----------------|
|`Flags`|`win:UInt8`|`0` для управляемого; `1` для машинного кода.|
|`ClrInstanceID`|`win:UInt16`|Уникальный идентификатор для экземпляра CoreCLR.|

## <a name="contentionstop_v1-event"></a>ContentionStop_V1 событие

Это событие создается в конце конфликта блокировки монитора.

|Ключевое слово для вызова события|Level|
|-----------------------------------|-----------|
|`ContentionKeyword` (0x4000)|Информационный (4)|

 В таблице ниже представлены сведения о событии.

|Событие|Идентификатор события|Условие вызова|
|-----------|--------------|-----------------|
|`ContentionStop_V1`|91|Завершено состязание за блокировку монитора.|

|Имя поля|Тип данных|Описание|
|----------------|---------------|-----------------|
|`Flags`|`win:UInt8`|`0` для управляемого; `1` для машинного кода.|
|`ClrInstanceID`|`win:UInt16`|Уникальный идентификатор для экземпляра CoreCLR.|
|`DurationNs`|`win:Double`|Длительность состязания в наносекундах.|
