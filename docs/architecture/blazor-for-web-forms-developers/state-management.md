<span data-ttu-id="7bbe5-104">Управление состоянием — это ключевая концепция приложений веб-форм, которая упрощает работу с состоянием просмотра, состоянием сеанса, состоянием приложения и обратной передачей.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-104">State management is a key concept of Web Forms applications, facilitated through View State, Session State, Application State, and Postback features.</span></span> <span data-ttu-id="7bbe5-105">Эти функции платформы с отслеживанием состояния позволяют скрыть управление состоянием, необходимое для приложения, и позволить разработчикам приложений сосредоточиться на предоставлении их функциональности.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-105">These stateful features of the framework helped to hide the state management required for an application and allow application developers to focus on delivering their functionality.</span></span> <span data-ttu-id="7bbe5-106">При использовании ASP.NET Core и Блазор некоторые из этих функций были перемещены, а некоторые из них были удалены полностью.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-106">With ASP.NET Core and Blazor, some of these features have been relocated and some have been removed altogether.</span></span> <span data-ttu-id="7bbe5-107">В этой главе рассматриваются способы поддержания состояния и предоставления тех же функций с помощью новых функций в Блазор.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-107">This chapter reviews how to maintain state and deliver the same functionality with the new features in Blazor.</span></span>

<span data-ttu-id="7bbe5-109">При обсуждении управления состоянием в приложении веб-форм многие разработчики сразу же считают ViewState.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-109">When discussing state management in Web Forms application, many developers will immediately think of ViewState.</span></span> <span data-ttu-id="7bbe5-110">В Web Forms ViewState управляет состоянием содержимого между запросами HTTP, отправляя большой кодированный блок текста назад и вперед в браузер.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-110">In Web Forms, ViewState manages the state of the content between HTTP requests by sending a large encoded block of text back and forth to the browser.</span></span> <span data-ttu-id="7bbe5-111">Поле ViewState может быть перегружено содержимым страницы, содержащей много элементов, что может увеличиться до нескольких мегабайт.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-111">The ViewState field could be overwhelmed with content from a page containing many elements, potentially expanding to several megabytes in size.</span></span>

<span data-ttu-id="7bbe5-112">При использовании Блазор Server приложение сохраняет текущее подключение к серверу.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-112">With Blazor Server, the app maintains an ongoing connection with the server.</span></span> <span data-ttu-id="7bbe5-113">Состояние приложения, называемое *цепью*, хранится в памяти сервера, пока подключение считается активным.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-113">The app's state, called a *circuit*, is held in server memory while the connection is considered active.</span></span> <span data-ttu-id="7bbe5-114">Состояние будет удалено только при переходе пользователя из приложения или определенной страницы в приложении.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-114">State will only be disposed when the user navigates away from the app or a particular page in the app.</span></span> <span data-ttu-id="7bbe5-115">Все члены активных компонентов доступны между взаимодействием с сервером.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-115">All members of the active components are available between interactions with the server.</span></span>

<span data-ttu-id="7bbe5-116">Существует несколько преимуществ этой функции.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-116">There are several advantages of this feature:</span></span>

- <span data-ttu-id="7bbe5-117">Состояние компонента легко доступно и не перестраивается между взаимодействиями.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-117">Component state is readily available and not rebuilt between interactions.</span></span>
- <span data-ttu-id="7bbe5-118">Состояние не передается в браузер.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-118">State isn't transmitted to the browser.</span></span>

<span data-ttu-id="7bbe5-119">Однако существуют некоторые недостатки сохранения состояния компонента в памяти для того, чтобы иметь в виду следующее.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-119">However, there are some disadvantages to in-memory component state persistence to be aware of:</span></span>

- <span data-ttu-id="7bbe5-120">Если сервер перезапускается между запросами, состояние теряется.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-120">If the server restarts between request, state is lost.</span></span>
- <span data-ttu-id="7bbe5-121">Решение для балансировки нагрузки веб-сервера приложения должно включать прикрепленные сеансы, чтобы все запросы из одного браузера возвращались на один и тот же сервер.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-121">Your application web server load-balancing solution must include sticky sessions to ensure that all requests from the same browser return to the same server.</span></span> <span data-ttu-id="7bbe5-122">Если запрос переходит на другой сервер, состояние будет потеряно.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-122">If a request goes to a different server, state will be lost.</span></span>
- <span data-ttu-id="7bbe5-123">Сохранение состояния компонента на сервере может привести к нехватке памяти на веб-сервере.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-123">Persistence of component state on the server can lead to memory pressure on the web server.</span></span>

<span data-ttu-id="7bbe5-124">По этим причинам не следует полагаться на то, что компонент должен находиться в памяти на сервере.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-124">For the preceding reasons, don't rely on just the state of the component to reside in-memory on the server.</span></span> <span data-ttu-id="7bbe5-125">Приложение также должно включать в себя резервное хранилище данных для данных между запросами.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-125">Your application should also include some backing data store for data between requests.</span></span> <span data-ttu-id="7bbe5-126">Вот несколько простых примеров этой стратегии:</span><span class="sxs-lookup"><span data-stu-id="7bbe5-126">Some simple examples of this strategy:</span></span>

- <span data-ttu-id="7bbe5-127">В приложении корзины для покупок сохраняет содержимое новых элементов, добавленных в корзину, в записи базы данных.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-127">In a shopping cart application, persist the content of new items added to the cart in a database record.</span></span> <span data-ttu-id="7bbe5-128">Если состояние на сервере потеряно, его можно восстановить из записей базы данных.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-128">If the state on the server is lost, you can reconstitute it from the database records.</span></span>
- <span data-ttu-id="7bbe5-129">В многокомпонентной веб-форме пользователи хотят, чтобы ваше приложение запомнило значения между каждым запросом.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-129">In a multi-part web form, your users will expect your application to remember values between each request.</span></span> <span data-ttu-id="7bbe5-130">Запишите данные между каждой записью пользователя в хранилище данных, чтобы их можно было извлечь и собрать в итоговую структуру ответа формы при завершении многокомпонентной формы.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-130">Write the data between each of your user's posts to a data store so that they can be fetched and assembled into the final form response structure when the multi-part form is completed.</span></span>

<span data-ttu-id="7bbe5-131">Дополнительные сведения об управлении состоянием в приложениях Блазор см. в разделе [ASP.NET Core блазор State Management](/aspnet/core/blazor/state-management).</span><span class="sxs-lookup"><span data-stu-id="7bbe5-131">For additional details on managing state in Blazor apps, see [ASP.NET Core Blazor state management](/aspnet/core/blazor/state-management).</span></span>

## <a name="application-state"></a><span data-ttu-id="7bbe5-141">Состояние приложения</span><span class="sxs-lookup"><span data-stu-id="7bbe5-141">Application state</span></span>

<span data-ttu-id="7bbe5-142">`Application`Объект в платформе веб-форм предоставляет обширный репозиторий между запросами для взаимодействия с конфигурацией и состоянием области приложения.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-142">The `Application` object in the Web Forms framework provides a massive, cross-request repository for interacting with application-scope configuration and state.</span></span> <span data-ttu-id="7bbe5-143">Состояние приложения было идеальным местом для хранения различных свойств конфигурации приложения, на которые могут ссылаться все запросы, независимо от пользователя, выполняющего запрос.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-143">Application state was an ideal place to store various application configuration properties that would be referenced by all requests, regardless of the user making the request.</span></span> <span data-ttu-id="7bbe5-144">Проблема с объектом заключается в `Application` том, что данные не сохранялись на нескольких серверах.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-144">The problem with the `Application` object was that data didn't persist across multiple servers.</span></span> <span data-ttu-id="7bbe5-145">Состояние объекта приложения было потеряно между перезапусками.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-145">The state of the application object was lost between restarts.</span></span>

<span data-ttu-id="7bbe5-146">Как и в `Session` случае, рекомендуется перемещать данные в постоянное резервное хранилище, доступное нескольким экземплярам сервера.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-146">As with `Session`, it's recommended that data move to a persistent backing store that could be accessed by multiple server instances.</span></span> <span data-ttu-id="7bbe5-147">Если есть постоянные данные, к которым вы хотите получить доступ по запросам и пользователям, вы можете легко сохранить их в одноэлементной службе, которую можно внедрить в компоненты, требующие этой информации или взаимодействия.</span><span class="sxs-lookup"><span data-stu-id="7bbe5-147">If there is volatile data that you would like to be able to access across requests and users, you could easily store it in a singleton service that can be injected into components that require this information or interaction.</span></span>

<span data-ttu-id="7bbe5-148">Создание объекта для поддержки состояния приложения и его потребления может напоминать следующую реализацию:</span><span class="sxs-lookup"><span data-stu-id="7bbe5-148">The construction of an object to maintain application state and its consumption could resemble the following implementation:</span></span>
