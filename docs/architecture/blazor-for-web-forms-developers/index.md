## <a name="introduction"></a>Вступление

.NET уже давно поддерживает разработку веб-приложений через ASP.NET — комплексный набор платформ и средств для создания любых типов приложений. ASP.NET имеет собственную линейку веб-платформ и технологий, начиная с классического ASP. Такие платформы, как ASP.NET Web Forms, ASP.NET MVC, веб-страницы ASP.NET и более поздняя ASP.NET Core, предоставляют эффективный и функциональный способ создания веб-приложений, *подготавливаемых к просмотру на сервере*, в которых содержимое пользовательского интерфейса динамически формируется на сервере в ответ на HTTP-запросы. Каждая платформа ASP.NET ориентирована на определенную аудиторию и подход к созданию приложений. ASP.NET Web Forms поставляется с исходным выпуском .NET Framework и поддержкой веб-разработки с использованием множества шаблонов, знакомых разработчикам классических приложений, таких как многократно используемые элементы управления пользовательского интерфейса с простой обработкой событий. Однако ни одно из предложений ASP.NET не предоставляет способа для выполнения кода, выполняемого в браузере пользователя. Для этого требуется написать код JavaScript и использовать любые из множества платформ и средств JavaScript, которые набирали и теряли популярность в течение многих лет: jQuery, Knockout, Angular, React и т. д.

[Blazor](https://blazor.net) — это новая веб-платформа, которая расширяет возможности создания веб-приложений с помощью .NET. Blazor представляет собой клиентскую платформу веб-интерфейса, основанную на C#, а не JavaScript. С помощью Blazor вы можете писать клиентскую логику и компоненты пользовательского интерфейса на C#, компилировать их в обычные сборки .NET, а затем запускать их непосредственно в браузере с помощью нового открытого веб-стандарта WebAssembly. Кроме того, Blazor позволяет запускать компоненты пользовательского интерфейса .NET на сервере и гибко управлять всеми взаимодействиями пользовательского интерфейса через подключение в режиме реального времени с помощью браузера. При использовании вместе с .NET, работающем на сервере, Blazor обеспечивает комплексную веб-разработку на базе .NET. Хотя платформа Blazor имеет множество общих черт с ASP.NET Web Forms, таких как многократно используемая модель компонентов и простая обработка пользовательских событий, она также основана на .NET Core, предоставляя современный и высокопроизводительный процесс веб-разработки.

Эта книга привычным и удобным образом знакомит разработчиков ASP.NET Web Forms с Blazor. В ней представлены концепции Blazor вместе с аналогичными концепциями в ASP.NET Web Forms, а также разобраны новые концепции, которые могут оказаться непривычными. Она охватывает обширный спектр вопросов и проблем, включая создание компонентов, маршрутизацию, макеты, настройку и безопасность. И хотя эта книга в основном посвящена формированию новой процедуры разработки, в ней также описаны рекомендации и стратегии переноса существующих решений ASP.NET Web Forms в Blazor, когда требуется модернизировать существующее приложение.

## <a name="who-should-use-the-book"></a>Для кого предназначена эта книга

Эта книга предназначена для разработчиков ASP.NET Web Forms, желающих познакомиться с Blazor в контексте имеющихся у них знаний и навыков. Эта книга поможет быстро приступить к работе над новым проектом Blazor или составить план модернизации существующего приложения ASP.NET Web Forms.

## <a name="how-to-use-the-book"></a>Как пользоваться книгой

В первой части этой книги приведено описание платформы Blazor и ее сравнение с разработкой веб-приложений с использованием ASP.NET Web Forms. Затем в отдельных главах книги описываются различные аспекты Blazor и приводятся сведения о каждой концепции Blazor к соответствующей концепции в ASP.NET Web Forms либо рассматриваются совершенно новые концепции. Книга также регулярно ссылается на полный пример приложения, реализованный как в ASP.NET Web Forms, так и в Blazor, чтобы продемонстрировать функции Blazor и предоставить пример перехода с ASP.NET Web Forms на Blazor. Обе реализации примера приложения (ASP.NET Web Forms и Blazor) можно найти на сайте [GitHub.](https://github.com/dotnet-architecture/eshoponblazor)

## <a name="what-this-book-doesnt-cover"></a>Темы, которые выходят за рамки этой книги

Эта книга представляет собой введение в Blazor, а не исчерпывающее руководство по миграции. Хотя она включает в себя рекомендации по переносу проекта из ASP.NET Web Forms в Blazor, в ней отсутствует описание всех особенностей и деталей. Общие рекомендации по переходу с ASP.NET на ASP.NET Core см. в [руководстве по миграции](https://docs.microsoft.com/aspnet/core/migration/proper-to-2x/) в документации по ASP.NET Core.
