<span data-ttu-id="dfd5c-105">В распределенных системах, таких как приложения на базе микрослужб, всегда есть риск частичного сбоя.</span><span class="sxs-lookup"><span data-stu-id="dfd5c-105">In distributed systems like microservices-based applications, there's an ever-present risk of partial failure.</span></span> <span data-ttu-id="dfd5c-106">Например, одна микрослужба или контейнер могут отказать или не отвечать какое-то время или одна виртуальная машина или сервер могут дать сбой.</span><span class="sxs-lookup"><span data-stu-id="dfd5c-106">For instance, a single microservice/container can fail or might not be available to respond for a short time, or a single VM or server can crash.</span></span> <span data-ttu-id="dfd5c-107">Поскольку клиенты и службы являются отдельными процессами, служба может не ответить вовремя на запрос клиента.</span><span class="sxs-lookup"><span data-stu-id="dfd5c-107">Since clients and services are separate processes, a service might not be able to respond in a timely way to a client’s request.</span></span> <span data-ttu-id="dfd5c-108">Служба может быть перегружена, очень медленно отвечать на запросы, быть недоступна некоторое время из-за проблем в сети.</span><span class="sxs-lookup"><span data-stu-id="dfd5c-108">The service might be overloaded and responding very slowly to requests or might simply not be accessible for a short time because of network issues.</span></span>

<span data-ttu-id="dfd5c-109">Рассмотрим страницу сведений о заказе в примере приложения eShopOnContainers.</span><span class="sxs-lookup"><span data-stu-id="dfd5c-109">For example, consider the Order details page from the eShopOnContainers sample application.</span></span> <span data-ttu-id="dfd5c-110">Если микрослужба заказа не отвечает при попытке оформить заказ, в результате неправильного <span data-ttu-id="dfd5c-125">Периодические сбои неизбежны в распределенной и облачной системе, даже если каждая зависимость в отдельности хорошо доступна.</span><span class="sxs-lookup"><span data-stu-id="dfd5c-125">Intermittent failure is guaranteed in a distributed and cloud-based system, even if every dependency itself has excellent availability.</span></span> <span data-ttu-id="dfd5c-126">Этот факт необходимо учитывать.</span><span class="sxs-lookup"><span data-stu-id="dfd5c-126">It's a fact you need to consider.</span></span>
