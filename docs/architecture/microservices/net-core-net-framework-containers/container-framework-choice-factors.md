# <a name="decision-table-net-frameworks-to-use-for-docker"></a>Таблица для принятия решений. Использование платформ .NET для Docker

В таблице ниже для принятия решений приводятся сводные сведения о том, когда следует использовать .NET Framework или .NET 5. Помните, что для контейнеров Linux требуются узлы Docker на основе Linux (виртуальные машины или серверы), а для контейнеров Windows — узлы Docker на основе Windows Server (виртуальные машины или серверы).

> [!IMPORTANT]
> На компьютерах разработки выполняется один узел Docker: Linux или Windows. Все связанные микрослужбы, которые должны запускаться и тестироваться вместе в рамках одного решения, должны выполняться на одной платформе контейнеров.

| Архитектура и тип приложения | Контейнеры Linux | Контейнеры Windows |
|-------------------------|------------------|--------------------|
| Микрослужбы в контейнерах | .NET 5 | .NET 5 |
| Монолитные приложения | .NET 5 | .NET Framework <br/> .NET 5 |
| Достижение высочайшей производительности и масштабируемости | .NET 5 | .NET 5 |
| Перенос имеющихся (устаревших) приложений Windows Server в контейнеры | -- | .NET Framework |
| Разработка новых приложений на основе контейнеров (с нуля) | .NET 5 | .NET 5 |
| ASP.NET Core | .NET 5 | .NET 5 (рекомендуется) <br/> .NET Framework |
| ASP.NET 4 (MVC 5, веб-API 2 и веб-формы) | -- | .NET Framework |
| Службы SignalR | .NET Core 2.1 или более поздней версии | .NET Framework <br/> .NET Core 2.1 или более поздней версии |
| WCF, WF и другие устаревшие платформы | WCF в .NET Core (только клиентская библиотека) | .NET Framework <br/> WCF в .NET 5 (только клиентская библиотека) |
| Использование служб Azure | .NET 5 <br/> (в конечном итоге большинство служб Azure будет предоставлять клиентские пакеты SDK для .NET 5) | .NET Framework <br/> .NET 5 <br/> (в конечном итоге большинство служб Azure будет предоставлять клиентские пакеты SDK для .NET 5) |
