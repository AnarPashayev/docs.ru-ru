<span data-ttu-id="d9a39-163">Blazor входит в состав ASP.NET Core 3.0 и более поздних версий.</span><span class="sxs-lookup"><span data-stu-id="d9a39-163">Blazor is included with ASP.NET Core 3.0 and later.</span></span> <span data-ttu-id="d9a39-164">Он предоставляет новый механизм для создания многофункциональных интерактивных клиентских веб-приложений с помощью Razor, C# и ASP.NET Core.</span><span class="sxs-lookup"><span data-stu-id="d9a39-164">It provides a new mechanism for building rich interactive web client applications using Razor, C#, and ASP.NET Core.</span></span> <span data-ttu-id="d9a39-165">Он предлагает еще одно решение, которое следует учитывать при разработке современных веб-приложений.</span><span class="sxs-lookup"><span data-stu-id="d9a39-165">It offers another solution to consider when developing modern web applications.</span></span> <span data-ttu-id="d9a39-166">Существует две версии Blazor: на стороне сервера и на стороне клиента.</span><span class="sxs-lookup"><span data-stu-id="d9a39-166">There are two versions of Blazor to consider: server-side and client-side.</span></span>

<span data-ttu-id="d9a39-167">Функция Blazor на стороне сервера выпущена в 2019 г. с ASP.NET Core 3.0.</span><span class="sxs-lookup"><span data-stu-id="d9a39-167">Server-side Blazor was released in 2019 with ASP.NET Core 3.0.</span></span> <span data-ttu-id="d9a39-168">Как следует из названия, функция работает на сервере, преобразуя изменения в клиентском документе для отображения в браузер по сети.</span><span class="sxs-lookup"><span data-stu-id="d9a39-168">As its name implies, it runs on the server, rendering changes to the client document back to the browser over the network.</span></span> <span data-ttu-id="d9a39-169">Blazor на стороне сервера обеспечивает широкие возможности для взаимодействия с пользователем, не требуя наличия JavaScript на стороне клиента и отдельной загрузки страниц для каждого взаимодействия с клиентской страницей.</span><span class="sxs-lookup"><span data-stu-id="d9a39-169">Server-side Blazor provides a rich client experience without requiring client-side JavaScript and without requiring separate page loads for each client page interaction.</span></span> <span data-ttu-id="d9a39-170">Изменения на загруженной странице запрашиваются и обрабатываются сервером, а затем отправляются обратно клиенту с помощью SignalR.</span><span class="sxs-lookup"><span data-stu-id="d9a39-170">Changes in the loaded page are requested from and processed by the server and then sent back to the client using SignalR.</span></span>

<span data-ttu-id="d9a39-171">Функция Blazor на стороне клиента, выпущенная в мае 2020 года, устраняет необходимость в отрисовке изменений на сервере.</span><span class="sxs-lookup"><span data-stu-id="d9a39-171">Client-side Blazor released in May 2020 eliminates the need to render changes on the server.</span></span> <span data-ttu-id="d9a39-172">Вместо этого для запуска кода .NET в клиенте функция использует WebAssembly.</span><span class="sxs-lookup"><span data-stu-id="d9a39-172">Instead, it leverages WebAssembly to run .NET code within the client.</span></span> <span data-ttu-id="d9a39-173">Клиент по-прежнему сможет выполнять вызовы API на сервер, если необходимо запросить данные, но все поведение на стороне клиента выполняется в клиенте через WebAssembly, которая уже поддерживается всеми основными браузерами и является простой библиотекой JavaScript.</span><span class="sxs-lookup"><span data-stu-id="d9a39-173">The client can still make API calls to the server if needed to request data, but all client-side behavior runs in the client via WebAssembly, which is already supported by all major browsers and is just a Javascript library.</span></span>
