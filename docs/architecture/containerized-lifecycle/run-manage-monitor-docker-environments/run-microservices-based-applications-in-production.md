# <a name="run-composed-and-microservices-based-applications-in-production-environments"></a><span data-ttu-id="d6bea-103">Запуск составных и основанных на микрослужбах приложений в рабочих средах</span><span class="sxs-lookup"><span data-stu-id="d6bea-103">Run composed and microservices-based applications in production environments</span></span>

<span data-ttu-id="d6bea-104">Приложения, состоящие из нескольких микрослужб, нужно развертывать в кластерах оркестратора, чтобы упростить развертывание и сделать его возможным с учетом доступных ИТ-ресурсов.</span><span class="sxs-lookup"><span data-stu-id="d6bea-104">Applications composed by multiple microservices do need to be deployed into orchestrator clusters in order to simplify the complexity of deployment and make it viable from an IT point of view.</span></span> <span data-ttu-id="d6bea-105">Без кластера оркестратора было бы трудно развернуть и масштабировать сложное приложение микрослужб.</span><span class="sxs-lookup"><span data-stu-id="d6bea-105">Without an orchestrator cluster, it would be difficult to deploy and scale out a complex microservices application.</span></span>

## <a name="introduction-to-orchestrators-schedulers-and-container-clusters"></a><span data-ttu-id="d6bea-106">Введение в оркестраторы, планировщики и кластеры контейнеров</span><span class="sxs-lookup"><span data-stu-id="d6bea-106">Introduction to orchestrators, schedulers, and container clusters</span></span>

<span data-ttu-id="d6bea-107">Ранее в этой электронной книге *кластеры* и *планировщики* упоминались в рамках обсуждения архитектуры и разработки программного обеспечения.</span><span class="sxs-lookup"><span data-stu-id="d6bea-107">Earlier in this e-book, *clusters* and *schedulers* were introduced as part of the discussion on software architecture and development.</span></span> <span data-ttu-id="d6bea-108">Kubernetes и Service Fabric являются примерами кластеров Docker.</span><span class="sxs-lookup"><span data-stu-id="d6bea-108">Kubernetes and Service Fabric are examples of Docker clusters.</span></span> <span data-ttu-id="d6bea-109">Оба эти оркестратора могут выполняться в составе инфраструктуры Службы Azure Kubernetes.</span><span class="sxs-lookup"><span data-stu-id="d6bea-109">Both of these orchestrators can run as a part of the infrastructure provided by Microsoft Azure Kubernetes Service.</span></span>

<span data-ttu-id="d6bea-110">Когда приложения масштабируются по нескольким системам узла, привлекательной становится возможность управления каждой системой узла и абстрагирования от сложности базовой платформы.</span><span class="sxs-lookup"><span data-stu-id="d6bea-110">When applications are scaled-out across multiple host systems, the ability to manage each host system and abstract away the complexity of the underlying platform becomes attractive.</span></span> <span data-ttu-id="d6bea-111">Именно за это и отвечают оркестраторы и планировщики.</span><span class="sxs-lookup"><span data-stu-id="d6bea-111">That's precisely what orchestrators and schedulers provide.</span></span> <span data-ttu-id="d6bea-112">Давайте кратко рассмотрим их:</span><span class="sxs-lookup"><span data-stu-id="d6bea-112">Let's take a brief look at them here:</span></span>

- <span data-ttu-id="d6bea-113">**Планировщики**.</span><span class="sxs-lookup"><span data-stu-id="d6bea-113">**Schedulers**.</span></span><span data-ttu-id="d6bea-114"> "Планирование" обозначает способность администратора загрузить на систему узла файл службы, который описывает выполнение конкретного контейнера.</span><span class="sxs-lookup"><span data-stu-id="d6bea-114"> "Scheduling" refers to the ability for an administrator to load a service file onto a host system that establishes how to run a specific container.</span></span> <span data-ttu-id="d6bea-115">Запуск контейнеров в кластере Docker обычно называется планированием.</span><span class="sxs-lookup"><span data-stu-id="d6bea-115">Launching containers in a Docker cluster tends to be known as scheduling.</span></span> <span data-ttu-id="d6bea-116">Хотя планирование обозначает конкретную процедуру загрузки определения службы, в более общем смысле планировщики отвечают за подключение к системе инициализации узла для управления службами с любой необходимой емкостью.</span><span class="sxs-lookup"><span data-stu-id="d6bea-116">Although scheduling refers to the specific act of loading the service definition, in a more general sense, schedulers are responsible for hooking into a host's init system to manage services in whatever capacity needed.</span></span>

   <span data-ttu-id="d6bea-117">Планировщик кластера преследует несколько задач: эффективное использование ресурсов кластера, работа с заданными пользователем ограничениями размещения, быстрое планирование приложений, чтобы они не оставались в состоянии ожидания, а также обеспечение определенной степени равнодоступности и устойчивости к ошибкам постоянной доступности.</span><span class="sxs-lookup"><span data-stu-id="d6bea-117">A cluster scheduler has multiple goals: using the cluster's resources efficiently, working with user-supplied placement constraints, scheduling applications rapidly to not leave them in a pending state, having a degree of "fairness," being robust to errors, and always be available.</span></span>

- <span data-ttu-id="d6bea-118">**Оркестраторы**.</span><span class="sxs-lookup"><span data-stu-id="d6bea-118">**Orchestrators**.</span></span><span data-ttu-id="d6bea-119"> Платформы расширяют возможности управления жизненным циклом до сложных многоконтейнерных рабочих нагрузок, развернутых в кластере узлов.</span><span class="sxs-lookup"><span data-stu-id="d6bea-119"> Platforms extend life-cycle management capabilities to complex, multi-container workloads deployed on a cluster of hosts.</span></span> <span data-ttu-id="d6bea-120">Используя абстрагирование инфраструктуры узлов, инструменты оркестрации позволяют пользователям обрабатывать весь кластер как единый целевой объект развертывания.</span><span class="sxs-lookup"><span data-stu-id="d6bea-120">By abstracting the host infrastructure, orchestration tools give users a way to treat the entire cluster as a single deployment target.</span></span>

   <span data-ttu-id="d6bea-121">Процесс оркестрации включает в себя средства и платформы, позволяющие автоматизировать все аспекты управления приложениями — первоначальное размещение или развертывания для каждого контейнера, перемещение контейнеров на разные узлы в зависимости от состояния или работоспособности узла, а также управление версиями, развертывание обновлений и функции мониторинга работоспособности, которые поддерживают масштабирование и отработку отказа, и многое другое.</span><span class="sxs-lookup"><span data-stu-id="d6bea-121">The process of orchestration involves tooling and a platform that can automate all aspects of application management from initial placement or deployment per container; moving containers to different hosts depending on its host's health or performance; versioning and rolling updates and health monitoring functions that support scaling and failover; and many more.</span></span>

   <span data-ttu-id="d6bea-122">Оркестрация — это общий термин, который охватывает планирование контейнеров, управление кластерами и, возможно, подготовку дополнительных узлов.</span><span class="sxs-lookup"><span data-stu-id="d6bea-122">Orchestration is a broad term that refers to container scheduling, cluster management, and possibly the provisioning of additional hosts.</span></span>

<span data-ttu-id="d6bea-123">Возможности, предоставляемые оркестраторами и планировщиками, сложно разработать и создать с нуля, поэтому обычно рекомендуется использовать решения оркестрации, предлагаемые поставщиками.</span><span class="sxs-lookup"><span data-stu-id="d6bea-123">The capabilities provided by orchestrators and schedulers are complex to develop and create from scratch, therefore you usually would want to use orchestration solutions offered by vendors.</span></span>
