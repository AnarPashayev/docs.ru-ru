<span data-ttu-id="321b2-130">gRPC использует другую технологию с открытым исходным кодом, которая называется [буфером протоколов](https://developers.google.com/protocol-buffers/docs/overview) или сообщениями protobuf для отправки и получения данных.</span><span class="sxs-lookup"><span data-stu-id="321b2-130">gRPC embraces another open-source technology called [Protocol Buffers](https://developers.google.com/protocol-buffers/docs/overview) or Protobuf messages to send and receive data.</span></span> <span data-ttu-id="321b2-131">Как и в случае с [контрактом данных WCF](https://docs.microsoft.com/dotnet/framework/wcf/feature-details/using-data-contracts), protobuf сериализует структурированные данные для систем для чтения и записи.</span><span class="sxs-lookup"><span data-stu-id="321b2-131">Similar to a [WCF Data Contract](https://docs.microsoft.com/dotnet/framework/wcf/feature-details/using-data-contracts), Protobuf serializes structured data for systems to read and write.</span></span> <span data-ttu-id="321b2-132">Это сокращает затраты, которые могут быть доступны для удобочитаемых форматов, таких как XML или JSON.</span><span class="sxs-lookup"><span data-stu-id="321b2-132">It reduces the overhead that human-readable formats like XML or JSON incur.</span></span>
