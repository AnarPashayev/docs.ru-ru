# <a name="protobuf-messages"></a><span data-ttu-id="e5542-103">Сообщения Protobuf</span><span class="sxs-lookup"><span data-stu-id="e5542-103">Protobuf messages</span></span>

<span data-ttu-id="e5542-104">В этом разделе описано, как объявить сообщения protobuf в файлах `.proto`, объясняются фундаментальные понятия чисел и типов полей, а также C# рассматривается код, создаваемый компилятором `protoc`.</span><span class="sxs-lookup"><span data-stu-id="e5542-104">This section covers how to declare Protobuf messages in `.proto` files, explains the fundamental concepts of field numbers and types, and looks at the C# code that is generated by the `protoc` compiler.</span></span> <span data-ttu-id="e5542-105">Оставшаяся часть главы посвящена более подробному рассмотрению того, как различные типы данных представлены в protobuf.</span><span class="sxs-lookup"><span data-stu-id="e5542-105">The rest of the chapter will look in more detail at how different types of data are represented in Protobuf.</span></span>

## <a name="declaring-a-message"></a><span data-ttu-id="e5542-106">Объявление сообщения</span><span class="sxs-lookup"><span data-stu-id="e5542-106">Declaring a message</span></span>

<span data-ttu-id="e5542-107">В WCF класс `Stock` для торгового рынка фондовой биржи может быть определен как в следующем примере:</span><span class="sxs-lookup"><span data-stu-id="e5542-107">In WCF, a `Stock` class for a stock market trading application might be defined like the following example:</span></span>

```csharp
namespace TraderSys
{
    [DataContract]
    public class Stock
    {
        [DataMember]
        public int Id { get; set;}
        [DataMember]
        public string Symbol { get; set;}
        [DataMember]
        public string DisplayName { get; set;}
        [DataMember]
        public int MarketId { get; set; }
    }
}
```

<span data-ttu-id="e5542-108">Чтобы реализовать эквивалентный класс в protobuf, он должен быть объявлен в файле `.proto`.</span><span class="sxs-lookup"><span data-stu-id="e5542-108">To implement the equivalent class in Protobuf, it must be declared in the `.proto` file.</span></span> <span data-ttu-id="e5542-109">Компилятор `protoc` создаст класс .NET как часть процесса сборки.</span><span class="sxs-lookup"><span data-stu-id="e5542-109">The `protoc` compiler will then generate the .NET class as part of the build process.</span></span>

```protobuf
syntax "proto3";

option csharp_namespace = "TraderSys";

message Stock {

    int32 id = 1;
    string symbol = 2;
    string display_name = 3;
    int32 market_id = 4;

}  
```

<span data-ttu-id="e5542-110">В первой строке объявляется используемая версия синтаксиса.</span><span class="sxs-lookup"><span data-stu-id="e5542-110">The first line declares the syntax version being used.</span></span> <span data-ttu-id="e5542-111">Версия 3 языка была выпущена в 2016 и является рекомендуемой версией для gRPC Services.</span><span class="sxs-lookup"><span data-stu-id="e5542-111">Version 3 of the language was released in 2016 and is the recommended version for gRPC services.</span></span>

<span data-ttu-id="e5542-112">В `option csharp_namespace` строке указывается пространство имен, используемое для создаваемых C# типов.</span><span class="sxs-lookup"><span data-stu-id="e5542-112">The `option csharp_namespace` line specifies the namespace to be used for the generated C# types.</span></span> <span data-ttu-id="e5542-113">Этот параметр будет проигнорирован при компиляции файла `.proto` для других языков.</span><span class="sxs-lookup"><span data-stu-id="e5542-113">This option will be ignored when the `.proto` file is compiled for other languages.</span></span> <span data-ttu-id="e5542-114">Часто protobuf файлы содержат параметры, зависящие от языка, для нескольких языков.</span><span class="sxs-lookup"><span data-stu-id="e5542-114">It is common for Protobuf files to contain language-specific options for several languages.</span></span>

<span data-ttu-id="e5542-115">В `Stock` определении сообщения указываются четыре поля, каждый из которых имеет тип, имя и номер поля.</span><span class="sxs-lookup"><span data-stu-id="e5542-115">The `Stock` message definition specifies four fields, each with a type, a name, and a field number.</span></span>

## <a name="field-numbers"></a><span data-ttu-id="e5542-116">Номера полей</span><span class="sxs-lookup"><span data-stu-id="e5542-116">Field numbers</span></span>

<span data-ttu-id="e5542-117">Номера полей являются важной частью protobuf.</span><span class="sxs-lookup"><span data-stu-id="e5542-117">Field numbers are an important part of Protobuf.</span></span> <span data-ttu-id="e5542-118">Они используются для указания полей в зашифрованных двоичных данных, что означает, что они не могут быть изменены с версии до версии вашей службы.</span><span class="sxs-lookup"><span data-stu-id="e5542-118">They're used to identify fields in the binary encoded data, which means they can't change from version to version of your service.</span></span> <span data-ttu-id="e5542-119">Преимущество заключается в том, что возможна обратная и прямая совместимость.</span><span class="sxs-lookup"><span data-stu-id="e5542-119">The advantage is that backward and forward compatibility is possible.</span></span> <span data-ttu-id="e5542-120">Клиенты и службы будут просто игнорировать номера полей, о которых они не известны, при условии, что вероятность отсутствующих значений будет обработана.</span><span class="sxs-lookup"><span data-stu-id="e5542-120">Clients and services will simply ignore field numbers they don't know about, as long as the possibility of missing values is handled.</span></span>

<span data-ttu-id="e5542-121">В двоичном формате номер поля объединяется с идентификатором типа.</span><span class="sxs-lookup"><span data-stu-id="e5542-121">In the binary format, the field number is combined with a type identifier.</span></span> <span data-ttu-id="e5542-122">Номера полей от 1 до 15 могут быть закодированы с помощью одного байта. числа от 16 до 2047 принимают 2 байта.</span><span class="sxs-lookup"><span data-stu-id="e5542-122">Field numbers from 1 to 15 can be encoded with their type as a single byte; numbers from 16 to 2047 take 2 bytes.</span></span> <span data-ttu-id="e5542-123">Если в сообщении по какой-либо причине требуется более 2047 полей для сообщения, можно увеличить значение.</span><span class="sxs-lookup"><span data-stu-id="e5542-123">You can go higher if you need more than 2047 fields on a message for any reason.</span></span> <span data-ttu-id="e5542-124">Однобайтовые идентификаторы полей с номерами от 1 до 15 обеспечивают лучшую производительность, поэтому их следует использовать для самых базовых и часто используемых полей.</span><span class="sxs-lookup"><span data-stu-id="e5542-124">The single byte identifiers for field numbers 1 to 15 offer better performance, so you should use them for the most basic, frequently used fields.</span></span>

## <a name="the-generated-code"></a><span data-ttu-id="e5542-130">Сформированный код</span><span class="sxs-lookup"><span data-stu-id="e5542-130">The generated code</span></span>

<span data-ttu-id="e5542-131">При сборке приложения protobuf создает классы для каждого сообщения, сопоставляя собственные типы с C# типами.</span><span class="sxs-lookup"><span data-stu-id="e5542-131">When you build your application, Protobuf creates classes for each of your messages, mapping its native types to C# types.</span></span> <span data-ttu-id="e5542-132">Созданный тип `Stock` будет иметь следующую сигнатуру:</span><span class="sxs-lookup"><span data-stu-id="e5542-132">The generated `Stock` type would have the following signature:</span></span>

```csharp
public class Stock
{
    public int Id { get; set; }
    public string Symbol { get; set; }
    public string DisplayName { get; set; }
    public int MarketId { get; set; }
}
```

<span data-ttu-id="e5542-133">Фактически созданный код гораздо более сложен, поскольку каждый класс содержит весь код, необходимый для сериализации и десериализации в двоичном формате.</span><span class="sxs-lookup"><span data-stu-id="e5542-133">The actual code that is generated is far more complicated than this, because each class contains all the code necessary to serialize and deserialize itself to the binary wire format.</span></span>
