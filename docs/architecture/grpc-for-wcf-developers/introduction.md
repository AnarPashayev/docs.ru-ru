## <a name="microservices"></a><span data-ttu-id="ba754-137">Микрослужбы</span><span class="sxs-lookup"><span data-stu-id="ba754-137">Microservices</span></span>

<span data-ttu-id="ba754-138">В архитектурах микрослужб крупные приложения создаются в виде коллекции небольших модульных служб.</span><span class="sxs-lookup"><span data-stu-id="ba754-138">In microservice architectures, large applications are built as a collection of smaller modular services.</span></span> <span data-ttu-id="ba754-139">Каждый компонент выполняет определенную задачу или процесс, и компоненты предназначены для работы интероперабли, но могут быть изолированы по мере необходимости.</span><span class="sxs-lookup"><span data-stu-id="ba754-139">Each component does a specific task or process, and components are designed to work interoperably but can be isolated as necessary.</span></span>

<span data-ttu-id="ba754-140">К микрослужбам относятся следующие преимущества.</span><span class="sxs-lookup"><span data-stu-id="ba754-140">Advantages to microservices include:</span></span>

- <span data-ttu-id="ba754-141">Изменения и обновления могут обрабатываться независимо друг от друга.</span><span class="sxs-lookup"><span data-stu-id="ba754-141">Changes and upgrades can be handled independently.</span></span>
- <span data-ttu-id="ba754-142">Обработка ошибок стала более эффективной, так как проблемы можно отслеживать с помощью отдельных служб, которые затем изолированы, перестраиваются, тестируются и развертываются повторно, независимо от других служб.</span><span class="sxs-lookup"><span data-stu-id="ba754-142">Error handling becomes more efficient because problems can be traced to specific services that are then isolated, rebuilt, tested, and redeployed independently of the other services.</span></span>
