# <a name="protocol-buffers"></a>Буферы протоколов

службы gRPC Services отправляют и получают данные в виде сообщений в виде *буфера протокола (protobuf)* , как и контракты данных в Windows Communication Foundation (WCF). Protobuf — это эффективный способ сериализации структурированных данных для компьютеров, предназначенных для чтения и записи, без издержек, которые могут возникнет в удобочитаемых форматах, таких как XML или JSON.

В этой главе описывается работа protobuf и определение собственных сообщений protobuf.

## <a name="how-protobuf-works"></a>Как работает protobuf

Большинство методов сериализации объектов .NET, включая контракты данных WCF, работают с помощью отражения для анализа структуры объектов во время выполнения. В большинстве случаев для большинства библиотек protobuf требуется заранее определить структуру с помощью выделенного языка (*языкового буфера протокола*) в `.proto` файле. Затем компилятор использует этот файл для создания кода для любой из поддерживаемых платформ. Поддерживаемыми платформами являются .NET, Java,C++C/, JavaScript и многие другие. 

Компилятор protobuf, `protoc`, обслуживается Google, хотя доступны альтернативные реализации. Созданный код является эффективным и оптимизирован для быстрой сериализации и десериализации данных.

Формат сети protobuf — это двоичная кодировка. Он использует некоторые полезные приемы, чтобы максимально сокращать число байтов, используемых для представления сообщений. Знание формата двоичного кодирования не обязательно для использования protobuf. Но если вас интересует, вы можете узнать больше о нем на [веб-сайте буферов протоколов](https://developers.google.com/protocol-buffers/docs/encoding).
