# <a name="protocol-buffers"></a>Буферы протоколов

gRPC Services отправляют и получают данные в виде сообщений в виде *буфера протокола (protobuf)* , как и контракты данных WCF. Protobuf — это эффективный способ сериализации структурированных данных для компьютеров, предназначенных для чтения и записи, без издержек, которые могут возникнет в удобочитаемых форматах, таких как XML или JSON.

В этой главе описывается работа protobuf и определение собственных сообщений protobuf.

## <a name="how-protobuf-works"></a>Как работает protobuf

Большинство методов сериализации объектов .NET, включая контракты данных WCF, работают с помощью отражения для анализа структуры объектов во время выполнения. Для большинства библиотек protobuf необходимо заранее определить структуру, используя выделенный язык (*язык буфера протокола*) в файле `.proto`. Этот файл затем используется компилятором для создания кода для любой из поддерживаемых платформ, включая .NET, Java, C/C++, JavaScript и многие другие. Компилятор protobuf, `protoc`, обслуживается Google, хотя доступны альтернативные реализации. Созданный код является эффективным и оптимизирован для быстрой сериализации и десериализации данных.

Сам формат подключения protobuf — это двоичная кодировка, которая использует некоторые полезные приемы для минимального числа байтов, используемых для представления сообщений. Знание формата двоичного кодирования не обязательно для использования protobuf, но если вас интересует, вы можете узнать больше о нем на [веб-сайте буферов протоколов](https://developers.google.com/protocol-buffers/docs/encoding).
