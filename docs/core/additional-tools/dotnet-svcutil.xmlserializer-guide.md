## <a name="getting-started"></a><span data-ttu-id="63a58-111">Начало работы</span><span class="sxs-lookup"><span data-stu-id="63a58-111">Getting started</span></span>

<span data-ttu-id="63a58-112">Чтобы использовать `dotnet-svcutil.xmlserializer` в консольном приложении .NET Core, выполните следующие действия.</span><span class="sxs-lookup"><span data-stu-id="63a58-112">To use `dotnet-svcutil.xmlserializer` in a .NET Core console application:</span></span>

1. <span data-ttu-id="63a58-113">Создайте службу WCF с именем MyWCFService с помощью шаблона по умолчанию "Приложение WCF-службы" в .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="63a58-113">Create a WCF Service named 'MyWCFService' using the default template 'WCF Service Application' in .NET Framework.</span></span> <span data-ttu-id="63a58-114">Добавьте атрибут `[XmlSerializerFormat]` в метод службы.</span><span class="sxs-lookup"><span data-stu-id="63a58-114">Add `[XmlSerializerFormat]` attribute on the service method like the following:</span></span>

   ```csharp

    public interface IService1
    {
        [XmlSerializerFormat]
        [OperationContract(Action = "http://tempuri.org/IService1/GetData", ReplyAction = "http://tempuri.org/IService1/GetDataResponse")]
        string GetData(int value);
    }
    ```

2. <span data-ttu-id="63a58-115">Для этого создайте консольное приложение .NET Core как клиентское приложение WCF, ориентированное на .NET Core 2.1 или более поздних версий.</span><span class="sxs-lookup"><span data-stu-id="63a58-115">Create a .NET Core console application as WCF client application that targets at .NET Core 2.1 or later versions.</span></span> <span data-ttu-id="63a58-116">Например, с помощью приведенной ниже команды создайте приложение с именем MyWCFClient.</span><span class="sxs-lookup"><span data-stu-id="63a58-116">For example, create an app named 'MyWCFClient' with the following command:</span></span>

    ```dotnetcli
    dotnet new console --name MyWCFClient
    ```

    <span data-ttu-id="63a58-117">Чтобы убедиться, что проект предназначен для .NET Core 2.1 или более поздней версии, в файле проекта проверьте XML-элемент `TargetFramework`.</span><span class="sxs-lookup"><span data-stu-id="63a58-117">To ensure your project is targeting .NET Core 2.1 or later, inspect the `TargetFramework` XML element in your project file:</span></span>

    ```xml
    <TargetFramework>netcoreapp2.1</TargetFramework>
    ```

3. <span data-ttu-id="63a58-118">Добавьте ссылку на пакет в `System.ServiceModel.Http`, выполнив приведенную ниже команду.</span><span class="sxs-lookup"><span data-stu-id="63a58-118">Add a package reference to `System.ServiceModel.Http` by running the following command:</span></span>


4. <span data-ttu-id="63a58-119">Добавьте код клиента WCF:</span><span class="sxs-lookup"><span data-stu-id="63a58-119">Add the WCF Client code:</span></span>

    ```csharp
    using System.ServiceModel;

        class Program
        {
            static void Main(string[] args)
            {
                var myBinding = new BasicHttpBinding();
                var myEndpoint = new EndpointAddress("http://localhost:2561/Service1.svc"); //Fill your service url here
                var myChannelFactory = new ChannelFactory<IService1>(myBinding, myEndpoint);
                IService1 client = myChannelFactory.CreateChannel();
                string s = client.GetData(1);
                ((ICommunicationObject)client).Close();
            }
        }

    [ServiceContract]
    public interface IService1
    {
        [XmlSerializerFormat]
        [OperationContract(Action = "http://tempuri.org/IService1/GetData", ReplyAction = "http://tempuri.org/IService1/GetDataResponse")]
        string GetData(int value);
    }
    ```

5. <span data-ttu-id="63a58-120">Добавьте ссылку на пакет `dotnet-svcutil.xmlserializer`, выполнив приведенную ниже команду.</span><span class="sxs-lookup"><span data-stu-id="63a58-120">Add a reference to the `dotnet-svcutil.xmlserializer` package by running the following command:</span></span>
  
    ```dotnetcli
    dotnet add package dotnet-svcutil.xmlserializer
    ```

    <span data-ttu-id="63a58-121">При выполнении команды следует добавить запись в файл проекта, как показано ниже.</span><span class="sxs-lookup"><span data-stu-id="63a58-121">Running the command should add an entry to your project file similar to this:</span></span>

    ```xml
    </ItemGroup>
    ```

6. <span data-ttu-id="63a58-122">Выполните сборку приложения с помощью команды `dotnet build`.</span><span class="sxs-lookup"><span data-stu-id="63a58-122">Build the application by running `dotnet build`.</span></span> <span data-ttu-id="63a58-123">Если все пройдет успешно, в папке выходных данных генерируется сборка с именем *MyWCFClient.XmlSerializers.dll*.</span><span class="sxs-lookup"><span data-stu-id="63a58-123">If everything succeeds, an assembly named *MyWCFClient.XmlSerializers.dll* is generated in the output folder.</span></span> <span data-ttu-id="63a58-124">В выходных данных сборки будут отображаться предупреждения, если создание сборки средством завершилось сбоем.</span><span class="sxs-lookup"><span data-stu-id="63a58-124">If the tool failed to generate the assembly, you'll see warnings in the build output.</span></span>
