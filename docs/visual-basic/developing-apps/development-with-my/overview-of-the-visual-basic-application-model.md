# <a name="overview-of-the-visual-basic-application-model"></a><span data-ttu-id="3787c-102">Обзор модели приложения в Visual Basic</span><span class="sxs-lookup"><span data-stu-id="3787c-102">Overview of the Visual Basic Application Model</span></span>

<span data-ttu-id="3787c-103">Visual Basic предоставляет четко определенную модель для управления поведением приложений Windows Forms: модель приложения Visual Basic.</span><span class="sxs-lookup"><span data-stu-id="3787c-103">Visual Basic provides a well-defined model for controlling the behavior of Windows Forms applications: the Visual Basic Application model.</span></span> <span data-ttu-id="3787c-104">Эта модель включает события для обработки запуска и завершения работы приложения, а также события для перехвата необработанных исключений.</span><span class="sxs-lookup"><span data-stu-id="3787c-104">This model includes events for handling the application's startup and shutdown, as well as events for catching unhandled exceptions.</span></span> <span data-ttu-id="3787c-105">Она также обеспечивает поддержку разработки приложений с одним экземпляром.</span><span class="sxs-lookup"><span data-stu-id="3787c-105">It also provides support for developing single-instance applications.</span></span> <span data-ttu-id="3787c-106">Модель приложения является расширяемой, поэтому разработчики, которым требуется больший контроль, могут настраивать переопределяемые методы.</span><span class="sxs-lookup"><span data-stu-id="3787c-106">The application model is extensible, so developers that need more control can customize its overridable methods.</span></span>

 <span data-ttu-id="3787c-108">Типичное приложение должно выполнять задачи при запуске и завершении работы.</span><span class="sxs-lookup"><span data-stu-id="3787c-108">A typical application needs to perform tasks when it starts up and shuts down.</span></span> <span data-ttu-id="3787c-109">Например, при запуске приложение может отобразить экран-заставку, установить подключения к базе данных, загрузить сохраненное состояние и т. д.</span><span class="sxs-lookup"><span data-stu-id="3787c-109">For example, when it starts up, the application can display a splash screen, make database connections, load a saved state, and so on.</span></span> <span data-ttu-id="3787c-110">Когда приложение завершает работу, оно может закрыть подключения к базе данных, сохранить текущее состояние и т. д.</span><span class="sxs-lookup"><span data-stu-id="3787c-110">When the application shuts down, it can close database connections, save the current state, and so on.</span></span> <span data-ttu-id="3787c-111">Кроме того, приложение может выполнять конкретный код при неожиданном завершении работы, например во время необработанного исключения.</span><span class="sxs-lookup"><span data-stu-id="3787c-111">In addition, the application can execute specific code when the application shuts down unexpectedly, such as during an unhandled exception.</span></span>

 <span data-ttu-id="3787c-117">Приложение с одним экземпляром запускается и проверяет, является ли оно первым или последующим экземпляром приложения:</span><span class="sxs-lookup"><span data-stu-id="3787c-117">A single-instance application starts and checks whether it is the first instance or a subsequent instance of the application:</span></span>

 <span data-ttu-id="3787c-124">Обрабатывая событие `StartupNextInstance`, можно управлять поведением приложения с одним экземпляром.</span><span class="sxs-lookup"><span data-stu-id="3787c-124">By handling the `StartupNextInstance` event, you can control how your single-instance application behaves.</span></span> <span data-ttu-id="3787c-125">Например, Microsoft Outlook обычно выполняется как приложение с одним экземпляром.
