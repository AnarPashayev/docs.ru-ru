## <a name="designing-class-libraries-for-my"></a><span data-ttu-id="ec673-165">Разработка библиотек классов для `My`</span><span class="sxs-lookup"><span data-stu-id="ec673-165">Designing class libraries for `My`</span></span>

<span data-ttu-id="ec673-166">Как и в случае с большинством объектных моделей, некоторые шаблоны проектирования хорошо работают в пространстве имен `My`, а другие — нет.</span><span class="sxs-lookup"><span data-stu-id="ec673-166">As is the case with most object models, some design patterns work well in the `My` namespace and others do not.</span></span> <span data-ttu-id="ec673-167">При проектировании расширения для пространства имен `My` учитывайте следующие принципы.</span><span class="sxs-lookup"><span data-stu-id="ec673-167">When designing an extension to the `My` namespace, consider the following principles:</span></span>

- <span data-ttu-id="ec673-168">**Методы без отслеживания состояния.**</span><span class="sxs-lookup"><span data-stu-id="ec673-168">**Stateless methods.**</span></span> <span data-ttu-id="ec673-169">Методы в пространстве имен `My` должны обеспечивать полное решение для конкретной задачи.</span><span class="sxs-lookup"><span data-stu-id="ec673-169">Methods in the `My` namespace should provide a complete solution to a specific task.</span></span> <span data-ttu-id="ec673-170">Убедитесь, что значения параметров, передаваемые в метод, предоставляют все входные данные, необходимые для выполнения конкретной задачи.</span><span class="sxs-lookup"><span data-stu-id="ec673-170">Ensure that the parameter values that are passed to the method provide all the input required to complete the particular task.</span></span> <span data-ttu-id="ec673-171">Избегайте создания методов, основанных на предыдущем состоянии, например открытых соединений с ресурсами.</span><span class="sxs-lookup"><span data-stu-id="ec673-171">Avoid creating methods that rely on prior state, such as open connections to resources.</span></span>
- <span data-ttu-id="ec673-172">**Глобальные экземпляры.**</span><span class="sxs-lookup"><span data-stu-id="ec673-172">**Global instances.**</span></span> <span data-ttu-id="ec673-173">Единственное состояние, поддерживаемое в `My` пространстве имен, является глобальным по отношению к проекту.</span><span class="sxs-lookup"><span data-stu-id="ec673-173">The only state that is maintained in the `My` namespace is global to the project.</span></span> <span data-ttu-id="ec673-174">Например, `My.Application.Info` инкапсулирует состояние, которое является общим для всего приложения.</span><span class="sxs-lookup"><span data-stu-id="ec673-174">For example, `My.Application.Info` encapsulates state that is shared throughout the application.</span></span>

- <span data-ttu-id="ec673-178">**Фабричные методы.**</span><span class="sxs-lookup"><span data-stu-id="ec673-178">**Factory methods.**</span></span> <span data-ttu-id="ec673-179">Создание экземпляров некоторых типов обязательно затрудняется.</span><span class="sxs-lookup"><span data-stu-id="ec673-179">Some types are necessarily difficult to instantiate.</span></span> <span data-ttu-id="ec673-180">Предоставление фабричных методов в качестве расширений пространства имен `My` позволяет более легко обнаруживать и использовать типы, попадающие в эту категорию.</span><span class="sxs-lookup"><span data-stu-id="ec673-180">Providing factory methods as extensions to the `My` namespace enables you to more easily discover and consume types that fall into this category.</span></span> <span data-ttu-id="ec673-181">Пример метода фабрики, который хорошо работает, `My.Computer.FileSystem.OpenTextFileReader`.</span><span class="sxs-lookup"><span data-stu-id="ec673-181">An example of a factory method that works well is `My.Computer.FileSystem.OpenTextFileReader`.</span></span> <span data-ttu-id="ec673-182">В .NET Framework доступно несколько типов потоков.</span><span class="sxs-lookup"><span data-stu-id="ec673-182">There are several stream types available in the .NET Framework.</span></span> <span data-ttu-id="ec673-183">Указывая текстовые файлы в частности, `OpenTextFileReader` помогает пользователю понять, какой поток использовать.</span><span class="sxs-lookup"><span data-stu-id="ec673-183">By specifying text files specifically, the `OpenTextFileReader` helps the user understand which stream to use.</span></span>

<span data-ttu-id="ec673-184">Эти рекомендации не исключают общие принципы разработки библиотек классов.</span><span class="sxs-lookup"><span data-stu-id="ec673-184">These guidelines do not preclude general design principles for class libraries.</span></span> <span data-ttu-id="ec673-185">Вместо этого они являются рекомендациями, оптимизированными для разработчиков, использующих Visual Basic и пространство имен `My`.</span><span class="sxs-lookup"><span data-stu-id="ec673-185">Rather, they are recommendations that are optimized for developers who are using Visual Basic and the `My` namespace.</span></span> <span data-ttu-id="ec673-186">Общие принципы создания библиотек классов см. в разделе [рекомендации по проектированию платформы](../../../standard/design-guidelines/index.md).</span><span class="sxs-lookup"><span data-stu-id="ec673-186">For general design principles for creating class libraries, see [Framework Design Guidelines](../../../standard/design-guidelines/index.md).</span></span>
