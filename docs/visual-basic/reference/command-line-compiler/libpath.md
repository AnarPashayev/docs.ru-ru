# <a name="-libpath"></a>-libpath
Задает расположение сборок, на которые указывают ссылки.  
  
## <a name="syntax"></a>Синтаксис  
  
```console  
-libpath:dirList  
```  
  
## <a name="arguments"></a>Arguments  
  
|Термин|Определение|  
|---|---|  
|`dirList`|Обязательное Разделенный точками с запятой список каталогов, в которых компилятор должен искать, если сборка, на которую указывает ссылка, не найдена ни в текущем рабочем каталоге (каталоге, из которого вызывается компилятор), ни в системном каталоге среды CLR. Если имя каталога содержит пробел, заключите его в кавычки ("").|  
  
## <a name="remarks"></a>Заметки  
 Параметр `-libpath` задает расположение сборок, на которые ссылается параметр [-Reference](../../../visual-basic/reference/command-line-compiler/reference.md) .  
  
 Компилятор выполняет поиск связанных сборок, для которых не указано полное имя, в следующем порядке:  
  
1. Текущая рабочая папка. Это папка, из которой был вызван компилятор.  
  
2. Системный каталог среды CLR.  
  
3. Каталоги, заданные `-libpath`.  
  
4. Каталоги, указанные переменной среды LIB.  
  
 Параметр `-libpath` является аддитивным; При указании более одного раза к любым предшествующим значениям добавляется.  
  
 Используйте `-reference`, чтобы указать ссылку на сборку.  
  
|Установка параметра-LIBPATH в интегрированной среде разработки Visual Studio|  
|---|  
|1. Выберите проект в **Обозреватель решений**. В меню **Проект** выберите пункт **Свойства**. <br />2. Перейдите на вкладку **ссылки** .<br />3. Нажмите кнопку **пути для ссылок...** .<br />4. в диалоговом окне **пути для ссылок** введите имя каталога в поле **Папка:** .<br />5. Нажмите кнопку **Добавить папку**.|  
  
## <a name="example"></a>Пример  
 Следующий код компилирует `T2.vb`, чтобы создать exe файл. Компилятор выполняет поиск в рабочем каталоге, в корневом каталоге диска C: и в каталоге новых сборок диска C: для ссылок на сборки.  
  
```console  
vbc -libpath:c:\;"c:\New Assemblies" -reference:t2.dll t2.vb  
```
