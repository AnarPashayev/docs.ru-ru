## <a name="what-is-pure-functional-transformation"></a><span data-ttu-id="45b39-107">Что такое чисто функциональное преобразование?</span><span class="sxs-lookup"><span data-stu-id="45b39-107">What Is Pure Functional Transformation?</span></span>  
 <span data-ttu-id="45b39-108">При *чисто функциональном преобразовании* набор функций, называемых *чистыми функциями*, определяет способ преобразования набора структурированных данных из одной формы в другую.</span><span class="sxs-lookup"><span data-stu-id="45b39-108">In *pure functional transformation*, a set of functions, called *pure functions*, define how to transform a set of structured data from its original form into another form.</span></span> <span data-ttu-id="45b39-109">Слово "чистые" указывает на то, что функции *допускают композицию*, то есть отвечают следующим требованиям:</span><span class="sxs-lookup"><span data-stu-id="45b39-109">The word "pure" indicates that the functions are *composable*, which requires that they are:</span></span>  
  
- <span data-ttu-id="45b39-110">*Замкнутые*, то есть они должны свободно упорядочиваться и переупорядочиваться без вовлечения остальных частей программы и взаимных зависимостей от них.</span><span class="sxs-lookup"><span data-stu-id="45b39-110">*Self-contained*, so that they can be freely ordered and rearranged without entanglement or interdependencies with the rest of the program.</span></span> <span data-ttu-id="45b39-111">Чистые преобразования не имеют представления о своей среде и не влияют на нее.</span><span class="sxs-lookup"><span data-stu-id="45b39-111">Pure transformations have no knowledge of or effect upon their environment.</span></span> <span data-ttu-id="45b39-112">Иными словами, использованные при преобразовании функции не имеют *побочных эффектов*.</span><span class="sxs-lookup"><span data-stu-id="45b39-112">That is, the functions used in the transformation have no *side effects*.</span></span>  
  
- <span data-ttu-id="45b39-113">*Без учета состояния*, то есть исполнение одной и той же функции или определенного набора функций с одинаковыми входными данными будет всегда приводить к одинаковым выходным данным.</span><span class="sxs-lookup"><span data-stu-id="45b39-113">*Stateless*, so that executing the same function or specific set of functions on the same input will always result in the same output.</span></span> <span data-ttu-id="45b39-114">Чистые преобразования не знают о предыдущих использованиях.</span><span class="sxs-lookup"><span data-stu-id="45b39-114">Pure transformations have no memory of their prior use.</span></span>  
  
> [!IMPORTANT]
> <span data-ttu-id="45b39-115">До конца данного учебника термин «чистая возможность» используется в основном для указания подхода к программированию, а не специальной характеристики языка.</span><span class="sxs-lookup"><span data-stu-id="45b39-115">In the rest of this tutorial, the term "pure function" is used in a general sense to indicate a programming approach, and not a specific language feature.</span></span>  
>   
>  <span data-ttu-id="45b39-116">Обратите внимание, что чистые функции должны реализовываться как функции в Visual Basic.</span><span class="sxs-lookup"><span data-stu-id="45b39-116">Note that pure functions must be implemented as functions in Visual Basic.</span></span>  
>   
>  <span data-ttu-id="45b39-117">Также не надо путать чистые функции с чисто виртуальными методами в C++.</span><span class="sxs-lookup"><span data-stu-id="45b39-117">Also, you should not confuse pure functions with pure virtual methods in C++.</span></span> <span data-ttu-id="45b39-118">Второй случай показывает, что содержащийся класс абстрактный и текст метода не указан.</span><span class="sxs-lookup"><span data-stu-id="45b39-118">The latter indicates that the containing class is abstract and that no method body is supplied.</span></span>

 <span data-ttu-id="45b39-120">Термин *функциональное программирование* обозначает такой подход к программированию, при котором непосредственно поддерживаются чисто функциональные преобразования.</span><span class="sxs-lookup"><span data-stu-id="45b39-120">*Functional programming* is a programming approach that directly supports pure functional transformation.</span></span>  
  
 <span data-ttu-id="45b39-121">Исторически языки функционального программирования общего назначения, такие как ML, Scheme, Haskell и F#, представляли интерес в первую очередь для научного сообщества.</span><span class="sxs-lookup"><span data-stu-id="45b39-121">Historically, general-purpose functional programming languages, such as ML, Scheme, Haskell, and F#, have been primarily of interest to the academic community.</span></span> <span data-ttu-id="45b39-122">Хотя всегда можно написать чисто функциональные преобразования в Visual Basic, сложность этого не стала привлекательным вариантом для большинства программистов.</span><span class="sxs-lookup"><span data-stu-id="45b39-122">Although it has always been possible to write pure functional transformations in Visual Basic, the difficulty of doing so has not made it an attractive option to most programmers.</span></span> <span data-ttu-id="45b39-123">Однако в более поздних версиях Visual Basic новые языковые конструкции, такие как лямбда-выражения и определение типов, делают функциональное программирование гораздо проще и эффективнее.</span><span class="sxs-lookup"><span data-stu-id="45b39-123">With later versions of Visual Basic, however, new language constructs such as lambda expressions and type inference make it functional programming much easier and more productive.</span></span>  
  
 <span data-ttu-id="45b39-124">Дополнительные сведения о функциональном программировании см. в разделе [Сравнение функционального и Императивное программирование (Visual Basic](../../../../visual-basic/programming-guide/concepts/linq/functional-programming-vs-imperative-programming.md)).</span><span class="sxs-lookup"><span data-stu-id="45b39-124">For more information about functional programming, see [Functional Programming vs. Imperative Programming (Visual Basic)](../../../../visual-basic/programming-guide/concepts/linq/functional-programming-vs-imperative-programming.md).</span></span>
