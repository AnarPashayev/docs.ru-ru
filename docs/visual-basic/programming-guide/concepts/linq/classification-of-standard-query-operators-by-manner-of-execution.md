## <a name="manners-of-execution"></a><span data-ttu-id="e1eec-108">Способ выполнения</span><span class="sxs-lookup"><span data-stu-id="e1eec-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="e1eec-109">Немедленный</span><span class="sxs-lookup"><span data-stu-id="e1eec-109">Immediate</span></span>  
 <span data-ttu-id="e1eec-110">Немедленное выполнение означает, что источник данных считывается и операция выполняется в той точке кода, где объявлен запрос.</span><span class="sxs-lookup"><span data-stu-id="e1eec-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="e1eec-111">Все стандартные операторы запросов, возвращающие один, неперечислимый результат, выполняются немедленно.</span><span class="sxs-lookup"><span data-stu-id="e1eec-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="e1eec-112">Отложено</span><span class="sxs-lookup"><span data-stu-id="e1eec-112">Deferred</span></span>  
 <span data-ttu-id="e1eec-113">Отложенное выполнение означает, что операция не выполняется в той точке кода, где объявлен запрос.</span><span class="sxs-lookup"><span data-stu-id="e1eec-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="e1eec-114">Она выполняется только после перечисления переменной запроса, например с помощью оператора `For Each`.</span><span class="sxs-lookup"><span data-stu-id="e1eec-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="e1eec-115">Это означает, что результаты выполнения запроса зависят от содержимого источника данных при выполнении запроса, а не при его определении.</span><span class="sxs-lookup"><span data-stu-id="e1eec-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="e1eec-116">Если переменная запроса перечисляется несколько раз, результаты могут каждый раз отличаться.</span><span class="sxs-lookup"><span data-stu-id="e1eec-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="e1eec-117">Практически все стандартные операторы запроса, которые возвращают значения типа <xref:System.Collections.Generic.IEnumerable%601> или <xref:System.Linq.IOrderedEnumerable%601>, выполняются отложенным способом.</span><span class="sxs-lookup"><span data-stu-id="e1eec-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="e1eec-118">Операторы запросов, использующие отложенное выполнение, можно дополнительно разделить на потоковые и непотоковые.</span><span class="sxs-lookup"><span data-stu-id="e1eec-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="e1eec-119">Потоковые операторы</span><span class="sxs-lookup"><span data-stu-id="e1eec-119">Streaming</span></span>  
 <span data-ttu-id="e1eec-120">Потоковые операторы не считывают все исходные данные до создания элементов.</span><span class="sxs-lookup"><span data-stu-id="e1eec-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="e1eec-121">Во время выполнения потоковый оператор выполняет свою операцию с каждым исходным элементом по мере считывания и при необходимости создает элемент.</span><span class="sxs-lookup"><span data-stu-id="e1eec-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="e1eec-122">Потоковый оператор продолжает считывание исходных элементов до того момента, когда можно будет создать итоговый элемент.</span><span class="sxs-lookup"><span data-stu-id="e1eec-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="e1eec-123">Это означает, что для получения одного итогового элемента может быть считано несколько исходных элементов.</span><span class="sxs-lookup"><span data-stu-id="e1eec-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="e1eec-124">Непотоковые</span><span class="sxs-lookup"><span data-stu-id="e1eec-124">Non-Streaming</span></span>  
 <span data-ttu-id="e1eec-125">Непотоковые операторы должны считать все исходные данные до создания итогового элемента.</span><span class="sxs-lookup"><span data-stu-id="e1eec-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="e1eec-126">В эту категорию попадают операции сортировки и группировки.</span><span class="sxs-lookup"><span data-stu-id="e1eec-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="e1eec-127">Во время выполнения непотоковые операторы запросов считывают все исходные данные, помещают их в структуру данных, выполняют операцию и создают итоговые элементы.</span><span class="sxs-lookup"><span data-stu-id="e1eec-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
