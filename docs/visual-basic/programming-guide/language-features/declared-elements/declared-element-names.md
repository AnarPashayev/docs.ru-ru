<span data-ttu-id="82234-103">Каждый объявленный элемент имеет имя, также называемое *идентификатором*, которое используется кодом для ссылки на него.</span><span class="sxs-lookup"><span data-stu-id="82234-103">Every declared element has a name, also called an *identifier*, which is what the code uses to refer to it.</span></span>  
  
## <a name="rules"></a><span data-ttu-id="82234-104">Правила</span><span class="sxs-lookup"><span data-stu-id="82234-104">Rules</span></span>  
 <span data-ttu-id="82234-105">Имя элемента в Visual Basic должно соответствовать следующим правилам.</span><span class="sxs-lookup"><span data-stu-id="82234-105">An element name in Visual Basic must observe the following rules:</span></span>  
  
- <span data-ttu-id="82234-106">Оно должно начинаться с буквы или знака подчеркивания ( `_` ).</span><span class="sxs-lookup"><span data-stu-id="82234-106">It must begin with an alphabetic character or an underscore (`_`).</span></span>  
  
- <span data-ttu-id="82234-107">Оно должно содержать только буквы, цифры и знаки подчеркивания.</span><span class="sxs-lookup"><span data-stu-id="82234-107">It must only contain alphabetic characters, decimal digits, and underscores.</span></span>  
  
- <span data-ttu-id="82234-108">Он должен содержать по крайней мере одну букву или цифру в десятичном формате, если начинается с символа подчеркивания.</span><span class="sxs-lookup"><span data-stu-id="82234-108">It must contain at least one alphabetic character or decimal digit if it begins with an underscore.</span></span>  
  
- <span data-ttu-id="82234-109">Длина не должна превышать 1023 символов.</span><span class="sxs-lookup"><span data-stu-id="82234-109">It must not be more than 1023 characters long.</span></span>  
  
 <span data-ttu-id="82234-110">Ограничение длины в 1023 символов также применяется ко всей строке полного имени, например `outerNamespace.middleNamespace.innerNamespace.thisClass.thisElement` .</span><span class="sxs-lookup"><span data-stu-id="82234-110">The length limit of 1023 characters also applies to the entire string of a fully qualified name, such as `outerNamespace.middleNamespace.innerNamespace.thisClass.thisElement`.</span></span>  
  
 <span data-ttu-id="82234-111">В следующем примере показаны некоторые допустимые имена элементов.</span><span class="sxs-lookup"><span data-stu-id="82234-111">The following example shows some valid element names.</span></span>

 `' Three INVALID element names`  
  
 `_`  
  
 `12ABC`  
  
 `xyz$wv`  
  
> [!CAUTION]
> <span data-ttu-id="82234-114">Имена элементов, начинающиеся с символа подчеркивания ( `_` ), не являются частью [независимость от языка и независимых от языка компонентов](../../../../standard/language-independence-and-language-independent-components.md) (CLS), поэтому CLS-совместимый код не может использовать компонент, определяющий такие имена.</span><span class="sxs-lookup"><span data-stu-id="82234-114">Element names starting with an underscore (`_`) are not part of the [Language Independence and Language-Independent Components](../../../../standard/language-independence-and-language-independent-components.md) (CLS), so CLS-compliant code cannot use a component that defines such names.</span></span> <span data-ttu-id="82234-115">Однако символ подчеркивания в любой другой должности в имени элемента является CLS-совместимым.</span><span class="sxs-lookup"><span data-stu-id="82234-115">However, an underscore in any other position in an element name is CLS-compliant.</span></span>

 <span data-ttu-id="82234-119">С другой стороны, ваше имя не должно быть настолько коротким, что оно не имеет адекватного описания того, что представляет элемент и как код использует его.</span><span class="sxs-lookup"><span data-stu-id="82234-119">On the other hand, your name should not be so short that it does not adequately describe what the element represents and how your code uses it.</span></span> <span data-ttu-id="82234-120">Это важно для удобочитаемости кода.</span><span class="sxs-lookup"><span data-stu-id="82234-120">This is important for the readability of your code.</span></span> <span data-ttu-id="82234-121">Если кто-то другой пытается понять его, или если вы самостоятельно просматриваете его после написания, подходящие имена элементов могут сэкономить на значительном количестве времени.</span><span class="sxs-lookup"><span data-stu-id="82234-121">If somebody else is trying to understand it, or if you yourself are looking at it a long time after you wrote it, suitable element names can save a considerable amount of time.</span></span>

 <span data-ttu-id="82234-123">Как правило, имя элемента не должно совпадать ни с одним из ключевых слов, зарезервированных Visual Basic, таких как `Case` или `Friend` .</span><span class="sxs-lookup"><span data-stu-id="82234-123">Generally, an element name must not match any of the keywords reserved by Visual Basic, such as `Case` or `Friend`.</span></span> <span data-ttu-id="82234-124">Однако можно определить *escape-имя*, заключенное в квадратные скобки ( `[ ]` ).</span><span class="sxs-lookup"><span data-stu-id="82234-124">However, you can define an *escaped name*, which is enclosed by brackets (`[ ]`).</span></span> <span data-ttu-id="82234-125">Экранированное имя может соответствовать любому ключевому слову Visual Basic, так как квадратные скобки удаляют неоднозначность.</span><span class="sxs-lookup"><span data-stu-id="82234-125">An escaped name can match any Visual Basic keyword, since the brackets remove any ambiguity.</span></span> <span data-ttu-id="82234-126">Скобки также используются при ссылке на имя позже в коде.</span><span class="sxs-lookup"><span data-stu-id="82234-126">You also use the brackets when you refer to the name later in your code.</span></span>

## <a name="case-sensitivity-in-names"></a><span data-ttu-id="82234-133">Чувствительность к регистру в именах</span><span class="sxs-lookup"><span data-stu-id="82234-133">Case Sensitivity in Names</span></span>  
 <span data-ttu-id="82234-134">Имена элементов в Visual Basic не учитывают регистр.</span><span class="sxs-lookup"><span data-stu-id="82234-134">Element names in Visual Basic are case-insensitive.</span></span> <span data-ttu-id="82234-135">Это означает, что при сравнении двух имен, которые различаются только регистром букв, компилятор интерпретирует их как одно и то же имя.</span><span class="sxs-lookup"><span data-stu-id="82234-135">This means that when the compiler compares two names that differ in alphabetic case only, it interprets them as the same name.</span></span> <span data-ttu-id="82234-136">Например, он считает, что `ABC` и `abc` являются одним и тем же объявленным элементом.</span><span class="sxs-lookup"><span data-stu-id="82234-136">For example, it considers `ABC` and `abc` to refer to the same declared element.</span></span>  
  
 <span data-ttu-id="82234-137">Однако среда CLR использует привязку с учетом регистра.</span><span class="sxs-lookup"><span data-stu-id="82234-137">However, the common language runtime (CLR) uses case-sensitive binding.</span></span> <span data-ttu-id="82234-138">Таким образом, при создании сборки или DLL и предоставлении к ней доступа другим сборкам, в именах регистр учитываться не будет.</span><span class="sxs-lookup"><span data-stu-id="82234-138">Therefore, when you produce an assembly or a DLL and make it available to other assemblies, your names are no longer case-insensitive.</span></span> <span data-ttu-id="82234-139">Например, если определен класс с элементом `ABC`и другие сборки используют класс с помощью среды CLR, они должны ссылаться на элемент как на `ABC`.</span><span class="sxs-lookup"><span data-stu-id="82234-139">For example, if you define a class with an element called `ABC`, and other assemblies make use of your class through the common language runtime, they must refer to the element as `ABC`.</span></span> <span data-ttu-id="82234-140">Если впоследствии вы перекомпилируете класс и измените имя элемента на `abc` , другие сборки, использующие ваш класс, больше не смогут получить доступ к этому элементу.</span><span class="sxs-lookup"><span data-stu-id="82234-140">If you subsequently recompile your class and change the element's name to `abc`, the other assemblies using your class could no longer access that element.</span></span> <span data-ttu-id="82234-141">Таким образом, при выпуске обновленной версии сборки нельзя изменять регистр имен открытых элементов.</span><span class="sxs-lookup"><span data-stu-id="82234-141">Therefore, when you release an updated version of an assembly, you should not change the alphabetic case of any public elements.</span></span>
