# <a name="references-to-declared-elements-visual-basic"></a><span data-ttu-id="f13a2-102">Ссылки на объявленные элементы (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="f13a2-102">References to Declared Elements (Visual Basic)</span></span>
<span data-ttu-id="f13a2-103">Когда код ссылается на объявленный элемент, компилятор Visual Basic совпадает с именем в ссылке соответствующий объявлению.</span><span class="sxs-lookup"><span data-stu-id="f13a2-103">When your code refers to a declared element, the Visual Basic compiler matches the name in your reference to the appropriate declaration of that name.</span></span> <span data-ttu-id="f13a2-104">Если более чем один элемент объявлен с тем же именем, вы можете контролировать, какие из этих элементов — на них ссылается *удовлетворяющим требованиям* его имя.</span><span class="sxs-lookup"><span data-stu-id="f13a2-104">If more than one element is declared with the same name, you can control which of those elements is to be referenced by *qualifying* its name.</span></span>  
  
 <span data-ttu-id="f13a2-105">Компилятор пытается сопоставить ссылку имени с объявлением с *наиболее узкой области*.</span><span class="sxs-lookup"><span data-stu-id="f13a2-105">The compiler attempts to match a name reference to a name declaration with the *narrowest scope*.</span></span> <span data-ttu-id="f13a2-106">Это означает, что он начинается с кода, делая ссылку и работая через последовательные уровни, содержащего элементы.</span><span class="sxs-lookup"><span data-stu-id="f13a2-106">This means it starts with the code making the reference and works outward through successive levels of containing elements.</span></span>  
  
 <span data-ttu-id="f13a2-107">Пример ссылки на две переменные с тем же именем.</span><span class="sxs-lookup"><span data-stu-id="f13a2-107">The following example shows references to two variables with the same name.</span></span> <span data-ttu-id="f13a2-108">В примере объявляются две переменные, каждая с именем `totalCount`, на разных уровнях области в модуле `container`.</span><span class="sxs-lookup"><span data-stu-id="f13a2-108">The example declares two variables, each named `totalCount`, at different levels of scope in module `container`.</span></span> <span data-ttu-id="f13a2-109">Когда процедура `showCount` отображает `totalCount` без квалификации, компилятор Visual Basic разрешает ссылку на объявление с самой узкой областью, а именно: локальное объявление внутри `showCount`.</span><span class="sxs-lookup"><span data-stu-id="f13a2-109">When the procedure `showCount` displays `totalCount` without qualification, the Visual Basic compiler resolves the reference to the declaration with the narrowest scope, namely the local declaration inside `showCount`.</span></span> <span data-ttu-id="f13a2-110">При уточнении `totalCount` с содержащим модулем `container`, компилятор разрешает ссылку на объявление с более широкой областью.</span><span class="sxs-lookup"><span data-stu-id="f13a2-110">When it qualifies `totalCount` with the containing module `container`, the compiler resolves the reference to the declaration with the broader scope.</span></span>  
  
```vb  
' Assume these two modules are both in the same assembly.  
Module container  
    Public totalCount As Integer = 1  
    Public Sub showCount()  
        Dim totalCount As Integer = 6000  
        ' The following statement displays the local totalCount (6000).  
        MsgBox("Unqualified totalCount is " & CStr(totalCount))  
        ' The following statement displays the module's totalCount (1).  
        MsgBox("container.totalCount is " & CStr(container.totalCount))  
    End Sub  
End Module  
Module callingModule  
    Public Sub displayCount()  
        container.showCount()  
        ' The following statement displays the containing module's totalCount (1).  
        MsgBox("container.totalCount is " & CStr(container.totalCount))  
    End Sub  
End Module  
```
