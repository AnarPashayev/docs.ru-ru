# <a name="white-space-in-xml-literals-visual-basic"></a><span data-ttu-id="62f68-102">Пробелы в XML-литералах (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="62f68-102">White Space in XML Literals (Visual Basic)</span></span>
<span data-ttu-id="62f68-103">Компилятор Visual Basic включает только символы значащих пробелов из XML-литерала, при создании [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] объекта.</span><span class="sxs-lookup"><span data-stu-id="62f68-103">The Visual Basic compiler incorporates only the significant white space characters from an XML literal when it creates a [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] object.</span></span> <span data-ttu-id="62f68-104">Незначащие пробелы не включены.</span><span class="sxs-lookup"><span data-stu-id="62f68-104">The insignificant white space characters are not incorporated.</span></span>  
  
## <a name="significant-and-insignificant-white-space"></a><span data-ttu-id="62f68-105">Значащий и незначащий пробел</span><span class="sxs-lookup"><span data-stu-id="62f68-105">Significant and Insignificant White Space</span></span>  
 <span data-ttu-id="62f68-106">Пробелы в XML-литералах значимы только в трех областях:</span><span class="sxs-lookup"><span data-stu-id="62f68-106">White space characters in XML literals are significant in only three areas:</span></span>  
  
- <span data-ttu-id="62f68-107">Когда они находятся в значении атрибута.</span><span class="sxs-lookup"><span data-stu-id="62f68-107">When they are in an attribute value.</span></span>  
  
- <span data-ttu-id="62f68-108">Когда они являются частью содержимого текстового элемента и текст также содержит другие символы.</span><span class="sxs-lookup"><span data-stu-id="62f68-108">When they are part of an element's text content and the text also contains other characters.</span></span>  
  
- <span data-ttu-id="62f68-109">Когда они находятся во внедренном выражении для текстового содержимого элемента.</span><span class="sxs-lookup"><span data-stu-id="62f68-109">When they are in an embedded expression for an element's text content.</span></span>  
  
 <span data-ttu-id="62f68-110">В противном случае компилятор считает незначащие пробелы и не включает его в [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] объекта для литерала.</span><span class="sxs-lookup"><span data-stu-id="62f68-110">Otherwise, the compiler treats white space characters as insignificant and does not include then in the [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] object for the literal.</span></span>  
  
 <span data-ttu-id="62f68-111">Чтобы включить незначащие пробелы в XML-литерал, используйте встроенное выражение, которое содержит строковый литерал с пробелом.</span><span class="sxs-lookup"><span data-stu-id="62f68-111">To include insignificant white space in an XML literal, use an embedded expression that contains a string literal with the white space.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="62f68-112">Если `xml:space` атрибут содержится в литерале XML-элемента, компилятор Visual Basic включает атрибут в <xref:System.Xml.Linq.XElement> объект, но добавление этого атрибута не изменяет способ обработки пробелов компилятором.</span><span class="sxs-lookup"><span data-stu-id="62f68-112">If the `xml:space` attribute appears in an XML element literal, the Visual Basic compiler includes the attribute in the <xref:System.Xml.Linq.XElement> object, but adding this attribute does not change how the compiler treats white space.</span></span>
