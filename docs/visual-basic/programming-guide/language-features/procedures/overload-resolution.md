# <a name="overload-resolution-visual-basic"></a><span data-ttu-id="f9bc6-102">Разрешение перегрузки (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="f9bc6-102">Overload Resolution (Visual Basic)</span></span>
<span data-ttu-id="f9bc6-103">Когда компилятор Visual Basic встречает вызов процедуры, определенной в нескольких перегруженных версиях, компилятор должен решить, какую из перегруженных версий вызывать.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-103">When the Visual Basic compiler encounters a call to a procedure that is defined in several overloaded versions, the compiler must decide which of the overloads to call.</span></span> <span data-ttu-id="f9bc6-104">Это достигается путем выполнения следующих действий.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-104">It does this by performing the following steps:</span></span>  
  
1. <span data-ttu-id="f9bc6-105">**Специальные возможности.**</span><span class="sxs-lookup"><span data-stu-id="f9bc6-105">**Accessibility.**</span></span> <span data-ttu-id="f9bc6-106">Он устраняет все перегрузки с уровнем доступа, который не позволяет вызывающему коду вызывать его.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-106">It eliminates any overload with an access level that prevents the calling code from calling it.</span></span>  
  
2. <span data-ttu-id="f9bc6-107">**Число параметров.**</span><span class="sxs-lookup"><span data-stu-id="f9bc6-107">**Number of Parameters.**</span></span> <span data-ttu-id="f9bc6-108">Он исключает все перегрузки, которые определяют разное количество параметров, чем указано в вызове.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-108">It eliminates any overload that defines a different number of parameters than are supplied in the call.</span></span>  
  
3. <span data-ttu-id="f9bc6-109">**Типы данных параметров.**</span><span class="sxs-lookup"><span data-stu-id="f9bc6-109">**Parameter Data Types.**</span></span> <span data-ttu-id="f9bc6-110">Компилятор передает предпочтение методам экземпляра по методам расширения.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-110">The compiler gives instance methods preference over extension methods.</span></span> <span data-ttu-id="f9bc6-111">Если найден любой метод экземпляра, требующий только расширяющего преобразования для сопоставления с вызовом процедуры, все методы расширения удаляются, а компилятор переходит только к методам экземпляра.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-111">If any instance method is found that requires only widening conversions to match the procedure call, all extension methods are dropped and the compiler continues with only the instance method candidates.</span></span> <span data-ttu-id="f9bc6-112">Если такой метод экземпляра не найден, он продолжится как для экземпляра, так и для методов расширения.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-112">If no such instance method is found, it continues with both instance and extension methods.</span></span>  
  
     <span data-ttu-id="f9bc6-113">На этом шаге удаляется любая перегрузка, для которой типы данных вызывающих аргументов не могут быть преобразованы в типы параметров, определенные в перегрузке.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-113">In this step, it eliminates any overload for which the data types of the calling arguments cannot be converted to the parameter types defined in the overload.</span></span>  
  
4. <span data-ttu-id="f9bc6-114">**Сужающие преобразования.**</span><span class="sxs-lookup"><span data-stu-id="f9bc6-114">**Narrowing Conversions.**</span></span> <span data-ttu-id="f9bc6-115">Он устраняет все перегрузки, требующие сужения преобразования из типов вызывающих аргументов в определенные типы параметров.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-115">It eliminates any overload that requires a narrowing conversion from the calling argument types to the defined parameter types.</span></span> <span data-ttu-id="f9bc6-116">Это верно, независимо от того, является ли параметр проверки типов ([оператор Option строго](../../../../visual-basic/language-reference/statements/option-strict-statement.md)) `On` или `Off`.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-116">This is true whether the type checking switch ([Option Strict Statement](../../../../visual-basic/language-reference/statements/option-strict-statement.md)) is `On` or `Off`.</span></span>  
  
5. <span data-ttu-id="f9bc6-117">**Минимальное расширение.**</span><span class="sxs-lookup"><span data-stu-id="f9bc6-117">**Least Widening.**</span></span> <span data-ttu-id="f9bc6-118">Компилятор рассматривает оставшиеся перегрузки парами.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-118">The compiler considers the remaining overloads in pairs.</span></span> <span data-ttu-id="f9bc6-119">Для каждой пары он сравнивает типы данных определенных параметров.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-119">For each pair, it compares the data types of the defined parameters.</span></span> <span data-ttu-id="f9bc6-120">Если типы в одной из перегрузок расширены до соответствующих типов в другом, компилятор исключает второй.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-120">If the types in one of the overloads all widen to the corresponding types in the other, the compiler eliminates the latter.</span></span> <span data-ttu-id="f9bc6-121">То есть он удерживает перегрузку, для которой требуется наименьший размер расширения.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-121">That is, it retains the overload that requires the least amount of widening.</span></span>  
  
6. <span data-ttu-id="f9bc6-122">**Один кандидат.**</span><span class="sxs-lookup"><span data-stu-id="f9bc6-122">**Single Candidate.**</span></span> <span data-ttu-id="f9bc6-123">Он продолжает рассматривать перегрузки парами до тех пор, пока не останется только одна перегрузка и разрешается вызов этой перегрузки.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-123">It continues considering overloads in pairs until only one overload remains, and it resolves the call to that overload.</span></span> <span data-ttu-id="f9bc6-124">Если компилятор не может уменьшить перегрузки для одного кандидата, возникает ошибка.</span><span class="sxs-lookup"><span data-stu-id="f9bc6-124">If the compiler cannot reduce the overloads to a single candidate, it generates an error.</span></span>
