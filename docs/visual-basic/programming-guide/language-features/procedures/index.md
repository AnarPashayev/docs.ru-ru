# <a name="procedures-in-visual-basic"></a><span data-ttu-id="82ce2-102">Процедуры в Visual Basic</span><span class="sxs-lookup"><span data-stu-id="82ce2-102">Procedures in Visual Basic</span></span>

<span data-ttu-id="82ce2-103">*Процедура* — это блок Visual Basic инструкций, заключенных в оператор объявления ( `Function` ,, `Sub` `Operator` , `Get` , `Set` ) и `End` объявлении сопоставления.</span><span class="sxs-lookup"><span data-stu-id="82ce2-103">A *procedure* is a block of Visual Basic statements enclosed by a declaration statement (`Function`, `Sub`, `Operator`, `Get`, `Set`) and a matching `End` declaration.</span></span> <span data-ttu-id="82ce2-104">Все исполняемые инструкции в Visual Basic должны находиться в пределах некоторой процедуры.</span><span class="sxs-lookup"><span data-stu-id="82ce2-104">All executable statements in Visual Basic must be within some procedure.</span></span>  
  
## <a name="calling-a-procedure"></a><span data-ttu-id="82ce2-105">Вызов процедуры</span><span class="sxs-lookup"><span data-stu-id="82ce2-105">Calling a Procedure</span></span>  

 <span data-ttu-id="82ce2-106">Процедура вызывается из другого места в коде.</span><span class="sxs-lookup"><span data-stu-id="82ce2-106">You invoke a procedure from some other place in the code.</span></span> <span data-ttu-id="82ce2-107">Это называется *вызовом процедуры*.</span><span class="sxs-lookup"><span data-stu-id="82ce2-107">This is known as a *procedure call*.</span></span> <span data-ttu-id="82ce2-108">После завершения процедуры она возвращает управление в код, из которого она была вызвана (*вызывающий код*).</span><span class="sxs-lookup"><span data-stu-id="82ce2-108">When the procedure is finished running, it returns control to the code that invoked it, which is known as the *calling code*.</span></span> <span data-ttu-id="82ce2-109">Вызывающий код — это оператор или выражение в составе оператора, в которых указывается имя процедуры и передается управление процедуре.</span><span class="sxs-lookup"><span data-stu-id="82ce2-109">The calling code is a statement, or an expression within a statement, that specifies the procedure by name and transfers control to it.</span></span>  
  
## <a name="returning-from-a-procedure"></a><span data-ttu-id="82ce2-110">Возврат из процедуры</span><span class="sxs-lookup"><span data-stu-id="82ce2-110">Returning from a Procedure</span></span>  

 <span data-ttu-id="82ce2-111">После завершения процедуры она возвращает управление вызывающему коду.</span><span class="sxs-lookup"><span data-stu-id="82ce2-111">A procedure returns control to the calling code when it has finished running.</span></span> <span data-ttu-id="82ce2-112">Для этого можно использовать [инструкцию Return](../../../language-reference/statements/return-statement.md), соответствующую инструкцию [Exit](../../../language-reference/statements/exit-statement.md) Function для процедуры или инструкцию [End \<keyword> инструкции](../../../language-reference/statements/end-keyword-statement.md) процедуры.</span><span class="sxs-lookup"><span data-stu-id="82ce2-112">To do this, it can use a [Return Statement](../../../language-reference/statements/return-statement.md), the appropriate [Exit Statement](../../../language-reference/statements/exit-statement.md) statement for the procedure, or the procedure's [End \<keyword> Statement](../../../language-reference/statements/end-keyword-statement.md) statement.</span></span> <span data-ttu-id="82ce2-113">Затем управление передается в вызывающий код, следующий за точкой вызова процедуры.</span><span class="sxs-lookup"><span data-stu-id="82ce2-113">Control then passes to the calling code following the point of the procedure call.</span></span>  
  
- <span data-ttu-id="82ce2-114">При использовании оператора `Return` управление немедленно возвращается в вызывающий код.</span><span class="sxs-lookup"><span data-stu-id="82ce2-114">With a `Return` statement, control returns immediately to the calling code.</span></span> <span data-ttu-id="82ce2-115">Операторы, следующие за оператором `Return`, не выполняются.</span><span class="sxs-lookup"><span data-stu-id="82ce2-115">Statements following the `Return` statement do not run.</span></span> <span data-ttu-id="82ce2-116">В одной и той же процедуре можно использовать несколько операторов `Return`.</span><span class="sxs-lookup"><span data-stu-id="82ce2-116">You can have more than one `Return` statement in the same procedure.</span></span>  
  
- <span data-ttu-id="82ce2-117">При использовании операторов `Exit Sub` или `Exit Function` управление немедленно возвращается в вызывающий код.</span><span class="sxs-lookup"><span data-stu-id="82ce2-117">With an `Exit Sub` or `Exit Function` statement, control returns immediately to the calling code.</span></span> <span data-ttu-id="82ce2-118">Операторы, следующие за оператором `Exit`, не выполняются.</span><span class="sxs-lookup"><span data-stu-id="82ce2-118">Statements following the `Exit` statement do not run.</span></span> <span data-ttu-id="82ce2-119">В одной и той же процедуре можно использовать несколько операторов `Exit` и одновременно использовать операторы `Return` и `Exit`.</span><span class="sxs-lookup"><span data-stu-id="82ce2-119">You can have more than one `Exit` statement in the same procedure, and you can mix `Return` and `Exit` statements in the same procedure.</span></span>  
  
- <span data-ttu-id="82ce2-120">Если процедура не содержит операторов `Return` или `Exit`, она завершается оператором `End Sub`, `End Function`, `End Get` или `End Set`, который следует за последним оператором в составе процедуры.</span><span class="sxs-lookup"><span data-stu-id="82ce2-120">If a procedure has no `Return` or `Exit` statements, it concludes with an `End Sub` or `End Function`, `End Get`, or `End Set` statement following the last statement of the procedure body.</span></span> <span data-ttu-id="82ce2-121">При использовании оператора `End` управление немедленно возвращается в вызывающий код.</span><span class="sxs-lookup"><span data-stu-id="82ce2-121">The `End` statement returns control immediately to the calling code.</span></span> <span data-ttu-id="82ce2-122">В процедуре можно использовать только один оператор `End`.</span><span class="sxs-lookup"><span data-stu-id="82ce2-122">You can have only one `End` statement in a procedure.</span></span>  
  
## <a name="parameters-and-arguments"></a><span data-ttu-id="82ce2-123">Параметры и аргументы</span><span class="sxs-lookup"><span data-stu-id="82ce2-123">Parameters and Arguments</span></span>  

 <span data-ttu-id="82ce2-124">В большинстве случаев процедура должна работать с разными данными при каждом вызове.</span><span class="sxs-lookup"><span data-stu-id="82ce2-124">In most cases, a procedure needs to operate on different data each time you call it.</span></span> <span data-ttu-id="82ce2-125">Эти данные можно передать в процедуру при ее вызове.</span><span class="sxs-lookup"><span data-stu-id="82ce2-125">You can pass this information to the procedure as part of the procedure call.</span></span> <span data-ttu-id="82ce2-126">В процедуре можно определить *параметры* (ноль или более), каждый из которых представляет одно из передаваемых значений.</span><span class="sxs-lookup"><span data-stu-id="82ce2-126">The procedure defines zero or more *parameters*, each of which represents a value it expects you to pass to it.</span></span> <span data-ttu-id="82ce2-127">Каждому параметру в определении процедуры соответствует *аргумент* при вызове процедуры.</span><span class="sxs-lookup"><span data-stu-id="82ce2-127">Corresponding to each parameter in the procedure definition is an *argument* in the procedure call.</span></span> <span data-ttu-id="82ce2-128">Аргумент представляет значение, которое передается в соответствующий параметр для заданного вызова процедуры.</span><span class="sxs-lookup"><span data-stu-id="82ce2-128">An argument represents the value you pass to the corresponding parameter in a given procedure call.</span></span>  
  
## <a name="types-of-procedures"></a><span data-ttu-id="82ce2-129">Типы процедур</span><span class="sxs-lookup"><span data-stu-id="82ce2-129">Types of Procedures</span></span>  

 <span data-ttu-id="82ce2-130">Visual Basic использует несколько типов процедур:</span><span class="sxs-lookup"><span data-stu-id="82ce2-130">Visual Basic uses several types of procedures:</span></span>  
  
- <span data-ttu-id="82ce2-131">[Подпрограммы](./sub-procedures.md) выполняют действия, но не возвращают значение в вызывающий код.</span><span class="sxs-lookup"><span data-stu-id="82ce2-131">[Sub Procedures](./sub-procedures.md) perform actions but do not return a value to the calling code.</span></span>  
  
- <span data-ttu-id="82ce2-132">Процедуры обработки событий — это процедуры `Sub`, которые выполняются в ответ на событие, вызванное действием пользователя или наступлением определенных условий в программе.</span><span class="sxs-lookup"><span data-stu-id="82ce2-132">Event-handling procedures are `Sub` procedures that execute in response to an event raised by user action or by an occurrence in a program.</span></span>

    <span data-ttu-id="82ce2-135">Некоторые функции, написанные на C#, возвращают *значение по ссылке*.</span><span class="sxs-lookup"><span data-stu-id="82ce2-135">Some functions written in C# return a *reference return value*.</span></span> <span data-ttu-id="82ce2-136">Вызывающие функции могут изменять возвращаемое значение, и это изменение отражается в состоянии вызываемого объекта.</span><span class="sxs-lookup"><span data-stu-id="82ce2-136">Function callers can modify the return value, and this modification is reflected in the state of the called object.</span></span> <span data-ttu-id="82ce2-137">Начиная с Visual Basic 2017, в коде можно использовать значения, возвращаемые по ссылке, но нельзя возвращать значения по ссылке.</span><span class="sxs-lookup"><span data-stu-id="82ce2-137">Starting with Visual Basic 2017, Visual Basic code can consume reference return values, although it cannot return a value by reference.</span></span> <span data-ttu-id="82ce2-138">Дополнительные сведения см. в разделе [Значения, возвращаемые по ссылке](ref-return-values.md).</span><span class="sxs-lookup"><span data-stu-id="82ce2-138">For more information, see [Reference return values](ref-return-values.md).</span></span>
  
- <span data-ttu-id="82ce2-139">[Процедуры свойств](./property-procedures.md) возвращают и задают значения свойств для объектов или модулей.</span><span class="sxs-lookup"><span data-stu-id="82ce2-139">[Property Procedures](./property-procedures.md) return and assign values of properties on objects or modules.</span></span>  
  
- <span data-ttu-id="82ce2-140">[Процедуры операторов](./operator-procedures.md) определяют поведение стандартного оператора, если один или оба операнда представляют собой недавно определенный класс или структуру.</span><span class="sxs-lookup"><span data-stu-id="82ce2-140">[Operator Procedures](./operator-procedures.md) define the behavior of a standard operator when one or both of the operands is a newly-defined class or structure.</span></span>  
  
- <span data-ttu-id="82ce2-141">В [универсальных процедурах в Visual Basic](../data-types/generic-procedures.md) определяются *параметры типа* в дополнение к обычным параметрам процедуры.  <span data-ttu-id="82ce2-145">Процедуры можно использовать для реализации повторяющихся или общих задач, таких как часто используемые вычисления, операции с текстом и элементами управления и операции с базами данных.</span><span class="sxs-lookup"><span data-stu-id="82ce2-145">Procedures are useful for performing repeated or shared tasks, such as frequently used calculations, text and control manipulation, and database operations.</span></span> <span data-ttu-id="82ce2-146">Процедуры можно вызывать из различных мест в коде; таким образом, их можно использовать в качестве стандартных блоков для создания приложения.</span><span class="sxs-lookup"><span data-stu-id="82ce2-146">You can call a procedure from many different places in your code, so you can use procedures as building blocks for your application.</span></span>
