 <span data-ttu-id="8b686-106">`Get` процедура получает значение свойства, а процедура `Set` сохраняет значение.</span><span class="sxs-lookup"><span data-stu-id="8b686-106">The `Get` procedure retrieves the property's value, and the `Set` procedure stores a value.</span></span> <span data-ttu-id="8b686-107">Если требуется, чтобы свойство имело доступ на чтение и запись, необходимо определить обе процедуры.</span><span class="sxs-lookup"><span data-stu-id="8b686-107">If you want the property to have read/write access, you must define both procedures.</span></span> <span data-ttu-id="8b686-108">Для свойства, доступного только для чтения, вы определяете только `Get`, а для свойства только для записи вы определяете только `Set`.</span><span class="sxs-lookup"><span data-stu-id="8b686-108">For a read-only property, you define only `Get`, and for a write-only property, you define only `Set`.</span></span>  
  
### <a name="to-create-a-property"></a><span data-ttu-id="8b686-109">Создание свойства</span><span class="sxs-lookup"><span data-stu-id="8b686-109">To create a property</span></span>  
  
1. <span data-ttu-id="8b686-110">За пределами любого свойства или процедуры используйте [инструкцию Property](../../../../visual-basic/language-reference/statements/property-statement.md), за которой следует оператор `End Property`.</span><span class="sxs-lookup"><span data-stu-id="8b686-110">Outside any property or procedure, use a [Property Statement](../../../../visual-basic/language-reference/statements/property-statement.md), followed by an `End Property` statement.</span></span>  
  
2. <span data-ttu-id="8b686-111">Если свойство принимает параметры, используйте ключевое слово `Property` с именем процедуры, а затем список параметров в круглых скобках.</span><span class="sxs-lookup"><span data-stu-id="8b686-111">If the property takes parameters, follow the `Property` keyword with the name of the procedure, then the parameter list in parentheses.</span></span>  
  
3. <span data-ttu-id="8b686-112">Используйте круглые скобки с предложением `As`, чтобы указать тип данных значения свойства.</span><span class="sxs-lookup"><span data-stu-id="8b686-112">Follow the parentheses with an `As` clause to specify the data type of the property's value.</span></span> <span data-ttu-id="8b686-113">Необходимо указать тип данных даже для свойства, доступного только на запись.</span><span class="sxs-lookup"><span data-stu-id="8b686-113">You must specify the data type even for a write-only property.</span></span>  
  
4. <span data-ttu-id="8b686-114">При необходимости добавьте `Get` и `Set` процедуры.</span><span class="sxs-lookup"><span data-stu-id="8b686-114">Add `Get` and `Set` procedures, as appropriate.</span></span> <span data-ttu-id="8b686-115">См. следующие направления.</span><span class="sxs-lookup"><span data-stu-id="8b686-115">See the following directions.</span></span>  
  
### <a name="to-create-a-get-procedure-that-retrieves-a-property-value"></a><span data-ttu-id="8b686-116">Создание процедуры Get, которая получает значение свойства</span><span class="sxs-lookup"><span data-stu-id="8b686-116">To create a Get procedure that retrieves a property value</span></span>  
  
1. <span data-ttu-id="8b686-117">Между инструкциями `Property` и `End Property` напишите [инструкцию GET](../../../../visual-basic/language-reference/statements/get-statement.md), за которой следует оператор `End Get`.</span><span class="sxs-lookup"><span data-stu-id="8b686-117">Between the `Property` and `End Property` statements, write a [Get Statement](../../../../visual-basic/language-reference/statements/get-statement.md), followed by an `End Get` statement.</span></span> <span data-ttu-id="8b686-118">Для процедуры `Get` не нужно определять никаких параметров.</span><span class="sxs-lookup"><span data-stu-id="8b686-118">You do not need to define any parameters for the `Get` procedure.</span></span>  
  
2. <span data-ttu-id="8b686-119">Поместите операторы кода, чтобы получить значение свойства между операторами `Get` и `End Get`.</span><span class="sxs-lookup"><span data-stu-id="8b686-119">Place the code statements to retrieve the property's value between the `Get` and `End Get` statements.</span></span> <span data-ttu-id="8b686-120">Этот код может включать другие вычисления и манипуляции с данными в дополнение к формированию и возврату значения свойства.</span><span class="sxs-lookup"><span data-stu-id="8b686-120">This code can include other calculations and data manipulations in addition to generating and returning the property's value.</span></span>  
  
3. <span data-ttu-id="8b686-121">Используйте оператор `Return`, чтобы вернуть значение свойства в вызывающий код.</span><span class="sxs-lookup"><span data-stu-id="8b686-121">Use a `Return` statement to return the property's value to the calling code.</span></span>  
  
 <span data-ttu-id="8b686-122">Необходимо написать `Get` процедуру для свойства, доступного для чтения и записи, и для свойства, доступного только для чтения.</span><span class="sxs-lookup"><span data-stu-id="8b686-122">You must write a `Get` procedure for a read-write property and for a read-only property.</span></span> <span data-ttu-id="8b686-123">Не следует определять `Get`ую процедуру для свойства, доступного только на запись.</span><span class="sxs-lookup"><span data-stu-id="8b686-123">You must not define a `Get` procedure for a write-only property.</span></span>  
  
### <a name="to-create-a-set-procedure-that-writes-a-propertys-value"></a><span data-ttu-id="8b686-124">Создание процедуры Set, которая записывает значение свойства</span><span class="sxs-lookup"><span data-stu-id="8b686-124">To create a Set procedure that writes a property's value</span></span>  
  
1. <span data-ttu-id="8b686-125">Между инструкциями `Property` и `End Property` напишите [инструкцию SET](../../../../visual-basic/language-reference/statements/set-statement.md), за которой следует оператор `End Set`.</span><span class="sxs-lookup"><span data-stu-id="8b686-125">Between the `Property` and `End Property` statements, write a [Set Statement](../../../../visual-basic/language-reference/statements/set-statement.md), followed by an `End Set` statement.</span></span>  
  
2. <span data-ttu-id="8b686-126">В операторе `Set` используйте ключевое слово `Set` со списком параметров в круглых скобках.</span><span class="sxs-lookup"><span data-stu-id="8b686-126">In the `Set` statement, follow the `Set` keyword with a parameter list in parentheses.</span></span> <span data-ttu-id="8b686-127">Этот список параметров должен включать по крайней мере параметр значения для значения, переданного вызывающим кодом.</span><span class="sxs-lookup"><span data-stu-id="8b686-127">This parameter list must include at least a value parameter for the value passed by the calling code.</span></span> <span data-ttu-id="8b686-128">Имя по умолчанию для этого параметра value — `Value`, но при необходимости можно использовать другое имя.</span><span class="sxs-lookup"><span data-stu-id="8b686-128">The default name for this value parameter is `Value`, but you can use a different name if appropriate.</span></span> <span data-ttu-id="8b686-129">Параметр значения должен иметь тот же тип данных, что и само свойство.</span><span class="sxs-lookup"><span data-stu-id="8b686-129">The value parameter must have the same data type as the property itself.</span></span>  
  
3. <span data-ttu-id="8b686-130">Поместите инструкции кода для хранения значения в свойстве между операторами `Set` и `End Set`.</span><span class="sxs-lookup"><span data-stu-id="8b686-130">Place the code statements to store a value in the property between the `Set` and `End Set` statements.</span></span> <span data-ttu-id="8b686-131">Этот код может включать другие вычисления и манипуляции с данными в дополнение к проверке и хранению значения свойства.</span><span class="sxs-lookup"><span data-stu-id="8b686-131">This code can include other calculations and data manipulations in addition to validating and storing the property's value.</span></span>  
  
4. <span data-ttu-id="8b686-132">Используйте параметр value, чтобы принять значение, предоставленное вызывающим кодом.</span><span class="sxs-lookup"><span data-stu-id="8b686-132">Use the value parameter to accept the value supplied by the calling code.</span></span> <span data-ttu-id="8b686-133">Это значение можно сохранить непосредственно в операторе присваивания или использовать в выражении для вычисления внутреннего значения, которое должно быть сохранено.</span><span class="sxs-lookup"><span data-stu-id="8b686-133">You can either store this value directly in an assignment statement, or use it in an expression to calculate the internal value to be stored.</span></span>  
  
 <span data-ttu-id="8b686-134">Для свойства, доступного для чтения и записи, и для свойства только для записи необходимо написать `Set` процедуру.</span><span class="sxs-lookup"><span data-stu-id="8b686-134">You must write a `Set` procedure for a read-write property and for a write-only property.</span></span> <span data-ttu-id="8b686-135">Не следует определять `Set`ую процедуру для свойства, доступного только для чтения.</span><span class="sxs-lookup"><span data-stu-id="8b686-135">You must not define a `Set` procedure for a read-only property.</span></span>
