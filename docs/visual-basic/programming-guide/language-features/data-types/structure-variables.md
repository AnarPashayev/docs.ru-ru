# <a name="structure-variables-visual-basic"></a><span data-ttu-id="d3e2b-102">Переменные структуры (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="d3e2b-102">Structure Variables (Visual Basic)</span></span>
<span data-ttu-id="d3e2b-103">После создания структуры, можно объявить переменные уровня модуля и процедуры для определенного типа.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-103">Once you have created a structure, you can declare procedure-level and module-level variables as that type.</span></span> <span data-ttu-id="d3e2b-104">Например можно создать структуру, которая записывает сведения о системе компьютера.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-104">For example, you can create a structure that records information about a computer system.</span></span> <span data-ttu-id="d3e2b-105">В следующем примере это показано.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-105">The following example demonstrates this.</span></span>  
  
```  
Public Structure systemInfo  
    Public cPU As String  
    Public memory As Long  
    Public purchaseDate As Date  
End Structure  
```  
  
 <span data-ttu-id="d3e2b-106">Можно объявить переменные этого типа.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-106">You can now declare variables of that type.</span></span> <span data-ttu-id="d3e2b-107">Это показано в следующем объявлении.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-107">The following declaration illustrates this.</span></span>  
  
```  
Dim mySystem, yourSystem As systemInfo  
```  
  
> [!NOTE]
>  <span data-ttu-id="d3e2b-108">В классы и модули, структуры объявляются с помощью [оператор Dim](../../../../visual-basic/language-reference/statements/dim-statement.md) по умолчанию для общего доступа.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-108">In classes and modules, structures declared using the [Dim Statement](../../../../visual-basic/language-reference/statements/dim-statement.md) default to public access.</span></span> <span data-ttu-id="d3e2b-109">Если структура должна быть частными, убедитесь, что необходимо объявить ее с [частного](../../../../visual-basic/language-reference/modifiers/private.md) ключевое слово.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-109">If you intend a structure to be private, make sure you declare it using the [Private](../../../../visual-basic/language-reference/modifiers/private.md) keyword.</span></span>  
  
## <a name="access-to-structure-values"></a><span data-ttu-id="d3e2b-110">Доступ к значениям структуры</span><span class="sxs-lookup"><span data-stu-id="d3e2b-110">Access to Structure Values</span></span>  
 <span data-ttu-id="d3e2b-111">Для присвоения и извлечения значений из элементов структуры переменной, используется тот же синтаксис, как используются для задания и получения свойств объекта.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-111">To assign and retrieve values from the elements of a structure variable, you use the same syntax as you use to set and get properties on an object.</span></span> <span data-ttu-id="d3e2b-112">Поместите оператор доступа к членам (`.`) между именем переменной структуры и имя элемента.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-112">You place the member access operator (`.`) between the structure variable name and the element name.</span></span> <span data-ttu-id="d3e2b-113">Следующий пример получает доступ к элементам переменных, ранее объявленных как тип `systemInfo`.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-113">The following example accesses elements of the variables previously declared as type `systemInfo`.</span></span>  
  
```  
mySystem.cPU = "486"  
Dim tooOld As Boolean  
If yourSystem.purchaseDate < #1/1/1992# Then tooOld = True  
```  
  
## <a name="assigning-structure-variables"></a><span data-ttu-id="d3e2b-114">Назначение переменных структуры</span><span class="sxs-lookup"><span data-stu-id="d3e2b-114">Assigning Structure Variables</span></span>  
 <span data-ttu-id="d3e2b-115">Можно также присвоить одну переменную в другой, если оба имеют один и тот же тип структуры.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-115">You can also assign one variable to another if both are of the same structure type.</span></span> <span data-ttu-id="d3e2b-116">Это копирует все элементы из одной структуры в соответствующие элементы в другой.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-116">This copies all the elements of one structure to the corresponding elements in the other.</span></span> <span data-ttu-id="d3e2b-117">Это показано в следующем объявлении.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-117">The following declaration illustrates this.</span></span>  
  
```  
yourSystem = mySystem  
```  
  
 <span data-ttu-id="d3e2b-118">Если элемент структуры является ссылочным типом, такие как `String`, `Object`, или массив, указатель на данные, копируется.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-118">If a structure element is a reference type, such as a `String`, `Object`, or array, the pointer to the data is copied.</span></span> <span data-ttu-id="d3e2b-119">В предыдущем примере если `systemInfo` бы были включены в объектной переменной, а затем копируется указатель из предыдущего примера `mySystem` для `yourSystem`, и изменение данных объекта через одну структуру будет действовать при обращении к через другую структуру.</span><span class="sxs-lookup"><span data-stu-id="d3e2b-119">In the previous example, if `systemInfo` had included an object variable, then the preceding example would have copied the pointer from `mySystem` to `yourSystem`, and a change to the object's data through one structure would be in effect when accessed through the other structure.</span></span>
