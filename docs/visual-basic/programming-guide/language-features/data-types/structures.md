<span data-ttu-id="5343a-103">*Структура* представляет собой обобщение определяемого пользователем типа (UDT), поддерживаемого предыдущими версиями Visual Basic.</span><span class="sxs-lookup"><span data-stu-id="5343a-103">A *structure* is a generalization of the user-defined type (UDT) supported by previous versions of Visual Basic.</span></span> <span data-ttu-id="5343a-104">В дополнение к полям, структуры могут предоставлять свойства, методы и события.</span><span class="sxs-lookup"><span data-stu-id="5343a-104">In addition to fields, structures can expose properties, methods, and events.</span></span> <span data-ttu-id="5343a-105">Структура может реализовывать один или несколько интерфейсов, и можно объявить отдельные уровни доступа для каждого поля.</span><span class="sxs-lookup"><span data-stu-id="5343a-105">A structure can implement one or more interfaces, and you can declare individual access levels for each field.</span></span>  
  
 <span data-ttu-id="5343a-106">Для создания структуры можно сочетать элементы данных различных типов.</span><span class="sxs-lookup"><span data-stu-id="5343a-106">You can combine data items of different types to create a structure.</span></span> <span data-ttu-id="5343a-107">Структура связывает один или несколько *элементов* друг с другом и с самой структурой.</span><span class="sxs-lookup"><span data-stu-id="5343a-107">A structure associates one or more *elements* with each other and with the structure itself.</span></span> <span data-ttu-id="5343a-108">При объявлении структуры она превращается в *составной тип данных*, и можно объявлять переменные этого типа.</span><span class="sxs-lookup"><span data-stu-id="5343a-108">When you declare a structure, it becomes a *composite data type*, and you can declare variables of that type.</span></span>  
  
 <span data-ttu-id="5343a-109">Структуры полезны, если требуется, чтобы одна переменная содержала несколько связанных частей информации.</span><span class="sxs-lookup"><span data-stu-id="5343a-109">Structures are useful when you want a single variable to hold several related pieces of information.</span></span> <span data-ttu-id="5343a-110">Например, может потребоваться одновременное отслеживание имени сотрудника, телефонного расширения и оклада.</span><span class="sxs-lookup"><span data-stu-id="5343a-110">For example, you might want to keep an employee's name, telephone extension, and salary together.</span></span> <span data-ttu-id="5343a-111">Для этой информации можно использовать несколько переменных или определить структуру и использовать ее для одной переменной сотрудника.</span><span class="sxs-lookup"><span data-stu-id="5343a-111">You could use several variables for this information, or you could define a structure and use it for a single employee variable.</span></span> <span data-ttu-id="5343a-112">Преимущество структуры станет очевидным, если у вас много сотрудников и, следовательно, много экземпляров переменной.</span><span class="sxs-lookup"><span data-stu-id="5343a-112">The advantage of the structure becomes apparent when you have many employees and therefore many instances of the variable.</span></span>
