## <a name="conversion-to-an-object-array"></a><span data-ttu-id="8cbd6-117">Преобразование в массив объектов</span><span class="sxs-lookup"><span data-stu-id="8cbd6-117">Conversion to an Object Array</span></span>  
 <span data-ttu-id="8cbd6-118">При объявлении `Object` массива без инициализации, его тип элемента — `Object` до тех пор, пока он не будет инициализирован.</span><span class="sxs-lookup"><span data-stu-id="8cbd6-118">When you declare an `Object` array without initializing it, its element type is `Object` as long as it remains uninitialized.</span></span> <span data-ttu-id="8cbd6-119">При установке его на массив определенного класса, он принимает тип этого класса.</span><span class="sxs-lookup"><span data-stu-id="8cbd6-119">When you set it to an array of a specific class, it takes on the type of that class.</span></span> <span data-ttu-id="8cbd6-120">Тем не менее, его базовым типом является по-прежнему `Object`, и можно впоследствии разместить его в другой массив несвязанного класса.</span><span class="sxs-lookup"><span data-stu-id="8cbd6-120">However, its underlying type is still `Object`, and you can subsequently set it to another array of an unrelated class.</span></span> <span data-ttu-id="8cbd6-121">Так как все классы являются производными от `Object`, можно изменить тип элемента массива из любого класса к любому другому классу.</span><span class="sxs-lookup"><span data-stu-id="8cbd6-121">Since all classes derive from `Object`, you can change the array's element type from any class to any other class.</span></span>  
  
 <span data-ttu-id="8cbd6-122">В следующем примере, не существует преобразования между типами `student` и `String`, но оба являются производными от `Object`, поэтому все назначения являются допустимыми.</span><span class="sxs-lookup"><span data-stu-id="8cbd6-122">In the following example, no conversion exists between types `student` and `String`, but both derive from `Object`, so all assignments are valid.</span></span>  
  
```  
' Assume student has already been defined as a class.  
Dim testArray() As Object  
' testArray is still an Object array at this point.  
Dim names() As String = New String(3) {"Name0", "Name1", "Name2", "Name3"}  
testArray = New student(3) {}  
' testArray is now of type student().  
testArray = names  
' testArray is now a String array.  
```  
  
### <a name="underlying-type-of-an-array"></a><span data-ttu-id="8cbd6-123">Базовый тип массива</span><span class="sxs-lookup"><span data-stu-id="8cbd6-123">Underlying Type of an Array</span></span>  
 <span data-ttu-id="8cbd6-124">Если изначально объявляется массив с определенным классом, его базовый тип элемента является этот класс.</span><span class="sxs-lookup"><span data-stu-id="8cbd6-124">If you originally declare an array with a specific class, its underlying element type is that class.</span></span> <span data-ttu-id="8cbd6-125">Если в дальнейшем массив другого класса, необходимо выполнить преобразование между двумя классами.</span><span class="sxs-lookup"><span data-stu-id="8cbd6-125">If you subsequently set it to an array of another class, there must be a conversion between the two classes.</span></span>  
  
 <span data-ttu-id="8cbd6-126">В следующем примере `students` является `student` массива.</span><span class="sxs-lookup"><span data-stu-id="8cbd6-126">In the following example, `students` is a `student` array.</span></span> <span data-ttu-id="8cbd6-127">Так как не существует преобразования между `String` и `student`, последняя инструкция завершается неудачно.</span><span class="sxs-lookup"><span data-stu-id="8cbd6-127">Since no conversion exists between `String` and `student`, the last statement fails.</span></span>  
  
```  
Dim students() As student  
Dim names() As String = New String(3) {"Name0", "Name1", "Name2", "Name3"}  
students = New Student(3) {}  
' The following statement fails at compile time.  
students = names  
```
