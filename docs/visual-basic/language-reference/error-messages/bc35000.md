<span data-ttu-id="a9997-103">Visual Basic пытается выполнить осуществляют внутренний вызов функции в среде выполнения Visual Basic (Microsoft.VisualBasic.dll), который не удается найти.</span><span class="sxs-lookup"><span data-stu-id="a9997-103">Visual Basic is attempting to make an internal call to a function in the Visual Basic Runtime (Microsoft.VisualBasic.dll) that cannot be found.</span></span>

<span data-ttu-id="a9997-104">**Идентификатор ошибки:** BC35000</span><span class="sxs-lookup"><span data-stu-id="a9997-104">**Error ID:** BC35000</span></span>

<span data-ttu-id="a9997-105">Эта ошибка возникает в стиле пакетов SDK для проектов (проектов с помощью \*VBPROJ-файл, который начинается со строки `<Project Sdk="Microsoft.NET.Sdk">`).</span><span class="sxs-lookup"><span data-stu-id="a9997-105">This error occurs in SDK-style projects (projects with a \*.vbproj file that begins with the line `<Project Sdk="Microsoft.NET.Sdk">`).</span></span> <span data-ttu-id="a9997-106">По умолчанию только подмножество сборки Microsoft.VisualBasic.dll внедряется в сборку приложения и  *\<функция >* не включается в это подмножество.</span><span class="sxs-lookup"><span data-stu-id="a9997-106">By default, only a subset of the Microsoft.VisualBasic.dll assembly is embedded in the application assembly, and *\<function>* is not included in that subset.</span></span>
