# <a name="and-expected"></a><span data-ttu-id="17a8c-102">Требуется And</span><span class="sxs-lookup"><span data-stu-id="17a8c-102">'And' expected</span></span>
<span data-ttu-id="17a8c-103">Для объединения двух или более операторов `And` в предложении `Equals` или `Join` используется оператор сравнения, отличный от `Group Join` .</span><span class="sxs-lookup"><span data-stu-id="17a8c-103">A comparison operator other than `And` has been used to combine two or more `Equals` operators in a `Join` or `Group Join` clause.</span></span> <span data-ttu-id="17a8c-104">Только оператор `And` может объединять несколько операторов `Equals` в предложении `Join` или `Group Join` .</span><span class="sxs-lookup"><span data-stu-id="17a8c-104">Only the `And` operator is allowed to combine multiple `Equals` operators in a `Join` or `Group Join` clause.</span></span>

## <a name="to-correct-this-error"></a><span data-ttu-id="17a8c-106">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="17a8c-106">To correct this error</span></span>  
  
1. <span data-ttu-id="17a8c-107">Реструктурируйте предложения `Equals` , чтобы проводить сравнения только с помощью оператора `And` .</span><span class="sxs-lookup"><span data-stu-id="17a8c-107">Restructure the `Equals` clauses to make comparisons by using only the `And` operator.</span></span> <span data-ttu-id="17a8c-108">Ниже представлен пример такого кода.</span><span class="sxs-lookup"><span data-stu-id="17a8c-108">The following is an example:</span></span>

                    On pet.Owner Equals pers And _  
                       pet.Name = pers.PetName_
