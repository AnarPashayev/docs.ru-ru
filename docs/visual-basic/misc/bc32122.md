<span data-ttu-id="3d327-103">Универсальный интерфейс наследует от двух или более универсальных интерфейсов, и два из этих наследований могут конфликтовать для некоторых значений аргументов типов.</span><span class="sxs-lookup"><span data-stu-id="3d327-103">A generic interface inherits from two or more generic interfaces, and two of the inheritances could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="3d327-104">Следующие операторы могут привести к этой ошибке.</span><span class="sxs-lookup"><span data-stu-id="3d327-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface interfaceA(Of u)

    Inherits interfaceA(Of v)  
End Interface  
Public Interface interfaceY(Of w)  
    Inherits interfaceA(Of w)  
End Interface  
Public Interface derivedInterface(Of t1, t2)  
    Inherits interfaceX(Of t1), interfaceY(Of t2)  
End Interface  
```  
  
 <span data-ttu-id="3d327-105">Если `derivedInterface` создается или реализуется, предоставляя один и тот же тип как для `t1` , так и для `t2`, он должен наследовать две версии `interfaceA` с одинаковыми аргументами типов.</span><span class="sxs-lookup"><span data-stu-id="3d327-105">If `derivedInterface` is constructed or implemented supplying the same type to both `t1` and `t2`, it must inherit two versions of `interfaceA` with identical type arguments.</span></span> <span data-ttu-id="3d327-106">Это приведет к неоднозначности определения, к какой версии следует получить доступ.</span><span class="sxs-lookup"><span data-stu-id="3d327-106">Doing so would produce an ambiguity about which version to access.</span></span>
