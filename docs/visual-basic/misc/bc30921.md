# <a name="derivedtypename-cannot-inherit-from-type-constructedbasetypename-because-it-expands-the-access-of-type-internaltypename-to-region-regionname"></a><span data-ttu-id="7f14c-102">" \<derivedtypename> " не может наследовать от \<type> " \<constructedbasetypename> ", так как он расширяет доступ типа " \<internaltypename> " на \<region> " \<regionname> "</span><span class="sxs-lookup"><span data-stu-id="7f14c-102">'\<derivedtypename>' cannot inherit from \<type> '\<constructedbasetypename>' because it expands the access of type '\<internaltypename>' to \<region> '\<regionname>'</span></span>
<span data-ttu-id="7f14c-103">Производный класс или интерфейс пытается расширить уровень доступа внутреннего типа, используя его как аргумент типа для базового класса или интерфейса.</span><span class="sxs-lookup"><span data-stu-id="7f14c-103">A derived class or interface attempts to expand the access level of an internal type by using it as a type argument to a base class or interface.</span></span>  
  
 <span data-ttu-id="7f14c-104">Приведенный ниже код может вызвать эту ошибку.</span><span class="sxs-lookup"><span data-stu-id="7f14c-104">The following code can generate this error.</span></span>  
  
```vb  
Public Class containingClass  
    Public Class baseClass(Of t)  
    End Class  
    Friend Class derivedClass  
        Inherits baseClass(Of internalStructure)  
    End Class  
    Private Structure internalStructure  
        Dim firstMember As Integer  
    End Structure  
End Class  
```  
  
 <span data-ttu-id="7f14c-105">Коду вне `containingClass` не разрешен доступ к `internalStructure`.</span><span class="sxs-lookup"><span data-stu-id="7f14c-105">Code outside `containingClass` is not allowed to access `internalStructure`.</span></span> <span data-ttu-id="7f14c-106">Однако доступ к `derivedClass` может осуществляться из любого кода в той же сборке.</span><span class="sxs-lookup"><span data-stu-id="7f14c-106">However, `derivedClass` can be accessed from any code in the same assembly.</span></span> <span data-ttu-id="7f14c-107">Таким образом, `derivedClass` не может наследовать `baseClass` , если он использует `internalStructure` в качестве аргумента типа, так как это может предоставить доступ к `internalStructure` во всей заданной области кода.</span><span class="sxs-lookup"><span data-stu-id="7f14c-107">Therefore, `derivedClass` cannot inherit `baseClass` if it uses `internalStructure` as a type argument, because that could expose `internalStructure` throughout the defining code region.</span></span>  
  
 <span data-ttu-id="7f14c-108">**Идентификатор ошибки:** BC30921</span><span class="sxs-lookup"><span data-stu-id="7f14c-108">**Error ID:** BC30921</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="7f14c-109">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="7f14c-109">To correct this error</span></span>  
  
- <span data-ttu-id="7f14c-110">Настройте уровни доступа классов или интерфейсов таким образом, чтобы производный тип не расширял уровень доступа внутреннего типа.</span><span class="sxs-lookup"><span data-stu-id="7f14c-110">Adjust the access levels of the classes or interfaces so that the derived type does not expand the access level of the internal type.</span></span>  
  
     <span data-ttu-id="7f14c-111">-или-</span><span class="sxs-lookup"><span data-stu-id="7f14c-111">-or-</span></span>  
  
- <span data-ttu-id="7f14c-112">Если вы не можете настроить уровни доступа, не используйте внутренний тип в качестве аргумента типа при создании базового класса или интерфейса.</span><span class="sxs-lookup"><span data-stu-id="7f14c-112">If you cannot adjust the access levels, then do not use the internal type as a type argument when constructing the base class or interface.</span></span>
