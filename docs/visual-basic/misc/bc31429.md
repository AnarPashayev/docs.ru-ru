# <a name="elementname-is-ambiguous-because-multiple-kinds-of-members-with-this-name-exist-in-type-typename"></a><span data-ttu-id="d3f6b-102">'\<elementname >' является неоднозначным, поскольку несколько видов членов с таким именем существует в \<тип > '\<typename >'</span><span class="sxs-lookup"><span data-stu-id="d3f6b-102">'\<elementname>' is ambiguous because multiple kinds of members with this name exist in \<type> '\<typename>'</span></span>
<span data-ttu-id="d3f6b-103">Выражение обращается к программному элементу, определенному в классе, структуре, модуле или интерфейсе, содержащем несколько членов с тем же именем.</span><span class="sxs-lookup"><span data-stu-id="d3f6b-103">An expression accesses a programming element defined in a class, structure, module, or interface that contains more than one member with the same name.</span></span>  
  
 <span data-ttu-id="d3f6b-104">Наиболее вероятной причиной этой ошибки является *учет регистра*.</span><span class="sxs-lookup"><span data-stu-id="d3f6b-104">The most likely cause of this error is *case sensitivity*.</span></span> <span data-ttu-id="d3f6b-105">В именах Visual Basic регистр не учитывается, что означает, что в разных местах в коде эти имена можно указывать в разном регистре.</span><span class="sxs-lookup"><span data-stu-id="d3f6b-105">Visual Basic names are case-insensitive, which means you can capitalize them differently at different places in your code.</span></span> <span data-ttu-id="d3f6b-106">Например, если определить переменную с именем `XYZ` и позже обратиться к ней как к `xyz`, компилятор сочтет эти два имени равнозначными.</span><span class="sxs-lookup"><span data-stu-id="d3f6b-106">For example, if you define a variable with the name `XYZ` and later access it as `xyz`, the compiler considers the two names to be equivalent.</span></span>  
  
 <span data-ttu-id="d3f6b-107">Однако в других языках, таких как [C#](../../csharp/index.yml) и [Visual C++ ](/cpp/index), учитывается регистр.</span><span class="sxs-lookup"><span data-stu-id="d3f6b-107">However, other languages, such as [C#](../../csharp/index.yml) and [Visual C++](/cpp/index), are case-sensitive.</span></span> <span data-ttu-id="d3f6b-108">В таких языках `XYZ` и `xyz` не считаются одинаковыми.</span><span class="sxs-lookup"><span data-stu-id="d3f6b-108">In such a language, `XYZ` and `xyz` are not considered to be the same name.</span></span> <span data-ttu-id="d3f6b-109">Следовательно, в классе, написанном на таком языке, должна быть задана переменная с именем `XYZ` и свойство с именем `xyz`.</span><span class="sxs-lookup"><span data-stu-id="d3f6b-109">Therefore, a class written in such a language could define a variable named `XYZ` and a property named `xyz`.</span></span> <span data-ttu-id="d3f6b-110">Среда CLR сохраняет учет регистра в сборках.</span><span class="sxs-lookup"><span data-stu-id="d3f6b-110">The common language runtime (CLR) preserves case sensitivity in assemblies.</span></span> <span data-ttu-id="d3f6b-111">Однако если приложение Visual Basic обращается к сборке с именами `XYZ` и `xyz`, они отображаются как одно и то же имя.</span><span class="sxs-lookup"><span data-stu-id="d3f6b-111">However, if a Visual Basic application accesses an assembly with names `XYZ` and `xyz`, they appear as the same name.</span></span>  
  
 <span data-ttu-id="d3f6b-112">**Код ошибки:** BC31429</span><span class="sxs-lookup"><span data-stu-id="d3f6b-112">**Error ID:** BC31429</span></span>
