# <a name="elementname-is-ambiguous-because-multiple-kinds-of-members-with-this-name-exist-in-type-typename"></a><span data-ttu-id="32e2f-102">" \<elementname> " является неоднозначным, так как несколько видов членов с этим именем существуют в \<type> " \<typename> "</span><span class="sxs-lookup"><span data-stu-id="32e2f-102">'\<elementname>' is ambiguous because multiple kinds of members with this name exist in \<type> '\<typename>'</span></span>

<span data-ttu-id="32e2f-103">Выражение обращается к программному элементу, определенному в классе, структуре, модуле или интерфейсе, содержащем несколько членов с тем же именем.</span><span class="sxs-lookup"><span data-stu-id="32e2f-103">An expression accesses a programming element defined in a class, structure, module, or interface that contains more than one member with the same name.</span></span>  
  
 <span data-ttu-id="32e2f-104">Наиболее вероятной причиной этой ошибки является *учет регистра*.</span><span class="sxs-lookup"><span data-stu-id="32e2f-104">The most likely cause of this error is *case sensitivity*.</span></span> <span data-ttu-id="32e2f-105">В именах Visual Basic регистр не учитывается, что означает, что в разных местах в коде эти имена можно указывать в разном регистре.</span><span class="sxs-lookup"><span data-stu-id="32e2f-105">Visual Basic names are case-insensitive, which means you can capitalize them differently at different places in your code.</span></span> <span data-ttu-id="32e2f-106">Например, если определить переменную с именем `XYZ` и позже обратиться к ней как к `xyz`, компилятор сочтет эти два имени равнозначными.</span><span class="sxs-lookup"><span data-stu-id="32e2f-106">For example, if you define a variable with the name `XYZ` and later access it as `xyz`, the compiler considers the two names to be equivalent.</span></span>  
  
 <span data-ttu-id="32e2f-107">Однако в других языках, таких как [C#](../../csharp/index.yml) и [Visual C++](/cpp/index), учитывается регистр.</span><span class="sxs-lookup"><span data-stu-id="32e2f-107">However, other languages, such as [C#](../../csharp/index.yml) and [Visual C++](/cpp/index), are case-sensitive.</span></span> <span data-ttu-id="32e2f-108">В таких языках `XYZ` и `xyz` не считаются одинаковыми.</span><span class="sxs-lookup"><span data-stu-id="32e2f-108">In such a language, `XYZ` and `xyz` are not considered to be the same name.</span></span> <span data-ttu-id="32e2f-109">Следовательно, в классе, написанном на таком языке, должна быть задана переменная с именем `XYZ` и свойство с именем `xyz`.</span><span class="sxs-lookup"><span data-stu-id="32e2f-109">Therefore, a class written in such a language could define a variable named `XYZ` and a property named `xyz`.</span></span> <span data-ttu-id="32e2f-110">Среда CLR сохраняет учет регистра в сборках.</span><span class="sxs-lookup"><span data-stu-id="32e2f-110">The common language runtime (CLR) preserves case sensitivity in assemblies.</span></span> <span data-ttu-id="32e2f-111">Однако если приложение Visual Basic обращается к сборке с именами `XYZ` и `xyz`, они отображаются как одно и то же имя.</span><span class="sxs-lookup"><span data-stu-id="32e2f-111">However, if a Visual Basic application accesses an assembly with names `XYZ` and `xyz`, they appear as the same name.</span></span>  
  
 <span data-ttu-id="32e2f-112">**Код ошибки:** BC31429</span><span class="sxs-lookup"><span data-stu-id="32e2f-112">**Error ID:** BC31429</span></span>

1. <span data-ttu-id="32e2f-114">Если вы можете управлять исходным кодом определяющего типа, рассмотрите возможность переименования членов, чтобы они отличались не только регистром.</span><span class="sxs-lookup"><span data-stu-id="32e2f-114">If you have control over the source code of the defining type, consider renaming the members so that they differ by more than only casing.</span></span> <span data-ttu-id="32e2f-115">Это может оказаться невозможным, если определяющий тип уже был опубликован и используется другими приложениями.</span><span class="sxs-lookup"><span data-stu-id="32e2f-115">This might not be possible if the defining type has already been published and is being used by other applications.</span></span>  
  
2. <span data-ttu-id="32e2f-116">Если вы не можете переименовать элементы в определяющем типе, удалите рассматриваемый программный элемент из кода.</span><span class="sxs-lookup"><span data-stu-id="32e2f-116">If you cannot rename the members in the defining type, remove the cited programming element from your code.</span></span> <span data-ttu-id="32e2f-117">Вы не можете получить доступ к элементу, который присутствует в Visual Basic, чтобы иметь несколько определений.</span><span class="sxs-lookup"><span data-stu-id="32e2f-117">You cannot access an element that appears to Visual Basic to have multiple definitions.</span></span>
