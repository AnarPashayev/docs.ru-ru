# <a name="cannot-inherit-interface-interfacename1-because-it-could-be-identical-to-interface-interfacename2-for-some-type-arguments"></a><span data-ttu-id="d4f66-102">Невозможно наследовать интерфейс "\<interfacename1>", поскольку он может совпадать с интерфейсом "\<interfacename2>" для некоторых аргументов типа</span><span class="sxs-lookup"><span data-stu-id="d4f66-102">Cannot inherit interface '\<interfacename1>' because it could be identical to interface '\<interfacename2>' for some type arguments</span></span>

<span data-ttu-id="d4f66-103">Универсальный интерфейс наследует более одного раза от другого универсального интерфейса, и два этих наследования могут конфликтовать для некоторых значений аргументов типов.</span><span class="sxs-lookup"><span data-stu-id="d4f66-103">A generic interface inherits more than once from another generic interface, and two of the inheritances could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="d4f66-104">Эта ошибка может быть вызвана следующими инструкциями.</span><span class="sxs-lookup"><span data-stu-id="d4f66-104">The following statements can generate this error.</span></span>  
  
 `Public Interface interfaceA(Of u)`  
  
 `End Interface`  
  
 `Public Interface derivedInterface(Of t1, t2)`  
  
 `Inherits interfaceA(Of t1), interfaceA(Of t2)`  
  
 `End Interface`  
  
 <span data-ttu-id="d4f66-105">Если `derivedInterface` создается или реализуется, предоставляя один и тот же тип как для `t1` , так и для `t2`, он должен наследовать две версии `interfaceA` с одинаковыми аргументами типов.</span><span class="sxs-lookup"><span data-stu-id="d4f66-105">If `derivedInterface` is constructed or implemented supplying the same type to both `t1` and `t2`, it must inherit two versions of `interfaceA` with identical type arguments.</span></span> <span data-ttu-id="d4f66-106">Это приведет к неоднозначности определения, к какой версии следует получить доступ.</span><span class="sxs-lookup"><span data-stu-id="d4f66-106">Doing so would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="d4f66-107">**Идентификатор ошибки:** BC32120</span><span class="sxs-lookup"><span data-stu-id="d4f66-107">**Error ID:** BC32120</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="d4f66-108">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="d4f66-108">To correct this error</span></span>  
  
- <span data-ttu-id="d4f66-109">Измените один из аргументов типов, предоставленных унаследованному интерфейсу, чтобы не было конфликтов.</span><span class="sxs-lookup"><span data-stu-id="d4f66-109">Change one of the type arguments supplied to the derived interface so that there is no conflict.</span></span>  
  
     <span data-ttu-id="d4f66-110">-или-</span><span class="sxs-lookup"><span data-stu-id="d4f66-110">-or-</span></span>  
  
- <span data-ttu-id="d4f66-111">Удалите из оператора `Inherits` один из интерфейсов, вызывающих возможный конфликт наследования или реализации.</span><span class="sxs-lookup"><span data-stu-id="d4f66-111">Remove from the `Inherits` statement one of the interfaces causing the potential inheritance or implementation conflict.</span></span>
