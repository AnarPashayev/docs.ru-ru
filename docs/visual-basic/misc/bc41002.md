 <span data-ttu-id="e6af4-104">Если производный класс не объявляет конструктор, Visual Basic пытается создать неявный конструктор без параметров, который вызывает `MyBase.New()` .</span><span class="sxs-lookup"><span data-stu-id="e6af4-104">When a derived class does not declare a constructor, Visual Basic attempts to generate an implicit parameterless constructor that calls `MyBase.New()`.</span></span> <span data-ttu-id="e6af4-105">Если в базовом классе нет доступного конструктора, который можно вызвать без аргументов, Visual Basic не может создать неявный конструктор.</span><span class="sxs-lookup"><span data-stu-id="e6af4-105">If there is no accessible constructor in the base class that can be called without arguments, Visual Basic cannot generate an implicit constructor.</span></span> <span data-ttu-id="e6af4-106">В этом случае обязательный конструктор помечается <xref:System.ObsoleteAttribute> атрибутом, поэтому Visual Basic не может вызвать его.</span><span class="sxs-lookup"><span data-stu-id="e6af4-106">In this case, the required constructor is marked with the <xref:System.ObsoleteAttribute> attribute, so Visual Basic cannot call it.</span></span>
