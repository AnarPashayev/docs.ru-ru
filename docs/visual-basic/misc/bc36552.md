 <span data-ttu-id="42e61-105">Метод расширения без параметров является недопустимым, так как первый параметр задает тип данных, который расширяет метод.</span><span class="sxs-lookup"><span data-stu-id="42e61-105">An extension method without parameters is not valid because the first parameter specifies which data type the method extends.</span></span> <span data-ttu-id="42e61-106">Первый параметр привязан к экземпляру типа данных, который вызывает метод.</span><span class="sxs-lookup"><span data-stu-id="42e61-106">The first parameter is bound to the instance of the data type that invokes the method.</span></span>

## <a name="example"></a><span data-ttu-id="42e61-110">Пример</span><span class="sxs-lookup"><span data-stu-id="42e61-110">Example</span></span>  

 <span data-ttu-id="42e61-111">Первый параметр в приведенном ниже примере указывает на то, что метод `Print` расширяет тип данных `String` .</span><span class="sxs-lookup"><span data-stu-id="42e61-111">The first parameter in the following example indicates that the `Print` method extends the `String` data type.</span></span>  
  
```vb  
<Extension()> _  
Public Sub Print (ByVal str As String)  
    Console.WriteLine(str)  
End Sub  
```  
  
 <span data-ttu-id="42e61-112">Когда метод расширения вызывается указанным ниже образом, параметр `str` в методе привязывается к `greeting`, экземпляру `String` , который вызывает метод `Print`.</span><span class="sxs-lookup"><span data-stu-id="42e61-112">When the extension method is called as follows, parameter `str` in the method is bound to `greeting`, the instance of `String` that calls `Print`.</span></span> <span data-ttu-id="42e61-113">Компилятор использует `greeting` в качестве аргумента для метода расширения `Print`.</span><span class="sxs-lookup"><span data-stu-id="42e61-113">The compiler will use `greeting` as the argument to extension method `Print`.</span></span>  
  
```vb  
Dim greeting As String = "Hello"  
greeting.Print()  
```
