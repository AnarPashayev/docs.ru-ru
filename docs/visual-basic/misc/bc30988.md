# <a name="type-or-with-expected"></a><span data-ttu-id="2cb16-102">Требуется тип или "With"</span><span class="sxs-lookup"><span data-stu-id="2cb16-102">Type or 'With' expected</span></span>

<span data-ttu-id="2cb16-103">При объявлении экземпляра класса за ключевым словом `New` должно следовать имя типа или `With`.</span><span class="sxs-lookup"><span data-stu-id="2cb16-103">When you declare an instance of a class, the `New` keyword must be followed by a type name or by `With`.</span></span> <span data-ttu-id="2cb16-104">Например, в следующих инструкциях объявляется `client` как экземпляр класса `Customer` .</span><span class="sxs-lookup"><span data-stu-id="2cb16-104">For example, the following statements each declare `client` to be an instance of the `Customer` class.</span></span> <span data-ttu-id="2cb16-105">Имя типа `Customer` , следующее за `New`.</span><span class="sxs-lookup"><span data-stu-id="2cb16-105">The type name `Customer` follows `New`.</span></span>  
  
```vb  
' Dim client As New Customer()  
' The next declaration uses an object initializer.  
Dim client As New Customer() With {.Name = "Litware, Inc."}  
```  
  
 <span data-ttu-id="2cb16-106">Начиная с Visual Basic 2008, можно объявить объект как экземпляр анонимного типа. в этом случае тип данных указывать не нужно.</span><span class="sxs-lookup"><span data-stu-id="2cb16-106">Beginning with Visual Basic 2008, you can declare an object to be an instance of an anonymous type, in which case you do not specify a data type.</span></span> <span data-ttu-id="2cb16-107">В объявлениях анонимных типов за `With` следует ключевое слово `New`.</span><span class="sxs-lookup"><span data-stu-id="2cb16-107">In anonymous type declarations, the keyword `With` follows `New`.</span></span>  
  
```vb  
Dim person = New With {.Name ="Mike Nash", .Age = 27}  
```  
  
 <span data-ttu-id="2cb16-108">**Идентификатор ошибки:** BC30988</span><span class="sxs-lookup"><span data-stu-id="2cb16-108">**Error ID:** BC30988</span></span>
