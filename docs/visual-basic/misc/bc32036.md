<span data-ttu-id="80df3-103">Производный класс не объявляет конструктор и Visual Basic не может создать его, так как он не может определить, какой конструктор базового класса для вызова.</span><span class="sxs-lookup"><span data-stu-id="80df3-103">A derived class does not declare a constructor, and Visual Basic cannot generate one because it cannot determine which base class constructor to call.</span></span>  
  
 <span data-ttu-id="80df3-104">Если производный класс не объявляет конструктор, Visual Basic пытается создать неявный конструктор без параметров, который вызывает `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="80df3-104">When a derived class does not declare a constructor, Visual Basic attempts to generate an implicit parameterless constructor that calls `MyBase.New()`.</span></span> <span data-ttu-id="80df3-105">Если нет доступного конструктора в базовом классе, которые могут вызываться без аргументов или если имеется более одного, Visual Basic не удается создать неявный конструктор.</span><span class="sxs-lookup"><span data-stu-id="80df3-105">If there is no accessible constructor in the base class that can be called without arguments, or if there is more than one, Visual Basic cannot generate an implicit constructor.</span></span>  
  
 <span data-ttu-id="80df3-106">Эта ситуация может возникнуть, например, если один конструктор базового класса содержит один аргумент `Optional` , а другой — один аргумент `ParamArray` .</span><span class="sxs-lookup"><span data-stu-id="80df3-106">This situation can arise, for example, if one base class constructor has a single `Optional` argument and another has a single `ParamArray` argument.</span></span> <span data-ttu-id="80df3-107">Каждый из них можно вызвать без аргументов.</span><span class="sxs-lookup"><span data-stu-id="80df3-107">Each of these can be called with no arguments.</span></span>
