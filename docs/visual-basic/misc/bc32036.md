# <a name="class-classname1-must-declare-a-sub-new-because-its-base-class-classname2-has-more-than-one-accessible-sub-new-that-can-be-called-with-no-arguments"></a><span data-ttu-id="84b94-102">Класс "\<classname1>" должен объявить "Sub New", так как его базовый класс "\<classname2>" имеет более чем один доступный "Sub New", который может быть вызван без аргументов</span><span class="sxs-lookup"><span data-stu-id="84b94-102">Class '\<classname1>' must declare a 'Sub New' because its base class '\<classname2>' has more than one accessible 'Sub New' that can be called with no arguments</span></span>

<span data-ttu-id="84b94-103">Производный класс не объявляет конструктор, и Visual Basic не может создать его, поскольку он не может определить, какой конструктор базового класса следует вызвать.</span><span class="sxs-lookup"><span data-stu-id="84b94-103">A derived class does not declare a constructor, and Visual Basic cannot generate one because it cannot determine which base class constructor to call.</span></span>  
  
 <span data-ttu-id="84b94-104">Если производный класс не объявляет конструктор, Visual Basic пытается создать неявный конструктор без параметров, который вызывает `MyBase.New()` .</span><span class="sxs-lookup"><span data-stu-id="84b94-104">When a derived class does not declare a constructor, Visual Basic attempts to generate an implicit parameterless constructor that calls `MyBase.New()`.</span></span> <span data-ttu-id="84b94-105">Если в базовом классе нет доступного конструктора, который может вызываться без аргументов, или если имеется более одного, Visual Basic не может создать неявный конструктор.</span><span class="sxs-lookup"><span data-stu-id="84b94-105">If there is no accessible constructor in the base class that can be called without arguments, or if there is more than one, Visual Basic cannot generate an implicit constructor.</span></span>  
  
 <span data-ttu-id="84b94-106">Эта ситуация может возникнуть, например, если один конструктор базового класса содержит один аргумент `Optional` , а другой — один аргумент `ParamArray` .</span><span class="sxs-lookup"><span data-stu-id="84b94-106">This situation can arise, for example, if one base class constructor has a single `Optional` argument and another has a single `ParamArray` argument.</span></span> <span data-ttu-id="84b94-107">Каждый из них можно вызвать без аргументов.</span><span class="sxs-lookup"><span data-stu-id="84b94-107">Each of these can be called with no arguments.</span></span>  
  
 <span data-ttu-id="84b94-108">**Идентификатор ошибки:** BC32036</span><span class="sxs-lookup"><span data-stu-id="84b94-108">**Error ID:** BC32036</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="84b94-109">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="84b94-109">To correct this error</span></span>  
  
1. <span data-ttu-id="84b94-110">Объявите и реализуйте по крайней мере один конструктор `Sub New` где-нибудь в производном классе.</span><span class="sxs-lookup"><span data-stu-id="84b94-110">Declare and implement at least one `Sub New` constructor somewhere in the derived class.</span></span>  
  
2. <span data-ttu-id="84b94-111">Добавьте вызов конструктора базового класса `MyBase.New()`в качестве первой строки каждого `Sub New`.</span><span class="sxs-lookup"><span data-stu-id="84b94-111">Add a call to a base class constructor, `MyBase.New()`, as the first line of every `Sub New`.</span></span>
