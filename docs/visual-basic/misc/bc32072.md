# <a name="cannot-implement-interface-interfacename1-because-its-implementation-could-conflict-with-the-implementation-of-another-implemented-interface-interfacename2-for-some-type-arguments"></a><span data-ttu-id="8f8a7-102">Невозможно реализовать интерфейс '\<имя_интерфейса1 >', поскольку может вызвать конфликт с реализацией другого существующего интерфейса '\<имя_интерфейса2 >' для некоторых аргументов типа</span><span class="sxs-lookup"><span data-stu-id="8f8a7-102">Cannot implement interface '\<interfacename1>' because its implementation could conflict with the implementation of another implemented interface '\<interfacename2>' for some type arguments</span></span>

<span data-ttu-id="8f8a7-103">Объявление класса включает инструкцию `Implements` , которая указывает два или более интерфейсов, но по крайней мере один из интерфейсов является универсальным, и две реализации могут конфликтовать для определенных значений аргументов типа.</span><span class="sxs-lookup"><span data-stu-id="8f8a7-103">A class declaration includes an `Implements` statement that specifies two or more interfaces, but at least one of the interfaces is generic and two of the implementations could conflict for certain values of type arguments.</span></span>

<span data-ttu-id="8f8a7-104">Эта ошибка может быть вызвана следующими инструкциями.</span><span class="sxs-lookup"><span data-stu-id="8f8a7-104">The following statements can generate this error.</span></span>

```vb
Public Interface iFace1
    Sub testSub(ByVal arg As String)
End Interface
Public Interface iFace2(Of t)
    Sub testSub(ByVal arg As t)
End Interface
Public Class testClass
    Implements iFace1, iFace2(Of String)
End Class
```

<span data-ttu-id="8f8a7-105">Так как `iFace2` формируется с помощью `String`, `testClass` должен реализовать две версии `testSub` с одинаковыми сигнатурами.</span><span class="sxs-lookup"><span data-stu-id="8f8a7-105">Because `iFace2` is constructed using `String`, `testClass` must implement two versions of `testSub` with identical signatures.</span></span> <span data-ttu-id="8f8a7-106">Это приведет к неоднозначности определения, к какой версии следует получить доступ.</span><span class="sxs-lookup"><span data-stu-id="8f8a7-106">Doing so would produce an ambiguity about which version to access.</span></span>

<span data-ttu-id="8f8a7-107">**Идентификатор ошибки:** BC32072</span><span class="sxs-lookup"><span data-stu-id="8f8a7-107">**Error ID:** BC32072</span></span>

## <a name="to-correct-this-error"></a><span data-ttu-id="8f8a7-108">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="8f8a7-108">To correct this error</span></span>

- <span data-ttu-id="8f8a7-109">Измените аргумент типа, предоставленный универсальному интерфейсу, чтобы не было конфликтов.</span><span class="sxs-lookup"><span data-stu-id="8f8a7-109">Change the type argument supplied to the generic interface so that there is no conflict.</span></span>
