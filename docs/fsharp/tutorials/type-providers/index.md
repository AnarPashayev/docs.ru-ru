# <a name="type-providers"></a><span data-ttu-id="a695b-103">Поставщики типов</span><span class="sxs-lookup"><span data-stu-id="a695b-103">Type Providers</span></span>

<span data-ttu-id="a695b-104">Поставщик типов F# — это компонент, предоставляющий типы, свойства и методы для использования в программе.</span><span class="sxs-lookup"><span data-stu-id="a695b-104">An F# type provider is a component that provides types, properties, and methods for use in your program.</span></span> <span data-ttu-id="a695b-105">Поставщики типов формируют, что известно как **предоставленные типы**, создаваемые F# компилятором и основанные на внешнем источнике данных.</span><span class="sxs-lookup"><span data-stu-id="a695b-105">Type Providers generate what are known as **Provided Types**, which are generated by the F# compiler and are based on an external data source.</span></span>

<span data-ttu-id="a695b-106">Например, поставщик F# типов для SQL может формировать типы, представляющие таблицы и столбцы в реляционной базе данных.</span><span class="sxs-lookup"><span data-stu-id="a695b-106">For example, an F# Type Provider for SQL can generate types representing tables and columns in a relational database.</span></span> <span data-ttu-id="a695b-107">Фактически, это именно то, что делает поставщик типов [дескриптора SqlProvider](https://fsprojects.github.io/SQLProvider/) .</span><span class="sxs-lookup"><span data-stu-id="a695b-107">In fact, this is what the [SQLProvider](https://fsprojects.github.io/SQLProvider/) Type Provider does.</span></span>

<span data-ttu-id="a695b-108">Предоставленные типы зависят от входных параметров от поставщика типов.</span><span class="sxs-lookup"><span data-stu-id="a695b-108">Provided Types depend on input parameters to a Type Provider.</span></span> <span data-ttu-id="a695b-109">Такие входные данные могут представлять собой образец источника данных (например, файл схемы JSON), URL-адрес, указывающий непосредственно на внешнюю службу или строку подключения к источнику данных.</span><span class="sxs-lookup"><span data-stu-id="a695b-109">Such input can be a sample data source (such as a JSON schema file), a URL pointing directly to an external service, or a connection string to a data source.</span></span> <span data-ttu-id="a695b-110">Поставщик типов также может гарантировать, что группы типов развертываются только по запросу; то есть они развертываются, если в программе на самом деле ссылаются ваши типы.</span><span class="sxs-lookup"><span data-stu-id="a695b-110">A Type Provider can also ensure that groups of types are only expanded on demand; that is, they are expanded if the types are actually referenced by your program.</span></span> <span data-ttu-id="a695b-111">Это позволяет применять прямую интеграцию по запросу широкомасштабных информационных пространств (например, рынков оперативных данных) в строго типизированном виде.</span><span class="sxs-lookup"><span data-stu-id="a695b-111">This allows for the direct, on-demand integration of large-scale information spaces such as online data markets in a strongly typed way.</span></span>

## <a name="generative-and-erased-type-providers"></a><span data-ttu-id="a695b-112">Поставщики регенеративного и стирания типов</span><span class="sxs-lookup"><span data-stu-id="a695b-112">Generative and Erased Type Providers</span></span>

<span data-ttu-id="a695b-113">Поставщики типов бывают двух видов: регенеративные и стертые.</span><span class="sxs-lookup"><span data-stu-id="a695b-113">Type Providers come in two forms: Generative and Erased.</span></span>

<span data-ttu-id="a695b-114">Поставщики типов регенератив создают типы, которые могут быть записаны как типы .NET в сборку, в которой они создаются.</span><span class="sxs-lookup"><span data-stu-id="a695b-114">Generative Type Providers produce types that can be written as .NET types into the assembly in which they are produced.</span></span> <span data-ttu-id="a695b-115">Это позволяет использовать их из кода в других сборках.</span><span class="sxs-lookup"><span data-stu-id="a695b-115">This allows them to be consumed from code in other assemblies.</span></span> <span data-ttu-id="a695b-116">Это означает, что типизированное представление источника данных обычно должно иметь тип, который можно представить с помощью типов .NET.</span><span class="sxs-lookup"><span data-stu-id="a695b-116">This means that the typed representation of the data source must generally be one that is feasible to represent with .NET types.</span></span>

<span data-ttu-id="a695b-117">При удалении поставщиков типов создаются типы, которые можно использовать только в сборке или проекте, из которых они созданы.</span><span class="sxs-lookup"><span data-stu-id="a695b-117">Erasing Type Providers produce types that can only be consumed in the assembly or project they are generated from.</span></span> <span data-ttu-id="a695b-118">Типы являются временными; то есть они не записываются в сборку и не могут использоваться кодом в других сборках.</span><span class="sxs-lookup"><span data-stu-id="a695b-118">The types are ephemeral; that is, they are not written into an assembly and cannot be consumed by code in other assemblies.</span></span> <span data-ttu-id="a695b-119">Они могут содержать *отложенные* члены, что позволяет использовать предоставленные типы из потенциально бесконечного пространства сведений.</span><span class="sxs-lookup"><span data-stu-id="a695b-119">They can contain *delayed* members, allowing you to use provided types from a potentially infinite information space.</span></span> <span data-ttu-id="a695b-120">Они полезны при использовании небольшого подмножества большого и взаимосвязанных источников данных.</span><span class="sxs-lookup"><span data-stu-id="a695b-120">They are useful for using a small subset of a large and interconnected data source.</span></span>
