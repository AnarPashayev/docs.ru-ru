## <a name="how-to-start-async-code-in-f"></a><span data-ttu-id="57b94-123">Запуск асинхронного кода на языке f #\#</span><span class="sxs-lookup"><span data-stu-id="57b94-123">How to start Async Code in F\#</span></span>

<span data-ttu-id="57b94-124">Как уже упоминалось, асинхронный код представляет собой спецификацию должны быть выполнены в другом контексте, который должен быть явным образом запущено.</span><span class="sxs-lookup"><span data-stu-id="57b94-124">As mentioned earlier, async code is a specification of work to be done in another context which needs to be explicitly started.</span></span> <span data-ttu-id="57b94-125">Ниже приведены два основных способа выполнения этой задачи:</span><span class="sxs-lookup"><span data-stu-id="57b94-125">Here are two primary ways to accomplish this:</span></span>

1. <span data-ttu-id="57b94-126">`Async.RunSynchronously` запускается поток async в другом потоке и ожидает его результат.</span><span class="sxs-lookup"><span data-stu-id="57b94-126">`Async.RunSynchronously` will start an async workflow on another thread and await its result.</span></span>

### <a name="a-note-on-threads"></a><span data-ttu-id="57b94-130">Примечание о потоках</span><span class="sxs-lookup"><span data-stu-id="57b94-130">A Note on Threads</span></span>

<span data-ttu-id="57b94-131">Фраза «в другом потоке» упомянутой выше, но очень важно знать, что **это не означает, что асинхронных рабочих потоков являются фасадом для многопоточности**.</span><span class="sxs-lookup"><span data-stu-id="57b94-131">The phrase "on another thread" is mentioned above, but it is important to know that **this does not mean that async workflows are a facade for multithreading**.</span></span> <span data-ttu-id="57b94-132">Рабочий процесс фактически «переходит» между потоками, заимствуя их для небольшой объем выполнять полезную работу.</span><span class="sxs-lookup"><span data-stu-id="57b94-132">The workflow actually "jumps" between threads, borrowing them for a small amount of time to do useful work.</span></span> <span data-ttu-id="57b94-133">Когда поток async эффективно «ожидает» (например, ожидания сетевого вызова, чтобы он возвращал нечто), любой поток, который он займов во время освобождается до перейдите выполнять полезную работу на что-то еще.</span><span class="sxs-lookup"><span data-stu-id="57b94-133">When an async workflow is effectively "waiting" (for example, waiting for a network call to return something), any thread it was borrowing at the time is freed up to go do useful work on something else.</span></span> <span data-ttu-id="57b94-134">Это позволяет асинхронных рабочих потоков для использования системы, в которой они выполняются так же эффективно и делает их особенно высоки сценариях с большим количеством операций ввода-вывода.</span><span class="sxs-lookup"><span data-stu-id="57b94-134">This allows async workflows to utilize the system they run on as effectively as possible, and makes them especially strong for high-volume I/O scenarios.</span></span>

## <a name="how-to-add-parallelism-to-async-code"></a><span data-ttu-id="57b94-135">Добавление параллелизма в асинхронный код</span><span class="sxs-lookup"><span data-stu-id="57b94-135">How to Add Parallelism to Async Code</span></span>

<span data-ttu-id="57b94-136">Иногда можно требуется для выполнения нескольких асинхронных заданий в параллельном режиме, собирать их результаты и интерпретировать их каким-либо образом.</span><span class="sxs-lookup"><span data-stu-id="57b94-136">Sometimes you may need to perform multiple asynchronous jobs in parallel, collect their results, and interpret them in some way.</span></span> <span data-ttu-id="57b94-137">`Async.Parallel` позволяет это сделать без необходимости использования библиотеки параллельных задач, на которой будет включать в себя необходимости приводимое `Task<'T>` и `Async<'T>` типов.</span><span class="sxs-lookup"><span data-stu-id="57b94-137">`Async.Parallel` allows you to do this without needing to use the Task Parallel Library, which would involve needing to coerce `Task<'T>` and `Async<'T>` types.</span></span>

<span data-ttu-id="57b94-138">Следующий пример будет использоваться `Async.Parallel` Чтобы загрузить HTML-код из четырех популярных сайтов в параллельном режиме, дождитесь завершения выполнения этих задач и затем напечатайте HTML, который был загружен.</span><span class="sxs-lookup"><span data-stu-id="57b94-138">The following example will use `Async.Parallel` to download the HTML from four popular sites in parallel, wait for those tasks to complete, and then print the HTML which was downloaded.</span></span>

```fsharp
open System
open System.Net

let urlList =
    [ "https://www.microsoft.com"
      "https://www.google.com"
      "https://www.amazon.com"
      "https://www.facebook.com" ]

let fetchHtmlAsync url =
    async {
        let uri = Uri(url)
        use webClient = new WebClient()
        let! html = webClient.AsyncDownloadString(uri)
        return html
    }

let getHtmlList urls =
    urls
    |> Seq.map fetchHtmlAsync   // Build an Async<'T> for each site
    |> Async.Parallel           // Returns an Async<'T []>
    |> Async.RunSynchronously   // Wait for the result of the parallel work

let htmlList = getHtmlList urlList

// We now have the downloaded HTML for each site!
for html in htmlList do
    printfn "%s" html
```

  <span data-ttu-id="57b94-164">В отличие от `await`, которой могут быть вложенными неопределенно долгое время, `let!` не может и должен иметь результат привязан перед его использованием внутрь другой `let!`, `do!`, или `use!`.</span><span class="sxs-lookup"><span data-stu-id="57b94-164">Unlike `await`, which can be nested indefinitely, `let!` cannot and must have its result bound before using it inside of another `let!`, `do!`, or `use!`.</span></span>

* <span data-ttu-id="57b94-165">Поддержка отмены проще в F# чем в C#/VB.</span><span class="sxs-lookup"><span data-stu-id="57b94-165">Cancellation support is simpler in F# than in C#/VB.</span></span>

  <span data-ttu-id="57b94-166">Поддержка отмены задачи во время его выполнения C#и Visual Basic требуется выполнить проверку `IsCancellationRequested` свойства или вызов `ThrowIfCancellationRequested()` на `CancellationToken` объект, который передается в асинхронный метод.</span><span class="sxs-lookup"><span data-stu-id="57b94-166">Supporting cancellation of a task midway through its execution in C#/VB requires checking the `IsCancellationRequested` property or calling `ThrowIfCancellationRequested()` on a `CancellationToken` object that’s passed into the async method.</span></span>

<span data-ttu-id="57b94-167">Напротив F# более естественным образом будет поддерживать отмену асинхронных рабочих потоков.</span><span class="sxs-lookup"><span data-stu-id="57b94-167">In contrast, F# async workflows are more naturally cancellable.</span></span> <span data-ttu-id="57b94-168">Отмена является простой трехшаговый процесс.</span><span class="sxs-lookup"><span data-stu-id="57b94-168">Cancellation is a simple three-step process.</span></span>

1. <span data-ttu-id="57b94-169">Создайте объект `CancellationTokenSource`.</span><span class="sxs-lookup"><span data-stu-id="57b94-169">Create a new `CancellationTokenSource`.</span></span>
2. <span data-ttu-id="57b94-170">Передайте его в функция запуска.</span><span class="sxs-lookup"><span data-stu-id="57b94-170">Pass it into a starting function.</span></span>
3. <span data-ttu-id="57b94-171">Вызовите `Cancel` в маркере.</span><span class="sxs-lookup"><span data-stu-id="57b94-171">Call `Cancel` on the token.</span></span>

<span data-ttu-id="57b94-172">Пример</span><span class="sxs-lookup"><span data-stu-id="57b94-172">Example:</span></span>

```fsharp
open System.Threading

// Create a workflow which will loop forever.
let workflow =
    async {
        while true do
            printfn "Working..."
            do! Async.Sleep 1000
    }

let tokenSource = new CancellationTokenSource()

// Start the workflow in the background
Async.Start (workflow, tokenSource.Token)

// Executing the next line will stop the workflow
tokenSource.Cancel()
```
