## <a name="remarks"></a><span data-ttu-id="1682e-107">Примечания</span><span class="sxs-lookup"><span data-stu-id="1682e-107">Remarks</span></span>

<span data-ttu-id="1682e-108">Перечисление выглядит очень похожим это размеченное объединение, имеющее простые значения, за исключением того, что значения могут быть указаны.</span><span class="sxs-lookup"><span data-stu-id="1682e-108">An enumeration looks much like a discriminated union that has simple values, except that the values can be specified.</span></span> <span data-ttu-id="1682e-109">Значения обычно являются целыми числами, которые начинаются с 0 или 1 или целых чисел, представляющих позиции битов.</span><span class="sxs-lookup"><span data-stu-id="1682e-109">The values are typically integers that start at 0 or 1, or integers that represent bit positions.</span></span> <span data-ttu-id="1682e-110">Если перечисление предназначен для представления позиции битов, также следует использовать [флаги](xref:System.FlagsAttribute) атрибута.</span><span class="sxs-lookup"><span data-stu-id="1682e-110">If an enumeration is intended to represent bit positions, you should also use the [Flags](xref:System.FlagsAttribute) attribute.</span></span>

<span data-ttu-id="1682e-111">Базовый тип перечисления определяется из литерал, который используется, чтобы, например, можно использовать литералы с суффиксами, такие как `1u`, `2u`, и т. д., для целого числа без знака (`uint32`) типа.</span><span class="sxs-lookup"><span data-stu-id="1682e-111">The underlying type of the enumeration is determined from the literal that is used, so that, for example, you can use literals with a suffix, such as `1u`, `2u`, and so on, for an unsigned integer (`uint32`) type.</span></span>

<span data-ttu-id="1682e-112">При ссылке на именованные значения, необходимо использовать имя самого типа перечисления как квалификатор, то есть `enum-name.value1`, а не только `value1`.</span><span class="sxs-lookup"><span data-stu-id="1682e-112">When you refer to the named values, you must use the name of the enumeration type itself as a qualifier, that is, `enum-name.value1`, not just `value1`.</span></span> <span data-ttu-id="1682e-113">Это поведение отличается от размеченных объединений.</span><span class="sxs-lookup"><span data-stu-id="1682e-113">This behavior differs from that of discriminated unions.</span></span> <span data-ttu-id="1682e-114">Это обусловлено перечисления всегда [RequireQualifiedAccess](https://msdn.microsoft.com/library/8b9b6ade-0471-4413-ac5d-638cd0de5f15) атрибута.</span><span class="sxs-lookup"><span data-stu-id="1682e-114">This is because enumerations always have the [RequireQualifiedAccess](https://msdn.microsoft.com/library/8b9b6ade-0471-4413-ac5d-638cd0de5f15) attribute.</span></span>
