## <a name="remarks"></a>Примечания

Строки Юникода могут содержать явные кодировки, которые можно указать с помощью `\u` следуют 16-разрядный шестнадцатеричный код или кодировки UTF-32, можно указать с помощью `\U` следуют 32-разрядных шестнадцатеричный код, который представляет Юникода пары символов-заместителей.

Начиная с версии F# 3.1, можно использовать `+` входа для объединения строковых литералов. Можно также использовать побитового или (`|||`) оператор, чтобы объединить флаги перечисления. Например, следующий код является допустимым в F# 3.1:

```fsharp
[<Literal>]
let Literal1 = "a" + "b"

[<Literal>]
let FileLocation =   __SOURCE_DIRECTORY__ + "/" + __SOURCE_FILE__

[<Literal>]
let Literal2 = 1 ||| 64

[<Literal>]
let Literal3 = System.IO.FileAccess.Read ||| System.IO.FileAccess.Write
```

Использование других побитовых операторов запрещено.

## <a name="named-literals"></a>Именованные литералы

Значения, которые предназначены быть константами могут маркироваться [литерала](https://msdn.microsoft.com/library/465f36ce-d146-41c0-b425-679c509cd285) атрибута. Этот атрибут имеет вследствие значение компилируется как константа.

В регулярных выражениях идентификаторов, начинающихся с символов нижнего регистра всегда обрабатываются как переменные для привязки, а не как литералы, поэтому следует использовать прописные буквы при определении литералов.

## <a name="integers-in-other-bases"></a>Целые числа в других базовых классов

32-разрядных целых чисел со знаком, также может быть указан при шестнадцатеричные, восьмеричные и двоичные `0x`, `0o` или `0b` префикса соответственно.

```fsharp
let numbers = (0x9F, 0o77, 0b1010)
// Result: numbers : int * int * int = (159, 63, 10)
```

## <a name="underscores-in-numeric-literals"></a>Символы подчеркивания в числовых литералах

Начиная с F# 4.1, можно отделить цифр с символа подчеркивания (`_`).

```fsharp
let value = 0xDEAD_BEEF

let valueAsBits = 0b1101_1110_1010_1101_1011_1110_1110_1111

let exampleSSN = 123_456_7890
```
