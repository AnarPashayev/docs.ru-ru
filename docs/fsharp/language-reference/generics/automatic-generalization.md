## <a name="value-restriction"></a><span data-ttu-id="7d2f0-122">Ограничение значения</span><span class="sxs-lookup"><span data-stu-id="7d2f0-122">Value Restriction</span></span>

<span data-ttu-id="7d2f0-123">Компилятор выполняет Автоматическое обобщение только для полных определений функций, которые имеют явные аргументы и для простых постоянных значений.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-123">The compiler performs automatic generalization only on complete function definitions that have explicit arguments, and on simple immutable values.</span></span>

<span data-ttu-id="7d2f0-124">Это означает, что компилятор выдает ошибку при попытке скомпилировать код, достаточно не обязательно должен быть определенного типа, но не также обобщению.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-124">This means that the compiler issues an error if you try to compile code that is not sufficiently constrained to be a specific type, but is also not generalizable.</span></span> <span data-ttu-id="7d2f0-125">Сообщение об ошибке для этой проблемы ссылается на это ограничение на Автоматическое обобщение для значения, что *ограничение значения*.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-125">The error message for this problem refers to this restriction on automatic generalization for values as the *value restriction*.</span></span>

<span data-ttu-id="7d2f0-126">Как правило ошибки ограничения значений происходит, если требуется, чтобы конструкцию как универсальный, но компилятор не получает достаточно информации, чтобы обобщить его или при достаточной информации о типе в неуниверсальной конструкции непреднамеренно опускаются.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-126">Typically, the value restriction error occurs either when you want a construct to be generic but the compiler has insufficient information to generalize it, or when you unintentionally omit sufficient type information in a nongeneric construct.</span></span> <span data-ttu-id="7d2f0-127">Решения для ошибки ограничения значений — для предоставления более явную информацию для более полного устранения проблемы вывода типа, в одном из следующих способов:</span><span class="sxs-lookup"><span data-stu-id="7d2f0-127">The solution to the value restriction error is to provide more explicit information to more fully constrain the type inference problem, in one of the following ways:</span></span>

- <span data-ttu-id="7d2f0-128">Сделайте тип неуниверсальным, добавив заметку явный тип значения или параметра.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-128">Constrain a type to be nongeneric by adding an explicit type annotation to a value or parameter.</span></span>

- <span data-ttu-id="7d2f0-129">Если проблемы с помощью конструкции необобщаемой для определения универсальной функции, такие как объединения функций или неполностью примененных каррированных аргументов, попробуйте переписать эту функцию определение обычной функции.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-129">If the problem is using a nongeneralizable construct to define a generic function, such as a function composition or incompletely applied curried function arguments, try to rewrite the function as an ordinary function definition.</span></span>

- <span data-ttu-id="7d2f0-130">Если проблема заключается выражение, которое слишком сложен, чтобы подготовить к использованию, когда-нибудь функцию, добавив дополнительный неиспользуемый параметр.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-130">If the problem is an expression that is too complex to be generalized, make it into a function by adding an extra, unused parameter.</span></span>

- <span data-ttu-id="7d2f0-131">Добавьте параметры явной универсального типа.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-131">Add explicit generic type parameters.</span></span> <span data-ttu-id="7d2f0-132">Этот параметр используется редко.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-132">This option is rarely used.</span></span>

<span data-ttu-id="7d2f0-134">Вариант 1. Слишком сложное выражение.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-134">Case 1: Too complex an expression.</span></span> <span data-ttu-id="7d2f0-135">В этом примере список `counter` должен быть `int option ref`, но он не определен как простое постоянное значение.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-135">In this example, the list `counter` is intended to be `int option ref`, but it is not defined as a simple immutable value.</span></span>

<span data-ttu-id="7d2f0-136">Вариант 2. <span data-ttu-id="7d2f0-138">Вариант 3. <span data-ttu-id="7d2f0-140">Вариант 4. Добавление параметров типа.</span><span class="sxs-lookup"><span data-stu-id="7d2f0-140">Case 4: Adding type parameters.</span></span>

<span data-ttu-id="7d2f0-141">В последнем случае значение становится тип функции, который может использоваться для создания значения разных типов, например следующим образом:</span><span class="sxs-lookup"><span data-stu-id="7d2f0-141">In the last case, the value becomes a type function, which may be used to create values of many different types, for example as follows:</span></span>
