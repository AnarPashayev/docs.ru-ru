# <a name="perform-inner-joins"></a><span data-ttu-id="58730-103">Выполнение внутренних соединений</span><span class="sxs-lookup"><span data-stu-id="58730-103">Perform inner joins</span></span>

<span data-ttu-id="58730-104">В терминах реляционных баз данных *внутреннее соединение* формирует результирующий набор, в котором каждый элемент первой коллекции отображается по одному разу для каждого соответствующего элемента во второй коллекции.</span><span class="sxs-lookup"><span data-stu-id="58730-104">In relational database terms, an *inner join* produces a result set in which each element of the first collection appears one time for every matching element in the second collection.</span></span> <span data-ttu-id="58730-105">Если для элемента в первой коллекции нет соответствующих элементов, он не отображается в результирующем наборе.</span><span class="sxs-lookup"><span data-stu-id="58730-105">If an element in the first collection has no matching elements, it does not appear in the result set.</span></span> <span data-ttu-id="58730-106">Метод <xref:System.Linq.Enumerable.Join%2A>, который вызывается предложением `join` в C#, реализует внутреннее соединение.</span><span class="sxs-lookup"><span data-stu-id="58730-106">The <xref:System.Linq.Enumerable.Join%2A> method, which is called by the `join` clause in C#, implements an inner join.</span></span>

<span data-ttu-id="58730-107">В этой статье показано выполнение четырех видов внутреннего соединения:</span><span class="sxs-lookup"><span data-stu-id="58730-107">This article shows you how to perform four variations of an inner join:</span></span>

- <span data-ttu-id="58730-108">Простое внутреннее соединение, сопоставляющее элементы из двух источников данных на основе простого ключа.</span><span class="sxs-lookup"><span data-stu-id="58730-108">A simple inner join that correlates elements from two data sources based on a simple key.</span></span>

- <span data-ttu-id="58730-109">Внутреннее соединение, сопоставляющее элементы из двух источников данных на основе *составного* ключа.</span><span class="sxs-lookup"><span data-stu-id="58730-109">An inner join that correlates elements from two data sources based on a *composite* key.</span></span> <span data-ttu-id="58730-110">Составной ключ — это ключ, который состоит из нескольких значений, позволяющих сопоставлять элементы на основе сразу нескольких свойств.</span><span class="sxs-lookup"><span data-stu-id="58730-110">A composite key, which is a key that consists of more than one value, enables you to correlate elements based on more than one property.</span></span>

- <span data-ttu-id="58730-111">*Множественное соединение*, при котором несколько последовательных операций соединения добавляются друг к другу.</span><span class="sxs-lookup"><span data-stu-id="58730-111">A *multiple join* in which successive join operations are appended to each other.</span></span>

- <span data-ttu-id="58730-112">Внутреннее соединение, реализуемое с помощью группового соединения.</span><span class="sxs-lookup"><span data-stu-id="58730-112">An inner join that is implemented by using a group join.</span></span>

<span data-ttu-id="58730-120">Вместо сопоставления элементов по одному единственному свойству можно воспользоваться составным ключом и сравнить элементы по нескольким свойствам.</span><span class="sxs-lookup"><span data-stu-id="58730-120">Instead of correlating elements based on just one property, you can use a composite key to compare elements based on multiple properties.</span></span> <span data-ttu-id="58730-121">Для этого задайте функцию селектора ключа для каждой коллекции, чтобы вернуть анонимный тип, состоящий из свойств, которые вы хотите сравнить.</span><span class="sxs-lookup"><span data-stu-id="58730-121">To do this, specify the key selector function for each collection to return an anonymous type that consists of the properties you want to compare.</span></span> <span data-ttu-id="58730-122">Если свойства помечаются, они должны иметь одинаковую метку в анонимном типе каждого ключа.</span><span class="sxs-lookup"><span data-stu-id="58730-122">If you label the properties, they must have the same label in each key's anonymous type.</span></span> <span data-ttu-id="58730-123">Кроме того, свойства должны отображаться в одинаковом порядке.</span><span class="sxs-lookup"><span data-stu-id="58730-123">The properties must also appear in the same order.</span></span>
