## <a name="query-expression-overview"></a><span data-ttu-id="b438d-120">Описание выражения запроса</span><span class="sxs-lookup"><span data-stu-id="b438d-120">Query expression overview</span></span>

- <span data-ttu-id="b438d-121">Выражение запроса можно использовать для получения и преобразования данных из любого источника данных, поддерживающего LINQ.</span><span class="sxs-lookup"><span data-stu-id="b438d-121">Query expressions can be used to query and to transform data from any LINQ-enabled data source.</span></span> <span data-ttu-id="b438d-122">Например, можно одним запросом получить данные из базы данных SQL и создать на их основе выходной XML-поток.</span><span class="sxs-lookup"><span data-stu-id="b438d-122">For example, a single query can retrieve data from a SQL database, and produce an XML stream as output.</span></span>
- <span data-ttu-id="b438d-123">Выражение запроса очень легко освоить, поскольку в нем используется много знакомых конструкций языка C#.</span><span class="sxs-lookup"><span data-stu-id="b438d-123">Query expressions are easy to master because they use many familiar C# language constructs.</span></span>
- <span data-ttu-id="b438d-124">Все переменные в выражениях запросов строго типизированы, хотя во многих случаях вам не нужно указывать тип явным образом, поскольку компилятор определит его автоматически.</span><span class="sxs-lookup"><span data-stu-id="b438d-124">The variables in a query expression are all strongly typed, although in many cases you do not have to provide the type explicitly because the compiler can infer it.</span></span> <span data-ttu-id="b438d-125">Дополнительные сведения см. в статье [Отношения между типами в операциях запросов LINQ (C#)](type-relationships-in-linq-query-operations.md).</span><span class="sxs-lookup"><span data-stu-id="b438d-125">For more information, see [Type relationships in LINQ query operations](type-relationships-in-linq-query-operations.md).</span></span>
- <span data-ttu-id="b438d-126">Запрос не будет выполняться, пока вы не начнете обращаться к переменной запроса, например, с помощью инструкции `foreach`.</span><span class="sxs-lookup"><span data-stu-id="b438d-126">A query is not executed until you iterate over the query variable, for example, in a `foreach` statement.</span></span> <span data-ttu-id="b438d-127">Дополнительные сведения см. в статье [Введение в запросы LINQ (C#)](introduction-to-linq-queries.md).</span><span class="sxs-lookup"><span data-stu-id="b438d-127">For more information, see [Introduction to LINQ queries](introduction-to-linq-queries.md).</span></span>
- <span data-ttu-id="b438d-128">Во время компиляции выражения запроса преобразуются в вызовы метода стандартного оператора запроса. - <span data-ttu-id="b438d-132">Мы рекомендуем при написании запросов LINQ использовать синтаксис запросов везде, где это возможно, а синтаксис метода — только если это совершенно необходимо.</span><span class="sxs-lookup"><span data-stu-id="b438d-132">As a rule when you write LINQ queries, we recommend that you use query syntax whenever possible and method syntax whenever necessary.</span></span> <span data-ttu-id="b438d-133">Между этими формами синтаксиса нет никакой разницы в семантике или производительности.</span><span class="sxs-lookup"><span data-stu-id="b438d-133">There is no semantic or performance difference between the two different forms.</span></span> <span data-ttu-id="b438d-134">Выражения запросов обычно более удобочитаемыми, чем аналогичные выражения с использованием синтаксиса метода.</span><span class="sxs-lookup"><span data-stu-id="b438d-134">Query expressions are often more readable than equivalent expressions written in method syntax.</span></span>
- <span data-ttu-id="b438d-135">Некоторых операции запросов, например <xref:System.Linq.Enumerable.Count%2A> или <xref:System.Linq.Enumerable.Max%2A>, не имеют эквивалентных предложений выражения для запросов и должны выражаться как вызовы методов.</span><span class="sxs-lookup"><span data-stu-id="b438d-135">Some query operations, such as <xref:System.Linq.Enumerable.Count%2A> or <xref:System.Linq.Enumerable.Max%2A>, have no equivalent query expression clause and must therefore be expressed as a method call.</span></span> <span data-ttu-id="b438d-136">Синтаксис запросов и синтаксис методов можно сочетать друг с другом различными способами.</span><span class="sxs-lookup"><span data-stu-id="b438d-136">Method syntax can be combined with query syntax in various ways.</span></span> <span data-ttu-id="b438d-137">Дополнительные сведения см. в статье [Синтаксис запросов и синтаксис методов в LINQ (C#)](query-syntax-and-method-syntax-in-linq.md).</span><span class="sxs-lookup"><span data-stu-id="b438d-137">For more information, see [Query syntax and method syntax in LINQ](query-syntax-and-method-syntax-in-linq.md).</span></span>
- <span data-ttu-id="b438d-138">Выражения запросов могут компилироваться в деревья выражений или в делегаты, в зависимости от типа, к которому применяется конкретный запрос.</span><span class="sxs-lookup"><span data-stu-id="b438d-138">Query expressions can be compiled to expression trees or to delegates, depending on the type that the query is applied to.</span></span> <span data-ttu-id="b438d-139">Запросы <xref:System.Collections.Generic.IEnumerable%601> компилируются в делегаты.</span><span class="sxs-lookup"><span data-stu-id="b438d-139"><xref:System.Collections.Generic.IEnumerable%601> queries are compiled to delegates.</span></span> <span data-ttu-id="b438d-140">Запросы <xref:System.Linq.IQueryable> и <xref:System.Linq.IQueryable%601> компилируются в деревья выражений.</span><span class="sxs-lookup"><span data-stu-id="b438d-140"><xref:System.Linq.IQueryable> and <xref:System.Linq.IQueryable%601> queries are compiled to expression trees.</span></span> <span data-ttu-id="b438d-141">Дополнительные сведения см. в статье [Деревья выражений](../../../expression-trees.md).</span><span class="sxs-lookup"><span data-stu-id="b438d-141">For more information, see [Expression trees](../../../expression-trees.md).</span></span>
