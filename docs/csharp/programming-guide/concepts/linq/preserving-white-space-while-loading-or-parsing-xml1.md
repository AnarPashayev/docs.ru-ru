# <a name="preserving-white-space-while-loading-or-parsing-xml"></a><span data-ttu-id="d4e2b-103">Сохранение пробелов при загрузке и анализе XML</span><span class="sxs-lookup"><span data-stu-id="d4e2b-103">Preserving White Space while Loading or Parsing XML</span></span>
<span data-ttu-id="d4e2b-104">В этой статье показан способ управления тем, как обрабатываются пробелы в [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)].</span><span class="sxs-lookup"><span data-stu-id="d4e2b-104">This topic describes how to control the white-space behavior of [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)].</span></span>  
  
 <span data-ttu-id="d4e2b-105">Типичный сценарий состоит в чтении XML с отступами, создании XML-дерева в памяти без текстовых узлов с пробелами (то есть без сохранения пробелов), выполнении определенных операций над XML и сохранении XML с отступами.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-105">A common scenario is to read indented XML, create an in-memory XML tree without any white space text nodes (that is, not preserving white space), perform some operations on the XML, and then save the XML with indentation.</span></span> <span data-ttu-id="d4e2b-106">При сериализации XML с форматированием сохраняются только значимые пробелы XML-дерева.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-106">When you serialize the XML with formatting, only significant white space in the XML tree is preserved.</span></span> <span data-ttu-id="d4e2b-107">Это поведение [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-107">This is the default behavior for [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)].</span></span>  
  
 <span data-ttu-id="d4e2b-108">Другой типичный сценарий заключается в чтении и изменении XML с уже существующими преднамеренными отступами.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-108">Another common scenario is to read and modify XML that has already been intentionally indented.</span></span> <span data-ttu-id="d4e2b-109">Эти отступы ни в коем случае изменять нельзя.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-109">You might not want to change this indentation in any way.</span></span> <span data-ttu-id="d4e2b-110">Для этого в [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)] можно сохранить пробелы при загрузке или синтаксическом анализе XML и отключить форматирование при сериализации XML.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-110">To do this in [!INCLUDE[sqltecxlinq](~/includes/sqltecxlinq-md.md)], you preserve white space when you load or parse the XML and disable formatting when you serialize the XML.</span></span>  
  
 <span data-ttu-id="d4e2b-111">В этой статье описывается, как методы, заполняющие XML-деревья, обрабатывают пробелы.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-111">This topic describes the white-space behavior of methods that populate XML trees.</span></span> <span data-ttu-id="d4e2b-112">Сведения об управлении пробелами при сериализации деревьев XML см. в разделе [Сохранение пробелов при сериализации](./preserving-white-space-while-serializing.md).</span><span class="sxs-lookup"><span data-stu-id="d4e2b-112">For information about controlling white space when you serialize XML trees, see [Preserving White Space While Serializing](./preserving-white-space-while-serializing.md).</span></span>

 <span data-ttu-id="d4e2b-114">Следующие методы в классах <xref:System.Xml.Linq.XElement> и <xref:System.Xml.Linq.XDocument> заполняют XML-дерево.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-114">The following methods in the <xref:System.Xml.Linq.XElement> and <xref:System.Xml.Linq.XDocument> classes populate an XML tree.</span></span> <span data-ttu-id="d4e2b-115">XML-дерево можно заполнить из файла, объекта <xref:System.IO.TextReader>, объекта <xref:System.Xml.XmlReader> или строки:</span><span class="sxs-lookup"><span data-stu-id="d4e2b-115">You can populate an XML tree from a file, a <xref:System.IO.TextReader>, an <xref:System.Xml.XmlReader>, or a string:</span></span>

- <xref:System.Xml.Linq.XElement.Parse%2A?displayProperty=nameWithType>

 <span data-ttu-id="d4e2b-117">В большинстве случаев, если метод принимает объект <xref:System.Xml.Linq.LoadOptions> в качестве аргумента, позволяет сохранять незначащие пробелы в XML-дереве как текстовые узлы.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-117">In most cases, if the method takes <xref:System.Xml.Linq.LoadOptions> as an argument, you can optionally preserve insignificant white space as text nodes in the XML tree.</span></span> <span data-ttu-id="d4e2b-118">Однако если метод загружает XML из объекта <xref:System.Xml.XmlReader>, то объект <xref:System.Xml.XmlReader> определяет, будут ли сохраняться пробелы или нет.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-118">However, if the method is loading the XML from an <xref:System.Xml.XmlReader>, then the <xref:System.Xml.XmlReader> determines whether white space will be preserved or not.</span></span> <span data-ttu-id="d4e2b-119">Задание параметра <xref:System.Xml.Linq.LoadOptions.PreserveWhitespace> не оказывает никакого воздействия.</span><span class="sxs-lookup"><span data-stu-id="d4e2b-119">Setting <xref:System.Xml.Linq.LoadOptions.PreserveWhitespace> will have no effect.</span></span>
