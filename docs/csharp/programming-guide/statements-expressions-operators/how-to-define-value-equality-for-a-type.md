# <a name="how-to-define-value-equality-for-a-type-c-programming-guide"></a><span data-ttu-id="ed34c-104">Руководство по программированию на C#. Как определить равенства значений для типа</span><span class="sxs-lookup"><span data-stu-id="ed34c-104">How to define value equality for a type (C# Programming Guide)</span></span>

<span data-ttu-id="ed34c-105">При определении класса или структуры необходимо решить, имеет ли смысл создавать пользовательское определение равенства значений (или эквивалентности) для этого типа.</span><span class="sxs-lookup"><span data-stu-id="ed34c-105">When you define a class or struct, you decide whether it makes sense to create a custom definition of value equality (or equivalence) for the type.</span></span> <span data-ttu-id="ed34c-106">Обычно правила определения равенства реализуются, если объекты этого типа будут добавляться в коллекции или если они в первую очередь предназначены для хранения набора полей или свойств.</span><span class="sxs-lookup"><span data-stu-id="ed34c-106">Typically, you implement value equality when objects of the type are expected to be added to a collection of some sort, or when their primary purpose is to store a set of fields or properties.</span></span> <span data-ttu-id="ed34c-107">В основу определения равенства значений можно положить сравнение всех полей и свойств в типе или только их части.</span><span class="sxs-lookup"><span data-stu-id="ed34c-107">You can base your definition of value equality on a comparison of all the fields and properties in the type, or you can base the definition on a subset.</span></span>

<span data-ttu-id="ed34c-108">В любом случае (как для классов, так и для структур) реализация должна соответствовать следующим пяти гарантиям эквивалентности (для следующих правил предполагается, что `x`, `y` и `z` не равны NULL).</span><span class="sxs-lookup"><span data-stu-id="ed34c-108">In either case, and in both classes and structs, your implementation should follow the five guarantees of equivalence (For the following rules, assume that `x`, `y` and `z` are not null):</span></span>  
  
1. <span data-ttu-id="ed34c-109">`x.Equals(x)` возвращает `true`.</span><span class="sxs-lookup"><span data-stu-id="ed34c-109">`x.Equals(x)` returns `true`.</span></span> <span data-ttu-id="ed34c-110">Это называется свойством рефлексивности.</span><span class="sxs-lookup"><span data-stu-id="ed34c-110">This is called the reflexive property.</span></span>  
  
2. <span data-ttu-id="ed34c-111">`x.Equals(y)` возвращает то же значение, что и `y.Equals(x)`.</span><span class="sxs-lookup"><span data-stu-id="ed34c-111">`x.Equals(y)` returns the same value as `y.Equals(x)`.</span></span> <span data-ttu-id="ed34c-112">Это называется свойством симметрии.</span><span class="sxs-lookup"><span data-stu-id="ed34c-112">This is called the symmetric property.</span></span>  
  
3. <span data-ttu-id="ed34c-113">Если `(x.Equals(y) && y.Equals(z))` возвращает `true`, `x.Equals(z)` возвращает `true`.</span><span class="sxs-lookup"><span data-stu-id="ed34c-113">if `(x.Equals(y) && y.Equals(z))` returns `true`, then `x.Equals(z)` returns `true`.</span></span> <span data-ttu-id="ed34c-114">Это называется свойством транзитивности.</span><span class="sxs-lookup"><span data-stu-id="ed34c-114">This is called the transitive property.</span></span>  
  
4. <span data-ttu-id="ed34c-115">Последовательные вызовы `x.Equals(y)` возвращают одно и то же значение до тех пор, пока объекты, на которые ссылаются x и y, не будут изменены.</span><span class="sxs-lookup"><span data-stu-id="ed34c-115">Successive invocations of `x.Equals(y)` return the same value as long as the objects referenced by x and y are not modified.</span></span>  
  
5. <span data-ttu-id="ed34c-116">Любое значение, отличное от NULL, не равно NULL.</span><span class="sxs-lookup"><span data-stu-id="ed34c-116">Any non-null value is not equal to null.</span></span> <span data-ttu-id="ed34c-117">Однако среда CLR проверяет наличие значений NULL для всех вызовов методов и выдает исключение `NullReferenceException`, если ссылка `this` будет равна NULL.</span><span class="sxs-lookup"><span data-stu-id="ed34c-117">However, the CLR checks for null on all method calls and throws a `NullReferenceException` if the `this` reference would be null.</span></span> <span data-ttu-id="ed34c-118">Поэтому `x.Equals(y)` выдает исключение, когда `x` имеет значение NULL.</span><span class="sxs-lookup"><span data-stu-id="ed34c-118">Therefore, `x.Equals(y)` throws an exception when `x` is null.</span></span> <span data-ttu-id="ed34c-119">Это нарушает правила 1 или 2 в зависимости от аргумента для `Equals`.</span><span class="sxs-lookup"><span data-stu-id="ed34c-119">That breaks rules 1 or 2, depending on the argument to `Equals`.</span></span>

<span data-ttu-id="ed34c-120">Любая определяемая вами структура имеет заданную по умолчанию реализацию равенства значений, которая наследуется от переопределения <xref:System.ValueType?displayProperty=nameWithType> метода <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="ed34c-120">Any struct that you define already has a default implementation of value equality that it inherits from the <xref:System.ValueType?displayProperty=nameWithType> override of the <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="ed34c-121">Эта реализация использует отражение для проверки всех полей и свойств в типе.</span><span class="sxs-lookup"><span data-stu-id="ed34c-121">This implementation uses reflection to examine all the fields and properties in the type.</span></span> <span data-ttu-id="ed34c-122">Хотя эта реализация возвращает верный результат, она отличается невысокой скоростью по сравнению с пользовательской реализацией, которую можно написать специально для конкретного типа.</span><span class="sxs-lookup"><span data-stu-id="ed34c-122">Although this implementation produces correct results, it is relatively slow compared to a custom implementation that you write specifically for the type.</span></span>  
  
<span data-ttu-id="ed34c-123">Детали реализации равенства значений для классов и структур различаются.</span><span class="sxs-lookup"><span data-stu-id="ed34c-123">The implementation details for value equality are different for classes and structs.</span></span> <span data-ttu-id="ed34c-124">Однако для реализации равенства как для классов, так и для структур, необходимо выполнить одни и те же базовые действия.</span><span class="sxs-lookup"><span data-stu-id="ed34c-124">However, both classes and structs require the same basic steps for implementing equality:</span></span>  
  
1. <span data-ttu-id="ed34c-125">Переопределите [виртуальный](../../language-reference/keywords/virtual.md) метод <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="ed34c-125">Override the [virtual](../../language-reference/keywords/virtual.md) <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="ed34c-126">В большинстве случаев пользовательская реализация `bool Equals( object obj )` должна вызывать относящийся к конкретному типу метод `Equals`, который является реализацией интерфейса <xref:System.IEquatable%601?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="ed34c-126">In most cases, your implementation of `bool Equals( object obj )` should just call into the type-specific `Equals` method that is the implementation of the <xref:System.IEquatable%601?displayProperty=nameWithType> interface.</span></span> <span data-ttu-id="ed34c-127">(См. шаг 2.)</span><span class="sxs-lookup"><span data-stu-id="ed34c-127">(See step 2.)</span></span>  
  
2. <span data-ttu-id="ed34c-128">Реализуйте интерфейс <xref:System.IEquatable%601?displayProperty=nameWithType>, предоставив метод `Equals` для конкретного типа.</span><span class="sxs-lookup"><span data-stu-id="ed34c-128">Implement the <xref:System.IEquatable%601?displayProperty=nameWithType> interface by providing a type-specific `Equals` method.</span></span> <span data-ttu-id="ed34c-129">Именно на этом этапе происходит фактическое сравнение значений.</span><span class="sxs-lookup"><span data-stu-id="ed34c-129">This is where the actual equivalence comparison is performed.</span></span> <span data-ttu-id="ed34c-130">Например, функцию равенства можно определить путем сравнения только одного из двух полей в типе.</span><span class="sxs-lookup"><span data-stu-id="ed34c-130">For example, you might decide to define equality by comparing only one or two fields in your type.</span></span> <span data-ttu-id="ed34c-131">Не вызывайте исключения из `Equals`.</span><span class="sxs-lookup"><span data-stu-id="ed34c-131">Do not throw exceptions from `Equals`.</span></span> <span data-ttu-id="ed34c-132">Только для классов: этот метод должен проверять только те поля, которые объявлены в классе.</span><span class="sxs-lookup"><span data-stu-id="ed34c-132">For classes only: This method should examine only fields that are declared in the class.</span></span> <span data-ttu-id="ed34c-133">Он должен вызывать метод `base.Equals` для проверки полей в базовом классе.</span><span class="sxs-lookup"><span data-stu-id="ed34c-133">It should call `base.Equals` to examine fields that are in the base class.</span></span> <span data-ttu-id="ed34c-134">(Не делайте этого, если тип наследуется напрямую от <xref:System.Object>, поскольку реализация <xref:System.Object> для <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> выполняет проверку равенства ссылок.)</span><span class="sxs-lookup"><span data-stu-id="ed34c-134">(Do not do this if the type inherits directly from <xref:System.Object>, because the <xref:System.Object> implementation of <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> performs a reference equality check.)</span></span>  
  
3. <span data-ttu-id="ed34c-135">Необязательно, но рекомендуется: перегрузите операторы [==](../../language-reference/operators/equality-operators.md#equality-operator-) и [!=](../../language-reference/operators/equality-operators.md#inequality-operator-).</span><span class="sxs-lookup"><span data-stu-id="ed34c-135">Optional but recommended: Overload the [==](../../language-reference/operators/equality-operators.md#equality-operator-) and [!=](../../language-reference/operators/equality-operators.md#inequality-operator-) operators.</span></span>  
  
4. <span data-ttu-id="ed34c-136">Переопределите <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType> таким образом, чтобы два объекта с равными значениями создавали одинаковый хэш-код.</span><span class="sxs-lookup"><span data-stu-id="ed34c-136">Override <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType> so that two objects that have value equality produce the same hash code.</span></span>  
  
5. <span data-ttu-id="ed34c-137">Необязательные: Для поддержки определений "больше" и "меньше" реализуйте для типа интерфейс <xref:System.IComparable%601>, а также перегрузите операторы [<=](../../language-reference/operators/comparison-operators.md#less-than-or-equal-operator-) и [>=](../../language-reference/operators/comparison-operators.md#greater-than-or-equal-operator-).</span><span class="sxs-lookup"><span data-stu-id="ed34c-137">Optional: To support definitions for "greater than" or "less than," implement the <xref:System.IComparable%601> interface for your type, and also overload the [<=](../../language-reference/operators/comparison-operators.md#less-than-or-equal-operator-) and [>=](../../language-reference/operators/comparison-operators.md#greater-than-or-equal-operator-) operators.</span></span>
