# <a name="nullable-types-c-programming-guide"></a><span data-ttu-id="16ef3-103">Типы, допускающие значение NULL (Руководство по программированию на C#)</span><span class="sxs-lookup"><span data-stu-id="16ef3-103">Nullable types (C# Programming Guide)</span></span>

<span data-ttu-id="16ef3-104">Типы, допускающие значения NULL, являются экземплярами структуры <xref:System.Nullable%601?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="16ef3-104">Nullable types are instances of the <xref:System.Nullable%601?displayProperty=nameWithType> struct.</span></span> <span data-ttu-id="16ef3-105">Типы, допускающие значение NULL, могут представлять все значения своего базового типа `T`, а также дополнительное значение [NULL](../../language-reference/keywords/null.md).</span><span class="sxs-lookup"><span data-stu-id="16ef3-105">Nullable types can represent all the values of an underlying type `T`, and an additional [null](../../language-reference/keywords/null.md) value.</span></span> <span data-ttu-id="16ef3-106">Базовый тип `T` может быть любым [типом значения](../../language-reference/keywords/value-types.md), не допускающим значение NULL.</span><span class="sxs-lookup"><span data-stu-id="16ef3-106">The underlying type `T` can be any non-nullable [value type](../../language-reference/keywords/value-types.md).</span></span> <span data-ttu-id="16ef3-107">`T` не может быть ссылочным типом.</span><span class="sxs-lookup"><span data-stu-id="16ef3-107">`T` cannot be a reference type.</span></span>

<span data-ttu-id="16ef3-108">Например, вы можете назначить значение `null` или любое целое число от <xref:System.Int32.MinValue?displayProperty=nameWithType> до <xref:System.Int32.MaxValue?displayProperty=nameWithType> для структуры `Nullable<int>`, а также [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md) или `Nullable<bool>` для `null`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-108">For example, you can assign `null` or any integer value from <xref:System.Int32.MinValue?displayProperty=nameWithType> to <xref:System.Int32.MaxValue?displayProperty=nameWithType> to a `Nullable<int>` and [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md), or `null` to a `Nullable<bool>`.</span></span>

<span data-ttu-id="16ef3-109">Тип, допускающий значение NULL, следует использовать, когда нужно представить неопределенное значение его базового типа.</span><span class="sxs-lookup"><span data-stu-id="16ef3-109">You use a nullable type when you need to represent the undefined value of an underlying type.</span></span> <span data-ttu-id="16ef3-110">У логической переменной может быть только два значения: true (истина) и false (ложь).</span><span class="sxs-lookup"><span data-stu-id="16ef3-110">A Boolean variable can have only two values: true and false.</span></span> <span data-ttu-id="16ef3-111">Значение "не определено" у нее отсутствует.</span><span class="sxs-lookup"><span data-stu-id="16ef3-111">There is no "undefined" value.</span></span> <span data-ttu-id="16ef3-112">Во многих сценариях программирования, в первую очередь при взаимодействии с базами данных, значение переменной может быть неопределенным или отсутствовать.</span><span class="sxs-lookup"><span data-stu-id="16ef3-112">In many programming applications, most notably database interactions, a variable value can be undefined or missing.</span></span> <span data-ttu-id="16ef3-113">Например, поле в базе данных может содержать значение true или false, или вообще не содержать никакого значения.</span><span class="sxs-lookup"><span data-stu-id="16ef3-113">For example, a field in a database may contain the values true or false, or it may contain no value at all.</span></span> <span data-ttu-id="16ef3-114">В этом случае используйте тип `Nullable<bool>`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-114">You use a `Nullable<bool>` type in that case.</span></span>

<span data-ttu-id="16ef3-115">Типы, допускающие значения NULL, имеют следующие характеристики.</span><span class="sxs-lookup"><span data-stu-id="16ef3-115">Nullable types have the following characteristics:</span></span>
  
- <span data-ttu-id="16ef3-116">Типы, допускающие значение NULL, представляют переменные типа значения, которым может быть присвоено значение `null`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-116">Nullable types represent value-type variables that can be assigned the `null` value.</span></span> <span data-ttu-id="16ef3-117">Нельзя создать тип, допускающий значение NULL, на основе ссылочного типа.</span><span class="sxs-lookup"><span data-stu-id="16ef3-117">You cannot create a nullable type based on a reference type.</span></span> <span data-ttu-id="16ef3-118">(Ссылочные типы всегда поддерживают значение `null`.)</span><span class="sxs-lookup"><span data-stu-id="16ef3-118">(Reference types already support the `null` value.)</span></span>  
  
- <span data-ttu-id="16ef3-119">Синтаксис `T?` является сокращением `Nullable<T>`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-119">The syntax `T?` is shorthand for `Nullable<T>`.</span></span> <span data-ttu-id="16ef3-120">Эти две формы записи являются взаимозаменяемыми.</span><span class="sxs-lookup"><span data-stu-id="16ef3-120">The two forms are interchangeable.</span></span>  
  
- <span data-ttu-id="16ef3-121">Типу, допускающему значение NULL, можно присвоить значение так же, как и типу базового значения, например `int? x = 10;` или `double? d = 4.108;`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-121">Assign a value to a nullable type just as you would for an underlying value type: `int? x = 10;` or `double? d = 4.108;`.</span></span> <span data-ttu-id="16ef3-122">Вы также можете присвоить значение `null`: `int? x = null;`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-122">You also can assign the `null` value: `int? x = null;`.</span></span>  
  
- <span data-ttu-id="16ef3-123">Используйте доступные только для чтения свойства <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> и <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> для проверки на NULL и получения значения, как показано в следующем примере: `if (x.HasValue) y = x.Value;`</span><span class="sxs-lookup"><span data-stu-id="16ef3-123">Use the <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> and <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> readonly properties to test for null and retrieve the value, as shown in the following example: `if (x.HasValue) y = x.Value;`</span></span>  
  
  - <span data-ttu-id="16ef3-124">Свойство <xref:System.Nullable%601.HasValue%2A> возвращает `true`, если переменная содержит значение, или `false`, если она содержит `null`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-124">The <xref:System.Nullable%601.HasValue%2A> property returns `true` if the variable contains a value, or `false` if it's `null`.</span></span>
  
  - <span data-ttu-id="16ef3-125">Свойство <xref:System.Nullable%601.Value%2A> возвращает значение, если <xref:System.Nullable%601.HasValue%2A> возвращает `true`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-125">The <xref:System.Nullable%601.Value%2A> property returns a value if <xref:System.Nullable%601.HasValue%2A> returns `true`.</span></span> <span data-ttu-id="16ef3-126">В противном случае возникает исключение <xref:System.InvalidOperationException>.</span><span class="sxs-lookup"><span data-stu-id="16ef3-126">Otherwise, an <xref:System.InvalidOperationException> is thrown.</span></span>  
  
- <span data-ttu-id="16ef3-127">Вы также можете использовать с типом, допускающим значение NULL, операторы `==` и `!=`, как показано в следующем примере: `if (x != null) y = x.Value;`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-127">You can also use the `==` and `!=` operators with a nullable type, as shown in the following example: `if (x != null) y = x.Value;`.</span></span> <span data-ttu-id="16ef3-128">Если `a` и `b` равны NULL, `a == b` дает значение `true`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-128">If `a` and `b` are both null, `a == b` evaluates to `true`.</span></span>  

- <span data-ttu-id="16ef3-129">Начиная с версии C# 7.0, для проверки и получения значения типа, допускающего значение NULL, можно использовать [сопоставление шаблонов](../../pattern-matching.md#the-is-type-pattern-expression): `if (x is int valueOfX) y = valueOfX;`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-129">Beginning with C# 7.0, you can use [pattern matching](../../pattern-matching.md#the-is-type-pattern-expression) to both examine and get a value of a nullable type: `if (x is int valueOfX) y = valueOfX;`.</span></span>
  
- <span data-ttu-id="16ef3-130">Значение по умолчанию `T?` является экземпляром, свойство <xref:System.Nullable%601.HasValue%2A> которого возвращает `false`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-130">The default value of `T?` is an instance whose <xref:System.Nullable%601.HasValue%2A> property returns `false`.</span></span>  

- <span data-ttu-id="16ef3-131">Используйте метод <xref:System.Nullable%601.GetValueOrDefault>, чтобы вернуть либо присвоенное значение, либо значение [по умолчанию](../../language-reference/keywords/default-values-table.md) для типа базового значения, если значение типа, допускающего значение NULL, равно `null`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-131">Use the <xref:System.Nullable%601.GetValueOrDefault> method to return either the assigned value, or the [default](../../language-reference/keywords/default-values-table.md) value of the underlying value type if the value of the nullable type is `null`.</span></span>  

- <span data-ttu-id="16ef3-132">Используйте метод <xref:System.Nullable%601.GetValueOrDefault(%600)>, чтобы вернуть либо присвоенное значение, либо предоставленное значение по умолчанию, если значение типа, допускающего значение NULL, равно `null`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-132">Use the <xref:System.Nullable%601.GetValueOrDefault(%600)> method to return either the assigned value, or the provided default value if the value of the nullable type is `null`.</span></span>
  
- <span data-ttu-id="16ef3-133">Используйте [оператор объединения со значением NULL](../../language-reference/operators/null-coalescing-operator.md), `??`, чтобы присвоить значение базовому типу на основе значения типа, допускающего значение NULL: `int? x = null; int y = x ?? -1;`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-133">Use the [null-coalescing operator](../../language-reference/operators/null-coalescing-operator.md), `??`, to assign a value to an underlying type based on a value of the nullable type: `int? x = null; int y = x ?? -1;`.</span></span> <span data-ttu-id="16ef3-134">В указанном примере, так как `x` равно NULL, итоговое значение `y` будет равно `-1`.</span><span class="sxs-lookup"><span data-stu-id="16ef3-134">In the example, since `x` is null, the result value of `y` is `-1`.</span></span>

- <span data-ttu-id="16ef3-135">Если между двумя типами данных определено пользовательское преобразование типов, то это же преобразование можно также использовать с версиями этих типов, допускающими значение null.</span><span class="sxs-lookup"><span data-stu-id="16ef3-135">If a user-defined conversion is defined between two data types, the same conversion can also be used with the nullable versions of these data types.</span></span>
  
- <span data-ttu-id="16ef3-136">Нельзя создавать вложенные типы, допускающие значение NULL.</span><span class="sxs-lookup"><span data-stu-id="16ef3-136">Nested nullable types are not allowed.</span></span> <span data-ttu-id="16ef3-137">Эта строка компилироваться не будет: `Nullable<Nullable<int>> n;`</span><span class="sxs-lookup"><span data-stu-id="16ef3-137">The following line doesn't compile: `Nullable<Nullable<int>> n;`</span></span>
