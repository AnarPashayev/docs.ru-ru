# <a name="nullable-value-types-c-programming-guide"></a><span data-ttu-id="bb9a4-103">Типы значений, допускающие значение NULL (руководство по программированию на C#)</span><span class="sxs-lookup"><span data-stu-id="bb9a4-103">Nullable value types (C# Programming Guide)</span></span>

<span data-ttu-id="bb9a4-104">Типы значений, допускающие значение NULL, являются экземплярами структуры <xref:System.Nullable%601?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-104">Nullable value types are instances of the <xref:System.Nullable%601?displayProperty=nameWithType> structure.</span></span> <span data-ttu-id="bb9a4-105">Типы значений, допускающие значение NULL, могут представлять все значения своего базового типа `T`, а также дополнительное значение [NULL](../../language-reference/keywords/null.md).</span><span class="sxs-lookup"><span data-stu-id="bb9a4-105">Nullable value types can represent all the values of an underlying type `T`, and an additional [null](../../language-reference/keywords/null.md) value.</span></span> <span data-ttu-id="bb9a4-106">Базовый тип `T` может быть любым [типом значения](../../language-reference/keywords/value-types.md), не допускающим значение NULL.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-106">The underlying type `T` can be any non-nullable [value type](../../language-reference/keywords/value-types.md).</span></span> <span data-ttu-id="bb9a4-107">`T` не может быть ссылочным типом.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-107">`T` cannot be a reference type.</span></span>

> [!NOTE]
> <span data-ttu-id="bb9a4-108">В C# 8.0 вводится функция ссылочных типов, допускающих значение NULL.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-108">C# 8.0 introduces the nullable reference types feature.</span></span> <span data-ttu-id="bb9a4-109">Дополнительные сведения см. в разделе [Ссылочные типы, допускающие значение NULL](../../nullable-references.md).</span><span class="sxs-lookup"><span data-stu-id="bb9a4-109">For more information, see [Nullable reference types](../../nullable-references.md).</span></span> <span data-ttu-id="bb9a4-110">Типы значений, допускающие значение NULL, доступны начиная с C# 2.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-110">The nullable value types are available starting with C# 2.</span></span>

<span data-ttu-id="bb9a4-111">Например, вы можете назначить значение `null` или любое целое число от <xref:System.Int32.MinValue?displayProperty=nameWithType> до <xref:System.Int32.MaxValue?displayProperty=nameWithType> для структуры `Nullable<int>`, а также [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md) или `Nullable<bool>` для `null`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-111">For example, you can assign `null` or any integer value from <xref:System.Int32.MinValue?displayProperty=nameWithType> to <xref:System.Int32.MaxValue?displayProperty=nameWithType> to a `Nullable<int>` and [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md), or `null` to a `Nullable<bool>`.</span></span>

<span data-ttu-id="bb9a4-112">Тип значения, допускающий значение NULL, следует использовать, когда нужно представить неопределенное значение его базового типа.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-112">You use a nullable value type when you need to represent the undefined value of an underlying type.</span></span> <span data-ttu-id="bb9a4-113">У логической переменной может быть только два значения: `true` и `false`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-113">A Boolean variable can have only two values: `true` and `false`.</span></span> <span data-ttu-id="bb9a4-114">Значение "не определено" у нее отсутствует.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-114">There is no "undefined" value.</span></span> <span data-ttu-id="bb9a4-115">Во многих сценариях программирования, в первую очередь при взаимодействии с базами данных, значение переменной может быть неопределенным или отсутствовать.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-115">In many programming applications, most notably database interactions, a variable value can be undefined or missing.</span></span> <span data-ttu-id="bb9a4-116">Например, поле в базе данных может содержать значение true или false, или вообще не содержать никакого значения.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-116">For example, a field in a database may contain the values true or false, or it may contain no value at all.</span></span> <span data-ttu-id="bb9a4-117">В этом случае используйте тип `Nullable<bool>`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-117">You use a `Nullable<bool>` type in that case.</span></span>

<span data-ttu-id="bb9a4-118">Типы значений, допускающие значение NULL, имеют следующие характеристики.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-118">Nullable value types have the following characteristics:</span></span>

- <span data-ttu-id="bb9a4-119">Типы значений, допускающие значение NULL, представляют переменные типа значения, которым может быть назначено значение `null`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-119">Nullable value types represent value-type variables that can be assigned the `null` value.</span></span>

- <span data-ttu-id="bb9a4-120">Синтаксис `T?` является сокращением `Nullable<T>`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-120">The syntax `T?` is shorthand for `Nullable<T>`.</span></span> <span data-ttu-id="bb9a4-121">Эти две формы записи являются взаимозаменяемыми.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-121">The two forms are interchangeable.</span></span>

- <span data-ttu-id="bb9a4-122">Типу значения, допускающему значение NULL, можно присвоить значение так же, как и типу базового значения, например `int? x = 10;` или `double? d = 4.108;`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-122">Assign a value to a nullable value type just as you would for an underlying value type: `int? x = 10;` or `double? d = 4.108;`.</span></span> <span data-ttu-id="bb9a4-123">Вы также можете присвоить значение `null`: `int? x = null;`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-123">You also can assign the `null` value: `int? x = null;`.</span></span>

- <span data-ttu-id="bb9a4-124">Используйте доступные только для чтения свойства <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> и <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> для проверки на NULL и получения значения, как показано в следующем примере: `if (x.HasValue) y = x.Value;`</span><span class="sxs-lookup"><span data-stu-id="bb9a4-124">Use the <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> and <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> readonly properties to test for null and retrieve the value, as shown in the following example: `if (x.HasValue) y = x.Value;`</span></span>

  - <span data-ttu-id="bb9a4-125">Свойство <xref:System.Nullable%601.HasValue%2A> возвращает `true`, если переменная содержит значение, или `false`, если она содержит `null`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-125">The <xref:System.Nullable%601.HasValue%2A> property returns `true` if the variable contains a value, or `false` if it's `null`.</span></span>

  - <span data-ttu-id="bb9a4-126">Свойство <xref:System.Nullable%601.Value%2A> возвращает значение, если <xref:System.Nullable%601.HasValue%2A> возвращает `true`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-126">The <xref:System.Nullable%601.Value%2A> property returns a value if <xref:System.Nullable%601.HasValue%2A> returns `true`.</span></span> <span data-ttu-id="bb9a4-127">В противном случае возникает исключение <xref:System.InvalidOperationException>.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-127">Otherwise, an <xref:System.InvalidOperationException> is thrown.</span></span>

- <span data-ttu-id="bb9a4-128">Вы также можете использовать с типом значения, допускающим значение NULL, операторы `==` и `!=`, как показано в следующем примере: `if (x != null) y = x.Value;`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-128">You can also use the `==` and `!=` operators with a nullable value type, as shown in the following example: `if (x != null) y = x.Value;`.</span></span> <span data-ttu-id="bb9a4-129">Если `a` и `b` равны NULL, `a == b` дает значение `true`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-129">If `a` and `b` are both null, `a == b` evaluates to `true`.</span></span>

- <span data-ttu-id="bb9a4-130">Начиная с версии C# 7.0, для проверки и получения значения типа, допускающего значение NULL, можно использовать [сопоставление шаблонов](../../pattern-matching.md#the-is-type-pattern-expression): `if (x is int valueOfX) y = valueOfX;`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-130">Beginning with C# 7.0, you can use [pattern matching](../../pattern-matching.md#the-is-type-pattern-expression) to both examine and get a value of a nullable type: `if (x is int valueOfX) y = valueOfX;`.</span></span>

- <span data-ttu-id="bb9a4-131">Значение по умолчанию `T?` является экземпляром, свойство <xref:System.Nullable%601.HasValue%2A> которого возвращает `false`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-131">The default value of `T?` is an instance whose <xref:System.Nullable%601.HasValue%2A> property returns `false`.</span></span>

- <span data-ttu-id="bb9a4-132">Используйте метод <xref:System.Nullable%601.GetValueOrDefault>, чтобы вернуть либо присвоенное значение, либо значение [по умолчанию](../../language-reference/keywords/default-values-table.md) для типа базового значения, если значение типа, допускающего значение NULL, равно `null`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-132">Use the <xref:System.Nullable%601.GetValueOrDefault> method to return either the assigned value, or the [default](../../language-reference/keywords/default-values-table.md) value of the underlying value type if the value of the nullable type is `null`.</span></span>

- <span data-ttu-id="bb9a4-133">Используйте метод <xref:System.Nullable%601.GetValueOrDefault(%600)>, чтобы вернуть либо присвоенное значение, либо предоставленное значение по умолчанию, если значение типа, допускающего значение NULL, равно `null`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-133">Use the <xref:System.Nullable%601.GetValueOrDefault(%600)> method to return either the assigned value, or the provided default value if the value of the nullable type is `null`.</span></span>

- <span data-ttu-id="bb9a4-134">Используйте [оператор объединения со значением NULL](../../language-reference/operators/null-coalescing-operator.md), `??`, чтобы присвоить значение переменной базового типа на основе типа значения, допускающего значение NULL: `int? x = null; int y = x ?? -1;`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-134">Use the [null-coalescing operator](../../language-reference/operators/null-coalescing-operator.md), `??`, to assign a value to a variable of an underlying value type based on a nullable-type value: `int? x = null; int y = x ?? -1;`.</span></span> <span data-ttu-id="bb9a4-135">В указанном примере, так как `x` равно `null`, итоговое значение `y` будет равно `-1`.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-135">In the example, since `x` is `null`, the result value of `y` is `-1`.</span></span>

- <span data-ttu-id="bb9a4-136">Если между двумя типами данных определено пользовательское преобразование типов, то это же преобразование можно также использовать с соответствующими типами, допускающими значение NULL.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-136">If a user-defined conversion is defined between two value types, the same conversion can also be used with the corresponding nullable types.</span></span>

- <span data-ttu-id="bb9a4-137">Нельзя создавать вложенные типы значений, допускающие значение NULL.</span><span class="sxs-lookup"><span data-stu-id="bb9a4-137">Nested nullable value types are not allowed.</span></span> <span data-ttu-id="bb9a4-138">Эта строка компилироваться не будет: `Nullable<Nullable<int>> n;`</span><span class="sxs-lookup"><span data-stu-id="bb9a4-138">The following line doesn't compile: `Nullable<Nullable<int>> n;`</span></span>
