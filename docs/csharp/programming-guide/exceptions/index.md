- <span data-ttu-id="d037e-116">Когда внутри такого блока `try` возникает исключение, поток управления переходит к первому подходящему обработчику исключений в стеке вызовов.</span><span class="sxs-lookup"><span data-stu-id="d037e-116">Once an exception occurs in the `try` block, the flow of control jumps to the first associated exception handler that is present anywhere in the call stack.</span></span> <span data-ttu-id="d037e-117">В C# ключевое слово `catch` обозначает обработчик исключений.</span><span class="sxs-lookup"><span data-stu-id="d037e-117">In C#, the `catch` keyword is used to define an exception handler.</span></span>

- <span data-ttu-id="d037e-119">Не перехватывайте исключение, если вы не намерены его обрабатывать с сохранением известного состояния приложения.</span><span class="sxs-lookup"><span data-stu-id="d037e-119">Do not catch an exception unless you can handle it and leave the application in a known state.</span></span> <span data-ttu-id="d037e-120">Если вы перехватываете `System.Exception`, создайте его заново в конце блока `catch`, используя ключевое слово `throw`.</span><span class="sxs-lookup"><span data-stu-id="d037e-120">If you catch `System.Exception`, re-throw it using the `throw` keyword at the end of the `catch` block.</span></span>
- <span data-ttu-id="d037e-121">Если блок `catch` определяет переменную исключения, ее можно использовать для получения дополнительных сведений о типе созданного исключения.</span><span class="sxs-lookup"><span data-stu-id="d037e-121">If a `catch` block defines an exception variable, you can use it to obtain more information about the type of exception that occurred.</span></span>
- <span data-ttu-id="d037e-122">Программа может явным образом создавать исключения с помощью ключевого слова `throw`.</span><span class="sxs-lookup"><span data-stu-id="d037e-122">Exceptions can be explicitly generated by a program by using the `throw` keyword.</span></span>
- <span data-ttu-id="d037e-123">Объекты исключения содержат подробные сведения об ошибке, например состояние стека вызовов и текстовое описание ошибки.</span><span class="sxs-lookup"><span data-stu-id="d037e-123">Exception objects contain detailed information about the error, such as the state of the call stack and a text description of the error.</span></span>
- <span data-ttu-id="d037e-124">Код в блоке `finally` выполняется даже в том случае, если создано исключение.</span><span class="sxs-lookup"><span data-stu-id="d037e-124">Code in a `finally` block is executed even if an exception is thrown.</span></span> <span data-ttu-id="d037e-125">Используйте блок `finally`, чтобы высвободить ресурсы, например закрыть потоки и файлы, которые были открыты внутри блока `try`.</span><span class="sxs-lookup"><span data-stu-id="d037e-125">Use a `finally` block to release resources, for example to close any streams or files that were opened in the `try` block.</span></span>
- <span data-ttu-id="d037e-126">Управляемые исключения реализованы в платформе .NET Framework на основе структурированного механизма обработки исключений Win32.</span><span class="sxs-lookup"><span data-stu-id="d037e-126">Managed exceptions in the .NET Framework are implemented on top of the Win32 structured exception handling mechanism.</span></span> <span data-ttu-id="d037e-127">Дополнительные сведения см. в статьях [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) (Структурированная обработка исключений в C и C++) и [A Crash Course on the Depths of Win32 Structured Exception Handling](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm) (Интенсивное погружение в структурированную обработку исключений на платформе Win32).</span><span class="sxs-lookup"><span data-stu-id="d037e-127">For more information, see [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) and [A Crash Course on the Depths of Win32 Structured Exception Handling](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm).</span></span>
