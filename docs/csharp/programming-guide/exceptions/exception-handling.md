## <a name="finally-blocks"></a><span data-ttu-id="9f6e4-126">Блоки "Finally"</span><span class="sxs-lookup"><span data-stu-id="9f6e4-126">Finally Blocks</span></span>  

 <span data-ttu-id="9f6e4-127">Блок `finally` позволяет удалить действия, выполненные в блоке `try`.</span><span class="sxs-lookup"><span data-stu-id="9f6e4-127">A `finally` block enables you to clean up actions that are performed in a `try` block.</span></span> <span data-ttu-id="9f6e4-128">При наличии блока `finally` он выполняется последним, после блока `try` и всех выполняемых блоков `catch`.</span><span class="sxs-lookup"><span data-stu-id="9f6e4-128">If present, the `finally` block executes last, after the `try` block and any matched `catch` block.</span></span> <span data-ttu-id="9f6e4-129">Блок `finally` выполняется всегда вне зависимости от возникновения исключения или обнаружения блока `catch`, соответствующего типу исключения.</span><span class="sxs-lookup"><span data-stu-id="9f6e4-129">A `finally` block always runs, regardless of whether an exception is thrown or a `catch` block matching the exception type is found.</span></span>  
  
 <span data-ttu-id="9f6e4-130">Блок `finally` можно использовать для высвобождения ресурсов, например потоков данных, подключений к базам данных, графических дескрипторов, не ожидая финализации объектов сборщиком мусора во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="9f6e4-130">The `finally` block can be used to release resources such as file streams, database connections, and graphics handles without waiting for the garbage collector in the runtime to finalize the objects.</span></span> <span data-ttu-id="9f6e4-131">Дополнительные сведения см. в разделе [Оператор using](../../language-reference/keywords/using-statement.md).</span><span class="sxs-lookup"><span data-stu-id="9f6e4-131">See [using Statement](../../language-reference/keywords/using-statement.md) for more information.</span></span>  
  
 <span data-ttu-id="9f6e4-132">В приведенном ниже примере с помощью блока `finally` закрывается файл, открытый в блоке `try`.</span><span class="sxs-lookup"><span data-stu-id="9f6e4-132">In the following example, the `finally` block is used to close a file that is opened in the `try` block.</span></span> <span data-ttu-id="9f6e4-133">Обратите внимание, что состояние дескриптора файла проверяется до закрытия файла.</span><span class="sxs-lookup"><span data-stu-id="9f6e4-133">Notice that the state of the file handle is checked before the file is closed.</span></span> <span data-ttu-id="9f6e4-134">Если блок `try` не может открыть этот файл, дескриптор файла по-прежнему имеет значение `null`, и блок `finally` не пытается закрыть его.</span><span class="sxs-lookup"><span data-stu-id="9f6e4-134">If the `try` block cannot open the file, the file handle still has the value `null` and the `finally` block does not try to close it.</span></span> <span data-ttu-id="9f6e4-135">Кроме того, если файл успешно открыт в блоке `try`, блок `finally` закрывает открытый файл.</span><span class="sxs-lookup"><span data-stu-id="9f6e4-135">Alternatively, if the file is opened successfully in the `try` block, the `finally` block closes the open file.</span></span>
