# <a name="compiler-error-cs0160"></a><span data-ttu-id="af565-102">Ошибка компилятора CS0160</span><span class="sxs-lookup"><span data-stu-id="af565-102">Compiler Error CS0160</span></span>
<span data-ttu-id="af565-103">Предыдущее предложение catch уже перехватывает все исключения этого типа или супертипа ("тип").</span><span class="sxs-lookup"><span data-stu-id="af565-103">A previous catch clause already catches all exceptions of this or of a super type ('type')</span></span>  
  
<span data-ttu-id="af565-104">Ряд операторов [catch](../language-reference/keywords/try-catch.md) должен быть указан в порядке убывания поколения наследования.</span><span class="sxs-lookup"><span data-stu-id="af565-104">A series of [catch](../language-reference/keywords/try-catch.md) statements needs to be in decreasing order of derivation.</span></span> <span data-ttu-id="af565-105">Например, наиболее производные объекты должны быть первыми.</span><span class="sxs-lookup"><span data-stu-id="af565-105">For example, the most derived objects must appear first.</span></span>
  
 <span data-ttu-id="af565-106">Дополнительные сведения см. в разделе [исключения и обработка исключений](../programming-guide/exceptions/index.md).</span><span class="sxs-lookup"><span data-stu-id="af565-106">For more information, see [Exceptions and Exception Handling](../programming-guide/exceptions/index.md).</span></span>  
  
 <span data-ttu-id="af565-107">В следующем примере возникает ошибка CS0160:</span><span class="sxs-lookup"><span data-stu-id="af565-107">The following sample generates CS0160:</span></span>  
  
```csharp  
// CS0160.cs  
public class MyClass2 : System.Exception {}  
public class MyClass  
{  
   public static void Main()  
   {  
      try {}  
  
      catch(System.Exception) {}   // Second-most derived; should be second catch  
      catch(MyClass2) {}   // CS0160  Most derived; should be first catch  
   }  
}  
```
