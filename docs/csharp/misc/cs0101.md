 <span data-ttu-id="3ed67-105">Объект [namespace](../language-reference/keywords/namespace.md) содержит повторяющиеся идентификаторы.</span><span class="sxs-lookup"><span data-stu-id="3ed67-105">A [namespace](../language-reference/keywords/namespace.md) has duplicate identifiers.</span></span> <span data-ttu-id="3ed67-106">Переименуйте или удалите один из повторяющихся идентификаторов.</span><span class="sxs-lookup"><span data-stu-id="3ed67-106">Rename or delete one of the duplicate identifiers.</span></span> <span data-ttu-id="3ed67-107">Дополнительные сведения см. в разделе [пространства имен](../programming-guide/namespaces/index.md) .</span><span class="sxs-lookup"><span data-stu-id="3ed67-107">For more information, see [Namespaces](../programming-guide/namespaces/index.md)</span></span>  
  
 <span data-ttu-id="3ed67-108">Следующий пример приводит к возникновению ошибки CS0101:</span><span class="sxs-lookup"><span data-stu-id="3ed67-108">The following sample generates CS0101:</span></span>  
  
```csharp  
// CS0101.cs  
namespace MyNamespace  
{  
   public class MyClass  
   {  
      static public void Main()  
      {  
      }  
   }  
  
   public class MyClass   // CS0101  
   {  
   }  
}  
```

<span data-ttu-id="3ed67-109">CS0101 также создается, если имя класса конфликтует с именем пространства имен.</span><span class="sxs-lookup"><span data-stu-id="3ed67-109">A CS0101 is also generated when your class name clashes with your namespace name.</span></span>  <span data-ttu-id="3ed67-110">Это может произойти при расширении с вспомогательными классами для базового класса, в котором вы пытаетесь разместить одинаковый маршрут пространства имен.</span><span class="sxs-lookup"><span data-stu-id="3ed67-110">This can happen when expanding with helper classes for the base class where you attempt to keep the namespace route the same.</span></span>  <span data-ttu-id="3ed67-111">В приведенном ниже примере класс UTF8 должен четко быть дочерним по отношению к классу String, но при попытке принудительного применения этого пространства имен в качестве служебных программ будет использоваться одно и то же имя. строка приведет к ошибке CS0101:</span><span class="sxs-lookup"><span data-stu-id="3ed67-111">In the below example, the UTF8 class should clearly be a subsidiary of the String class, but attempting to force it into the same name space by declaring said namespace as Utilities.String will cause a CS0101 error:</span></span>

```csharp
//CS0101-Utilities.String.cs
namespace Utilities
{  
   public class String
   {  
        public string MyString;
   }  
}

//CS0101-Utilities.String.UTF8.cs
namespace Utilities.String  // CS0101  
{  
   public class UTF8
   {  
        public string MySecondString;
   }  
}  
```
