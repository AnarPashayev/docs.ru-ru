# <a name="compiler-warning-level-2-cs0440"></a><span data-ttu-id="4d51e-103">Предупреждение компилятора (уровень 2) CS0440</span><span class="sxs-lookup"><span data-stu-id="4d51e-103">Compiler Warning (level 2) CS0440</span></span>
<span data-ttu-id="4d51e-104">Определение псевдонима с именем "global" не рекомендуется из-за того, что "global::" всегда указывает на глобальное пространство имен и не является псевдонимом.</span><span class="sxs-lookup"><span data-stu-id="4d51e-104">Defining an alias named 'global' is ill-advised since 'global::' always references the global namespace and not an alias</span></span>  
  
 <span data-ttu-id="4d51e-105">Это предупреждение возникает при определении псевдонима с именем global.</span><span class="sxs-lookup"><span data-stu-id="4d51e-105">This warning is issued when you define an alias named global.</span></span>  
  
## <a name="example"></a><span data-ttu-id="4d51e-106">Пример</span><span class="sxs-lookup"><span data-stu-id="4d51e-106">Example</span></span>  
 <span data-ttu-id="4d51e-107">В следующем примере возникает ошибка CS0440:</span><span class="sxs-lookup"><span data-stu-id="4d51e-107">The following example generates CS0440:</span></span>  
  
```csharp  
// CS0440.cs  
// Compile with: /W:1  
  
using global = MyClass;   // CS0440  
class MyClass  
{  
    static void Main()  
    {  
        // Note how global refers to the global namespace  
        // even though it is redefined above.  
        global::System.Console.WriteLine();  
    }  
}  
```
