# <a name="compiler-error-cs0313"></a><span data-ttu-id="34e17-103">Ошибка компилятора CS0313</span><span class="sxs-lookup"><span data-stu-id="34e17-103">Compiler Error CS0313</span></span>
<span data-ttu-id="34e17-104">Тип "тип1" нельзя использовать в качестве параметра типа "имя параметра" в универсальном типе или методе "тип2".</span><span class="sxs-lookup"><span data-stu-id="34e17-104">The type 'type1' cannot be used as type parameter 'parameter name' in the generic type or method 'type2'.</span></span> <span data-ttu-id="34e17-105">Допускающий значение null тип "тип1" не удовлетворяет ограничению типа "тип2".</span><span class="sxs-lookup"><span data-stu-id="34e17-105">The nullable type 'type1' does not satisfy the constraint of 'type2'.</span></span> <span data-ttu-id="34e17-106">Допускающие значение NULL типы не могу отвечать никаким ограничениям интерфейса.</span><span class="sxs-lookup"><span data-stu-id="34e17-106">Nullable types cannot satisfy any interface constraints.</span></span>  
  
 <span data-ttu-id="34e17-107">Тип значения, допускающий значение null, не эквивалентен его аналогу, не допускающему значения NULL.</span><span class="sxs-lookup"><span data-stu-id="34e17-107">A nullable value type is not equivalent to its non-nullable counterpart.</span></span> <span data-ttu-id="34e17-108">В представленном ниже примере структура `ImplStruct` удовлетворяет ограничению `BaseInterface` , однако структура `ImplStruct?` этому ограничению не удовлетворяет, так как метод `Nullable<ImplStruct>` не реализует интерфейс `BaseInterface`.</span><span class="sxs-lookup"><span data-stu-id="34e17-108">In the example that follows, `ImplStruct` satisfies the `BaseInterface` constraint but `ImplStruct?` does not because `Nullable<ImplStruct>` does not implement `BaseInterface`.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="34e17-109">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="34e17-109">To correct this error</span></span>  
  
1. <span data-ttu-id="34e17-110">Одно из решений показано в представленном ниже примере кода: определите обычную структуру `ImplStruct` в качестве первого аргумента типа в вызове метода `TestMethod`.</span><span class="sxs-lookup"><span data-stu-id="34e17-110">Using the code that follows as an example, one solution is to specify an ordinary `ImplStruct` as the first type argument in the call to `TestMethod`.</span></span> <span data-ttu-id="34e17-111">Затем измените метод `TestMethod` , чтобы создать допускающую значение NULL версию структуры `Implstruct` в его операторе return.</span><span class="sxs-lookup"><span data-stu-id="34e17-111">Then modify `TestMethod` to create a nullable version of `Implstruct` in its return statement:</span></span>  
  
    ```csharp  
    return new Nullable<T>(t);  
    ```  
  
## <a name="example"></a><span data-ttu-id="34e17-112">Пример</span><span class="sxs-lookup"><span data-stu-id="34e17-112">Example</span></span>

<span data-ttu-id="34e17-113">Следующий код приводит к возникновению ошибки CS0313:</span><span class="sxs-lookup"><span data-stu-id="34e17-113">The following code generates CS0313:</span></span>  
  
```csharp  
// cs0313.cs  
public interface BaseInterface { }  
public struct ImplStruct : BaseInterface { }  
  
public class TestClass  
{  
    public T? TestMethod<T, U>(T t) where T : struct, U  
    {  
        return t;  
    }  
}  
  
public class NullableTest  
{  
    public static void Run()  
    {  
  
        TestClass tc = new TestClass();  
        tc.TestMethod<ImplStruct?, BaseInterface>(new ImplStruct?()); // CS0313  
    }  
    public static void Main()  
    { }  
}  
```
