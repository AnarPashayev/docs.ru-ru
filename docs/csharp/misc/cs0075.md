# <a name="compiler-error-cs0075"></a><span data-ttu-id="495be-103">Ошибка компилятора CS0075</span><span class="sxs-lookup"><span data-stu-id="495be-103">Compiler Error CS0075</span></span>

<span data-ttu-id="495be-104">Для приведения отрицательного значения следует заключить значение в круглые скобки</span><span class="sxs-lookup"><span data-stu-id="495be-104">To cast a negative value, you must enclose the value in parentheses</span></span>  
  
 <span data-ttu-id="495be-105">Если выполняется приведение с помощью ключевого слова, идентифицирующего предопределенный тип, то скобки не требуются.</span><span class="sxs-lookup"><span data-stu-id="495be-105">If you are casting using a keyword that identifies a predefined type, then you do not need parentheses.</span></span> <span data-ttu-id="495be-106">В противном случае необходимо использовать скобки, так как (x) –y не будет считаться выражением приведения.</span><span class="sxs-lookup"><span data-stu-id="495be-106">Otherwise, you must put the parentheses because (x) –y will not be considered a cast expression.</span></span> <span data-ttu-id="495be-107">Из спецификации C#, раздел 7.6.6:</span><span class="sxs-lookup"><span data-stu-id="495be-107">From the C# Specification, Section 7.6.6:</span></span>  
  
 <span data-ttu-id="495be-108">*В правиле устранения неоднозначности следует, что, если x и y являются идентификаторами, (x) y, (x) (y) и (x) (-y) являются выражениями CAST, но (x)-y нет, даже если x определяет тип. Однако если x является ключевым словом, определяющим предопределенный тип (например, int), то все четыре формы являются выражениями CAST (поскольку такое ключевое слово не может быть выражением само по себе).*</span><span class="sxs-lookup"><span data-stu-id="495be-108">*From the disambiguation rule it follows that, if x and y are identifiers, (x)y, (x)(y), and (x)(-y) are cast-expressions, but (x)-y is not, even if x identifies a type. However, if x is a keyword that identifies a predefined type (such as int), then all four forms are cast-expressions (because such a keyword could not possibly be an expression by itself).*</span></span>  
  
 <span data-ttu-id="495be-109">Следующий код приводит к возникновению ошибки CS0075:</span><span class="sxs-lookup"><span data-stu-id="495be-109">The following code generates CS0075:</span></span>  
  
```csharp  
// CS0075  
namespace MyNamespace  
{  
    enum MyEnum { }  
    public class MyClass  
    {  
        public static void Main()  
        {  
            // To fix the error, place the negative  
            // values below in parentheses  
            int i = (System.Int32) - 4; //CS0075  
            MyEnum e = (MyEnum) - 1;    //CS0075  
            System.Console.WriteLine(i); //to avoid warning  
            System.Console.WriteLine(e); //to avoid warning  
        }  
    }  
}  
```
