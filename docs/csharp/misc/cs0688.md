## <a name="example"></a><span data-ttu-id="d9e9b-108">Пример</span><span class="sxs-lookup"><span data-stu-id="d9e9b-108">Example</span></span>  
 <span data-ttu-id="d9e9b-109">В следующем примере возникает ошибка CS0688.</span><span class="sxs-lookup"><span data-stu-id="d9e9b-109">The following sample generates CS0688.</span></span> <span data-ttu-id="d9e9b-110">Чтобы устранить это предупреждение без изменения базового класса, удалите атрибут безопасности из переопределяющего метода.</span><span class="sxs-lookup"><span data-stu-id="d9e9b-110">To resolve the warning without modifying the base class, remove the security attribute from the overriding method.</span></span> <span data-ttu-id="d9e9b-111">Это не решит проблему безопасности.</span><span class="sxs-lookup"><span data-stu-id="d9e9b-111">This will not solve the security problem.</span></span>  
  
```csharp  
// CS0688.cs  
// compile with: /W:1  
using System;  
using System.Security.Permissions;  
  
class Base   
{  
    //Uncomment the following line to close the security hole  
    //[FileIOPermission(SecurityAction.LinkDemand, All=@"C:\\")]  
    public virtual void DoScaryFileStuff()  
    {  
    }  
}  
  
class Derived: Base  
{  
    [FileIOPermission(SecurityAction.LinkDemand, All=@"C:\\")] // CS0688  
    public override void DoScaryFileStuff()  
    {  
    }  
    static void Main()  
    {  
    }  
}  
```
