<span data-ttu-id="a0d3d-104">Метод "метод1" содержит запрос компоновки, но переопределяет или реализует метод "метод2", который не имеет запрос компоновки.</span><span class="sxs-lookup"><span data-stu-id="a0d3d-104">'method1' has a link demand, but overrides or implements 'method2' which does not have a link demand.</span></span> <span data-ttu-id="a0d3d-105">Может возникнуть нарушение защиты.</span><span class="sxs-lookup"><span data-stu-id="a0d3d-105">A security hole may exist.</span></span>  
  
 <span data-ttu-id="a0d3d-106">Запрос компоновки, установленный в методе производного класса, можно легко обойти, вызвав метод базового класса.</span><span class="sxs-lookup"><span data-stu-id="a0d3d-106">The link demand set up on the derived class method can easily be circumvented by calling the base class method.</span></span> <span data-ttu-id="a0d3d-107">Чтобы закрыть эту брешь в системе безопасности, метод базового класса также должен использовать запрос компоновки.</span><span class="sxs-lookup"><span data-stu-id="a0d3d-107">To close the security hole, the base class method needs to also use the link demand.</span></span> <span data-ttu-id="a0d3d-108">Дополнительные сведения см. в разделе [Demand и LinkDemand](../../framework/misc/securing-wrapper-code.md#demand-vs-linkdemand).</span><span class="sxs-lookup"><span data-stu-id="a0d3d-108">For more information, see [Demand vs. LinkDemand](../../framework/misc/securing-wrapper-code.md#demand-vs-linkdemand).</span></span>  
  
## <a name="example"></a><span data-ttu-id="a0d3d-109">Пример</span><span class="sxs-lookup"><span data-stu-id="a0d3d-109">Example</span></span>  
 <span data-ttu-id="a0d3d-110">В следующем примере возникает ошибка CS0688.</span><span class="sxs-lookup"><span data-stu-id="a0d3d-110">The following sample generates CS0688.</span></span> <span data-ttu-id="a0d3d-111">Чтобы устранить это предупреждение без изменения базового класса, удалите атрибут безопасности из переопределяющего метода.</span><span class="sxs-lookup"><span data-stu-id="a0d3d-111">To resolve the warning without modifying the base class, remove the security attribute from the overriding method.</span></span> <span data-ttu-id="a0d3d-112">Это не решит проблему безопасности.</span><span class="sxs-lookup"><span data-stu-id="a0d3d-112">This will not solve the security problem.</span></span>  
  
```csharp  
// CS0688.cs  
// compile with: /W:1  
using System;  
using System.Security.Permissions;  
  
class Base
{  
    //Uncomment the following line to close the security hole  
    //[FileIOPermission(SecurityAction.LinkDemand, All=@"C:\\")]  
    public virtual void DoScaryFileStuff()  
    {  
    }  
}  
  
class Derived: Base  
{  
    [FileIOPermission(SecurityAction.LinkDemand, All=@"C:\\")] // CS0688  
    public override void DoScaryFileStuff()  
    {  
    }  
    static void Main()  
    {  
    }  
}  
```
