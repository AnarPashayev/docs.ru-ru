# <a name="compiler-error-cs0505"></a><span data-ttu-id="55031-102">Ошибка компилятора CS0505</span><span class="sxs-lookup"><span data-stu-id="55031-102">Compiler Error CS0505</span></span>
<span data-ttu-id="55031-103">"член1": невозможно переопределить, так как "член2" не является функцией</span><span class="sxs-lookup"><span data-stu-id="55031-103">'member1': cannot override because 'member2' is not a function</span></span>  
  
 <span data-ttu-id="55031-104">Объявление класса пыталось переопределить не являющийся методом объект в базовом классе.</span><span class="sxs-lookup"><span data-stu-id="55031-104">A class declaration attempted to override a non-method in a base class.</span></span> <span data-ttu-id="55031-105">Переопределения должны соответствовать типу члена.</span><span class="sxs-lookup"><span data-stu-id="55031-105">Overrides must match the member type.</span></span> <span data-ttu-id="55031-106">Если нужен метод с тем же именем, что и метод в базовом классе, используйте ключевое слово [new](../language-reference/keywords/new-modifier.md) (а не [override](../language-reference/keywords/override.md)) в объявлении метода в базовом классе.</span><span class="sxs-lookup"><span data-stu-id="55031-106">If a method with the same name as a method in a base class is desired, use [new](../language-reference/keywords/new-modifier.md) (and not [override](../language-reference/keywords/override.md)) on the method declaration in the base class.</span></span>  
  
 <span data-ttu-id="55031-107">В следующем примере возникает ошибка CS0505:</span><span class="sxs-lookup"><span data-stu-id="55031-107">The following sample generates CS0505:</span></span>  
  
```csharp  
// CS0505.cs  
// compile with: /target:library  
public class clx  
{  
   public int i;  
}  
  
public class cly : clx  
{  
   public override int i() { return 0; }   // CS0505  
}  
```
