# <a name="compiler-error-cs0503"></a><span data-ttu-id="64418-103">Ошибка компилятора CS0503</span><span class="sxs-lookup"><span data-stu-id="64418-103">Compiler Error CS0503</span></span>
<span data-ttu-id="64418-104">Абстрактный метод "метод" не может быть помечен как виртуальный</span><span class="sxs-lookup"><span data-stu-id="64418-104">The abstract method 'method' cannot be marked virtual</span></span>  
  
 <span data-ttu-id="64418-105">Излишне одновременно помечать метод как [абстрактный](../language-reference/keywords/abstract.md) и как [виртуальный](../language-reference/keywords/virtual.md) , потому что подразумевается, что **абстрактный** метод является **виртуальным**.</span><span class="sxs-lookup"><span data-stu-id="64418-105">It is redundant to mark a member method as both [abstract](../language-reference/keywords/abstract.md) and [virtual](../language-reference/keywords/virtual.md) because **abstract** implies **virtual**.</span></span>  
  
 <span data-ttu-id="64418-106">Следующий пример приводит к возникновению ошибки CS0503:</span><span class="sxs-lookup"><span data-stu-id="64418-106">The following sample generates CS0503:</span></span>  
  
```csharp  
// CS0503.cs  
namespace x  
{  
   abstract public class clx  
   {  
      abstract virtual public void f();   // CS0503  
   }  
}  
```
