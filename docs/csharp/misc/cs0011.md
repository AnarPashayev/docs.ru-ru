# <a name="compiler-error-cs0011"></a><span data-ttu-id="6a018-102">Ошибка компилятора CS0011</span><span class="sxs-lookup"><span data-stu-id="6a018-102">Compiler Error CS0011</span></span>
<span data-ttu-id="6a018-103">Не удается разрешить базовый класс или интерфейс "класс" в сборке "сборка", на которую ссылается тип "тип"</span><span class="sxs-lookup"><span data-stu-id="6a018-103">The base class or interface 'class' in assembly 'assembly' referenced by type 'type' could not be resolved</span></span>  
  
 <span data-ttu-id="6a018-104">Класс, который был импортирован из файла с помощью **/reference**, является производным от не найденного класса или реализует не найденный интерфейс.</span><span class="sxs-lookup"><span data-stu-id="6a018-104">A class that was imported from a file with **/reference**, is derived from a class or implements an interface that is not found.</span></span> <span data-ttu-id="6a018-105">Это может произойти, если требуемая библиотека DLL не включена в компиляцию с помощью **/reference**.</span><span class="sxs-lookup"><span data-stu-id="6a018-105">This can occur if a required DLL is not also included in the compilation with **/reference**.</span></span>  
  
 <span data-ttu-id="6a018-106">Дополнительные сведения см. в разделе [диалоговое окно "Добавление ссылки"](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) и [-Reference (C# параметры компилятора)](../language-reference/compiler-options/reference-compiler-option.md).</span><span class="sxs-lookup"><span data-stu-id="6a018-106">For more information, see [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) and [-reference (C# Compiler Options)](../language-reference/compiler-options/reference-compiler-option.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="6a018-107">Пример</span><span class="sxs-lookup"><span data-stu-id="6a018-107">Example</span></span>  
  
```csharp  
// CS0011_1.cs  
// compile with: /target:library  
  
public class Outer   
{  
   public class B { }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="6a018-108">Пример</span><span class="sxs-lookup"><span data-stu-id="6a018-108">Example</span></span>  
 <span data-ttu-id="6a018-109">Второй файл создает библиотеку DLL, которая определяет класс `C` , производный от класса `B` , созданного в предыдущем примере.</span><span class="sxs-lookup"><span data-stu-id="6a018-109">The second file creates a DLL that defines a class `C` that is derived from the class `B` that was created in the previous example.</span></span>  
  
```csharp  
// CS0011_2.cs  
// compile with: /target:library /reference:CS0011_1.dll  
// post-build command: del /f CS0011_1.dll  
public class C : Outer.B {}  
```  
  
## <a name="example"></a><span data-ttu-id="6a018-110">Пример</span><span class="sxs-lookup"><span data-stu-id="6a018-110">Example</span></span>  
 <span data-ttu-id="6a018-111">Третий файл заменяет библиотеку DLL, созданную на первом шаге, и пропускает определение внутреннего класса `B`.</span><span class="sxs-lookup"><span data-stu-id="6a018-111">The third file replaces the DLL created by the first step, and omits the definition of the inner class `B`.</span></span>  
  
```csharp  
// CS0011_3.cs  
// compile with: /target:library /out:cs0011_1.dll  
public class Outer {}  
```  
  
## <a name="example"></a><span data-ttu-id="6a018-112">Пример</span><span class="sxs-lookup"><span data-stu-id="6a018-112">Example</span></span>  
 <span data-ttu-id="6a018-113">Наконец, четвертый файл ссылается на класс `C` , определенный во втором примере, являющийся производным от класса `B`, который в данный момент отсутствует.</span><span class="sxs-lookup"><span data-stu-id="6a018-113">Finally, the fourth file references the class `C` defined in the second example, which is derived from class `B`, and which is now missing.</span></span>  
  
 <span data-ttu-id="6a018-114">Следующий пример приводит к возникновению ошибки CS0011.</span><span class="sxs-lookup"><span data-stu-id="6a018-114">The following sample generates CS0011.</span></span>  
  
```csharp  
// CS0011_4.cs

// CS0011 expected  
  
class M  
{  
   public static void Main()  
   {  
      C c = new C();  
   }  
}  
```
