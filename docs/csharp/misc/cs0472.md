<span data-ttu-id="595b5-104">Результат выражения всегда равен "значение1", поскольку значение типа "значение2" никогда не равно Null типа "значение3"</span><span class="sxs-lookup"><span data-stu-id="595b5-104">The result of the expression is always 'value1' since a value of type 'value2' is never equal to 'null' of type 'value3'</span></span>

## <a name="example"></a><span data-ttu-id="595b5-106">Пример</span><span class="sxs-lookup"><span data-stu-id="595b5-106">Example</span></span>  

 <span data-ttu-id="595b5-107">Следующий пример приводит к возникновению предупреждения CS0472:</span><span class="sxs-lookup"><span data-stu-id="595b5-107">The following sample generates CS0472.</span></span>  
  
```csharp  
public class Test  
{  
    public static int Main()  
    {  
        int i = 5;  
        int counter = 0;  
  
        // Comparison:  
        if (i == null)  // CS0472  
        // To resolve, use a valid value for i.  
            counter++;
        return counter;  
    }  
}  
```
