# <a name="compiler-error-cs1613"></a><span data-ttu-id="6c06f-103">Ошибка компилятора CS1613</span><span class="sxs-lookup"><span data-stu-id="6c06f-103">Compiler Error CS1613</span></span>

<span data-ttu-id="6c06f-104">Управляемый класс-оболочка coclass "класс" для интерфейса "интерфейс" не найден (возможно, была пропущена ссылка на сборку).</span><span class="sxs-lookup"><span data-stu-id="6c06f-104">The managed coclass wrapper class 'class' for interface 'interface' cannot be found (are you missing an assembly reference?)</span></span>  
  
 <span data-ttu-id="6c06f-105">Предпринята попытка создать экземпляр COM-объекта из интерфейса.</span><span class="sxs-lookup"><span data-stu-id="6c06f-105">An attempt was made to instantiate a COM object from an interface.</span></span> <span data-ttu-id="6c06f-106">Интерфейс имеет атрибуты **ComImport** и `CoClass` , однако компилятору не удается найти тип, предоставленный для атрибута `CoClass` .</span><span class="sxs-lookup"><span data-stu-id="6c06f-106">The interface has the **ComImport** and `CoClass` attributes, but the compiler cannot find the type given for the `CoClass` attribute.</span></span>  
  
 <span data-ttu-id="6c06f-107">Чтобы устранить эту неполадку, можно попробовать выполнить указанные ниже действия.</span><span class="sxs-lookup"><span data-stu-id="6c06f-107">To resolve this error, you can try one of the following:</span></span>  
  
- <span data-ttu-id="6c06f-108">Добавьте ссылку на сборку, которая содержит кокласс (в большинстве случаев интерфейс и кокласс должны находиться в одной сборке).</span><span class="sxs-lookup"><span data-stu-id="6c06f-108">Add a reference to the assembly that has the coclass (most of the time the interface and coclass should be in the same assembly).</span></span> <span data-ttu-id="6c06f-109">Для получения сведений см. раздел [-Reference](../language-reference/compiler-options/reference-compiler-option.md) или [диалоговое окно "Добавление ссылки"](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) .</span><span class="sxs-lookup"><span data-stu-id="6c06f-109">See [-reference](../language-reference/compiler-options/reference-compiler-option.md) or [Add Reference Dialog Box](/visualstudio/ide/how-to-add-or-remove-references-by-using-the-reference-manager) for information.</span></span>  
  
- <span data-ttu-id="6c06f-110">Исправьте атрибут `CoClass` интерфейса.</span><span class="sxs-lookup"><span data-stu-id="6c06f-110">Fix the `CoClass` attribute on the interface.</span></span>  
  
 <span data-ttu-id="6c06f-111">В приведенном ниже примере демонстрируется правильное применение атрибута **CoClassAttribute**.</span><span class="sxs-lookup"><span data-stu-id="6c06f-111">The following sample demonstrates correct usage of **CoClassAttribute**:</span></span>  
  
```csharp  
// CS1613.cs  
using System;  
using System.Runtime.InteropServices;  
  
[Guid("1FFD7840-E82D-4268-875C-80A160C23296")]  
[ComImport()]  
[CoClass(typeof(A))]  
public interface IA{}  
public class A : IA {}  
  
public class AA  
{  
   public static void Main()  
   {  
      IA i;  
      i = new IA(); // This is equivalent to new A().  
                    // because of the CoClass attribute on IA  
   }  
}  
```
