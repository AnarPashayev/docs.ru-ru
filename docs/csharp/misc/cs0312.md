 <span data-ttu-id="9f730-106">Тип значения, допускающий значение null, отличается от его аналога, не допускающего значения NULL. между ними не существует неявного преобразования ссылок или преобразования.</span><span class="sxs-lookup"><span data-stu-id="9f730-106">A nullable value type is distinct from its non-nullable counterpart; no implicit reference conversion or identify conversion exists between them.</span></span> <span data-ttu-id="9f730-107">Допускающая значения null упаковка-преобразование не удовлетворяет ограничению универсального типа.</span><span class="sxs-lookup"><span data-stu-id="9f730-107">A nullable boxing conversion does not satisfy a generic type constraint.</span></span> <span data-ttu-id="9f730-108">В следующем примере первый параметр типа является `Nullable<int>` , а второй параметр типа является `System.Int32`.</span><span class="sxs-lookup"><span data-stu-id="9f730-108">In the example that follows, the first type parameter is a `Nullable<int>` and the second type parameter is a `System.Int32`.</span></span>

## <a name="example"></a><span data-ttu-id="9f730-112">Пример</span><span class="sxs-lookup"><span data-stu-id="9f730-112">Example</span></span>

```csharp

    static void MTyVar<T, U>() where T : U { }

        MTyVar<int?, int>(); // CS0312  
        return 1;  
    }  
}  
```
