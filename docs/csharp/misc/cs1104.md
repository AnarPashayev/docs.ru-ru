# <a name="compiler-error-cs1104"></a><span data-ttu-id="9ee90-103">Ошибка компилятора CS1104</span><span class="sxs-lookup"><span data-stu-id="9ee90-103">Compiler Error CS1104</span></span>

## <a name="to-correct-this-error"></a><span data-ttu-id="9ee90-106">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="9ee90-106">To correct this error</span></span>  
  
1. <span data-ttu-id="9ee90-107">Обратите внимание, что первый параметр определения метода расширения указывает, какой тип метод будет "расширять".</span><span class="sxs-lookup"><span data-stu-id="9ee90-107">Remember that the first parameter of an extension method definition specifies which type the method will "extend".</span></span> <span data-ttu-id="9ee90-108">Это не входной параметр.</span><span class="sxs-lookup"><span data-stu-id="9ee90-108">It is not an input parameter.</span></span> <span data-ttu-id="9ee90-109">Таким образом, в этом месте массив параметров не имеет смысла.</span><span class="sxs-lookup"><span data-stu-id="9ee90-109">Therefore, it makes no sense to have a params array in this location.</span></span> <span data-ttu-id="9ee90-110">Если нужно передать массив параметров, передайте его вторым параметром.</span><span class="sxs-lookup"><span data-stu-id="9ee90-110">If you do have to pass in a params array, make it the second parameter.</span></span>
