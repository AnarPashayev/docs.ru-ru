# <a name="compiler-error-cs0172"></a><span data-ttu-id="1ea07-103">Ошибка компилятора CS0172</span><span class="sxs-lookup"><span data-stu-id="1ea07-103">Compiler Error CS0172</span></span>
<span data-ttu-id="1ea07-104">Не удается определить тип условного выражения, так как "тип1" и "тип2" неявно преобразуются друг в друга</span><span class="sxs-lookup"><span data-stu-id="1ea07-104">Type of conditional expression cannot be determined because 'type1' and 'type2' implicitly convert to one another</span></span>  
  
<span data-ttu-id="1ea07-105">В условном операторе необходимо иметь возможность неявного преобразования типов с обеих сторон `:` токена.</span><span class="sxs-lookup"><span data-stu-id="1ea07-105">In a conditional statement, you must be able to implicitly convert the types on either side of the `:` token.</span></span> <span data-ttu-id="1ea07-106">Кроме того, не может быть взаимно неявных преобразований; требуется только одно преобразование.</span><span class="sxs-lookup"><span data-stu-id="1ea07-106">Also, there cannot be mutual implicit conversions; you only need one conversion.</span></span>
  
<span data-ttu-id="1ea07-107">Следующий пример приводит к возникновению ошибки CS0172:</span><span class="sxs-lookup"><span data-stu-id="1ea07-107">The following sample generates CS0172:</span></span>
  
```csharp  
// CS0172.cs  
public class Square  
{  
   public class Circle  
   {  
      public static implicit operator Circle(Square aa)  
      {  
         return null;  
      }  
  
      public static implicit operator Square(Circle aa)  
      // using explicit resolves this error  
      // public static explicit operator Square(Circle aa)  
      {  
         return null;  
      }  
   }  
  
   public static void Main()  
   {  
      Circle aa = new Circle();  
      Square ii = new Square();  
      object o = (1 == 1) ? aa : ii;   // CS0172  
      // the following cast would resolve this error  
      // (1 == 1) ? aa : (Circle)ii;  
   }  
}  
```
