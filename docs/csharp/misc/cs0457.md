# <a name="compiler-error-cs0457"></a><span data-ttu-id="68a1f-102">Ошибка компилятора CS0457</span><span class="sxs-lookup"><span data-stu-id="68a1f-102">Compiler Error CS0457</span></span>
<span data-ttu-id="68a1f-103">Неоднозначные определенные пользователем преобразования "имя метода преобразования 1" и "имя метода преобразования 2" при преобразовании из "имя типа 1" в "имя типа 2"</span><span class="sxs-lookup"><span data-stu-id="68a1f-103">Ambiguous user defined conversions 'Conversion method name 1' and 'Conversion method name 2' when converting from 'type name 1' to 'type name 2'</span></span>  
  
 <span data-ttu-id="68a1f-104">Применяются два метода преобразования, и компилятор не может выбрать, какой из них использовать.</span><span class="sxs-lookup"><span data-stu-id="68a1f-104">Two conversion methods are applicable, and the compiler is unable to decide which one to use.</span></span>  
  
 <span data-ttu-id="68a1f-105">Один из сценариев, в котором может возникнуть такая ошибка, выглядит следующим образом:</span><span class="sxs-lookup"><span data-stu-id="68a1f-105">One scenario that can cause this error is as follows:</span></span>  
  
- <span data-ttu-id="68a1f-106">Вы хотите преобразовать класс A в класс B, где A и B не связаны.</span><span class="sxs-lookup"><span data-stu-id="68a1f-106">You want to convert class A to class B where A and B are unrelated.</span></span>  
  
- <span data-ttu-id="68a1f-107">Класс A является производным от A0, и существует метод, который преобразует A0 в B.</span><span class="sxs-lookup"><span data-stu-id="68a1f-107">A is derived from A0, and there is a method that converts from A0 to B.</span></span>  
  
- <span data-ttu-id="68a1f-108">Класс B имеет подкласс B1, и существует метод, который преобразует A в B1.</span><span class="sxs-lookup"><span data-stu-id="68a1f-108">B has a subclass B1 and there is a method that converts from A to B1.</span></span>  
  
 <span data-ttu-id="68a1f-109">Компилятор будет оценивать оба метода преобразования одинаково, так как первое преобразование обеспечивает наилучший тип назначения, а второе преобразование обеспечивает наилучший тип источника.</span><span class="sxs-lookup"><span data-stu-id="68a1f-109">The compiler will weight the two conversion methods equally, because the first conversion provides the best destination type, and the second conversion provides the best source type.</span></span> <span data-ttu-id="68a1f-110">Поскольку компилятор не сможет выбрать одно из этих преобразований, возникнет эта ошибка.</span><span class="sxs-lookup"><span data-stu-id="68a1f-110">Since the compiler will be unable to choose, this error will be generated.</span></span> <span data-ttu-id="68a1f-111">Чтобы разрешить ее, напишите новый явный метод преобразования A в B.</span><span class="sxs-lookup"><span data-stu-id="68a1f-111">To resolve, write a new explicit method converting A to B.</span></span>  
  
 <span data-ttu-id="68a1f-112">В другом сценарии, приводящем к этой ошибке, существует два метода, которые преобразуют A в B. Чтобы устранить проблему, укажите нужное преобразование посредством явного приведения.</span><span class="sxs-lookup"><span data-stu-id="68a1f-112">Another scenario that causes this error is if there are two methods that convert A to B. To fix, specify which conversion to use through an explicit cast.</span></span>  
  
## <a name="example"></a><span data-ttu-id="68a1f-113">Пример</span><span class="sxs-lookup"><span data-stu-id="68a1f-113">Example</span></span>  
 <span data-ttu-id="68a1f-114">В следующем примере возникает ошибка CS0457.</span><span class="sxs-lookup"><span data-stu-id="68a1f-114">The following sample generates CS0457.</span></span>  
  
```csharp  
// CS0457.cs  
using System;  
public class A { }  
  
public class G0 {  }  
public class G1<R> : G0 {  }  
  
public class H0 {  
   public static implicit operator G0(H0 h) {  
      return new G0();  
   }  
}  
public class H1<R> : H0 {  
   public static implicit operator G1<R>(H1<R> h) {  
      return new G1<R>();  
   }  
}  
  
public class Test   
{  
   public static void F0(G0 g) {  }  
   public static void Main()   
   {  
      H1<A> h1a = new H1<A>();  
      F0(h1a);   // CS0457  
   }  
}  
```
