# <a name="compiler-warning-level-2-cs0728"></a><span data-ttu-id="2df61-103">Предупреждение компилятора (уровень 2) CS0728</span><span class="sxs-lookup"><span data-stu-id="2df61-103">Compiler Warning (level 2) CS0728</span></span>
<span data-ttu-id="2df61-104">Возможно, используется недопустимое назначение для локальной переменной "переменная", которая является аргументом оператора using или lock.</span><span class="sxs-lookup"><span data-stu-id="2df61-104">Possibly incorrect assignment to local 'variable' which is the argument to a using or lock statement.</span></span>  <span data-ttu-id="2df61-105">В исходном значении локального объекта произойдет вызов Dispose или разблокирование.</span><span class="sxs-lookup"><span data-stu-id="2df61-105">The Dispose call or unlocking will happen on the original value of the local.</span></span>  
  
 <span data-ttu-id="2df61-106">Существует несколько сценариев, где блоки `using` или `lock` приведут к временной утечке ресурсов.</span><span class="sxs-lookup"><span data-stu-id="2df61-106">There are several scenarios where `using` or `lock` blocks will result in a temporary leak of resources.</span></span> <span data-ttu-id="2df61-107">Ниже приведен один пример.</span><span class="sxs-lookup"><span data-stu-id="2df61-107">Here is one example:</span></span>  
  
 `thisType f = null;`  
  
 `using (f)`  
  
 `{`  
  
 `f = new thisType();`  
  
 `...`  
  
 `}`  
  
 <span data-ttu-id="2df61-108">В этом случае исходное значение, например null, переменной `thisType` будет удалено, когда блок `using` закончит выполнение, но объект `thisType` , созданный в этом блоке, не будет удален, несмотря на то, что он будет в конечном итоге удален сборщиком мусора.</span><span class="sxs-lookup"><span data-stu-id="2df61-108">In this case, the original value, such as null, of the variable `thisType` will be disposed of when the `using` block finishes executing, but the `thisType` object created inside the block will not be, although it will eventually get garbage collected.</span></span>  
  
 <span data-ttu-id="2df61-109">Чтоб устранить эту ошибку, используйте следующую форму:</span><span class="sxs-lookup"><span data-stu-id="2df61-109">To resolve this error, use the following form:</span></span>  
  
 `using (thisType f = new thisType())`  
  
 `{`  
  
 `...`  
  
 `}`  
  
 <span data-ttu-id="2df61-110">В данном случае вновь выделенный объект `thisType` будет удален.</span><span class="sxs-lookup"><span data-stu-id="2df61-110">In this case, the newly allocated `thisType` object will be disposed of.</span></span>  
  
## <a name="example"></a><span data-ttu-id="2df61-111">Пример</span><span class="sxs-lookup"><span data-stu-id="2df61-111">Example</span></span>  
 <span data-ttu-id="2df61-112">Следующий код приводит к возникновению предупреждения CS0728.</span><span class="sxs-lookup"><span data-stu-id="2df61-112">The following code will generate warning CS0728.</span></span>  
  
```csharp
// CS0728.cs  
  
using System;  
public class ValidBase : IDisposable  
{  
    public void Dispose() {  }  
}  
  
public class Logger  
{  
    public static void dummy()  
    {  
        ValidBase vb = null;  
        using (vb)
        {  
            vb = null;  // CS0728  
        }  
        vb = null;  
    }  
    public static void Main() { }  
}  
```
