 <span data-ttu-id="ac76c-104">При использовании [-doc](../language-reference/compiler-options/doc-compiler-option.md)любые комментарии в исходном коде должны быть в формате XML.</span><span class="sxs-lookup"><span data-stu-id="ac76c-104">When using [-doc](../language-reference/compiler-options/doc-compiler-option.md), any comments in the source code must be in XML.</span></span> <span data-ttu-id="ac76c-105">Любая ошибка разметки XML вызывает предупреждение CS1570.</span><span class="sxs-lookup"><span data-stu-id="ac76c-105">Any error with your XML markup will generate CS1570.</span></span> <span data-ttu-id="ac76c-106">Пример:</span><span class="sxs-lookup"><span data-stu-id="ac76c-106">For example:</span></span>

- <span data-ttu-id="ac76c-108">Если вы используете тег, например [\<seealso >](../programming-guide/xmldoc/seealso.md), у которого нет закрывающего тега, перед закрывающей угловой скобкой необходимо указать прямую косую черту.</span><span class="sxs-lookup"><span data-stu-id="ac76c-108">If you are using a tag, such as [\<seealso>](../programming-guide/xmldoc/seealso.md), which does not have a closing tag, you must specify a forward slash before the closing angle bracket.</span></span>  
  
- <span data-ttu-id="ac76c-109">Если в тексте описания необходимо использовать символ "больше" или "меньше", следует представить эти символы как **&gt;** или **&lt;** .</span><span class="sxs-lookup"><span data-stu-id="ac76c-109">If you need to use a greater-than or less-than symbol in the text of description, you need to represent them with **&gt;** or **&lt;**.</span></span>  
  
- <span data-ttu-id="ac76c-110">Атрибут File или Path в теге [\<include >](../programming-guide/xmldoc/include.md) отсутствовал или неправильно сформирован.</span><span class="sxs-lookup"><span data-stu-id="ac76c-110">The file or path attribute on an [\<include>](../programming-guide/xmldoc/include.md) tag was missing or improperly formed.</span></span>
