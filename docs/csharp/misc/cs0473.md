<span data-ttu-id="43bde-104">Реализация явного интерфейса "имя_метода" соответствует более чем одному членом интерфейса.</span><span class="sxs-lookup"><span data-stu-id="43bde-104">Explicit interface implementation 'method name' matches more than one interface member.</span></span> <span data-ttu-id="43bde-105">Фактический выбор конкретного члена интерфейса зависит от реализации.</span><span class="sxs-lookup"><span data-stu-id="43bde-105">Which interface member is actually chosen is implementation-dependent.</span></span> <span data-ttu-id="43bde-106">Попробуйте вместо явной реализации использовать неявную реализацию интерфейса.</span><span class="sxs-lookup"><span data-stu-id="43bde-106">Consider using a non-explicit implementation instead.</span></span>

<span data-ttu-id="43bde-107">В некоторых случаях универсальный метод может получить такую же сигнатуру, что и неуниверсальный метод.</span><span class="sxs-lookup"><span data-stu-id="43bde-107">In some cases a generic method might acquire the same signature as a non-generic method.</span></span> <span data-ttu-id="43bde-108">Проблема заключается в том, что система метаданных Common Language Infrastructure (CLI) никаким способом не может однозначно установить, какой метод привязан к какому слоту.</span><span class="sxs-lookup"><span data-stu-id="43bde-108">The problem is that there is no way in the common language infrastructure (CLI) metadata system to unambiguously state which method binds to which slot.</span></span> <span data-ttu-id="43bde-109">Это определение делается на усмотрение CLI.</span><span class="sxs-lookup"><span data-stu-id="43bde-109">It is up to the CLI to make that determination.</span></span>

## <a name="to-correct-this-error"></a><span data-ttu-id="43bde-110">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="43bde-110">To correct this error</span></span>

<span data-ttu-id="43bde-111">Чтобы исправить эту ошибку, устраните явную реализацию и реализуйте оба метода интерфейса в неявной реализации `public int TestMethod(int)` .</span><span class="sxs-lookup"><span data-stu-id="43bde-111">To correct the error, eliminate the explicit implementation and implement both of the interface methods in the implicit implementation `public int TestMethod(int)`.</span></span>

## <a name="example"></a><span data-ttu-id="43bde-112">Пример</span><span class="sxs-lookup"><span data-stu-id="43bde-112">Example</span></span>

<span data-ttu-id="43bde-113">Следующий код приводит к возникновению ошибки CS0473:</span><span class="sxs-lookup"><span data-stu-id="43bde-113">The following code generates CS0473:</span></span>

```csharp
public interface ITest<T>
{
    int TestMethod(int i);
    int TestMethod(T i);
}

public class ImplementingClass : ITest<int>
{
    int ITest<int>.TestMethod(int i) // CS0473
    {
        return i + 1;
    }

    public int TestMethod(int i)
    {
        return i - 1;
    }
}

class T
{
    static int Main()
    {
        ImplementingClass a = new ImplementingClass();
        if (a.TestMethod(0) != -1)
            return -1;

        ITest<int> i_a = a;
        System.Console.WriteLine(i_a.TestMethod(0).ToString());
        if (i_a.TestMethod(0) != 1)
            return -1;

        return 0;
    }
}
```
