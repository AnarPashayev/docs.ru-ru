# <a name="compiler-error-cs0837"></a><span data-ttu-id="93b4c-102">Ошибка компилятора CS0837</span><span class="sxs-lookup"><span data-stu-id="93b4c-102">Compiler Error CS0837</span></span>
<span data-ttu-id="93b4c-103">Первый операнд оператора "is" или "AS" не может быть лямбда-выражением, анонимным методом или группой методов.</span><span class="sxs-lookup"><span data-stu-id="93b4c-103">The first operand of an 'is' or 'as' operator may not be a lambda expression, anonymous method, or method group.</span></span>

 <span data-ttu-id="93b4c-104">Лямбда-выражения, анонимные методы и группы методов нельзя использовать с левой стороны [—](../language-reference/operators/type-testing-and-cast.md#is-operator) или [как](../language-reference/operators/type-testing-and-cast.md#as-operator).</span><span class="sxs-lookup"><span data-stu-id="93b4c-104">Lambda expressions, anonymous methods and method groups may not be used on the left side of [is](../language-reference/operators/type-testing-and-cast.md#is-operator) or [as](../language-reference/operators/type-testing-and-cast.md#as-operator).</span></span>

## <a name="to-correct-this-error"></a><span data-ttu-id="93b4c-105">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="93b4c-105">To correct this error</span></span>

- <span data-ttu-id="93b4c-106">Если ошибка включает оператор `is` , учтите, что `is` принимает значение и тип и пытается преобразовать значение к этому типу с помощью ссылки, упаковки-преобразования или распаковки-преобразования.</span><span class="sxs-lookup"><span data-stu-id="93b4c-106">If the error involves the `is` operator, remember that `is` takes a value and a type and tells you whether the value can be made into that type by a reference, boxing, or unboxing conversion.</span></span> <span data-ttu-id="93b4c-107">Так как лямбда-выражения не являются значениями и к ним неприменимы ссылки, упаковки-преобразования и распаковки-преобразования, лямбда-выражения не являются кандидатами для `is`.</span><span class="sxs-lookup"><span data-stu-id="93b4c-107">Because lambdas are not values and have no reference, boxing, or unboxing conversions, lambdas are not candidates for `is`.</span></span>

- <span data-ttu-id="93b4c-108">Если код неправильно использует `as`, исправьте его для правильного выполнения приведения.</span><span class="sxs-lookup"><span data-stu-id="93b4c-108">If the code misuses `as`, the correction is probably to change it to a cast.</span></span>

## <a name="example"></a><span data-ttu-id="93b4c-109">Пример</span><span class="sxs-lookup"><span data-stu-id="93b4c-109">Example</span></span>

<span data-ttu-id="93b4c-110">Следующий пример приводит к возникновению ошибок CS0837:</span><span class="sxs-lookup"><span data-stu-id="93b4c-110">The following example generates CS0837:</span></span>

```csharp
// cs0837.cs
namespace TestNamespace
{
    public delegate void Del();

    class Test
    {
        static int Main()
        {
            bool b1 = (() => { }) is Del;   // CS0837
            bool b2 = delegate() { } is Del;// CS0837
            Del d1 = () => { } as Del;      // CS0837  
            Del d2 = delegate() { } as Del; // CS0837
            return 1;
        }
    }
}
```
