# <a name="string-c-reference"></a><span data-ttu-id="b3120-102">string (Справочник по C#)</span><span class="sxs-lookup"><span data-stu-id="b3120-102">string (C# Reference)</span></span>

<span data-ttu-id="b3120-103">Тип `string` представляет последовательность, состоящую из нуля или более символов в кодировке Юникод.</span><span class="sxs-lookup"><span data-stu-id="b3120-103">The `string` type represents a sequence of zero or more Unicode characters.</span></span> <span data-ttu-id="b3120-104">`string` является псевдонимом для <xref:System.String> в .NET.</span><span class="sxs-lookup"><span data-stu-id="b3120-104">`string` is an alias for <xref:System.String> in .NET.</span></span>

<span data-ttu-id="b3120-105">Несмотря на то, что `string` представляет собой ссылочный тип, операторы равенства (`==` и`!=`) по определению сравнивают не ссылки, а значения объектов `string`.</span><span class="sxs-lookup"><span data-stu-id="b3120-105">Although `string` is a reference type, the equality operators (`==` and `!=`) are defined to compare the values of `string` objects, not references.</span></span> <span data-ttu-id="b3120-106">Это делает проверку равенства строк более интуитивно понятной.</span><span class="sxs-lookup"><span data-stu-id="b3120-106">This makes testing for string equality more intuitive.</span></span> <span data-ttu-id="b3120-107">Например:</span><span class="sxs-lookup"><span data-stu-id="b3120-107">For example:</span></span>

```csharp
string a = "hello";
string b = "h";
// Append to contents of 'b'
b += "ello";
Console.WriteLine(a == b);
Console.WriteLine((object)a == (object)b);
```

<span data-ttu-id="b3120-108">Отображается значение True, а затем False, поскольку содержимое строк эквивалентно, однако `a` и `b` не относятся к одному и тому же экземпляру строки.</span><span class="sxs-lookup"><span data-stu-id="b3120-108">This displays "True" and then "False" because the content of the strings are equivalent, but `a` and `b` do not refer to the same string instance.</span></span>

<span data-ttu-id="b3120-109">Оператор + объединяет строки:</span><span class="sxs-lookup"><span data-stu-id="b3120-109">The + operator concatenates strings:</span></span>

```csharp
string a = "good " + "morning";
```

<span data-ttu-id="b3120-110">При этом создается строковый объект, содержащий слова "good morning".</span><span class="sxs-lookup"><span data-stu-id="b3120-110">This creates a string object that contains "good morning".</span></span>

<span data-ttu-id="b3120-111">Строки *неизменяемы*, т. е. содержимое созданного строкового объекта изменить нельзя, хотя синтаксис выглядит так, будто это возможно.</span><span class="sxs-lookup"><span data-stu-id="b3120-111">Strings are *immutable*--the contents of a string object cannot be changed after the object is created, although the syntax makes it appear as if you can do this.</span></span> <span data-ttu-id="b3120-112">Например, при написании кода компилятор фактически создает новый строковый объект для хранения новой последовательности символов, а затем этот новый объект назначается b.</span><span class="sxs-lookup"><span data-stu-id="b3120-112">For example, when you write this code, the compiler actually creates a new string object to hold the new sequence of characters, and that new object is assigned to b.</span></span> <span data-ttu-id="b3120-113">После этого строка h может быть отправлена в мусор.</span><span class="sxs-lookup"><span data-stu-id="b3120-113">The string "h" is then eligible for garbage collection.</span></span>

```csharp
string b = "h";
b += "ello";
```

<span data-ttu-id="b3120-114">Оператор [] позволяет предоставить к отдельным символам `string` доступ только для чтения:</span><span class="sxs-lookup"><span data-stu-id="b3120-114">The [] operator can be used for readonly access to individual characters of a `string`:</span></span>

```csharp
string str = "test";
char x = str[2];  // x = 's';
```

<span data-ttu-id="b3120-115">Также оператор [] можно использовать для итерации каждого символа в объекте `string`:</span><span class="sxs-lookup"><span data-stu-id="b3120-115">In similar fashion, the [] operator can also be used for iterating over each character in a `string`:</span></span>

```csharp
string str = "test";

for (int i = 0; i < str.Length; i++)
{
  Console.Write(str[i] + " ");
}
// Output: t e s t
``` 

<span data-ttu-id="b3120-116">Строковые литералы имеют тип `string` и могут быть записаны в двух формах: в кавычках (") или в кавычках с префиксом @-quoted.</span><span class="sxs-lookup"><span data-stu-id="b3120-116">String literals are of type `string` and can be written in two forms, quoted and @-quoted.</span></span> <span data-ttu-id="b3120-117">Строковые литералы в кавычках заключаются в двойные кавычки ("):</span><span class="sxs-lookup"><span data-stu-id="b3120-117">Quoted string literals are enclosed in double quotation marks ("):</span></span>

```csharp
"good morning"  // a string literal
```

<span data-ttu-id="b3120-118">Строковые литералы могут содержать любые символьные литералы.</span><span class="sxs-lookup"><span data-stu-id="b3120-118">String literals can contain any character literal.</span></span> <span data-ttu-id="b3120-119">Escape-последовательности включены.</span><span class="sxs-lookup"><span data-stu-id="b3120-119">Escape sequences are included.</span></span> <span data-ttu-id="b3120-120">В следующем примере escape-последовательность `\\` используется для получения обратной косой черты, `\u0066` — для получения буквы f, и `\n` — для получения новой строки.</span><span class="sxs-lookup"><span data-stu-id="b3120-120">The following example uses escape sequence `\\` for backslash, `\u0066` for the letter f, and `\n` for newline.</span></span>

```csharp
string a = "\\\u0066\n";
Console.WriteLine(a);
```

> [!NOTE]
> <span data-ttu-id="b3120-121">Escape-код `\udddd` (где `dddd` состоит из четырех цифр) представляет символ Юникода U+`dddd`.</span><span class="sxs-lookup"><span data-stu-id="b3120-121">The escape code `\udddd` (where `dddd` is a four-digit number) represents the Unicode character U+`dddd`.</span></span> <span data-ttu-id="b3120-122">Также распознаются восьмизначные escape-коды Юникода: `\Udddddddd`.</span><span class="sxs-lookup"><span data-stu-id="b3120-122">Eight-digit Unicode escape codes are also recognized: `\Udddddddd`.</span></span>

<span data-ttu-id="b3120-123">Буквальные строковые литералы начинаются с `@` и также заключаются в двойные кавычки.</span><span class="sxs-lookup"><span data-stu-id="b3120-123">Verbatim string literals start with `@` and are also enclosed in double quotation marks.</span></span> <span data-ttu-id="b3120-124">Пример:</span><span class="sxs-lookup"><span data-stu-id="b3120-124">For example:</span></span>

```csharp
@"good morning"  // a string literal
```

<span data-ttu-id="b3120-125">Преимущество буквальных строк состоит в том, что escape-последовательности *не* обрабатываются, что позволяет легко написать, например, полное имя файла:</span><span class="sxs-lookup"><span data-stu-id="b3120-125">The advantage of verbatim strings is that escape sequences are *not* processed, which makes it easy to write, for example, a fully qualified file name:</span></span>

```csharp
@"c:\Docs\Source\a.txt"  // rather than "c:\\Docs\\Source\\a.txt"
```

<span data-ttu-id="b3120-126">Чтобы добавить в строку с префиксом @-quoted двойные кавычки, продублируйте их:</span><span class="sxs-lookup"><span data-stu-id="b3120-126">To include a double quotation mark in an @-quoted string, double it:</span></span>

```csharp
@"""Ahoy!"" cried the captain." // "Ahoy!" cried the captain.
```

<span data-ttu-id="b3120-127">Другие варианты использования специального символа `@` см. в разделе [@ — буквальный идентификатор](../tokens/verbatim.md).</span><span class="sxs-lookup"><span data-stu-id="b3120-127">For other uses of the `@` special character, see [@ -- verbatim identifier](../tokens/verbatim.md).</span></span>

<span data-ttu-id="b3120-128">Дополнительные сведения о строках в С# см. в разделе [Строки](../../programming-guide/strings/index.md).</span><span class="sxs-lookup"><span data-stu-id="b3120-128">For more information about strings in C#, see [Strings](../../programming-guide/strings/index.md).</span></span>
