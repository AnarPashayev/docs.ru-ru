## <a name="ref-struct-types"></a><span data-ttu-id="e9810-176">Типы ссылочных структур</span><span class="sxs-lookup"><span data-stu-id="e9810-176">Ref struct types</span></span>

<span data-ttu-id="e9810-177">Добавление модификатора `ref` в объявление `struct` определяет, что экземпляры этого типа должны размещаться в стеке.</span><span class="sxs-lookup"><span data-stu-id="e9810-177">Adding the `ref` modifier to a `struct` declaration defines that instances of that type must be stack allocated.</span></span> <span data-ttu-id="e9810-178">Другими словами, экземпляры этих типов никогда не создаются из кучи в роли члена другого класса.</span><span class="sxs-lookup"><span data-stu-id="e9810-178">In other words, instances of these types can never be created on the heap as a member of another class.</span></span> <span data-ttu-id="e9810-179">Главным стимулом для создания этой функции была структура <xref:System.Span%601> и связанные структуры.</span><span class="sxs-lookup"><span data-stu-id="e9810-179">The primary motivation for this feature was <xref:System.Span%601> and related structures.</span></span>

<span data-ttu-id="e9810-180">Необходимость поддержки типа `ref struct` как выделенной в стеке переменной вводит ряд правил, применяемых компилятором ко всем типам `ref struct`.</span><span class="sxs-lookup"><span data-stu-id="e9810-180">The goal of keeping a `ref struct` type as a stack-allocated variable introduces several rules that the compiler enforces for all `ref struct` types.</span></span>

- <span data-ttu-id="e9810-181">`ref struct` не поддерживает упаковку.</span><span class="sxs-lookup"><span data-stu-id="e9810-181">You can't box a `ref struct`.</span></span> <span data-ttu-id="e9810-182">Тип `ref struct` невозможно присвоить переменной типа `object`, `dynamic` или любому типу интерфейса.</span><span class="sxs-lookup"><span data-stu-id="e9810-182">You cannot assign a `ref struct` type to a variable of type `object`, `dynamic`, or any interface type.</span></span>
- <span data-ttu-id="e9810-183">Типы `ref struct` не могут реализовывать интерфейсы.</span><span class="sxs-lookup"><span data-stu-id="e9810-183">`ref struct` types cannot implement interfaces.</span></span>
- <span data-ttu-id="e9810-184">`ref struct` невозможно объявить как член поля класса или обычной структуры.</span><span class="sxs-lookup"><span data-stu-id="e9810-184">You can't declare a `ref struct` as a field member of a class or a normal struct.</span></span> <span data-ttu-id="e9810-185">Сюда входит объявление автоматически реализуемого свойства, которое создает резервное поле, созданное компилятором.</span><span class="sxs-lookup"><span data-stu-id="e9810-185">This includes declaring an auto-implemented property, which creates a compiler generated backing field.</span></span>

- <span data-ttu-id="e9810-188">Локальные переменные `ref struct` невозможно объявить в итераторах.</span><span class="sxs-lookup"><span data-stu-id="e9810-188">You cannot declare `ref struct` local variables in iterators.</span></span>
- <span data-ttu-id="e9810-189">Невозможно захватить переменные `ref struct` в лямбда-выражениях или локальных функциях.</span><span class="sxs-lookup"><span data-stu-id="e9810-189">You cannot capture `ref struct` variables in lambda expressions or local functions.</span></span>

<span data-ttu-id="e9810-190">Эти ограничения позволяют избежать случайного использования `ref struct`, которое может повысить его уровень до управляемой кучи.</span><span class="sxs-lookup"><span data-stu-id="e9810-190">These restrictions ensure you don't accidentally use a `ref struct` in a manner that could promote it to the managed heap.</span></span>

<span data-ttu-id="e9810-191">Вы можете сочетать модификаторы и объявить структуру как `readonly ref`.</span><span class="sxs-lookup"><span data-stu-id="e9810-191">You can combine modifiers to declare a struct as `readonly ref`.</span></span> <span data-ttu-id="e9810-192">Объявление `readonly ref struct` объединяет все преимущества и ограничения объявлений `ref struct` и `readonly struct`.</span><span class="sxs-lookup"><span data-stu-id="e9810-192">A `readonly ref struct` combines the benefits and restrictions of `ref struct` and `readonly struct` declarations.</span></span>
