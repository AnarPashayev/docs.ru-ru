# <a name="compiler-error-cs0029"></a><span data-ttu-id="efe69-103">Ошибка компилятора CS0029</span><span class="sxs-lookup"><span data-stu-id="efe69-103">Compiler Error CS0029</span></span>

<span data-ttu-id="efe69-104">Не удается неявно преобразовать тип "type" в "type"</span><span class="sxs-lookup"><span data-stu-id="efe69-104">Cannot implicitly convert type 'type' to 'type'</span></span>

 <span data-ttu-id="efe69-105">Компилятору требуется явное преобразование.</span><span class="sxs-lookup"><span data-stu-id="efe69-105">The compiler requires an explicit conversion.</span></span> <span data-ttu-id="efe69-106">Например, может потребоваться приведение r-значения к тому же типу, который имеет l-значение.</span><span class="sxs-lookup"><span data-stu-id="efe69-106">For example, you may need to cast an r-value to be the same type as an l-value.</span></span> <span data-ttu-id="efe69-107">Или может быть необходимо предоставить подпрограммы преобразования для поддержки перегрузки определенных операторов.</span><span class="sxs-lookup"><span data-stu-id="efe69-107">Or, you must provide conversion routines to support certain operator overloads.</span></span>

 <span data-ttu-id="efe69-108">Преобразование должно выполняться в том случае, когда переменная одного типа присваивается переменной другого типа.</span><span class="sxs-lookup"><span data-stu-id="efe69-108">Conversions must occur when assigning a variable of one type to a variable of a different type.</span></span> <span data-ttu-id="efe69-109">При присвоении между переменными разных типов компилятор должен преобразовать тип в правой части оператора присваивания в тип в его левой части.</span><span class="sxs-lookup"><span data-stu-id="efe69-109">When making an assignment between variables of different types, the compiler must convert the type on the right-hand side of the assignment operator to the type on the left-hand side of the assignment operator.</span></span> <span data-ttu-id="efe69-110">Рассмотрим следующий код:</span><span class="sxs-lookup"><span data-stu-id="efe69-110">Take the following the code:</span></span>

```csharp
int i = 50;
long lng = 100;
i = lng;
```

 <span data-ttu-id="efe69-111">`i = lng;` выполняет присваивание, однако типы данных переменных в левой и правой частях оператора присваивания не совпадают.</span><span class="sxs-lookup"><span data-stu-id="efe69-111">`i = lng;` makes an assignment, but the data types of the variables on the left and right-hand side of the assignment operator don't match.</span></span> <span data-ttu-id="efe69-112">Прежде чем выполнять присваивание, компилятор неявно преобразует переменную `lng` типа long в тип int. Выполнение этого преобразования не предписывается явно какими-либо инструкциями кода и поэтому называется неявным.</span><span class="sxs-lookup"><span data-stu-id="efe69-112">Before making the assignment the compiler is implicitly converting the variable `lng`, which is of type long, to an int. This is implicit because no code explicitly instructed the compiler to perform this conversion.</span></span> <span data-ttu-id="efe69-113">Обратите внимание, что в этом коде реализуется неявное сужающее преобразование, которое не допускается компилятором из-за риска потери данных.</span><span class="sxs-lookup"><span data-stu-id="efe69-113">The problem with this code is that this is considered a narrowing conversion, and the compiler does not allow implicit narrowing conversions because there could be a potential loss of data.</span></span>

 <span data-ttu-id="efe69-114">Преобразование является сужающим в тех случаях, когда целевой тип данных занимает в памяти меньше места, чем исходный.</span><span class="sxs-lookup"><span data-stu-id="efe69-114">A narrowing conversion exists when converting to a data type that occupies less storage space in memory than the data type we are converting from.</span></span> <span data-ttu-id="efe69-115">Например, сужающим является преобразование из long в int.</span><span class="sxs-lookup"><span data-stu-id="efe69-115">For example, converting a long to an int would be considered a narrowing conversion.</span></span> <span data-ttu-id="efe69-116">Тип long занимает в памяти 8 байт, а тип int — 4 байта.</span><span class="sxs-lookup"><span data-stu-id="efe69-116">A long occupies 8 bytes of memory while an int occupies 4 bytes.</span></span> <span data-ttu-id="efe69-117">Ниже показан пример потери данных в результате такого преобразования:</span><span class="sxs-lookup"><span data-stu-id="efe69-117">To see how data loss can occur, consider the following sample:</span></span>

```csharp
int i = 50;
long lng = 3147483647;
i = lng;
```

 <span data-ttu-id="efe69-118">Переменная `lng` содержит слишком большое значение, которое нельзя сохранить в переменной `i`.</span><span class="sxs-lookup"><span data-stu-id="efe69-118">The variable `lng` now contains a value that cannot be stored in the variable `i` because it is too large.</span></span> <span data-ttu-id="efe69-119">Таким образом, при преобразовании этого значения в тип int потеряется часть данных, а преобразованное значение не будет в точности равно исходному.</span><span class="sxs-lookup"><span data-stu-id="efe69-119">If we were to convert this value to an int type we would be losing some of our data and the converted value would not be the same as the value before the conversion.</span></span>

 <span data-ttu-id="efe69-120">Расширяющие преобразования являются противоположностью сужающих.</span><span class="sxs-lookup"><span data-stu-id="efe69-120">A widening conversion would be the opposite of a narrowing conversion.</span></span> <span data-ttu-id="efe69-121">В этом случае целевой тип данных занимает в памяти больше места, чем исходный.</span><span class="sxs-lookup"><span data-stu-id="efe69-121">With widening conversions, we are converting to a data type that occupies more storage space in memory than the data type we are converting from.</span></span> <span data-ttu-id="efe69-122">Ниже приведен пример расширяющего преобразования:</span><span class="sxs-lookup"><span data-stu-id="efe69-122">Here is an example of a widening conversion:</span></span>

```csharp
int i = 50;
long lng = 100;
lng = i;
```

 <span data-ttu-id="efe69-123">Обратите внимание на отличие этого примера кода от первого.</span><span class="sxs-lookup"><span data-stu-id="efe69-123">Notice the difference between this code sample and the first.</span></span> <span data-ttu-id="efe69-124">На этот раз переменная `lng` находится в левой части оператора присваивания и является его целевой переменной.</span><span class="sxs-lookup"><span data-stu-id="efe69-124">This time the variable `lng` is on the left-hand side of the assignment operator, so it is the target of our assignment.</span></span> <span data-ttu-id="efe69-125">Перед присвоением компилятор должен неявно преобразовать переменную `i` типа int в тип long.</span><span class="sxs-lookup"><span data-stu-id="efe69-125">Before the assignment can be made, the compiler must implicitly convert the variable `i`, which is of type int, to type long.</span></span> <span data-ttu-id="efe69-126">Это будет расширяющее преобразование, поскольку исходный тип (int) занимает в памяти 4 байта, а целевой (long) — 8 байт.</span><span class="sxs-lookup"><span data-stu-id="efe69-126">This is a widening conversion since we are converting from a type that occupies 4 bytes of memory (an int) to a type that occupies 8 bytes of memory (a long).</span></span> <span data-ttu-id="efe69-127">Неявные расширяющие преобразования не связаны с риском потери данных и допускаются к применению.</span><span class="sxs-lookup"><span data-stu-id="efe69-127">Implicit widening conversions are allowed because there is no potential loss of data.</span></span> <span data-ttu-id="efe69-128">Любое значение типа int может быть сохранено в переменной типа long.</span><span class="sxs-lookup"><span data-stu-id="efe69-128">Any value that can be stored in an int can also be stored in a long.</span></span>

 <span data-ttu-id="efe69-129">Поскольку неявные сужающие преобразования не допускаются, для успешной компиляции этого кода необходимо явно выполнить преобразование типа данных.</span><span class="sxs-lookup"><span data-stu-id="efe69-129">We know that implicit narrowing conversions are not allowed, so to be able to compile this code we need to explicitly convert the data type.</span></span> <span data-ttu-id="efe69-130">Явные преобразования выполняются путем приведения типов.</span><span class="sxs-lookup"><span data-stu-id="efe69-130">Explicit conversions are done using casting.</span></span> <span data-ttu-id="efe69-131">Понятие приведения типов в языке C# описывает преобразование одного типа в другой.</span><span class="sxs-lookup"><span data-stu-id="efe69-131">Casting is the term used in C# to describe converting one data type to another.</span></span> <span data-ttu-id="efe69-132">Чтобы гарантировать успешную компиляцию кода, необходимо использовать следующий синтаксис:</span><span class="sxs-lookup"><span data-stu-id="efe69-132">To get the code to compile we would need to use the following syntax:</span></span>

```csharp

long lng = 100;
i = (int) lng;   // Cast to int.
```

 <span data-ttu-id="efe69-133">В третьей строке кода содержится инструкция для явного преобразования переменной `lng` типа long в тип int перед выполнением присвоения.</span><span class="sxs-lookup"><span data-stu-id="efe69-133">The third line of code tells the compiler to explicitly convert the variable `lng`, which is of type long, to an int before making the assignment.</span></span> <span data-ttu-id="efe69-134">Помните, что при сужающем преобразовании существует риск потери данных.</span><span class="sxs-lookup"><span data-stu-id="efe69-134">Remember that with a narrowing conversion, there is a potential loss of data.</span></span> <span data-ttu-id="efe69-135">Сужающие преобразования следует использовать с осторожностью, поскольку даже в случае успешной компиляции кода во время выполнения могут быть получены непредвиденные результаты.</span><span class="sxs-lookup"><span data-stu-id="efe69-135">Narrowing conversions should be used with caution and even though the code will compile you may get unexpected results at run-time.</span></span>

 <span data-ttu-id="efe69-136">Приведенная здесь информация относится только к типам значений.</span><span class="sxs-lookup"><span data-stu-id="efe69-136">This discussion has only been for value types.</span></span> <span data-ttu-id="efe69-137">При использовании типов значений вы работаете напрямую с данными, хранящимися в переменной.</span><span class="sxs-lookup"><span data-stu-id="efe69-137">When working with value types you work directly with the data stored in the variable.</span></span> <span data-ttu-id="efe69-138">Но в .NET также есть ссылочные типы.</span><span class="sxs-lookup"><span data-stu-id="efe69-138">However, .NET also has reference types.</span></span> <span data-ttu-id="efe69-139">В случае с ними вы работаете не с фактическими данными, а со ссылками на переменные.</span><span class="sxs-lookup"><span data-stu-id="efe69-139">When working with reference types you are working with a reference to a variable, not the actual data.</span></span> <span data-ttu-id="efe69-140">Например, к ссылочным типам относятся классы, интерфейсы и массивы.</span><span class="sxs-lookup"><span data-stu-id="efe69-140">Examples of reference types would be classes, interfaces and arrays.</span></span> <span data-ttu-id="efe69-141">Неявные или явные преобразования между ссылочными типами не поддерживаются кроме случаев, когда компилятор допускает конкретные преобразования или реализованы соответствующие операторы преобразования.</span><span class="sxs-lookup"><span data-stu-id="efe69-141">You cannot implicitly or explicitly convert one reference type to another unless the compiler allows the specific conversion or the appropriate conversion operators are implemented.</span></span>

 <span data-ttu-id="efe69-142">В следующем примере возникает ошибка CS0029:</span><span class="sxs-lookup"><span data-stu-id="efe69-142">The following sample generates CS0029:</span></span>

```csharp
// CS0029.cs
public class MyInt
{
    private int x = 0;

    // Uncomment this conversion routine to resolve CS0029.
    /*
    public static implicit operator int(MyInt i)
    {
        return i.x;
    }
    */

    public static void Main()
    {
        var myInt = new MyInt();
        int i = myInt; // CS0029
    }
```
