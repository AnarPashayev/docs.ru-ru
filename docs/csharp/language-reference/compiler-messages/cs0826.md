# <a name="compiler-error-cs0826"></a><span data-ttu-id="ed7e3-102">Ошибка компилятора CS0826</span><span class="sxs-lookup"><span data-stu-id="ed7e3-102">Compiler Error CS0826</span></span>
<span data-ttu-id="ed7e3-103">Нет подходящего типа для неявно типизированного массива.</span><span class="sxs-lookup"><span data-stu-id="ed7e3-103">No best type found for implicitly typed array.</span></span>  
  
 <span data-ttu-id="ed7e3-104">Все элементы массива должны иметь одинаковый тип или допускать неявное преобразование в один тип в зависимости от правил определения типа, используемых компилятором.</span><span class="sxs-lookup"><span data-stu-id="ed7e3-104">Array elements must all be the same type or implicitly convertible to the same type according to the type inference rules used by the compiler.</span></span> <span data-ttu-id="ed7e3-105">В качестве наилучшего типа должен использоваться один из типов, представленных в выражении массива.</span><span class="sxs-lookup"><span data-stu-id="ed7e3-105">The best type must be one of the types present in the array expression.</span></span> <span data-ttu-id="ed7e3-106">Элементы не будут преобразовываться в новый тип, например `object`.</span><span class="sxs-lookup"><span data-stu-id="ed7e3-106">Elements will not be converted to a new type such as `object`.</span></span> <span data-ttu-id="ed7e3-107">Для неявно типизированного массива компилятор должен определять тип массива на основе назначенного ему типа элементов.</span><span class="sxs-lookup"><span data-stu-id="ed7e3-107">For an implicitly typed array, the compiler must infer the array type based on the type of elements assigned to it.</span></span>

## <a name="example"></a><span data-ttu-id="ed7e3-112">Пример</span><span class="sxs-lookup"><span data-stu-id="ed7e3-112">Example</span></span>  
 <span data-ttu-id="ed7e3-113">В следующем коде возникает ошибка CS0826, поскольку элементы массива имеют разные типы, а логика определения типа компилятора не позволяет выявить один наилучший тип:</span><span class="sxs-lookup"><span data-stu-id="ed7e3-113">The following code generates CS0826 because the array elements are not all the same type, and the compiler's type inference logic does not find a single best type:</span></span>  
  
```csharp  
// cs0826.cs  
public class C  
{  
    public static int Main()  
    {  
        var x = new[] { 1, "str" }; // CS0826  
  
        char c = 'c';  
        short s1 = 0;  
        short s2 = -0;  
        short s3 = 1;  
        short s4 = -1;  
  
        var array1 = new[] { s1, s2, s3, s4, c, '1' }; // CS0826  
        return 1;  
    }  
}  
```
