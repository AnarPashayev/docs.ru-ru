# <a name="compiler-warning-level-1-cs0465"></a><span data-ttu-id="6424c-102">Предупреждение компилятора (уровень 1) CS0465</span><span class="sxs-lookup"><span data-stu-id="6424c-102">Compiler Warning (level 1) CS0465</span></span>
<span data-ttu-id="6424c-103">Введение метода "Finalize" может конфликтовать с вызовом деструктора.</span><span class="sxs-lookup"><span data-stu-id="6424c-103">Introducing a 'Finalize' method can interfere with destructor invocation.</span></span> <span data-ttu-id="6424c-104">Деструктор был объявлен намеренно?</span><span class="sxs-lookup"><span data-stu-id="6424c-104">Did you intend to declare a destructor?</span></span>  
  
 <span data-ttu-id="6424c-105">Это предупреждение возникает при создании класса, содержащего метод с сигнатурой `public virtual void Finalize`.</span><span class="sxs-lookup"><span data-stu-id="6424c-105">This warning occurs when you create a class with a method whose signature is `public virtual void Finalize`.</span></span>  
  
 <span data-ttu-id="6424c-106">Если такой класс используется в качестве базового класса и в производном классе определяется деструктор, этот деструктор переопределит метод базового класса `Finalize`, а не <xref:System.Object.Finalize%2A>.</span><span class="sxs-lookup"><span data-stu-id="6424c-106">If such a class is used as a base class and if the deriving class defines a destructor, the destructor will override the base class `Finalize` method, not <xref:System.Object.Finalize%2A>.</span></span>  
  
## <a name="example"></a><span data-ttu-id="6424c-107">Пример</span><span class="sxs-lookup"><span data-stu-id="6424c-107">Example</span></span>  
 <span data-ttu-id="6424c-108">В следующем примере возникает предупреждение CS0465.</span><span class="sxs-lookup"><span data-stu-id="6424c-108">The following sample generates CS0465.</span></span>  
  
```csharp  
// CS0465.cs  
// compile with: /target:library  
class A  
{  
   public virtual void Finalize() {}   // CS0465  
}  
  
// OK  
class B  
{  
   ~B() {}  
}  
```
