<span data-ttu-id="3e15d-104">Если ссылка на сборку "Имя сборки 1" соответствует "Имя сборки 2", может понадобиться предоставить политику времени выполнения</span><span class="sxs-lookup"><span data-stu-id="3e15d-104">Assuming assembly reference "Assembly Name #1" matches "Assembly Name #2", you may need to supply runtime policy</span></span>  
  
 <span data-ttu-id="3e15d-105">Две сборки имеют разные номера выпуска и (или) версии.</span><span class="sxs-lookup"><span data-stu-id="3e15d-105">The two assemblies differ in release and/or version number.</span></span> <span data-ttu-id="3e15d-106">Для унификации необходимо указать директивы в CONFIG-файле приложения и предоставить правильное строгое имя сборки, как показано в следующем примере кода.</span><span class="sxs-lookup"><span data-stu-id="3e15d-106">For unification to occur, you must specify directives in the application's .config file, and you must provide the correct strong name of an assembly, as demonstrated in the following example code.</span></span>  
  
## <a name="example"></a><span data-ttu-id="3e15d-107">Пример</span><span class="sxs-lookup"><span data-stu-id="3e15d-107">Example</span></span>  

 <span data-ttu-id="3e15d-108">В следующем примере с несколькими файлами ссылка на сборку задается с использованием двух разных внешних псевдонимов.</span><span class="sxs-lookup"><span data-stu-id="3e15d-108">The following multifile sample references an assembly using two different external aliases.</span></span> <span data-ttu-id="3e15d-109">В первом примере выполняется построение старой версии кода, в результате чего создается сборка CS1701_d.</span><span class="sxs-lookup"><span data-stu-id="3e15d-109">This first sample builds the older version of the code that creates assembly CS1701_d.</span></span>  
  
```csharp  
// CS1701_a.cs  
// compile with: /target:library /out:cs1701_d.dll /keyfile:mykey.snk  
using System.Reflection;  
[assembly:AssemblyVersion("1.0")]  
public class A {  
   public void M1() {}  
}  
  
public class C1 {}  
```  
  
## <a name="example"></a><span data-ttu-id="3e15d-110">Пример</span><span class="sxs-lookup"><span data-stu-id="3e15d-110">Example</span></span>  

 <span data-ttu-id="3e15d-111">Этот код создает более новую версию сборки CS1701_d.</span><span class="sxs-lookup"><span data-stu-id="3e15d-111">This is the code that creates the newer version of assembly CS1701_d.</span></span> <span data-ttu-id="3e15d-112">Обратите внимание, что компиляция осуществляется в каталог, отличный от каталога старой версии, поскольку выходные файлы имеют одинаковые имена.</span><span class="sxs-lookup"><span data-stu-id="3e15d-112">Note that it compiles into a different directory than the older version, necessary since the output files have the same names.</span></span>

```csharp  
// compile with: /target:library /out:c:\\cs1701_d.dll /keyfile:mykey.snk  
using System.Reflection;  
[assembly:AssemblyVersion("2.0")]  
public class A {  
   public void M2() {}  
   public void M1() {}  
}  
  
public class C2 {}  
public class C1 {}  
```  
  
## <a name="example"></a><span data-ttu-id="3e15d-113">Пример</span><span class="sxs-lookup"><span data-stu-id="3e15d-113">Example</span></span>  

 <span data-ttu-id="3e15d-114">В этом примере задаются внешние псевдонимы A1 и A2.</span><span class="sxs-lookup"><span data-stu-id="3e15d-114">This sample sets up the external aliases A1 and A2.</span></span>  
  
```csharp  
// CS1701_c.cs  
// compile with: /target:library /reference:A2=c:\\cs1701_d.dll /reference:A1=cs1701_d.dll  
  
extern alias A1;  
extern alias A2;  
// using System;  
using a1 = A1::A;  
using a2 = A2::A;  
  
public class Ref {  
   public static a1 A1() { return new a1(); }  
   public static a2 A2() { return new a2(); }  
  
   public static A1::C1 M1() { return new A1::C1(); }  
   public static A2::C2 M2() { return new A2::C2(); }  
}  
```
