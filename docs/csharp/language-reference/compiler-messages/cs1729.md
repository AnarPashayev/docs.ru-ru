# <a name="compiler-error-cs1729"></a><span data-ttu-id="6811a-103">Ошибка компилятора CS1729</span><span class="sxs-lookup"><span data-stu-id="6811a-103">Compiler Error CS1729</span></span>

<span data-ttu-id="6811a-104">"type" не содержит конструктор, принимающий аргументы: "number".</span><span class="sxs-lookup"><span data-stu-id="6811a-104">'type' does not contain a constructor that takes 'number' arguments.</span></span>  
  
 <span data-ttu-id="6811a-105">Эта ошибка возникает, когда прямо или косвенно вызывается конструктор какого-либо класса, но компилятор не может найти ни один конструктор с указанным числом параметров.</span><span class="sxs-lookup"><span data-stu-id="6811a-105">This error occurs when you either directly or indirectly invoke the constructor of a class but the compiler cannot find any constructors with the same number of parameters.</span></span> <span data-ttu-id="6811a-106">В следующем примере класс `test` не имеет конструкторов, которые принимают аргументы.</span><span class="sxs-lookup"><span data-stu-id="6811a-106">In the following example, the `test` class has no constructors that take any arguments.</span></span> <span data-ttu-id="6811a-107">Таким образом, у него есть только конструктор без параметров, который принимает ноль аргументов.</span><span class="sxs-lookup"><span data-stu-id="6811a-107">It therefore has only a parameterless constructor that takes zero arguments.</span></span> <span data-ttu-id="6811a-108">Так как во второй строке, где возникает ошибка, производный класс не объявляет никакие конструкторы, компилятор предоставляет конструктор без параметров.</span><span class="sxs-lookup"><span data-stu-id="6811a-108">Because in the second line in which the error is generated, the derived class declares no constructors of its own, the compiler provides a parameterless constructor.</span></span> <span data-ttu-id="6811a-109">Этот конструктор вызывает конструктор базового класса без параметров.</span><span class="sxs-lookup"><span data-stu-id="6811a-109">That constructor invokes a parameterless constructor in the base class.</span></span> <span data-ttu-id="6811a-110">Поскольку базовый класс не имеет такого конструктора, возникает ошибка CS1729.</span><span class="sxs-lookup"><span data-stu-id="6811a-110">Because the base class has no such constructor, CS1729 is generated.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="6811a-111">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="6811a-111">To correct this error</span></span>

## <a name="example"></a><span data-ttu-id="6811a-115">Пример</span><span class="sxs-lookup"><span data-stu-id="6811a-115">Example</span></span>  

 <span data-ttu-id="6811a-116">Следующий пример приводит к возникновению ошибки CS1729:</span><span class="sxs-lookup"><span data-stu-id="6811a-116">The following example generates CS1729:</span></span>  
  
```csharp  
// cs1729.cs  
class Test  
{  
    static int Main()  
    {  
        // Class Test has only a parameterless constructor, which takes no arguments.  
        Test test1 = new Test(2); // CS1729  
        // The following line resolves the error.  
        Test test2 = new Test();  
  
        // Class Parent has only one constructor, which takes two int parameters.  
        Parent exampleParent1 = new Parent(10); // CS1729  
        // The following line resolves the error.  
        Parent exampleParent2 = new Parent(10, 1);  
  
        return 1;  
    }  
}  
  
public class Parent  
{  
    // The only constructor for this class has two parameters.  
    public Parent(int i, int j) { }  
}  
  
// The following declaration causes a compiler error because class Parent  
// does not have a constructor that takes no arguments. The declaration of  
// class Child2 shows how to resolve this error.  
public class Child : Parent { } // CS1729  
  
public class Child2 : Parent  
{  
    // The constructor for Child2 has only one parameter. To access the
    // constructor in Parent, and prevent this compiler error, you must provide
    // a value for the second parameter of Parent. The following example provides 0.  
    public Child2(int k)  
        : base(k, 0)  
    {  
        // Add the body of the constructor here.  
    }  
}  
```
