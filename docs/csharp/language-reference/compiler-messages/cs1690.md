# <a name="compiler-warning-level-1-cs1690"></a><span data-ttu-id="847f8-102">Предупреждение компилятора (уровень 1) CS1690</span><span class="sxs-lookup"><span data-stu-id="847f8-102">Compiler Warning (level 1) CS1690</span></span>
<span data-ttu-id="847f8-103">Доступ к члену в "член" может вызвать исключение времени исполнения, поскольку он является полем класса, который маршалируется по ссылке</span><span class="sxs-lookup"><span data-stu-id="847f8-103">Accessing a member on 'member' may cause a runtime exception because it is a field of a marshal-by-reference class</span></span>  
  
 <span data-ttu-id="847f8-104">Это предупреждение возникает при попытке вызвать метод, свойство или индексатор для члена класса, производного от <xref:System.MarshalByRefObject>, если этот член имеет тип значения.</span><span class="sxs-lookup"><span data-stu-id="847f8-104">This warning occurs when you try to call a method, property, or indexer on a member of a class that derives from <xref:System.MarshalByRefObject>, and the member is a value type.</span></span> <span data-ttu-id="847f8-105">Объекты, наследуемые от `MarshalByRefObject`, как правило, предназначены для маршалирования по ссылке на уровне домена приложения.</span><span class="sxs-lookup"><span data-stu-id="847f8-105">Objects that inherit from `MarshalByRefObject` are typically intended to be marshaled by reference across an application domain.</span></span> <span data-ttu-id="847f8-106">Если в коде предпринимается попытка получить прямой доступ к члену такого объекта с типом значения на уровне домена приложения, возникнет исключение <xref:System.InvalidOperationException> времени выполнения.</span><span class="sxs-lookup"><span data-stu-id="847f8-106">If any code ever attempts to directly access the value-type member of such an object across an application domain, a runtime <xref:System.InvalidOperationException> will occur.</span></span> <span data-ttu-id="847f8-107">Чтобы устранить это предупреждение, скопируйте член в локальную переменную и вызовите метод для этой переменной.</span><span class="sxs-lookup"><span data-stu-id="847f8-107">To resolve the warning, first copy the member into a local variable and call the method on that variable.</span></span>
  
 <span data-ttu-id="847f8-108">В следующем примере возникает предупреждение CS1690:</span><span class="sxs-lookup"><span data-stu-id="847f8-108">The following sample generates CS1690:</span></span>  
  
```csharp  
// CS1690.cs  
using System;

class WarningCS1690 : MarshalByRefObject
{
    int i = 5;

    public static void Main()
    {
        AppDomain domain = AppDomain.CreateDomain("MyDomain");
        Type t = typeof(WarningCS1690);
        WarningCS1690 e = (WarningCS1690)domain.CreateInstanceAndUnwrap(t.Assembly.FullName,t.FullName);

        e.i.ToString(); // CS1690

        // OK  
        int i = e.i;
        i.ToString();
        e.i = i;
    }
}
```
