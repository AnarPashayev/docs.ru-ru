<span data-ttu-id="b389d-104">Член структуры "поле структуры2" типа "структура1" приводит к циклу в этом макете структуры</span><span class="sxs-lookup"><span data-stu-id="b389d-104">Struct member 'struct2 field' of type 'struct1' causes a cycle in the struct layout</span></span>  
  
 <span data-ttu-id="b389d-105">Определения двух структур содержат рекурсивные ссылки.</span><span class="sxs-lookup"><span data-stu-id="b389d-105">The definitions of two structs include recursive references.</span></span> <span data-ttu-id="b389d-106">Измените определения [структур](../builtin-types/struct.md) таким образом, чтобы в каждом из них не задавалось определение самой структуры на основе другой.</span><span class="sxs-lookup"><span data-stu-id="b389d-106">Change the [struct](../builtin-types/struct.md) definitions such that each does not define itself on the other.</span></span> <span data-ttu-id="b389d-107">Это ограничение относится только к структурам, поскольку они являются типами значений.</span><span class="sxs-lookup"><span data-stu-id="b389d-107">This limitation applies only to structs, since structs are value types.</span></span> <span data-ttu-id="b389d-108">При использовании рекурсивных ссылок объявите типы в качестве классов.</span><span class="sxs-lookup"><span data-stu-id="b389d-108">If using recursive references, declare your types as classes.</span></span>  
  
 <span data-ttu-id="b389d-109">В следующем примере возникает ошибка CS0523:</span><span class="sxs-lookup"><span data-stu-id="b389d-109">The following sample generates CS0523:</span></span>  
  
```csharp

// compile with: /target:library

{  
   public RecursiveLayoutStruct2 field;  
}  
  
struct RecursiveLayoutStruct2  
{  
   public RecursiveLayoutStruct1 field;   // CS0523  
}  
```
