## <a name="example-of-ambiguous-conversion"></a><span data-ttu-id="19bc0-107">Пример неоднозначного преобразования</span><span class="sxs-lookup"><span data-stu-id="19bc0-107">Example of ambiguous conversion</span></span>

<span data-ttu-id="19bc0-108">Эта ошибка может возникнуть при отсутствии преобразования или в том случае, если доступные неявные преобразования приводят к возникновению неоднозначности.</span><span class="sxs-lookup"><span data-stu-id="19bc0-108">This error can occur when there is no conversion or if the available implicit conversions result in an ambiguous situation.</span></span> <span data-ttu-id="19bc0-109">В следующем примере возникает ошибка CS0151:</span><span class="sxs-lookup"><span data-stu-id="19bc0-109">The following example generates CS0151:</span></span>

```csharp
public class MyClass
{
   public static implicit operator int (MyClass aa)
   {
      return 0;
   }

   public static implicit operator long (MyClass aa)
   {
      return 0;
   }

   public static void Main()
   {
      MyClass a = new MyClass();

      // Compiler cannot choose between int and long.
      switch (a)   // CS0151
      // try the following line instead
      // switch ((int)a)
      {
         case 1:
            break;
      }
   }
}
```

## <a name="example-of-void-method"></a><span data-ttu-id="19bc0-110">Пример метода void</span><span class="sxs-lookup"><span data-stu-id="19bc0-110">Example of void method</span></span>

<span data-ttu-id="19bc0-111">Вызов метода [void](../builtin-types/void.md) в выражении сопоставления [switch](../keywords/switch.md) приводит к ошибке CS0151.</span><span class="sxs-lookup"><span data-stu-id="19bc0-111">A [void](../builtin-types/void.md) method invocation in a [switch](../keywords/switch.md) match expression generates CS0151.</span></span> <span data-ttu-id="19bc0-112">Чтобы исправить эту ошибку, вызовите метод, возвращающий вместо этого целочисленное значение, например [int](../builtin-types/integral-numeric-types.md) или [long](../builtin-types/integral-numeric-types.md).</span><span class="sxs-lookup"><span data-stu-id="19bc0-112">You can fix the error by calling a method that returns an integral type such as [int](../builtin-types/integral-numeric-types.md) or [long](../builtin-types/integral-numeric-types.md) instead.</span></span>

```csharp
class C
{
    static void Main()
    {
        switch (M()) // CS0151
        {
            default:
                break;
        }
    }

    static void M()
    {
    }
}
```
