## <a name="example-of-ambiguous-conversion"></a><span data-ttu-id="fec25-106">Пример неоднозначного преобразования</span><span class="sxs-lookup"><span data-stu-id="fec25-106">Example of ambiguous conversion</span></span>

<span data-ttu-id="fec25-107">Эта ошибка может возникнуть при отсутствии преобразования или в том случае, если доступные неявные преобразования приводят к возникновению неоднозначности.</span><span class="sxs-lookup"><span data-stu-id="fec25-107">This error can occur when there is no conversion or if the available implicit conversions result in an ambiguous situation.</span></span> <span data-ttu-id="fec25-108">В следующем примере возникает ошибка CS0151:</span><span class="sxs-lookup"><span data-stu-id="fec25-108">The following example generates CS0151:</span></span>

```csharp
public class MyClass
{
   public static implicit operator int (MyClass aa)
   {
      return 0;
   }

   public static implicit operator long (MyClass aa)
   {
      return 0;
   }

   public static void Main()
   {
      MyClass a = new MyClass();

      // Compiler cannot choose between int and long.
      switch (a)   // CS0151
      // try the following line instead
      // switch ((int)a)
      {
         case 1:
            break;
      }
   }
}
```

## <a name="example-of-void-method"></a><span data-ttu-id="fec25-109">Пример метода void</span><span class="sxs-lookup"><span data-stu-id="fec25-109">Example of void method</span></span>

<span data-ttu-id="fec25-110">Вызов метода [void](../../../csharp/language-reference/keywords/void.md) в выражении сопоставления [switch](../keywords/switch.md) приводит к ошибке CS0151.</span><span class="sxs-lookup"><span data-stu-id="fec25-110">A [void](../../../csharp/language-reference/keywords/void.md) method invocation in a [switch](../keywords/switch.md) match expression generates CS0151.</span></span> <span data-ttu-id="fec25-111">Чтобы исправить эту ошибку, вызовите метод, возвращающий вместо этого целочисленное значение, например [int](../../../csharp/language-reference/builtin-types/integral-numeric-types.md) или [long](../../../csharp/language-reference/builtin-types/integral-numeric-types.md).</span><span class="sxs-lookup"><span data-stu-id="fec25-111">You can fix the error by calling a method that returns an integral type such as [int](../../../csharp/language-reference/builtin-types/integral-numeric-types.md) or [long](../../../csharp/language-reference/builtin-types/integral-numeric-types.md) instead.</span></span>

```csharp
class C
{
    static void Main()
    {
        switch (M()) // CS0151
        {
            default:
                break;
        }
    }

    static void M()
    {
    }
}
```
