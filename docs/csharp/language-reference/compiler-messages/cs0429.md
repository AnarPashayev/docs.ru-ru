# <a name="compiler-warning-level-4-cs0429"></a><span data-ttu-id="55a11-103">Предупреждение компилятора (уровень 4) CS0429</span><span class="sxs-lookup"><span data-stu-id="55a11-103">Compiler Warning (level 4) CS0429</span></span>

<span data-ttu-id="55a11-104">Обнаружен недостижимый код в выражении</span><span class="sxs-lookup"><span data-stu-id="55a11-104">Unreachable expression code detected</span></span>  
  
 <span data-ttu-id="55a11-105">Эта ошибка возникает в тех случаях, когда невозможно достижение какой-либо части выражения в коде.</span><span class="sxs-lookup"><span data-stu-id="55a11-105">This error occurs whenever part of an expression in your code is unreachable.</span></span> <span data-ttu-id="55a11-106">В следующем примере условие `false && myTest()` соответствует этому критерию, поскольку метод `myTest()` никогда не будет выполнен из-за того, что левая часть операции `&&` всегда будет возвращать значение false.</span><span class="sxs-lookup"><span data-stu-id="55a11-106">In the following example, the condition `false && myTest()` meets this criteria because the `myTest()` method will never get evaluated due to the fact that the left side of the `&&` operation is always false.</span></span> <span data-ttu-id="55a11-107">Как только оператор `&&` вычисляет инструкцию `false` как значение false, обработка прекращается, и вычисление правой части выражения ни при каких условиях не производится.</span><span class="sxs-lookup"><span data-stu-id="55a11-107">As soon as the `&&` operator evaluates the `false` statement as false, it stops the evaluation, and will never evaluate the right side.</span></span>

```csharp  
    public static void Main()
    {  
        if (false && myTest())  // CS0429

        }  
        else  
        {  
            int i = 0;  
            i++;  
        }  
    }  
  
    static bool myTest() { return true; }  
}  
```
