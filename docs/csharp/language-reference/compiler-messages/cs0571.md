# <a name="compiler-error-cs0571"></a><span data-ttu-id="34312-103">Ошибка компилятора CS0571</span><span class="sxs-lookup"><span data-stu-id="34312-103">Compiler Error CS0571</span></span>

<span data-ttu-id="34312-104">"функция": явный вызов оператора или метода доступа невозможен</span><span class="sxs-lookup"><span data-stu-id="34312-104">'function' : cannot explicitly call operator or accessor</span></span>  
  
 <span data-ttu-id="34312-105">Некоторые операторы имеют внутренние имена.</span><span class="sxs-lookup"><span data-stu-id="34312-105">Certain operators have internal names.</span></span> <span data-ttu-id="34312-106">Например, оператор ++ имеет внутреннее имя **op_Increment**.</span><span class="sxs-lookup"><span data-stu-id="34312-106">For example, **op_Increment** is the internal name of the ++ operator.</span></span> <span data-ttu-id="34312-107">Такие имена методов не следует использовать или явно вызывать.</span><span class="sxs-lookup"><span data-stu-id="34312-107">You should not use or explicitly call such method names.</span></span>  
  
 <span data-ttu-id="34312-108">В следующем примере возникает ошибка CS0571:</span><span class="sxs-lookup"><span data-stu-id="34312-108">The following sample generates CS0571:</span></span>  
  
```csharp  
// CS0571.cs  
public class MyClass  
{  
   public static MyClass operator ++ (MyClass c)  
   {  
      return null;  
   }  
  
   public static int prop  
   {  
      get  
      {  
         return 1;  
      }  
      set  
      {  
      }  
   }  
  
   public static void Main()  
   {  
      op_Increment(null);   // CS0571  
      // use the increment operator as follows  
      // MyClass x = new MyClass();  
      // x++;  
  
      set_prop(1);      // CS0571  
      // try the following line instead  
      // prop = 1;  
   }  
}  
```
