## <a name="example"></a><span data-ttu-id="4a227-119">Пример</span><span class="sxs-lookup"><span data-stu-id="4a227-119">Example</span></span>  
 <span data-ttu-id="4a227-120">Следующий код ссылается на две версии библиотеки DLL, которые были определены в выше приведенном коде.</span><span class="sxs-lookup"><span data-stu-id="4a227-120">The following code references the two DLL versions that are defined in the preceding code.</span></span> <span data-ttu-id="4a227-121">`AssemblyA` ссылается на библиотеку DLL, созданную файлом CS1705a.cs (версии 1.0).</span><span class="sxs-lookup"><span data-stu-id="4a227-121">`AssemblyA` refers to the DLL created by CS1705a.cs (version 1.0).</span></span> <span data-ttu-id="4a227-122">`AssemblyB` ссылается на библиотеку DLL, созданную файлом CS1705b.cs (версии 2.0).</span><span class="sxs-lookup"><span data-stu-id="4a227-122">`AssemblyB` refers to the DLL created by CS1705b.cs (version 2.0).</span></span> <span data-ttu-id="4a227-123">В `ClassC` определены два метода.</span><span class="sxs-lookup"><span data-stu-id="4a227-123">In `ClassC`, two methods are defined.</span></span> <span data-ttu-id="4a227-124">Первый, `Return1A`, возвращает объект типа `Class1A`, который является псевдонимом для `Class1` из версии 1.0 библиотеки DLL.</span><span class="sxs-lookup"><span data-stu-id="4a227-124">The first, `Return1A`, returns an object of type `Class1A`, which is an alias for `Class1` from version 1.0 of the DLL.</span></span> <span data-ttu-id="4a227-125">Второй, `Return1B`, возвращает объект типа `Class1B`, который является псевдонимом для `Class1` из версии 2.0 библиотеки DLL.</span><span class="sxs-lookup"><span data-stu-id="4a227-125">The second, `Return1B`, returns an object of type `Class1B`, which is an alias for `Class1` from version 2.0 of the DLL.</span></span> <span data-ttu-id="4a227-126">Определение `Return1A` создает зависимость от версии 1.0; определение `Return1B` создает зависимость от версии 2.0.</span><span class="sxs-lookup"><span data-stu-id="4a227-126">The definition of `Return1A` creates a dependency on version 1.0; the definition of `Return1B` creates a dependency on version 2.0.</span></span>  
  
```csharp  
// CS1705c.cs  
  
// Compile by using the following command. AssemblyA refers to the DLL created by  
// CS1705a.cs (version 1.0). AssemblyB refers to the DLL created by CS1705b.cs  
// (version 2.0).  
//    csc /target:library /r:AssemblyA=C:\\CS1705.dll /r:AssemblyB=CS1705.dll CS1705c.cs  
  
extern alias AssemblyA;  
extern alias AssemblyB;  
  
// Class1A is an alias for type Class1 from VS1705a.cs, which is in version 1.0   
// of the assembly. Class1B is an alias for type Class1 from CS1705b.cs, which  
// is in version 2.0 of the assembly.  
  
using Class1A = AssemblyA::Class1;  
using Class1B = AssemblyB::Class1;  
  
// Method Return1A in ClassC returns an object of type Class1A, which is  
// Class1 from version 1.0 of the DLL. Method Return1B returns an object  
// of type Class1B, which is Class1 from version 2.0 of the DLL.  
  
public class ClassC  
{  
    // The following line creates a dependency on version 1.0 of CS1705.dll.  
    // This is not the source of the problem when ClassC is accessed from  
    // CS1705d.cs because CS1705d.cs references version 1.0 of the DLL.   
    // Therefore, type Class1A and the assembly have the same version.  
    public static Class1A Return1A() { return new Class1A(); }  
  
    // The following line creates a dependency on version 2.0 of CS1705.dll.  
    // This causes compiler error CS1705 when ClassC is accessed from     
    // CS1705d.cs, because CS1705d.cs does not reference version 2.0 of the   
    // DLL. Class1B is the alias for Class1 in version 2.0, and CS1705d.cs   
    // references version 1.0.  
    public static Class1B Return1B() { return new Class1B(); }  
}  
```

    }
