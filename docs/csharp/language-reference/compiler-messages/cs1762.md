# <a name="compiler-warning-level-1-cs1762"></a><span data-ttu-id="ad4d5-102">Предупреждение компилятора (уровень 1) CS1762</span><span class="sxs-lookup"><span data-stu-id="ad4d5-102">Compiler Warning (level 1) CS1762</span></span>

<span data-ttu-id="ad4d5-103">Была создана ссылка на внедренную сборку взаимодействия "\<сборка1>" из-за косвенной ссылки на эту сборку из сборки "\<сборка2>".</span><span class="sxs-lookup"><span data-stu-id="ad4d5-103">A reference was created to embedded interop assembly '\<assembly1>' because of an indirect reference to that assembly from assembly '\<assembly2>'.</span></span> <span data-ttu-id="ad4d5-104">Рекомендуется изменить свойство "Внедрить типы взаимодействия" в одной из сборок.</span><span class="sxs-lookup"><span data-stu-id="ad4d5-104">Consider changing the 'Embed Interop Types' property on either assembly.</span></span>  
  
 <span data-ttu-id="ad4d5-105">Была добавлена ссылка на сборку (сборка1), для которой свойству `Embed Interop Types` присвоено значение `True`.</span><span class="sxs-lookup"><span data-stu-id="ad4d5-105">You have added a reference to an assembly (assembly1) that has the `Embed Interop Types` property set to `True`.</span></span> <span data-ttu-id="ad4d5-106">Это указывает компилятору на необходимость внедрить сведения о типе взаимодействия из этой сборки.</span><span class="sxs-lookup"><span data-stu-id="ad4d5-106">This instructs the compiler to embed interop type information from that assembly.</span></span> <span data-ttu-id="ad4d5-107">Тем не менее компилятор не может внедрить такие сведения из этой сборки, поскольку другая сборка, на которую задаются ссылки (сборка2), также ссылается на эту сборку (сборка1) и содержит свойство `Embed Interop Types` со значением `False`.</span><span class="sxs-lookup"><span data-stu-id="ad4d5-107">However, the compiler cannot embed interop type information from that assembly because another assembly that you have referenced (assembly2) also references that assembly (assembly1) and has the `Embed Interop Types` property set to `False`.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="ad4d5-108">Если присвоить свойству `Embed Interop Types` для ссылки на сборку значение `True`, это будет эквивалентно ссылке на сборку с использованием параметра `/link` для компилятора командной строки.</span><span class="sxs-lookup"><span data-stu-id="ad4d5-108">Setting the `Embed Interop Types` property on an assembly reference to `True` is equivalent to referencing the assembly by using the `/link` option for the command-line compiler.</span></span>  
  
## <a name="to-address-this-warning"></a><span data-ttu-id="ad4d5-109">Устранение предупреждения</span><span class="sxs-lookup"><span data-stu-id="ad4d5-109">To address this warning</span></span>
  
- <span data-ttu-id="ad4d5-110">Чтобы внедрить сведения о типе взаимодействия для обеих сборок, присвойте свойству `Embed Interop Types` во всех ссылках на сборку сборка1 значение `True`.</span><span class="sxs-lookup"><span data-stu-id="ad4d5-110">To embed interop type information for both assemblies, set the `Embed Interop Types` property on all references to assembly1 to `True`.</span></span> <span data-ttu-id="ad4d5-111">Дополнительные сведения см. в разделе [Пошаговое руководство. внедрению типов из управляемых сборок](../../programming-guide/concepts/assemblies-gac/walkthrough-embedding-types-from-managed-assemblies-in-visual-studio.md).</span><span class="sxs-lookup"><span data-stu-id="ad4d5-111">For more information about how to set that property, see [Walkthrough: Embedding Types from Managed Assemblies](../../programming-guide/concepts/assemblies-gac/walkthrough-embedding-types-from-managed-assemblies-in-visual-studio.md).</span></span>
