# <a name="introduction-to-delegates"></a><span data-ttu-id="29140-103">Общие сведения о делегатах</span><span class="sxs-lookup"><span data-stu-id="29140-103">Introduction to Delegates</span></span>

<span data-ttu-id="29140-104">Делегаты предоставляют механизм *позднего связывания* в .NET.</span><span class="sxs-lookup"><span data-stu-id="29140-104">Delegates provide a *late binding* mechanism in .NET.</span></span> <span data-ttu-id="29140-105">Позднее связывание означает, что создается алгоритм, где вызывающий объект также предоставляет по крайней мере один метод, который реализует часть алгоритма.</span><span class="sxs-lookup"><span data-stu-id="29140-105">Late Binding means that you create an algorithm where the caller also supplies at least one method that implements part of the algorithm.</span></span>

<span data-ttu-id="29140-106">Например, рассмотрим сортировку списка звезд в астрономическом приложении.</span><span class="sxs-lookup"><span data-stu-id="29140-106">For example, consider sorting a list of stars in an astronomy application.</span></span>
<span data-ttu-id="29140-107">Можно отсортировать звезды по расстоянию от Земли, по величине или по воспринимаемой яркости.</span><span class="sxs-lookup"><span data-stu-id="29140-107">You may choose to sort those stars by their distance from the earth, or the magnitude of the star, or their perceived brightness.</span></span>

<span data-ttu-id="29140-108">Во всех этих случаях метод Sort() выполняет, по сути, одно и то же: упорядочивает элементы в списке на основе некоего сравнения.</span><span class="sxs-lookup"><span data-stu-id="29140-108">In all those cases, the Sort() method does essentially the same thing: arranges the items in the list based on some comparison.</span></span> <span data-ttu-id="29140-109">Для каждого порядка сортировки используется разный код, сравнивающий две звезды.</span><span class="sxs-lookup"><span data-stu-id="29140-109">The code that compares two stars is different for each of the sort orderings.</span></span>

<span data-ttu-id="29140-110">Такого рода решения использовались в программном обеспечении в течение полувека.</span><span class="sxs-lookup"><span data-stu-id="29140-110">These kinds of solutions have been used in software for half a century.</span></span>
<span data-ttu-id="29140-111">Концепция использования делегатов в языке C# обеспечивает первоклассную поддержку языка и безопасность типов.</span><span class="sxs-lookup"><span data-stu-id="29140-111">The C# language delegate concept provides first class language support, and type safety around the concept.</span></span>

<span data-ttu-id="29140-114">Разработчики, использующие язык, определили несколько целей для функций, которые в итоге стали делегатами.</span><span class="sxs-lookup"><span data-stu-id="29140-114">The language designers enumerated several goals for the feature that eventually became delegates.</span></span>

<span data-ttu-id="29140-115">Группе разработчиков требовалась общая языковая конструкция, которую можно было бы использовать для любых алгоритмов позднего связывания.</span><span class="sxs-lookup"><span data-stu-id="29140-115">The team wanted a common language construct that could be used for any late binding algorithms.</span></span> <span data-ttu-id="29140-116">Разработчикам удалось изучить одну концепцию и применить ее для решения многих самых разных проблем программного обеспечения.</span><span class="sxs-lookup"><span data-stu-id="29140-116">That enables developers to learn one concept, and use that same concept across many different software problems.</span></span>
