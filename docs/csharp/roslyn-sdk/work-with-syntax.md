## <a name="syntax-trees"></a><span data-ttu-id="4eb06-110">Деревья синтаксиса</span><span class="sxs-lookup"><span data-stu-id="4eb06-110">Syntax trees</span></span>

<span data-ttu-id="4eb06-123">Третий атрибут деревьев синтаксиса заключается в том, что они неизменяемы и потокобезопасны.</span><span class="sxs-lookup"><span data-stu-id="4eb06-123">The third attribute of syntax trees is that they are immutable and thread-safe.</span></span>  <span data-ttu-id="4eb06-124">Это означает, что после получения дерево представляет собой моментальный снимок текущего состояния кода и никогда не меняется.</span><span class="sxs-lookup"><span data-stu-id="4eb06-124">This means that after a tree is obtained, it is a snapshot of the current state of the code, and never changes.</span></span> <span data-ttu-id="4eb06-125">Это позволяет нескольким пользователям одновременно взаимодействовать с одним деревом синтаксиса в разных потоках без блокировки или дублирования.</span><span class="sxs-lookup"><span data-stu-id="4eb06-125">This allows multiple users to interact with the same syntax tree at the same time in different threads without locking or duplication.</span></span> <span data-ttu-id="4eb06-126">Так как деревья являются неизменяемыми, то есть напрямую в них невозможно внести никакие изменения, фабричные методы помогают создавать и изменять деревья синтаксиса путем создания их дополнительных снимков.</span><span class="sxs-lookup"><span data-stu-id="4eb06-126">Because the trees are immutable and no modifications can be made directly to a tree, factory methods help create and modify syntax trees by creating additional snapshots of the tree.</span></span> <span data-ttu-id="4eb06-127">Деревья эффективно используют базовые узлы повторно, поэтому новую версию можно перестроить быстро и с небольшими затратами памяти.</span><span class="sxs-lookup"><span data-stu-id="4eb06-127">The trees are efficient in the way they reuse underlying nodes, so a new version can be rebuilt fast and with little extra memory.</span></span>

<span data-ttu-id="4eb06-128">Дерево синтаксиса фактически является древовидной структурой данных, где нетерминальные структурные элементы являются родительскими для других элементов.</span><span class="sxs-lookup"><span data-stu-id="4eb06-128">A syntax tree is literally a tree data structure, where non-terminal structural elements parent other elements.</span></span> <span data-ttu-id="4eb06-129">Каждое дерево синтаксиса состоит из узлов, токенов и дополнительной синтаксической информации (trivia).</span><span class="sxs-lookup"><span data-stu-id="4eb06-129">Each syntax tree is made up of nodes, tokens, and trivia.</span></span>
