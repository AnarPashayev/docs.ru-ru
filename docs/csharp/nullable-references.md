# <a name="nullable-reference-types"></a><span data-ttu-id="336f8-104">Ссылочные типы, допускающие значение null</span><span class="sxs-lookup"><span data-stu-id="336f8-104">Nullable reference types</span></span>

<span data-ttu-id="336f8-105">C# 8.0 представляет **ссылочные типы, допускающие значение NULL** и **ссылочные типы, не допускающие значение NULL**, которые позволяют делать важные заявления о свойствах для переменных ссылочного типа:</span><span class="sxs-lookup"><span data-stu-id="336f8-105">C# 8.0 introduces **nullable reference types** and **non-nullable reference types** that enable you to make important statements about the properties for reference type variables:</span></span>

- <span data-ttu-id="336f8-106">**Ссылка не должна иметь значение NULL**.</span><span class="sxs-lookup"><span data-stu-id="336f8-106">**A reference is not supposed to be null**.</span></span> <span data-ttu-id="336f8-107">Когда переменные не должны иметь значение NULL, компилятор принудительно применяет правила, которые гарантируют, что можно разыменовать эти переменные без предварительной проверки того, что они не имеют значение NULL:</span><span class="sxs-lookup"><span data-stu-id="336f8-107">When variables aren't supposed to be null, the compiler enforces rules that ensure it is safe to dereference these variables without first checking that it isn't null:</span></span>
  - <span data-ttu-id="336f8-108">Переменную необходимо инициализировать со значением, отличным от NULL.</span><span class="sxs-lookup"><span data-stu-id="336f8-108">The variable must be initialized to a non-null value.</span></span>
  - <span data-ttu-id="336f8-109">Переменной не может быть присвоено значение `null`.</span><span class="sxs-lookup"><span data-stu-id="336f8-109">The variable can never be assigned the value `null`.</span></span>
- <span data-ttu-id="336f8-110">**Ссылка может иметь значение NULL**.</span><span class="sxs-lookup"><span data-stu-id="336f8-110">**A reference may be null**.</span></span> <span data-ttu-id="336f8-111">Когда переменные могут иметь значение NULL, компилятор применяет различные правила, чтобы убедиться, что вы правильно проверили наличие пустой ссылки:</span><span class="sxs-lookup"><span data-stu-id="336f8-111">When variables may be null, the compiler enforces different rules to ensure that you've correctly checked for a null reference:</span></span>
  - <span data-ttu-id="336f8-112">Переменная может быть разыменована только в том случае, когда компилятор может гарантировать, что значение не равно NULL.</span><span class="sxs-lookup"><span data-stu-id="336f8-112">The variable may only be dereferenced when the compiler can guarantee that the value isn't null.</span></span>
  - <span data-ttu-id="336f8-113">Эти переменные могут быть инициализированы со значением по умолчанию `null`, и им можно присвоить значение `null` в другом коде.</span><span class="sxs-lookup"><span data-stu-id="336f8-113">These variables may be initialized with the default `null` value and may be assigned the value `null` in other code.</span></span>

<span data-ttu-id="336f8-114">Эта новая функция предоставляет значительные преимущества по сравнению с обработкой ссылочных переменных в более ранних версиях C#, где назначение кода невозможно было определить по объявлению переменной.</span><span class="sxs-lookup"><span data-stu-id="336f8-114">This new feature provides significant benefits over the handling of reference variables in earlier versions of C# where the design intent couldn't be determined from the variable declaration.</span></span> <span data-ttu-id="336f8-115">Компилятор не обеспечивает безопасность от исключений, связанных с пустой ссылкой, для ссылочных типов:</span><span class="sxs-lookup"><span data-stu-id="336f8-115">The compiler didn't provide safety against null reference exceptions for reference types:</span></span>

- <span data-ttu-id="336f8-116">**Ссылка может иметь значение NULL**.</span><span class="sxs-lookup"><span data-stu-id="336f8-116">**A reference can be null**.</span></span> <span data-ttu-id="336f8-117">Предупреждения не выдаются, если ссылочный тип инициализируется со значением NULL либо позднее получает значение NULL.</span><span class="sxs-lookup"><span data-stu-id="336f8-117">No warnings are issued when a reference type is initialized to null, or later assigned to null.</span></span>
- <span data-ttu-id="336f8-118">**Предполагается, что ссылка не имеет значение NULL**.</span><span class="sxs-lookup"><span data-stu-id="336f8-118">**A reference is assumed to be not null**.</span></span> <span data-ttu-id="336f8-119">Компилятор не выдает предупреждения, когда ссылочные типы разыменованы.</span><span class="sxs-lookup"><span data-stu-id="336f8-119">The compiler doesn't issue any warnings when reference types are dereferenced.</span></span> <span data-ttu-id="336f8-120">(Со ссылками, допускающими значение NULL, компилятор выдает предупреждения, когда вы разыменовываете переменную, которая может иметь значение NULL).</span><span class="sxs-lookup"><span data-stu-id="336f8-120">(With nullable references,  the compiler issues warnings whenever you dereference a variable that may be null).</span></span>

<span data-ttu-id="336f8-121">С появлением ссылочных типов, допускающих значение NULL, вы можете более четко объявить свое намерение.</span><span class="sxs-lookup"><span data-stu-id="336f8-121">With the addition of nullable reference types, you can declare your intent more clearly.</span></span> <span data-ttu-id="336f8-122">Значение `null` — это правильный способ представить, что переменная не ссылается на значение.</span><span class="sxs-lookup"><span data-stu-id="336f8-122">The `null` value is the correct way to represent that a variable doesn't refer to a value.</span></span> <span data-ttu-id="336f8-123">Эта функция не предназначена для удаления всех значений `null` из кода.</span><span class="sxs-lookup"><span data-stu-id="336f8-123">Don't use this feature to remove all `null` values from your code.</span></span> <span data-ttu-id="336f8-124">Вместо этого следует объявить свое намерение компилятору и другим разработчикам, которые читают ваш код.</span><span class="sxs-lookup"><span data-stu-id="336f8-124">Rather, you should declare your intent to the compiler and other developers that read your code.</span></span> <span data-ttu-id="336f8-125">Если вы объявите свое намерение, компилятор сообщит, когда ваш код не соответствует этому намерению.</span><span class="sxs-lookup"><span data-stu-id="336f8-125">By declaring your intent, the compiler informs you when you write code that is inconsistent with that intent.</span></span>

<span data-ttu-id="336f8-126">**Ссылочный тип, допускающий значение NULL** использует тот же синтаксис, что и [тип значения, допускающего значение NULL](programming-guide/nullable-types/index.md): к типу переменной добавляется `?`.</span><span class="sxs-lookup"><span data-stu-id="336f8-126">A **nullable reference type** is noted using the same syntax as [nullable value types](programming-guide/nullable-types/index.md): a `?` is appended to the type of the variable.</span></span> <span data-ttu-id="336f8-127">Например, следующее объявление переменной представляет строковую переменную, допускающую значение NULL, `name`:</span><span class="sxs-lookup"><span data-stu-id="336f8-127">For example, the following variable declaration represents a nullable string variable, `name`:</span></span>

```csharp
string? name;
```

<span data-ttu-id="336f8-128">Любая переменная, где `?` не добавляется к имени типа, является **ссылочным типом, не допускающим значение NULL**.</span><span class="sxs-lookup"><span data-stu-id="336f8-128">Any variable where the `?` is not appended to the type name is a **non-nullable reference type**.</span></span> <span data-ttu-id="336f8-129">Сюда входят все переменные ссылочных типов в существующем коде, если вы включили эту функцию.</span><span class="sxs-lookup"><span data-stu-id="336f8-129">That includes all reference type variables in existing code when you have enabled this feature.</span></span>

<span data-ttu-id="336f8-130">Компилятор использует статический анализ, чтобы определить, что ссылка, допускающая значение NULL, имеет значение, отличное от NULL.</span><span class="sxs-lookup"><span data-stu-id="336f8-130">The compiler uses static analysis to determine if a nullable reference is known to be non-null.</span></span> <span data-ttu-id="336f8-131">Компилятор выдает предупреждение, если вы разыменовываете ссылку, допускающую значение NULL, когда она может иметь значение NULL.</span><span class="sxs-lookup"><span data-stu-id="336f8-131">The compiler warns you if you dereference a nullable reference when it may be null.</span></span> <span data-ttu-id="336f8-132">Это поведение можно переопределить с помощью **оператора допуска NULL** (`!`) после имени переменной.</span><span class="sxs-lookup"><span data-stu-id="336f8-132">You can override this behavior by using the **null-forgiving operator** (`!`) following a variable name.</span></span> <span data-ttu-id="336f8-133">Например, если вы знаете, что переменная `name` не имеет значение NULL, а компилятор выдает предупреждение, напишите следующий код, чтобы переопределить анализ компилятора:</span><span class="sxs-lookup"><span data-stu-id="336f8-133">For example, if you know the `name` variable isn't null but the compiler issues a warning, you can write the following code to override the compiler's analysis:</span></span>

```csharp
name!.Length;
```

<span data-ttu-id="336f8-134">Дополнительные сведения об этом операторе см. в черновике спецификации [ссылочных типов, допускающих значение NULL](../../_csharplang/proposals/csharp-8.0/nullable-reference-types-specification.md#the-null-forgiving-operator) на GitHub.</span><span class="sxs-lookup"><span data-stu-id="336f8-134">You can read details about this operator in the [draft nullable reference types](../../_csharplang/proposals/csharp-8.0/nullable-reference-types-specification.md#the-null-forgiving-operator) specification proposal on GitHub.</span></span>

## <a name="nullability-of-types"></a><span data-ttu-id="336f8-135">Допустимость значений NULL для типов</span><span class="sxs-lookup"><span data-stu-id="336f8-135">Nullability of types</span></span>

<span data-ttu-id="336f8-136">Любой ссылочный тип может иметь один из четырех уровней *допустимости значений NULL*, который описывает, когда создаются предупреждения:</span><span class="sxs-lookup"><span data-stu-id="336f8-136">Any reference type can have one of four *nullabilities*, which describes when warnings are generated:</span></span>

- <span data-ttu-id="336f8-137">*Не допускает значения NULL*: значение NULL нельзя присвоить переменным этого типа.</span><span class="sxs-lookup"><span data-stu-id="336f8-137">*Nonnullable*: Null can't be assigned to variables of this type.</span></span> <span data-ttu-id="336f8-138">Переменные этого типа не нужно проверять на значение NULL перед разыменованием.</span><span class="sxs-lookup"><span data-stu-id="336f8-138">Variables of this type don't need to be null-checked before dereferencing.</span></span>
- <span data-ttu-id="336f8-139">*Допускает значение NULL*: значение NULL можно присвоить переменным этого типа.</span><span class="sxs-lookup"><span data-stu-id="336f8-139">*Nullable*: Null can be assigned to variables of this type.</span></span> <span data-ttu-id="336f8-140">Разыменование переменных этого типа без предварительной проверки `null` вызывает предупреждение.</span><span class="sxs-lookup"><span data-stu-id="336f8-140">Dereferencing variables of this type without first checking for `null` causes a warning.</span></span>

- <span data-ttu-id="336f8-143">*Неизвестно*: обычно относится к параметрам типа, где ограничения не сообщают компилятору, что тип должен *допускать* или *не допускать значение NULL*.</span><span class="sxs-lookup"><span data-stu-id="336f8-143">*Unknown*: This is generally for type parameters where constraints don't tell the compiler that the type must be *nullable* or *nonnullable*.</span></span>
