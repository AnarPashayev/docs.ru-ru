# <a name="nullable-reference-types"></a><span data-ttu-id="b1b88-104">Ссылочные типы, допускающие значение null</span><span class="sxs-lookup"><span data-stu-id="b1b88-104">Nullable reference types</span></span>

<span data-ttu-id="b1b88-105">C# 8.0 представляет **ссылочные типы, допускающие значение NULL** и **ссылочные типы, не допускающие значение NULL**, которые позволяют делать важные заявления о свойствах для переменных ссылочного типа:</span><span class="sxs-lookup"><span data-stu-id="b1b88-105">C# 8.0 introduces **nullable reference types** and **non-nullable reference types** that enable you to make important statements about the properties for reference type variables:</span></span>

- <span data-ttu-id="b1b88-106">**Ссылка не должна иметь значение NULL**.</span><span class="sxs-lookup"><span data-stu-id="b1b88-106">**A reference is not supposed to be null**.</span></span> <span data-ttu-id="b1b88-107">Когда переменные не должны иметь значение NULL, компилятор принудительно применяет правила, которые гарантируют, что можно разыменовать эти переменные без предварительной проверки того, что они не имеют значение NULL:</span><span class="sxs-lookup"><span data-stu-id="b1b88-107">When variables aren't supposed to be null, the compiler enforces rules that ensure it is safe to dereference these variables without first checking that it isn't null:</span></span>

  - <span data-ttu-id="b1b88-109">Переменной не может быть присвоено значение `null`.</span><span class="sxs-lookup"><span data-stu-id="b1b88-109">The variable can never be assigned the value `null`.</span></span>
- <span data-ttu-id="b1b88-110">**Ссылка может иметь значение NULL**.</span><span class="sxs-lookup"><span data-stu-id="b1b88-110">**A reference may be null**.</span></span> <span data-ttu-id="b1b88-111">Когда переменные могут иметь значение NULL, компилятор применяет различные правила, чтобы убедиться, что вы правильно проверили наличие пустой ссылки:</span><span class="sxs-lookup"><span data-stu-id="b1b88-111">When variables may be null, the compiler enforces different rules to ensure that you've correctly checked for a null reference:</span></span>
  - <span data-ttu-id="b1b88-112">Переменная может быть разыменована только в том случае, когда компилятор может гарантировать, что значение не равно NULL.</span><span class="sxs-lookup"><span data-stu-id="b1b88-112">The variable may only be dereferenced when the compiler can guarantee that the value isn't null.</span></span>

<span data-ttu-id="b1b88-114">Эта новая функция предоставляет значительные преимущества по сравнению с обработкой ссылочных переменных в более ранних версиях C#, где назначение кода невозможно было определить по объявлению переменной.</span><span class="sxs-lookup"><span data-stu-id="b1b88-114">This new feature provides significant benefits over the handling of reference variables in earlier versions of C# where the design intent couldn't be determined from the variable declaration.</span></span> <span data-ttu-id="b1b88-115">Компилятор не обеспечивает безопасность от исключений, связанных с пустой ссылкой, для ссылочных типов:</span><span class="sxs-lookup"><span data-stu-id="b1b88-115">The compiler didn't provide safety against null reference exceptions for reference types:</span></span>

- <span data-ttu-id="b1b88-116">**Ссылка может иметь значение NULL**.</span><span class="sxs-lookup"><span data-stu-id="b1b88-116">**A reference can be null**.</span></span> <span data-ttu-id="b1b88-117">Предупреждения не выдаются, если ссылочный тип инициализируется со значением NULL либо позднее получает значение NULL.</span><span class="sxs-lookup"><span data-stu-id="b1b88-117">No warnings are issued when a reference type is initialized to null, or later assigned to null.</span></span>
- <span data-ttu-id="b1b88-118">**Предполагается, что ссылка не имеет значение NULL**.</span><span class="sxs-lookup"><span data-stu-id="b1b88-118">**A reference is assumed to be not null**.</span></span> <span data-ttu-id="b1b88-119">Компилятор не выдает предупреждения, когда ссылочные типы разыменованы.</span><span class="sxs-lookup"><span data-stu-id="b1b88-119">The compiler doesn't issue any warnings when reference types are dereferenced.</span></span> <span data-ttu-id="b1b88-120">(Со ссылками, допускающими значение NULL, компилятор выдает предупреждения, когда вы разыменовываете переменную, которая может иметь значение NULL).</span><span class="sxs-lookup"><span data-stu-id="b1b88-120">(With nullable references,  the compiler issues warnings whenever you dereference a variable that may be null).</span></span>

<span data-ttu-id="b1b88-121">С появлением ссылочных типов, допускающих значение NULL, вы можете более четко объявить свое намерение.</span><span class="sxs-lookup"><span data-stu-id="b1b88-121">With the addition of nullable reference types, you can declare your intent more clearly.</span></span> <span data-ttu-id="b1b88-122">Значение `null` — это правильный способ представить, что переменная не ссылается на значение.</span><span class="sxs-lookup"><span data-stu-id="b1b88-122">The `null` value is the correct way to represent that a variable doesn't refer to a value.</span></span> <span data-ttu-id="b1b88-123">Эта функция не предназначена для удаления всех значений `null` из кода.</span><span class="sxs-lookup"><span data-stu-id="b1b88-123">Don't use this feature to remove all `null` values from your code.</span></span> <span data-ttu-id="b1b88-124">Вместо этого следует объявить свое намерение компилятору и другим разработчикам, которые читают ваш код.</span><span class="sxs-lookup"><span data-stu-id="b1b88-124">Rather, you should declare your intent to the compiler and other developers that read your code.</span></span> <span data-ttu-id="b1b88-125">Если вы объявите свое намерение, компилятор сообщит, когда ваш код не соответствует этому намерению.</span><span class="sxs-lookup"><span data-stu-id="b1b88-125">By declaring your intent, the compiler informs you when you write code that is inconsistent with that intent.</span></span>

<span data-ttu-id="b1b88-128">Любая переменная, где `?` не добавляется к имени типа, является **ссылочным типом, не допускающим значение NULL**.</span><span class="sxs-lookup"><span data-stu-id="b1b88-128">Any variable where the `?` is not appended to the type name is a **non-nullable reference type**.</span></span> <span data-ttu-id="b1b88-129">Сюда входят все переменные ссылочных типов в существующем коде, если вы включили эту функцию.</span><span class="sxs-lookup"><span data-stu-id="b1b88-129">That includes all reference type variables in existing code when you have enabled this feature.</span></span>

<span data-ttu-id="b1b88-130">Компилятор использует статический анализ, чтобы определить, что ссылка, допускающая значение NULL, имеет значение, отличное от NULL.</span><span class="sxs-lookup"><span data-stu-id="b1b88-130">The compiler uses static analysis to determine if a nullable reference is known to be non-null.</span></span> <span data-ttu-id="b1b88-131">Компилятор выдает предупреждение, если вы разыменовываете ссылку, допускающую значение NULL, когда она может иметь значение NULL.</span><span class="sxs-lookup"><span data-stu-id="b1b88-131">The compiler warns you if you dereference a nullable reference when it may be null.</span></span> <span data-ttu-id="b1b88-132">Это поведение можно переопределить с помощью **оператора допуска NULL** (`!`) после имени переменной.</span><span class="sxs-lookup"><span data-stu-id="b1b88-132">You can override this behavior by using the **null-forgiving operator** (`!`) following a variable name.</span></span> <span data-ttu-id="b1b88-133">Например, если вы знаете, что переменная `name` не имеет значение NULL, а компилятор выдает предупреждение, напишите следующий код, чтобы переопределить анализ компилятора:</span><span class="sxs-lookup"><span data-stu-id="b1b88-133">For example, if you know the `name` variable isn't null but the compiler issues a warning, you can write the following code to override the compiler's analysis:</span></span>

<span data-ttu-id="b1b88-134">Дополнительные сведения об этом операторе см. в черновике спецификации [ссылочных типов, допускающих значение NULL](../../_csharplang/proposals/csharp-8.0/nullable-reference-types-specification.md#the-null-forgiving-operator) на GitHub.</span><span class="sxs-lookup"><span data-stu-id="b1b88-134">You can read details about this operator in the [draft nullable reference types](../../_csharplang/proposals/csharp-8.0/nullable-reference-types-specification.md#the-null-forgiving-operator) specification proposal on GitHub.</span></span>

## <a name="nullability-of-types"></a><span data-ttu-id="b1b88-135">Допустимость значений NULL для типов</span><span class="sxs-lookup"><span data-stu-id="b1b88-135">Nullability of types</span></span>

<span data-ttu-id="b1b88-136">Любой ссылочный тип может иметь один из четырех уровней *допустимости значений NULL*, который описывает, когда создаются предупреждения:</span><span class="sxs-lookup"><span data-stu-id="b1b88-136">Any reference type can have one of four *nullabilities*, which describes when warnings are generated:</span></span>
