## <a name="translating-is-visiting"></a><span data-ttu-id="96a1f-110">Преобразование — это обход</span><span class="sxs-lookup"><span data-stu-id="96a1f-110">Translating is Visiting</span></span>

<span data-ttu-id="96a1f-111">Код, создаваемый для преобразования дерева выражения, является расширением кода, который используется для обхода всех узлов в дереве.</span><span class="sxs-lookup"><span data-stu-id="96a1f-111">The code you build to translate an expression tree is an extension of what you've already seen to visit all the nodes in a tree.</span></span> <span data-ttu-id="96a1f-112">Во время преобразования дерева выражения вы выполняете обход всех узлов и, таким образом, построение нового дерева.</span><span class="sxs-lookup"><span data-stu-id="96a1f-112">When you translate an expression tree, you visit all the nodes, and while visiting them, build the new tree.</span></span> <span data-ttu-id="96a1f-113">Новое дерево может содержать ссылки на исходные узлы или новые узлы, которые были помещены в дерево.</span><span class="sxs-lookup"><span data-stu-id="96a1f-113">The new tree may contain references to the original nodes, or new nodes that you have placed in the tree.</span></span>

<span data-ttu-id="96a1f-114">Рассмотрим это на примере, выполнив обход дерева выражения и создав новое дерево, заменяя в нем несколько узлов.</span><span class="sxs-lookup"><span data-stu-id="96a1f-114">Let's see this in action by visiting an expression tree, and creating a new tree with some replacement nodes.</span></span> <span data-ttu-id="96a1f-115">В этом примере мы заменим любую константу константой, которая в десять раз больше.</span><span class="sxs-lookup"><span data-stu-id="96a1f-115">In this example, let's replace any constant with a constant that is ten times larger.</span></span>
<span data-ttu-id="96a1f-116">В противном случае мы оставим дерево выражения без изменений.</span><span class="sxs-lookup"><span data-stu-id="96a1f-116">Otherwise, we'll leave the expression tree intact.</span></span> <span data-ttu-id="96a1f-117">Вместо того чтобы считывать значение константы и заменять ее новой константой, мы осуществим такую подстановку, заменив узел константы новым узлом, который выполняет умножение.</span><span class="sxs-lookup"><span data-stu-id="96a1f-117">Rather than reading the value of the constant, and replacing it with a new constant, we'll make this replacement by replacing the constant node with a new node that performs the multiplication.</span></span>

<span data-ttu-id="96a1f-118">Итак, вы находите узел константы, создаете новый узел умножения, дочерний элемент которого является исходной константой, и константу `10`:</span><span class="sxs-lookup"><span data-stu-id="96a1f-118">Here, once you find a constant node, you create a new multiplication node whose children are the original constant, and the constant `10`:</span></span>

```csharp
private static Expression ReplaceNodes(Expression original)
{
    if (original.NodeType == ExpressionType.Constant)
    {
        return Expression.Multiply(original, Expression.Constant(10));
    }
    else if (original.NodeType == ExpressionType.Add)
    {
        var binaryExpression = (BinaryExpression)original;
        return Expression.Add(
            ReplaceNodes(binaryExpression.Left),
            ReplaceNodes(binaryExpression.Right));
    }
    return original;
}
```

<span data-ttu-id="96a1f-119">При замене исходного узла новым узлом формируется новое дерево, содержащее наши изменения.</span><span class="sxs-lookup"><span data-stu-id="96a1f-119">By replacing the original node with the substitute, a new tree is formed that contains our modifications.</span></span> <span data-ttu-id="96a1f-120">Мы можем проверить это, скомпилировав и выполнив измененное дерево.</span><span class="sxs-lookup"><span data-stu-id="96a1f-120">We can verify that by compiling and executing the replaced tree.</span></span>

```csharp
var one = Expression.Constant(1, typeof(int));
var two = Expression.Constant(2, typeof(int));
var addition = Expression.Add(one, two);
var sum = ReplaceNodes(addition);
var executableFunc = Expression.Lambda(sum);

var func = (Func<int>)executableFunc.Compile();
var answer = func();
Console.WriteLine(answer);
```

<span data-ttu-id="96a1f-121">Создание нового дерева представляет собой сочетание обхода узлов в существующем дереве и создания новых узлов и вставки их в дерево.</span><span class="sxs-lookup"><span data-stu-id="96a1f-121">Building a new tree is a combination of visiting the nodes in the existing tree, and creating new nodes and inserting them into the tree.</span></span>

<span data-ttu-id="96a1f-122">Этот пример демонстрирует значимость неизменяемых деревьев выражений.</span><span class="sxs-lookup"><span data-stu-id="96a1f-122">This example shows the importance of expression trees being immutable.</span></span> <span data-ttu-id="96a1f-123">Обратите внимание, что созданное нами новое дерево содержит сочетание вновь созданных узлов и узлов из существующего дерева.</span><span class="sxs-lookup"><span data-stu-id="96a1f-123">Notice that the new tree created above contains a mixture of newly created nodes, and nodes from the existing tree.</span></span> <span data-ttu-id="96a1f-124">Это безопасно, поскольку узлы в существующем дереве изменить нельзя.</span><span class="sxs-lookup"><span data-stu-id="96a1f-124">That's safe, because the nodes in the existing tree cannot be modified.</span></span> <span data-ttu-id="96a1f-125">Это позволяет существенно сократить потребление памяти.</span><span class="sxs-lookup"><span data-stu-id="96a1f-125">This can result in significant memory efficiencies.</span></span>
<span data-ttu-id="96a1f-126">Одни и те же узлы можно использовать в одном дереве или в нескольких деревьях выражений.</span><span class="sxs-lookup"><span data-stu-id="96a1f-126">The same nodes can be used throughout a tree, or in multiple expression trees.</span></span> <span data-ttu-id="96a1f-127">Поскольку узлы нельзя изменить, один узел можно при необходимости использовать повторно.</span><span class="sxs-lookup"><span data-stu-id="96a1f-127">Since nodes can't be modified, the same node can be reused whenever it's needed.</span></span>

## <a name="traversing-and-executing-an-addition"></a><span data-ttu-id="96a1f-128">Обход и выполнение добавления</span><span class="sxs-lookup"><span data-stu-id="96a1f-128">Traversing and Executing an Addition</span></span>

<span data-ttu-id="96a1f-129">Давайте проверим это, создав второй посетитель, который обходит дерево дополнительных узлов и вычисляет результат.</span><span class="sxs-lookup"><span data-stu-id="96a1f-129">Let's verify this by building a second visitor that walks the tree of addition nodes and computes the result.</span></span> <span data-ttu-id="96a1f-130">Для этого вы можете просто внести пару изменений в уже готовый посетитель.</span><span class="sxs-lookup"><span data-stu-id="96a1f-130">You can do this by making a couple modifications to the visitor that you've seen so far.</span></span> <span data-ttu-id="96a1f-131">В этой новой версии посетитель вернет частичную сумму операции сложения до этой точки.</span><span class="sxs-lookup"><span data-stu-id="96a1f-131">In this new version, the visitor will return the partial sum of the addition operation up to this point.</span></span> <span data-ttu-id="96a1f-132">Для константного выражения это будет просто значением константного выражения.</span><span class="sxs-lookup"><span data-stu-id="96a1f-132">For a constant expression, that is simply the value of the constant expression.</span></span> <span data-ttu-id="96a1f-133">Для выражения добавления результатом будет сумма левого и правого операндов, поскольку выполняется обход этих деревьев.</span><span class="sxs-lookup"><span data-stu-id="96a1f-133">For an addition expression, the result is the sum of the left and right operands, once those trees have been traversed.</span></span>

```csharp
var one = Expression.Constant(1, typeof(int));
var two = Expression.Constant(2, typeof(int));
var three= Expression.Constant(3, typeof(int));
var four = Expression.Constant(4, typeof(int));
var addition = Expression.Add(one, two);
var add2 = Expression.Add(three, four);
var sum = Expression.Add(addition, add2);

// Declare the delegate, so we can call it
// from itself recursively:
Func<Expression, int> aggregate = null;
// Aggregate, return constants, or the sum of the left and right operand.
// Major simplification: Assume every binary expression is an addition.
aggregate = (exp) =>
    exp.NodeType == ExpressionType.Constant ?
    (int)((ConstantExpression)exp).Value :
    aggregate(((BinaryExpression)exp).Left) + aggregate(((BinaryExpression)exp).Right);

var theSum = aggregate(sum);
Console.WriteLine(theSum);
```

<span data-ttu-id="96a1f-134">Нам потребуется написать небольшой фрагмент кода, однако основные принципы очень просты.</span><span class="sxs-lookup"><span data-stu-id="96a1f-134">There's quite a bit of code here, but the concepts are very approachable.</span></span>
<span data-ttu-id="96a1f-135">Этот код обходит дочерние элементы при поиске в глубину.</span><span class="sxs-lookup"><span data-stu-id="96a1f-135">This code visits children in a depth first search.</span></span> <span data-ttu-id="96a1f-136">При обнаружении узла константы посетитель возвращает значение этой константы.</span><span class="sxs-lookup"><span data-stu-id="96a1f-136">When it encounters a constant node, the visitor returns the value of the constant.</span></span> <span data-ttu-id="96a1f-137">После обхода обоих дочерних элементов для них вычисляется сумма для этого поддерева.</span><span class="sxs-lookup"><span data-stu-id="96a1f-137">After the visitor has visited both children, those children will have computed the sum computed for that subtree.</span></span> <span data-ttu-id="96a1f-138">Теперь добавленный узел может вычислить свою сумму.</span><span class="sxs-lookup"><span data-stu-id="96a1f-138">The addition node can now compute its sum.</span></span>
<span data-ttu-id="96a1f-139">После обхода всех узлов в дереве выражений будут вычислена сумма.</span><span class="sxs-lookup"><span data-stu-id="96a1f-139">Once all the nodes in the expression tree have been visited, the sum will have been computed.</span></span> <span data-ttu-id="96a1f-140">Вы можете отслеживать выполнение, запустив пример в отладчике с трассировкой выполнения.</span><span class="sxs-lookup"><span data-stu-id="96a1f-140">You can trace the execution by running the sample in the debugger and tracing the execution.</span></span>

```csharp
private static int Aggregate(Expression exp)
{
    if (exp.NodeType == ExpressionType.Constant)
    {
        var constantExp = (ConstantExpression)exp;
        Console.Error.WriteLine($"Found Constant: {constantExp.Value}");
        return (int)constantExp.Value;
    }
    else if (exp.NodeType == ExpressionType.Add)
    {
        var addExp = (BinaryExpression)exp;
        Console.Error.WriteLine("Found Addition Expression");
        Console.Error.WriteLine("Computing Left node");
        var leftOperand = Aggregate(addExp.Left);
        Console.Error.WriteLine($"Left is: {leftOperand}");
        Console.Error.WriteLine("Computing Right node");
        var rightOperand = Aggregate(addExp.Right);
        Console.Error.WriteLine($"Right is: {rightOperand}");
        var sum = leftOperand + rightOperand;
        Console.Error.WriteLine($"Computed sum: {sum}");
        return sum;
    }
    else throw new NotSupportedException("Haven't written this yet");
}
```
