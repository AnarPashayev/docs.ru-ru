# <a name="weak-references"></a><span data-ttu-id="a8a83-103">Слабые ссылки</span><span class="sxs-lookup"><span data-stu-id="a8a83-103">Weak References</span></span>
<span data-ttu-id="a8a83-104">Сборщик мусора не может собрать объект, используемый приложением, пока код приложения взаимодействует с этим объектом.</span><span class="sxs-lookup"><span data-stu-id="a8a83-104">The garbage collector cannot collect an object in use by an application while the application's code can reach that object.</span></span> <span data-ttu-id="a8a83-105">Говорят, что приложение имеет строгую ссылку на объект.</span><span class="sxs-lookup"><span data-stu-id="a8a83-105">The application is said to have a strong reference to the object.</span></span>  
  
 <span data-ttu-id="a8a83-106">Слабая ссылка дает сборщику мусора возможность удалить объект, но в то же время обеспечивает приложению доступ к этому объекту.</span><span class="sxs-lookup"><span data-stu-id="a8a83-106">A weak reference permits the garbage collector to collect the object while still allowing the application to access the object.</span></span> <span data-ttu-id="a8a83-107">Слабая ссылка допустима только в течение неопределенного количества времени до сборки объекта при отсутствии строгих ссылок.</span><span class="sxs-lookup"><span data-stu-id="a8a83-107">A weak reference is valid only during the indeterminate amount of time until the object is collected when no strong references exist.</span></span> <span data-ttu-id="a8a83-108">При использовании слабой ссылки приложение все еще может получить строгую ссылку в объекте, что предотвратит удаление последнего.</span><span class="sxs-lookup"><span data-stu-id="a8a83-108">When you use a weak reference, the application can still obtain a strong reference to the object, which prevents it from being collected.</span></span> <span data-ttu-id="a8a83-109">Однако всегда существует вероятность удаления объекта сборщиком мусора до повторного создания строгой ссылки.</span><span class="sxs-lookup"><span data-stu-id="a8a83-109">However, there is always the risk that the garbage collector will get to the object first before a strong reference is reestablished.</span></span>  
  
 <span data-ttu-id="a8a83-110">Слабые ссылки полезны для объектов, которые используют большой объем памяти, но могут быть созданы повторно без особых усилий, если они были удалены сборщиком мусора.</span><span class="sxs-lookup"><span data-stu-id="a8a83-110">Weak references are useful for objects that use a lot of memory, but can be recreated easily if they are reclaimed by garbage collection.</span></span>  
  
 <span data-ttu-id="a8a83-111">Для примера рассмотрим представление в виде дерева в приложении Windows Forms, которое отображает для пользователя разветвленную иерархическую структуру нескольких вариантов.</span><span class="sxs-lookup"><span data-stu-id="a8a83-111">Suppose a tree view in a Windows Forms application displays a complex hierarchical choice of options to the user.</span></span> <span data-ttu-id="a8a83-112">Если базовые данные большие по объему, хранение этого дерева в памяти является неэффективным, особенно если пользователь делает в этом приложении что-либо еще.</span><span class="sxs-lookup"><span data-stu-id="a8a83-112">If the underlying data is large, keeping the tree in memory is inefficient when the user is involved with something else in the application.</span></span>  
  
 <span data-ttu-id="a8a83-113">Если пользователь переключается в другую часть приложения, можно использовать класс <xref:System.WeakReference> для создания слабой ссылки на дерево и удалить все строгие ссылки.</span><span class="sxs-lookup"><span data-stu-id="a8a83-113">When the user switches away to another part of the application, you can use the <xref:System.WeakReference> class to create a weak reference to the tree and destroy all strong references.</span></span> <span data-ttu-id="a8a83-114">Когда пользователь переключится обратно к этому дереву, приложение попытается получить строгую ссылку на дерево и, при успешном выполнении этой операции, избежать повторного создания дерева.</span><span class="sxs-lookup"><span data-stu-id="a8a83-114">When the user switches back to the tree, the application attempts to obtain a strong reference to the tree and, if successful, avoids reconstructing the tree.</span></span>  
  
 <span data-ttu-id="a8a83-115">Чтобы установить слабую ссылку на объект, создайте <xref:System.WeakReference>, используя экземпляр объекта для отслеживания.</span><span class="sxs-lookup"><span data-stu-id="a8a83-115">To establish a weak reference with an object, you create a <xref:System.WeakReference> using the instance of the object to be tracked.</span></span> <span data-ttu-id="a8a83-116">Затем следует присвоить свойству <xref:System.WeakReference.Target%2A> этот объект и задать значение `null` для исходной ссылки на объект.</span><span class="sxs-lookup"><span data-stu-id="a8a83-116">You then set the <xref:System.WeakReference.Target%2A> property to that object and set the original reference to the object to `null`.</span></span> <span data-ttu-id="a8a83-117">Пример кода см. в разделе <xref:System.WeakReference> в библиотеке классов.</span><span class="sxs-lookup"><span data-stu-id="a8a83-117">For a code example, see <xref:System.WeakReference> in the class library.</span></span>  
  
## <a name="short-and-long-weak-references"></a><span data-ttu-id="a8a83-118">Краткие и длинные слабые ссылки</span><span class="sxs-lookup"><span data-stu-id="a8a83-118">Short and Long Weak References</span></span>  
 <span data-ttu-id="a8a83-119">Можно создать краткую слабую ссылку или длинную слабую ссылку:</span><span class="sxs-lookup"><span data-stu-id="a8a83-119">You can create a short weak reference or a long weak reference:</span></span>  
  
- <span data-ttu-id="a8a83-120">Short</span><span class="sxs-lookup"><span data-stu-id="a8a83-120">Short</span></span>  
  
     <span data-ttu-id="a8a83-121">Назначением краткой ссылки становится `null`, если объект удален сборщиком мусора.</span><span class="sxs-lookup"><span data-stu-id="a8a83-121">The target of a short weak reference becomes `null` when the object is reclaimed by garbage collection.</span></span> <span data-ttu-id="a8a83-122">Сама по себе слабая ссылка является управляемым объектом и подлежит сборке мусора, как и любые другие управляемые объекты.</span><span class="sxs-lookup"><span data-stu-id="a8a83-122">The weak reference is itself a managed object, and is subject to garbage collection just like any other managed object.</span></span>  <span data-ttu-id="a8a83-123">Краткая слабая ссылка является конструктором без параметров для <xref:System.WeakReference>.</span><span class="sxs-lookup"><span data-stu-id="a8a83-123">A short weak reference is the parameterless constructor for <xref:System.WeakReference>.</span></span>  
  
- <span data-ttu-id="a8a83-124">Long</span><span class="sxs-lookup"><span data-stu-id="a8a83-124">Long</span></span>  
  
     <span data-ttu-id="a8a83-125">Длинная слабая ссылка сохраняется после вызова метода <xref:System.Object.Finalize%2A> объекта.</span><span class="sxs-lookup"><span data-stu-id="a8a83-125">A long weak reference is retained after the object's <xref:System.Object.Finalize%2A> method has been called.</span></span> <span data-ttu-id="a8a83-126">Это позволяет повторно создавать объект, однако состояние объекта остается непредсказуемым.</span><span class="sxs-lookup"><span data-stu-id="a8a83-126">This allows the object to be recreated, but the state of the object remains unpredictable.</span></span> <span data-ttu-id="a8a83-127">Чтобы использовать длинную ссылку, укажите значение `true` в конструкторе <xref:System.WeakReference>.</span><span class="sxs-lookup"><span data-stu-id="a8a83-127">To use a long reference, specify `true` in the <xref:System.WeakReference> constructor.</span></span>  
  
     <span data-ttu-id="a8a83-128">Если тип объекта не имеет метода <xref:System.Object.Finalize%2A>, используется функция краткой слабой ссылки, а сама слабая ссылка становится допустимой только до сборки целевых объектов, что может произойти в любое время после запуска метода завершения.</span><span class="sxs-lookup"><span data-stu-id="a8a83-128">If the object's type does not have a <xref:System.Object.Finalize%2A> method, the short weak reference functionality applies and the weak reference is valid only until the target is collected, which can occur anytime after the finalizer is run.</span></span>  
  
 <span data-ttu-id="a8a83-129">Чтобы установить строгую ссылку и повторно использовать объект, приведите свойство <xref:System.WeakReference.Target%2A> объекта <xref:System.WeakReference> к типу этого объекта.</span><span class="sxs-lookup"><span data-stu-id="a8a83-129">To establish a strong reference and use the object again, cast the <xref:System.WeakReference.Target%2A> property of a <xref:System.WeakReference> to the type of the object.</span></span> <span data-ttu-id="a8a83-130">Если свойство <xref:System.WeakReference.Target%2A> возвращает значение `null`, значит объект был удален сборщиком. В противном случае можно продолжить использование объекта, так как приложение восстановило строгую ссылку на этот объект.</span><span class="sxs-lookup"><span data-stu-id="a8a83-130">If the <xref:System.WeakReference.Target%2A> property returns `null`, the object was collected; otherwise, you can continue to use the object because the application has regained a strong reference to it.</span></span>  
  
## <a name="guidelines-for-using-weak-references"></a><span data-ttu-id="a8a83-131">Правила использования слабых ссылок</span><span class="sxs-lookup"><span data-stu-id="a8a83-131">Guidelines for Using Weak References</span></span>  
 <span data-ttu-id="a8a83-132">Используйте длинные слабые ссылки только при необходимости, так как состояние объекта после выполнения завершения становится непредсказуемым.</span><span class="sxs-lookup"><span data-stu-id="a8a83-132">Use long weak references only when necessary as the state of the object is unpredictable after finalization.</span></span>  
  
 <span data-ttu-id="a8a83-133">Избегайте использования слабых ссылок на небольшие объекты, потому что сам указатель может быть таким же по объему или даже больше.</span><span class="sxs-lookup"><span data-stu-id="a8a83-133">Avoid using weak references to small objects because the pointer itself may be as large or larger.</span></span>  
  
 <span data-ttu-id="a8a83-134">Избегайте использования слабых ссылок в качестве автоматического решения проблем, связанных с управлением памятью.</span><span class="sxs-lookup"><span data-stu-id="a8a83-134">Avoid using weak references as an automatic solution to memory management problems.</span></span> <span data-ttu-id="a8a83-135">Вместо этого разработайте эффективную политику кэширования для обработки объектов приложения.</span><span class="sxs-lookup"><span data-stu-id="a8a83-135">Instead, develop an effective caching policy for handling your application's objects.</span></span>
