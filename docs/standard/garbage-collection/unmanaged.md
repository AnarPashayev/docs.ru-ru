<span data-ttu-id="46d47-104">Для большинства объектов, создаваемых приложением, управление памятью осуществляется [сборщиком мусора .NET](index.md).</span><span class="sxs-lookup"><span data-stu-id="46d47-104">For the majority of the objects that your app creates, you can rely on the [.NET garbage collector](index.md) to handle memory management.</span></span> <span data-ttu-id="46d47-105">Но при создании объектов, которые включают неуправляемые ресурсы, эти ресурсы необходимо явно освобождать после использования.</span><span class="sxs-lookup"><span data-stu-id="46d47-105">However, when you create objects that include unmanaged resources, you must explicitly release those resources when you finish using them.</span></span> <span data-ttu-id="46d47-106">Основным типом неуправляемых ресурсов являются объекты, заключающие ресурсы операционной системы, такие как файлы, окна, сетевые подключения и подключения к базам данным.</span><span class="sxs-lookup"><span data-stu-id="46d47-106">The most common types of unmanaged resources are objects that wrap operating system resources, such as files, windows, network connections, or database connections.</span></span> <span data-ttu-id="46d47-107">Хотя сборщик мусора может отслеживать время жизни управляемого объекта, инкапсулирующего неуправляемые ресурсы, он не имеет сведений о том, как освобождать такие ресурсы.</span><span class="sxs-lookup"><span data-stu-id="46d47-107">Although the garbage collector is able to track the lifetime of an object that encapsulates an unmanaged resource, it doesn't know how to release and clean up the unmanaged resource.</span></span>

  - <span data-ttu-id="46d47-115">Используйте безопасный дескриптор в качестве оболочки для неуправляемого ресурса.</span><span class="sxs-lookup"><span data-stu-id="46d47-115">Use a safe handle to wrap your unmanaged resource.</span></span> <span data-ttu-id="46d47-116">Это рекомендуемая методика.</span><span class="sxs-lookup"><span data-stu-id="46d47-116">This is the recommended technique.</span></span> <span data-ttu-id="46d47-117">Безопасные дескрипторы являются производными от абстрактного класса <xref:System.Runtime.InteropServices.SafeHandle?displayProperty=nameWithType>.   - <span data-ttu-id="46d47-121">Переопределите метод <xref:System.Object.Finalize%2A?displayProperty=nameWithType> .</span><span class="sxs-lookup"><span data-stu-id="46d47-121">Override the <xref:System.Object.Finalize%2A?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="46d47-122">Завершение включает недетерминированное освобождение неуправляемых ресурсов, когда объекту-получателю типа не удается вызвать метод <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> для их детерминированного удаления.</span><span class="sxs-lookup"><span data-stu-id="46d47-122">Finalization enables the non-deterministic release of unmanaged resources when the consumer of a type fails to call <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> to dispose of them deterministically.</span></span> <span data-ttu-id="46d47-123">Определите [метод завершения](../../csharp/programming-guide/classes-and-structs/destructors.md), переопределив метод <xref:System.Object.Finalize%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="46d47-123">Define a [finalizer](../../csharp/programming-guide/classes-and-structs/destructors.md) by overriding the <xref:System.Object.Finalize%2A?displayProperty=nameWithType> method.</span></span>

  > [!WARNING]
  > <span data-ttu-id="46d47-124">Но так как завершение объекта может быть сложной операцией с высокой вероятностью возникновения ошибок, мы рекомендуем использовать безопасный дескриптор, а не указывать собственный метод завершения.</span><span class="sxs-lookup"><span data-stu-id="46d47-124">However, because object finalization can be a complex and an error-prone operation, we recommend that you use a safe handle instead of providing your own finalizer.</span></span>
