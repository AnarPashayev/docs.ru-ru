<span data-ttu-id="fedc4-170">__Рассмотрим приложение, для которого классическая модель установки заменяется облачной.__</span><span class="sxs-lookup"><span data-stu-id="fedc4-170">__Consider an app that moves from a desktop-install model to a cloud-first model.__</span></span> <span data-ttu-id="fedc4-171">Например, этот сценарий может включать в себя классические приложения, для которых осуществляется переход к модели полнофункционального клиента или веб-приложения.</span><span class="sxs-lookup"><span data-stu-id="fedc4-171">This scenario includes apps that move from a desktop app or rich client model into a web-based model.</span></span> <span data-ttu-id="fedc4-172">Модели угроз, характерные для классических приложений, не обязательно будут относиться к облачным службам.</span><span class="sxs-lookup"><span data-stu-id="fedc4-172">Any threat models drawn for the desktop app aren't necessarily applicable to the cloud-based service.</span></span> <span data-ttu-id="fedc4-173">К примеру, определенная угроза может исключаться, поскольку клиент не заинтересован в атаке на собственную систему.</span><span class="sxs-lookup"><span data-stu-id="fedc4-173">The threat model for the desktop app might dismiss a given threat as "not interesting for the client to attack itself."</span></span> <span data-ttu-id="fedc4-174">Тем не менее, та же самая угроза становится вполне реальной в контексте намерения удаленного пользователя (клиента) атаковать саму облачную службу.</span><span class="sxs-lookup"><span data-stu-id="fedc4-174">But that same threat might become interesting when it considers a remote user (the client) attacking the cloud service itself.</span></span>
