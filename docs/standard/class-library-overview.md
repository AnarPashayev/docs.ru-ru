## <a name="naming-conventions"></a><span data-ttu-id="6abfd-118">Соглашения об именах</span><span class="sxs-lookup"><span data-stu-id="6abfd-118">Naming conventions</span></span>

 <span data-ttu-id="6abfd-119">Для типов платформы .NET используется схема именования через точку, описывающая иерархию.</span><span class="sxs-lookup"><span data-stu-id="6abfd-119">.NET types use a dot syntax naming scheme that connotes a hierarchy.</span></span> <span data-ttu-id="6abfd-120">При таком подходе связанные типы группируются в пространства имен, что упрощает их поиск и создание ссылок.</span><span class="sxs-lookup"><span data-stu-id="6abfd-120">This technique groups related types into namespaces so they can be searched and referenced more easily.</span></span> <span data-ttu-id="6abfd-121">Первая часть полного имени — до крайней правой точки — это имя пространства имен.</span><span class="sxs-lookup"><span data-stu-id="6abfd-121">The first part of the full name — up to the rightmost dot — is the namespace name.</span></span> <span data-ttu-id="6abfd-122">Последняя часть имени — это имя типа.</span><span class="sxs-lookup"><span data-stu-id="6abfd-122">The last part of the name is the type name.</span></span> <span data-ttu-id="6abfd-123">Например `System.Collections.Generic.List<T>` представляет тип `List<T>`, который принадлежит пространству имен `System.Collections.Generic`.</span><span class="sxs-lookup"><span data-stu-id="6abfd-123">For example, `System.Collections.Generic.List<T>` represents the `List<T>` type, which belongs to the `System.Collections.Generic` namespace.</span></span> <span data-ttu-id="6abfd-124">Типы в <xref:System.Collections.Generic> можно использовать для работы с универсальными коллекциями.</span><span class="sxs-lookup"><span data-stu-id="6abfd-124">The types in <xref:System.Collections.Generic> can be used to work with generic collections.</span></span>  
  
 <span data-ttu-id="6abfd-125">Такая схема именования упрощает разработчикам библиотек задачу расширения .NET Framework с целью создания иерархических групп типов и присвоения им согласованных между собой и понятных имен.</span><span class="sxs-lookup"><span data-stu-id="6abfd-125">This naming scheme makes it easy for library developers extending the .NET Framework to create hierarchical groups of types and name them in a consistent, informative manner.</span></span> <span data-ttu-id="6abfd-126">Она также позволяет однозначно идентифицировать типы по их полным именам (то есть по пространству имен и имени типа), что предотвращает конфликты имен типов.</span><span class="sxs-lookup"><span data-stu-id="6abfd-126">It also allows types to be unambiguously identified by their full name (that is, by their namespace and type name), which prevents type name collisions.</span></span> <span data-ttu-id="6abfd-127">Ожидается, при именовании пространств имен разработчики библиотек будут руководствоваться следующим соглашением:</span><span class="sxs-lookup"><span data-stu-id="6abfd-127">Library developers are expected to use the following convention when creating names for their namespaces:</span></span>  
  
 <span data-ttu-id="6abfd-128">*CompanyName*.*TechnologyName*</span><span class="sxs-lookup"><span data-stu-id="6abfd-128">*CompanyName*.*TechnologyName*</span></span>  
  
 <span data-ttu-id="6abfd-129">Например, пространство имен `Microsoft.Word` соответствует этому правилу.</span><span class="sxs-lookup"><span data-stu-id="6abfd-129">For example, the namespace `Microsoft.Word` conforms to this guideline.</span></span>  
  
 <span data-ttu-id="6abfd-130">Использование шаблонов именования для группировки связанных типов в пространства имен очень полезно при создании и документировании библиотек классов.</span><span class="sxs-lookup"><span data-stu-id="6abfd-130">The use of naming patterns to group related types into namespaces is a very useful way to build and document class libraries.</span></span> <span data-ttu-id="6abfd-131">Однако такая схема именования не влияет на видимость, доступ к членам, наследование, безопасность и привязки.</span><span class="sxs-lookup"><span data-stu-id="6abfd-131">However, this naming scheme has no effect on visibility, member access, inheritance, security, or binding.</span></span> <span data-ttu-id="6abfd-132">Пространство имен может быть разделено между несколькими сборками, и одна сборка может содержать типы из нескольких пространств имен.</span><span class="sxs-lookup"><span data-stu-id="6abfd-132">A namespace can be partitioned across multiple assemblies and a single assembly can contain types from multiple namespaces.</span></span> <span data-ttu-id="6abfd-133">Сборка представляет собой формальную структуру для управления версиями, развертывания, обеспечения безопасности, загрузки и обеспечения видимости в среде CLR.</span><span class="sxs-lookup"><span data-stu-id="6abfd-133">The assembly provides the formal structure for versioning, deployment, security, loading, and visibility in the common language runtime.</span></span>
