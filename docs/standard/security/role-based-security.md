<span data-ttu-id="34df1-103">Роли часто используются в финансовых и деловых приложениях для применения политики.</span><span class="sxs-lookup"><span data-stu-id="34df1-103">Roles are often used in financial or business applications to enforce policy.</span></span> <span data-ttu-id="34df1-104">Например, приложение может установить ограничение на размер транзакции в зависимости от того, является ли сделавший запрос пользователь участником определенной роли.</span><span class="sxs-lookup"><span data-stu-id="34df1-104">For example, an application might impose limits on the size of the транзакций, размер которых меньше определенного порога, менеджер может иметь больший предел, а вице-президент — еще больший или неограниченный.</span><span class="sxs-lookup"><span data-stu-id="34df1-105">Clerks might have authorization to process transactions that are less than a specified threshold, supervisors might have a higher limit, and vice-presidents might have a still higher limit (or no limit at all).</span></span> <span data-ttu-id="34df1-106">Безопасность на основе ролей также можно использовать, когда приложению для завершения действия требуются многократные <span data-ttu-id="34df1-107">В качестве примера можно привести систему закупок, в которой каждый сотрудник может создать заявку на покупку, но преобразовать эту заявку в заказ на покупку для отправки поставщику может только агент по закупкам.</span><span class="sxs-lookup"><span data-stu-id="34df1-107">Such a case might be a purchasing system in which any employee can generate a purchase request,  <span data-ttu-id="34df1-108">Безопасность на основе ролей в .NET Framework поддерживает авторизацию путем предоставления текущему потоку информации о субъекте, который создается из связанного удостоверения.</span><span class="sxs-lookup"><span data-stu-id="34df1-108">.NET Framework role-based security supports authorization by making information about the principal, which is constructed from an associated identity, available to the current thread.</span></span> <span data-ttu-id="34df1-109">Удостоверение (и субъект, который определяется с его помощью) может быть основано на учетной principal it helps to define) can be either based on a Windows account or be a custom identity unrelated to a Windows account.</span></span> <span data-ttu-id="34df1-110">Приложения .NET Framework могут принимать решения об авторизации либо на основе удостоверения субъекта, либо на основе членства в роли, либо на основе того и другого.</span><span class="sxs-lookup"><span data-stu-id="34df1-110">.NET Framework applications can make authorization decisions based on the principal's identity or role membership, or both.</span></span> <span data-ttu-id="34df1-111">Роль — membership to determine whether a principal is authorized to perform a requested action.</span></span>  
  
 <span data-ttu-id="34df1-114">Для обеспечения простоты использования и согласованности с системой управления доступом для кода механизм безопасности на основе ролей в .NET Framework предоставляет объекты <xref:System.Security.Permissions.PrincipalPermission?displayProperty=nameWithType>, которые позволяют среде CLR проводить авторизацию таким же способом, как и при проверке управления доступом для кода.</span><span class="sxs-lookup"><span data-stu-id="34df1-114">To provide ease of use and consistency with code access security, .NET Framework role-based security provides <xref:System.Security.Permissions.PrincipalPermission?displayProperty=nameWithType> objects that enable the common language runtime to perform authorization in a way that is similar to code access security checks.</span></span> <span data-ttu-id="34df1-115">Класс <xref:System.Security.Permissions.PrincipalPermission> представляет удостоверение или роль, которой должен соответствовать субъект.  <span data-ttu-id="34df1-117">Платформа .NET Framework обеспечивает поддержку безопасности на основе ролей, которая является достаточно гибкой и расширяемой для того, чтобы удовлетворять требованиям широкого спектра приложений.</span><span class="sxs-lookup"><span data-stu-id="34df1-117">The .NET Framework provides role-based security support that is flexible and extensible enough to meet the needs of a wide spectrum of applications.</span></span> <span data-ttu-id="34df1-118">Можно взаимодействовать с существующей инфраструктурой аутентификации, например службами COM+ 1.0, или создать пользовательскую систему аутентификации.</span><span class="sxs-lookup"><span data-stu-id="34df1-118">You can choose to interoperate with existing authentication infrastructures, such as COM+ 1.0 Services, or to create a custom authentication system.</span></span> <span data-ttu-id="34df1-119">Безопасность на основе ролей особенно хорошо подходит для использования в веб-приложениях ASP.NET, которые обрабатываются в основном на сервере.</span><span class="sxs-lookup"><span data-stu-id="34df1-119">Role-based security is particularly well-suited for use in ASP.NET Web applications, which are processed primarily on the server.</span></span> <span data-ttu-id="34df1-120">Тем не менее безопасность на основе ролей в .NET Framework можно использовать как на стороне клиента, так и на стороне сервера.</span><span class="sxs-lookup"><span data-stu-id="34df1-120">However, .NET Framework role-based security can be used on either the client or the server.</span></span>
