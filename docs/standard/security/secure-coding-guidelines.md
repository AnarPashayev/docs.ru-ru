<span data-ttu-id="70cf6-149">Следующий подход наиболее мощный и поэтому потенциально опасный (при неправильном использовании) к написанию безопасного кода: библиотека служит как интерфейс для другого кода для доступа к определенным ресурсам, которые недоступны в противном случае, так же, как применять классы .NET разрешения для ресурсов, которые они используют.</span><span class="sxs-lookup"><span data-stu-id="70cf6-149">The following approach is the most powerful and hence potentially dangerous (if done incorrectly) for security coding: your library serves as an interface for other code to access certain resources that aren't otherwise available, just as the .NET classes enforce permissions for the resources they use.</span></span> <span data-ttu-id="70cf6-150">При предоставлении доступа к ресурсу код должен сначала запросить соответствующее разрешение на ресурс (т. е. он должен выполнить проверку безопасности), а затем объявить свои права на выполнение текущей операции.</span><span class="sxs-lookup"><span data-stu-id="70cf6-150">Wherever you expose a resource, your code must first demand the permission appropriate to the resource (that is, it must perform a security check) and then typically assert its rights to perform the actual operation.</span></span>
