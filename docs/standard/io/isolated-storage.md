## <a name="data-compartments-and-stores"></a><span data-ttu-id="44990-121">Секции данных и хранилища</span><span class="sxs-lookup"><span data-stu-id="44990-121">Data Compartments and Stores</span></span>

<span data-ttu-id="44990-242">Изолированное хранилище может оказаться полезным во многих случаях, в том числе в следующих четырех ситуациях:</span><span class="sxs-lookup"><span data-stu-id="44990-242">Isolated storage is useful in many situations, including these four scenarios:</span></span>

- <span data-ttu-id="44990-243">Загружаемые элементы управления.</span><span class="sxs-lookup"><span data-stu-id="44990-243">Downloaded controls.</span></span> <span data-ttu-id="44990-244">Элементам управления, загружаемым через Интернет, не разрешено записывать данные на жесткий диск посредством обычных классов ввода-вывода, но они могут использовать изолированное хранилище для хранения параметров пользователя и состояний приложения.</span><span class="sxs-lookup"><span data-stu-id="44990-244">Managed code controls downloaded from the Internet are not allowed to write to the hard drive through normal I/O classes, but they can use isolated storage to persist users' settings and application states.</span></span>

- <span data-ttu-id="44990-245">Хранение общих компонентов.</span><span class="sxs-lookup"><span data-stu-id="44990-245">Shared component storage.</span></span> <span data-ttu-id="44990-246">Компоненты, общие для нескольких приложений, могут использовать изолированное хранилище для обеспечения управляемого доступа к хранилищам данных.</span><span class="sxs-lookup"><span data-stu-id="44990-246">Components that are shared between applications can use isolated storage to provide controlled access to data stores.</span></span>

- <span data-ttu-id="44990-247">Хранение на сервере.</span><span class="sxs-lookup"><span data-stu-id="44990-247">Server storage.</span></span> <span data-ttu-id="44990-248">Серверные приложения могут использовать изолированные хранилища для представления отдельных хранилищ большому числу пользователей, обращающихся с запросами к приложению.</span><span class="sxs-lookup"><span data-stu-id="44990-248">Server applications can use isolated storage to provide individual stores for a large number of users making requests to the application.</span></span> <span data-ttu-id="44990-249">Так как изолированное хранилище всегда выделяется пользователем, сервер должен выполнять роль пользователя, создающего запрос.</span><span class="sxs-lookup"><span data-stu-id="44990-249">Because isolated storage is always segregated by user, the server must impersonate the user making the request.</span></span> <span data-ttu-id="44990-250">В этом случае данные изолируются на основе идентификатора субъекта, который является тем же идентификатором, который используется приложением для того, чтобы различать пользователей.</span><span class="sxs-lookup"><span data-stu-id="44990-250">In this case, data is isolated based on the identity of the principal, which is the same identity the application uses to distinguish between its users.</span></span>

- <span data-ttu-id="44990-251">Перемещение.</span><span class="sxs-lookup"><span data-stu-id="44990-251">Roaming.</span></span> <span data-ttu-id="44990-252">Приложения могут также использовать изолированное хранилище с перемещаемыми профилями пользователей.</span><span class="sxs-lookup"><span data-stu-id="44990-252">Applications can also use isolated storage with roaming user profiles.</span></span> <span data-ttu-id="44990-253">Это позволяет изолированным данным пользователя перемещаться вслед за его профилем.</span><span class="sxs-lookup"><span data-stu-id="44990-253">This allows a user's isolated stores to roam with the profile.</span></span>

<span data-ttu-id="44990-254">Изолированное хранилище не следует использовать в следующих ситуациях:</span><span class="sxs-lookup"><span data-stu-id="44990-254">You should not use isolated storage in the following situations:</span></span>

- <span data-ttu-id="44990-255">Для хранения важных конфиденциальных данных, таких как незашифрованные ключи или пароли, поскольку изолированное хранилище не защищено от высоконадежного кода, от неуправляемого кода и от доверенных пользователей компьютера.</span><span class="sxs-lookup"><span data-stu-id="44990-255">To store high-value secrets, such as unencrypted keys or passwords, because isolated storage is not protected from highly trusted code, from unmanaged code, or from trusted users of the computer.</span></span>

- <span data-ttu-id="44990-256">Для хранения кода.</span><span class="sxs-lookup"><span data-stu-id="44990-256">To store code.</span></span>

- <span data-ttu-id="44990-257">Для хранения параметров конфигурации и развертывания, которые контролируются администраторами.</span><span class="sxs-lookup"><span data-stu-id="44990-257">To store configuration and deployment settings, which administrators control.</span></span> <span data-ttu-id="44990-258">(настройки пользователя не считаются параметрами конфигурации, т.к. они не управляются администраторами).</span><span class="sxs-lookup"><span data-stu-id="44990-258">(User preferences are not considered to be configuration settings because administrators do not control them.)</span></span>

<span data-ttu-id="44990-259">Многие приложения используют базы данных для хранения и изолирования данных, в которых одна или несколько строк могут представлять хранилище для определенного пользователя.</span><span class="sxs-lookup"><span data-stu-id="44990-259">Many applications use a database to store and isolate data, in which case one or more rows in a database might represent storage for a specific user.</span></span> <span data-ttu-id="44990-260">Применение изолированное хранилища вместо базы данных как правило выбирается в тех случаях, когда число пользователей невелико, когда использование базы данных влечет значительные потери производительности или когда отсутствуют средства для поддержания баз данных.</span><span class="sxs-lookup"><span data-stu-id="44990-260">You might choose to use isolated storage instead of a database when the number of users is small, when the overhead of using a database is significant, or when no database facility exists.</span></span> <span data-ttu-id="44990-261">Изолированное хранилище также представляет вполне реальную альтернативу в тех случаях, когда приложению требуется более гибкое и сложное хранилище, чем строка в базе данных.</span><span class="sxs-lookup"><span data-stu-id="44990-261">Also, when the application requires storage that is more flexible and complex than what a row in a database provides, isolated storage can provide a viable alternative.</span></span>
