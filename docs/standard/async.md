Избежать этого можно, только научившись пользоваться сложными шаблонами.</span><span class="sxs-lookup"><span data-stu-id="a5675-115">I/O APIs traditionally block by default, resulting in poor user experiences and hardware utilization unless you want to learn and use challenging patterns.</span></span> <span data-ttu-id="a5675-116">Основанные на задачах асинхронные интерфейсы API и модель асинхронного программирования на уровне языка меняют эту диспозицию и делают асинхронное выполнение режимом по умолчанию, при этом для их использования требуется освоить совсем немного новых понятий.</span><span class="sxs-lookup"><span data-stu-id="a5675-116">Task-based async APIs and the language-level asynchronous programming model invert this model, making async execution the default with few new concepts to learn.</span></span>

- <span data-ttu-id="a5675-118">Обрабатывает больше запросов сервера, предоставляя потокам возможность обрабатывать больше запросов во время ожидания результата от запросов ввода-вывода.</span><span class="sxs-lookup"><span data-stu-id="a5675-118">Handles more server requests by yielding threads to handle more requests while waiting for I/O requests to return.</span></span>
- <span data-ttu-id="a5675-119">Делает пользовательский интерфейс более быстрым, выделяя потоки для обработки действий в пользовательском интерфейсе во время ожидания запросов ввода-вывода и передавая затратные по времени операции другим ядрам ЦП.</span><span class="sxs-lookup"><span data-stu-id="a5675-119">Enables UIs to be more responsive by yielding threads to UI interaction while waiting for I/O requests and by transitioning long-running work to other CPU cores.</span></span>
