# <a name="runtime-callable-wrapper"></a><span data-ttu-id="0f537-103">Вызываемая оболочка времени выполнения</span><span class="sxs-lookup"><span data-stu-id="0f537-103">Runtime Callable Wrapper</span></span>
<span data-ttu-id="0f537-104">Среда CLR предоставляет доступ к COM-объектам через посредник, называемый вызываемой оболочкой времени выполнения.</span><span class="sxs-lookup"><span data-stu-id="0f537-104">The common language runtime exposes COM objects through a proxy called the runtime callable wrapper (RCW).</span></span> <span data-ttu-id="0f537-105">Хотя вызываемая оболочка времени выполнения выглядит для клиентов .NET обычным объектом, ее основная функция состоит в маршалинге вызовов между клиентом .NET и COM-объектом.</span><span class="sxs-lookup"><span data-stu-id="0f537-105">Although the RCW appears to be an ordinary object to .NET clients, its primary function is to marshal calls between a .NET client and a COM object.</span></span>  
  
 <span data-ttu-id="0f537-106">Среда выполнения создает по одной вызываемой оболочке времени выполнения для каждого COM-объекта независимо от числа существующих ссылок на этот объект.</span><span class="sxs-lookup"><span data-stu-id="0f537-106">The runtime creates exactly one RCW for each COM object, regardless of the number of references that exist on that object.</span></span> <span data-ttu-id="0f537-107">Среда выполнения поддерживает одну вызываемую оболочку времени выполнения на процесс для каждого объекта.</span><span class="sxs-lookup"><span data-stu-id="0f537-107">The runtime maintains a single RCW per process for each object.</span></span>  <span data-ttu-id="0f537-108">Если вы создаете вызываемую оболочку времени выполнения в одном домене приложения или подразделении, а затем передаете ссылку в другой домен приложения или подразделение, будет использоваться посредник для первого объекта.</span><span class="sxs-lookup"><span data-stu-id="0f537-108">If you create an RCW in one application domain or apartment, and then pass a reference to another application domain or apartment, a proxy to the first object will be used.</span></span>  <span data-ttu-id="0f537-109">Как показано на схеме ниже, любое количество управляемых клиентов может содержать ссылку на COM-объекты, предоставляющие интерфейсы INew и INewer.</span><span class="sxs-lookup"><span data-stu-id="0f537-109">As the following illustration shows, any number of managed clients can hold a reference to the COM objects that expose INew and INewer interfaces.</span></span>  

<span data-ttu-id="0f537-110">На следующем рисунке показана процедура доступа к COM-объектам с помощью вызываемой оболочки времени выполнения:</span><span class="sxs-lookup"><span data-stu-id="0f537-110">The following image shows the process for accessing COM objects through the runtime callable wrapper:</span></span>

 ![Процедура доступа к COM-объектам с помощью вызываемой оболочки времени выполнения.](./media/runtime-callable-wrapper/runtime-callable-wrapper.gif)  

 <span data-ttu-id="0f537-112">Используя метаданные, полученные из библиотеки типов, среда выполнения создает и вызываемый COM-объект, и оболочку для него.</span><span class="sxs-lookup"><span data-stu-id="0f537-112">Using metadata derived from a type library, the runtime creates both the COM object being called and a wrapper for that object.</span></span> <span data-ttu-id="0f537-113">Каждая вызываемая оболочка времени выполнения содержит кэш указателей интерфейса для COM-объекта, для которого она используется, и освобождает ссылку на COM-объект, когда она больше не нужна.</span><span class="sxs-lookup"><span data-stu-id="0f537-113">Each RCW maintains a cache of interface pointers on the COM object it wraps and releases its reference on the COM object when the RCW is no longer needed.</span></span> <span data-ttu-id="0f537-114">Среда выполнения выполняет сборку мусора для вызываемой оболочки времени выполнения.</span><span class="sxs-lookup"><span data-stu-id="0f537-114">The runtime performs garbage collection on the RCW.</span></span>  
  
 <span data-ttu-id="0f537-115">Среди прочих операций вызываемая оболочка времени выполнения осуществляет маршалинг данных между управляемым и неуправляемым кодом от имени упакованного в оболочку объекта.</span><span class="sxs-lookup"><span data-stu-id="0f537-115">Among other activities, the RCW marshals data between managed and unmanaged code, on behalf of the wrapped object.</span></span> <span data-ttu-id="0f537-116">В частности, вызываемая оболочка времени выполнения выполняет маршалинг аргументов и возвращаемых значений метода, если данные, которыми обмениваются клиент и сервер, представлены в них по-разному.</span><span class="sxs-lookup"><span data-stu-id="0f537-116">Specifically, the RCW provides marshaling for method arguments and method return values whenever the client and server have different representations of the data passed between them.</span></span>  
  
 <span data-ttu-id="0f537-117">Стандартная оболочка обеспечивает выполнение встроенных правил маршалинга.</span><span class="sxs-lookup"><span data-stu-id="0f537-117">The standard wrapper enforces built-in marshaling rules.</span></span> <span data-ttu-id="0f537-118">Например, когда клиент .NET передает в неуправляемый объект тип String как часть аргумента, оболочка преобразует эту строку в тип BSTR.</span><span class="sxs-lookup"><span data-stu-id="0f537-118">For example, when a .NET client passes a String type as part of an argument to an unmanaged object, the wrapper converts the string to a BSTR type.</span></span> <span data-ttu-id="0f537-119">Если COM-объект возвращает управляемому вызывающему объекту данные типа BSTR, вызывающий объект получит данные типа String.</span><span class="sxs-lookup"><span data-stu-id="0f537-119">Should the COM object return a BSTR to its managed caller, the caller receives a String.</span></span> <span data-ttu-id="0f537-120">И клиент, и сервер отправляют и получают данные в понятном им представлении.</span><span class="sxs-lookup"><span data-stu-id="0f537-120">Both the client and the server send and receive data that is familiar to them.</span></span> <span data-ttu-id="0f537-121">Преобразование других типов не требуется.</span><span class="sxs-lookup"><span data-stu-id="0f537-121">Other types require no conversion.</span></span> <span data-ttu-id="0f537-122">Например, стандартная оболочка всегда передает 4-байтовое целое число между управляемым и неуправляемым кодом, не преобразуя тип.</span><span class="sxs-lookup"><span data-stu-id="0f537-122">For instance, a standard wrapper will always pass a 4-byte integer between managed and unmanaged code without converting the type.</span></span>
