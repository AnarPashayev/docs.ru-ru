<span data-ttu-id="d643f-104">При выборе целевой платформы для приложения или библиотеки вы указываете набор API-интерфейсов, которые вы хотите сделать доступными для приложения или библиотеки.</span><span class="sxs-lookup"><span data-stu-id="d643f-104">When you target a framework in an app or library, you're specifying the set of APIs that you'd like to make available to the app or library.</span></span> <span data-ttu-id="d643f-105">Целевая платформа указывается в файле проекта с помощью моникеров целевой платформы (TFM).</span><span class="sxs-lookup"><span data-stu-id="d643f-105">You specify the target framework in your project file using Target Framework Monikers (TFMs).</span></span>

<span data-ttu-id="d643f-106">Приложение или библиотека могут быть предназначены для версии [.NET Standard](net-standard.md).</span><span class="sxs-lookup"><span data-stu-id="d643f-106">An app or library can target a version of [.NET Standard](net-standard.md).</span></span> <span data-ttu-id="d643f-107">Версии .NET Standard представляют стандартные наборы API-интерфейсов во всех реализациях .NET.</span><span class="sxs-lookup"><span data-stu-id="d643f-107">.NET Standard versions represent standardized sets of APIs across all .NET implementations.</span></span> <span data-ttu-id="d643f-108">Например, библиотека может быть предназначена для .NET Standard 1.6 и получить доступ к API-интерфейсам, которые работают в .NET Core и .NET Framework с одной и той же базой кода.</span><span class="sxs-lookup"><span data-stu-id="d643f-108">For example, a library can target .NET Standard 1.6 and gain access to APIs that function across .NET Core and .NET Framework using the same codebase.</span></span>

<span data-ttu-id="d643f-109">Приложение или библиотека могут также работать в конкретной реализации .NET. В этом случае они получают доступ к API-интерфейсам конкретной реализации.</span><span class="sxs-lookup"><span data-stu-id="d643f-109">An app or library can also target a specific .NET implementation to gain access to implementation-specific APIs.</span></span> <span data-ttu-id="d643f-110">Например, приложение, предназначенное для Xamarin.iOS (например, `Xamarin.iOS10`), получает доступ к предоставленным Xamarin iOS программам-оболочкам API для iOS 10 или приложение, ориентированное на универсальную платформу Windows (UWP, `uap10.0`) имеет доступ к API-интерфейсам, которые компилируется для устройств под управлением Windows 10.</span><span class="sxs-lookup"><span data-stu-id="d643f-110">For example, an app that targets Xamarin.iOS (for example, `Xamarin.iOS10`) gets access to Xamarin-provided iOS API wrappers for iOS 10, or an app that targets the Universal Windows Platform (UWP, `uap10.0`) has access to APIs that compile for devices that run Windows 10.</span></span>

<span data-ttu-id="d643f-111">Для некоторых целевых платформ (например, .NET Framework) API-интерфейсы определяются сборками, устанавливаемыми платформой в системе, в число которых могут входить API-интерфейсы платформы приложений (например, ASP.NET).</span><span class="sxs-lookup"><span data-stu-id="d643f-111">For some target frameworks (for example, the .NET Framework), the APIs are defined by the assemblies that the framework installs on a system and may include application framework APIs (for example, ASP.NET).</span></span>

<span data-ttu-id="d643f-112">Для целевых платформ на основе пакетов (например, .NET Standard и .NET Core) API-интерфейсы определяются пакетами в составе приложения или библиотеки.</span><span class="sxs-lookup"><span data-stu-id="d643f-112">For package-based target frameworks (for example, .NET Standard and .NET Core), the APIs are defined by the packages included in the app or library.</span></span> <span data-ttu-id="d643f-113">*Метапакет* — это пакет NuGet, не имеющий собственного содержимого, но имеющий список зависимостей (другие пакеты).</span><span class="sxs-lookup"><span data-stu-id="d643f-113">A *metapackage* is a NuGet package that has no content of its own but is a list of dependencies (other packages).</span></span> <span data-ttu-id="d643f-114">В этом случае целевая платформа на основе пакетов NuGet неявно задает метапакет, который ссылается на все пакеты, составляющие платформу.</span><span class="sxs-lookup"><span data-stu-id="d643f-114">A NuGet package-based target framework implicitly specifies a metapackage that references all the packages that together make up the framework.</span></span>

## <a name="how-to-specify-target-frameworks"></a><span data-ttu-id="d643f-198">Как указать целевые платформы</span><span class="sxs-lookup"><span data-stu-id="d643f-198">How to specify target frameworks</span></span>

<span data-ttu-id="d643f-199">Целевые платформы указываются в файле проекта.</span><span class="sxs-lookup"><span data-stu-id="d643f-199">Target frameworks are specified in your project file.</span></span> <span data-ttu-id="d643f-200">Если указана одна целевая платформа, используйте элемент **TargetFramework**.</span><span class="sxs-lookup"><span data-stu-id="d643f-200">When a single target framework is specified, use the **TargetFramework** element.</span></span> <span data-ttu-id="d643f-201">В следующем файле проекта консольного приложения показано, как указать целевую платформу .NET Core 3.0:</span><span class="sxs-lookup"><span data-stu-id="d643f-201">The following console app project file demonstrates how to target .NET Core 3.0:</span></span>

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>netcoreapp3.0</TargetFramework>
  </PropertyGroup>

</Project>
```

<span data-ttu-id="d643f-202">При указании нескольких целевых платформ можно условно ссылаться на сборки для каждой целевой платформы.</span><span class="sxs-lookup"><span data-stu-id="d643f-202">When you specify multiple target frameworks, you may conditionally reference assemblies for each target framework.</span></span> <span data-ttu-id="d643f-203">В коде можно условно компилировать эти сборки с использованием символов препроцессора с логикой *if-then-else*.</span><span class="sxs-lookup"><span data-stu-id="d643f-203">In your code, you can conditionally compile against those assemblies by using preprocessor symbols with *if-then-else* logic.</span></span>

<span data-ttu-id="d643f-204">Следующий файл проекта библиотеки предназначен для API-интерфейсов .NET Standard (`netstandard1.4`) и API-интерфейсов .NET Framework (`net40` и `net45`).</span><span class="sxs-lookup"><span data-stu-id="d643f-204">The following library project file targets APIs of .NET Standard (`netstandard1.4`) and APIs of the .NET Framework (`net40` and `net45`).</span></span> <span data-ttu-id="d643f-205">Используйте множественный элемент **TargetFrameworks** с несколькими целевыми платформами.</span><span class="sxs-lookup"><span data-stu-id="d643f-205">Use the plural **TargetFrameworks** element with multiple target frameworks.</span></span> <span data-ttu-id="d643f-206">Обратите внимание, каким образом атрибуты `Condition` включают пакеты, связанные с конкретной реализацией, при компиляции библиотеки для двух TFM .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="d643f-206">Note how the `Condition` attributes include implementation-specific packages when the library is compiled for the two .NET Framework TFMs:</span></span>

```xml

    <TargetFrameworks>netstandard1.4;net40;net45</TargetFrameworks>
  </PropertyGroup>

  <!-- Conditionally obtain references for the .NET Framework 4.0 target -->

  </ItemGroup>
```
