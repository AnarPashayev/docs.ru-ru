<span data-ttu-id="c8792-103">Сущность является декларацией, определяющей имя, которое будет использоваться в XML-документе вместо содержимого или разметки.</span><span class="sxs-lookup"><span data-stu-id="c8792-103">An entity is a declaration that states a name to be used in the XML in place of content or markup.</span></span> <span data-ttu-id="c8792-104">Сущность состоит из двух частей.</span><span class="sxs-lookup"><span data-stu-id="c8792-104">There are two parts to entities.</span></span> <span data-ttu-id="c8792-105">Во-первых, в декларации сущности необходимо связать имя с заменяющим его содержимым.</span><span class="sxs-lookup"><span data-stu-id="c8792-105">First, you must tie a name to the replacement content using an entity declaration.</span></span> <span data-ttu-id="c8792-106">Декларация сущности создается с помощью синтаксиса `<!ENTITY name "value">` определения DTD или схемы XML.</span><span class="sxs-lookup"><span data-stu-id="c8792-106">An used in the XML, it is called an entity reference.</span></span> <span data-ttu-id="c8792-109">Например, следующая декларация сущности определяет сущность с именем `publisher`, связанную с содержимым «Microsoft Press».</span><span class="sxs-lookup"><span data-stu-id="c8792-109">For example, the following entity declaration declares an entity of the name `publisher` being associated with the ```  
  
 <span data-ttu-id="c8792-110">В следующем примере показано использование декларации сущности в XML в качестве ссылки на сущность.</span><span class="sxs-lookup"><span data-stu-id="c8792-110">The following example shows the use of this entity declaration in XML as an entity reference.</span></span>

  
 <span data-ttu-id="c8792-111">Некоторые средства синтаксического анализа автоматически развертывают сущности при загрузке документа в память.</span><span class="sxs-lookup"><span data-stu-id="c8792-111">Some parsers automatically expand entities when a document is loaded into memory.</span></span> <span data-ttu-id="c8792-112">Поэтому, когда XML-документ считывается в память, декларации сущностей запоминаются и сохраняются.</span><span class="sxs-lookup"><span data-stu-id="c8792-112">Therefore, when the XML is being read into memory, entity declarations are remembered and saved.</span></span> <span data-ttu-id="c8792-113">Если средство синтаксического анализа впоследствии встречает символы `&;`, определяющие ссылку на общую сущность, оно ищет это имя в таблице деклараций сущностей.</span><span class="sxs-lookup"><span data-stu-id="c8792-113">When the parser subsequently encounters `&;` characters, which identify a general entity reference, the parser looks up that name in an entity declaration table.</span></span> <span data-ttu-id="c8792-114">Ссылка `&publisher;` заменяется содержимым, которое она представляет.</span><span class="sxs-lookup"><span data-stu-id="c8792-114">The reference, `&publisher;` is replaced by the content that it represents.</span></span> <span data-ttu-id="c8792-115">Используя следующий XML-код,</span><span class="sxs-lookup"><span data-stu-id="c8792-115">Using the following XML,</span></span>  
  
```xml  
<author>Fred</author>  
<pubinfo>Published by &publisher;</pubinfo>  
```  
  
 <span data-ttu-id="c8792-116">разворачивает ссылку на сущность и заменяет `&publisher;` содержимым Microsoft Press, формируя следующий расширенный XML.</span><span class="sxs-lookup"><span data-stu-id="c8792-116">expanding the entity reference and replacing the `&publisher;` with the Microsoft Press content gives the following expanded XML.</span></span>  
  
 <span data-ttu-id="c8792-117">**Выходные данные**</span><span class="sxs-lookup"><span data-stu-id="c8792-117">**Output**</span></span>  
  

 <span data-ttu-id="c8792-120">![Блок-схема иерархии типов сущностей](../../../../docs/standard/data/xml/media/entity-hierarchy.gif "Entity_hierarchy")</span><span class="sxs-lookup"><span data-stu-id="c8792-120">![flow chart of entity type hierarchy](../../../../docs/standard/data/xml/media/entity-hierarchy.gif "Entity_hierarchy")</span></span>  
  
 <span data-ttu-id="c8792-121">По умолчанию реализация модели DOM на платформе Microsoft .NET Framework сохраняет ссылки на сущности и не развертывает их при загрузке XML.</span><span class="sxs-lookup"><span data-stu-id="c8792-121">The default for the Microsoft .NET Framework implementation of the XML Document Object Model (DOM) is to preserve the entities references and not expand the entities when the XML is loaded.</span></span> <span data-ttu-id="c8792-122">Это означает, что при загрузке документа в модель DOM создается узел **XmlEntityReference`&publisher;`, содержащий ссылочную переменную** , с дочерними узлами, представляющими содержимое сущности, объявленной в определении DTD.</span><span class="sxs-lookup"><span data-stu-id="c8792-122">The implication of this is that as a document is loaded in the DOM, an **XmlEntityReference** node containing the reference variable `&publisher;` is created, with child nodes representing the content in the entity declared in the DTD.</span></span>  
  
 <span data-ttu-id="c8792-123">Следующая схема на примере объявления сущности `<!ENTITY publisher "Microsoft Press">` демонстрирует узлы **XmlEntity** и **XmlText**, создаваемые на основе этого объявления.</span><span class="sxs-lookup"><span data-stu-id="c8792-123">Using the `<!ENTITY publisher "Microsoft Press">` entity declaration, the following diagram shows the **XmlEntity** and **XmlText** nodes created from this declaration.</span></span>
