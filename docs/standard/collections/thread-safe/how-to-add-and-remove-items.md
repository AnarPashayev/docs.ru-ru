<span data-ttu-id="82ecc-121">Кроме того, несмотря на то, что все методы <xref:System.Collections.Concurrent.ConcurrentDictionary%602> потокобезопасны, не все методы атомарны, например <xref:System.Collections.Concurrent.ConcurrentDictionary%602.GetOrAdd%2A> и <xref:System.Collections.Concurrent.ConcurrentDictionary%602.AddOrUpdate%2A>.</span><span class="sxs-lookup"><span data-stu-id="82ecc-121">Also, although all methods of <xref:System.Collections.Concurrent.ConcurrentDictionary%602> are thread-safe, not all methods are atomic, specifically <xref:System.Collections.Concurrent.ConcurrentDictionary%602.GetOrAdd%2A> and <xref:System.Collections.Concurrent.ConcurrentDictionary%602.AddOrUpdate%2A>.</span></span> <span data-ttu-id="82ecc-122">Чтобы предотвратить блокировку всех потоков неизвестным кодом, пользовательский делегат, передаваемый этим методам, вызывается вне внутренней блокировки словаря.</span><span class="sxs-lookup"><span data-stu-id="82ecc-122">To prevent unknown code from blocking all threads, the user delegate that's passed to these methods is invoked outside of the dictionary's internal lock.</span></span> <span data-ttu-id="82ecc-123">Поэтому может произойти следующая последовательность событий.</span><span class="sxs-lookup"><span data-stu-id="82ecc-123">Therefore, it's possible for this sequence of events to occur:</span></span>

1. <span data-ttu-id="82ecc-124">_Поток threadA_ вызывает <xref:System.Collections.Concurrent.ConcurrentDictionary%602.GetOrAdd%2A>, не находит элемент и создает элемент для добавления, вызывая делегат `valueFactory`.</span><span class="sxs-lookup"><span data-stu-id="82ecc-124">_threadA_ calls <xref:System.Collections.Concurrent.ConcurrentDictionary%602.GetOrAdd%2A>, finds no item, and creates a new item to add by invoking the `valueFactory` delegate.</span></span>

1. <span data-ttu-id="82ecc-125">_Поток threadB_ одновременно вызывает <xref:System.Collections.Concurrent.ConcurrentDictionary%602.GetOrAdd%2A>, вызывается его делегат `valueFactory`, и этот поток оказывается в состоянии внутренней блокировки раньше _потока threadA_, поэтому его новая пара "ключ — значение" добавляется в словарь.</span><span class="sxs-lookup"><span data-stu-id="82ecc-125">_threadB_ calls <xref:System.Collections.Concurrent.ConcurrentDictionary%602.GetOrAdd%2A> concurrently, its `valueFactory` delegate is invoked and it arrives at the internal lock before _threadA_, and so its new key-value pair is added to the dictionary.</span></span>

1. <span data-ttu-id="82ecc-126">Пользовательский делегат _потока threadA_ завершает работу, поток достигает блокировки, но видит, что теперь элемент уже существует.</span><span class="sxs-lookup"><span data-stu-id="82ecc-126">_threadA's_ user delegate completes, and the thread arrives at the lock, but now sees that the item exists already.</span></span>
