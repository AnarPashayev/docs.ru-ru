# <a name="when-to-deploy-windows-containers-to-azure-vms-iaas-cloud"></a>Когда следует развертывать контейнеры Windows на виртуальных машинах Azure (облачная среда IaaS)

Если ваша организация использует виртуальные машины Azure, даже если используются контейнеры Windows, вы по-прежнему освободили IaaS. Это означает, освободили операций инфраструктуры, исправления ОС виртуальной Машины и сложность инфраструктуры для масштабируемых приложений, при необходимости развертывания нескольких виртуальных машин в инфраструктуре с балансировкой нагрузки. Ниже приведены основные сценарии использования контейнеров Windows на виртуальной Машине Azure.

- **Среда разработки и тестирования**: Виртуальную Машину в облаке идеально подходит для разработки и тестирования в облаке. Можно быстро создать или остановка среды зависит от ваших потребностей.

- **Требуется масштабируемость малых и средних**: В сценариях, где могут потребоваться всего за несколько виртуальных машин для рабочей среды управление небольшое количество виртуальных машин может быть доступной пока можно переместить в более сложных средах PaaS, такие как оркестраторы.

- **Рабочей среде с существующими средствами развертывания**: Которые могут перемещаться из локальной среды, в котором вы вложили средства в средства, упрощающие сложного развертывания на виртуальных машинах или серверах без операционной системы (например, Puppet или аналогичных средств). Чтобы переместить в облако с минимальными изменениями процедуры развертывания рабочей среды, могут продолжить использовать эти средства для развертывания на виртуальных машинах Azure. Тем не менее стоит использовать контейнеры Windows как единицей развертывания для улучшения развертывания.
