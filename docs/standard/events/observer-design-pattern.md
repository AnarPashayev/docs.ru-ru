<span data-ttu-id="7544d-111">Шаблон разработки наблюдателя подходит для распределенных push-уведомлений, так как он поддерживает четкое разделение двух разных компонентов или уровней приложения, таких как уровень источника данных (бизнес-логика) и уровень пользовательского интерфейса (отображение).</span><span class="sxs-lookup"><span data-stu-id="7544d-111">The observer design pattern is suitable for distributed push-based notifications, because it supports a clean separation between two different components or application layers, such as a data source (business logic) layer and a user interface (display) layer.</span></span> <span data-ttu-id="7544d-112">Шаблон можно реализовывать во всех случаях, когда поставщик использует обратные вызовы для предоставления текущих сведений клиентам.</span><span class="sxs-lookup"><span data-stu-id="7544d-112">The pattern can be implemented whenever a provider uses callbacks to supply its clients with current information.</span></span>

<span data-ttu-id="7544d-113">Для реализации шаблона необходимы перечисленные ниже элементы.</span><span class="sxs-lookup"><span data-stu-id="7544d-113">Implementing the pattern requires that you provide the following:</span></span>

- <span data-ttu-id="7544d-117">Наблюдатель, то есть объект, получающий уведомления от поставщика.</span><span class="sxs-lookup"><span data-stu-id="7544d-117">An observer, which is an object that receives notifications from a provider.</span></span> <span data-ttu-id="7544d-118">Наблюдатель — это класс или структура, реализующие интерфейс <xref:System.IObserver%601>.</span><span class="sxs-lookup"><span data-stu-id="7544d-118">An observer is a class or structure that implements the <xref:System.IObserver%601> interface.</span></span> <span data-ttu-id="7544d-119">Наблюдатель должен реализовывать три метода, которые вызываются поставщиком:</span><span class="sxs-lookup"><span data-stu-id="7544d-119">The observer must implement three methods, all of which are called by the provider:</span></span>

  - <span data-ttu-id="7544d-120">метод <xref:System.IObserver%601.OnNext%2A?displayProperty=nameWithType>, предоставляющий наблюдателю новые или текущие сведения;</span><span class="sxs-lookup"><span data-stu-id="7544d-120"><xref:System.IObserver%601.OnNext%2A?displayProperty=nameWithType>, which supplies the observer with new or current information.</span></span>

  - <span data-ttu-id="7544d-121">метод <xref:System.IObserver%601.OnError%2A?displayProperty=nameWithType>, информирующий наблюдатель о том, что произошла ошибка;</span><span class="sxs-lookup"><span data-stu-id="7544d-121"><xref:System.IObserver%601.OnError%2A?displayProperty=nameWithType>, which informs the observer that an error has occurred.</span></span>

  - <span data-ttu-id="7544d-122">метод <xref:System.IObserver%601.OnCompleted%2A?displayProperty=nameWithType>, указывающий, что поставщик завершил отправку уведомлений.</span><span class="sxs-lookup"><span data-stu-id="7544d-122"><xref:System.IObserver%601.OnCompleted%2A?displayProperty=nameWithType>, which indicates that the provider has finished sending notifications.</span></span>

- <span data-ttu-id="7544d-123">Механизм, который позволяет поставщику отслеживать наблюдатели.</span><span class="sxs-lookup"><span data-stu-id="7544d-123">A mechanism that allows the provider to keep track of observers.</span></span> <span data-ttu-id="7544d-124">Как правило, поставщик использует объект контейнера, например объект <xref:System.Collections.Generic.List%601?displayProperty=nameWithType>, для хранения ссылок на реализации <xref:System.IObserver%601>, которые подписаны на уведомления.</span><span class="sxs-lookup"><span data-stu-id="7544d-124">Typically, the provider uses a container object, such as a <xref:System.Collections.Generic.List%601?displayProperty=nameWithType> object, to hold references to the <xref:System.IObserver%601> implementations that have subscribed to notifications.</span></span> <span data-ttu-id="7544d-125">Использование контейнера хранилища в этих целях позволяет поставщику обрабатывать от нуля до неограниченного числа наблюдателей.</span><span class="sxs-lookup"><span data-stu-id="7544d-125">Using a storage container for this purpose enables the provider to handle zero to an unlimited number of observers.</span></span> <span data-ttu-id="7544d-126">Порядок, в котором наблюдатели получают уведомления, не определен. - <span data-ttu-id="7544d-127">Реализация <xref:System.IDisposable>, которая позволяет поставщику удалять наблюдатели по завершении уведомления.</span><span class="sxs-lookup"><span data-stu-id="7544d-127">An <xref:System.IDisposable> implementation that enables the provider to remove observers when notification is complete.</span></span> <span data-ttu-id="7544d-128">Наблюдатели получают ссылку на реализацию <xref:System.IDisposable> из метода <xref:System.IObservable%601.Subscribe%2A>, поэтому они также могут вызывать метод <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType>, чтобы отменить подписку, прежде чем поставщик завершит отправку уведомлений.</span><span class="sxs-lookup"><span data-stu-id="7544d-128">Observers receive a reference to the <xref:System.IDisposable> implementation from the <xref:System.IObservable%601.Subscribe%2A> method, so they can also call the <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> method to unsubscribe before the provider has finished sending notifications.</span></span>
