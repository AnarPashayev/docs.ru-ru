# <a name="reference-assemblies"></a><span data-ttu-id="1542c-103">Базовые сборки</span><span class="sxs-lookup"><span data-stu-id="1542c-103">Reference assemblies</span></span>

<span data-ttu-id="1542c-104">*Базовые сборки* являются особым типом сборки, которая содержит только минимальный объем метаданных, необходимый для представления общедоступного API-интерфейса библиотеки.</span><span class="sxs-lookup"><span data-stu-id="1542c-104">*Reference assemblies* are a special type of assembly that contain only the minimum amount of metadata required to represent the library's public API surface.</span></span> <span data-ttu-id="1542c-105">Такие сборки включают в себя объявления для всех элементов, которые важны при указании ссылки на сборку в средствах сборки, но исключают все реализации элементов, а также объявления закрытых элементов, не имеющих наблюдаемого влияния на их контракт API.</span><span class="sxs-lookup"><span data-stu-id="1542c-105">They include declarations for all members that are significant when referencing an assembly in build tools, but exclude all member implementations and declarations of private members that have no observable impact on their API contract.</span></span> <span data-ttu-id="1542c-106">Обычные сборки, напротив, называются *сборками реализации*.</span><span class="sxs-lookup"><span data-stu-id="1542c-106">In contrast, regular assemblies are called *implementation assemblies*.</span></span>

<span data-ttu-id="1542c-107">Базовые сборки не могут быть загружены для выполнения, но они могут передаваться как входные данные компилятора так же, как сборки реализации.</span><span class="sxs-lookup"><span data-stu-id="1542c-107">Reference assemblies can't be loaded for execution, but they can be passed as compiler input in the same way as implementation assemblies.</span></span> <span data-ttu-id="1542c-108">Базовые сборки обычно распределяются с помощью пакета средств разработки программного обеспечения (SDK) определенной платформы или библиотеки.</span><span class="sxs-lookup"><span data-stu-id="1542c-108">Reference assemblies are usually distributed with the Software Development Kit (SDK) of a particular platform or library.</span></span>

## <a name="reference-assemblies-structure"></a><span data-ttu-id="1542c-154">Структура базовых сборок</span><span class="sxs-lookup"><span data-stu-id="1542c-154">Reference assemblies structure</span></span>

<span data-ttu-id="1542c-155">Базовые сборки являются расширением связанной концепции — *сборок, содержащих только метаданные*.</span><span class="sxs-lookup"><span data-stu-id="1542c-155">Reference assemblies are an expansion of the related concept, *metadata-only assemblies*.</span></span> <span data-ttu-id="1542c-156">Тела методов в сборках, состоящих только из метаданных, заменяются одним телом `throw null`, но такие сборки включают все члены, кроме анонимных типов.</span><span class="sxs-lookup"><span data-stu-id="1542c-156">Metadata-only assemblies have their method bodies replaced with a single `throw null` body, but include all members except anonymous types.</span></span> <span data-ttu-id="1542c-157">Тело `throw null` используется для того, чтобы могло выполняться средство **PEVerify** для проверки полноты метаданных, что было бы невозможно при отсутствии тела.</span><span class="sxs-lookup"><span data-stu-id="1542c-157">The reason for using `throw null` bodies (as opposed to no bodies) is so that **PEVerify** can run and pass (thus validating the completeness of the metadata).</span></span>

<span data-ttu-id="1542c-158">Базовые сборки удаляют метаданные (закрытые члены) из сборок, содержащих только метаданные.</span><span class="sxs-lookup"><span data-stu-id="1542c-158">Reference assemblies further remove metadata (private members) from metadata-only assemblies:</span></span>

- <span data-ttu-id="1542c-159">Базовая сборка содержит ссылки только на необходимые компоненты в слое доступа API.</span><span class="sxs-lookup"><span data-stu-id="1542c-159">A reference assembly only has references for what it needs in the API surface.</span></span> <span data-ttu-id="1542c-160">Реальная сборка может иметь дополнительные ссылки, связанные с конкретной реализацией.</span><span class="sxs-lookup"><span data-stu-id="1542c-160">The real assembly may have additional references related to specific implementations.</span></span> <span data-ttu-id="1542c-161">Например, базовая сборка для `class C { private void M() { dynamic d = 1; ... } }` не ссылается на типы, требуемые для `dynamic`.</span><span class="sxs-lookup"><span data-stu-id="1542c-161">For instance, the reference assembly for `class C { private void M() { dynamic d = 1; ... } }` doesn't reference any types required for `dynamic`.</span></span>
- <span data-ttu-id="1542c-162">Закрытые функции-члены (методы, свойства и события) удаляются в случае, если их удаление не скажется заметно на компиляции.</span><span class="sxs-lookup"><span data-stu-id="1542c-162">Private function-members (methods, properties, and events) are removed in cases where their removal doesn't observably impact compilation.</span></span> <span data-ttu-id="1542c-163">Если нет атрибутов [InternalsVisibleTo](xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute), внутренние элементы функции также удаляются.</span><span class="sxs-lookup"><span data-stu-id="1542c-163">If there are no [InternalsVisibleTo](xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute) attributes, internal function members are also removed.</span></span>

<span data-ttu-id="1542c-164">Метаданные в базовых сборках продолжают хранить следующие сведения:</span><span class="sxs-lookup"><span data-stu-id="1542c-164">The metadata in reference assemblies continues to keep the following information:</span></span>

- <span data-ttu-id="1542c-165">все типы, включая закрытые и вложенные;</span><span class="sxs-lookup"><span data-stu-id="1542c-165">All types, including private and nested types.</span></span>
- <span data-ttu-id="1542c-166">все атрибуты, даже внутренние;</span><span class="sxs-lookup"><span data-stu-id="1542c-166">All attributes, even internal ones.</span></span>
- <span data-ttu-id="1542c-167">все виртуальные методы;</span><span class="sxs-lookup"><span data-stu-id="1542c-167">All virtual methods.</span></span>
- <span data-ttu-id="1542c-168">явные реализации интерфейса;</span><span class="sxs-lookup"><span data-stu-id="1542c-168">Explicit interface implementations.</span></span>
- <span data-ttu-id="1542c-169">явно реализованные свойства и события, так как их методы доступа являются виртуальными;</span><span class="sxs-lookup"><span data-stu-id="1542c-169">Explicitly implemented properties and events, because their accessors are virtual.</span></span>
- <span data-ttu-id="1542c-170">все поля структур.</span><span class="sxs-lookup"><span data-stu-id="1542c-170">All fields of structures.</span></span>

<span data-ttu-id="1542c-171">Базовые сборки содержат атрибут [ReferenceAssembly](xref:System.Runtime.CompilerServices.ReferenceAssemblyAttribute) уровня сборки.</span><span class="sxs-lookup"><span data-stu-id="1542c-171">Reference assemblies include an assembly-level [ReferenceAssembly](xref:System.Runtime.CompilerServices.ReferenceAssemblyAttribute) attribute.</span></span> <span data-ttu-id="1542c-172">Этот атрибут может быть задан в исходном коде, в таком случае компилятору не требуется его синтезировать.</span><span class="sxs-lookup"><span data-stu-id="1542c-172">This attribute may be specified in source; then the compiler won't need to synthesize it.</span></span> <span data-ttu-id="1542c-173">Из-за этого атрибута среды выполнения не будут загружать базовые сборки для выполнения (однако они могут загружаться в режиме только для отражения).</span><span class="sxs-lookup"><span data-stu-id="1542c-173">Because of this attribute, runtimes will refuse to load reference assemblies for execution (but they can be loaded in reflection-only mode).</span></span>

<span data-ttu-id="1542c-174">Точные сведения о структуре базовой сборки зависят от версии компилятора.</span><span class="sxs-lookup"><span data-stu-id="1542c-174">Exact reference assembly structure details depend on the compiler version.</span></span> <span data-ttu-id="1542c-175">Более новые версии могут исключить дополнительные метаданные, если они не влияют на общедоступный API-интерфейса.</span><span class="sxs-lookup"><span data-stu-id="1542c-175">Newer versions may choose to exclude more metadata if it's determined as not affecting the public API surface.</span></span>

> [!NOTE]
> <span data-ttu-id="1542c-176">Сведения в этом разделе применимы только к базовым сборкам, которые созданы компиляторами Roslyn, начиная с C# версии 7.1 или Visual Basic версии 15.3.</span><span class="sxs-lookup"><span data-stu-id="1542c-176">Information in this section is applicable only to reference assemblies generated by Roslyn compilers starting from C# version 7.1 or Visual Basic version 15.3.</span></span> <span data-ttu-id="1542c-177">Структура базовых сборок для библиотек .NET Framework и .NET Core может отличаться некоторыми деталями, так как они используют собственный механизм создания базовых сборок.</span><span class="sxs-lookup"><span data-stu-id="1542c-177">The structure of reference assemblies for .NET Framework and .NET Core libraries can differ in some details, because they use their own mechanism of generating reference assemblies.</span></span> <span data-ttu-id="1542c-178">Например, они могут иметь совершенно пустые тела методов вместо тела `throw null`.</span><span class="sxs-lookup"><span data-stu-id="1542c-178">For example, they might have totally empty method bodies instead of the `throw null` body.</span></span> <span data-ttu-id="1542c-179">Но общие принципы по-прежнему применяются: у них нет пригодных для использования реализаций методов и они содержат метаданные только для элементов, которые имеют наблюдаемый эффект с точки зрения общедоступного API-интерфейса.</span><span class="sxs-lookup"><span data-stu-id="1542c-179">But the general principle still applies: they don't have usable method implementations and contain metadata only for members that have an observable impact from a public API perspective.</span></span>
