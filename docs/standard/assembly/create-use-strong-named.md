<span data-ttu-id="9558d-103">Строгое имя состоит из удостоверения сборки, включающего ее простое текстовое имя, номер версии и сведения о языке и региональных параметрах (если они имеются), а также открытый ключ и цифровую подпись.</span><span class="sxs-lookup"><span data-stu-id="9558d-103">A strong name consists of the assembly's identity—its simple text name, version number, and culture information (if provided)—plus a public key and a digital signature.</span></span> <span data-ttu-id="9558d-104">Оно создается из файла сборки с использованием соответствующего закрытого ключа.</span><span class="sxs-lookup"><span data-stu-id="9558d-104">It is generated from an assembly file using the corresponding private key.</span></span> <span data-ttu-id="9558d-105">(В файле сборки содержится манифест сборки, в котором указаны имена и хэши всех файлов, составляющих эту сборку.)</span><span class="sxs-lookup"><span data-stu-id="9558d-105">(The assembly file contains the assembly manifest, which contains the names and hashes of all the files that make up the assembly.)</span></span>

## <a name="strong-name-scenario"></a><span data-ttu-id="9558d-112">Сценарий со строгим именем</span><span class="sxs-lookup"><span data-stu-id="9558d-112">Strong name scenario</span></span>

<span data-ttu-id="9558d-113">В следующем сценарии описывается процедура подписи сборки со строгим именем и последующая ссылка на сборку с использованием этого имени.</span><span class="sxs-lookup"><span data-stu-id="9558d-113">The following scenario outlines the process of signing an assembly with a strong name and later referencing it by that name.</span></span>

1. <span data-ttu-id="9558d-114">Сборка A создается со строгим именем с помощью одного из следующих методов.</span><span class="sxs-lookup"><span data-stu-id="9558d-114">Assembly A is created with a strong name using one of the following methods:</span></span>

    - <span data-ttu-id="9558d-115">Использование среды разработки, поддерживающей создание строгих имен, например Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="9558d-115">Using a development environment that supports creating strong names, such as Visual Studio.</span></span>

    - <span data-ttu-id="9558d-116">Создание пары ключей шифрования с использованием [Программы строгих имен (Sn.exe)](../../framework/tools/sn-exe-strong-name-tool.md) и назначение этой пары ключей сборке с использованием компилятора командной строки или [компоновщика сборок (Al.exe)](../../framework/tools/al-exe-assembly-linker.md).</span><span class="sxs-lookup"><span data-stu-id="9558d-116">Creating a cryptographic key pair using the [Strong Name tool (Sn.exe)](../../framework/tools/sn-exe-strong-name-tool.md) and assigning that key pair to the assembly using either a command-line compiler or the [Assembly Linker (Al.exe)](../../framework/tools/al-exe-assembly-linker.md).</span></span> <span data-ttu-id="9558d-117">Windows SDK содержит оба инструмента: Sn.exe и Al.exe.</span><span class="sxs-lookup"><span data-stu-id="9558d-117">The Windows SDK provides both Sn.exe and Al.exe.</span></span>

2. <span data-ttu-id="9558d-118">Среда разработки или средство подписывает хэш файла, содержащего манифест сборки, закрытым ключом разработчика.</span><span class="sxs-lookup"><span data-stu-id="9558d-118">The development environment or tool signs the hash of the file containing the assembly's manifest with the developer's private key.</span></span> <span data-ttu-id="9558d-119">Эта цифровая подпись хранится в переносимом исполняемом файле (PE), который содержит манифест сборки А.</span><span class="sxs-lookup"><span data-stu-id="9558d-119">This digital signature is stored in the portable executable (PE) file that contains Assembly A's manifest.</span></span>

3. <span data-ttu-id="9558d-120">Сборка B является потребителем сборки A. Раздел ссылок манифеста сборки B содержит токен, представляющий открытый ключ сборки А.</span><span class="sxs-lookup"><span data-stu-id="9558d-120">Assembly B is a consumer of Assembly A. The reference section of Assembly B's manifest includes a token that represents Assembly A's public key.</span></span> <span data-ttu-id="9558d-121">Токен является частью полного открытого ключа и используется вместо самого ключа в целях экономии пространства.</span><span class="sxs-lookup"><span data-stu-id="9558d-121">A token is a portion of the full public key and is used rather than the key itself to save space.</span></span>

4. <span data-ttu-id="9558d-122">Среда CLR проверяет подпись строгого имени, когда сборка помещается в глобальный кэш сборок.</span><span class="sxs-lookup"><span data-stu-id="9558d-122">The common language runtime verifies the strong name signature when the assembly is placed in the global assembly cache.</span></span> <span data-ttu-id="9558d-123">При привязке по строгому имени во время выполнения среда CLR сравнивает ключ, сохраненный в манифесте сборки B, с ключом, используемым для создания строгого имени для сборки А. Если проверки безопасности .NET Framework пройдены и привязка выполнена успешно, у сборки B есть гарантия, что биты сборки A не были изменены и действительно получены от разработчиков сборки А.</span><span class="sxs-lookup"><span data-stu-id="9558d-123">When binding by strong name at run time, the common language runtime compares the key stored in Assembly B's manifest with the key used to generate the strong name for Assembly A. If the .NET Framework security checks pass and the bind succeeds, Assembly B has a guarantee that Assembly A's bits have not been tampered with and that these bits actually come from the developers of Assembly A.</span></span>

> [!NOTE]
> <span data-ttu-id="9558d-124">В этом сценарии не рассматриваются вопросы доверия.</span><span class="sxs-lookup"><span data-stu-id="9558d-124">This scenario doesn't address trust issues.</span></span> <span data-ttu-id="9558d-125">Помимо строгого имени сборки могут содержать полные подписи кода аутентификации Microsoft Authenticode.</span><span class="sxs-lookup"><span data-stu-id="9558d-125">Assemblies can carry full Microsoft Authenticode signatures in addition to a strong name.</span></span> <span data-ttu-id="9558d-126">Подписи Authenticode включают сертификат, который устанавливает отношения доверия.</span><span class="sxs-lookup"><span data-stu-id="9558d-126">Authenticode signatures include a certificate that establishes trust.</span></span> <span data-ttu-id="9558d-127">Важно отметить, что строгие имена не требуют подписи кода таким способом.</span><span class="sxs-lookup"><span data-stu-id="9558d-127">It's important to note that strong names don't require code to be signed in this way.</span></span> <span data-ttu-id="9558d-128">Строгие имена служат только для однозначной идентификации.</span><span class="sxs-lookup"><span data-stu-id="9558d-128">Strong names only provide a unique identity.</span></span>

## <a name="bypass-signature-verification-of-trusted-assemblies"></a><span data-ttu-id="9558d-129">Обход проверки подписей в доверенных сборках</span><span class="sxs-lookup"><span data-stu-id="9558d-129">Bypass signature verification of trusted assemblies</span></span>

<span data-ttu-id="9558d-130">Начиная с .NET Framework 3.5 с пакетом обновления 1 (SP1), подписи строгого имени не проверяются, если сборка загружается в домен приложения с полным доверием, например в домен приложения по умолчанию для зоны `MyComputer`.</span><span class="sxs-lookup"><span data-stu-id="9558d-130">Starting with the .NET Framework 3.5 Service Pack 1, strong-name signatures are not validated when an assembly is loaded into a full-trust application domain, such as the default application domain for the `MyComputer` zone.</span></span> <span data-ttu-id="9558d-131">Это называется функцией обхода строгих имен.</span><span class="sxs-lookup"><span data-stu-id="9558d-131">This is referred to as the strong-name bypass feature.</span></span> <span data-ttu-id="9558d-132">В среде с полным доверием всегда успешно обрабатываются запросы <xref:System.Security.Permissions.StrongNameIdentityPermission> для подписанных сбором с полным доверием независимо от их подписи.</span><span class="sxs-lookup"><span data-stu-id="9558d-132">In a full-trust environment, demands for <xref:System.Security.Permissions.StrongNameIdentityPermission> always succeed for signed, full-trust assemblies, regardless of their signature.</span></span> <span data-ttu-id="9558d-133">Функция обхода строгих имен позволяет избежать ненужных затрат ресурсов для проверки подписей строгих имен в сборках с полным доверием, что позволяет быстрее загружать такие сборки.</span><span class="sxs-lookup"><span data-stu-id="9558d-133">The strong-name bypass feature avoids the unnecessary overhead of strong-name signature verification of full-trust assemblies in this situation, allowing the assemblies to load faster.</span></span>

<span data-ttu-id="9558d-134">Функция обхода применима к любой сборке, подписанной со строгим именем и имеющей следующие характеристики.</span><span class="sxs-lookup"><span data-stu-id="9558d-134">The bypass feature applies to any assembly that is signed with a strong name and that has the following characteristics:</span></span>

- <span data-ttu-id="9558d-135">Полное доверие без доказательства <xref:System.Security.Policy.StrongName> (например, при наличии доказательства зоны `MyComputer`).</span><span class="sxs-lookup"><span data-stu-id="9558d-135">Fully trusted without <xref:System.Security.Policy.StrongName> evidence (for example, has `MyComputer` zone evidence).</span></span>

- <span data-ttu-id="9558d-136">Загрузка в домен <xref:System.AppDomain> с полным доверием.</span><span class="sxs-lookup"><span data-stu-id="9558d-136">Loaded into a fully trusted <xref:System.AppDomain>.</span></span>

- <span data-ttu-id="9558d-137">Загрузка из расположения со свойством <xref:System.AppDomainSetup.ApplicationBase%2A> домена <xref:System.AppDomain>.</span><span class="sxs-lookup"><span data-stu-id="9558d-137">Loaded from a location under the <xref:System.AppDomainSetup.ApplicationBase%2A> property of that <xref:System.AppDomain>.</span></span>

- <span data-ttu-id="9558d-138">Подпись осуществлена без задержки.</span><span class="sxs-lookup"><span data-stu-id="9558d-138">Not delay-signed.</span></span>

<span data-ttu-id="9558d-139">Эту возможность можно отключить для отдельных приложений или для компьютера.</span><span class="sxs-lookup"><span data-stu-id="9558d-139">This feature can be disabled for individual applications or for a computer.</span></span> <span data-ttu-id="9558d-140">См. практическое руководство по [ Отключение функции пропуска строгих имен](disable-strong-name-bypass-feature.md).</span><span class="sxs-lookup"><span data-stu-id="9558d-140">See [How to: Disable the strong-name bypass feature](disable-strong-name-bypass-feature.md).</span></span>
