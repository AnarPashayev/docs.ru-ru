<span data-ttu-id="77de7-104">Любая сборка, статическая или динамическая, содержит коллекцию данных с описанием того, как ее элементы связаны друг с другом.</span><span class="sxs-lookup"><span data-stu-id="77de7-104">Every assembly, whether static or dynamic, contains a collection of data that describes how the elements in the assembly relate to each other.</span></span> <span data-ttu-id="77de7-105">Эти метаданные содержатся в манифесте сборки.</span><span class="sxs-lookup"><span data-stu-id="77de7-105">The assembly manifest contains this assembly metadata.</span></span> <span data-ttu-id="77de7-106">Манифест сборки содержит все метаданные, необходимые для задания требований сборки к версиям и удостоверения безопасности, а также все метаданные, необходимые для определения области действия сборки и разрешения ссылок на ресурсы и классы.</span><span class="sxs-lookup"><span data-stu-id="77de7-106">An assembly manifest contains all the metadata needed to specify the assembly's version requirements and security identity, and all metadata needed to define the scope of the assembly and resolve references to resources and classes.</span></span> <span data-ttu-id="77de7-107">Манифест сборки может храниться в PE-файле ( *EXE* или *DLL* ) с кодом MSIL или же в отдельном PE-файле, содержащем только данные манифеста.</span><span class="sxs-lookup"><span data-stu-id="77de7-107">The assembly manifest can be stored in either a PE file (an *.exe* or *.dll* ) with Microsoft intermediate language (MSIL) code or in a standalone PE file that contains only assembly manifest information.</span></span>

- <span data-ttu-id="77de7-113">перечисление файлов, составляющих сборку;</span><span class="sxs-lookup"><span data-stu-id="77de7-113">Enumerates the files that make up the assembly.</span></span>  
  
- <span data-ttu-id="77de7-114">сопоставление ссылок на типы и ресурсы сборки с файлами, содержащими объявления и реализации этих типов и ресурсов;</span><span class="sxs-lookup"><span data-stu-id="77de7-114">Governs how references to the assembly's types and resources map to the files that contain their declarations and implementations.</span></span>
