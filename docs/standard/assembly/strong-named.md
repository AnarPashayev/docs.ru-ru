# <a name="strong-named-assemblies"></a><span data-ttu-id="3078d-103">Сборки со строгими именами</span><span class="sxs-lookup"><span data-stu-id="3078d-103">Strong-named assemblies</span></span>

<span data-ttu-id="3078d-104">При задании строгого имени для сборки для нее создается уникальный идентификатор, что позволяет избежать конфликтов сборок.</span><span class="sxs-lookup"><span data-stu-id="3078d-104">Strong-naming an assembly creates a unique identity for the assembly, and can prevent assembly conflicts.</span></span>

## <a name="what-makes-a-strong-named-assembly"></a><span data-ttu-id="3078d-105">Что такое сборка со строгим именем?</span><span class="sxs-lookup"><span data-stu-id="3078d-105">What makes a strong-named assembly?</span></span>

<span data-ttu-id="3078d-106">Сборка со строгим именем создается с помощью закрытого ключа, который соответствует открытому ключу, распространяемому со сборкой, и самой сборке.</span><span class="sxs-lookup"><span data-stu-id="3078d-106">A strong named assembly is generated by using the private key that corresponds to the public key distributed with the assembly, and the assembly itself.</span></span> <span data-ttu-id="3078d-107">Сборка включает манифест сборки, который содержит имена и хэши всех файлов, из которых состоит сборка.</span><span class="sxs-lookup"><span data-stu-id="3078d-107">The assembly includes the assembly manifest, which contains the names and hashes of all the files that make up the assembly.</span></span> <span data-ttu-id="3078d-108">Сборки с одинаковым строгим именем должны быть идентичны.</span><span class="sxs-lookup"><span data-stu-id="3078d-108">Assemblies that have the same strong name should be identical.</span></span>

<span data-ttu-id="3078d-109">Задавать строгие имена для сборок можно с помощью Visual Studio или программы командной строки.</span><span class="sxs-lookup"><span data-stu-id="3078d-109">You can strong-name assemblies by using Visual Studio or a command-line tool.</span></span> <span data-ttu-id="3078d-110">Дополнительные сведения см. в разделе [Практическое руководство. Подписывание сборки строгим именем](sign-strong-name.md) и [Sn.exe (средство создания строгих имен)](../../framework/tools/sn-exe-strong-name-tool.md).</span><span class="sxs-lookup"><span data-stu-id="3078d-110">For more information, see [How to: Sign an assembly with a strong name](sign-strong-name.md) or [Sn.exe (Strong Name tool)](../../framework/tools/sn-exe-strong-name-tool.md).</span></span>

<span data-ttu-id="3078d-111">При создании сборки со строгим именем она содержит простое текстовое имя сборки, номер версии, дополнительные сведения о языке и языковых параметрах, цифровую подпись и открытый ключ, который соответствует закрытому ключу, используемому для подписи.</span><span class="sxs-lookup"><span data-stu-id="3078d-111">When a strong-named assembly is created, it contains the simple text name of the assembly, the version number, optional culture information, a digital signature, and the public key that corresponds to the private key used for signing.</span></span>

> [!WARNING]
> <span data-ttu-id="3078d-112">Строгие имена не являются средством обеспечения безопасности.</span><span class="sxs-lookup"><span data-stu-id="3078d-112">Do not rely on strong names for security.</span></span> <span data-ttu-id="3078d-113">Они служат только для однозначной идентификации.</span><span class="sxs-lookup"><span data-stu-id="3078d-113">They provide a unique identity only.</span></span>

## <a name="why-strong-name-your-assemblies"></a><span data-ttu-id="3078d-114">Зачем присваивать сборкам строгие имена?</span><span class="sxs-lookup"><span data-stu-id="3078d-114">Why strong-name your assemblies?</span></span>

<span data-ttu-id="3078d-115">При использовании ссылки на сборку со строгим именем можно пользоваться определенными преимуществами, например отслеживанием версий и защитой имен.</span><span class="sxs-lookup"><span data-stu-id="3078d-115">When you reference a strong-named assembly, you can expect certain benefits, such as versioning and naming protection.</span></span> <span data-ttu-id="3078d-116">В .NET Framework сборки со строгими именами могут устанавливаться в глобальный кэш сборок, который необходим для реализации некоторых сценариев.</span><span class="sxs-lookup"><span data-stu-id="3078d-116">In the .NET Framework, strong-named assemblies can be installed in the global assembly cache, which is required to enable some scenarios.</span></span>

<span data-ttu-id="3078d-117">Сборки со строгими именами полезны в следующих сценариях:</span><span class="sxs-lookup"><span data-stu-id="3078d-117">Strong-named assemblies are useful in the following scenarios:</span></span>

- <span data-ttu-id="3078d-118">Необходимо обеспечить возможность ссылок на ваши сборки сборкам со строгими именами или предоставить доступ `friend` к вашим сборкам из других сборок со строгими именами.</span><span class="sxs-lookup"><span data-stu-id="3078d-118">You want to enable your assemblies to be referenced by strong-named assemblies, or you want to give `friend` access to your assemblies from other strong-named assemblies.</span></span>

- <span data-ttu-id="3078d-119">Приложению требуется доступ к различным версиям одной и той же сборки.</span><span class="sxs-lookup"><span data-stu-id="3078d-119">An app needs access to different versions of the same assembly.</span></span> <span data-ttu-id="3078d-120">Это означает, что требуются разные версии сборки для параллельной загрузки в одном домене приложений без возникновения конфликтов.</span><span class="sxs-lookup"><span data-stu-id="3078d-120">This means  you need different versions of an assembly to load side by side in the same app domain without conflict.</span></span> <span data-ttu-id="3078d-121">(например, если в сборках с одним простым именем существуют разные расширения API, назначение строгих имен обеспечивает однозначную идентификацию всех версий сборки);</span><span class="sxs-lookup"><span data-stu-id="3078d-121">For example, if different extensions of an API exist in assemblies that have the same simple name, strong-naming provides a unique identity for each version of the assembly.</span></span>

- <span data-ttu-id="3078d-122">Использование вашей сборки не должно отрицательно сказываться на производительности приложений, поэтому сборка должна быть нейтральной к доменам.</span><span class="sxs-lookup"><span data-stu-id="3078d-122">You do not want to negatively affect performance of apps using your assembly, so you want the assembly to be domain neutral.</span></span> <span data-ttu-id="3078d-123">Для этого требуется строгое именование, так как нейтральная к доменам сборка должна устанавливаться в глобальный кэш сборок.</span><span class="sxs-lookup"><span data-stu-id="3078d-123">This requires strong-naming because a domain-neutral assembly must be installed in the global assembly cache.</span></span>

- <span data-ttu-id="3078d-124">Требуется применять централизованное обслуживание приложений с помощью политики издателя (это означает, что сборка должна устанавливаться в глобальном кэше сборок).</span><span class="sxs-lookup"><span data-stu-id="3078d-124">You want to centralize servicing for your app by applying publisher policy, which means the assembly must be installed in the global assembly cache.</span></span>

<span data-ttu-id="3078d-125">Если вы являетесь разработчиком открытого кода и хотите воспользоваться преимуществами идентификации, которые обеспечивают сборки со строгими именами, рекомендуется возвращать закрытый ключ, связанный со сборкой, в систему управления версиями.</span><span class="sxs-lookup"><span data-stu-id="3078d-125">If you are an open-source developer and you want the identity benefits of a strong-named assembly, consider checking in the private key associated with an assembly to your source control system.</span></span>
