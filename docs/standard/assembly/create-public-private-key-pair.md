<span data-ttu-id="00f75-103">Для подписи сборки строгим именем необходимо иметь пару, состоящую из открытого и закрытого ключа.</span><span class="sxs-lookup"><span data-stu-id="00f75-103">To sign an assembly with a strong name, you must have a public/private key pair.</span></span> <span data-ttu-id="00f75-104">Эта пара криптографических ключей используется во время компиляции для создания сборки со строгим именем.</span><span class="sxs-lookup"><span data-stu-id="00f75-104">This public and private cryptographic key pair is used during compilation to create a strong-named assembly.</span></span> <span data-ttu-id="00f75-105">Пару ключей можно создать с помощью [средства для работы со строгими именами (Sn.exe)](../../framework/tools/sn-exe-strong-name-tool.md).</span><span class="sxs-lookup"><span data-stu-id="00f75-105">You can create a key pair using the [Strong Name tool (Sn.exe)](../../framework/tools/sn-exe-strong-name-tool.md).</span></span> <span data-ttu-id="00f75-106">Файлы пары ключей обычно имеют расширение *SNK*.</span><span class="sxs-lookup"><span data-stu-id="00f75-106">Key pair files usually have an *.snk* extension.</span></span>

## <a name="create-a-key-pair"></a><span data-ttu-id="00f75-110">Создание пары ключей</span><span class="sxs-lookup"><span data-stu-id="00f75-110">Create a key pair</span></span>

<span data-ttu-id="00f75-111">Чтобы создать пару ключей, в командной строке введите следующую команду:</span><span class="sxs-lookup"><span data-stu-id="00f75-111">To create a key pair, at a command prompt, type the following command:</span></span>

<span data-ttu-id="00f75-112">**sn –k** \<*имя файла*></span><span class="sxs-lookup"><span data-stu-id="00f75-112">**sn –k** \<*file name*></span></span>

<span data-ttu-id="00f75-113">В этой команде *имя файла* — это имя выходного файла, содержащего пару ключей.</span><span class="sxs-lookup"><span data-stu-id="00f75-113">In this command, *file name* is the name of the output file containing the key pair.</span></span>

<span data-ttu-id="00f75-114">В следующем примере создается пара ключей с именем *sgKey.snk*.</span><span class="sxs-lookup"><span data-stu-id="00f75-114">The following example creates a key pair called *sgKey.snk*.</span></span>

```cmd
sn -k sgKey.snk
```

<span data-ttu-id="00f75-115">Если требуется отложить подписание сборки, а также управлять парой ключей целиком (что может потребоваться разве что для тестирования), можно использовать следующие команды для создания пары ключей и извлечения открытого ключа из нее в отдельный файл.</span><span class="sxs-lookup"><span data-stu-id="00f75-115">If you intend to delay sign an assembly and you control the whole key pair (which is unlikely outside test scenarios), you can use the following commands to generate a key pair and then extract the public key from it into a separate file.</span></span> <span data-ttu-id="00f75-116">Во-первых, создайте пару ключей:</span><span class="sxs-lookup"><span data-stu-id="00f75-116">First, create the key pair:</span></span>

```cmd
sn -k keypair.snk
```

<span data-ttu-id="00f75-117">Затем извлеките открытый ключ из пары ключей и скопируйте его в отдельный файл:</span><span class="sxs-lookup"><span data-stu-id="00f75-117">Next, extract the public key from the key pair and copy it to a separate file:</span></span>

```cmd
sn -p keypair.snk public.snk
```

<span data-ttu-id="00f75-118">После создания пары ключей необходимо поместить файл в расположение, в котором его смогут найти инструменты создания подписи строгого имени.</span><span class="sxs-lookup"><span data-stu-id="00f75-118">Once you create the key pair, you must put the file where the strong name signing tools can find it.</span></span>

<span data-ttu-id="00f75-119">При подписании сборки строгим именем [компоновщик сборок (Al.exe)](../../framework/tools/al-exe-assembly-linker.md) выполняет поиск файла ключа относительно текущей и выходной папки.</span><span class="sxs-lookup"><span data-stu-id="00f75-119">When signing an assembly with a strong name, the [Assembly Linker (Al.exe)](../../framework/tools/al-exe-assembly-linker.md) looks for the key file relative to the current directory and to the output directory.</span></span> <span data-ttu-id="00f75-120">При использовании компиляторов, работающих в режиме командной строки, достаточно просто скопировать ключ в текущий каталог, содержащий модули кода.</span><span class="sxs-lookup"><span data-stu-id="00f75-120">When using command-line compilers, you can simply copy the key to the current directory containing your code modules.</span></span>

<span data-ttu-id="00f75-121">При использовании более ранней версии Visual Studio, в которой нет вкладки **Подписание** в свойствах проекта, рекомендуемым расположением файла ключа является каталог проекта с атрибутом файла, заданным следующим образом:</span><span class="sxs-lookup"><span data-stu-id="00f75-121">If you are using an earlier version of Visual Studio that does not have a **Signing** tab in the project properties, the recommended key file location is the project directory with the file attribute specified as follows:</span></span>

```cpp
[assembly:AssemblyKeyFileAttribute("keyfile.snk")];
```

```csharp
[assembly:AssemblyKeyFileAttribute("keyfile.snk")]
```

```vb
<Assembly:AssemblyKeyFileAttribute("keyfile.snk")>
```
