# <a name="constructor-design"></a>Разработка конструктора

Есть два вида конструкторов: тип конструкторов и экземпляр конструкторов.

Конструкторы типа статичны и управляются CLR до использования типа. Конструкторы экземпляров работают при создании экземпляра типа.

Конструкторы типа не могут принимать какие-либо параметры. Инстанции конструкторы могут. Конструкторы экземпляров, которые не принимают никаких параметров, часто называют беспараметрыными конструкторами.

Конструкторы являются наиболее естественным способом создания экземпляров типа. Большинство разработчиков будут искать и пытаться использовать конструктор, прежде чем они рассматривают альтернативные способы создания экземпляров (например, заводские методы).

✔️ CONSIDER, предоставляя простые, идеально по умолчанию, конструкторы.

Простой конструктор имеет очень небольшое количество параметров, и все параметры примитивы или перечисления. Такие простые конструкторы повышают удобство использования каркаса.

✔️ CONSIDER с помощью статического фабричного метода вместо конструктора, если семантика желаемой операции не отображает непосредственно конструкцию нового экземпляра, или если следовать руководящим принципам проектирования конструктора кажется неестественным.

✔️ DO используют параметры конструктора в качестве ярлыков для настройки основных свойств.

Не должно быть никакой разницы в семантике между использованием пустого конструктора, за которым следуют некоторые наборы свойств, и использованием конструктора с несколькими аргументами.

✔️ do использовать одно и то же имя для параметров конструктора и свойство, если параметры конструктора используются для простого набора свойства.

Единственное различие между такими параметрами и свойствами должно быть оболочкой.

✔️ сделать минимальную работу в конструкторе.

Конструкторы не должны делать много работы, кроме захвата параметров конструктора. Стоимость любой другой обработки должна быть отложена до тех пор, пока требуется.

✔️, если это необходимо, выбрасывает исключения из конструкторов экземпляров.

✔️ совершенно прямо объявить общественный безпараметр конструкции в классах, если такой конструктор не требуется.

Если вы явно не объявите о каких-либо конструкторах по типу, многие языки (например, C) автоматически добавят непосильным параметрам. (Абстрактные классы получают защищенный конструктор.)

Добавление параметризированного конструктора в класс не позволяет компилятору добавлять безпараметр. Это часто приводит к случайным изменениям нарушения.

❌AVOID явно определяет беспараметрыные конструкторы на структурах.

Это ускоряет создание массива, потому что если конструктор без параметров не определен, он не должен быть запущен на каждом слоте в массиве. Обратите внимание, что многие компиляторы, в том числе C, не позволяют структураторам иметь беспараметрыные конструкторы по этой причине.

❌AVOID вызывает виртуальных членов на объект внутри своего конструктора.

Вызов виртуального участника вызовет наиболее производный переопределение, даже если конструктор наиболее производного типа еще не был полностью запущен.

## <a name="type-constructor-guidelines"></a>Руководящие принципы ввода-конструктора

✔️ сделать статические конструкторы частными.

Для инициализации типа используется статический конструктор, также называемый конструктором класса. CLR вызывает статический конструктор до создания первого экземпляра типа или вызова каких-либо статических членов этого типа. Пользователь не может контролировать, когда называется статический конструктор. Если статический конструктор не является закрытым, его можно назвать кодом, кроме CLR. В зависимости от операций, выполняемых в конструкторе, это может привести к неожиданному поведению. Компилятор C' заставляет статических конструкторов быть частными.

❌НЕ выбрасывайте исключения из статических конструкторов.

Если исключение выбрасывается из конструктора типа, тип не используется в текущем домене приложения.

✔️ CONSIDER инициализации статические поля встроенной, а не явно с помощью статических конструкторов, потому что время выполнения способно оптимизировать производительность типов, которые не имеют четко определенного статического конструктора.
