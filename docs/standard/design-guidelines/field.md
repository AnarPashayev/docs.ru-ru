# <a name="field-design"></a><span data-ttu-id="969ac-102">Разработка полей</span><span class="sxs-lookup"><span data-stu-id="969ac-102">Field Design</span></span>
<span data-ttu-id="969ac-103">Принцип инкапсуляции является одним из наиболее важные понятия об удостоверении в объектно ориентированного проектирования.</span><span class="sxs-lookup"><span data-stu-id="969ac-103">The principle of encapsulation is one of the most important notions in object-oriented design.</span></span> <span data-ttu-id="969ac-104">Этот принцип утверждает, что данные, хранящиеся в объекте должен быть доступен только для этого объекта.</span><span class="sxs-lookup"><span data-stu-id="969ac-104">This principle states that data stored inside an object should be accessible only to that object.</span></span>  
  
 <span data-ttu-id="969ac-105">Полезный способ интерпретации принцип — скажем, что тип должны разрабатываться таким образом, чтобы изменения поля этого типа (изменения имени или типу) могут выполняться без нарушения кода, отличное от для членов типа.</span><span class="sxs-lookup"><span data-stu-id="969ac-105">A useful way to interpret the principle is to say that a type should be designed so that changes to fields of that type (name or type changes) can be made without breaking code other than for members of the type.</span></span> <span data-ttu-id="969ac-106">Эту интерпретацию немедленно подразумевает, что все поля должно быть закрытым.</span><span class="sxs-lookup"><span data-stu-id="969ac-106">This interpretation immediately implies that all fields must be private.</span></span>  
  
 <span data-ttu-id="969ac-107">Мы исключаем константы и статические поля только для чтения из этих ограничений strict, поскольку такие поля практически по определению, никогда не требуется изменить.</span><span class="sxs-lookup"><span data-stu-id="969ac-107">We exclude constant and static read-only fields from this strict restriction, because such fields, almost by definition, are never required to change.</span></span>  
  
 <span data-ttu-id="969ac-108">**X DO NOT** предоставляют открытые или защищенные поля экземпляра.</span><span class="sxs-lookup"><span data-stu-id="969ac-108">**X DO NOT** provide instance fields that are public or protected.</span></span>  
  
 <span data-ttu-id="969ac-109">Необходимо указать свойства для доступа к полям, вместо того, чтобы их открытый или защищенный.</span><span class="sxs-lookup"><span data-stu-id="969ac-109">You should provide properties for accessing fields instead of making them public or protected.</span></span>  
  
 <span data-ttu-id="969ac-110">**✓ DO** для констант, которые никогда не изменяются, используйте поля.</span><span class="sxs-lookup"><span data-stu-id="969ac-110">**✓ DO** use constant fields for constants that will never change.</span></span>  
  
 <span data-ttu-id="969ac-111">Компилятор горит значения const полей непосредственно в вызывающем коде.</span><span class="sxs-lookup"><span data-stu-id="969ac-111">The compiler burns the values of const fields directly into calling code.</span></span> <span data-ttu-id="969ac-112">Таким образом значения const не может быть изменено без риска нарушения совместимости.</span><span class="sxs-lookup"><span data-stu-id="969ac-112">Therefore, const values can never be changed without the risk of breaking compatibility.</span></span>  
  
 <span data-ttu-id="969ac-113">**✓ DO** использовать открытые статические `readonly` поля для предварительно определенных экземпляров объекта.</span><span class="sxs-lookup"><span data-stu-id="969ac-113">**✓ DO** use public static `readonly` fields for predefined object instances.</span></span>  
  
 <span data-ttu-id="969ac-114">Если существуют предварительно определенные экземпляры типа, объявите их как открытые только для чтения статического поля самого типа.</span><span class="sxs-lookup"><span data-stu-id="969ac-114">If there are predefined instances of the type, declare them as public read-only static fields of the type itself.</span></span>  
  
 <span data-ttu-id="969ac-115">**X DO NOT** назначить экземпляры изменяемых типов для `readonly` поля.</span><span class="sxs-lookup"><span data-stu-id="969ac-115">**X DO NOT** assign instances of mutable types to `readonly` fields.</span></span>  
  
 <span data-ttu-id="969ac-116">Изменяемый тип является типом с экземплярами, которые могут быть изменены после их создания.</span><span class="sxs-lookup"><span data-stu-id="969ac-116">A mutable type is a type with instances that can be modified after they are instantiated.</span></span> <span data-ttu-id="969ac-117">Например, изменяемые типам относятся массивы, большинства коллекций и потоков, но <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Uri?displayProperty=nameWithType>, и <xref:System.String?displayProperty=nameWithType> все неизменяемы.</span><span class="sxs-lookup"><span data-stu-id="969ac-117">For example, arrays, most collections, and streams are mutable types, but <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Uri?displayProperty=nameWithType>, and <xref:System.String?displayProperty=nameWithType> are all immutable.</span></span> <span data-ttu-id="969ac-118">Модификатор "только для чтения" в поле типа ссылки не позволяет экземпляру, хранящиеся в поле замену, но не запрещает данных экземпляра поля изменяется путем вызова участников, изменение экземпляра.</span><span class="sxs-lookup"><span data-stu-id="969ac-118">The read-only modifier on a reference type field prevents the instance stored in the field from being replaced, but it does not prevent the field’s instance data from being modified by calling members changing the instance.</span></span>
