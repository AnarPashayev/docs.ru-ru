# <a name="names-of-classes-structs-and-interfaces"></a>Имена классов, структур и интерфейсов
Руководящие принципы именования, которые следуют, применяются к общему наименованию типов.

 ✔️ классы имен и структур с существительными или существительными фразами, используя PascalCasing.

 Это отличает имена типов от методов, которые называются глагольных фраз.

 ✔️ интерфейсы имен DO с прилагательными фразами, или иногда с существительными или существительными фразами.

 Существить и существить фразы должны использоваться редко, и они могут указывать на то, что тип должен быть абстрактным классом, а не интерфейсом.

 ❌НЕ давайте названия классов приставку (например, "C").

 ✔️ CONSIDER, заканчивая название выведенных классов с названием базового класса.

 Это очень читаемо и объясняет отношения ясно. Некоторые примеры этого в `ArgumentOutOfRangeException`коде: , `Exception`который `SerializableAttribute`является своего рода `Attribute`, и , который является своего рода . Однако при применении этого руководства важно использовать разумные суждения; например, `Button` класс является своего `Control` рода `Control` событием, хотя и не отображается в его названии.

 ✔️ do приставки интерфейса имена с буквой I, чтобы указать, что тип интерфейса.

 Например, `IComponent` (описательное `ICustomAttributeProvider` существительное), (существительное) и `IPersistable` (прилагательное) являются подходящими именами интерфейса. Как и в других именах типов, избегайте сокращений.

 ✔️ убедиться, что имена отличаются только префиксом "I" на названии интерфейса при определении пары класса и интерфейса, где класс является стандартной реализацией интерфейса.

## <a name="names-of-generic-type-parameters"></a>Названия параметров общего типа
 Общие предложения были добавлены в .NET Framework 2.0. Функция представила новый вид идентификатора, называемый *параметром типа.*

 ✔️ значения общих параметров типа DO с описательными именами, если только однобуквенное имя не является полностью понятным и описательное имя не добавит ценности.

 ✔️ CONSIDER `T` в качестве типа параметра для типов с одним параметром типа одной буквы.

```csharp
public int IComparer<T> { ... }
public delegate bool Predicate<T>(T item);
public struct Nullable<T> where T:struct { ... }
```

 ✔️ DO прикрепительное описательное тип параметр имена с `T`.

```csharp
public interface ISessionChannel<TSession> where TSession : ISession {
    TSession Session { get; }
}
```

 ✔️ CONSIDER, указывающие ограничения, наложенные на параметр типа в названии параметра.

 Например, параметр, `ISession` ограниченный `TSession`может быть вызван.

## <a name="naming-enumerations"></a>Именование ичислений
 Названия типов перечислений (также называемых перечислениями) в целом должны следовать стандартным правилам именования типов (PascalCasing и т.д.). Однако существуют дополнительные руководящие принципы, которые применяются конкретно к перечислениям.

 ✔️ do использовать имя единственного типа для перечисления, если его значения не являются битовыми полями.

 ✔️ do использовать имя типа множественного числа для перечисления с битовыми полями в качестве значений, также называемых флагами enum.

 ❌НЕ используйте суффикс "Enum" в именах типов enum.

 ❌НЕ используйте суффиксы "Flag" или "Flags" в именах типов enum.

 ❌НЕ используйте префикс на именах значений перечисления (например, "объявление" для enums ADO, "rtf" для богатых текстовых перечислений и т.д.).
