 <span data-ttu-id="3464d-164">Например элементы с помощью параметров-массивов байтов практически никогда не будет называться путем передачи отдельных байтов.</span><span class="sxs-lookup"><span data-stu-id="3464d-164">For example, members with byte array parameters would almost never be called by passing individual bytes.</span></span> <span data-ttu-id="3464d-165">По этой причине параметров массива байтов в .NET Framework не используйте ключевое слово params.</span><span class="sxs-lookup"><span data-stu-id="3464d-165">For this reason, byte array parameters in the .NET Framework do not use the params keyword.</span></span>

 <span data-ttu-id="3464d-167">Из-за того, что многие компилятор преобразует аргументы в элемент во временный массив во время вызова массив может быть временным объектом, и таким образом будут потеряны все изменения в массив.</span><span class="sxs-lookup"><span data-stu-id="3464d-167">Because of the fact that many compilers turn the arguments to the member into a temporary array at the call site, the array might be a temporary object, and therefore any modifications to the array will be lost.</span></span>  
  
 <span data-ttu-id="3464d-168">**✓ CONSIDER** с помощью ключевого слова params в простой перегрузке, даже если более сложная перегрузка не может использовать его.</span><span class="sxs-lookup"><span data-stu-id="3464d-168">**✓ CONSIDER** using the params keyword in a simple overload, even if a more complex overload could not use it.</span></span>  
  
 <span data-ttu-id="3464d-169">Спросите себя, если бы значение для пользователей с массивом параметров в одной перегрузке, даже если он не был в все перегрузки.</span><span class="sxs-lookup"><span data-stu-id="3464d-169">Ask yourself if users would value having the params array in one overload even if it wasn’t in all overloads.</span></span>  
  
 <span data-ttu-id="3464d-170">**✓ DO** попытайтесь Упорядочить параметры, чтобы сделать возможным использование ключевого слова params.</span><span class="sxs-lookup"><span data-stu-id="3464d-170">**✓ DO** try to order parameters to make it possible to use the params keyword.</span></span>  
  
 <span data-ttu-id="3464d-171">**✓ CONSIDER** предусматривать особые перегрузки и ветви кода для вызовов с небольшим числом аргументов в API-интерфейсы важна высокая производительность.</span><span class="sxs-lookup"><span data-stu-id="3464d-171">**✓ CONSIDER** providing special overloads and code paths for calls with a small number of arguments in extremely performance-sensitive APIs.</span></span>  
  
 <span data-ttu-id="3464d-172">Это позволяет избежать создания массива объектов при вызове API с небольшим числом аргументов.</span><span class="sxs-lookup"><span data-stu-id="3464d-172">This makes it possible to avoid creating array objects when the API is called with a small number of arguments.</span></span> <span data-ttu-id="3464d-173">Имена параметров образуют единственном параметра массива и добавление числовой суффикс.</span><span class="sxs-lookup"><span data-stu-id="3464d-173">Form the names of the parameters by taking a singular form of the array parameter and adding a numeric suffix.</span></span>  
  
 <span data-ttu-id="3464d-174">Это следует только сделать Если предполагается особым путь весь код, не просто создать массив и вызвать метод более общими.</span><span class="sxs-lookup"><span data-stu-id="3464d-174">You should only do this if you are going to special-case the entire code path, not just create an array and call the more general method.</span></span>  
  
 <span data-ttu-id="3464d-175">**✓ DO** Имейте в виду, null может быть передан в качестве аргумента массива params.</span><span class="sxs-lookup"><span data-stu-id="3464d-175">**✓ DO** be aware that null could be passed as a params array argument.</span></span>  
  
 <span data-ttu-id="3464d-176">Обязательно проверяйте, что массив не равен null, если перед обработкой.</span><span class="sxs-lookup"><span data-stu-id="3464d-176">You should validate that the array is not null before processing.</span></span>  
  
 <span data-ttu-id="3464d-177">**X DO NOT** использовать `varargs` методы, также известный как кнопку с многоточием.</span><span class="sxs-lookup"><span data-stu-id="3464d-177">**X DO NOT** use the `varargs` methods, otherwise known as the ellipsis.</span></span>  
  
 <span data-ttu-id="3464d-178">Некоторых языков среды CLR, например C++, поддерживают альтернативные соглашение о передаче вызывается списки параметров переменных `varargs` методы.</span><span class="sxs-lookup"><span data-stu-id="3464d-178">Some CLR languages, such as C++, support an alternative convention for passing variable parameter lists called `varargs` methods.</span></span> <span data-ttu-id="3464d-179">Соглашение не следует на платформах, так как он не является CLS-совместимым.</span><span class="sxs-lookup"><span data-stu-id="3464d-179">The convention should not be used in frameworks, because it is not CLS compliant.</span></span>  
  
### <a name="pointer-parameters"></a><span data-ttu-id="3464d-180">Параметры-указатели</span><span class="sxs-lookup"><span data-stu-id="3464d-180">Pointer Parameters</span></span>  
 <span data-ttu-id="3464d-181">В общем случае указатели не должны отображаться в общую контактную зону структуры хорошо спроектированной управляемого кода.</span><span class="sxs-lookup"><span data-stu-id="3464d-181">In general, pointers should not appear in the public surface area of a well-designed managed code framework.</span></span> <span data-ttu-id="3464d-182">В большинстве случаев, должна быть включена указатели.</span><span class="sxs-lookup"><span data-stu-id="3464d-182">Most of the time, pointers should be encapsulated.</span></span> <span data-ttu-id="3464d-183">Однако в некоторых случаях указатели являются обязательными в целях взаимодействия, и с использованием указателей в таких случаях подходит.</span><span class="sxs-lookup"><span data-stu-id="3464d-183">However, in some cases pointers are required for interoperability reasons, and using pointers in such cases is appropriate.</span></span>  
  
 <span data-ttu-id="3464d-184">**✓ DO** предоставляют альтернативный для любого элемента, который принимает указатель в качестве аргумента, так как указатели не являются CLS-совместимыми.</span><span class="sxs-lookup"><span data-stu-id="3464d-184">**✓ DO** provide an alternative for any member that takes a pointer argument, because pointers are not CLS-compliant.</span></span>  
  
 <span data-ttu-id="3464d-185">**X AVOID** затратного контроля аргументов указатель аргументов.</span><span class="sxs-lookup"><span data-stu-id="3464d-185">**X AVOID** doing expensive argument checking of pointer arguments.</span></span>  
  
 <span data-ttu-id="3464d-186">**✓ DO** соглашениям общих указателей при разработке члены с указателями.</span><span class="sxs-lookup"><span data-stu-id="3464d-186">**✓ DO** follow common pointer-related conventions when designing members with pointers.</span></span>  
  
 <span data-ttu-id="3464d-187">Например нет необходимости передавать начальный индекс, так как простой указателями может использоваться для достижения такого же результата.</span><span class="sxs-lookup"><span data-stu-id="3464d-187">For example, there is no need to pass the start index, because simple pointer arithmetic can be used to accomplish the same result.</span></span>
