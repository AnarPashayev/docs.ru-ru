# <a name="exception-throwing"></a><span data-ttu-id="c3408-102">Создание исключений</span><span class="sxs-lookup"><span data-stu-id="c3408-102">Exception Throwing</span></span>
<span data-ttu-id="c3408-103">Исключения рекомендации, приведенные в этом разделе требуется хорошее определение значение сбой при выполнении.</span><span class="sxs-lookup"><span data-stu-id="c3408-103">Exception-throwing guidelines described in this section require a good definition of the meaning of execution failure.</span></span> <span data-ttu-id="c3408-104">Сбой при выполнении происходит всякий раз, когда член не может это было целью (что член названия).</span><span class="sxs-lookup"><span data-stu-id="c3408-104">Execution failure occurs whenever a member cannot do what it was designed to do (what the member name implies).</span></span> <span data-ttu-id="c3408-105">Например если `OpenFile` метод не может возвращать дескриптор открытого файла вызывающей стороне, он будет считаться ошибкой выполнения.</span><span class="sxs-lookup"><span data-stu-id="c3408-105">For example, if the `OpenFile` method cannot return an opened file handle to the caller, it would be considered an execution failure.</span></span>  
  
 <span data-ttu-id="c3408-106">Большинство разработчиков ознакомления с использования исключений для использования ошибок, например деление на 0 или пустых ссылок.</span><span class="sxs-lookup"><span data-stu-id="c3408-106">Most developers have become comfortable with using exceptions for usage errors such as division by zero or null references.</span></span> <span data-ttu-id="c3408-107">В платформе исключения используются для всех ошибок, включая ошибки выполнения.</span><span class="sxs-lookup"><span data-stu-id="c3408-107">In the Framework, exceptions are used for all error conditions, including execution errors.</span></span>  
  
 <span data-ttu-id="c3408-108">**X DO NOT** возвращенный код ошибки.</span><span class="sxs-lookup"><span data-stu-id="c3408-108">**X DO NOT** return error codes.</span></span>  
  
 <span data-ttu-id="c3408-109">Исключениями являются основным средством отчетов об ошибках в платформах.</span><span class="sxs-lookup"><span data-stu-id="c3408-109">Exceptions are the primary means of reporting errors in frameworks.</span></span>  
  
 <span data-ttu-id="c3408-110">**✓ DO** отчета сбоев при выполнении путем вызова исключений.</span><span class="sxs-lookup"><span data-stu-id="c3408-110">**✓ DO** report execution failures by throwing exceptions.</span></span>  
  
 <span data-ttu-id="c3408-111">**✓ CONSIDER** завершение процесса путем вызова `System.Environment.FailFast` (функции .NET Framework версии 2.0) вместо создания исключения при обнаружении в коде ситуации, когда он небезопасных для дальнейшего выполнения.</span><span class="sxs-lookup"><span data-stu-id="c3408-111">**✓ CONSIDER** terminating the process by calling `System.Environment.FailFast` (.NET Framework 2.0 feature) instead of throwing an exception if your code encounters a situation where it is unsafe for further execution.</span></span>  
  
 <span data-ttu-id="c3408-112">**X DO NOT** использовать исключения для обычного потока управления, если это возможно.</span><span class="sxs-lookup"><span data-stu-id="c3408-112">**X DO NOT** use exceptions for the normal flow of control, if possible.</span></span>  
  
 <span data-ttu-id="c3408-113">За исключением системных сбоев и операций с помощью потенциальных состояний гонки конструкторах framework следует разрабатывать API-интерфейсов, поэтому пользователи могут писать код, который не создает исключение.</span><span class="sxs-lookup"><span data-stu-id="c3408-113">Except for system failures and operations with potential race conditions, framework designers should design APIs so users can write code that does not throw exceptions.</span></span> <span data-ttu-id="c3408-114">Например можно предоставить способ проверки предварительных условий до вызова члена, поэтому пользователи могут писать код, который не вызывает исключения.</span><span class="sxs-lookup"><span data-stu-id="c3408-114">For example, you can provide a way to check preconditions before calling a member so users can write code that does not throw exceptions.</span></span>  
  
 <span data-ttu-id="c3408-115">Элемент, используемый для проверки предварительных условий для другого элемента часто называют тест-инженер, а член, который фактически выполняет работу, называется doer.</span><span class="sxs-lookup"><span data-stu-id="c3408-115">The member used to check preconditions of another member is often referred to as a tester, and the member that actually does the work is called a doer.</span></span>  
  
 <span data-ttu-id="c3408-116">Бывают случаи, когда шаблон Tester-Doer может привести к снижению неприемлемо производительности.</span><span class="sxs-lookup"><span data-stu-id="c3408-116">There are cases when the Tester-Doer Pattern can have an unacceptable performance overhead.</span></span> <span data-ttu-id="c3408-117">В таких случаях следует рассматривать так называемые шаблон Try-Parse (см. в разделе [исключения и производительность](../../../docs/standard/design-guidelines/exceptions-and-performance.md) Дополнительные сведения).</span><span class="sxs-lookup"><span data-stu-id="c3408-117">In such cases, the so-called Try-Parse Pattern should be considered (see [Exceptions and Performance](../../../docs/standard/design-guidelines/exceptions-and-performance.md) for more information).</span></span>  
  
 <span data-ttu-id="c3408-118">**✓ CONSIDER** создание исключений влияет на производительность.</span><span class="sxs-lookup"><span data-stu-id="c3408-118">**✓ CONSIDER** the performance implications of throwing exceptions.</span></span> <span data-ttu-id="c3408-119">Тарифы throw выше 100 в секунду, скорее всего существенно повлиять на производительность большинства приложений.</span><span class="sxs-lookup"><span data-stu-id="c3408-119">Throw rates above 100 per second are likely to noticeably impact the performance of most applications.</span></span>  
  
 <span data-ttu-id="c3408-120">**✓ DO** документ, все исключения, создаваемые публично вызываемыми членами из-за нарушения элемента контракта (а не системной ошибки) и обрабатывает их как часть вашего контракта.</span><span class="sxs-lookup"><span data-stu-id="c3408-120">**✓ DO** document all exceptions thrown by publicly callable members because of a violation of the member contract (rather than a system failure) and treat them as part of your contract.</span></span>  
  
 <span data-ttu-id="c3408-121">Исключения, которые являются частью контракта не должны изменяться от одной версии к следующему (т. е. не изменяйте тип исключения и не нужно добавлять новые исключения).</span><span class="sxs-lookup"><span data-stu-id="c3408-121">Exceptions that are a part of the contract should not change from one version to the next (i.e., exception type should not change, and new exceptions should not be added).</span></span>  
  
 <span data-ttu-id="c3408-122">**X DO NOT** имеет общих членов, которые можно либо throw или не на основе некоторых параметра.</span><span class="sxs-lookup"><span data-stu-id="c3408-122">**X DO NOT** have public members that can either throw or not based on some option.</span></span>  
  
 <span data-ttu-id="c3408-123">**X DO NOT** иметь открытые члены, которые возвращают исключения в качестве возвращаемого значения или `out` параметра.</span><span class="sxs-lookup"><span data-stu-id="c3408-123">**X DO NOT** have public members that return exceptions as the return value or an `out` parameter.</span></span>  
  
 <span data-ttu-id="c3408-124">Возвращать исключения из общедоступных интерфейсов API, а не вызывает их противоречит преимущества отчетов об ошибках, основанной на исключении.</span><span class="sxs-lookup"><span data-stu-id="c3408-124">Returning exceptions from public APIs instead of throwing them defeats many of the benefits of exception-based error reporting.</span></span>  
  
 <span data-ttu-id="c3408-125">**✓ CONSIDER** с помощью методы построителя исключений.</span><span class="sxs-lookup"><span data-stu-id="c3408-125">**✓ CONSIDER** using exception builder methods.</span></span>  
  
 <span data-ttu-id="c3408-126">Чаще всего же исключение из различных мест.</span><span class="sxs-lookup"><span data-stu-id="c3408-126">It is common to throw the same exception from different places.</span></span> <span data-ttu-id="c3408-127">Чтобы избежать Раздувание кода, используйте вспомогательные методы, создающие исключения и инициализировать их свойства.</span><span class="sxs-lookup"><span data-stu-id="c3408-127">To avoid code bloat, use helper methods that create exceptions and initialize their properties.</span></span>  
  
 <span data-ttu-id="c3408-128">Кроме того, члены, вызывающие исключения не получаете как встроенный.</span><span class="sxs-lookup"><span data-stu-id="c3408-128">Also, members that throw exceptions are not getting inlined.</span></span> <span data-ttu-id="c3408-129">Перемещение инструкцию throw в построитель может разрешить член для встраивания.</span><span class="sxs-lookup"><span data-stu-id="c3408-129">Moving the throw statement inside the builder might allow the member to be inlined.</span></span>  
  
 <span data-ttu-id="c3408-130">**X DO NOT** вызывать исключения из блоков фильтрации исключений.</span><span class="sxs-lookup"><span data-stu-id="c3408-130">**X DO NOT** throw exceptions from exception filter blocks.</span></span>  
  
 <span data-ttu-id="c3408-131">Когда фильтр исключений создает исключение, исключение перехватывается в среде CLR и фильтр возвращает значение false.</span><span class="sxs-lookup"><span data-stu-id="c3408-131">When an exception filter raises an exception, the exception is caught by the CLR, and the filter returns false.</span></span> <span data-ttu-id="c3408-132">Это поведение не отличается от фильтра, выполнение и возвращает значение false, явным образом, поэтому очень трудно отлаживать.</span><span class="sxs-lookup"><span data-stu-id="c3408-132">This behavior is indistinguishable from the filter executing and returning false explicitly and is therefore very difficult to debug.</span></span>  
  
 <span data-ttu-id="c3408-133">**X AVOID** явное создание исключений из блоков finally.</span><span class="sxs-lookup"><span data-stu-id="c3408-133">**X AVOID** explicitly throwing exceptions from finally blocks.</span></span> <span data-ttu-id="c3408-134">Неявно возникающих исключениях, полученным в результате вызова методов, которые выдают являются допустимыми.</span><span class="sxs-lookup"><span data-stu-id="c3408-134">Implicitly thrown exceptions resulting from calling methods that throw are acceptable.</span></span>
