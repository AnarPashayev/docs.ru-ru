# <a name="type-design-guidelines"></a><span data-ttu-id="eee78-102">Правила разработки типов</span><span class="sxs-lookup"><span data-stu-id="eee78-102">Type Design Guidelines</span></span>
<span data-ttu-id="eee78-103">С точки зрения среды CLR существует только две категории типов — ссылочные типы и типы значений, но в целях обсуждения структуры платформы мы разделены на несколько логических групп, каждый из которых имеет собственные собственные правила проектирования.</span><span class="sxs-lookup"><span data-stu-id="eee78-103">From the CLR perspective, there are only two categories of types—reference types and value types—but for the purpose of a discussion about framework design, we divide types into more logical groups, each with its own specific design rules.</span></span>

 <span data-ttu-id="eee78-104">Классы являются общими вариантами ссылочных типов.</span><span class="sxs-lookup"><span data-stu-id="eee78-104">Classes are the general case of reference types.</span></span> <span data-ttu-id="eee78-105">Они составляют большую часть типов в большинстве платформ.</span><span class="sxs-lookup"><span data-stu-id="eee78-105">They make up the bulk of types in the majority of frameworks.</span></span> <span data-ttu-id="eee78-106">Классы заменяют популярность до обширного набора объектно-ориентированных функций, которые они поддерживают, и их общей применимости.</span><span class="sxs-lookup"><span data-stu-id="eee78-106">Classes owe their popularity to the rich set of object-oriented features they support and to their general applicability.</span></span> <span data-ttu-id="eee78-107">Базовые классы и абстрактные классы — это специальные логические группы, связанные с расширяемостью.</span><span class="sxs-lookup"><span data-stu-id="eee78-107">Base classes and abstract classes are special logical groups related to extensibility.</span></span>

 <span data-ttu-id="eee78-108">Интерфейсы — это типы, которые могут быть реализованы как ссылочными типами, так и типами значений.</span><span class="sxs-lookup"><span data-stu-id="eee78-108">Interfaces are types that can be implemented by both reference types and value types.</span></span> <span data-ttu-id="eee78-109">Таким образом, они служат в качестве корней для однозначных иерархий ссылочных типов и типов значений.</span><span class="sxs-lookup"><span data-stu-id="eee78-109">They can thus serve as roots of polymorphic hierarchies of reference types and value types.</span></span> <span data-ttu-id="eee78-110">Кроме того, интерфейсы можно использовать для моделирования множественного наследования, которое изначально не поддерживается средой CLR.</span><span class="sxs-lookup"><span data-stu-id="eee78-110">In addition, interfaces can be used to simulate multiple inheritance, which is not natively supported by the CLR.</span></span>

 <span data-ttu-id="eee78-111">Структуры являются общими вариантами типов значений и должны быть зарезервированы для небольших простых типов, аналогичных примитивам языка.</span><span class="sxs-lookup"><span data-stu-id="eee78-111">Structs are the general case of value types and should be reserved for small, simple types, similar to language primitives.</span></span>

 <span data-ttu-id="eee78-112">Перечисления — это особый случай типов значений, используемых для определения коротких наборов значений, таких как дни недели, цвета консоли и т. д.</span><span class="sxs-lookup"><span data-stu-id="eee78-112">Enums are a special case of value types used to define short sets of values, such as days of the week, console colors, and so on.</span></span>

 <span data-ttu-id="eee78-113">Статические классы — это типы, предназначенные для контейнеров для статических членов.</span><span class="sxs-lookup"><span data-stu-id="eee78-113">Static classes are types intended to be containers for static members.</span></span> <span data-ttu-id="eee78-114">Они обычно используются для предоставления сочетаний клавиш для других операций.</span><span class="sxs-lookup"><span data-stu-id="eee78-114">They are commonly used to provide shortcuts to other operations.</span></span>

 <span data-ttu-id="eee78-115">Делегаты, исключения, атрибуты, массивы и коллекции являются специальными вариантами ссылочных типов, предназначенных для определенных целей, а рекомендации по их проектированию и использованию обсуждаются в других местах этой книги.</span><span class="sxs-lookup"><span data-stu-id="eee78-115">Delegates, exceptions, attributes, arrays, and collections are all special cases of reference types intended for specific uses, and guidelines for their design and usage are discussed elsewhere in this book.</span></span>

 <span data-ttu-id="eee78-116">✔️ гарантировать, что каждый тип является четко определенным набором связанных элементов, а не только случайным набором несвязанных функций.</span><span class="sxs-lookup"><span data-stu-id="eee78-116">✔️ DO ensure that each type is a well-defined set of related members, not just a random collection of unrelated functionality.</span></span>
