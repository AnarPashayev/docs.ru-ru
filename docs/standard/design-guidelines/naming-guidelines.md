# <a name="naming-guidelines"></a>Правила именования
В соответствии с единообразным набором соглашений об именовании в разработке платформы может быть основная составляющая удобства использования платформы. Он позволяет использовать платформу многими разработчиками в широко разделенных проектах. Помимо согласованности форм, имена элементов платформы должны быть легко понятны и должны передавать функцию каждого элемента.  
  
 Цель этой главы — предоставить единообразный набор соглашений об именовании, которые приводят к непосредственным именам для разработчиков.  
  
 Хотя применение этих соглашений об именовании в качестве общего руководства по разработке кода приведет к более согласованному именованию в коде, необходимо только применить их к интерфейсам API, которые являются общедоступными (открытые или защищенные типы и члены, а также явно реализованные интерфейсы).
