# <a name="property-design"></a><span data-ttu-id="4055a-102">Разработка свойств</span><span class="sxs-lookup"><span data-stu-id="4055a-102">Property Design</span></span>
<span data-ttu-id="4055a-103">Хотя свойства технически похожи на методы, они сильно отличаются в плане их сценариев использования.</span><span class="sxs-lookup"><span data-stu-id="4055a-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="4055a-104">Они должны отображаться как интеллектуальные поля.</span><span class="sxs-lookup"><span data-stu-id="4055a-104">They should be seen as smart fields.</span></span> <span data-ttu-id="4055a-105">Они имеют синтаксис для вызова полей и гибкость методов.</span><span class="sxs-lookup"><span data-stu-id="4055a-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>  
  
 <span data-ttu-id="4055a-106">**✓ DO** Создание свойства только для чтения, если вызывающий объект не следует изменять значение свойства.</span><span class="sxs-lookup"><span data-stu-id="4055a-106">**✓ DO** create get-only properties if the caller should not be able to change the value of the property.</span></span>  
  
 <span data-ttu-id="4055a-107">Помните, что если тип свойства является изменяемым ссылочным типом, значение свойства можно изменить, даже если это свойство доступно только для получения.</span><span class="sxs-lookup"><span data-stu-id="4055a-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>  
  
 <span data-ttu-id="4055a-108">**X DO NOT** предоставить только для набора свойств или свойств задания, имеющих широкий доступ, нежели метод считывания.</span><span class="sxs-lookup"><span data-stu-id="4055a-108">**X DO NOT** provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>  
  
 <span data-ttu-id="4055a-109">Например, не используйте свойства с открытым методом задания и защищенным методом считывания.</span><span class="sxs-lookup"><span data-stu-id="4055a-109">For example, do not use properties with a public setter and a protected getter.</span></span>  
  
 <span data-ttu-id="4055a-110">Если метод получения свойства не может быть предоставлен, реализуйте в качестве метода функцию.</span><span class="sxs-lookup"><span data-stu-id="4055a-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="4055a-111">Рассмотрите возможность запуска имени метода с `Set` и выполните действия с именем свойства.</span><span class="sxs-lookup"><span data-stu-id="4055a-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="4055a-112">Например, <xref:System.AppDomain> имеет метод с именем `SetCachePath` вместо того, чтобы иметь свойство только для набора, именуемое `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="4055a-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>  
  
 <span data-ttu-id="4055a-113">**✓ DO** предоставить значения по умолчанию для всех свойств, убедившись, что значения по умолчанию не приводят к брешь в системе безопасности или совсем неэффективный код.</span><span class="sxs-lookup"><span data-stu-id="4055a-113">**✓ DO** provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>  
  
 <span data-ttu-id="4055a-114">**✓ DO** позволяют устанавливать в любом порядке, даже если это приводит к временной недопустимое состояние объекта свойства.</span><span class="sxs-lookup"><span data-stu-id="4055a-114">**✓ DO** allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>  
  
 <span data-ttu-id="4055a-115">Обычно два или более свойства взаимосвязаны с точкой, в которой некоторые значения одного свойства могут быть недопустимыми с учетом значений других свойств одного и того же объекта.</span><span class="sxs-lookup"><span data-stu-id="4055a-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="4055a-116">В таких случаях исключения, являющиеся результатом недопустимого состояния, должны быть отложены до тех пор, пока взаимосвязанные свойства фактически не будут использоваться объектом вместе.</span><span class="sxs-lookup"><span data-stu-id="4055a-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>  
  
 <span data-ttu-id="4055a-117">**✓ DO** сохраняют предыдущее значение, если метод задания свойства вызывает исключение.</span><span class="sxs-lookup"><span data-stu-id="4055a-117">**✓ DO** preserve the previous value if a property setter throws an exception.</span></span>  
  
 <span data-ttu-id="4055a-118">**X AVOID** исключения из методов получения свойства.</span><span class="sxs-lookup"><span data-stu-id="4055a-118">**X AVOID** throwing exceptions from property getters.</span></span>  
  
 <span data-ttu-id="4055a-119">Методы получения свойств должны быть простыми операциями и не должны иметь никаких предусловий.</span><span class="sxs-lookup"><span data-stu-id="4055a-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="4055a-120">Если метод получения может вызвать исключение, то, возможно, он будет переработано, чтобы быть методом.</span><span class="sxs-lookup"><span data-stu-id="4055a-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="4055a-121">Обратите внимание, что это правило не применяется к индексаторам, в которых мы предполагаем исключения в результате проверки аргументов.</span><span class="sxs-lookup"><span data-stu-id="4055a-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>  
  
### <a name="indexed-property-design"></a><span data-ttu-id="4055a-122">Разработка индексированных свойств</span><span class="sxs-lookup"><span data-stu-id="4055a-122">Indexed Property Design</span></span>  
 <span data-ttu-id="4055a-123">Индексированное свойство — это специальное свойство, которое может иметь параметры и может вызываться с помощью специального синтаксиса, аналогичного индексированию массива.</span><span class="sxs-lookup"><span data-stu-id="4055a-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>  
  
 <span data-ttu-id="4055a-124">Индексированные свойства обычно называются индексаторами.</span><span class="sxs-lookup"><span data-stu-id="4055a-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="4055a-125">Индексаторы следует использовать только в интерфейсах API, которые предоставляют доступ к элементам в логической коллекции.</span><span class="sxs-lookup"><span data-stu-id="4055a-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="4055a-126">Например, строка представляет собой набор символов, а индексатор для <xref:System.String?displayProperty=nameWithType> был добавлен для доступа к его символам.</span><span class="sxs-lookup"><span data-stu-id="4055a-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>  
  
 <span data-ttu-id="4055a-127">**✓ CONSIDER** использовать индексаторы для предоставления доступа к данным, хранящимся во внутреннем массиве.</span><span class="sxs-lookup"><span data-stu-id="4055a-127">**✓ CONSIDER** using indexers to provide access to data stored in an internal array.</span></span>  
  
 <span data-ttu-id="4055a-128">**✓ CONSIDER** обеспечить индексаторы для типов, представляющих коллекции элементов.</span><span class="sxs-lookup"><span data-stu-id="4055a-128">**✓ CONSIDER** providing indexers on types representing collections of items.</span></span>  
  
 <span data-ttu-id="4055a-129">**X AVOID** использование индексированных свойств с более чем одним параметром.</span><span class="sxs-lookup"><span data-stu-id="4055a-129">**X AVOID** using indexed properties with more than one parameter.</span></span>  
  
 <span data-ttu-id="4055a-130">Если для проектирования требуется несколько параметров, следует подумать, действительно ли свойство представляет метод доступа к логической коллекции.</span><span class="sxs-lookup"><span data-stu-id="4055a-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="4055a-131">Если это не так, используйте вместо них методы.</span><span class="sxs-lookup"><span data-stu-id="4055a-131">If it does not, use methods instead.</span></span> <span data-ttu-id="4055a-132">Попробуйте запустить имя метода с помощью `Get` или `Set`.</span><span class="sxs-lookup"><span data-stu-id="4055a-132">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="4055a-133">**X AVOID** индексаторы с типами параметров, отличных от <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, или перечисления.</span><span class="sxs-lookup"><span data-stu-id="4055a-133">**X AVOID** indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>  
  
 <span data-ttu-id="4055a-134">Если для разработки требуются другие типы параметров, настоятельно повторно оцените, действительно ли API представляет метод доступа к логической коллекции.</span><span class="sxs-lookup"><span data-stu-id="4055a-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="4055a-135">В противном случае используйте метод.</span><span class="sxs-lookup"><span data-stu-id="4055a-135">If it does not, use a method.</span></span> <span data-ttu-id="4055a-136">Попробуйте запустить имя метода с помощью `Get` или `Set`.</span><span class="sxs-lookup"><span data-stu-id="4055a-136">Consider starting the method name with `Get` or `Set`.</span></span>  
  
 <span data-ttu-id="4055a-137">**✓ DO** используйте имя `Item` для индексированных свойств при отсутствии очевидно, что более понятные имена (например, см. <xref:System.String.Chars%2A> свойство `System.String`).</span><span class="sxs-lookup"><span data-stu-id="4055a-137">**✓ DO** use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>  
  
 <span data-ttu-id="4055a-138">В C#индексаторы по умолчанию имеют именованный элемент.</span><span class="sxs-lookup"><span data-stu-id="4055a-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="4055a-139">Для настройки этого имени можно использовать <xref:System.Runtime.CompilerServices.IndexerNameAttribute>.</span><span class="sxs-lookup"><span data-stu-id="4055a-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>  
  
 <span data-ttu-id="4055a-140">**X DO NOT** предоставляют индексатор и методы, которые семантически эквивалентны.</span><span class="sxs-lookup"><span data-stu-id="4055a-140">**X DO NOT** provide both an indexer and methods that are semantically equivalent.</span></span>  
  
 <span data-ttu-id="4055a-141">**X DO NOT** предоставляют более одного семейства перегруженных индексаторов в одном типе.</span><span class="sxs-lookup"><span data-stu-id="4055a-141">**X DO NOT** provide more than one family of overloaded indexers in one type.</span></span>  
  
 <span data-ttu-id="4055a-142">Это обеспечивается C# компилятором.</span><span class="sxs-lookup"><span data-stu-id="4055a-142">This is enforced by the C# compiler.</span></span>  
  
 <span data-ttu-id="4055a-143">**X DO NOT** нестандартное использование индексированных свойств.</span><span class="sxs-lookup"><span data-stu-id="4055a-143">**X DO NOT** use nondefault indexed properties.</span></span>  
  
 <span data-ttu-id="4055a-144">Это обеспечивается C# компилятором.</span><span class="sxs-lookup"><span data-stu-id="4055a-144">This is enforced by the C# compiler.</span></span>  
  
### <a name="property-change-notification-events"></a><span data-ttu-id="4055a-145">События уведомления об изменении свойств</span><span class="sxs-lookup"><span data-stu-id="4055a-145">Property Change Notification Events</span></span>  
 <span data-ttu-id="4055a-146">Иногда бывает полезно указать событие, уведомляющее пользователя об изменениях в значении свойства.</span><span class="sxs-lookup"><span data-stu-id="4055a-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="4055a-147">Например, `System.Windows.Forms.Control` вызывает событие `TextChanged` после изменения значения свойства `Text`.</span><span class="sxs-lookup"><span data-stu-id="4055a-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>  
  
 <span data-ttu-id="4055a-148">**✓ CONSIDER** вызов изменения события уведомления при изменении значений свойств в API высокого уровня (обычно компонентов конструктора).</span><span class="sxs-lookup"><span data-stu-id="4055a-148">**✓ CONSIDER** raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>  
  
 <span data-ttu-id="4055a-149">Если есть хороший сценарий, при котором пользователю известно, когда изменяется свойство объекта, объект должен создать событие уведомления об изменении для свойства.</span><span class="sxs-lookup"><span data-stu-id="4055a-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>  
  
 <span data-ttu-id="4055a-150">Однако вряд ли стоит тратить издержки на создание таких событий для низкоуровневых API, таких как базовые типы или коллекции.</span><span class="sxs-lookup"><span data-stu-id="4055a-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="4055a-151">Например, <xref:System.Collections.Generic.List%601> не будет вызывать такие события при добавлении нового элемента в список и изменении свойства `Count`.</span><span class="sxs-lookup"><span data-stu-id="4055a-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>  
  
 <span data-ttu-id="4055a-152">**✓ CONSIDER** вызов изменения события уведомления при изменении значения свойства через внешних событий.</span><span class="sxs-lookup"><span data-stu-id="4055a-152">**✓ CONSIDER** raising change notification events when the value of a property changes via external forces.</span></span>  
  
 <span data-ttu-id="4055a-153">Если значение свойства изменяется с помощью некоторой внешней силы (не путем вызова методов объекта), события вызываются разработчиком, что значение изменяется и изменилось.</span><span class="sxs-lookup"><span data-stu-id="4055a-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="4055a-154">Хорошим примером является свойство `Text` элемента управления "текстовое поле".</span><span class="sxs-lookup"><span data-stu-id="4055a-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="4055a-155">Когда пользователь вводит текст в `TextBox`, значение свойства изменяется автоматически.</span><span class="sxs-lookup"><span data-stu-id="4055a-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
