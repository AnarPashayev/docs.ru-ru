# <a name="sealing"></a>Запечатывание
Одной из особенностей платформ объектно ориентированного — что разработчики могут расширять и настраивать их способами, непредвиденным, проектировщики этой инфраструктуры. Это power и опасность расширяемой архитектуры. При создании вашей платформы, это так, поэтому очень важно тщательно разрабатывать приложения для расширения его при необходимости, а также ограничить расширяемости, когда это представляет опасность.  
  
 Мощный механизм, который запрещает расширяемости запечатывания. Можно запечатать класс или отдельные члены. Запечатывание класс запрещает наследование класса. Запечатывание член запрещает пользователям переопределять конкретного элемента.  
  
 **X DO NOT** запечатывайте классы без необходимости веская причина для этого.  
  
 Запечатывание класса, так как вы не могу представить сценарий расширяемости не веских причин. Пользователям, платформы и наследование от классов по различным причинам nonobvious, таких как добавление членов удобства. См. в разделе [незапечатанные классы](../../../docs/standard/design-guidelines/unsealed-classes.md) примеры nonobvious причинам требуется наследовать от типа.  
  
 Следующие веские причины для запечатывания класса:  
  
- Этот класс является статическим классом. См. в разделе [разработка статичных классов](../../../docs/standard/design-guidelines/static-class.md).  
  
- Класс сохраняет конфиденциальные секреты в наследуемые защищенные члены.  
  
- Этот класс наследует многие виртуальные члены и затраты на их по отдельности запечатывание бы перевешивают преимущества выходя из незапечатанного класса.  
  
- Класс является атрибут, который требуется очень быстро среды выполнения поиска. Запечатанный атрибуты имеют немного более высоких уровней производительности, чем незапечатанный из них. см. в разделе [атрибуты](../../../docs/standard/design-guidelines/attributes.md).  
  
 **X DO NOT** объявлять защищенные или виртуальные члены в запечатанных типах.  
  
 По определению не может наследоваться от запечатанные типы. Это означает, что не может быть вызвана защищенные члены в запечатанных типах и виртуальных методов для запечатанных типов не могут быть переопределены.  
  
 **✓ CONSIDER** запечатывание члены, которые можно переопределить.  
  
 Проблемы, которые могут быть результатом Общие сведения о виртуальных членах (подробно [виртуальные члены](../../../docs/standard/design-guidelines/virtual-members.md)) применить переопределения, несмотря на то что для немного в меньшей степени. Запечатывание переопределение скрывает от вас этих проблем, начиная с этого момента в иерархии наследования.
