# <a name="attributes"></a><span data-ttu-id="6f1f3-102">Атрибуты</span><span class="sxs-lookup"><span data-stu-id="6f1f3-102">Attributes</span></span>
<span data-ttu-id="6f1f3-103"><xref:System.Attribute?displayProperty=nameWithType> является базовым классом, используемым для определения пользовательских атрибутов.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-103"><xref:System.Attribute?displayProperty=nameWithType> is a base class used to define custom attributes.</span></span>  
  
 <span data-ttu-id="6f1f3-104">Атрибуты — это заметки, которые могут быть добавлены к элементам программирования, таким как сборки, типы, члены и параметры.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-104">Attributes are annotations that can be added to programming elements such as assemblies, types, members, and parameters.</span></span> <span data-ttu-id="6f1f3-105">Они хранятся в метаданных сборки и доступны во время выполнения с помощью API-интерфейсов отражения.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-105">They are stored in the metadata of the assembly and can be accessed at runtime using the reflection APIs.</span></span> <span data-ttu-id="6f1f3-106">Например, платформа определяет <xref:System.ObsoleteAttribute>, которую можно применить к типу или члену, чтобы указать, что тип или член является устаревшим.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-106">For example, the Framework defines the <xref:System.ObsoleteAttribute>, which can be applied to a type or a member to indicate that the type or member has been deprecated.</span></span>  
  
 <span data-ttu-id="6f1f3-107">Атрибуты могут иметь одно или несколько свойств, которые содержат дополнительные данные, связанные с атрибутом.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-107">Attributes can have one or more properties that carry additional data related to the attribute.</span></span> <span data-ttu-id="6f1f3-108">Например, `ObsoleteAttribute` может содержать дополнительные сведения о выпуске, в котором тип или член устарел, а также описание новых интерфейсов API, заменяющих устаревший API.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-108">For example, `ObsoleteAttribute` could carry additional information about the release in which a type or a member got deprecated and the description of the new APIs replacing the obsolete API.</span></span>  
  
 <span data-ttu-id="6f1f3-109">При применении атрибута необходимо указать некоторые свойства атрибута.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-109">Some properties of an attribute must be specified when the attribute is applied.</span></span> <span data-ttu-id="6f1f3-110">Они называются обязательными свойствами или требуемыми аргументами, так как они представлены как параметры позиционированного конструктора.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-110">These are referred to as the required properties or required arguments, because they are represented as positional constructor parameters.</span></span> <span data-ttu-id="6f1f3-111">Например, свойство <xref:System.Diagnostics.ConditionalAttribute.ConditionString%2A> <xref:System.Diagnostics.ConditionalAttribute> является обязательным свойством.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-111">For example, the <xref:System.Diagnostics.ConditionalAttribute.ConditionString%2A> property of the <xref:System.Diagnostics.ConditionalAttribute> is a required property.</span></span>  
  
 <span data-ttu-id="6f1f3-112">Свойства, которые не обязательно должны указываться при применении атрибута, называются необязательными свойствами (или необязательными аргументами).</span><span class="sxs-lookup"><span data-stu-id="6f1f3-112">Properties that do not necessarily have to be specified when the attribute is applied are called optional properties (or optional arguments).</span></span> <span data-ttu-id="6f1f3-113">Они представлены настраиваемыми свойствами.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-113">They are represented by settable properties.</span></span> <span data-ttu-id="6f1f3-114">Компиляторы предоставляют специальный синтаксис для установки этих свойств при применении атрибута.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-114">Compilers provide special syntax to set these properties when an attribute is applied.</span></span> <span data-ttu-id="6f1f3-115">Например, свойство <xref:System.AttributeUsageAttribute.Inherited%2A?displayProperty=nameWithType> представляет необязательный аргумент.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-115">For example, the <xref:System.AttributeUsageAttribute.Inherited%2A?displayProperty=nameWithType> property represents an optional argument.</span></span>  
  
 <span data-ttu-id="6f1f3-116">**✓ DO** имя классы настраиваемых атрибутов с суффиксом «Attribute».</span><span class="sxs-lookup"><span data-stu-id="6f1f3-116">**✓ DO** name custom attribute classes with the suffix "Attribute."</span></span>  
  
 <span data-ttu-id="6f1f3-117">**✓ DO** применить <xref:System.AttributeUsageAttribute> для настраиваемых атрибутов.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-117">**✓ DO** apply the <xref:System.AttributeUsageAttribute> to custom attributes.</span></span>  
  
 <span data-ttu-id="6f1f3-118">**✓ DO** предоставляют настраиваемые свойства для необязательных аргументов.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-118">**✓ DO** provide settable properties for optional arguments.</span></span>  
  
 <span data-ttu-id="6f1f3-119">**✓ DO** обеспечения обязательные аргументы свойства только для чтения.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-119">**✓ DO** provide get-only properties for required arguments.</span></span>  
  
 <span data-ttu-id="6f1f3-120">**✓ DO** предоставляют параметры конструктора для инициализации свойства, соответствующие обязательные аргументы.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-120">**✓ DO** provide constructor parameters to initialize properties corresponding to required arguments.</span></span> <span data-ttu-id="6f1f3-121">Каждый параметр должен иметь одно и то же имя (хотя с другим регистром) в качестве соответствующего свойства.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-121">Each parameter should have the same name (although with different casing) as the corresponding property.</span></span>  
  
 <span data-ttu-id="6f1f3-122">**X AVOID** указав аргументы конструктора для инициализации свойства, соответствующие необязательные аргументы.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-122">**X AVOID** providing constructor parameters to initialize properties corresponding to the optional arguments.</span></span>  
  
 <span data-ttu-id="6f1f3-123">Иными словами, не имеют свойств, которые могут быть заданы с помощью конструктора и метода задания.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-123">In other words, do not have properties that can be set with both a constructor and a setter.</span></span> <span data-ttu-id="6f1f3-124">В этой рекомендации явно указаны аргументы, которые являются необязательными и являются обязательными, и в них не предусмотрены два способа выполнения одного и того же действия.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-124">This guideline makes very explicit which arguments are optional and which are required, and avoids having two ways of doing the same thing.</span></span>  
  
 <span data-ttu-id="6f1f3-125">**X AVOID** перегрузку конструкторов настраиваемых атрибутов.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-125">**X AVOID** overloading custom attribute constructors.</span></span>  
  
 <span data-ttu-id="6f1f3-126">Наличие только одного конструктора явно взаимодействует с пользователем, какие аргументы являются обязательными, а какие — необязательными.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-126">Having only one constructor clearly communicates to the user which arguments are required and which are optional.</span></span>  
  
 <span data-ttu-id="6f1f3-127">**✓ DO** запечатать классы настраиваемых атрибутов, если это возможно.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-127">**✓ DO** seal custom attribute classes, if possible.</span></span> <span data-ttu-id="6f1f3-128">Это позволяет ускорить поиск атрибута.</span><span class="sxs-lookup"><span data-stu-id="6f1f3-128">This makes the look-up for the attribute faster.</span></span>
