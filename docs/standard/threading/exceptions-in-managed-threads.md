# <a name="exceptions-in-managed-threads"></a><span data-ttu-id="a13db-102">Исключения в управляемых потоках</span><span class="sxs-lookup"><span data-stu-id="a13db-102">Exceptions in Managed Threads</span></span>
<span data-ttu-id="a13db-103">Начиная с .NET Framework версии 2.0, среда CLR позволяет большинству необработанных исключений выполняться в потоках.</span><span class="sxs-lookup"><span data-stu-id="a13db-103">Starting with the .NET Framework version 2.0, the common language runtime allows most unhandled exceptions in threads to proceed naturally.</span></span> <span data-ttu-id="a13db-104">Как правило, это означает, что необработанное исключение будет вызывать завершение работы приложения.</span><span class="sxs-lookup"><span data-stu-id="a13db-104">In most cases this means that the unhandled exception causes the application to terminate.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="a13db-105">Это значительное изменение по сравнению с .NET Framework 1.0 и 1.1, в которых поддерживались многие необработанные исключения, например необработанные исключения в потоках из пула потоков.</span><span class="sxs-lookup"><span data-stu-id="a13db-105">This is a significant change from the .NET Framework versions 1.0 and 1.1, which provide a backstop for many unhandled exceptions — for example, unhandled exceptions in thread pool threads.</span></span> <span data-ttu-id="a13db-106">См. [Отличия от предыдущих версий](#ChangeFromPreviousVersions) ниже в этом разделе.</span><span class="sxs-lookup"><span data-stu-id="a13db-106">See [Change from Previous Versions](#ChangeFromPreviousVersions) later in this topic.</span></span>  
  
 <span data-ttu-id="a13db-107">Среда CLR предоставляет поддержку для определенных необработанных исключений, которые используются для управления потоком выполнения программы:</span><span class="sxs-lookup"><span data-stu-id="a13db-107">The common language runtime provides a backstop for certain unhandled exceptions that are used for controlling program flow:</span></span>  
  
- <span data-ttu-id="a13db-108">В потоке создается исключение <xref:System.Threading.ThreadAbortException> из-за вызова <xref:System.Threading.Thread.Abort%2A>.</span><span class="sxs-lookup"><span data-stu-id="a13db-108">A <xref:System.Threading.ThreadAbortException> is thrown in a thread because <xref:System.Threading.Thread.Abort%2A> was called.</span></span>  
  
- <span data-ttu-id="a13db-109">В потоке создается исключение <xref:System.AppDomainUnloadedException> из-за того, что в данный момент выгружается домен приложения, в котором выполняется поток.</span><span class="sxs-lookup"><span data-stu-id="a13db-109">An <xref:System.AppDomainUnloadedException> is thrown in a thread because the application domain in which the thread is executing is being unloaded.</span></span>  
  
- <span data-ttu-id="a13db-110">Среда CLR или ведущий процесс прерывает выполнение потока путем создания внутреннего исключения.</span><span class="sxs-lookup"><span data-stu-id="a13db-110">The common language runtime or a host process terminates the thread by throwing an internal exception.</span></span>  
  
 <span data-ttu-id="a13db-111">Если любые из этих исключений не обрабатываются в потоках, созданных в среде CLR, исключение завершает поток, однако среда CLR не позволяет исключению выполнять какие-либо дополнительные действия.</span><span class="sxs-lookup"><span data-stu-id="a13db-111">If any of these exceptions are unhandled in threads created by the common language runtime, the exception terminates the thread, but the common language runtime does not allow the exception to proceed further.</span></span>  
  
 <span data-ttu-id="a13db-112">Если эти исключения не обрабатываются в главном потоке или в потоках, которые перешли в среду выполнения из неуправляемого кода, они продолжают выполнение, что приводит к завершению работы приложения.</span><span class="sxs-lookup"><span data-stu-id="a13db-112">If these exceptions are unhandled in the main thread, or in threads that entered the runtime from unmanaged code, they proceed normally, resulting in termination of the application.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="a13db-113">Среда выполнения может создать необработанное исключение до того, как любой управляемый код сможет установить обработчик исключения.</span><span class="sxs-lookup"><span data-stu-id="a13db-113">It is possible for the runtime to throw an unhandled exception before any managed code has had a chance to install an exception handler.</span></span> <span data-ttu-id="a13db-114">Даже если управляемый код не в состоянии обработать такое исключение, исключение может продолжать выполняться.</span><span class="sxs-lookup"><span data-stu-id="a13db-114">Even though managed code had no chance to handle such an exception, the exception is allowed to proceed naturally.</span></span>  
  
## <a name="exposing-threading-problems-during-development"></a><span data-ttu-id="a13db-115">Создание проблем при работе с потоками во время разработки</span><span class="sxs-lookup"><span data-stu-id="a13db-115">Exposing Threading Problems During Development</span></span>
