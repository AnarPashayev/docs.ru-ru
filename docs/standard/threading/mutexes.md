## <a name="using-mutexes"></a><span data-ttu-id="464aa-107">Использование мьютексов</span><span class="sxs-lookup"><span data-stu-id="464aa-107">Using Mutexes</span></span>  

 <span data-ttu-id="464aa-108">Поток вызывает метод <xref:System.Threading.WaitHandle.WaitOne%2A> из мьютекса, чтобы затребовать владение.</span><span class="sxs-lookup"><span data-stu-id="464aa-108">A thread calls the <xref:System.Threading.WaitHandle.WaitOne%2A> method of a mutex to request ownership.</span></span> <span data-ttu-id="464aa-109">Вызов блокируется до тех пор, пока мьютекс не будет доступен или пока не истечет дополнительное время ожидания.</span><span class="sxs-lookup"><span data-stu-id="464aa-109">The call blocks until the mutex is available, or until the optional timeout interval elapses.</span></span> <span data-ttu-id="464aa-110">Если мьютексом не владеет ни один поток, сообщается состояние мьютекса.</span><span class="sxs-lookup"><span data-stu-id="464aa-110">The state of a mutex is signaled if no thread owns it.</span></span>  
  
 <span data-ttu-id="464aa-111">Поток освобождает мьютекс, вызывая его метод <xref:System.Threading.Mutex.ReleaseMutex%2A>.</span><span class="sxs-lookup"><span data-stu-id="464aa-111">A thread releases a mutex by calling its <xref:System.Threading.Mutex.ReleaseMutex%2A> method.</span></span> <span data-ttu-id="464aa-112">Мьютексы поддерживают сходство потоков, т. е. мьютекс может быть освобожден только тем потоком, который им владеет.</span><span class="sxs-lookup"><span data-stu-id="464aa-112">Mutexes have thread affinity; that is, the mutex can be released only by the thread that owns it.</span></span> <span data-ttu-id="464aa-113">Если поток освобождает мьютекс, которым он не владеет, в потоке создается исключение <xref:System.ApplicationException>.</span><span class="sxs-lookup"><span data-stu-id="464aa-113">If a thread releases a mutex it does not own, an <xref:System.ApplicationException> is thrown in the thread.</span></span>  
  
 <span data-ttu-id="464aa-114">Так как класс <xref:System.Threading.Mutex> является производным от <xref:System.Threading.WaitHandle>, вы также можете вызвать статические методы <xref:System.Threading.WaitHandle.WaitAll%2A> или <xref:System.Threading.WaitHandle.WaitAny%2A> объекта <xref:System.Threading.WaitHandle>, чтобы запросить владение <xref:System.Threading.Mutex> в сочетании с другими дескрипторами ожидания.</span><span class="sxs-lookup"><span data-stu-id="464aa-114">Because the <xref:System.Threading.Mutex> class derives from <xref:System.Threading.WaitHandle>, you can also call the static <xref:System.Threading.WaitHandle.WaitAll%2A> or <xref:System.Threading.WaitHandle.WaitAny%2A> methods of <xref:System.Threading.WaitHandle> to request ownership of a <xref:System.Threading.Mutex> in combination with other wait handles.</span></span>  
  
 <span data-ttu-id="464aa-115">Если поток владеет <xref:System.Threading.Mutex>, он может указывать этот же <xref:System.Threading.Mutex> в повторных запросах ожидания, не прерывая выполнение. Но в этом случае, чтобы отказаться от владения, потребуется освободить <xref:System.Threading.Mutex> столько же раз, сколько он был вызван.</span><span class="sxs-lookup"><span data-stu-id="464aa-115">If a thread owns a <xref:System.Threading.Mutex>, that thread can specify the same <xref:System.Threading.Mutex> in repeated wait-request calls without blocking its execution; however, it must release the <xref:System.Threading.Mutex> as many times to release ownership.</span></span>  
  
## <a name="abandoned-mutexes"></a><span data-ttu-id="464aa-116">Брошенные мьютексы</span><span class="sxs-lookup"><span data-stu-id="464aa-116">Abandoned Mutexes</span></span>  

 <span data-ttu-id="464aa-117">Если поток завершается без освобождения <xref:System.Threading.Mutex>, мьютекс считается отмененным.</span><span class="sxs-lookup"><span data-stu-id="464aa-117">If a thread terminates without releasing a <xref:System.Threading.Mutex>, the mutex is said to be abandoned.</span></span> <span data-ttu-id="464aa-118">Это часто указывает на серьезную ошибку программирования, поскольку ресурс, защищаемый мьютексом, может остаться в несогласованном состоянии.</span><span class="sxs-lookup"><span data-stu-id="464aa-118">This often indicates a serious programming error because the resource the mutex is protecting might be left in an inconsistent state.</span></span> <span data-ttu-id="464aa-119">Исключение <xref:System.Threading.AbandonedMutexException> создается в следующем потоке, который завладеет этим мьютексом.</span><span class="sxs-lookup"><span data-stu-id="464aa-119">An <xref:System.Threading.AbandonedMutexException> is thrown in the next thread that acquires the mutex.</span></span>
  
 <span data-ttu-id="464aa-120">В случае системного мьютекса брошенный мьютекс может указывать на то, что работа приложения была внезапно прекращена (например, с помощью диспетчера задач Windows).</span><span class="sxs-lookup"><span data-stu-id="464aa-120">In the case of a system-wide mutex, an abandoned mutex might indicate that an application has been terminated abruptly (for example, by using Windows Task Manager).</span></span>

 <span data-ttu-id="464aa-126">Локальный мьютекс существует только в вашем процессе.</span><span class="sxs-lookup"><span data-stu-id="464aa-126">A local mutex exists only within your process.</span></span> <span data-ttu-id="464aa-127">Его может использовать любой поток в вашем процессе, имеющий ссылку на локальный объект <xref:System.Threading.Mutex>.</span><span class="sxs-lookup"><span data-stu-id="464aa-127">It can be used by any thread in your process that has a reference to the local <xref:System.Threading.Mutex> object.</span></span> <span data-ttu-id="464aa-128">Каждый объект <xref:System.Threading.Mutex> является отдельным локальным мьютексом.</span><span class="sxs-lookup"><span data-stu-id="464aa-128">Each <xref:System.Threading.Mutex> object is a separate local mutex.</span></span>
