  <span data-ttu-id="276be-113">Когда *указатель точности* ограничивает число цифр дробной части в итоговой строке, числа в итоговых строках округляются до представимого результата, ближайшего к бесконечно точному.</span><span class="sxs-lookup"><span data-stu-id="276be-113">When *precision specifier* controls the number of fractional digits in the result string, the result string reflects a number that is rounded to a representable result nearest to the infinitely precise result.</span></span> <span data-ttu-id="276be-114">Если имеются два одинаково близких представимых результата:</span><span class="sxs-lookup"><span data-stu-id="276be-114">If there are two equally near representable results:</span></span>
  - <span data-ttu-id="276be-115">**В .NET Framework и .NET Core вплоть до версии 2.0** среда выполнения выбирает результат с большей наименее значащей цифрой (то есть с использованием <xref:System.MidpointRounding.AwayFromZero?displayProperty=nameWithType>).</span><span class="sxs-lookup"><span data-stu-id="276be-115">**On the .NET Framework and .NET Core up to .NET Core 2.0**, the runtime selects the result with the greater least significant digit (that is, using <xref:System.MidpointRounding.AwayFromZero?displayProperty=nameWithType>).</span></span>
  - <span data-ttu-id="276be-116">**В .NET Core 2.1 и более поздних версиях** среда выполнения выбирает результат с четной наименее значащей цифрой (то есть с использованием <xref:System.MidpointRounding.ToEven?displayProperty=nameWithType>).</span><span class="sxs-lookup"><span data-stu-id="276be-116">**On .NET Core 2.1 and later**, the runtime selects the result with an even least significant digit (that is, using <xref:System.MidpointRounding.ToEven?displayProperty=nameWithType>).</span></span>

  > <span data-ttu-id="276be-117">Описатель точности определяет число цифр в результирующей строке.</span><span class="sxs-lookup"><span data-stu-id="276be-117">The precision specifier determines the number of digits in the result string.</span></span> <span data-ttu-id="276be-118">Чтобы заполнить строку результата начальными или конечными пробелами, используйте функцию [составного форматирования](../../../docs/standard/base-types/composite-formatting.md) и определите  *компонент выравнивания* в элементе форматирования.</span><span class="sxs-lookup"><span data-stu-id="276be-118">To pad a result string with leading or trailing spaces, use the [composite formatting](../../../docs/standard/base-types/composite-formatting.md) feature and define an *alignment component* in the format item.</span></span>
