## <a name="explicitly-named-captures"></a><span data-ttu-id="a9527-127">Явно именованные шаблоны</span><span class="sxs-lookup"><span data-stu-id="a9527-127">Explicitly Named Captures</span></span>  
 <span data-ttu-id="a9527-128">В обычных регулярных выражениях круглые скобки, обозначающие отдельные шаблоны, автоматически последовательно нумеруются.</span><span class="sxs-lookup"><span data-stu-id="a9527-128">In traditional regular expressions, capturing parentheses are automatically numbered sequentially.</span></span> <span data-ttu-id="a9527-129">В связи с этим возникают две проблемы.</span><span class="sxs-lookup"><span data-stu-id="a9527-129">This leads to two problems.</span></span> <span data-ttu-id="a9527-130">Во-первых, если регулярное выражение изменяется из-за вставки или удаления пары круглых скобок, все части кода, которые ссылаются на нумерованные шаблоны, необходимо переписать, чтобы отразить новую нумерацию.</span><span class="sxs-lookup"><span data-stu-id="a9527-130">First, if a regular expression is modified by inserting or removing a set of parentheses, all code that refers to the numbered captures must be rewritten to reflect the new numbering.</span></span> <span data-ttu-id="a9527-131">Во-вторых, вследствие того, что различные пары круглых скобок часто используются для определения двух альтернативных выражений для поиска, трудно определить, какое из двух выражений в действительности вернуло результат.</span><span class="sxs-lookup"><span data-stu-id="a9527-131">Second, because different sets of parentheses often are used to provide two alternative expressions for an acceptable match, it might be difficult to determine which of the two expressions actually returned a result.</span></span>
