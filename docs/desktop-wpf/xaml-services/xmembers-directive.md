# <a name="xmembers-directive"></a>Директива x:Members
Содержит набор элементов, определяемых в разметке, которые применяются к x:Class родительского элемента.

## <a name="xaml-attribute-usage"></a>Использование атрибута XAML

```xaml
<object x:Class="className">
<x:Members>
  oneOrMoreMembers
</x:Members
</object>
```

## <a name="xaml-values"></a>Значения XAML

|||
|-|-|
|`className`|Имя класса резервирования или разделяемого класса для рабочей среды XAML. См. заметки.|
|`oneOrMoreMembers`|Один или несколько элементов объекта, представляющих определения членов. Как правило, `x:Property` это элементы объектов. См. заметки.|

## <a name="remarks"></a>Примечания

В реализации .NET XAML Services нет типа поддержки `x:Members`или базовой реализации для . `x:Members`является специальным членом XAML, который может существовать в качестве члена любого типа. В потоке `x:Members` узлов XAML, представлен как `Members`член по имени , из XAML язык XAML пространства имен. Участник `Members` содержит общий список `Member` объектов только для чтения. В типичной разметке `x:Property` отдельные участники указаны как элементы свойства. `x:Property`является более точным типом специально для свойств типов `x:Member`и присваивается . Для получения дополнительной информации [см.](xproperty-directive.md)

Для поддержки практического использования `x:Members` как средства указания определений членов в разметке эти члены должны быть связаны с классом, который может быть изменен. Предполагаемая модель состоит в том, что `x:Members` существует в качестве члена типа, указывающего `x:Class`. Однако механизм связывания типов и членов или для создания динамических определений членов не поддерживается на уровне .NET XAML Services. Это отводится отдельным платформам, имеющим модели приложений, поддерживающие определения членов из XAML. Как правило, для поддержки этой функции требуются действия MSBUILD при построении, которые компилируют разметку XAML и либо интегрируют его с выделенным кодом, либо создают чистые сборки из XAML.

## <a name="xmembers-for-windows-workflow-foundation"></a>x:Члены фонда рабочего процесса Windows

Для Windows Workflow `x:Members` Foundation содержит участников пользовательского действия, полностью составленного в XAML, или XAML-определенных динамических членов для разработчика деятельности с кодовым кодом. `x:Class` также должен быть указан в корневом элементе рабочей среды XAML. Это не является требованием на уровне .NET XAML Services, но становится требованием, когда производство XAML загружается действиями msBUILD, которые поддерживают пользовательские действия и Фонд обработки данных Windows XAML в целом. `x:Members`должен быть первым элементом ребенка в разметке `x:Class`элемента объекта, который декларирует .
