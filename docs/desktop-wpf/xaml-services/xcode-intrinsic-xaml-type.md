# <a name="xcode-intrinsic-xaml-type"></a>Встроенный тип XAML x:Code
Позволяет разместить код в рамках производства XAML. Такой код может быть компилирован любой реализацией процессора XAML, которая компилирует XAML, либо оставлен в производстве XAML для последующего использования, например, интерпретация времен выполнения.

## <a name="xaml-object-element-usage"></a>Использование элемента объекта XAML

```xaml
<x:Code>
   // code instructions, usually enclosed by CDATA...
</x:Code>
```

## <a name="remarks"></a>Примечания

Код в `x:Code` элементе директивы XAML по-прежнему интерпретируется в общем пространстве имен XML и предоставленных пространствах имен XAML. Поэтому обычно необходимо приложить код, используемый `x:Code` `CDATA` для внутри сегмента.

`x:Code`не допускается для всех возможных механизмов развертывания производства XAML. В конкретных рамках (например, WPF) код должен быть скомпилирован. В других средах `x:Code` использование может быть обычно запрещено.

Для инфраструктур, позволяющих управлять `x:Code` контентом, правильный `x:Code` компилятор языка для использования в контенте определяется настройками и целями содержащегося проекта, который используется для компиляции приложения.

## <a name="wpf-usage-notes"></a>Примечания об использовании WPF

Код, `x:Code` декларированный в рамках для WPF, имеет несколько заметных ограничений:

- Элемент `x:Code` директивы должен быть непосредственным элементом ребенка корневого элемента производства XAML.

- [x: Директива класса](xclass-directive.md) должна быть предоставлена на элементе родительского корневого элемента.

- Код, размещенный внутри, `x:Code` будет обрабатываться компиляцией, которая будет находиться в рамках частичный класс, который уже создается для этой страницы XAML. Поэтому весь код, который вы определяете, должен быть членами или переменными этого частичного класса.

- Нельзя определить дополнительные классы, кроме как вложения класса внутри частичного класса (гнездо разрешено, но это не типично, поскольку вложенные классы не могут быть отнесены к XAML). Пространства имен CLR, отличные от пространства имен, используемого для существующего частичного класса, не могут быть определены или добавлены.

- Ссылки на кодовые сущности за пределами частичного пространства имен класса CLR должны быть полностью квалифицированы. Если объявленные участники переопределяются с частичным классом, которые перекрываются, это должно быть указано с помощью ключевого слова, конкретного языка переопределения. Если участники `x:Code` объявили в сфере конфликта с членами частичного класса, созданными из XAML, таким образом, что компилятор сообщает о конфликте, файл XAML не может компилировать или загружать.
