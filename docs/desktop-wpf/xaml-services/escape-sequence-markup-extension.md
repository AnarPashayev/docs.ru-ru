# <a name="-escape-sequence--markup-extension"></a>{}Побег последовательность / разметка расширение

Обеспечивает последовательность побега XAML для значений атрибутов. Последовательность побега позволяет интерпретировать последующие значения в атрибуте как буквальные.

## <a name="xaml-attribute-usage"></a>Использование атрибута XAML

```xaml
<object property="{} literalValue" .../>
```

## <a name="xaml-property-element-usage"></a>Использование элемента свойства XAML

```xaml
<object>
  <object.property>
    {} literalValue
  </object.property>
</object>
```

## <a name="xaml-values"></a>Значения XAML

|||
|-|-|
|*буквальнаястоимость*|Буквальная строка, которая следует за последовательностью побега. Обычно эта строка содержит открытую или закрытую скобку (я. или я).|

## <a name="remarks"></a>Примечания

Последовательность побега ({}) используется так, что открытая скобка (я) может быть использована в качестве буквального символа в XAML.

Читатели XAML обычно используют открытую скобку (к) для обозначения точки входа в расширение разметки; однако, они сначала проверяют следующий символ, чтобы определить, является ли это закрытие скобки (я). Только тогда, когда{}две скобки ( ) являются смежными, они считаются побега последовательности.

При возникновении последовательности побега считыватель XAML должен обработать оставшуюся часть строки в виде строки. Однако, если последовательность побега применяется к члену, который имеет преобразователь типа, строка может подвергнуться преобразованию типа, когда она интерпретируется автором XAML.

Последовательность побега не является расширением разметки и не поддерживается классом. Тем не менее, это конвенция, что XAML читателей (в том числе пользовательских xAML читателей) должны уважать.

Таким образом, знак котировки () не может использоваться в качестве последовательности побега. Если вам нужно установить отметку котировки в качестве значения свойства для свойства, используйте синтаксис элемента свойства и поместите отметку котировки в качестве строки внутри элемента свойства, или используйте сущность символа XML. Для свойства содержимого знак котировки может быть всем содержанием.

Последовательность побега (){}часто требуется при указании типа XML, который должен включать квалификатор пространства имен в месте, где может появиться расширение разметки XAML. Это место включает в себя начало значения атрибута XAML и расширение разметки сразу после равного знака.
