## <a name="implementation-guidelines-and-conventions"></a><span data-ttu-id="bf760-112">Правила и соглашения реализации</span><span class="sxs-lookup"><span data-stu-id="bf760-112">Implementation Guidelines and Conventions</span></span>  
 <span data-ttu-id="bf760-113">При реализации шаблона элемента управления Grid обратите внимание на следующие правила и соглашения.</span><span class="sxs-lookup"><span data-stu-id="bf760-113">When implementing the Grid control pattern, note the following guidelines and conventions:</span></span>  
  
- <span data-ttu-id="bf760-114">Координаты сетки отсчитываются начиная с нуля и от левой верхней ячейки (или правой верхней ячейки в зависимости от языкового стандарта), имеющей координаты (0, 0).</span><span class="sxs-lookup"><span data-stu-id="bf760-114">Grid coordinates are zero-based with the upper left (or upper right cell depending on locale) having coordinates (0, 0).</span></span>  
  
- <span data-ttu-id="bf760-115">Если ячейка пуста, элемент модели автоматизации пользовательского интерфейса по-прежнему должен возвращаться для поддержки свойства <xref:System.Windows.Automation.Provider.IGridItemProvider.ContainingGrid%2A> для этой ячейки.</span><span class="sxs-lookup"><span data-stu-id="bf760-115">If a cell is empty, a UI Automation element must still be returned in order to support the <xref:System.Windows.Automation.Provider.IGridItemProvider.ContainingGrid%2A> property for that cell.</span></span> <span data-ttu-id="bf760-116">Это возможно, когда макет дочерних элементов сетки подобен массиву с переменной длиной (см. пример ниже).</span><span class="sxs-lookup"><span data-stu-id="bf760-116">This is possible when the layout of child elements in the grid is similar to a ragged array (see example below).</span></span>  
  
 <span data-ttu-id="bf760-117">![Windows Explorer, отображающий неровную структуру.](./media/uia-gridpattern-ragged-array.PNG "UIA_GridPattern_Ragged_Array")</span><span class="sxs-lookup"><span data-stu-id="bf760-117">![Windows Explorer view showing ragged layout.](./media/uia-gridpattern-ragged-array.PNG "UIA_GridPattern_Ragged_Array")</span></span>  
<span data-ttu-id="bf760-118">Пример элемента управления "Сетка" с пустыми координатами</span><span class="sxs-lookup"><span data-stu-id="bf760-118">Example of a Grid Control with Empty Coordinates</span></span>  
  
- <span data-ttu-id="bf760-119">Сетка с единственным элементом по-прежнему должна реализовывать <xref:System.Windows.Automation.Provider.IGridProvider> , если она логически считается сеткой.</span><span class="sxs-lookup"><span data-stu-id="bf760-119">A grid with a single item is still required to implement <xref:System.Windows.Automation.Provider.IGridProvider> if it is logically considered to be a grid.</span></span> <span data-ttu-id="bf760-120">Количество дочерних элементов в сетке не имеет значения.</span><span class="sxs-lookup"><span data-stu-id="bf760-120">The number of child items in the grid is immaterial.</span></span>  
  
- <span data-ttu-id="bf760-121">Скрытые строки и столбцы, в зависимости от реализации поставщика, могут быть загружены в дерево [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] и поэтому будут отражаться в свойствах <xref:System.Windows.Automation.GridPattern.GridPatternInformation.RowCount%2A> и <xref:System.Windows.Automation.GridPattern.GridPatternInformation.ColumnCount%2A> .</span><span class="sxs-lookup"><span data-stu-id="bf760-121">Hidden rows and columns, depending on the provider implementation, may be loaded in the [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] tree and therefore will be reflected in the <xref:System.Windows.Automation.GridPattern.GridPatternInformation.RowCount%2A> and <xref:System.Windows.Automation.GridPattern.GridPatternInformation.ColumnCount%2A> properties.</span></span> <span data-ttu-id="bf760-122">Если скрытые строки и столбцы еще не загружены, они не должны учитываться.</span><span class="sxs-lookup"><span data-stu-id="bf760-122">If the hidden rows and columns have not yet been loaded, they should not be counted.</span></span>  
  
- <span data-ttu-id="bf760-123"><xref:System.Windows.Automation.Provider.IGridProvider> не позволяет активно манипулировать сеткой; для включения этой функциональности необходимо реализовать <xref:System.Windows.Automation.Provider.ITransformProvider> .</span><span class="sxs-lookup"><span data-stu-id="bf760-123"><xref:System.Windows.Automation.Provider.IGridProvider> does not enable active manipulation of a grid; <xref:System.Windows.Automation.Provider.ITransformProvider> must be implemented to enable this functionality.</span></span>  
  
- <span data-ttu-id="bf760-124">Используйте <xref:System.Windows.Automation.StructureChangedEventHandler> для прослушивания изменений структуры или макета сетки, таких как добавление, удаление или слияние ячеек.</span><span class="sxs-lookup"><span data-stu-id="bf760-124">Use a <xref:System.Windows.Automation.StructureChangedEventHandler> to listen for structural or layout changes to the grid such as cells that have been added, removed, or merged.</span></span>  
  
- <span data-ttu-id="bf760-125">Используйте <xref:System.Windows.Automation.AutomationFocusChangedEventHandler> для отслеживания прохождения по элементам или ячейкам сетки.</span><span class="sxs-lookup"><span data-stu-id="bf760-125">Use a <xref:System.Windows.Automation.AutomationFocusChangedEventHandler> to track traversal through the items or cells of a grid.</span></span>
