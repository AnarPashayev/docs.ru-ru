# <a name="how-to-add-installers-to-your-service-application"></a><span data-ttu-id="981be-104">Практическое руководство. Добавление установщиков в приложение-службу</span><span class="sxs-lookup"><span data-stu-id="981be-104">How to: Add Installers to Your Service Application</span></span>

<span data-ttu-id="981be-105">В состав Visual Studio входят компоненты установки, которые могут устанавливать ресурсы, связанные с приложениями-службами.</span><span class="sxs-lookup"><span data-stu-id="981be-105">Visual Studio ships installation components that can install resources associated with your service applications.</span></span> <span data-ttu-id="981be-106">Компоненты установки регистрируют отдельную службу в целевой системе и сообщают диспетчеру служб об этой службе.</span><span class="sxs-lookup"><span data-stu-id="981be-106">Installation components register an individual service on the system to which it is being installed and let the Services Control Manager know that the service exists.</span></span> <span data-ttu-id="981be-107">При работе с приложением-службой можно выбрать ссылку в окне свойств для автоматического добавления соответствующих установщиков в проект.</span><span class="sxs-lookup"><span data-stu-id="981be-107">When you work with a service application, you can select a link in the Properties window to automatically add the appropriate installers to your project.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="981be-108">Значения свойств для службы копируются из класса службы в класс установщика.</span><span class="sxs-lookup"><span data-stu-id="981be-108">Property values for your service are copied from the service class to the installer class.</span></span> <span data-ttu-id="981be-109">Обновляемые значения свойств в классе службы не обновляются автоматически в установщике.</span><span class="sxs-lookup"><span data-stu-id="981be-109">If you update the property values on the service class, they are not automatically updated in the installer.</span></span>  
  
 <span data-ttu-id="981be-110">При добавлении установщика в проект в нем создается класс (который по умолчанию называется `ProjectInstaller`), а в нем создаются экземпляры соответствующих компонентов установки.</span><span class="sxs-lookup"><span data-stu-id="981be-110">When you add an installer to your project, a new class (which, by default, is named `ProjectInstaller`) is created in the project, and instances of the appropriate installation components are created within it.</span></span> <span data-ttu-id="981be-111">Этот класс служит центральной точкой для всех компонентов установки, которые требуются для проекта.</span><span class="sxs-lookup"><span data-stu-id="981be-111">This class acts as a central point for all of the installation components your project needs.</span></span> <span data-ttu-id="981be-112">Например, если добавить в приложение вторую службу и щелкнуть ссылку "Добавить установщик", второй класс установщика не будет создан. Вместо этого в существующий класс будет добавлен дополнительный необходимый компонент установки для второй службы.</span><span class="sxs-lookup"><span data-stu-id="981be-112">For example, if you add a second service to your application and click the Add Installer link, a second installer class is not created; instead, the necessary additional installation component for the second service is added to the existing class.</span></span>  
  
 <span data-ttu-id="981be-113">Вам не нужно добавлять специальный код в установщики, чтобы обеспечить правильную установку службы.</span><span class="sxs-lookup"><span data-stu-id="981be-113">You do not need to do any special coding within the installers to make your services install correctly.</span></span> <span data-ttu-id="981be-114">Тем не менее иногда может потребоваться изменить содержимое установщиков, если вам нужно добавить специальные функции в процесс установки.</span><span class="sxs-lookup"><span data-stu-id="981be-114">However, you may occasionally need to modify the contents of the installers if you need to add special functionality to the installation process.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="981be-115">Отображаемые диалоговые окна и команды меню могут отличаться от описанных в справке в зависимости от текущих параметров или выпуска.</span><span class="sxs-lookup"><span data-stu-id="981be-115">The dialog boxes and menu commands you see might differ from those described in Help depending on your active settings or edition.</span></span> <span data-ttu-id="981be-116">Чтобы изменить параметры, выберите в меню **Сервис** пункт **Импорт и экспорт параметров** .</span><span class="sxs-lookup"><span data-stu-id="981be-116">To change your settings, choose **Import and Export Settings** on the **Tools** menu.</span></span> <span data-ttu-id="981be-117">Дополнительные сведения см. в разделе [Персонализация интегрированной среды разработки Visual Studio](/visualstudio/ide/personalizing-the-visual-studio-ide).</span><span class="sxs-lookup"><span data-stu-id="981be-117">For more information, see [Personalize the Visual Studio IDE](/visualstudio/ide/personalizing-the-visual-studio-ide).</span></span>  
  
### <a name="to-add-installers-to-your-service-application"></a><span data-ttu-id="981be-118">Добавление установщиков в приложение-службу</span><span class="sxs-lookup"><span data-stu-id="981be-118">To add installers to your service application</span></span>  
  
1. <span data-ttu-id="981be-119">В **обозревателе решений** откройте **конструктор** для службы, в которую нужно добавить компонент установки.</span><span class="sxs-lookup"><span data-stu-id="981be-119">In **Solution Explorer**, access **Design** view for the service for which you want to add an installation component.</span></span>  
  
2. <span data-ttu-id="981be-120">Щелкните фон конструктора, чтобы выбрать саму службу, а не один из ее элементов.</span><span class="sxs-lookup"><span data-stu-id="981be-120">Click the background of the designer to select the service itself, rather than any of its contents.</span></span>  
  
3. <span data-ttu-id="981be-121">Щелкните правой кнопкой мыши в активном окне конструктора и выберите команду **Добавить установщик**.</span><span class="sxs-lookup"><span data-stu-id="981be-121">With the designer in focus, right-click, and then click **Add Installer**.</span></span>  
  
     <span data-ttu-id="981be-122">В проект будут добавлены новый класс `ProjectInstaller` и два компонента установки <xref:System.ServiceProcess.ServiceProcessInstaller> и <xref:System.ServiceProcess.ServiceInstaller>, в которые будут скопированы значения свойств для службы.</span><span class="sxs-lookup"><span data-stu-id="981be-122">A new class, `ProjectInstaller`, and two installation components, <xref:System.ServiceProcess.ServiceProcessInstaller> and <xref:System.ServiceProcess.ServiceInstaller>, are added to your project, and property values for the service are copied to the components.</span></span>  
  
4. <span data-ttu-id="981be-123">Щелкните компонент <xref:System.ServiceProcess.ServiceInstaller> и убедитесь, что для свойства <xref:System.ServiceProcess.ServiceInstaller.ServiceName%2A> задано то же значение, что и для свойства <xref:System.ServiceProcess.ServiceInstaller.ServiceName%2A> в службе.</span><span class="sxs-lookup"><span data-stu-id="981be-123">Click the <xref:System.ServiceProcess.ServiceInstaller> component and verify that the value of the <xref:System.ServiceProcess.ServiceInstaller.ServiceName%2A> property is set to the same value as the <xref:System.ServiceProcess.ServiceInstaller.ServiceName%2A> property on the service itself.</span></span>  
  
5. <span data-ttu-id="981be-124">Чтобы определить, как будет запущена служба, щелкните компонент <xref:System.ServiceProcess.ServiceInstaller> и задайте для свойства <xref:System.ServiceProcess.ServiceInstaller.StartType%2A> соответствующее значение.</span><span class="sxs-lookup"><span data-stu-id="981be-124">To determine how your service will be started, click the <xref:System.ServiceProcess.ServiceInstaller> component and set the <xref:System.ServiceProcess.ServiceInstaller.StartType%2A> property to the appropriate value.</span></span>

    |-----------|------------|

Запуск служб](how-to-start-services.md).</span><span class="sxs-lookup"><span data-stu-id="981be-128">For more information, see [How to: Start Services](how-to-start-services.md).</span></span>|  
    |<xref:System.ServiceProcess.ServiceStartMode.Automatic>|<span data-ttu-id="981be-129">Служба будет запускаться сама при перезагрузке компьютера.</span><span class="sxs-lookup"><span data-stu-id="981be-129">The service will start by itself whenever the computer reboots.</span></span>|  
    |<xref:System.ServiceProcess.ServiceStartMode.Disabled>|<span data-ttu-id="981be-130">Запуск службы невозможен.</span><span class="sxs-lookup"><span data-stu-id="981be-130">The service cannot be started.</span></span>|  
  
6. <span data-ttu-id="981be-131">Чтобы определить контекст безопасности, в котором будет запущена служба, щелкните компонент <xref:System.ServiceProcess.ServiceProcessInstaller> и задайте соответствующие значения свойств.</span><span class="sxs-lookup"><span data-stu-id="981be-131">To determine the security context in which your service will run, click the <xref:System.ServiceProcess.ServiceProcessInstaller> component and set the appropriate property values.</span></span> <span data-ttu-id="981be-132">Дополнительные сведения см. в разделе [Практическое руководство. Назначение службам контекста безопасности](how-to-specify-the-security-context-for-services.md).</span><span class="sxs-lookup"><span data-stu-id="981be-132">For more information, see [How to: Specify the Security Context for Services](how-to-specify-the-security-context-for-services.md).</span></span>  
  
7. <span data-ttu-id="981be-133">Переопределите все методы, для которых нужно выполнить дополнительную обработку.</span><span class="sxs-lookup"><span data-stu-id="981be-133">Override any methods for which you need to perform custom processing.</span></span>  
  
8. <span data-ttu-id="981be-134">Повторите шаги 1–7 для каждой дополнительной службы в проекте.</span><span class="sxs-lookup"><span data-stu-id="981be-134">Perform steps 1 through 7 for each additional service in your project.</span></span>  
  
    > [!NOTE]
    > <span data-ttu-id="981be-135">Для каждой дополнительной службы в проекте необходимо добавить в класс `ProjectInstaller` проекта дополнительный компонент <xref:System.ServiceProcess.ServiceInstaller>.</span><span class="sxs-lookup"><span data-stu-id="981be-135">For each additional service in your project, you must add an additional <xref:System.ServiceProcess.ServiceInstaller> component to the project's `ProjectInstaller` class.</span></span> <span data-ttu-id="981be-136">Компонент <xref:System.ServiceProcess.ServiceProcessInstaller>, добавленный на шаге 3, работает со всеми отдельными установщиками служб в проекте.</span><span class="sxs-lookup"><span data-stu-id="981be-136">The <xref:System.ServiceProcess.ServiceProcessInstaller> component added in step three works with all of the individual service installers in the project.</span></span>
