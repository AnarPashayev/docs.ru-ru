 <span data-ttu-id="38ceb-125">При проектировании, построении и развертывании приложения необходимо исходить из того, что оно будет атаковано.</span><span class="sxs-lookup"><span data-stu-id="38ceb-125">When you design, build, and deploy your application, you must assume that your application will be attacked.</span></span> <span data-ttu-id="38ceb-126">Часто атаки проводятся с помощью злонамеренного кода, который выполняется с разрешениями пользователя, запустившего этот код.</span><span class="sxs-lookup"><span data-stu-id="38ceb-126">Often these attacks come from malicious code that executes with the permissions of the user running the code.</span></span> <span data-ttu-id="38ceb-127">Другие атаки могут исходить из кода, разработанного с благими намерениями, который в действительности был использован потенциальным взломщиком для осуществления одного из приемов атаки.</span><span class="sxs-lookup"><span data-stu-id="38ceb-127">Others can originate with well-intentioned code that has been exploited by an attacker.</span></span> <span data-ttu-id="38ceb-128">При планировании защиты всегда следует предполагать возможность реализации самого худшего сценария.</span><span class="sxs-lookup"><span data-stu-id="38ceb-128">When planning security, always assume the worst-case scenario will occur.</span></span>
