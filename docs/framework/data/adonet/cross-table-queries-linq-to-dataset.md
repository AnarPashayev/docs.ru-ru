# <a name="cross-table-queries-linq-to-dataset"></a><span data-ttu-id="fe4d1-102">Запросы между таблицами (LINQ to DataSet)</span><span class="sxs-lookup"><span data-stu-id="fe4d1-102">Cross-Table Queries (LINQ to DataSet)</span></span>

<span data-ttu-id="fe4d1-103">В дополнение к запросам к одной таблице можно также выполнять запросы между таблицами в LINQ to DataSet.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-103">In addition to querying a single table, you can also perform cross-table queries in LINQ to DataSet.</span></span> <span data-ttu-id="fe4d1-104">Это делается с помощью *объединения*.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-104">This is done by using a *join*.</span></span> <span data-ttu-id="fe4d1-105">Соединение представляет собой взаимосвязь объектов одного источника данных с объектами, использующими общий атрибут, например идентификатор продукта или контактного лица, в другом источнике данных.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-105">A join is the association of objects in one data source with objects that share a common attribute in another data source, such as a product or contact ID.</span></span> <span data-ttu-id="fe4d1-106">В объектно-ориентированном программировании связи между объектами относительно просты для навигации, поскольку каждый объект имеет член, ссылающийся на другой объект.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-106">In object-oriented programming, relationships between objects are relatively easy to navigate because each object has a member that references another object.</span></span> <span data-ttu-id="fe4d1-107">Однако в таблицах внешних баз данных перемещение по связям не столь однозначно.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-107">In external database tables, however, navigating relationships is not as straightforward.</span></span> <span data-ttu-id="fe4d1-108">Таблицы баз данных не содержат встроенных связей.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-108">Database tables do not contain built-in relationships.</span></span> <span data-ttu-id="fe4d1-109">В таких случаях для соединения элементов из разных источников может использоваться операция объединения.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-109">In these cases, the join operation can be used to match elements from each source.</span></span> <span data-ttu-id="fe4d1-110">Например, если две таблицы содержат данные о продуктах и о продажах, нужно использовать операцию соединения для сопоставления данных о продажах и о продуктах, относящихся к одному и тому же заказу на продажу.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-110">For example, given two tables that contain product information and sales information, you could use a join operation to match sales information and products for the same sales order.</span></span>  
  
 <span data-ttu-id="fe4d1-111">Платформа, интегрированная с LINQ, предоставляет два оператора Join: <xref:System.Linq.Enumerable.Join%2A> и <xref:System.Linq.Enumerable.GroupJoin%2A> .</span><span class="sxs-lookup"><span data-stu-id="fe4d1-111">The Language-Integrated Query (LINQ) framework provides two join operators, <xref:System.Linq.Enumerable.Join%2A> and <xref:System.Linq.Enumerable.GroupJoin%2A>.</span></span> <span data-ttu-id="fe4d1-112">Эти операторы выполняют *эквивалентные объединения*: то есть объединения, которые соответствуют двум источникам данных, только если их ключи равны.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-112">These operators perform *equi-joins*: that is, joins that match two data sources only when their keys are equal.</span></span> <span data-ttu-id="fe4d1-113">(Напротив, Transact-SQL поддерживает операторы Join, отличные от `equals` , например `less than` оператор).</span><span class="sxs-lookup"><span data-stu-id="fe4d1-113">(By contrast, Transact-SQL supports join operators other than `equals`, such as the `less than` operator.)</span></span>  
  
 <span data-ttu-id="fe4d1-114">В терминах реляционной базы данных оператор <xref:System.Linq.Enumerable.Join%2A> выполняет внутреннее соединение.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-114">In relational database terms, <xref:System.Linq.Enumerable.Join%2A> implements an inner join.</span></span> <span data-ttu-id="fe4d1-115">Внутреннее соединение представляет собой соединение, при котором возвращаются только объекты, имеющие соответствия в другом наборе данных.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-115">An inner join is a type of join in which only those objects that have a match in the opposite data set are returned.</span></span>  
  
 <span data-ttu-id="fe4d1-116"><xref:System.Linq.Enumerable.GroupJoin%2A>Операторы не имеют прямого эквивалента в терминах реляционной базы данных; они реализуют надмножество внутренних соединений и левых внешних соединений.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-116">The <xref:System.Linq.Enumerable.GroupJoin%2A> operators have no direct equivalent in relational database terms; they implement a superset of inner joins and left outer joins.</span></span> <span data-ttu-id="fe4d1-117">Левое внешнее соединение — это соединение, возвращающее каждый элемент первой (левой) коллекции, даже если в второй коллекции нет коррелированных элементов.</span><span class="sxs-lookup"><span data-stu-id="fe4d1-117">A left outer join is a join that returns each element of the first (left) collection, even if it has no correlated elements in the second collection.</span></span>  
  
 <span data-ttu-id="fe4d1-118">Дополнительные сведения о объединениях см. в разделе [операции JOIN](/previous-versions/visualstudio/visual-studio-2013/bb397908(v=vs.120)).</span><span class="sxs-lookup"><span data-stu-id="fe4d1-118">For more information about joins, see [Join Operations](/previous-versions/visualstudio/visual-studio-2013/bb397908(v=vs.120)).</span></span>
