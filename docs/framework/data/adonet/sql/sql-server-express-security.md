<span data-ttu-id="063d7-103">Выпуск Microsoft SQL Server Express (SQL Server Express) основан на версии Microsoft SQL Server и поддерживает большинство средств ядра базы данных этой версии.</span><span class="sxs-lookup"><span data-stu-id="063d7-103">Microsoft SQL Server Express Edition (SQL Server Express) is based on Microsoft SQL Server, and supports most of the features of the database engine.</span></span> <span data-ttu-id="063d7-104">Этот выпуск спроектирован так, что несущественные возможности и средства обеспечения сетевого взаимодействия отменены по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="063d7-104">It is designed so that nonessential features and network connectivity are off by default.</span></span> <span data-ttu-id="063d7-105">Благодаря этому уменьшается контактная зона, доступная для атаки со стороны злонамеренного пользователя.</span><span class="sxs-lookup"><span data-stu-id="063d7-105">This reduces the surface area available for attack by a malicious user.</span></span>  
  
 <span data-ttu-id="063d7-106">Программное обеспечение SQL Server Express обычно устанавливается в виде именованного экземпляра.</span><span class="sxs-lookup"><span data-stu-id="063d7-106">SQL Server Express is usually installed as a named instance.</span></span> <span data-ttu-id="063d7-107">Стандартное имя экземпляра - `SQLExpress`.</span><span class="sxs-lookup"><span data-stu-id="063d7-107">The default name of the instance is `SQLExpress`.</span></span> <span data-ttu-id="063d7-108">Именованный экземпляр определяется по сетевому имени компьютера, а также по имени экземпляра, которое указывается во время установки.</span><span class="sxs-lookup"><span data-stu-id="063d7-108">A named instance is identified by the network name of the computer plus the instance name that you specify during installation.</span></span>  
  
## <a name="network-access"></a><span data-ttu-id="063d7-109">Сетевой доступ</span><span class="sxs-lookup"><span data-stu-id="063d7-109">Network Access</span></span>  
 <span data-ttu-id="063d7-110">По соображениям безопасности в выпуске SQL Server Express применение сетевых протоколов запрещено по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="063d7-110">For security reasons, networking protocols are disabled by default in SQL Server Express.</span></span> <span data-ttu-id="063d7-111">Это исключает возможность атаки со стороны внешних пользователей, которые могли бы поставить под угрозу компьютер, применяемый для эксплуатации экземпляра SQL Server Express.</span><span class="sxs-lookup"><span data-stu-id="063d7-111">This prevents attacks from outside users that might compromise the computer that hosts the instance of SQL Server Express.</span></span> <span data-ttu-id="063d7-112">Необходимо явно разрешить использование средств сетевого взаимодействия и запустить службу браузера SQL Server для подключения к экземпляру SQL Server Express с другого компьютера.</span><span class="sxs-lookup"><span data-stu-id="063d7-112">You must explicitly enable network connectivity and start the SQL Server Browser service to connect to a SQL Server Express instance from another computer.</span></span>  
  
 <span data-ttu-id="063d7-113">После ввода в действие средств сетевого взаимодействия необходимо соблюдать по отношению к экземпляру SQL Server Express такие же требования к безопасности, как и при эксплуатации других выпусков SQL Server.</span><span class="sxs-lookup"><span data-stu-id="063d7-113">Once network connectivity is enabled, a SQL Server Express instance has the same security requirements as the other editions of SQL Server.</span></span>  
  
## <a name="user-instances"></a><span data-ttu-id="063d7-114">Пользовательские экземпляры</span><span class="sxs-lookup"><span data-stu-id="063d7-114">User Instances</span></span>  
 <span data-ttu-id="063d7-115">Пользовательский экземпляр - это отдельный экземпляр ядра базы данных SQL Server Express, который создается родительским экземпляром SQL Server Express.</span><span class="sxs-lookup"><span data-stu-id="063d7-115">A user instance is a separate instance of the SQL Server Express database engine that is generated by a parent instance of SQL Server Express.</span></span> <span data-ttu-id="063d7-116">Основное назначение пользовательского экземпляра состоит в том, чтобы дать возможность пользователям, эксплуатирующим операционную систему Windows под управлением учетной записи пользователя с минимальными разрешениями, получить права системного администратора (`sysadmin`) по отношению к экземпляру SQL Server Express на своем локальном компьютере.</span><span class="sxs-lookup"><span data-stu-id="063d7-116">The primary goal of a user instance is to allow users who are running Windows under a least-privilege user account to have system administrator (`sysadmin`) privileges on the SQL Server Express instance on their local computer.</span></span> <span data-ttu-id="063d7-117">Пользовательские экземпляры не предназначены для пользователей, которые имеют права системных администраторов на своих собственных компьютерах.</span><span class="sxs-lookup"><span data-stu-id="063d7-117">User instances are not intended for users who are system administrators on their own computers.</span></span>  
  
 <span data-ttu-id="063d7-118">Пользовательский экземпляр создается на основе исходного экземпляра SQL Server или SQL Server Express от имени пользователя.</span><span class="sxs-lookup"><span data-stu-id="063d7-118">A user instance is generated from a primary instance of SQL Server or SQL Server Express on behalf of a user.</span></span> <span data-ttu-id="063d7-119">Он функционирует как пользовательский процесс в контексте безопасности Windows самого пользователя, а не службы.</span><span class="sxs-lookup"><span data-stu-id="063d7-119">It runs as a user process under the Windows security context of the user, not as a service.</span></span> <span data-ttu-id="063d7-120">Применение имен входа SQL Server не разрешено; поддерживаются только имена входа Windows.</span><span class="sxs-lookup"><span data-stu-id="063d7-120">SQL Server logins are disallowed; only Windows logins are supported.</span></span> <span data-ttu-id="063d7-121">Благодаря этому исключается возможность вносить с помощью программного обеспечения, применяемого в рамках пользовательского экземпляра, такие общесистемные изменения, разрешения на выполнение которых пользователь не имеет.</span><span class="sxs-lookup"><span data-stu-id="063d7-121">This prevents software executing on a user instance from making system-wide changes that the user would not have permissions to make.</span></span> <span data-ttu-id="063d7-122">Пользовательский экземпляр именуется также дочерним или клиентским экземпляром, а иногда обозначается с помощью сокращения RANU (run as normal user - эксплуатируемый с правами обычного пользователя).</span><span class="sxs-lookup"><span data-stu-id="063d7-122">A user instance is also known as a child or client instance, and is sometimes referred to by using the RANU acronym ("run as normal user").</span></span>  
  
 <span data-ttu-id="063d7-123">Каждый пользовательский экземпляр изолирован от своего родительского экземпляра и от других пользовательских экземпляров, работающих на том же компьютере.</span><span class="sxs-lookup"><span data-stu-id="063d7-123">Each user instance is isolated from its parent instance and from other user instances running on the same computer.</span></span> <span data-ttu-id="063d7-124">Базы данных, установленные для пользовательских экземпляров, открываются только в однопользовательском режиме; возможность подключения к ним нескольких пользователей отсутствует.</span><span class="sxs-lookup"><span data-stu-id="063d7-124">Databases installed on user instances are opened in single-user mode only; multiple users cannot connect to them.</span></span> <span data-ttu-id="063d7-125">Репликация, распределенные запросы и удаленные соединения для пользовательских экземпляров не разрешены.</span><span class="sxs-lookup"><span data-stu-id="063d7-125">Replication, distributed queries and remote connections are disabled for user instances.</span></span> <span data-ttu-id="063d7-126">Пользователи, подключенные к пользовательскому экземпляру, не имеют каких-либо особых прав по отношению к родительскому экземпляру SQL Server Express.</span><span class="sxs-lookup"><span data-stu-id="063d7-126">When connected to a user instance, users do not have any special privileges on the parent SQL Server Express instance.</span></span>
