<span data-ttu-id="7110f-103">SQL Server содержит функции шифрования и расшифровки данных с помощью сертификата и асимметричного или симметричного ключа.</span><span class="sxs-lookup"><span data-stu-id="7110f-103">SQL Server provides functions to encrypt and decrypt data using a certificate, asymmetric key, or symmetric key.</span></span> <span data-ttu-id="7110f-104">Все они содержатся во внутреннем хранилище сертификатов.</span><span class="sxs-lookup"><span data-stu-id="7110f-104">It manages all of these in an internal certificate store.</span></span> <span data-ttu-id="7110f-105">Хранилище использует иерархию шифрования, обеспечивающую безопасность сертификатов и ключей на уровне, находящемся выше в иерархии.</span><span class="sxs-lookup"><span data-stu-id="7110f-105">The store uses an encryption hierarchy that secures certificates and keys at one level with the layer above it in the hierarchy.</span></span> <span data-ttu-id="7110f-106">Эта область возможностей SQL Server называется секретным хранилищем.</span><span class="sxs-lookup"><span data-stu-id="7110f-106">This feature area of SQL Server is called Secret Storage.</span></span>  
  
 <span data-ttu-id="7110f-107">Самым быстрым режимом шифрования, поддерживаемым функциями шифрования, является шифрование с помощью симметричного ключа.</span><span class="sxs-lookup"><span data-stu-id="7110f-107">The fastest mode of encryption supported by the encryption functions is symmetric key encryption.</span></span> <span data-ttu-id="7110f-108">Этот режим подходит для управления большими томами данных.</span><span class="sxs-lookup"><span data-stu-id="7110f-108">This mode is suitable for handling large volumes of data.</span></span> <span data-ttu-id="7110f-109">Симметричные ключи могут быть зашифрованы сертификатами, паролями или другими симметричными ключами.</span><span class="sxs-lookup"><span data-stu-id="7110f-109">The symmetric keys can be encrypted by certificates, passwords or other symmetric keys.</span></span>  
  
## <a name="keys-and-algorithms"></a><span data-ttu-id="7110f-110">Ключи и алгоритмы</span><span class="sxs-lookup"><span data-stu-id="7110f-110">Keys and Algorithms</span></span>  

 <span data-ttu-id="7110f-111">SQL Server поддерживает несколько алгоритмов шифрования симметричным ключом, включая DES, Triple DES, RC2, RC4, 128-разрядный RC4, DESX, 128-разрядный AES, 192-разрядный AES и 256-разрядный AES.</span><span class="sxs-lookup"><span data-stu-id="7110f-111">SQL Server supports several symmetric key encryption algorithms, including DES, Triple DES, RC2, RC4, 128-bit RC4, DESX, 128-bit AES, 192-bit AES, and 256-bit AES.</span></span> <span data-ttu-id="7110f-112">Алгоритмы реализуются с помощью API-интерфейса Windows Crypto.</span><span class="sxs-lookup"><span data-stu-id="7110f-112">The algorithms are implemented using the Windows Crypto API.</span></span>  
  
 <span data-ttu-id="7110f-113">В пределах соединения с базой данных SQL Server может поддерживать несколько открытых симметричных ключей.</span><span class="sxs-lookup"><span data-stu-id="7110f-113">Within the scope of a database connection, SQL Server can maintain multiple open symmetric keys.</span></span> <span data-ttu-id="7110f-114">Открытый ключ получается из хранилища и доступен для расшифровки данных.</span><span class="sxs-lookup"><span data-stu-id="7110f-114">An open key is retrieved from the store and is available for decrypting data.</span></span> <span data-ttu-id="7110f-115">Не нужно указывать, какой симметричный ключ будет использоваться для расшифровки фрагмента данных.</span><span class="sxs-lookup"><span data-stu-id="7110f-115">When a piece of data is decrypted, there is no need to specify the symmetric key to use.</span></span> <span data-ttu-id="7110f-116">Каждое зашифрованное значение содержит идентификатор ключа (идентификатор GUID ключа), использованного для его шифрования.</span><span class="sxs-lookup"><span data-stu-id="7110f-116">Each encrypted value contains the key identifier (key GUID) of the key used to encrypt it.</span></span> <span data-ttu-id="7110f-117">Если расшифрован и открыт верный ключ, то ядро сопоставляет зашифрованный поток байтов с открытым симметричным ключом.</span><span class="sxs-lookup"><span data-stu-id="7110f-117">The engine matches the encrypted byte stream to an open symmetric key, if the correct key has been decrypted and is open.</span></span> <span data-ttu-id="7110f-118">Этот ключ затем используется для выполнения расшифровки и возвращения данных.</span><span class="sxs-lookup"><span data-stu-id="7110f-118">This key is then used to perform decryption and return the data.</span></span> <span data-ttu-id="7110f-119">Если верный ключ не открыт, возвращается значение NULL.</span><span class="sxs-lookup"><span data-stu-id="7110f-119">If the correct key is not open, NULL is returned.</span></span>  
  
 <span data-ttu-id="7110f-120">Пример, демонстрирующий работу с зашифрованными данными в базе данных, см. в разделе [Шифрование столбца данных](/sql/relational-databases/security/encryption/encrypt-a-column-of-data).</span><span class="sxs-lookup"><span data-stu-id="7110f-120">For an example that shows how to work with encrypted data in a database, see [Encrypt a Column of Data](/sql/relational-databases/security/encryption/encrypt-a-column-of-data).</span></span>
