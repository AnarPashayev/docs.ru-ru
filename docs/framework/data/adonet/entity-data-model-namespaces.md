# <a name="entity-data-model-namespaces"></a><span data-ttu-id="a7f7e-102">EDM (модель данных с использованием сущностей): Пространства имен</span><span class="sxs-lookup"><span data-stu-id="a7f7e-102">Entity Data Model: Namespaces</span></span>
<span data-ttu-id="a7f7e-103">Пространство имен в Entity Data Model (EDM) — это абстрактный контейнер для [типы сущностей](../../../../docs/framework/data/adonet/entity-type.md), [сложные типы](../../../../docs/framework/data/adonet/complex-type.md), и [ассоциации](../../../../docs/framework/data/adonet/association-type.md).</span><span class="sxs-lookup"><span data-stu-id="a7f7e-103">A namespace in the Entity Data Model (EDM) is an abstract container for [entity types](../../../../docs/framework/data/adonet/entity-type.md), [complex types](../../../../docs/framework/data/adonet/complex-type.md), and [associations](../../../../docs/framework/data/adonet/association-type.md).</span></span> <span data-ttu-id="a7f7e-104">Пространства имен в модели EDM схожи с пространствами имен в языке программирования: они обеспечивают контекст для объектов, которые они содержат, а также являются способом устранения неопределенности в отношении объектов, которые имеют одно и то же имя (но содержатся в разных пространствах имен).</span><span class="sxs-lookup"><span data-stu-id="a7f7e-104">Namespaces in the EDM are similar to namespaces in a programming language: they provide context for the objects that they contain and they provide a way to disambiguate objects that have the same name (but are contained in different namespaces).</span></span>
