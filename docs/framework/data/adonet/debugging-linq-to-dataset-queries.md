# <a name="debugging-linq-to-dataset-queries"></a>Отладка запросов в LINQ to DataSet

Visual Studio поддерживает отладку LINQ для набора данных. Однако существуют некоторые отличия между отладкой LINQ для набора данных и не LINQ to DataSet управляемого кода. Большинство возможностей отладки работают с LINQ для инструкций набора данных, включая пошаговое выполнение, установку точек останова и просмотр результатов, отображаемых в окнах отладчика. Тем не менее отложенное выполнение запроса в имеет некоторые побочные эффекты, которые следует учитывать при выполнении отладки LINQ код набора данных, и существуют некоторые ограничения на использование, изменить и продолжить. В этом разделе обсуждаются аспекты отладки, уникальные для LINQ to DataSet по сравнению с не LINQ to DataSet управляемого кода.  
  
## <a name="viewing-results"></a>Просмотр результатов  
 Результат LINQ to DataSet инструкции можно просмотреть с помощью подсказки, окно контрольных значений и диалоговом окне "Быстрая проверка". В окне исходного текста можно остановить указатель на запросе, после чего появится элемент DataTip. Можно скопировать LINQ переменная набора данных и вставьте его в окно просмотра или диалоговое окно "Быстрая проверка". В LINQ to DataSet запроса вычисляется не при его создании или объявлении, но только в том случае, если выполняется запрос. Это называется *отложенного выполнения*. Поэтому переменная запроса не имеет значения, пока запрос не выполнен. Дополнительные сведения см. в разделе [запросов в LINQ to DataSet](../../../../docs/framework/data/adonet/queries-in-linq-to-dataset.md).  
  
 Отладчик должен выполнить запрос, чтобы отобразить его результаты. Скрытое выполнение происходит при просмотре LINQ для результатов запроса набора данных в отладчике, и он имеет некоторые эффекты, которые следует учитывать. Каждое вычисление запроса занимает время. Разворачивание узла результатов занимает время. В некоторых запросах повторные вычисления могу вызвать заметное снижение производительности. Вычисление запроса может также вызвать побочные эффекты, которые изменяют значение данных или состояние программы. Не все запросы имеют побочные эффекты. Чтобы определить, может ли запрос быть выполнен безопасно, без побочных эффектов, нужно понимать программный код, которым реализован запрос. Дополнительные сведения см. в разделе [побочные эффекты и выражения](https://docs.microsoft.com/previous-versions/visualstudio/visual-studio-2013/a7a250bs(v=vs.120)).  
  
## <a name="edit-and-continue"></a>Изменить и продолжить  
 Изменить и продолжить не поддерживает изменения в LINQ для запросов к наборам данных. Если добавить, удалить или изменить LINQ к инструкции набора данных во время сеанса отладки, появится диалоговое окно о том, что изменение не поддерживается в режиме Edit и Continue. На этом этапе можно либо отменить изменения, либо остановить сеанс отладки и запустить новый сеанс с отредактированным кодом.  
  
 Кроме того, изменить и продолжить не поддерживает изменение типа или значения переменной, которая используется в LINQ to DataSet инструкции. Снова можно либо отменить изменения либо остановить и перезапустить сеанс отладки.  
  
 В визуальном элементе C# в Visual Studio нельзя использовать изменить и продолжить с любым кодом в методе, содержащем LINQ запрос к набору данных.  
  
 В Visual Basic в Visual Studio, вы можно изменить и продолжить использовать на не-кода LINQ to DataSet, даже в методе, содержащем LINQ запрос к набору данных. Можно добавить или удалить код перед LINQ инструкции набора данных, даже если изменения затронут количество строк в LINQ запрос к набору данных. Visual Basic отладки для не-кода LINQ to DataSet остается неизменным, так как она была до введения LINQ to DataSet. Невозможно изменить, добавить или удалить LINQ запрос к набору данных, тем не менее, в том случае, если вы не Остановите отладку, чтобы применить изменения.
