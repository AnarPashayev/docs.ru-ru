 <span data-ttu-id="6319d-109">Следующий запрос возвращает список продуктов и коллекцию количеств в строках заказа для каждого продукта:</span><span class="sxs-lookup"><span data-stu-id="6319d-109">The following query produces a list of products and a collection of order line quantities per each product:</span></span>

  group by ol.Product as p

  group by ol.Product as p  
```  
  
 <span data-ttu-id="6319d-111">Оператор `GROUPPARTITION` может использоваться вместе с определяемыми пользователем агрегатная функциями.</span><span class="sxs-lookup"><span data-stu-id="6319d-111">The `GROUPPARTITION` operator can be used in conjunction with user-defined aggregate functions.</span></span>

```  
  
 <span data-ttu-id="6319d-115">При использовании обычного предложения GROUP BY результаты группирования скрыты.</span><span class="sxs-lookup"><span data-stu-id="6319d-115">With a regular GROUP BY, the results of the grouping are hidden.</span></span> <span data-ttu-id="6319d-116">Эти результаты можно использовать только в агрегатной функции.</span><span class="sxs-lookup"><span data-stu-id="6319d-116">You can only use the results in an aggregate function.</span></span> <span data-ttu-id="6319d-117">Чтобы можно было видеть результаты группирования, необходимо сопоставить результаты группирования и входной набор с использованием вложенного запроса.</span><span class="sxs-lookup"><span data-stu-id="6319d-117">In order to see the results of the grouping, you have to correlate the results of the grouping and the input set by using a subquery.</span></span> <span data-ttu-id="6319d-118">Следующие два запроса эквивалентны.</span><span class="sxs-lookup"><span data-stu-id="6319d-118">The following two queries are equivalent:</span></span>

 <span data-ttu-id="6319d-119">Как показывает этот пример, применение статистического оператора GROUPPARTITION позволяет упростить получение ссылки на входной набор после группирования.</span><span class="sxs-lookup"><span data-stu-id="6319d-119">As seen from the example, the GROUPPARTITION aggregate operator makes it easier to get a reference to the input set after the grouping.</span></span>  
  
 <span data-ttu-id="6319d-120">В операторе GROUPPARTITION на входе может указываться любое выражение [!INCLUDE[esql](../../../../../../includes/esql-md.md)] , если используется параметр `expression` .</span><span class="sxs-lookup"><span data-stu-id="6319d-120">The GROUPPARTITION operator can specify any [!INCLUDE[esql](../../../../../../includes/esql-md.md)] expression in the operator input when you use the `expression` parameter.</span></span>  
  
 <span data-ttu-id="6319d-121">Например, все следующие входные выражения для секции группы являются допустимыми:</span><span class="sxs-lookup"><span data-stu-id="6319d-121">For instance all of the following input expressions to the group partition are valid:</span></span>
