 <span data-ttu-id="f469b-108">Операция запроса LINQ состоит из трех действий: получение одного или нескольких источников данных, создание запроса и его выполнение.</span><span class="sxs-lookup"><span data-stu-id="f469b-108">A LINQ query operation consists of three actions: obtain the data source or sources, create the query, and execute the query.</span></span>  
  
 <span data-ttu-id="f469b-109">К источникам данных, реализующим общий интерфейс <xref:System.Collections.Generic.IEnumerable%601> или <xref:System.Linq.IQueryable%601>, можно выполнять запрос с помощью LINQ.</span><span class="sxs-lookup"><span data-stu-id="f469b-109">Data sources that implement the <xref:System.Collections.Generic.IEnumerable%601> generic interface or the <xref:System.Linq.IQueryable%601> generic interface can be queried through LINQ.</span></span> <span data-ttu-id="f469b-110">Экземпляры универсального <xref:System.Data.Objects.ObjectQuery%601> класс, который реализует универсальный интерфейс <xref:System.Linq.IQueryable%601> интерфейсом, служат в качестве источника данных для [!INCLUDE[linq_entities](../../../../../../includes/linq-entities-md.md)] запросов.</span><span class="sxs-lookup"><span data-stu-id="f469b-110">Instances of the generic <xref:System.Data.Objects.ObjectQuery%601> class, which implements the generic <xref:System.Linq.IQueryable%601> interface, serve as the data source for [!INCLUDE[linq_entities](../../../../../../includes/linq-entities-md.md)] queries.</span></span> <span data-ttu-id="f469b-111">Универсальный класс <xref:System.Data.Objects.ObjectQuery%601> представляет запрос, который возвращает коллекцию, включающую ноль или более типизированных объектов.</span><span class="sxs-lookup"><span data-stu-id="f469b-111">The <xref:System.Data.Objects.ObjectQuery%601> generic class represents a query that returns a collection of zero or more typed objects.</span></span> <span data-ttu-id="f469b-112">Можно также разрешить компилятору вывести тип сущности с помощью ключевого слова C# `var` (Dim в Visual Basic).</span><span class="sxs-lookup"><span data-stu-id="f469b-112">You can also let the compiler infer the type of an entity by using the C# keyword `var` (Dim in Visual Basic).</span></span>  
  
 <span data-ttu-id="f469b-113">В запросе указываются данные, которые необходимо получить из источника данных.</span><span class="sxs-lookup"><span data-stu-id="f469b-113">In the query, you specify exactly the information that you want to retrieve from the data source.</span></span> <span data-ttu-id="f469b-114">В запросе можно также указать, как следует сортировать, группировать и формировать возвращаемую информацию.</span><span class="sxs-lookup"><span data-stu-id="f469b-114">A query can also specify how that information should be sorted, grouped, and shaped before it is returned.</span></span> <span data-ttu-id="f469b-115">В LINQ запрос хранится в переменной.</span><span class="sxs-lookup"><span data-stu-id="f469b-115">In LINQ, a query is stored in a variable.</span></span> <span data-ttu-id="f469b-116">Если запрос возвращает последовательность значений, то переменная запроса должна иметь тип данных, который может быть запрошен.</span><span class="sxs-lookup"><span data-stu-id="f469b-116">If the query returns a sequence of values, the query variable itself must be a queryable type.</span></span> <span data-ttu-id="f469b-117">Эта переменная запроса не выполняет никаких действий и не возвращает данные. Она только хранит информацию о запросе.</span><span class="sxs-lookup"><span data-stu-id="f469b-117">This query variable takes no action and returns no data; it only stores the query information.</span></span> <span data-ttu-id="f469b-118">После создания запроса его необходимо выполнить, чтобы получить данные.</span><span class="sxs-lookup"><span data-stu-id="f469b-118">After you create a query you must execute that query to retrieve any data.</span></span>  
  
## <a name="query-syntax"></a><span data-ttu-id="f469b-119">Синтаксис запроса</span><span class="sxs-lookup"><span data-stu-id="f469b-119">Query Syntax</span></span>  
 [!INCLUDE[linq_entities](../../../../../../includes/linq-entities-md.md)] <span data-ttu-id="f469b-120">запросы могут создаваться два разных синтаксиса: синтаксис выражений запросов и синтаксис запросов на основе методов.</span><span class="sxs-lookup"><span data-stu-id="f469b-120">queries can be composed in two different syntaxes: query expression syntax and method-based query syntax.</span></span> <span data-ttu-id="f469b-121">Синтаксис выражения запроса появился в C# 3.0 и Visual Basic 9.0 и состоит из набора предложений, написанных в декларативном синтаксисе, аналогичном Transact-SQL или XQuery.</span><span class="sxs-lookup"><span data-stu-id="f469b-121">Query expression syntax is new in C# 3.0 and Visual Basic 9.0, and it consists of a set of clauses written in a declarative syntax similar to Transact-SQL or XQuery.</span></span> <span data-ttu-id="f469b-122">Тем не менее среда CLR платформы [!INCLUDE[dnprdnshort](../../../../../../includes/dnprdnshort-md.md)] не может прочитать выражение запроса сама.</span><span class="sxs-lookup"><span data-stu-id="f469b-122">However, the [!INCLUDE[dnprdnshort](../../../../../../includes/dnprdnshort-md.md)] common language runtime (CLR) cannot read the query expression syntax itself.</span></span> <span data-ttu-id="f469b-123">Таким образом, во время компиляции выражения запроса преобразуются в то, что понятно CLR -
