# <a name="query-results"></a><span data-ttu-id="e22e1-102">Результаты запроса</span><span class="sxs-lookup"><span data-stu-id="e22e1-102">Query Results</span></span>
<span data-ttu-id="e22e1-103">После запроса LINQ to Entities преобразуются в деревья команд и выполнена, результаты запроса обычно возвращаются в одной из следующих:</span><span class="sxs-lookup"><span data-stu-id="e22e1-103">After a LINQ to Entities query is converted to command trees and executed, the query results are usually returned as one of the following:</span></span>  
  
- <span data-ttu-id="e22e1-104">Коллекция из нуля или большего числа типизированных объектов сущностей или проекция сложных типов в концептуальной модели.</span><span class="sxs-lookup"><span data-stu-id="e22e1-104">A collection of zero or more typed entity objects or a projection of complex types in the conceptual model.</span></span>  
  
- <span data-ttu-id="e22e1-105">Типы CLR, поддерживаемые концептуальной моделью.</span><span class="sxs-lookup"><span data-stu-id="e22e1-105">CLR types supported by the conceptual model.</span></span>  
  
- <span data-ttu-id="e22e1-106">Встроенные коллекции.</span><span class="sxs-lookup"><span data-stu-id="e22e1-106">Inline collections.</span></span>  
  
- <span data-ttu-id="e22e1-107">Анонимные типы.</span><span class="sxs-lookup"><span data-stu-id="e22e1-107">Anonymous types.</span></span>  
  
 <span data-ttu-id="e22e1-108">При выполнении запроса к источнику данных результаты материализуются в типы CLR и возвращаются клиенту.</span><span class="sxs-lookup"><span data-stu-id="e22e1-108">When the query has executed against the data source, the results are materialized into CLR types and returned to the client.</span></span> <span data-ttu-id="e22e1-109">Материализация объектов полностью обеспечивается платформой [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span><span class="sxs-lookup"><span data-stu-id="e22e1-109">All object materialization is performed by the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="e22e1-110">Любые ошибки, вызванные невозможностью сопоставления объектов платформы [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] и среды CLR, вызовут исключения во время материализации объекта.</span><span class="sxs-lookup"><span data-stu-id="e22e1-110">Any errors that result from an inability to map between the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] and the CLR will cause exceptions to be thrown during object materialization.</span></span>  
  
 <span data-ttu-id="e22e1-111">Если в результате выполнения запроса возвращаются типы-примитивы концептуальной модели, то результаты состоят из изолированных типов CLR, которые не связаны с платформой [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span><span class="sxs-lookup"><span data-stu-id="e22e1-111">If the query execution returns primitive conceptual model types, the запрос вернул коллекцию типизированных объектов сущностей, представленных <xref:System.Data.Objects.ObjectQuery%601>, то эти типы отслеживаются контекстом объекта.</span><span class="sxs-lookup"><span data-stu-id="e22e1-112">However, if the query returns a collection of typed entity objects, represented by <xref:System.Data.Objects.ObjectQuery%601>, those types are tracked by the object context.</span></span> <span data-ttu-id="e22e1-113">Все поведение объекта (например, дочерние и родительские коллекции, отслеживание изменений, полиморфизм и т. д.), как определено в [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span><span class="sxs-lookup"><span data-stu-id="e22e1-113">All object behavior (such as child/parent collections, change tracking, polymorphism, and so on) are as defined in the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="e22e1-114">Эти функции можно могут быть использованы в пределах, определяемых [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span><span class="sxs-lookup"><span data-stu-id="e22e1-114">This functionality can be used in its capacity, as defined in the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="e22e1-115">Дополнительные сведения см. в разделе [работа с объектами](../../../../../../docs/framework/data/adonet/ef/working-with-objects.md).</span><span class="sxs-lookup"><span data-stu-id="e22e1-115">For more information, see [Working with Objects](../../../../../../docs/framework/data/adonet/ef/working-with-objects.md).</span></span>  
  
 <span data-ttu-id="e22e1-116">Типы структур, возвращаемые из запросов (например, анонимные и сложные типы, допускающие значение null), могут иметь значение `null`.</span><span class="sxs-lookup"><span data-stu-id="e22e1-116">Struct types returned from queries (such as anonymous types and nullable complex types) can be of `null` value.</span></span> <span data-ttu-id="e22e1-117">Свойство <xref:System.Data.Objects.DataClasses.EntityCollection%601> возвращаемой сущности также может иметь значение `null`.</span><span class="sxs-lookup"><span data-stu-id="e22e1-117">An <xref:System.Data.Objects.DataClasses.EntityCollection%601> property of a returned entity can also be of `null` value.</span></span> <span data-ttu-id="e22e1-118">Это может произойти в результате проецирования свойства коллекции для сущности, имеющей значение `null`, например, при вызове метода <xref:System.Linq.Queryable.FirstOrDefault%2A> для объекта <xref:System.Data.Objects.ObjectQuery%601>, который не содержит элементов.</span><span class="sxs-lookup"><span data-stu-id="e22e1-118">This can result from projecting the collection property of an entity that is of `null` value, such as calling <xref:System.Linq.Queryable.FirstOrDefault%2A> on an <xref:System.Data.Objects.ObjectQuery%601> that has no elements.</span></span>  
  
 <span data-ttu-id="e22e1-119">В некоторых ситуациях может показаться, что запрос создает материализованный результат во время выполнения, но в действительности запрос будет выполняться на сервере и объект сущности никогда не окажется материализован в среде CLR.</span><span class="sxs-lookup"><span data-stu-id="e22e1-119">In certain situations, a query might appear to generate a materialized result during its execution, but the query will be executed on the server and the entity object will never be materialized in the CLR.</span></span> <span data-ttu-id="e22e1-120">Это может вызвать проблемы, если приложение рассчитывает на побочные эффекты материализации.</span><span class="sxs-lookup"><span data-stu-id="e22e1-120">This can cause problems if you are depending on side effects of object materialization.</span></span>  
  
 <span data-ttu-id="e22e1-121">В следующем примере имеется пользовательский класс `MyContact` со свойством `LastName`.</span><span class="sxs-lookup"><span data-stu-id="e22e1-121">The following example contains a custom class, `MyContact`, with a `LastName` property.</span></span> <span data-ttu-id="e22e1-122">При присваивании значения свойству `LastName` переменная `count` увеличивается на единицу.</span><span class="sxs-lookup"><span data-stu-id="e22e1-122">When the `LastName` property is set, a `count` variable is incremented.</span></span> <span data-ttu-id="e22e1-123">Если выполнить два приведенных ниже запроса, то первый запрос увеличит значение `count` на единицу, а второй - нет.</span><span class="sxs-lookup"><span data-stu-id="e22e1-123">If you execute the two following queries, the first query will increment `count` while the second query will not.</span></span> <span data-ttu-id="e22e1-124">Причина этого заключается в том, что во втором запросе свойство `LastName` спроецировано из результатов, а класс `MyContact` не создается, поскольку он не является необходимым для выполнения запроса в хранилище.</span><span class="sxs-lookup"><span data-stu-id="e22e1-124">This is because in the second query the `LastName` property is projected from the results and the `MyContact` class is never created, because it is not required to execute the query on the store.</span></span>
