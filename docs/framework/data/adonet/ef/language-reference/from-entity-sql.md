 `FROM (C AS c JOIN D AS d) CROSS APPLY c.Names AS e`  
  
## <a name="left-correlation"></a><span data-ttu-id="f67e2-166">Левая корреляция</span><span class="sxs-lookup"><span data-stu-id="f67e2-166">Left Correlation</span></span>  
 <span data-ttu-id="f67e2-167">Элементы в предложении `FROM` могут ссылаться на элементы, указанные в более ранних предложениях.</span><span class="sxs-lookup"><span data-stu-id="f67e2-167">Items in the `FROM` clause can refer to items specified in earlier clauses.</span></span> <span data-ttu-id="f67e2-168">В следующем примере `C` и `D` являются независимыми коллекциями, но `c.Names` зависит от `C`:</span><span class="sxs-lookup"><span data-stu-id="f67e2-168">In the following example, `C` and `D` are independent collections, but `c.Names` is dependent on `C`:</span></span>

from C as c, D as d, c.Names as e  
```  
  
 <span data-ttu-id="f67e2-169">Это логически равносильно:</span><span class="sxs-lookup"><span data-stu-id="f67e2-169">This is logically equivalent to:</span></span>

from (C as c join D as d) cross apply c.Names as e

## <a name="semantics"></a><span data-ttu-id="f67e2-170">Семантика</span><span class="sxs-lookup"><span data-stu-id="f67e2-170">Semantics</span></span>  
 <span data-ttu-id="f67e2-171">Логически предполагается, что коллекции в предложении `FROM` - часть `n`-стороннего перекрестного соединения (за исключением случая 1-стороннего перекрестного соединения).</span><span class="sxs-lookup"><span data-stu-id="f67e2-171">Logically, the collections in the `FROM` clause are assumed to be part of an `n`-way cross join (except in the case of a 1-way cross join).</span></span> <span data-ttu-id="f67e2-172">Псевдонимы в предложении `FROM` обрабатываются слева направо и добавляются в текущую область для последующего применения.</span><span class="sxs-lookup"><span data-stu-id="f67e2-172">Aliases in the `FROM` clause are processed left to right, and are added to the current scope for later reference.</span></span> <span data-ttu-id="f67e2-173">Предполагается, что предложение `FROM` формирует мультинабор строк.</span><span class="sxs-lookup"><span data-stu-id="f67e2-173">The `FROM` clause is assumed to produce a multiset of rows.</span></span> <span data-ttu-id="f67e2-174">В предложении `FROM` будет одно поле для каждого элемента, которое представляет единственный элемент из этого элемента сбора.</span><span class="sxs-lookup"><span data-stu-id="f67e2-174">There will be one field for each item in the `FROM` clause that represents a single element from that collection item.</span></span>  
  
 <span data-ttu-id="f67e2-175">Предложение `FROM` логически формирует мультинабор строк типа Row(c, d, e); при этом предполагается, что поля c, d и e являются элементами типа `C`, `D` и `c.Names`.</span><span class="sxs-lookup"><span data-stu-id="f67e2-175">The `FROM` clause logically produces a multiset of rows of type Row(c, d, e) where fields c, d, and e are assumed to be of the element type of `C`, `D`, and `c.Names`.</span></span>  
  
 [!INCLUDE[esql](../../../../../../includes/esql-md.md)] <span data-ttu-id="f67e2-176">появился псевдоним для каждого простого `FROM` элемент предложения в области видимости.</span><span class="sxs-lookup"><span data-stu-id="f67e2-176">introduces an alias for each simple `FROM` clause item in scope.</span></span> <span data-ttu-id="f67e2-177">Например, в следующем фрагменте предложения FROM в область введены имена c, d и e.</span><span class="sxs-lookup"><span data-stu-id="f67e2-177">For example, in the following FROM clause snippet, The names introduced into scope are c, d, and e.</span></span>

from (C as c join D as d) cross apply c.Names as e  
```  
  
 <span data-ttu-id="f67e2-178">В [!INCLUDE[esql](../../../../../../includes/esql-md.md)] (в отличие от [!INCLUDE[tsql](../../../../../../includes/tsql-md.md)]) предложение `FROM` только вводит псевдонимы в область.</span><span class="sxs-lookup"><span data-stu-id="f67e2-178">In [!INCLUDE[esql](../../../../../../includes/esql-md.md)] (unlike [!INCLUDE[tsql](../../../../../../includes/tsql-md.md)]), the `FROM` clause only introduces the aliases into scope.</span></span> <span data-ttu-id="f67e2-179">Любые обращения к столбцам (свойствам) этих коллекций нужно уточнить с помощью псевдонима.</span><span class="sxs-lookup"><span data-stu-id="f67e2-179">Any references to columns (properties) of these collections must be qualified with the alias.</span></span>
