<span data-ttu-id="4300d-103">Если приложение многократно выполняет похожие по структуре запросы на платформе Entity Framework, во многих случаях можно повысить производительность, скомпилировав запрос один раз, а затем выполняя его несколько раз с разными параметрами.</span><span class="sxs-lookup"><span data-stu-id="4300d-103">When you have an application that executes structurally similar queries many times in the Entity Framework, you can frequently increase performance by compiling the query one time and executing it several times with different parameters.</span></span> <span data-ttu-id="4300d-104">Например, приложению может понадобиться получить всех клиентов из определенного города; имя города указывается пользователем во время выполнения с помощью формы.</span><span class="sxs-lookup"><span data-stu-id="4300d-104">For example, an application might have to retrieve all the customers in a particular city; the city is specified at runtime by the user in a form.</span></span> <span data-ttu-id="4300d-105">Для этих целей технология LINQ to Entities поддерживает использование скомпилированных запросов.</span><span class="sxs-lookup"><span data-stu-id="4300d-105">LINQ to Entities supports using compiled queries for this purpose.</span></span>  
  
 <span data-ttu-id="4300d-106">Начиная с версии 4.5 платформы .NET Framework, запросы LINQ кэшируются автоматически.</span><span class="sxs-lookup"><span data-stu-id="4300d-106">Starting with the .NET Framework 4.5, LINQ queries are cached automatically.</span></span> <span data-ttu-id="4300d-107">Тем не менее можно использовать скомпилированные запросы LINQ для снижения затрат при последующем выполнении, и скомпилированные запросы могут быть более эффективными, чем запросы LINQ, которые автоматически сохраняются в кэше.</span><span class="sxs-lookup"><span data-stu-id="4300d-107">However, you can still use compiled LINQ queries to reduce this cost in later executions and compiled queries can be more efficient than LINQ queries that are automatically cached.</span></span> <span data-ttu-id="4300d-108">Обратите внимание, что запросы LINQ to Entities, которые применяют оператор `Enumerable.Contains` к коллекции в памяти, автоматически не кэшируются.</span><span class="sxs-lookup"><span data-stu-id="4300d-108">Note that LINQ to Entities queries that apply the `Enumerable.Contains` operator to in-memory collections are not automatically cached.</span></span> <span data-ttu-id="4300d-109">Также в скомпилированных запросах LINQ не допускаются коллекции в памяти с параметрами.</span><span class="sxs-lookup"><span data-stu-id="4300d-109">Also parameterizing in-memory collections in compiled LINQ queries is not allowed.</span></span>  
  
 <span data-ttu-id="4300d-110">Класс <xref:System.Data.Objects.CompiledQuery> обеспечивает компиляцию и кэширование запросов для повторного использования.</span><span class="sxs-lookup"><span только один раз во время первого выполнения.</span><span class="sxs-lookup"><span data-stu-id="4300d-114">The query compiles once during only the first execution.</span></span> <span data-ttu-id="4300d-115">Параметры слияния, которые заданы для запроса во время компиляции, далее не могут быть изменены.</span><span class="sxs-lookup"><span data-stu-id="4300d-115">The merge options set for the query at the time of the compilation cannot be changed later.</span></span> <span data-ttu-id="4300d-116">После компиляции запроса ему можно передавать только параметры примитивного типа,  <span data-ttu-id="4300d-118">Выражение запроса запроса LIN'to `Compile` Entities, которое компилирует метод `Func` <xref:System.Data.Objects.CompiledQuery>', <xref:System.Func%605>представлено одним из общих делегатов, например.</span><span class="sxs-lookup"><span data-stu-id="4300d-118">The LINQ to Entities query expression that the <xref:System.Data.Objects.CompiledQuery>'s `Compile` method compiles is represented by one of the generic `Func` delegates, such as <xref:System.Func%605>.</span></span> <span data-ttu-id="4300d-119">Выражение запроса может инкапсулировать не более одного параметра `ObjectContext`, одного возвращаемого параметра и 16 параметров запроса.</span><span class="sxs-lookup"><span data-stu-id="4300d-119">At most, the query expression can encapsulate an `ObjectContext` parameter, a return parameter, and 16 query parameters.</span></span> <span data-ttu-id="4300d-120">Если нужно больше 16 параметров запроса, то можно создать структуру, свойства которой будут соответствовать параметрам запроса.</span><span class="sxs-lookup"><span data-stu-id="4300d-120">If more than 16 query parameters are required, you can create a structure whose properties represent query parameters.</span></span> <span data-ttu-id="4300d-121">После задания свойств ими можно будет воспользоваться в выражении запроса из структуры.</span><span class="sxs-lookup"><span data-stu-id="4300d-121">You can then use the properties on the structure in the query expression after you set the properties.</span></span>
