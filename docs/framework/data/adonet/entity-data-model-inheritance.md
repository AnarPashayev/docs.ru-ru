# <a name="entity-data-model-inheritance"></a><span data-ttu-id="e90b1-102">Модель EDM: наследование</span><span class="sxs-lookup"><span data-stu-id="e90b1-102">Entity Data Model: Inheritance</span></span>
<span data-ttu-id="e90b1-103">EDM (модель EDM) поддерживает наследование для [типов сущностей](entity-type.md).</span><span class="sxs-lookup"><span data-stu-id="e90b1-103">The Entity Data Model (EDM) supports inheritance for [entity types](entity-type.md).</span></span> <span data-ttu-id="e90b1-104">Наследование в модели (EDM) схоже с наследованием для классов в языках объектно-ориентированного программирования.</span><span class="sxs-lookup"><span data-stu-id="e90b1-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="e90b1-105">Как и в случае с классами на объектно-ориентированных языках, в концептуальной модели можно определить тип сущности ( *производный тип*), наследующий от другого типа сущности ( *базовый тип*).</span><span class="sxs-lookup"><span data-stu-id="e90b1-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="e90b1-106">Однако в отличие от классов в объектно-ориентированном программировании в концептуальной модели производный тип всегда наследует все [Свойства](property.md) и [Свойства навигации](navigation-property.md) базового типа.</span><span class="sxs-lookup"><span data-stu-id="e90b1-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](property.md) and [navigation properties](navigation-property.md) of the base type.</span></span> <span data-ttu-id="e90b1-107">В производном типе нельзя отменить унаследованные свойства.</span><span class="sxs-lookup"><span data-stu-id="e90b1-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="e90b1-108">В концептуальной модели можно построить иерархию наследования, в которой один производный тип наследует свойства другого производного типа.</span><span class="sxs-lookup"><span data-stu-id="e90b1-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="e90b1-109">Тип в верхней части иерархии (один тип в иерархии, не являющийся производным типом) называется *корневым типом*.</span><span class="sxs-lookup"><span data-stu-id="e90b1-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="e90b1-110">В иерархии наследования [ключ сущности](entity-key.md) должен быть определен в корневом типе.</span><span class="sxs-lookup"><span data-stu-id="e90b1-110">In an inheritance hierarchy, the [entity key](entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="e90b1-111">Нельзя построить иерархии наследования, в которых один производный тип наследует свойства более чем одного типа.</span><span class="sxs-lookup"><span data-stu-id="e90b1-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="e90b1-112">Например, в концептуальной модели с типом сущности `Book` можно определить производные типы `FictionBook` и `NonFictionBook`, из которых каждый наследует свойства `Book`.</span><span class="sxs-lookup"><span data-stu-id="e90b1-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="e90b1-113">Тем не менее, в дальнейшем нельзя определить тип, который бы наследовал свойства обоих типов `FictionBook` и `NonFictionBook`.</span><span class="sxs-lookup"><span data-stu-id="e90b1-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
