# <a name="entity-data-model-inheritance"></a><span data-ttu-id="f5c38-102">EDM (модель данных с использованием сущностей): Наследование</span><span class="sxs-lookup"><span data-stu-id="f5c38-102">Entity Data Model: Inheritance</span></span>
<span data-ttu-id="f5c38-103">Entity Data Model (EDM) поддерживает наследование для [типы сущностей](../../../../docs/framework/data/adonet/entity-type.md).</span><span class="sxs-lookup"><span data-stu-id="f5c38-103">The Entity Data Model (EDM) supports inheritance for [entity types](../../../../docs/framework/data/adonet/entity-type.md).</span></span> <span data-ttu-id="f5c38-104">Наследование в модели (EDM) схоже с наследованием для классов в языках объектно-ориентированного программирования.</span><span class="sxs-lookup"><span data-stu-id="f5c38-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="f5c38-105">Как и с классами в объектно ориентированных языках, в концептуальной модели можно определить тип сущности ( *производный тип*), наследуемый от другого типа сущности ( *базовый тип*).</span><span class="sxs-lookup"><span data-stu-id="f5c38-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="f5c38-106">Тем не менее, в отличие от классов в объектно ориентированного программирования, в концептуальной модели производный тип всегда наследует все [свойства](../../../../docs/framework/data/adonet/property.md) и [свойства навигации](../../../../docs/framework/data/adonet/navigation-property.md) базового типа.</span><span class="sxs-lookup"><span data-stu-id="f5c38-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](../../../../docs/framework/data/adonet/property.md) and [navigation properties](../../../../docs/framework/data/adonet/navigation-property.md) of the base type.</span></span> <span data-ttu-id="f5c38-107">В производном типе нельзя отменить унаследованные свойства.</span><span class="sxs-lookup"><span data-stu-id="f5c38-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="f5c38-108">В концептуальной модели можно построить иерархию наследования, в которой один производный тип наследует свойства другого производного типа.</span><span class="sxs-lookup"><span data-stu-id="f5c38-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="f5c38-109">Тип в верхней части иерархии (одного типа в иерархии, который не является производным типом) называется *корневой тип*.</span><span class="sxs-lookup"><span data-stu-id="f5c38-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="f5c38-110">В иерархии наследования [ключ сущности](../../../../docs/framework/data/adonet/entity-key.md) должен быть определен в корневом типе.</span><span class="sxs-lookup"><span data-stu-id="f5c38-110">In an inheritance hierarchy, the [entity key](../../../../docs/framework/data/adonet/entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="f5c38-111">Нельзя построить иерархии наследования, в которых один производный тип наследует свойства более чем одного типа.</span><span class="sxs-lookup"><span data-stu-id="f5c38-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="f5c38-112">Например, в концептуальной модели с типом сущности `Book` можно определить производные типы `FictionBook` и `NonFictionBook`, из которых каждый наследует свойства `Book`.</span><span class="sxs-lookup"><span data-stu-id="f5c38-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="f5c38-113">Тем не менее, в дальнейшем нельзя определить тип, который бы наследовал свойства обоих типов `FictionBook` и `NonFictionBook`.</span><span class="sxs-lookup"><span data-stu-id="f5c38-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
