<span data-ttu-id="b2723-104">Сегодня многие бизнес-разработчики должны использовать два (или более) языка программирования: высокоуровневый язык для бизнес-логики и уровней представления (например, языками в среде разработки.</span><span class="sxs-lookup"><span data-stu-id="b2723-105">This requires the developer to be proficient in several languages to be effective, and also causes language соответствия типов параметров запроса и технологии `IntelliSense`.</span><span class="sxs-lookup"><span data-stu-id="b2723-108">There is no type checking of the query parameters and no `IntelliSense`  <span data-ttu-id="b2723-109">LINQ позволяет разработчикам формировать запросы на основе наборов в своем коде приложения без использования отдельного языка запросов.</span><span <xref:System.Collections.IEnumerable> интерфейс), например к структурам данных в памяти, XML-документам, базам данных SQL и <xref:System.Data.DataSet> объектам.</span><span class="sxs-lookup"><span программирования, нет необходимости использовать другой язык запросов, внедренный в виде строковых литералов, которые не могут быть проверены компилятором.</span><span class="sxs-lookup"><span data-stu-id="b2723-112">Because queries can be formed in the programming language itself, you do not have to use another query language that is embedded as string literals that cannot be understood or verified by the compiler.</span></span> <span data-ttu-id="b2723-113">Интеграция запросов в язык программирования также позволяет программистам Visual Studio работать более продуктивно, предоставляя проверку типов и синтаксиса во время компиляции `IntelliSense` .</span><span class="sxs-lookup"><span data-stu-id="b2723-113">Integrating queries into the programming language also enables Visual запросов и поиск ошибок.</span><span class="sxs-lookup"><span data-stu-id="b2723-114">These features reduce the need for query debugging and error fixing.</span></span>  
  
 <span data-ttu-id="b2723-115">Передача данных из таблиц SQL в объекты в памяти часто бывает сложной и способствует совершению ошибок.</span><span class="sxs-lookup"><span data-stu-id="b2723-115">Transferring data from SQL tables into objects in memory is often tedious and error-prone.</span></span> <span data-ttu-id="b2723-116">Поставщик LINQ, реализованный LINQ to DataSet, и [!INCLUDE[vbtecdlinq](../../../../includes/vbtecdlinq-md.md)] преобразует исходные данные в <xref:System.Collections.IEnumerable> коллекции объектов на основе.</span><span class="sxs-lookup"><span data-stu-id="b2723-116">The LINQ provider implemented by LINQ to DataSet and [!INCLUDE[vbtecdlinq](../../../../includes/vbtecdlinq-md.md)] converts the source data into <xref:System.Collections.IEnumerable>-based object collections.</span></span> <span data-ttu-id="b2723-117">Программист всегда видит данные как коллекции <xref:System.Collections.IEnumerable> как при запросе, так и при обновлении.</span><span class="sxs-lookup"><span data-stu-id="b2723-117">The programmer always views the data as an <xref:System.Collections.IEnumerable> collection, both when you query and when you update.</span></span> <span data-ttu-id="b2723-118">Для написания запросов к этим коллекциям предоставлена полная поддержка технологии `IntelliSense`.</span><span class="sxs-lookup"><span data-stu-id="b2723-118">Full `IntelliSense` support is provided for writing queries against those collections.</span></span>  
  
 <span data-ttu-id="b2723-119">Существует три отдельные технологии LINQ: LINQ to DataSet, [!INCLUDE[vbtecdlinq](../../../../includes/vbtecdlinq-md.md)] и LINQ to Entities.</span><span class="sxs-lookup"><span data-stu-id="b2723-119">There are three separate ADO.NET Language-Integrated Query (LINQ) technologies: LINQ to DataSet, [!INCLUDE[vbtecdlinq](../../../../includes/vbtecdlinq-md.md)], and LINQ to Entities.</span></span> <span data-ttu-id="b2723-120">LINQ to DataSet предоставляет более широкие, оптимизированные запросы <xref:System.Data.DataSet> [!INCLUDE[vbtecdlinq](../../../../includes/vbtecdlinq-md.md)] для и позволяет напрямую запрашивать SQL Server схемы баз данных, а LINQ to Entities позволяет запрашивать EDM.</span><span class="sxs-lookup"><span data-stu-id="b2723-120">LINQ to DataSet provides richer, optimized querying over the <xref:System.Data.DataSet> and [!INCLUDE[vbtecdlinq](../../../../includes/vbtecdlinq-md.md)] enables you to directly query SQL Server database schemas, and LINQ to Entities allows you to query an Entity Data Model.</span></span>
