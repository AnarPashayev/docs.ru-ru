# <a name="supportedruntime-element"></a>\<поддерживаемыйэлемент>Runtime

Определяет, какую общую версию выполнения языка и, по желанию, версию .NET Framework поддерживает приложение.  

[\<конфигурация>](../configuration-element.md)  
&nbsp;&nbsp;[\<>стартапа](startup-element.md)  
&nbsp;&nbsp;&nbsp;&nbsp;**\<поддерживаетRuntime>**  

## <a name="syntax"></a>Синтаксис

```xml
<supportedRuntime version="runtime version" sku="sku id"/>
```

## <a name="attributes"></a>Атрибуты

|attribute|Описание|
|---------------|-----------------|
|**version**|Необязательный атрибут.<br /><br /> Строковое значение, задающее версию среды CLR, которая поддерживается данным приложением. Для допустимых `version` значений атрибута см. ["runtime version" values](#version) **Примечание:**  Через .NET Framework 3.5 значение *«версии времени выполнения»* принимает форму *major*. *несовершеннолетний*. *построить*. Начиная с .NET Framework 4, требуются только основные и второстепенные номера версий (т.е. "v4.0" вместо "v4.0.30319"). Рекомендуется использовать более короткие строки.|
|**Sku**|Необязательный атрибут.<br /><br /> Строковое значение, которое указывает единицу складского хранения (SKU), которая, в свою очередь, указывает выпуск .NET Framework, поддерживаемый этим приложением.<br /><br /> Начиная с .NET Framework 4.0, рекомендуется использовать атрибут `sku`.  Если он присутствует, он указывает целевую версию .NET Framework приложения.<br /><br /> Для допустимых значений атрибута ["sku id" values](#sku) sku см.|

## <a name="remarks"></a>Remarks

Если ** \<поддерживаемыйэлементRuntime>** элемент не присутствует в файле конфигурации приложения, используется версия времени выполнения, используемого для создания приложения.

** \<Поддерживаемыйэлемент>** элемент должен использоваться всеми приложениями, построенными с использованием версии 1.1 или позже времени выполнения. Приложения, созданные для поддержки только версии 1.0 времени выполнения, должны использовать [ \<необходимый элементRuntime>.](../startup/requiredruntime-element.md)

> [!NOTE]
> Если вы используете функцию [CorBindToRuntimeByCfg](../../../unmanaged-api/hosting/corbindtoruntimebycfg-function.md) для указания `<requiredRuntime>` файла конфигурации, необходимо использовать элемент для всех версий времени выполнения. Элемент `<supportedRuntime>` игнорируется при использовании [CorbindToRuntimeByCfg](../../../unmanaged-api/hosting/corbindtoruntimebycfg-function.md).  
  
Для приложений, поддерживающих версии среды выполнения от .NET Framework 1.1 до 3.5, при поддержке нескольких версий среды выполнения первый элемент должен задавать наиболее предпочтительную версию среды, а последний — наименее предпочтительную. Для приложений, поддерживающих .NET Framework 4.0 или более поздние версии, `version` атрибут указывает на версию `sku` CLR, которая является общей для .NET Framework 4 и более поздних версий, а атрибут указывает на единую версию .NET Framework, на которую нацелено приложение.

Если ** \<поддерживаемыйПринцип>** элемент `sku` с атрибутом присутствует в файле конфигурации, а установленная версия .NET Framework ниже указанной поддерживаемой версии, приложение не выполняется и вместо этого отображает сообщение с просьбой установить поддерживаемую версию. В противном случае приложение пытается работать на любой установленной версии, но может вести себя неожиданно, если оно не полностью совместимо с этой версией. (Для различий в совместимости между версиями рамочного соглашения .NET [см.](https://docs.microsoft.com/dotnet/framework/migration-guide/application-compatibility) Поэтому мы рекомендуем включить этот элемент в файл конфигурации приложения для более легкой диагностики ошибок. (Файл конфигурации, автоматически генерируемый Visual Studio при создании нового проекта, уже содержит его.)
  
> [!NOTE]
> Если ваше приложение использует устаревшие пути активации, такие как [функция CorBindToRuntimeEx,](../../../unmanaged-api/hosting/corbindtoruntimeex-function.md)и вы хотите, чтобы эти пути активировали версию 4 CLR вместо более ранней версии, или если ваше приложение построено с помощью .NET Framework 4, но имеет зависимость от сборки смешанного режима, построенной с более ранней версией .NET Framework, недостаточно указать в списке поддерживаемых Runtimes рамку .NET. Кроме того, в [ \<начальной> элемента](../startup/startup-element.md) в файле `true`конфигурации необходимо настроить `useLegacyV2RuntimeActivationPolicy` атрибут. Однако установка этого `true` атрибута означает, что все компоненты, построенные с более ранними версиями рамочного соглашения .NET, запускаются с помощью .NET Framework 4 вместо времени выполнения, с которым они были построены.

Приложения рекомендуется тестировать со всеми версиями .NET Framework, в которых они могут работать.

## <a name="example"></a>Пример

В следующем примере показано задание в файле конфигурации поддерживаемых версий среды выполнения. Файл конфигурации указывает, что приложение нацелено на рамку .NET 4.7.

```xml
<configuration>
   <startup>
      <supportedRuntime version="v4.0" sku=".NETFramework,Version=v4.7" />
   </startup>
</configuration>
```
