# <a name="backuplist"></a>\<Баккуплист >
Представляет раздел конфигурации для определения списка резервных копий, который перечисляет набор конечных точек, которые должны использоваться службой маршрутизации в случае, если основная конечная точка недостижима. Если первая конечная точка в списке недоступна, то служба маршрутизации автоматически переключается на следующую точку в списке.  Этот метод позволяет быстро повысить надежность приложения, не реализуя в клиентском приложении обработку сложных схем и не задавая расположение всех служб.  
  
 \<> System. serviceModel  
\<> маршрутизации  
\<Баккуплистс >  
\<Баккуплист >  
  
## <a name="syntax"></a>Синтаксис  
  
```xml  
<routing>
  <backupLists>
    <backupList name="String">
      <add endpointName="String" />
    </backupList>
  </backupLists>
</routing>
```  
  
## <a name="attributes-and-elements"></a>Атрибуты и элементы  
 В следующих разделах описаны атрибуты, дочерние и родительские элементы.  
  
### <a name="attributes"></a>Атрибуты  
  
|Атрибут|Описание|  
|---------------|-----------------|  
|имя|Строка, в которой приведено имя, используемое для указания этого списка конечных точек.|  
  
### <a name="child-elements"></a>Дочерние элементы  
  
|Элемент|Описание|  
|-------------|-----------------|  
|[\<filter>](filter.md)||  
  
### <a name="parent-elements"></a>Родительские элементы  
  
|Элемент|Описание|  
|-------------|-----------------|  
|[\<> маршрутизации](routing.md)|Список резервных конечных точек.|  
  
## <a name="remarks"></a>Примечания  
 Этот раздел содержит упорядоченную коллекцию конечных точек, в которые будет передаваться сообщение в случае, если при отправке в основную конечную точку возникает исключение связи.  
  
 Если отправка в основную конечную точку, указанную `endpointName` в [ \<атрибуте Add >](add-of-entries.md) , завершается с исключением связи, служба Routing Service попытается отправить сообщение в первую конечную точку в этом разделе конфигурации. Если эта отправка также приведет к возникновению исключения связи, служба маршрутизации попытается отправить сообщение в следующую точку, указанную в этом разделе, пока отправка не завершится успешно или не возвратит ошибку, не связанную с исключением связи, либо пока ошибки не будут возвращены для всех конечных точек из коллекции.  
  
 В следующем примере, если отправка в основную конечную точку с именем "Destination" возвращает исключение связи, Служба попытается отправить сообщение в "Алтернатесервицекуеуе". Если эта попытка также возвращает исключение связи, то служба маршрутизации попытается отправить сообщение в следующую конечную точку в коллекции.  
  
```xml  
<filterTables>
  <filterTable name="filterTable1">
    <add filterName="MatchAllFilter1"
         endpointName="Destination"
         backupList="backupEndpointList" />
  </filterTable>
</filterTables>
<backupLists>
  <backupList name="backupEndpointList">
    <add endpointName="backupServiceQueue" />
    <add endpointName="alternateServiceQueue" />
  </backupList>
</backupLists>
```
