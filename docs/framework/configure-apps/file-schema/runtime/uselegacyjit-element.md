# <a name="uselegacyjit-element"></a>Элемент \<useLegacyJit>

Определяет, использует ли среда CLR устаревший 64-разрядный JIT-компилятор для JIT-компиляции.  
  
\<configuration>  
\<Среда выполнения >  
\<useLegacyJit >
  
## <a name="syntax"></a>Синтаксис  
  
```xml
<useLegacyJit enabled=0|1 />
```

Имя элемента `useLegacyJit` чувствителен к регистру.
  
## <a name="attributes-and-elements"></a>Элементы и атрибуты

В следующих разделах описаны атрибуты, дочерние и родительские элементы.  
  
### <a name="attributes"></a>Атрибуты  
  
| Атрибут | Описание                                                                                   |  
| --------- | --------------------------------------------------------------------------------------------- |  
| `enabled` | Обязательный атрибут.<br><br>Указывает, использует ли среда выполнения устаревший 64-разрядный компилятор JIT. |  
  
### <a name="enabled-attribute"></a>атрибут Enabled  
  
| Значение | Описание                                                                                                         |  
| ----- | ------------------------------------------------------------------------------------------------------------------- |  
| 0     | Среда CLR использует новый 64-разрядных JIT-компилятор, включенных в .NET Framework 4.6 и более поздних версий. |  
| 1     | Среда CLR использует старого 64-разрядных JIT-компилятора.                                                     |  
  
### <a name="child-elements"></a>Дочерние элементы

Нет
  
### <a name="parent-elements"></a>Родительские элементы  
  
| Элемент         | Описание                                                                                                       |  
| --------------- | ----------------------------------------------------------------------------------------------------------------- |  
| `configuration` | Корневой элемент в любом файле конфигурации, используемом средой CLR и приложениями .NET Framework. |  
| `runtime`       | Содержит сведения о параметрах инициализации среды выполнения.                                                        |  
  
## <a name="remarks"></a>Примечания  

Начиная с .NET Framework 4.6, среда CLR использует новый 64-разрядный компилятор для компиляции эту задачу выполняет JIT по умолчанию. В некоторых случаях это может привести различия в поведении из кода приложения, который был JIT-компиляции в предыдущей версии 64-разрядный компилятор JIT. Установив `enabled` атрибут `<useLegacyJit>` элемент `1`, можно отключить новый 64-разрядный компилятор JIT и вместо компиляции приложения с помощью устаревший 64-разрядный компилятор JIT.  
  
> [!NOTE]
> `<useLegacyJit>` Элемента влияет на 64-разрядных JIT-компиляции. Компиляция с помощью 32-разрядный компилятор JIT не затрагивается.  
  
Вместо настройки файла конфигурации, вы можете включить устаревший 64-разрядный компилятор JIT двумя способами:  
  
- Настройка переменной среды

  Задайте `COMPLUS_useLegacyJit` переменной среды либо `0` (используйте новый 64-разрядный компилятор JIT) или `1` (использовать старые 64-разрядный компилятор JIT):
  
  ```  
  COMPLUS_useLegacyJit=0|1  
  ```  
  
  Переменная среды имеет *глобальной области*, что означает, что это влияет на все приложения выполняются на компьютере. Если задано, его можно переопределить с помощью параметра файла конфигурации приложения. Имя переменной среды не учитывается.
  
- Добавление раздела реестра

  Вы можете включить устаревший 64-разрядных JIT-компилятор, добавив `REG_DWORD` значение либо `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\.NETFramework` или `HKEY_CURRENT_USER\SOFTWARE\Microsoft\.NETFramework` разделом реестра. Значение называется `useLegacyJit`. Если значение равно 0, используется новый компилятор. Если значение равно 1, устаревший 64-разрядный компилятор JIT включен. Имя значения реестра не учитывается.
  
  Добавление значения к `HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\.NETFramework` ключ влияет на все приложения, работающие на компьютере. Добавление значения к `HKEY_CURRENT_USER\SOFTWARE\Microsoft\.NETFramework` ключ влияет на все приложения, запустите текущим пользователем. Если компьютер настроен с несколькими учетными записями, затрагиваются только приложений, выполните текущим пользователем, если только значение добавляется к разделам реестра для других пользователей, а также. Добавление `<useLegacyJit>` элемент в файл конфигурации переопределяет параметры реестра, если они присутствуют.  
  
## <a name="example"></a>Пример  

В следующем файле конфигурации отключает компиляцию с помощью нового 64-разрядный компилятор JIT и вместо этого использует устаревший 64-разрядный компилятор JIT.  
  
```xml  
<?xml version ="1.0"?>  
<configuration>  
  <runtime>  
    <useLegacyJit enabled="1" />  
  </runtime>  
</configuration>  
```
