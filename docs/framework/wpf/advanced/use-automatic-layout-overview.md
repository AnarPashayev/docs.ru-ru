# <a name="use-automatic-layout-overview"></a>Обзор использования автоматической разметки

В этом разделе представлены рекомендации для разработчиков по написанию приложений [!INCLUDE[TLA#tla_winclient](../../../../includes/tlasharptla-winclient-md.md)] с помощью локализуемых пользовательских интерфейсов (UI). В прошлом процесс локализации пользовательского интерфейса осуществлялся в течение длительного времени. Каждый язык, для которого был адаптирован пользовательский интерфейс, требовал коррекции пикселя в пикселях. На сегодняшний день с верным стандартом проектирования и правильного программирования можно создать пользовательские интерфейсы, чтобы локализаторам было меньше изменений и их расположения. Подход к написанию приложений, которые могут быть более легко изменять размер и положение, называется автоматическим макетом и может быть достигнут с помощью конструкции приложения [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)].

<a name="advantages_of_autolayout"></a>

## <a name="advantages-of-using-automatic-layout"></a>Преимущества использования автоматической разметки

Поскольку система презентации [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] является мощным и гибкой, она обеспечивает возможность разметки элементов в приложении, которые могут быть скорректированы в соответствии с требованиями различных языков. В следующем списке отмечены преимущества автоматической разметки.

- Пользовательский интерфейс отображается хорошо на любом языке.

- Снижается необходимость перенастраивать расположение и размер элементов управления после перевода текста.

- Снижается необходимость перенастраивать размер окна.

- Макет пользовательского интерфейса правильно отображается на любом языке.

- Локализация может быть сведена не многим более чем к переводу строки.

## <a name="automatic-layout-and-coding-standards"></a>Автоматическая разметка и стандарты кодирования

При использовании автоматического макета требуется набор стандартов и правил разработки кода и проектирования для создания полностью локализуемого пользовательского интерфейса. Следующие рекомендации предназначены для помощи в кодировании автоматической разметки.

**Не использовать абсолютные позиции**

- Не используйте <xref:System.Windows.Controls.Canvas>, так как он размещает элементы абсолютно.

- Для позиционирования элементов управления используйте <xref:System.Windows.Controls.DockPanel>, <xref:System.Windows.Controls.StackPanel> и <xref:System.Windows.Controls.Grid>.

Обсуждение различных типов панелей см. в разделе [Общие сведения о панелях](../controls/panels-overview.md).

**Не устанавливайте фиксированный размер окна**

  [!code-xaml[LocalizationGrid#2](~/samples/snippets/csharp/VS_Snippets_Wpf/LocalizationGrid/CS/Pane1.xaml#2)]

**Добавление <xref:System.Windows.FrameworkElement.FlowDirection%2A>**

- Добавьте <xref:System.Windows.FrameworkElement.FlowDirection%2A> в корневой элемент приложения.

  WPF предоставляет удобный способ поддержки горизонтальных, двунаправленных и вертикальных макетов. В платформе Presentation Framework свойство <xref:System.Windows.FrameworkElement.FlowDirection%2A> можно использовать для определения макета. Ниже перечислены шаблоны направления текста.

  - <xref:System.Windows.FlowDirection.LeftToRight?displayProperty=nameWithType> (Лртб) — горизонтальный макет для латиницы, восточноазиатских языков и т. д.

  - <xref:System.Windows.FlowDirection.RightToLeft?displayProperty=nameWithType> (Рлтб) — двунаправленный для арабского, иврита и т. д.

**Использование составных шрифтов вместо физических шрифтов**

- При использовании составных шрифтов свойство <xref:System.Windows.Controls.Control.FontFamily%2A> не обязательно должно быть локализовано.

- Разработчики могут использовать один из следующих шрифтов или создать свой собственный.

  - Глобальный пользовательский интерфейс
  - Global San Serif
  - Global Serif

**Добавить XML: lang**

- Добавьте атрибут `xml:lang` в корневой элемент пользовательского интерфейса, например `xml:lang="en-US"` для приложения на английском языке.

- Поскольку составные шрифты используют `xml:lang` для определения используемого шрифта, установите это свойство для поддержки многоязычных сценариев.
