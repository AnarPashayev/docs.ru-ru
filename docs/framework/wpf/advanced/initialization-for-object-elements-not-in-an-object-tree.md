 <span data-ttu-id="7ef99-106">При создании экземпляра класса [!INCLUDE[TLA#tla_winclient](../../../../includes/tlasharptla-winclient-md.md)] в коде следует иметь в виду, что несколько аспектов инициализации объекта для класса [!INCLUDE[TLA#tla_winclient](../../../../includes/tlasharptla-winclient-md.md)] преднамеренно не являются частью кода, выполняющегося при вызове конструктора класса.</span><span class="sxs-lookup"><span data-stu-id="7ef99-106">When you create an instance of a [!INCLUDE[TLA#tla_winclient](../../../../includes/tlasharptla-winclient-md.md)] class in code, you should be aware that several aspects of object initialization for a [!INCLUDE[TLA#tla_winclient](../../../../includes/tlasharptla-winclient-md.md)] class are deliberately not a part of the code that is executed when calling the class constructor.</span></span> <span data-ttu-id="7ef99-107">Большая часть визуального представления элемента управления, особенно для класса элемента управления, не определяется конструктором.</span><span class="sxs-lookup"><span data-stu-id="7ef99-107">Particularly for a control class, most of the visual representation of that control is not defined by the constructor.</span></span> <span data-ttu-id="7ef99-108">Вместо этого визуальное представление определяется шаблоном элемента управления.</span><span class="sxs-lookup"><span data-stu-id="7ef99-108">Instead, the visual representation is defined by the control's template.</span></span> <span data-ttu-id="7ef99-109">Шаблон может быть получен из различных источников, но наиболее часто он берется из стилей темы.</span><span class="sxs-lookup"><span data-stu-id="7ef99-109">The template potentially comes from a variety of sources, but most often the template is obtained from theme styles.</span></span> <span data-ttu-id="7ef99-110">Шаблоны фактически выполняют позднее связывание.  <span data-ttu-id="7ef99-113">Визуальное дерево также участвует в этом процессе.</span><span class="sxs-lookup"><span data-stu-id="7ef99-113">The visual tree also participates in this process.</span></span> <span data-ttu-id="7ef99-114">Элементы, которые являются частью визуального дерева посредством шаблонов, также полностью не создаются до тех пор, пока не произойдет подключение.</span><span class="sxs-lookup"><span data-stu-id="7ef99-114">Elements that are part of the visual tree through the templates are also not fully instantiated until connected.</span></span>  
  
 <span data-ttu-id="7ef99-115">Последствием этого является то, что для определенных операций, зависящих от завершенных визуальных характеристик элемента, требуется выполнение дополнительных действий.</span><span class="sxs-lookup"><span data-stu-id="7ef99-115">The consequences of this behavior are that certain operations that rely on the completed visual characteristics of an element require additional steps.</span></span> <span data-ttu-id="7ef99-116">Примером является попытка получить визуальные характеристики класса, который был создан, но еще не присоединен к дереву.</span><span class="sxs-lookup"><span data-stu-id="7ef99-116">An example is if you are attempting to get the visual characteristics of a class that was constructed but not yet attached to a tree.</span></span> <span data-ttu-id="7ef99-117">Например если вы хотите вызвать <xref:System.Windows.Media.Imaging.RenderTargetBitmap.Render%2A> на <xref:System.Windows.Media.Imaging.RenderTargetBitmap> и визуальный элемент не подключен к дереву, этот элемент не визуально завершена, пока не будут выполнены дополнительные действия по инициализации.</span><span class="sxs-lookup"><span data-stu-id="7ef99-117">For instance, if you want to call <xref:System.Windows.Media.Imaging.RenderTargetBitmap.Render%2A> on a <xref:System.Windows.Media.Imaging.RenderTargetBitmap> and the visual you are passing is an element not connected to a tree, that element is not visually complete until additional initialization steps are completed.</span></span>

 <span data-ttu-id="7ef99-122">Если элемент при установке свойств на <xref:System.Windows.FrameworkElement> или <xref:System.Windows.FrameworkContentElement> производного класса, то можно вызвать версии класса <xref:System.Windows.FrameworkElement.BeginInit%2A> и <xref:System.Windows.FrameworkElement.EndInit%2A> вместо приведения к <xref:System.ComponentModel.ISupportInitialize>.</span><span class="sxs-lookup"><span data-stu-id="7ef99-122">If the element you are setting properties on is a <xref:System.Windows.FrameworkElement> or <xref:System.Windows.FrameworkContentElement> derived class, then you can call the class versions of <xref:System.Windows.FrameworkElement.BeginInit%2A> and <xref:System.Windows.FrameworkElement.EndInit%2A> rather than casting to <xref:System.ComponentModel.ISupportInitialize>.</span></span>
