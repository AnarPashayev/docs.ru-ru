## <a name="software-rendering-pipeline"></a><span data-ttu-id="90ed0-110">Программный конвейер отрисовки</span><span class="sxs-lookup"><span data-stu-id="90ed0-110">Software Rendering Pipeline</span></span>  
 <span data-ttu-id="90ed0-111">[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] Программного конвейера визуализации полностью ограничен ресурсами ЦП.</span><span class="sxs-lookup"><span data-stu-id="90ed0-111">The [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] software rendering pipeline is entirely CPU bound.</span></span> [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] <span data-ttu-id="90ed0-112">Использование инструкций SSE и SSE2 задает в ЦП для реализации растеризации оптимизированного, полнофункциональной программного обеспечения.</span><span class="sxs-lookup"><span data-stu-id="90ed0-112">takes advantage of the SSE and SSE2 instruction sets in the CPU to implement an optimized, fully-featured software rasterizer.</span></span> <span data-ttu-id="90ed0-113">К программному обеспечению, осуществляется плавный переход каждый раз, когда невозможно функциональных возможностей приложения, используя Аппаратный конвейер отрисовки.</span><span class="sxs-lookup"><span data-stu-id="90ed0-113">Fallback to software is seamless any time application functionality cannot be rendered using the hardware rendering pipeline.</span></span>

### <a name="graphics-rendering-tiers"></a><span data-ttu-id="90ed0-118">Уровни графической отрисовки</span><span class="sxs-lookup"><span data-stu-id="90ed0-118">Graphics Rendering Tiers</span></span>  
 <span data-ttu-id="90ed0-119">Возможно, очень сложно предсказать, приложение будет запущено в конфигурации оборудования.</span><span class="sxs-lookup"><span data-stu-id="90ed0-119">It may be very difficult to predict the hardware configuration that your application will be running on.</span></span> <span data-ttu-id="90ed0-120">Тем не менее может потребоваться рассмотрите вариант проектирования, которая позволяет приложению легко переключение функций при работе на другом оборудовании, благодаря чему он может использовать все преимущества каждой различные аппаратные конфигурации.</span><span class="sxs-lookup"><span data-stu-id="90ed0-120">However, you might want to consider a design that allows your application to seamlessly switch features when running on different hardware, so that it can take full advantage of each different hardware configuration.</span></span>  
  
 <span data-ttu-id="90ed0-121">Чтобы добиться этого, [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] предоставляет функциональные возможности для определения графических возможностей системы во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="90ed0-121">To achieve this, [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] provides functionality to determine the graphics capability of a system at runtime.</span></span> <span data-ttu-id="90ed0-122">Возможности работы с графикой, определяется категоризация видеоадаптер как один из трех уровней визуализации.</span><span class="sxs-lookup"><span data-stu-id="90ed0-122">Graphics capability is determined by categorizing the video card as one of three rendering capability tiers.</span></span> [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] <span data-ttu-id="90ed0-123">предоставляет интерфейс API, позволяющий приложению запрашивать уровней отрисовки.</span><span class="sxs-lookup"><span data-stu-id="90ed0-123">exposes an API that allows an application to query the rendering capability tier.</span></span> <span data-ttu-id="90ed0-124">Приложение может предпринять различные фрагменты кода во время выполнения в зависимости от уровня визуализации, который поддерживается оборудованием.</span><span class="sxs-lookup"><span data-stu-id="90ed0-124">Your application can then take different code paths at run time depending on the rendering tier supported by the hardware.</span></span>  
  
 <span data-ttu-id="90ed0-125">Ниже перечислены возможности графического оборудования, которые сильнее всего влияют на уровни отрисовки.</span><span class="sxs-lookup"><span data-stu-id="90ed0-125">The features of the graphics hardware that most impact the rendering tier levels are:</span></span>  
  
- <span data-ttu-id="90ed0-126">**Видеопамять**. - <span data-ttu-id="90ed0-127">**Построитель текстуры**. Построитель текстуры — это функция обработки графики, вычисляющая эффекты на уровне пикселей.</span><span class="sxs-lookup"><span data-stu-id="90ed0-127">**Pixel Shader** A pixel shader is a graphics processing function that calculates effects on a per-pixel basis.</span></span> <span data-ttu-id="90ed0-128">В зависимости от разрешения отображаемой графики может быть несколько миллионов пикселей, которые необходимо обработать для каждого отображаемого кадра.</span><span class="sxs-lookup"><span data-stu-id="90ed0-128">Depending on the resolution of the displayed graphics, there could be several million pixels that need to be processed for each display frame.</span></span>  
  
- <span data-ttu-id="90ed0-129">**Вершинный построитель текстуры**. - <span data-ttu-id="90ed0-130">**Поддержка мультитекстурирования**. Поддержка мультитекстурирования относится к возможности применить две и более разных текстур во время операции смешения на объекте трехмерной графики.</span><span class="sxs-lookup"><span data-stu-id="90ed0-130">**Multitexture Support** Multitexture support refers to the ability to apply two or more distinct textures during a blending operation on a 3D graphics object.</span></span> <span data-ttu-id="90ed0-131">Степень многотекстурной поддержки определяется числом мультитекстурных модулей на графическом оборудовании.</span><span class="sxs-lookup"><span data-stu-id="90ed0-131">The degree of multitexture support is determined by the number of multitexture units on the graphics hardware.</span></span>  
  
 <span data-ttu-id="90ed0-132">Построитель текстуры, шейдер вершин и мультитекстурирования используются для определения конкретных [!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] уровни версий, которые, в свою очередь, используются для определения различных уровней отрисовки в [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)].</span><span class="sxs-lookup"><span data-stu-id="90ed0-132">The pixel shader, vertex shader, and multitexture features are used to define specific [!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] version levels, which, in turn, are used to define the different rendering tiers in [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)].</span></span>  
  
 <span data-ttu-id="90ed0-133">Возможности графического оборудования определяют возможности отрисовки приложения [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)].</span><span class="sxs-lookup"><span data-stu-id="90ed0-133">The features of the graphics hardware determine the rendering capability of a [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] application.</span></span> <span data-ttu-id="90ed0-134">Система [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] определяет три уровня отрисовки.</span><span class="sxs-lookup"><span data-stu-id="90ed0-134">The [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] system defines three rendering tiers:</span></span>  
  
- <span data-ttu-id="90ed0-135">**Уровень отрисовки 0**. Аппаратное ускорение графики отсутствует.</span><span class="sxs-lookup"><span data-stu-id="90ed0-135">**Rendering Tier 0** No graphics hardware acceleration.</span></span> <span data-ttu-id="90ed0-136">[!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] Версия меньше, чем версии 7.0.</span><span class="sxs-lookup"><span data-stu-id="90ed0-136">The [!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] version level is less than version 7.0.</span></span>  
  
- <span data-ttu-id="90ed0-137">**Уровень отрисовки 1** Частичное аппаратное ускорение графики.</span><span class="sxs-lookup"><span data-stu-id="90ed0-137">**Rendering Tier 1** Partial graphics hardware acceleration.</span></span> <span data-ttu-id="90ed0-138">[!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] Уровень версии больше или равна 7.0, и **меньше** версии 9.0.</span><span class="sxs-lookup"><span data-stu-id="90ed0-138">The [!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] version level is greater than or equal to version 7.0, and **lesser** than version 9.0.</span></span>  
  
- <span data-ttu-id="90ed0-139">**Уровень отрисовки 2**. Большинство графических функций используют аппаратное ускорение графики.</span><span class="sxs-lookup"><span data-stu-id="90ed0-139">**Rendering Tier 2** Most graphics features use graphics hardware acceleration.</span></span> <span data-ttu-id="90ed0-140">Уровень версии [!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] больше или равен версии 9.0.</span><span class="sxs-lookup"><span data-stu-id="90ed0-140">The [!INCLUDE[TLA2#tla_dx](../../../../includes/tla2sharptla-dx-md.md)] version level is greater than or equal to version 9.0.</span></span>  
  
 <span data-ttu-id="90ed0-141">Дополнительные сведения о [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] уровнях отрисовки см. в разделе [уровни графической отрисовки](graphics-rendering-tiers.md).</span><span class="sxs-lookup"><span data-stu-id="90ed0-141">For more information on [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] rendering tiers, see [Graphics Rendering Tiers](graphics-rendering-tiers.md).</span></span>
