## <a name="implementing-a-collection-type-dependency-property"></a><span data-ttu-id="b2ebc-104">Реализация свойства зависимостей типа "коллекция"</span><span class="sxs-lookup"><span data-stu-id="b2ebc-104">Implementing a Collection-Type Dependency Property</span></span>  
 <span data-ttu-id="b2ebc-105">Для свойства зависимостей в целом шаблон реализации, который вы выполняете, заключается в том, что вы определяете оболочку свойств CLR, где это свойство поддерживается идентификатором <xref:System.Windows.DependencyProperty>, а не полем или другой конструкцией.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-105">For a dependency property in general, the implementation pattern that you follow is that you define a CLR property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="b2ebc-106">При реализации свойства типа "коллекция" вы следуете этому же шаблону.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="b2ebc-107">Однако свойство типа коллекции значительно усложняет шаблон, когда тип, содержащийся в коллекции, сам является <xref:System.Windows.DependencyObject> или <xref:System.Windows.Freezable> производном классе.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

## <a name="initializing-the-collection-beyond-the-default-value"></a><span data-ttu-id="b2ebc-108">Инициализация коллекции за пределами значения по умолчанию</span><span class="sxs-lookup"><span data-stu-id="b2ebc-108">Initializing the Collection Beyond the Default Value</span></span>  
 <span data-ttu-id="b2ebc-109">При создании свойства зависимостей вы не указываете значение свойства по умолчанию в качестве начального значения поля.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-109">When you create a dependency property, you do not specify the property default value as the initial field value.</span></span> <span data-ttu-id="b2ebc-110">Вместо этого значение по умолчанию указывается через метаданные свойства зависимостей.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-110">Instead, you specify the default value through the dependency property metadata.</span></span> <span data-ttu-id="b2ebc-111">Если свойство является ссылочным типом, значение по умолчанию, заданное в метаданных свойства зависимостей, не является значением по умолчанию для каждого экземпляра. Напротив, это значение по умолчанию, которое применяется ко всем экземплярам типа.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-111">If your property is a reference type, the default value specified in dependency property metadata is not a default value per instance; instead it is a default value that applies to all instances of the type.</span></span> <span data-ttu-id="b2ebc-112">Поэтому необходимо избегать использования единственной статической коллекции, определенной метаданными свойства коллекции, в качестве рабочего значения по умолчанию для вновь создаваемых экземпляров типа.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-112">Therefore you must be careful to not use the singular static collection defined by the collection property metadata as the working default value for newly created instances of your type.</span></span> <span data-ttu-id="b2ebc-113">Вместо этого необходимо явно задать в качестве значения коллекции уникальную коллекцию (экземпляр) как часть логики конструктора класса.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-113">Instead, you must make sure that you deliberately set the collection value to a unique (instance) collection as part of your class constructor logic.</span></span> <span data-ttu-id="b2ebc-114">В противном случае будет создан случайный одноэлементный класс.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-114">Otherwise you will have created an unintentional singleton class.</span></span>

## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="b2ebc-129">Передача сведений об изменении значений привязки из свойств коллекции</span><span class="sxs-lookup"><span data-stu-id="b2ebc-129">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="b2ebc-130">Свойство коллекции, которое само является свойством зависимостей, не сообщает об изменениях своих подсвойств автоматически.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-130">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="b2ebc-131">При создании привязок в коллекции это может мешать привязке сообщать об изменениях и, следовательно, нарушить некоторые сценарии привязки данных.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-131">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="b2ebc-132">Однако, если в качестве типа коллекции используется тип коллекции <xref:System.Windows.FreezableCollection%601>, то изменения подсвойств в содержащихся в коллекции элементах будут правильно отправлены, а привязка работает должным образом.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-132">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>  
  
 <span data-ttu-id="b2ebc-133">Чтобы включить привязку подсвойства в коллекции объектов зависимостей, создайте свойство коллекции как тип <xref:System.Windows.FreezableCollection%601>с ограничением типа для этой коллекции в любой <xref:System.Windows.DependencyObject> производный класс.</span><span class="sxs-lookup"><span data-stu-id="b2ebc-133">To enable subproperty binding in a dependency object collection, create the collection property as type <xref:System.Windows.FreezableCollection%601>, with a type constraint for that collection to any <xref:System.Windows.DependencyObject> derived class.</span></span>
