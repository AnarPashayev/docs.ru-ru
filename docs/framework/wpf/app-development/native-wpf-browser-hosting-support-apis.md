# <a name="native-wpf-browser-hosting-support-apis"></a>Интерфейсы API для поддержки размещения в собственном браузере WPF
Хостинг приложений WPF в веб-браузерах облегчается сервером Active Document (также известным как DocObject), зарегистрированным вне wPF Host. Internet Explorer может напрямую активировать и интегрироваться с активным документом. Для хостинга XBAPs и свободных документов XAML в браузерах Mozilla, WPF предоставляет плагин NPAPI, который обеспечивает аналогичную среду хостинга для сервера WPF Active Document, как Internet Explorer делает. Однако самый простой практический способ размещения документов XBAPs и XAML в других браузерах и автономных приложениях — это управление веб-браузером Internet Explorer. Управление web Browser обеспечивает сложную среду хостинга сервера Active Document, но позволяет своему собственному хосту настраивать и расширять эту среду и напрямую общаться с текущим объектом Active Document.  
  
 Сервер WPF Active Document реализует несколько общих интерфейсов хостинга, в ключая [IOleObject,](/windows/win32/api/oleidl/nn-oleidl-ioleobject) [IOleDocument,](/windows/win32/api/docobj/nn-docobj-ioledocument) [IOleInPlaceActiveObject,](/windows/win32/api/oleidl/nn-oleidl-ioleinplaceactiveobject) [IPersistMoniker](https://docs.microsoft.com/previous-versions/windows/internet-explorer/ie-developer/platform-apis/ms775042(v=vs.85)), [IOleCommandTarget](/windows/win32/api/docobj/nn-docobj-iolecommandtarget). При размещении в управлении веб-браузером эти интерфейсы могут быть запросами от объекта, возвращенного свойством [IWebBrowser2::Document.](https://docs.microsoft.com/previous-versions/aa752116(v=vs.85))  
  
## <a name="iolecommandtarget"></a>Iolecommandtarget  
 Внедрение сервера Active Document в WPF Active Document [IOleCommandTarget](/windows/win32/api/docobj/nn-docobj-iolecommandtarget) поддерживает многочисленные команды, связанные с навигацией и браузером, стандартной группы команд OLE (с нулевой командной группой GUID). Кроме того, он распознает пользовательскую группу команд под названием CGID_PresentationHost. В настоящее время в этой группе определена только одна команда.  
  
```cpp  
DEFINE_GUID(CGID_PresentationHost, 0xd0288c55, 0xd6, 0x4f5e, 0xa8, 0x51, 0x79, 0xde, 0xc5, 0x1b, 0x10, 0xec);  
enum PresentationHostCommands {
   PHCMDID_TABINTO = 1
};  
```  
  
 PHCMDID_TABINTO поручает PresentationHost переключить фокус ировку на первый или последний фокусируемый элемент в его содержимом в зависимости от состояния ключа Shift.
