### <a name="to-enable-message-flow-tracing-in-the-getting-started-tutorial"></a><span data-ttu-id="88967-113">Включение трассировки потока сообщений в проекте, который был создан в учебнике «Приступая к работе»</span><span class="sxs-lookup"><span data-stu-id="88967-113">To enable message flow tracing in the Getting Started tutorial</span></span>

1. <span data-ttu-id="88967-114">Откройте средство просмотра событий, щелкнув **запустить**, **запуска**и введя `eventvwr.exe`.</span><span class="sxs-lookup"><span data-stu-id="88967-114">Open Event Viewer by clicking **Start**, **Run**, and entering `eventvwr.exe`.</span></span>

2. <span data-ttu-id="88967-115">Если вы еще не включено аналитическое отслеживание, разверните **журналы приложений и служб**, **Microsoft**, **Windows**, **сервер приложений-приложения** .</span><span class="sxs-lookup"><span data-stu-id="88967-115">If you haven’t enabled analytic tracing, expand **Applications and Services Logs**, **Microsoft**, **Windows**, **Application Server-Applications**.</span></span> <span data-ttu-id="88967-116">Выберите **представление**, **Отобразить аналитический и отладочный журналы**.</span><span class="sxs-lookup"><span data-stu-id="88967-116">Select **View**, **Show Analytic and Debug Logs**.</span></span> <span data-ttu-id="88967-117">Щелкните правой кнопкой мыши **аналитический** и выберите **включить журнал**.</span><span class="sxs-lookup"><span data-stu-id="88967-117">Right-click **Analytic** and select **Enable Log**.</span></span> <span data-ttu-id="88967-118">Оставьте средство просмотра событий открытым, чтобы можно было просматривать трассировки.</span><span class="sxs-lookup"><span data-stu-id="88967-118">Leave Event Viewer open so that traces can be viewed.</span></span>

3. <span data-ttu-id="88967-119">Откройте образец, созданный в [Приступая к работе](../../../../docs/framework/wcf/getting-started-tutorial.md) в Visual Studio 2012.</span><span class="sxs-lookup"><span data-stu-id="88967-119">Open the sample created in the [Getting Started Tutorial](../../../../docs/framework/wcf/getting-started-tutorial.md) in Visual Studio 2012.</span></span> <span data-ttu-id="88967-120">Обратите внимание на то, что Visual Studio 2012 следует запустить с правами администратора для создания службы.</span><span class="sxs-lookup"><span data-stu-id="88967-120">Note that you must run Visual Studio 2012 as an administrator so that the service can be created.</span></span> <span data-ttu-id="88967-121">Если у вас есть установить образцы WCF, можно открыть [Приступая к работе](../../../../docs/framework/wcf/samples/getting-started-sample.md), который содержит выполненный проект, созданный в этом руководстве.</span><span class="sxs-lookup"><span data-stu-id="88967-121">If you have the WCF samples installed, you can open the [Getting Started](../../../../docs/framework/wcf/samples/getting-started-sample.md), which contains the completed project created in the tutorial.</span></span>

4. <span data-ttu-id="88967-122">Щелкните правой кнопкой мыши **службы** проекта и выберите **добавить**, **новый элемент**.</span><span class="sxs-lookup"><span data-stu-id="88967-122">Right-click the **Service** project and select **Add**, **New Item**.</span></span> <span data-ttu-id="88967-123">Выберите **файла конфигурации приложения** и нажмите кнопку **ОК**.</span><span class="sxs-lookup"><span data-stu-id="88967-123">Select **Application Configuration File** and click **OK**.</span></span>


9. <span data-ttu-id="88967-129">В методе Main файла program.cs клиентского проекта задайте распространение идентификатора GUID трассировки в журнале событий.</span><span class="sxs-lookup"><span data-stu-id="88967-129">In the Main method in the program.cs file in the client project, set the Trace GUID to be propagated in the event log.</span></span>

    ```xml
       </System>
    ```
