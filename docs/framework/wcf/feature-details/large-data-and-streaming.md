 <span data-ttu-id="e8b72-110">Тогда как сообщения с кодированием в виде текста XML прозрачны и имеют удобочитаемый вид, двоичные сообщения часто непонятны в сравнении с ними и сложны для декодирования без специальных средств.</span><span class="sxs-lookup"><span data-stu-id="e8b72-110">While XML-text encoded messages are transparent and "human readable", binary messages are often fairly obscure in comparison and difficult to decode without tools.</span></span> <span data-ttu-id="e8b72-111">Такая разница в читаемости становится причиной игнорирования того факта, что двоичные сообщения часто содержат встроенные метаданные в полезной нагрузке, что приводит к увеличению объема служебных данных как и в случае сообщений в виде текста XML.</span><span class="sxs-lookup"><span data-stu-id="e8b72-111">This difference in legibility leads one to overlook that binary messages also often carry inline metadata in the payload, which adds overhead just as with XML text messages.</span></span> <span data-ttu-id="e8b72-112">Это особенно актуально для двоичных форматов, направленных на обеспечение возможностей слабого соединения и динамического вызова.</span><span class="sxs-lookup"><span data-stu-id="e8b72-112">This is specifically true for binary formats that aim to provide loose-coupling and dynamic invocation capabilities.</span></span>

### <a name="special-security-considerations-for-large-data"></a><span data-ttu-id="e8b72-252">Особые вопросы по безопасности данных большого объема</span><span class="sxs-lookup"><span data-stu-id="e8b72-252">Special Security Considerations for Large Data</span></span>
