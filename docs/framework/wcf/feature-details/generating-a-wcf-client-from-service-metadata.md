<span data-ttu-id="51139-104">В этом разделе рассматривается использование различных ключей в Svcutil.exe для формирования клиентов из документов метаданных.</span><span class="sxs-lookup"><span data-stu-id="51139-104">This topic describes how to use the various switches in Svcutil.exe to generate clients from metadata documents.</span></span>  
  
 <span data-ttu-id="51139-105">Документы метаданных могут находиться в устойчивом хранилище или извлекаться в оперативном режиме.</span><span class="sxs-lookup"><span data-stu-id="51139-105">Metadata documents can be on a durable storage or be retrieved online.</span></span> <span data-ttu-id="51139-106">Извлечение в оперативном режиме происходит либо по протоколу WS-MetadataExchange, либо по протоколу Microsoft Discovery (DISCO).</span><span class="sxs-lookup"><span data-stu-id="51139-106">Online retrieval follows either the WS-MetadataExchange protocol or the Microsoft Discovery (DISCO) protocol.</span></span> <span data-ttu-id="51139-107">Svcutil.exe одновременно отправляет следующие запросы метаданных для извлечения метаданных:</span><span class="sxs-lookup"><span data-stu-id="51139-107">Svcutil.exe issues the following metadata requests simultaneously to retrieve metadata:</span></span>  
  
- <span data-ttu-id="51139-108">запрос WS-MetadataExchange (MEX) по указанному адресу;</span><span class="sxs-lookup"><span data-stu-id="51139-108">WS-MetadataExchange (MEX) request to the supplied address.</span></span>  
  
- <span data-ttu-id="51139-109">запрос MEX с присоединенным ключом `/mex` по указанному адресу;</span><span class="sxs-lookup"><span data-stu-id="51139-109">MEX request to the supplied address with `/mex` appended.</span></span>

 <span data-ttu-id="51139-111">Программа Svcutil.exe формирует клиент на основании файла WSDL или файла политики, полученного от службы.</span><span class="sxs-lookup"><span data-stu-id="51139-111">Svcutil.exe generates the client based on the Web Services Description Language (WSDL) or policy file received from the service.</span></span> <span data-ttu-id="51139-112">Имя участника-пользователя (UPN) создается путем сцепления имени пользователя с " \@ " и добавления полного доменного имени (FQDN).</span><span class="sxs-lookup"><span data-stu-id="51139-112">The user principal name (UPN) is generated by concatenating the user name with "\@" and then adding a fully-qualified domain name (FQDN).</span></span> <span data-ttu-id="51139-113">Однако для пользователей, которые зарегистрировались в Active Directory, этот формат является недопустимым, и имя участника-пользователя, которое создает средство, вызывает сбой в проверке подлинности Kerberos со следующим сообщением об ошибке: **попытка входа завершилась неудачей.**</span><span class="sxs-lookup"><span data-stu-id="51139-113">However, for users who registered on Active Directory, this format is not valid and the UPN that the tool generates causes a failure in the Kerberos authentication with the following error message: **The logon attempt failed.**</span></span> <span data-ttu-id="51139-114">Чтобы устранить эту проблему, необходимо вручную откорректировать файл клиента, сформированный Svcutil.exe.</span><span class="sxs-lookup"><span data-stu-id="51139-114">To resolve this problem, manually fix the client file that the tool generated.</span></span>  
  
```console

```
