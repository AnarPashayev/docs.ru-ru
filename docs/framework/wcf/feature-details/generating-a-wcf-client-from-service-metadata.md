# <a name="generating-a-wcf-client-from-service-metadata"></a><span data-ttu-id="251f9-102">Создание клиента WCF из метаданных службы</span><span class="sxs-lookup"><span data-stu-id="251f9-102">Generating a WCF Client from Service Metadata</span></span>
<span data-ttu-id="251f9-103">В этом разделе рассматривается использование различных ключей в Svcutil.exe для формирования клиентов из документов метаданных.</span><span class="sxs-lookup"><span data-stu-id="251f9-103">This topic describes how to use the various switches in Svcutil.exe to generate clients from metadata documents.</span></span>  
  
 <span data-ttu-id="251f9-104">Документы метаданных могут находиться в устойчивом хранилище или извлекаться в оперативном режиме.</span><span class="sxs-lookup"><span data-stu-id="251f9-104">Metadata documents can be on a durable storage or be retrieved online.</span></span> <span data-ttu-id="251f9-105">Извлечение в оперативном режиме происходит либо по протоколу WS-MetadataExchange, либо по протоколу Microsoft Discovery (DISCO).</span><span class="sxs-lookup"><span data-stu-id="251f9-105">Online retrieval follows either the WS-MetadataExchange protocol or the Microsoft Discovery (DISCO) protocol.</span></span> <span data-ttu-id="251f9-106">Svcutil.exe одновременно отправляет следующие запросы метаданных для извлечения метаданных:</span><span class="sxs-lookup"><span data-stu-id="251f9-106">Svcutil.exe issues the following metadata requests simultaneously to retrieve metadata:</span></span>  
  
- <span data-ttu-id="251f9-107">запрос WS-MetadataExchange (MEX) по указанному адресу;</span><span class="sxs-lookup"><span data-stu-id="251f9-107">WS-MetadataExchange (MEX) request to the supplied address.</span></span>  
  
- <span data-ttu-id="251f9-108">запрос MEX с присоединенным ключом `/mex` по указанному адресу;</span><span class="sxs-lookup"><span data-stu-id="251f9-108">MEX request to the supplied address with `/mex` appended.</span></span>  
  
- <span data-ttu-id="251f9-109">Запрос DISCO (с использованием [дисковериклиентпротокол](https://go.microsoft.com/fwlink/?LinkId=94777) из веб-служб ASP.NET) к заданному адресу.</span><span class="sxs-lookup"><span data-stu-id="251f9-109">DISCO request (using the [DiscoveryClientProtocol](https://go.microsoft.com/fwlink/?LinkId=94777) from ASP.NET Web services) to the supplied address.</span></span>  
  
 <span data-ttu-id="251f9-110">Программа Svcutil.exe формирует клиент на основании файла WSDL или файла политики, полученного от службы.</span><span class="sxs-lookup"><span data-stu-id="251f9-110">Svcutil.exe generates the client based on the Web Services Description Language (WSDL) or policy file received from the service.</span></span> <span data-ttu-id="251f9-111">Имя участника-пользователя (UPN) создается путем сцепления имени пользователя с "\@" и добавления полного доменного имени (FQDN).</span><span class="sxs-lookup"><span data-stu-id="251f9-111">The user principal name (UPN) is generated by concatenating the user name with "\@" and then adding a fully-qualified domain name (FQDN).</span></span> <span data-ttu-id="251f9-112">Однако для пользователей, которые зарегистрировались в Active Directory, этот формат является недопустимым, и имя участника-пользователя, создаваемое средством, вызывает сбой в проверке подлинности Kerberos со следующим сообщением об ошибке: **Сбой попытки входа.**</span><span class="sxs-lookup"><span data-stu-id="251f9-112">However, for users who registered on Active Directory, this format is not valid and the UPN that the tool generates causes a failure in the Kerberos authentication with the following error message: **The logon attempt failed.**</span></span> <span data-ttu-id="251f9-113">Чтобы устранить эту проблему, необходимо вручную откорректировать файл клиента, сформированный Svcutil.exe.</span><span class="sxs-lookup"><span data-stu-id="251f9-113">To resolve this problem, manually fix the client file that the tool generated.</span></span>  
  
```console

```
