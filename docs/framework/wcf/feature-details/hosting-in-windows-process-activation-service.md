<span data-ttu-id="24c9d-103">Служба активации процессов Windows (WAS) управляет активацией и временем существования рабочих процессов, содержащих приложения, размещения служб Windows Communication Foundation (WCF).</span><span class="sxs-lookup"><span data-stu-id="24c9d-103">The Windows Process Activation Service (WAS) manages the activation and lifetime of the worker processes that contain applications that host Windows Communication Foundation (WCF) services.</span></span> <span data-ttu-id="24c9d-104">Модель процесса WAS обобщает модель процесса IIS 6.0 для HTTP-сервера, устраняя зависимость от HTTP.</span><span class="sxs-lookup"><span data-stu-id="24c9d-104">The WAS process model generalizes the IIS 6.0 process model for the HTTP server by removing the dependency on HTTP.</span></span> <span data-ttu-id="24c9d-105">Это позволяет службам WCF использовать как HTTP, так и отличные от HTTP протоколы, такие как Net.TCP, в среде размещения, которая поддерживает активацию на основе сообщений и предоставляет возможность размещать большое количество приложений на данном компьютере.</span><span class="sxs-lookup"><span data-stu-id="24c9d-105">This allows WCF services to use both HTTP and non-HTTP protocols, such as Net.TCP, in a hosting environment that supports message-based activation and offers the ability to host a large number of applications on a given machine.</span></span>  
  
 <span data-ttu-id="24c9d-106">Дополнительные сведения о создании службы WCF, которая выполняется в среде размещения WAS, см. в разделе [как: Размещение службы WCF в WAS](../../../../docs/framework/wcf/feature-details/how-to-host-a-wcf-service-in-was.md).</span><span class="sxs-lookup"><span data-stu-id="24c9d-106">For more information about building a WCF service that runs in the WAS hosting environment, see [How to: Host a WCF Service in WAS](../../../../docs/framework/wcf/feature-details/how-to-host-a-wcf-service-in-was.md).</span></span>  
  
 <span data-ttu-id="24c9d-107">Модель процессов WAS предоставляет несколько функций, благодаря которым размещение приложений можно осуществлять более надежным и управляемым способом и при этом более эффективно использовать ресурсы.</span><span class="sxs-lookup"><span data-stu-id="24c9d-107">The WAS process model provides several features that enable applications to be hosted in a way that is more robust, more manageable, and that uses resources efficiently:</span></span>

- <span data-ttu-id="24c9d-109">Надежный перезапуск приложений и рабочих процессов для поддержания состояния выполняемых приложений.</span><span class="sxs-lookup"><span data-stu-id="24c9d-109">Robust application and worker process recycling to maintain the health of running applications.</span></span>  
  
- <span data-ttu-id="24c9d-110">Централизованные настройка приложений и управление ими.</span><span class="sxs-lookup"><span data-stu-id="24c9d-110">Centralized application configuration and management.</span></span>  
  
- <span data-ttu-id="24c9d-111">Возможность использования приложениями модели процессов IIS без развертывания полной установки IIS.</span><span class="sxs-lookup"><span data-stu-id="24c9d-111">Allows applications to take advantage of the IIS process model without requiring the deployment footprint of a full IIS installation.</span></span>  
<span data-ttu-id="24c9d-112">[Windows Server AppFabric](https://go.microsoft.com/fwlink/?LinkId=196496) работает с IIS 7.0 и служба активации процессов Windows (WAS), чтобы предоставить среду для служб NET4 WCF и WF для размещения функционально насыщенных приложений.</span><span class="sxs-lookup"><span data-stu-id="24c9d-112">[Windows Server AppFabric](https://go.microsoft.com/fwlink/?LinkId=196496) works with IIS 7.0 and Windows Process Activation Service (WAS) to provide a rich application hosting environment for NET4 WCF and WF services.</span></span> <span data-ttu-id="24c9d-113">К ее преимуществам относятся управление жизненным циклом, перезапуск процессов, совместное размещение, быстрая защита от сбоев, обработка потерянных процессов, активация по запросу и наблюдение за работоспособностью.</span><span class="sxs-lookup"><span data-stu-id="24c9d-113">These benefits include process life-cycle management, process recycling, shared hosting, rapid failure protection, process orphaning, on-demand activation, and health monitoring.</span></span> <span data-ttu-id="24c9d-114">Подробные сведения см. в разделе [AppFabric Hosting Features](https://go.microsoft.com/fwlink/?LinkId=196494) и [основы размещения AppFabric](https://go.microsoft.com/fwlink/?LinkId=196495).</span><span class="sxs-lookup"><span data-stu-id="24c9d-114">For detailed information, see [AppFabric Hosting Features](https://go.microsoft.com/fwlink/?LinkId=196494) and [AppFabric Hosting Concepts](https://go.microsoft.com/fwlink/?LinkId=196495).</span></span>
