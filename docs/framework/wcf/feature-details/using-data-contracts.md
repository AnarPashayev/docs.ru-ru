# <a name="using-data-contracts"></a><span data-ttu-id="74c68-103">Использование контрактов данных</span><span class="sxs-lookup"><span data-stu-id="74c68-103">Using Data Contracts</span></span>
<span data-ttu-id="74c68-104">*Контракт данных* - формальное соглашение между службой и клиентом, абстрактно описывающее данные, обмен которыми происходит.</span><span class="sxs-lookup"><span data-stu-id="74c68-104">A *data contract* is a formal agreement between a service and a client that abstractly describes the data to be exchanged.</span></span> <span data-ttu-id="74c68-105">Это значит, что для взаимодействия клиент и служба не обязаны совместно использовать одни и те же типы, достаточно совместно использовать одни и те же контракты данных.</span><span class="sxs-lookup"><span data-stu-id="74c68-105">That is, to communicate, the client and the service do not have to share the same types, only the same data contracts.</span></span> <span data-ttu-id="74c68-106">Контракт данных для каждого параметра и возвращаемого типа четко определяет, какие данные сериализуются (превращаются в XML) для обмена.</span><span class="sxs-lookup"><span data-stu-id="74c68-106">A data contract precisely defines, for each parameter or return type, what data is serialized (turned into XML) to be exchanged.</span></span>
