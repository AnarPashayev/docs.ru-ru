# <a name="soap-and-http-endpoints"></a>Конечные точки SOAP и HTTP
В этом примере показано, как реализовать службу на основе RPC и выставить ее в формате SOAP и формате "Plain Old XML" (POX) с использованием модели веб-программирования WCF. Более подробную информацию о привязке HTTP для службы можно получить в базовом примере [службы HTTP.](../../../../docs/framework/wcf/samples/basic-http-service.md) В данном образце акцент сделан на особенностях предоставления одной и той же службы через протокол SOAP и HTTP с использованием разных привязок.  
  
## <a name="demonstrates"></a>Что демонстрирует  
 Разоблачение rPC-сервиса через SOAP и HTTP с помощью WCF.  
  
## <a name="discussion"></a>Обсуждение  
 Этот пример состоит из двух компонентов: проекта web-приложений (Service), который содержит службу WCF и консольное приложение (Клиент), которое вызывает операции службы с использованием привязок SOAP и HTTP.  
  
 Служба WCF предоставляет 2`GetData` операции `PutData` - и - что эхо строки, которая была принята в качестве ввода. Операции службы помечены атрибутами <xref:System.ServiceModel.Web.WebGetAttribute> и <xref:System.ServiceModel.Web.WebInvokeAttribute>. Эти атрибуты управляют HTTP-проекцией операций. Кроме того, они помечены атрибутом <xref:System.ServiceModel.OperationContractAttribute>, который позволяет предоставлять их через привязки протокола SOAP. Метод службы `PutData` вызывает исключение <xref:System.ServiceModel.Web.WebFaultException>, которое отправляется обратно через HTTP с использованием кода состояния HTTP, а через SOAP как ошибка SOAP.  
  
 Файл Web.config настраивает службу WCF с 3 конечными точками:  
  
- Конечная точка ~/service.svc/mex предоставляет доступ к метаданным службы клиентам на основе SOAP.  
  
- Конечная точка ~/service.svc/http позволяет клиентам получать доступ к службе с использованием привязки HTTP.  
  
- Конечная точка ~/service.svc/soap позволяет клиентам получать доступ к службе с использованием привязки протокола SOAP.  
  
 Конечная точка HTTP настроена `webHttp` с <> `helpEnabled` стандартной `true`конечную точку, которая установлена на. В результате этого служба предоставляет справочную страницу на основе XHTML по адресу ~/service.svc/http/help, которую клиенты на основе HTTP могут использовать для доступа к службе.  
  
 Клиентский проект демонстрирует доступ к службе с помощью прокси-сервера SOAP <xref:System.Net.WebClient>(генерируемого через Add Service **Reference)** и доступ к сервису с помощью.  
  
 Образец состоит из службы, размещенной на веб-сервере, и консольного приложения. Во время выполнения консольного приложения клиент совершает запросы к службе и выводит в окно консоли нужные сведения из ответов.  
  
#### <a name="to-run-the-sample"></a>Запуск образца  
  
1. Откройте решение для образца «SOAP and HTTP Endpoints».  
  
2. Чтобы построить решение, нажмите CTRL+SHIFT+B.  
  
3. Если он еще не открыт, нажмите CTRL-W, S, чтобы открыть окно **Solution Explorer.**  
  
4. Из окна **Solution Explorer** нажмите правой кнопкой мыши проекта **Службы** и поместите курсор над опцией контекстного меню **Debug,** чтобы появится меню контекста **Start New Instance.** Нажмите **Кнопка Начало Новой инстанции**. Запускается сервер разработки ASP.NET, на котором размещается служба.  
  
5. Из окон Solution Explorer нажмите правой кнопкой мыши проекта Клиента и поместите курсор над опцией контекстного меню **Debug,** чтобы появится контекстное меню **Start New Instance.** Нажмите **Кнопка Начало Новой инстанции**.  
  
6. На клиенте открывается окно консоли с URI запущенной службы и URI HTML-страницы справки для запущенной службы. HTML-страницу справки можно просмотреть в любой момент времени, введя URI этой страницы в браузере.  
  
7. Во время работы образца клиент записывает состояние текущего действия.  
  
8. Чтобы завершить клиентское консольное приложение, нажмите любую клавишу.  
  
9. Чтобы прекратить отладку службы, нажмите клавиши SHIFT+F5.  
  
10. В зоне уведомлений Windows нажмите правой кнопкой мыши значок сервера ASP.NET разработки и выберите **«Стоп»** из контекстного меню.
