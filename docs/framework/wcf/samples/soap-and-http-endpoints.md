## <a name="discussion"></a><span data-ttu-id="859a7-108">Обсуждение</span><span class="sxs-lookup"><span data-stu-id="859a7-108">Discussion</span></span>  
 <span data-ttu-id="859a7-109">Этот образец состоит из двух компонентов: проекта веб-приложения (служба), содержащий службы WCF и консольное приложение (клиент), которое вызывает операции службы с помощью привязки SOAP и HTTP.</span><span class="sxs-lookup"><span data-stu-id="859a7-109">This sample consists of two components: a Web Application project (Service) that contains a WCF service and a console application (Client) that invokes service operations using SOAP and HTTP bindings.</span></span>  
  
 <span data-ttu-id="859a7-110">Служба WCF предоставляет 2 операции-`GetData` и `PutData` — которые повторяют строку, которая была передана в качестве входных данных.</span><span class="sxs-lookup"><span data-stu-id="859a7-110">The WCF service exposes 2 operations –`GetData` and `PutData` – that echo the string that was passed as input.</span></span> <span data-ttu-id="859a7-111">Операции службы помечены атрибутами <xref:System.ServiceModel.Web.WebGetAttribute> и <xref:System.ServiceModel.Web.WebInvokeAttribute>.</span><span class="sxs-lookup"><span data-stu-id="859a7-111">The service operations are annotated with <xref:System.ServiceModel.Web.WebGetAttribute> and <xref:System.ServiceModel.Web.WebInvokeAttribute>.</span></span> <span data-ttu-id="859a7-112">Эти атрибуты управляют HTTP-проекцией операций.</span><span class="sxs-lookup"><span data-stu-id="859a7-112">These attributes control the HTTP projection of these operations.</span></span> <span data-ttu-id="859a7-113">Кроме того, они помечены атрибутом <xref:System.ServiceModel.OperationContractAttribute>, который позволяет предоставлять их через привязки протокола SOAP.</span><span class="sxs-lookup"><span data-stu-id="859a7-113">In addition, they are annotated with <xref:System.ServiceModel.OperationContractAttribute>, which enables them to be exposed over SOAP bindings.</span></span> <span data-ttu-id="859a7-114">Метод службы `PutData` вызывает исключение <xref:System.ServiceModel.Web.WebFaultException>, которое отправляется обратно через HTTP с использованием кода состояния HTTP, а через SOAP как ошибка SOAP.</span><span class="sxs-lookup"><span data-stu-id="859a7-114">The service’s `PutData` method throws a <xref:System.ServiceModel.Web.WebFaultException>, which gets sent back over HTTP using the HTTP status code and gets sent back over SOAP as a SOAP fault.</span></span>  
  
 <span data-ttu-id="859a7-115">Файл Web.config Конфигурирует службы WCF с конечными точками, 3:</span><span class="sxs-lookup"><span data-stu-id="859a7-115">The Web.config file configures the WCF service with 3 endpoints:</span></span>  
  
- <span data-ttu-id="859a7-116">Конечная точка ~/service.svc/mex предоставляет доступ к метаданным службы клиентам на основе SOAP.</span><span class="sxs-lookup"><span data-stu-id="859a7-116">The ~/service.svc/mex endpoint that exposes the service metadata for access by SOAP-based clients.</span></span>  
  
- <span data-ttu-id="859a7-117">Конечная точка ~/service.svc/http позволяет клиентам получать доступ к службе с использованием привязки HTTP.</span><span class="sxs-lookup"><span data-stu-id="859a7-117">The ~/service.svc/http endpoint that enables clients to access the service using the HTTP binding.</span></span>  
  
- <span data-ttu-id="859a7-118">Конечная точка ~/service.svc/soap позволяет клиентам получать доступ к службе с использованием привязки протокола SOAP.</span><span class="sxs-lookup"><span data-stu-id="859a7-118">The ~/service.svc/soap endpoint that allows the clients to access the service using the SOAP over HTTP binding.</span></span>  
  
 <span data-ttu-id="859a7-119">Конечная точка HTTP настроена с помощью <`webHttp`> Стандартная конечная точка, имеющая `helpEnabled` присвоено `true`.</span><span class="sxs-lookup"><span data-stu-id="859a7-119">The HTTP endpoint is configured with a <`webHttp`> standard endpoint which has `helpEnabled` set to `true`.</span></span> <span data-ttu-id="859a7-120">В результате этого служба предоставляет справочную страницу на основе XHTML по адресу ~/service.svc/http/help, которую клиенты на основе HTTP могут использовать для доступа к службе.</span><span class="sxs-lookup"><span data-stu-id="859a7-120">As a result, the service exposes an XHTML based help page at ~/service.svc/http/help that HTTP-based clients can use to access the service.</span></span>  
  
 <span data-ttu-id="859a7-121">Клиентский проект демонстрирует доступ к службе, используя прокси SOAP (созданный с помощью **Add Service Reference**) и доступ к службе с помощью <xref:System.Net.WebClient>.</span><span class="sxs-lookup"><span data-stu-id="859a7-121">The client project demonstrates accessing the service using a SOAP proxy (generated through **Add Service Reference**) and accessing the service using <xref:System.Net.WebClient>.</span></span>  
  
 <span data-ttu-id="859a7-122">Образец состоит из службы, размещенной на веб-сервере, и консольного приложения.</span><span class="sxs-lookup"><span data-stu-id="859a7-122">The sample consists of a Web-hosted service and a console application.</span></span> <span data-ttu-id="859a7-123">Во время выполнения консольного приложения клиент совершает запросы к службе и выводит в окно консоли нужные сведения из ответов.</span><span class="sxs-lookup"><span data-stu-id="859a7-123">As the console application runs, the client makes requests to the service and writes the pertinent information from the responses to the console window.</span></span>
