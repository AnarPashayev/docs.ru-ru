 <span data-ttu-id="e4605-109">WCF реализует поставщик WMI, компонент, который предоставляет инструментирование во время выполнения через интерфейс, совместимый с WBEM.</span><span class="sxs-lookup"><span data-stu-id="e4605-109">WCF implements a WMI provider, a component that exposes instrumentation at runtime through a WBEM-compatible interface.</span></span> <span data-ttu-id="e4605-110">Средства управления могут подключаться к службам через интерфейс во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="e4605-110">Management tools can connect to the services through the interface at runtime.</span></span> <span data-ttu-id="e4605-111">WCF предоставляет атрибуты служб, такие как адреса, привязки, поведения и прослушиватели.</span><span class="sxs-lookup"><span data-stu-id="e4605-111">WCF exposes attributes of services such as addresses, bindings, behaviors, and listeners.</span></span>  
  
 <span data-ttu-id="e4605-112">Встроенный поставщик инструментария WMI активируется в файле конфигурации приложения.</span><span class="sxs-lookup"><span data-stu-id="e4605-112">The built-in WMI provider is activated in the configuration file of the application.</span></span> <span data-ttu-id="e4605-113">Это делается с помощью `wmiProviderEnabled` атрибута [ \<> диагностики](../../../../docs/framework/configure-apps/file-schema/wcf/diagnostics.md) в [ \<разделе > System.
