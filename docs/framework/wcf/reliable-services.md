<span data-ttu-id="5b006-103">Очереди и надежные сеансы — это функции Windows Communication Foundation (WCF), которые реализуют надежный обмен сообщениями.</span><span class="sxs-lookup"><span data-stu-id="5b006-103">Queues and reliable sessions are the Windows Communication Foundation (WCF) features that implement reliable messaging.</span></span> <span data-ttu-id="5b006-104">В этом разделе объясняются функции надежного обмена сообщениями в WCF.</span><span class="sxs-lookup"><span data-stu-id="5b006-104">This topic explains the reliable messaging features of WCF.</span></span>  
  
 <span data-ttu-id="5b006-105">*Надежный обмен* сообщениями заключается в том, как источник надежного обмена сообщениями (называемый *источником*) надежно передает сообщения в место назначения надежного обмена сообщениями (называемое *назначением).*</span><span class="sxs-lookup"><span data-stu-id="5b006-105">*Reliable messaging* is how a reliable messaging source (called the *source*) transfers messages reliably to a reliable messaging destination (called the *destination*).</span></span>  
  
 <span data-ttu-id="5b006-106">Ниже перечислены функции, выполняемые при надежном обмене сообщениями.</span><span class="sxs-lookup"><span data-stu-id="5b006-106">Reliable messaging performs the following functions:</span></span>

- <span data-ttu-id="5b006-108">Отделение источника от назначения.</span><span class="sxs-lookup"><span data-stu-id="5b006-108">Separates the source and the destination from each other.</span></span> <span data-ttu-id="5b006-109">Это обеспечивает независимость сбоев и восстановлений источника и назначения, а также надежность передачи и доставки сообщений даже при недоступности источника или назначения.</span><span class="sxs-lookup"><span data-stu-id="5b006-109">This provides independent failure and recovery of the source and the destination, as well as reliable transfer and delivery of messages, even when the source or destination is unavailable.</span></span>  
  
 <span data-ttu-id="5b006-110">Надежность обмена сообщениями часто обеспечивается ценой высокой задержки.</span><span class="sxs-lookup"><span data-stu-id="5b006-110">Reliable messaging frequently comes at the cost of high latency.</span></span> <span data-ttu-id="5b006-111">*Задержка* — это время, затрачиваемое на то, что сообщение достигает места назначения из источника.</span><span class="sxs-lookup"><span data-stu-id="5b006-111">*Latency* is the time it takes for the message to reach the destination from the source.</span></span> <span data-ttu-id="5b006-112">Таким образом, WCF предоставляет следующие типы надежного обмена сообщениями:</span><span class="sxs-lookup"><span data-stu-id="5b006-112">WCF, therefore, provides the following types of reliable messaging:</span></span>  
  
- <span data-ttu-id="5b006-113">[Надежные сеансы](./feature-details/reliable-sessions.md), которые обеспечивают надежную пересылку без затрат на высокую задержку.</span><span class="sxs-lookup"><span data-stu-id="5b006-113">[Reliable Sessions](./feature-details/reliable-sessions.md), which offers reliable transfer without the cost of high latency.</span></span>  
  
- <span data-ttu-id="5b006-114">[Очереди в WCF](./feature-details/queues-in-wcf.md), обеспечивающие надежную передачу и разделение между источником и назначением.</span><span class="sxs-lookup"><span data-stu-id="5b006-114">[Queues in WCF](./feature-details/queues-in-wcf.md), which offers both reliable transfers and separation between the source and the destination.</span></span>  
  
## <a name="reliable-sessions"></a><span data-ttu-id="5b006-115">Надежные сеансы</span><span class="sxs-lookup"><span data-stu-id="5b006-115">Reliable Sessions</span></span>  

 <span data-ttu-id="5b006-116">Надежные сеансы обеспечивают надежную сквозную передачу сообщений между источником и назначением с помощью протокола WS-Reliable Messaging независимо от количества и типа посредников между конечными точками обмена сообщениями (источником и назначением),</span><span class="sxs-lookup"><span data-stu-id="5b006-116">Reliable sessions provide end-to-end reliable transfer of messages between a source and a destination using the WS-Reliable Messaging protocol, regardless of the number or type of intermediaries that separate the messaging (source and destination) endpoints.</span></span> <span data-ttu-id="5b006-117">В него входят любые транспортные посредники, не использующие протокол SOAP (например, прокси-серверы HTTP), и посредники, использующие этот протокол (например, мосты и маршрутизаторы на базе SOAP), необходимые для продвижения сообщений между конечными точками.</span><span class="sxs-lookup"><span data-stu-id="5b006-117">This includes any transport intermediaries that do not use SOAP (for example, HTTP proxies) or intermediaries that use SOAP (for example, SOAP-based routers or bridges) that are required for messages to flow between the endpoints.</span></span> <span data-ttu-id="5b006-118">Надежные сеансы используют окно передачи в памяти для маскировки сбоев на уровне сообщений SOAP и повторной установки соединений при транспортных сбоях.</span><span class="sxs-lookup"><span data-stu-id="5b006-118">Reliable sessions use an in-memory transfer window to mask SOAP message-level failures and re-establish connections in the case of transport failures.</span></span>  
  
 <span data-ttu-id="5b006-119">Надежные сеансы обеспечивают надежную передачу сообщений с низкой задержкой.</span><span class="sxs-lookup"><span data-stu-id="5b006-119">Reliable sessions provide low-latency reliable message transfers.</span></span> <span data-ttu-id="5b006-120">Они обеспечивают доставку сообщений протокола SOAP через любые прокси или маршрутизаторы подобно тому, как протокол TCP обеспечивает доставку пакетов через IP-мосты.</span><span class="sxs-lookup"><span data-stu-id="5b006-120">They provide for SOAP messages over any proxies or intermediaries, equivalent to what TCP provides for packets over IP bridges.</span></span> <span data-ttu-id="5b006-121">Дополнительные сведения о надежных сеансах см. в разделе [Надежные сеансы](./feature-details/reliable-sessions.md).</span><span class="sxs-lookup"><span data-stu-id="5b006-121">For more information about reliable sessions, see [Reliable Sessions](./feature-details/reliable-sessions.md).</span></span>  
  
### <a name="queues"></a><span data-ttu-id="5b006-122">Очереди</span><span class="sxs-lookup"><span data-stu-id="5b006-122">Queues</span></span>  

 <span data-ttu-id="5b006-123">Очереди в WCF обеспечивают надежную передачу сообщений и разделение между источниками и местами назначения за счет высокой задержки.</span><span class="sxs-lookup"><span data-stu-id="5b006-123">Queues in WCF provide both reliable transfers of messages and separation between sources and destinations at the cost of high latency.</span></span> <span data-ttu-id="5b006-124">Взаимодействие в очереди WCF построено на основе очереди сообщений (MSMQ).</span><span class="sxs-lookup"><span data-stu-id="5b006-124">WCF queued communication is built on top of Message Queuing (MSMQ).</span></span>  
  
 <span data-ttu-id="5b006-125">MSMQ поставляется как дополнительный компонент с Windows.</span><span class="sxs-lookup"><span data-stu-id="5b006-125">MSMQ ships as an optional component with Windows.</span></span> <span data-ttu-id="5b006-126">Служба MSMQ работает как служба Windows.</span><span class="sxs-lookup"><span data-stu-id="5b006-126">The MSMQ service runs as a Windows Service.</span></span> <span data-ttu-id="5b006-127">MSMQ помещает передаваемые сообщения в очередь передачи от имени источника и доставляет их в целевую очередь.</span><span class="sxs-lookup"><span data-stu-id="5b006-127">It captures messages for transmission in a transmission queue on behalf of the source and delivers it to a target queue.</span></span> <span data-ttu-id="5b006-128">Целевая очередь принимает сообщения от имени назначения для последующей доставки при получении запроса от назначения.</span><span class="sxs-lookup"><span data-stu-id="5b006-128">The target queue accepts messages on behalf of the destination for later delivery whenever the destination requests messages.</span></span> <span data-ttu-id="5b006-129">Диспетчеры MSMQ реализуют протокол надежной передачи сообщений, благодаря которому сообщения не теряются при передаче.</span><span class="sxs-lookup"><span data-stu-id="5b006-129">The MSMQ managers implement a reliable message-transfer protocol so that messages are not lost in transmission.</span></span> <span data-ttu-id="5b006-130">Этот протокол может быть собственным или основанным на SOAP (протокол SRMP).</span><span class="sxs-lookup"><span data-stu-id="5b006-130">The protocol can be native or a SOAP-based protocol called SOAP Reliable Messaging Protocol (SRMP).</span></span>
