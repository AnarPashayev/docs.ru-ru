# <a name="windows-communication-foundation-bindings"></a>Привязки Windows Communication Foundation
Привязки указывают, как конечную точку службы Windows Communication Foundation (WCF) взаимодействует с другими конечными точками. В привязке должен быть указан как минимум используемый транспорт (например, HTTP или TCP). Через привязки также можно задать другие характеристики, такие как поддержка безопасности и транзакций.  
  
## <a name="in-this-section"></a>В этом разделе  
 [Общие сведения о привязках WCF](../../../docs/framework/wcf/bindings-overview.md)  
 Выполните обзор привязки WCF, какие привязки, система предоставляет и как можно определить или изменить их.  
  
 [Привязки, предоставляемые системой](../../../docs/framework/wcf/system-provided-bindings.md)  
 Список привязок, включенных в WCF. Эти привязки охватывают большую часть требований по безопасности и шаблонам сообщений.  
  
 [Использование привязок для настройки служб и клиентов](../../../docs/framework/wcf/using-bindings-to-configure-services-and-clients.md)  
 Привязка WCF содержит важные сведения, которые клиенты должны использовать для подключения к конечным точкам службы.  
  
 [Настройка привязок для служб](../../../docs/framework/wcf/configuring-bindings-for-wcf-services.md)  
 С помощью конфигурации администраторы и установщики могут настраивать привязки для конечных точек службы.
