## <a name="remarks"></a><span data-ttu-id="953de-132">Заметки</span><span class="sxs-lookup"><span data-stu-id="953de-132">Remarks</span></span>  
 <span data-ttu-id="953de-133">Объект `ICorDebugEval` создается в контексте конкретного потока, используемого для выполнения оценок.</span><span class="sxs-lookup"><span data-stu-id="953de-133">An `ICorDebugEval` object is created in the context of a specific thread that is used to perform the evaluations.</span></span> <span data-ttu-id="953de-134">Все объекты и типы, используемые в данной оценке, должны находиться в одном домене приложения.</span><span class="sxs-lookup"><span data-stu-id="953de-134">All objects and types used in a given evaluation must reside within the same application domain.</span></span> <span data-ttu-id="953de-135">Этот домен приложения не должен совпадать с именем текущего домена приложения потока.</span><span class="sxs-lookup"><span data-stu-id="953de-135">That application domain need not be the same as the current application domain of the thread.</span></span> <span data-ttu-id="953de-136">Вычисления могут быть вложенными.</span><span class="sxs-lookup"><span data-stu-id="953de-136">Evaluations can be nested.</span></span>  
  
 <span data-ttu-id="953de-137">Операции оценки не завершаются до тех пор, пока отладчик не вызовет [ICorDebugController:: Continue](icordebugcontroller-continue-method.md), а затем получает обратный вызов [ICorDebugManagedCallback:: евалкомплете](icordebugmanagedcallback-evalcomplete-method.md) .</span><span class="sxs-lookup"><span data-stu-id="953de-137">The evaluation's operations do not complete until the debugger calls [ICorDebugController::Continue](icordebugcontroller-continue-method.md), and then receives an [ICorDebugManagedCallback::EvalComplete](icordebugmanagedcallback-evalcomplete-method.md) callback.</span></span> <span data-ttu-id="953de-138">Если необходимо использовать функцию оценки, не разрешая выполнение других потоков, приостановите потоки с помощью [ICorDebugController:: SetAllThreadsDebugState](icordebugcontroller-setallthreadsdebugstate-method.md) или [ICorDebugController:: Остановите](icordebugcontroller-stop-method.md) перед вызовом [ICorDebugController:: Continue](icordebugcontroller-continue-method.md).</span><span class="sxs-lookup"><span data-stu-id="953de-138">If you need to use the evaluation functionality without allowing other threads to run, suspend the threads by using either [ICorDebugController::SetAllThreadsDebugState](icordebugcontroller-setallthreadsdebugstate-method.md) or [ICorDebugController::Stop](icordebugcontroller-stop-method.md) before calling [ICorDebugController::Continue](icordebugcontroller-continue-method.md).</span></span>  
  
 <span data-ttu-id="953de-139">Поскольку пользовательский код выполняется при выполнении оценки, могут возникать любые события отладки, включая загрузку классов и точки останова.</span><span class="sxs-lookup"><span data-stu-id="953de-139">Because user code is running when the evaluation is in progress, any debug events can occur, including class loads and breakpoints.</span></span> <span data-ttu-id="953de-140">Для этих событий отладчик будет принимать обратные вызовы, как обычные.</span><span class="sxs-lookup"><span data-stu-id="953de-140">The debugger will receive callbacks, as normal, for these events.</span></span> <span data-ttu-id="953de-141">Состояние оценки будет рассматриваться как часть проверки нормального состояния программы.</span><span class="sxs-lookup"><span data-stu-id="953de-141">The state of the evaluation will be seen as part of the inspection of the normal program state.</span></span> <span data-ttu-id="953de-142">Цепочка стека будет представлять собой цепочку `CHAIN_FUNC_EVAL` (см. перечисление "Кордебугстепреасон" и метод [ICorDebugChain::-Reason](icordebugchain-getreason-method.md) ).</span><span class="sxs-lookup"><span data-stu-id="953de-142">The stack chain will be a `CHAIN_FUNC_EVAL` chain (see the "CorDebugStepReason" enumeration and the [ICorDebugChain::GetReason](icordebugchain-getreason-method.md) method).</span></span> <span data-ttu-id="953de-143">Полный API отладчика продолжит работать в нормальном режиме.</span><span class="sxs-lookup"><span data-stu-id="953de-143">The full debugger API will continue to operate as normal.</span></span>  
  
 <span data-ttu-id="953de-144">В случае возникновения взаимоблокировки или бесконечной циклической ситуации код пользователя может никогда не завершиться.</span><span class="sxs-lookup"><span data-stu-id="953de-144">If a deadlocked or infinite looping situation arises, the user code may never complete.</span></span> <span data-ttu-id="953de-145">В этом случае необходимо вызвать метод [ICorDebugEval:: Abort](icordebugeval-abort-method.md) , прежде чем возобновить работу программы.</span><span class="sxs-lookup"><span data-stu-id="953de-145">In such a case, you must call [ICorDebugEval::Abort](icordebugeval-abort-method.md) before resuming the program.</span></span>
