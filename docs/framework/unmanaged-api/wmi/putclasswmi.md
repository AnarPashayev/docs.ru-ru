# <a name="putclasswmi-function"></a>Функция Путклассвми

Создает новый класс или обновляет существующий.

[!INCLUDE[internalonly-unmanaged](../../../../includes/internalonly-unmanaged.md)]

## <a name="syntax"></a>Синтаксис

```cpp
HRESULT PutClassWmi (
   [in] IWbemClassObject*    pObject,
   [in] long                 lFlags,
   [in] IWbemContext*        pCtx,
   [out] IWbemCallResult**   ppCallResult
);
```

## <a name="parameters"></a>Параметры

`pObject`\
окне Указатель на допустимое определение класса. Она должна быть правильно инициализирована всеми необходимыми значениями свойств.

`lFlags`\
окне Сочетание флагов, влияющих на поведение этой функции. Следующие значения определены в файле заголовка *вбемкли. h* , или их можно определить как константы в коде:

|Константа  |Значение  |Описание  |
|---------|---------|---------|
| `WBEM_FLAG_USE_AMENDED_QUALIFIERS` | 0x20000 | Если задано, Инструментарий WMI не сохраняет никаких квалификаторов с измененной разновидностью. <br> Если не задано, предполагается, что этот объект не локализован, а все квалификаторы хранятся в этом экземпляре. |
| `WBEM_FLAG_CREATE_OR_UPDATE` | 0 | Создайте класс, если он не существует, или перезапишите его, если он уже существует. |
| `WBEM_FLAG_UPDATE_ONLY` | 1 | Обновите класс. Для успешного вызова класс должен существовать. |
| `WBEM_FLAG_CREATE_ONLY` | 2 | Создайте класс. Вызов завершается ошибкой, если класс уже существует. |
| `WBEM_FLAG_RETURN_IMMEDIATELY` | 0x10 | Флаг вызывает вызов семисинчронаус. |
| `WBEM_FLAG_OWNER_UPDATE` | 0x10000 | Поставщики push-уведомлений должны указывать этот флаг `PutClassWmi` при вызове, чтобы указать, что этот класс изменился. |
| `WBEM_FLAG_UPDATE_COMPATIBLE` | 0 | Позволяет обновлять класс, если нет производных классов и экземпляров этого класса. Это также позволяет обновлять во всех случаях, если изменение относится только к неважным квалификаторам, например квалификатору Description. Если у класса есть экземпляры или изменения, которые относятся к важным квалификаторам, обновление завершается сбоем. |
| `WBEM_FLAG_UPDATE_SAFE_MODE` | 0x20 | Позволяет обновлять классы даже при наличии дочерних классов, если изменение не вызывает конфликтов с дочерними классами. Например, этот флаг позволяет добавить новое свойство в базовый класс, который ранее не упоминался ни в одном из дочерних классов. Если класс имеет экземпляры, обновление завершается ошибкой. |
| `WBEM_FLAG_UPDATE_FORCE_MODE` | 0x40 | принудительно обновляет классы при наличии конфликтующих дочерних классов. Например, этот флаг приводит к принудительному обновлению, если квалификатор класса определен в дочернем классе, а базовый класс пытается добавить тот же квалификатор, который конфликтует с существующим. В принудительном режиме конфликт TIS разрешается путем удаления конфликтующего квалификатора в дочернем классе. |

`pCtx`\
окне Как правило, это значение `null`равно. В противном случае он является указателем на экземпляр [ивбемконтекст](/windows/desktop/api/wbemcli/nn-wbemcli-iwbemcontext) , который может использоваться поставщиком, который предоставляет запрошенные классы.

`ppCallResult`\
заполняет Если `null`значение равно, этот параметр не используется. Если `lFlags` параметр `WBEM_FLAG_RETURN_IMMEDIATELY`содержит, функция возвращает значение сразу `WBEM_S_NO_ERROR`с параметром. Параметр `ppCallResult` получает указатель на новый объект [IWbemCallResult](/windows/desktop/api/wbemcli/nn-wbemcli-iwbemcallresult).

## <a name="return-value"></a>Возвращаемое значение

Следующие значения, возвращаемые этой функцией, определены в файле заголовка *вбемкли. h* , или их можно определить как константы в коде:

|Константа  |Значение  |Описание  |
|---------|---------|---------|
| `WBEM_E_ACCESS_DENIED` | 0x80041003 | У пользователя нет разрешения на создание или изменение классов. |
| `WBEM_E_FAILED` | 0x80041001 | Произошла неопределенная ошибка. |
| `WBEM_E_INVALID_CLASS` | 0x80041010 | Указан недопустимый класс. Как правило, это означает `pObject` , что указывает объект экземпляра. |
| `WBEM_E_INVALID_PARAMETER` | 0x80041008 | Недопустимый параметр. |
| `WBEM_E_INVALID OPERATION` | 0x80041016 | Указано недопустимое имя класса. |
| `WBEM_E_CLASS_HAS_CHILDREN` | 0x80041025 | Предпринята попытка внести изменение, которое сделает подкласс недействительным. |
| `WBEM_E_ALREADY_EXISTS` | 0x80041019 | Указан `WBEM_FLAG_CREATE_ONLY` флаг, но класс уже существует. |
| `WBEM_E_NOT_FOUND` | 0x80041002 | `WBEM_FLAG_UPDATE_ONLY`был указан в `lFlags`, а класс не найден. |
| `WBEM_E_INCOMPLETE_CLASS` | 0x80041020 | Обязательные свойства для классов не заданы. |
| `WBEM_E_OUT_OF_MEMORY` | 0x80041006 | Недостаточно памяти для завершения операции. |
| `WBEM_E_SHUTTING_DOWN` | 0x80041033 | Вероятно, Инструментарий WMI остановлен и перезапущен. Снова вызовите [коннектсервервми](connectserverwmi.md) . |
| `WBEM_E_TRANSPORT_FAILURE` | 0x80041015 | Не удалось создать ссылку на удаленный вызов процедур (RPC) между текущим процессом и WMI. |
| `WBEM_S_NO_ERROR` | 0 | Вызов функции выполнен успешно.  |

## <a name="remarks"></a>Примечания

Эта функция заключает вызов метода [IWbemServices::P уткласс](/windows/desktop/api/wbemcli/nf-wbemcli-iwbemservices-putclass) .

Пользователь может не создавать классы с именами, начинающимися или заканчивающимися символом подчеркивания.

Если вызов функции завершается неудачно, можно получить дополнительные сведения об ошибке, вызвав функцию [жетерроринфо](geterrorinfo.md) .
