# <a name="put-function"></a>Функция размещения

Задает новое значение для именованного свойства.

[!INCLUDE[internalonly-unmanaged](../../../../includes/internalonly-unmanaged.md)]

## <a name="syntax"></a>Синтаксис

```cpp
HRESULT Put (
   [in] int               vFunc,
   [in] IWbemClassObject* ptr,
   [in] LPCWSTR           wszName,
   [in] LONG              lFlags,
   [in] VARIANT*          pVal,
   [in] CIMTYPE           vtType
);
```

## <a name="parameters"></a>Параметры

`vFunc`\
окне Этот параметр не используется.

`ptr`\
окне Указатель на экземпляр [ивбемклассобжект](/windows/desktop/api/wbemcli/nn-wbemcli-iwbemclassobject) .

`wszName`\
окне Имя свойства. Этот параметр не может быть `null`.

`lFlags`\
[in] Зарезервировано. Этот параметр должен иметь значение 0.

`pVal`\
окне Указатель на допустимое `VARIANT`, которое станет новым значением свойства. Если `pVal` является `null` или указывает на `VARIANT` типа `VT_NULL`, свойству присваивается значение `null`.

`vtType`\
окне Тип `VARIANT` на который указывает `pVal`. Дополнительные сведения см. в разделе ["Примечания"](#remarks) .

## <a name="return-value"></a>Возвращаемое значение

Следующие значения, возвращаемые этой функцией, определены в файле заголовка *вбемкли. h* , или их можно определить как константы в коде:

|Константа  |значения  |Описание  |
|---------|---------|---------|
|`WBEM_E_FAILED` | 0x80041001 | Общий сбой. |
|`WBEM_E_INVALID_PARAMETER` | 0x80041008 | Один или несколько параметров недопустимы. |
|`WBEM_E_INVALID_PROPERTY_TYPE` | 0x8004102a | Тип свойства не распознан. Это значение возвращается при создании экземпляров класса, если класс уже существует. |
|`WBEM_E_OUT_OF_MEMORY` | 0x80041006 | Недостаточно памяти для завершения операции. |
| `WBEM_E_TYPE_MISMATCH` | 0x80041005 | Для экземпляров: указывает, что `pVal` указывает на `VARIANT` неверного типа для свойства. <br/> Для определений классов: свойство уже существует в родительском классе, а новый COM-тип отличается от старого типа COM. |
|`WBEM_S_NO_ERROR` | 0 | Вызов функции выполнен успешно. |

## <a name="remarks"></a>Заметки

Эта функция заключает вызов метода [ивбемклассобжект::P UT](/windows/desktop/api/wbemcli/nf-wbemcli-iwbemclassobject-put) .

Эта функция всегда перезаписывает текущее значение свойства новым. Если [ивбемклассобжект](/windows/desktop/api/wbemcli/nn-wbemcli-iwbemclassobject) указывает на определение класса, `Put` создает или обновляет значение свойства. Если [ивбемклассобжект](/windows/desktop/api/wbemcli/nn-wbemcli-iwbemclassobject) указывает на экземпляр CIM, `Put` обновляет только значение свойства; `Put` не может создать значение свойства.

Системное свойство `__CLASS` доступно для записи только во время создания класса, если оно не может быть пустым. Все остальные системные свойства доступны только для чтения.

Пользователь не может создавать свойства с именами, которые начинаются или заканчиваются символом подчеркивания ("_"). Он зарезервирован для системных классов и свойств.

Если свойство, заданное `Put`ной функцией, существует в родительском классе, значение свойства по умолчанию изменяется, если тип свойства не совпадает с типом родительского класса. Если свойство не существует и не является несоответствием типов, создается свойство.

Параметр `vtType` используется только при создании новых свойств в определении класса CIM, а `pVal` — `null` или указывает на `VARIANT` типа `VT_NULL`. В этом случае параметр `vType` указывает тип CIM для свойства. Во всех остальных случаях `vtType` должны быть равны 0. `vtType` также должен быть равен 0, если базовый объект является экземпляром (даже если `Val` `null`), так как тип свойства является фиксированным и не может быть изменен.
