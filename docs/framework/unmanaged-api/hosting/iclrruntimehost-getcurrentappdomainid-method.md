# <a name="iclrruntimehostgetcurrentappdomainid-method"></a>Метод ICLRRuntimeHost::GetCurrentAppDomainId
Возвращает числовой идентификатор <xref:System.AppDomain>, который выполняется в данный момент.  
  
## <a name="syntax"></a>Синтаксис  
  
```cpp  
HRESULT GetCurrentAppDomainId(  
    [out] DWORD* pdwAppDomainId  
);  
```  
  
## <a name="parameters"></a>Параметры  
 `pdwAppDomainId`  
 заполняет Числовой идентификатор <xref:System.AppDomain>, который выполняется в данный момент.  
  
## <a name="return-value"></a>Возвращаемое значение  
  
|HRESULT|Описание|  
|-------------|-----------------|  
|S_OK|`GetCurrentAppDomainId` успешно возвращено.|  
|HOST_E_CLRNOTAVAILABLE|Среда CLR не была загружена в процесс, или среда CLR находится в состоянии, в котором она не может выполнить управляемый код или успешно обработать вызов.|  
|HOST_E_TIMEOUT|Время ожидания вызова истекло.|  
|HOST_E_NOT_OWNER|Вызывающий объект не владеет блокировкой.|  
|HOST_E_ABANDONED|Событие было отменено, пока заблокированный поток или волокно ожидают его.|  
|E_FAIL|Произошла неизвестная фатальная ошибка. Если метод возвращает значение E_FAIL, среда CLR больше не может использоваться в процессе. Последующие вызовы методов размещения возвращают HOST_E_CLRNOTAVAILABLE.|  
  
## <a name="remarks"></a>Заметки  
 Параметру `pdwAppDomainId` присваивается значение свойства <xref:System.AppDomain.Id%2A> <xref:System.AppDomain>, в котором выполняется текущий поток.
