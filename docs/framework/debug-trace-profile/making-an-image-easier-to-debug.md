# <a name="making-an-image-easier-to-debug-in-net"></a><span data-ttu-id="78d78-103">Упрощение отладки образа в .NET</span><span class="sxs-lookup"><span data-stu-id="78d78-103">Making an image easier to debug in .NET</span></span>

<span data-ttu-id="78d78-104">При компиляции неуправляемого кода можно настроить исполняемый образ для отладки, задав параметры командной строки или интегрированной среды разработки.</span><span class="sxs-lookup"><span data-stu-id="78d78-104">When compiling unmanaged code, you can configure an executable image for debugging by setting IDE switches or command-line options.</span></span> <span data-ttu-id="78d78-105">Например, параметр командной строки /**Zi** в Visual C++ служит для создания файлов с отладочными символами (с расширением PDB).</span><span class="sxs-lookup"><span data-stu-id="78d78-105">For example, you can use the /**Zi** command-line option in Visual C++ to ask it to emit debug symbol files (file extension .pdb).</span></span> <span data-ttu-id="78d78-106">Аналогичным образом параметр командной строки /**Od** отключает оптимизацию компилятора.</span><span class="sxs-lookup"><span data-stu-id="78d78-106">Similarly, the /**Od** command-line option tells the compiler to disable optimization.</span></span> <span data-ttu-id="78d78-107">Результирующий код выполняется более медленно, но его отладка будет проще, если это необходимо.</span><span class="sxs-lookup"><span data-stu-id="78d78-107">The resulting code runs more slowly, but it's easier to debug, should this be necessary.</span></span>

<span data-ttu-id="78d78-108">При компиляции управляемого кода .NET Framework компиляторы, такие как Visual C++, Visual Basic, и C# компилируются их исходную программу в промежуточный язык Майкрософт (MSIL).</span><span class="sxs-lookup"><span data-stu-id="78d78-108">When compiling .NET Framework managed code, compilers such as Visual C++, Visual Basic, and C# compile their source program into Microsoft intermediate language (MSIL).</span></span> <span data-ttu-id="78d78-109">Затем MSIL компилируется JIT-компилятором непосредственно перед выполнением в машинный код машинного кода.</span><span class="sxs-lookup"><span data-stu-id="78d78-109">MSIL is then JIT-compiled, just before execution, into native machine code.</span></span> <span data-ttu-id="78d78-110">Как и в случае с неуправляемым кодом, можно настроить исполняемый образ для отладки, задав параметры командной строки или интегрированной среды разработки.</span><span class="sxs-lookup"><span data-stu-id="78d78-110">As with unmanaged code, you can configure an executable image for debugging by setting IDE switches or command-line options.</span></span> <span data-ttu-id="78d78-111">Также можно настроить JIT-компиляцию для отладки практически таким же образом.</span><span class="sxs-lookup"><span data-stu-id="78d78-111">You can also configure the JIT compilation for debugging in much the same way.</span></span>

<span data-ttu-id="78d78-112">Данная JIT-конфигурация имеет два аспекта.</span><span class="sxs-lookup"><span data-stu-id="78d78-112">This JIT configuration has two aspects:</span></span>

- <span data-ttu-id="78d78-113">Можно запросить JIT-компилятор для создания данных отслеживания.</span><span class="sxs-lookup"><span data-stu-id="78d78-113">You can request the JIT compiler to generate tracking information.</span></span> <span data-ttu-id="78d78-114">После этого отладчик сможет сопоставлять цепочки команд MSIL с соответствующими командами машинного кода, а также отслеживать место хранения локальных переменных и аргументов функций.</span><span class="sxs-lookup"><span data-stu-id="78d78-114">This makes it possible for the debugger to match up a chain of MSIL with its machine code counterpart, and to track where local variables and function arguments are stored.</span></span> <span data-ttu-id="78d78-115">Начиная с версии .NET Framework 2,0 JIT-компилятор всегда создает данные отслеживания, поэтому нет необходимости запрашивать его.</span><span class="sxs-lookup"><span data-stu-id="78d78-115">Starting with the .NET Framework version 2.0, the JIT compiler always generates tracking information, so there is no need to request it.</span></span>

- <span data-ttu-id="78d78-116">Можно запросить JIT-компилятор, чтобы не оптимизировать полученный машинный код.</span><span class="sxs-lookup"><span data-stu-id="78d78-116">You can request the JIT compiler to not optimize the resulting machine code.</span></span>

<span data-ttu-id="78d78-117">Как правило, компилятор, создающий MSIL, устанавливает эти параметры JIT-компилятора соответствующим образом, основываясь на параметрах интегрированной среды разработки или указанных параметрах командной строки, например,/**OD**.</span><span class="sxs-lookup"><span data-stu-id="78d78-117">Normally, the compiler that generates the MSIL sets these JIT compiler options appropriately, based upon the IDE switches or command-line options you specify, for example, /**Od**.</span></span>

<span data-ttu-id="78d78-118">В некоторых случаях может возникнуть необходимость изменить поведение JIT-компилятора, чтобы создаваемый машинный код было проще отлаживать.</span><span class="sxs-lookup"><span data-stu-id="78d78-118">In some cases, you might want to change the behavior of the JIT compiler so that the machine code it generates is easier to debug.</span></span> <span data-ttu-id="78d78-119">Например, для окончательной сборки или оптимизации управления может потребоваться создать сведения о JIT-отслеживании.</span><span class="sxs-lookup"><span data-stu-id="78d78-119">For example, you might want to generate JIT tracking information for a retail build or control optimization.</span></span> <span data-ttu-id="78d78-120">Это можно сделать с помощью файла инициализации (INI).</span><span class="sxs-lookup"><span data-stu-id="78d78-120">You can do so with an initialization (.ini) file.</span></span>

<span data-ttu-id="78d78-121">Например, если сборка, которую требуется отладить, называется *MyApp. exe*, то можно создать текстовый файл с именем *MyApp. ini*в той же папке, что и *файл MyApp. exe*, содержащий следующие три строки:</span><span class="sxs-lookup"><span data-stu-id="78d78-121">For example, if the assembly you want to debug is called *MyApp.exe*, then you can create a text file named *MyApp.ini*, in the same folder as *MyApp.exe*, which contains these three lines:</span></span>

```ini
[.NET Framework Debugging Control]
GenerateTrackingInfo=1
AllowOptimize=0
```

<span data-ttu-id="78d78-122">Значением каждого параметра может быть 0 или 1; отсутствующие параметры получают значение 0.</span><span class="sxs-lookup"><span data-stu-id="78d78-122">You can set the value of each option to 0 or 1, and any absent option defaults to 0.</span></span> <span data-ttu-id="78d78-123">Для упрощения отладки значение `GenerateTrackingInfo` должно быть равным 1, а значение `AllowOptimize` — 0.</span><span class="sxs-lookup"><span data-stu-id="78d78-123">Setting `GenerateTrackingInfo` to 1 and `AllowOptimize` to 0 provides the easiest debugging.</span></span>

<span data-ttu-id="78d78-124">Начиная с версии .NET Framework 2,0 JIT-компилятор всегда создает данные отслеживания независимо от значения для `GenerateTrackingInfo`. Однако значение `AllowOptimize` по-прежнему действует.</span><span class="sxs-lookup"><span data-stu-id="78d78-124">Starting with the .NET Framework version 2.0, the JIT compiler always generates tracking information regardless of the value for `GenerateTrackingInfo`; however, the `AllowOptimize` value still has an effect.</span></span> <span data-ttu-id="78d78-125">При использовании [Ngen.exe (средства для создания образов в машинном коде)](../tools/ngen-exe-native-image-generator.md) для предварительной компиляции образа в машинном коде без оптимизации INI-файл, содержащий параметр `AllowOptimize=0`, должен находиться в целевой папке во время выполнения Ngen.exe.</span><span class="sxs-lookup"><span data-stu-id="78d78-125">When using the [Ngen.exe (Native Image Generator)](../tools/ngen-exe-native-image-generator.md) to precompile the native image without optimization, the .ini file must be present in the target folder with `AllowOptimize=0` when Ngen.exe executes.</span></span> <span data-ttu-id="78d78-126">Если сборка предварительно скомпилирована без оптимизации, необходимо удалить предварительно скомпилированный код с помощью команды NGen. exe **/uninstall** перед повторным выполнением программы Ngen. exe, чтобы выполнить предварительную компиляцию кода как оптимизированного.</span><span class="sxs-lookup"><span data-stu-id="78d78-126">If you have precompiled an assembly without optimization, you must remove the precompiled code using NGen.exe **/uninstall** option before rerunning Ngen.exe to precompile the code as optimized.</span></span> <span data-ttu-id="78d78-127">Если ini-файл отсутствует в папке, по умолчанию Ngen. exe выполняет предварительную компиляцию кода как оптимизированного.</span><span class="sxs-lookup"><span data-stu-id="78d78-127">If the .ini file isn't present in the folder, by default Ngen.exe precompiles the code as optimized.</span></span>

<span data-ttu-id="78d78-128"><xref:System.Diagnostics.DebuggableAttribute?displayProperty=nameWithType> управляет параметрами для сборки.</span><span class="sxs-lookup"><span data-stu-id="78d78-128">The <xref:System.Diagnostics.DebuggableAttribute?displayProperty=nameWithType> controls the settings for an assembly.</span></span> <span data-ttu-id="78d78-129">**DebuggableAttribute** содержит два поля, которые определяют, должен ли JIT-компилятор оптимизировать и (или) создавать данные отслеживания.</span><span class="sxs-lookup"><span data-stu-id="78d78-129">**DebuggableAttribute** includes two fields that control whether the JIT compiler should optimize and/or generate tracking information.</span></span> <span data-ttu-id="78d78-130">Начиная с версии .NET Framework 2,0 JIT-компилятор всегда создает данные отслеживания.</span><span class="sxs-lookup"><span data-stu-id="78d78-130">Starting with the .NET Framework version 2.0, the JIT compiler always generates tracking information.</span></span>

<span data-ttu-id="78d78-131">Для розничной сборки компиляторы не устанавливают никаких **DebuggableAttribute**.</span><span class="sxs-lookup"><span data-stu-id="78d78-131">For a retail build, compilers don't set any **DebuggableAttribute**.</span></span> <span data-ttu-id="78d78-132">По умолчанию JIT-компилятор создает наивысшую производительность, что затрудняет отладку машинного кода.</span><span class="sxs-lookup"><span data-stu-id="78d78-132">By default, the JIT compiler generates the highest performance, hardest to debug machine code.</span></span> <span data-ttu-id="78d78-133">Включение JIT-отслеживания несколько снижает производительность; отключение оптимизации снижает производительность значительно.</span><span class="sxs-lookup"><span data-stu-id="78d78-133">Enabling JIT tracking lowers performance a little, and disabling optimization lowers performance a lot.</span></span>

<span data-ttu-id="78d78-134">**DebuggableAttribute** одновременно применяется ко всей сборке, а не к ее отдельным модулям.</span><span class="sxs-lookup"><span data-stu-id="78d78-134">The **DebuggableAttribute** applies to a whole assembly at a time, not to individual modules within the assembly.</span></span> <span data-ttu-id="78d78-135">Средства разработки должны, таким образом, подключать пользовательские атрибуты к классу **System.Runtime.CompilerServices.AssemblyAttributesGoHere** или, если сборка уже создана, к токену метаданных сборки.</span><span class="sxs-lookup"><span data-stu-id="78d78-135">Development tools must therefore attach custom attributes to the assembly metadata token, if an assembly has already been created, or to the class called **System.Runtime.CompilerServices.AssemblyAttributesGoHere**.</span></span> <span data-ttu-id="78d78-136">Затем средство ALink передает эти атрибуты **DebuggableAttribute** из каждого модуля в сборку, в которой они становятся частью.</span><span class="sxs-lookup"><span data-stu-id="78d78-136">The ALink tool then promotes these **DebuggableAttribute** attributes from each module to the assembly they become a part of.</span></span> <span data-ttu-id="78d78-137">В случае конфликта происходит сбой операции ALink.</span><span class="sxs-lookup"><span data-stu-id="78d78-137">If there's a conflict, the ALink operation fails.</span></span>

> <span data-ttu-id="78d78-138">В .NET Framework версии 1.0 **DebuggableAttribute** добавляется компилятором Microsoft Visual C++, если указаны параметры компилятора **/clr** и **/Zi**.</span><span class="sxs-lookup"><span data-stu-id="78d78-138">In version 1.0 of the .NET Framework, the Microsoft Visual C++ compiler adds the **DebuggableAttribute** when the **/clr** and **/Zi** compiler options are specified.</span></span> <span data-ttu-id="78d78-139">В версии 1,1 .NET Framework необходимо вручную добавить **DebuggableAttribute** в код или использовать параметр компоновщика **/ASSEMBLYDEBUG** .</span><span class="sxs-lookup"><span data-stu-id="78d78-139">In version 1.1 of the .NET Framework, you must either add the **DebuggableAttribute** manually in your code or use the **/ASSEMBLYDEBUG** linker option.</span></span>
