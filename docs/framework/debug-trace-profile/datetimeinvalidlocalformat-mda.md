## <a name="symptom"></a><span data-ttu-id="8cda0-105">Симптом</span><span class="sxs-lookup"><span data-stu-id="8cda0-105">Symptom</span></span>  
 <span data-ttu-id="8cda0-106">Приложение вручную сериализует экземпляр <xref:System.DateTime> в формате UTC с использованием локального формата:</span><span class="sxs-lookup"><span data-stu-id="8cda0-106">An application is manually serializing a UTC <xref:System.DateTime> instance using a local format:</span></span>

### <a name="cause"></a><span data-ttu-id="8cda0-107">Причина</span><span class="sxs-lookup"><span data-stu-id="8cda0-107">Cause</span></span>  
 <span data-ttu-id="8cda0-108">Формат "z" для метода <xref:System.DateTime.ToString%2A?displayProperty=nameWithType> включает сдвиг местного часового пояса, например "+03:00" для московского времени.</span><span class="sxs-lookup"><span data-stu-id="8cda0-108">The 'z' format for the <xref:System.DateTime.ToString%2A?displayProperty=nameWithType> method includes the local time zone offset, for example, "+10:00" for Sydney time.</span></span> <span data-ttu-id="8cda0-109">Таким образом, значащий результат будет получен только в том случае, если экземпляр <xref:System.DateTime> содержит значение местного времени.</span><span class="sxs-lookup"><span data-stu-id="8cda0-109">As such, it will only produce a meaningful result if the value of the <xref:System.DateTime> is local.</span></span> <span data-ttu-id="8cda0-110">Если время указано в формате UTC, <xref:System.DateTime.ToString%2A?displayProperty=nameWithType> содержит сдвиг местного часового пояса, однако не отображает и не позволяет изменять описатель часового пояса.</span><span class="sxs-lookup"><span data-stu-id="8cda0-110">If the value is UTC time, <xref:System.DateTime.ToString%2A?displayProperty=nameWithType> includes the local time zone offset, but it does not display or adjust the time zone specifier.</span></span>  
  
### <a name="resolution"></a><span data-ttu-id="8cda0-111">Решение</span><span class="sxs-lookup"><span data-stu-id="8cda0-111">Resolution</span></span>  
 <span data-ttu-id="8cda0-112">Экземпляры <xref:System.DateTime> в формате UTC должны иметь форматирование, явно указывающее на это.</span><span class="sxs-lookup"><span data-stu-id="8cda0-112">UTC <xref:System.DateTime> instances should be formatted in a way that indicates that they are UTC.</span></span> <span data-ttu-id="8cda0-113">Для работы со временем в формате UTC рекомендуется использовать формат "Z":</span><span class="sxs-lookup"><span data-stu-id="8cda0-113">The recommended format for UTC times to use a 'Z' to denote UTC time:</span></span>  
  
```csharp

Serialize(myDateTime.ToString("yyyy-MM-dd'T'HH:mm:ss.fffffffZ"));  
```  
  
 <span data-ttu-id="8cda0-114">Также доступен формат "o", который сериализует <xref:System.DateTime> с использованием свойства <xref:System.DateTime.Kind%2A> и обеспечивает корректную сериализацию независимо от того, содержит экземпляр местное время, время в формате UTC или формат не указан:</span><span class="sxs-lookup"><span data-stu-id="8cda0-114">There is also an "o" format that serializes a <xref:System.DateTime> making use of the <xref:System.DateTime.Kind%2A> property that serializes correctly regardless of whether the instance is local, UTC, or unspecified:</span></span>  
  
```csharp

Serialize(myDateTime.ToString("o"));  
```

## <a name="example"></a><span data-ttu-id="8cda0-120">Пример</span><span class="sxs-lookup"><span data-stu-id="8cda0-120">Example</span></span>  
 <span data-ttu-id="8cda0-121">Рассмотрите возможность косвенной сериализации значений <xref:System.DateTime> в формате UTC с использованием класса <xref:System.Xml.XmlConvert> или <xref:System.Data.DataSet>, как показано ниже.</span><span class="sxs-lookup"><span data-stu-id="8cda0-121">Consider an application that is indirectly serializing a UTC <xref:System.DateTime> value by using the <xref:System.Xml.XmlConvert> or <xref:System.Data.DataSet> class, in the following manner.</span></span>

 <span data-ttu-id="8cda0-122">В сериализациях <xref:System.Xml.XmlConvert> и <xref:System.Data.DataSet> по умолчанию используется локальный формат.</span><span class="sxs-lookup"><span data-stu-id="8cda0-122">The <xref:System.Xml.XmlConvert> and <xref:System.Data.DataSet> serializations use local formats for serialization by default.</span></span> <span data-ttu-id="8cda0-123">Для сериализации значений <xref:System.DateTime> другого вида, например в формате UTC, требуется указать дополнительные параметры.</span><span class="sxs-lookup"><span data-stu-id="8cda0-123">Additional options are required to serialize other kinds of <xref:System.DateTime> values, such as UTC.</span></span>
