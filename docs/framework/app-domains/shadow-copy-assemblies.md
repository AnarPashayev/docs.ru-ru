  <span data-ttu-id="d72db-119">По умолчанию этот параметр вызывает копирование всех сборок по пути приложения в кэш загрузки перед их загрузкой.</span><span class="sxs-lookup"><span data-stu-id="d72db-119">By default, this setting causes all assemblies in the application path to be copied to a download cache before they are loaded.</span></span> <span data-ttu-id="d72db-120">Это тот же кэш, который поддерживается средой CLR для хранения файлов, загруженных с других компьютеров. ## <a name="startup-performance"></a><span data-ttu-id="d72db-140">Производительность при запуске</span><span class="sxs-lookup"><span data-stu-id="d72db-140">Startup Performance</span></span>

<span data-ttu-id="d72db-141">Когда запускается домен приложения, использующий теневое копирование, возникает задержка, пока сборки копируются из каталога приложения в каталог теневого копирования или проверяются, если они уже находятся в этом каталоге.</span><span class="sxs-lookup"><span data-stu-id="d72db-141">When an application domain that uses shadow copying starts, there is a delay while assemblies in the application directory are copied to the shadow copy directory, or verified if they are already in that location.</span></span> <span data-ttu-id="d72db-142">До появления .NET Framework 4 все сборки копировались во временный каталог.</span><span class="sxs-lookup"><span data-stu-id="d72db-142">Before the .NET Framework 4, all assemblies were copied to a temporary directory.</span></span> <span data-ttu-id="d72db-143">Каждая сборка открывалась для проверки имени сборки и строгого имени.</span><span class="sxs-lookup"><span data-stu-id="d72db-143">Each assembly was opened to verify the assembly name, and the strong name was validated.</span></span> <span data-ttu-id="d72db-144">Каждая сборка проверялась на предмет того, обновлялась ли она позднее копии в каталоге теневого копирования.</span><span class="sxs-lookup"><span data-stu-id="d72db-144">Each assembly was checked to see whether it had been updated more recently than the copy in the shadow copy directory.</span></span> <span data-ttu-id="d72db-145">Если это было так, сборка копировалась в каталог теневого копирования.</span><span class="sxs-lookup"><span data-stu-id="d72db-145">If so, it was copied to the shadow copy directory.</span></span> <span data-ttu-id="d72db-146">Наконец, временные копии удалялись.</span><span class="sxs-lookup"><span data-stu-id="d72db-146">Finally, the temporary copies were discarded.</span></span>
