# <a name="working-with-assemblies-and-the-global-assembly-cache"></a><span data-ttu-id="23f74-102">Работа со сборками и глобальным кэшем сборок</span><span class="sxs-lookup"><span data-stu-id="23f74-102">Working with Assemblies and the Global Assembly Cache</span></span>

<span data-ttu-id="23f74-103">Если необходимо обеспечить возможность совместного использования сборки в нескольких приложениях, то ее можно поместить в глобальный кэш сборок.</span><span class="sxs-lookup"><span data-stu-id="23f74-103">If you intend to share an assembly among several applications, you can install it into the global assembly cache.</span></span> <span data-ttu-id="23f74-104">Этот кэш кода уровня компьютера присутствует на любом компьютере, где установлена среда CLR.</span><span class="sxs-lookup"><span data-stu-id="23f74-104">Each computer where the common language runtime is installed has this machine-wide code cache.</span></span> <span data-ttu-id="23f74-105">В глобальном кэше сборок сохраняются сборки, специально предназначенные для совместного использования на компьютере несколькими приложениями.</span><span class="sxs-lookup"><span data-stu-id="23f74-105">The global assembly cache stores assemblies specifically designated to be shared by several applications on the computer.</span></span> <span data-ttu-id="23f74-106">Для установки в глобальном кэше сборка должна иметь строгое имя.</span><span class="sxs-lookup"><span data-stu-id="23f74-106">An assembly must have a strong name to be installed in the global assembly cache.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="23f74-107">Имя сборки, установленной в глобальном кэше сборок, должно совпадать с именем файла (без учета расширения имени файла).</span><span class="sxs-lookup"><span data-stu-id="23f74-107">Assemblies placed in the global assembly cache must have the same assembly name and file name (not including the file name extension).</span></span> <span data-ttu-id="23f74-108">К примеру, файл сборки с именем myAssembly должен иметь имя myAssembly.exe или myAssembly.dll.</span><span class="sxs-lookup"><span data-stu-id="23f74-108">For example, an assembly with the assembly name of myAssembly must have a file name of either myAssembly.exe or myAssembly.dll.</span></span>  
  
<span data-ttu-id="23f74-109">Прибегать к совместному использованию сборок путем их установки в глобальном кэше сборок следует только при необходимости.</span><span class="sxs-lookup"><span data-stu-id="23f74-109">You should share assemblies by installing them into the global assembly cache only when necessary.</span></span> <span data-ttu-id="23f74-110">Как правило, зависимости между сборками следует сохранять закрытыми, а сами сборки нужно размещать в папке приложения, если они не предназначены для совместного использования.</span><span class="sxs-lookup"><span data-stu-id="23f74-110">As a general guideline, keep assembly dependencies private and locate assemblies in the application directory unless sharing an assembly is explicitly required.</span></span> <span data-ttu-id="23f74-111">Кроме того, установка сборок в глобальном кэше сборок для обеспечения доступа к ним с помощью COM-взаимодействия или из неуправляемого кода не является обязательной.</span><span class="sxs-lookup"><span data-stu-id="23f74-111">In addition, you do not have to install assemblies into the global assembly cache to make them accessible to COM interop or unmanaged code.</span></span>  
  
<span data-ttu-id="23f74-112">Существует несколько причин для установки сборки в глобальном кэше сборок.</span><span class="sxs-lookup"><span data-stu-id="23f74-112">There are several reasons why you might want to install an assembly into the global assembly cache:</span></span>  
  
- <span data-ttu-id="23f74-113">Общее расположение.</span><span class="sxs-lookup"><span data-stu-id="23f74-113">Shared location.</span></span>  
  
     <span data-ttu-id="23f74-114">Используемые несколькими приложениями сборки можно располагать в глобальном кэше сборок.</span><span class="sxs-lookup"><span data-stu-id="23f74-114">Assemblies that should be used by applications can be put in the global assembly cache.</span></span> <span data-ttu-id="23f74-115">Например, если все приложения используют сборку, расположенную в глобальном кэше сборок, то в файл Machine.config можно добавить оператор политики выбора версий, который перенаправляет ссылки на эту сборку.</span><span class="sxs-lookup"><span data-stu-id="23f74-115">For example, if all applications should use an assembly located in the global assembly cache, a version policy statement can be added to the Machine.config file that redirects references to the assembly.</span></span>  
  
- <span data-ttu-id="23f74-116">Безопасность файлов.</span><span class="sxs-lookup"><span data-stu-id="23f74-116">File security.</span></span>  
  
     <span data-ttu-id="23f74-117">Администраторы часто защищают папку systemroot с помощью списка управления доступом, определяющего права на запись и выполнение.</span><span class="sxs-lookup"><span data-stu-id="23f74-117">Administrators often protect the systemroot directory using an Access Control List (ACL) to control write and execute access.</span></span> <span data-ttu-id="23f74-118">Так как глобальный кэш сборок размещается в корневом каталоге системы, он наследует список управления доступом этого каталога.</span><span class="sxs-lookup"><span data-stu-id="23f74-118">Because the global assembly cache is installed in the systemroot directory, it inherits that directory's ACL.</span></span> <span data-ttu-id="23f74-119">Рекомендуется разрешать удаление файлов из глобального кэша сборок только пользователям, имеющим права доступа администратора.</span><span class="sxs-lookup"><span data-stu-id="23f74-119">It is recommended that only users with Administrator privileges be allowed to delete files from the global assembly cache.</span></span>  
  
- <span data-ttu-id="23f74-120">Управление параллельными версиями.</span><span class="sxs-lookup"><span data-stu-id="23f74-120">Side-by-side versioning.</span></span>  
  
     <span data-ttu-id="23f74-121">В глобальном кэше сборок может храниться несколько сборок, имеющих одинаковые имена, но различные сведения о версии.</span><span class="sxs-lookup"><span data-stu-id="23f74-121">Multiple copies of assemblies with the same name but different version information can be maintained in the global assembly cache.</span></span>  
  
- <span data-ttu-id="23f74-122">Дополнительное место для поиска.</span><span class="sxs-lookup"><span data-stu-id="23f74-122">Additional search location.</span></span>  
  
     <span data-ttu-id="23f74-123">Перед проверкой или использованием сведений о базе кода в файле конфигурации среда CLR ищет в глобальном кэше сборки, соответствующие запросу.</span><span class="sxs-lookup"><span data-stu-id="23f74-123">The common language runtime checks the global assembly cache for an assembly that matches the assembly request before probing or using the codebase information in a configuration file.</span></span>  
  
 <span data-ttu-id="23f74-124">Обратите внимание, что существуют сценарии, в которых установка сборки в глобальный кэш сборок явно не требуется.</span><span class="sxs-lookup"><span data-stu-id="23f74-124">Note that there are scenarios where you explicitly do not want to install an assembly into the global assembly cache.</span></span> <span data-ttu-id="23f74-125">Если одна из составляющих приложение сборок помещается в глобальный кэш сборок, то после этого нельзя будет скопировать или установить приложение с помощью команды XCOPY путем копирования каталога приложения.</span><span class="sxs-lookup"><span data-stu-id="23f74-125">If you place one of the assemblies that make up an application into the global assembly cache, you can no longer replicate or install the application by using XCOPY to copy the application directory.</span></span> <span data-ttu-id="23f74-126">В этом случае также требуется переместить сборку в глобальный кэш сборок.</span><span class="sxs-lookup"><span data-stu-id="23f74-126">In this case, you must also move the assembly into the global assembly cache.</span></span>
