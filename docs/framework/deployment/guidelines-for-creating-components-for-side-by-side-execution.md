<span data-ttu-id="e74b4-103">Выполните эти общие рекомендации для создания управляемых приложений или компонентов для параллельного выполнения.</span><span class="sxs-lookup"><span data-stu-id="e74b4-103">Follow these general guidelines to create managed applications or components designed for side-by-side execution:</span></span>

 <span data-ttu-id="e74b4-117">Одним из способов успешного создания приложения или компонента для параллельного выполнения является изоляция.</span><span class="sxs-lookup"><span data-stu-id="e74b4-117">One key to successfully designing an application or component for side-by-side execution is isolation.</span></span> <span data-ttu-id="e74b4-118">Приложение или компонент должен осуществлять управление всеми ресурсами, в частности вводом-выводом файлов, в изолированном режиме.</span><span class="sxs-lookup"><span data-stu-id="e74b4-118">The application or component must manage all resources, particularly file I/O, in an isolated manner.</span></span> <span data-ttu-id="e74b4-119">Соблюдайте следующие правила, чтобы гарантировать изолированную работу приложения или компонента.</span><span class="sxs-lookup"><span data-stu-id="e74b4-119">Follow these guidelines to make sure your application or component runs in isolation:</span></span>  
  
-   <span data-ttu-id="e74b4-120">Выполняйте запись в реестр с учетом версий.</span><span class="sxs-lookup"><span data-stu-id="e74b4-120">Write to the registry in a version-specific way.</span></span> <span data-ttu-id="e74b4-121">Храните значения в кустах или ключах с указанием версии, не осуществляйте совместное использование информации или сведений о состоянии в разных версиях компонента.</span><span class="sxs-lookup"><span data-stu-id="e74b4-121">Store values in hives or keys that indicate the version, and do not share information or state across versions of a component.</span></span> <span data-ttu-id="e74b4-122">В этом случае два приложения или компонента, работающие одновременно, не смогут перезаписывать информацию.</span><span class="sxs-lookup"><span data-stu-id="e74b4-122">This prevents two applications or components running at the same time from overwriting information.</span></span>

-   <span data-ttu-id="e74b4-127">Создавайте учетные записи пользователей и группы с учетом версий.</span><span class="sxs-lookup"><span data-stu-id="e74b4-127">Create user accounts and groups in a version-specific manner.</span></span> <span data-ttu-id="e74b4-128">Учетные записи пользователей и группы, созданные приложением, должны идентифицироваться по версии.</span><span class="sxs-lookup"><span data-stu-id="e74b4-128">User accounts and groups created by an application should be identified by version.</span></span> <span data-ttu-id="e74b4-129">Не осуществляйте совместное использование учетных записей пользователей и групп разными версиями приложения.</span><span class="sxs-lookup"><span data-stu-id="e74b4-129">Do not share user accounts and groups between versions of an application.</span></span>
