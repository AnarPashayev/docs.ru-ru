# <a name="ipv6-addressing"></a><span data-ttu-id="6cab7-102">Адресация IPv6</span><span class="sxs-lookup"><span data-stu-id="6cab7-102">IPv6 Addressing</span></span>
<span data-ttu-id="6cab7-103">Длина IPv6-адресов составляет 128 бит.</span><span class="sxs-lookup"><span data-stu-id="6cab7-103">In the Internet Protocol version 6 (IPv6), addresses are 128 bits long.</span></span> <span data-ttu-id="6cab7-104">Одна из причин такого большого адресного пространства заключается в том, чтобы разделить доступные адреса на иерархию доменов маршрутизации, отражающих топологию Интернета.</span><span class="sxs-lookup"><span data-stu-id="6cab7-104">One reason for such a large address space is to subdivide the available addresses into a hierarchy of routing domains that reflect the Internet's topology.</span></span> <span data-ttu-id="6cab7-105">Другая причина состоит в том, чтобы сопоставить адреса сетевых адаптеров (или интерфейсов), которые подключают устройства к сети.</span><span class="sxs-lookup"><span data-stu-id="6cab7-105">Another reason is to map the addresses of network adapters (or interfaces) that connect devices to the network.</span></span> <span data-ttu-id="6cab7-106">IPv6 обладает встроенной возможностью разрешать адреса на их самом низком уровне, который находится на уровне сетевого интерфейса, а также позволяет выполнять автоматическую настройку.</span><span class="sxs-lookup"><span data-stu-id="6cab7-106">IPv6 features an inherent capability to resolve addresses at their lowest level, which is at the network interface level, and also has auto-configuration capabilities.</span></span>  
  
## <a name="text-representation"></a><span data-ttu-id="6cab7-107">Текстовое представление</span><span class="sxs-lookup"><span data-stu-id="6cab7-107">Text Representation</span></span>  
 <span data-ttu-id="6cab7-108">Ниже приведены три стандартные формы, которые используются для представления адресов IPv6 в виде текстовых строк:</span><span class="sxs-lookup"><span data-stu-id="6cab7-108">The following are the three conventional forms used to represent the IPv6 addresses as text strings:</span></span>

-   <span data-ttu-id="6cab7-113">**Сжатая форма**.</span><span class="sxs-lookup"><span data-stu-id="6cab7-113">**Compressed form**.</span></span> <span data-ttu-id="6cab7-114">Из-за длины адреса представлена символом двойного двоеточия (::).</span><span class="sxs-lookup"><span data-stu-id="6cab7-115">To simplify writing these addresses, use the compressed form, in which a single contiguous `3FFE:FFFF::8:800:20C4:0`.</span></span> <span data-ttu-id="6cab7-119">Петлевой адрес `0:0:0:0:0:0:0:1` в сжатом виде выглядит как `::`1.</span><span class="sxs-lookup"><span `::`.</span><span class="sxs-lookup"><span data-stu-id="6cab7-120">The unspecified address `0:0:0:0:0:0:0:0` in compressed form is `::`.</span></span>

## <a name="address-types"></a><span data-ttu-id="6cab7-124">Типы адресов</span><span class="sxs-lookup"><span data-stu-id="6cab7-124">Address Types</span></span>

 <span data-ttu-id="6cab7-130">Ниже приведен пример адреса с 64-разрядным префиксом.</span><span class="sxs-lookup"><span data-stu-id="6cab7-130">The following is an example of an address with a 64-bit prefix.</span></span>  
  
 `3FFE:FFFF:0:CD30:0:0:0:0/64`<span data-ttu-id="6cab7-131">.</span><span class="sxs-lookup"><span data-stu-id="6cab7-131">.</span></span>  
  
 <span data-ttu-id="6cab7-132">В этом примере префикс — `3FFE:FFFF:0:CD30`.</span><span class="sxs-lookup"><span data-stu-id="6cab7-132">The prefix in this example is `3FFE:FFFF:0:CD30`.</span></span> <span data-ttu-id="6cab7-133">Адрес также может быть написан в сжатом виде, например `3FFE:FFFF:0:CD30::/64`.</span><span class="sxs-lookup"><span data-stu-id="6cab7-133">The address can also be written in a compressed form, as `3FFE:FFFF:0:CD30::/64`.</span></span>  
  
 <span data-ttu-id="6cab7-134">IPv6 определяет следующие типы адресов:</span><span class="sxs-lookup"><span data-stu-id="6cab7-134">IPv6 defines the following address types:</span></span>  
  
-   <span data-ttu-id="6cab7-135">**Адрес одноадресной рассылки**.</span><span class="sxs-lookup"><span data-stu-id="6cab7-135">**Unicast address**.</span></span> <span data-ttu-id="6cab7-136">Идентификатор для отдельного интерфейса.</span><span class="sxs-lookup"><span data-stu-id="6cab7-136">An identifier for a single interface.</span></span> <span data-ttu-id="6cab7-137">Пакет, отправленный на этот адрес, доставляется в определенный интерфейс.</span><span class="sxs-lookup"><span data-stu-id="6cab7-137">A packet sent to this address is delivered to the identified interface.</span></span> <span data-ttu-id="6cab7-138">Адреса одноадресной рассылки отличаются от адресов многоадресной рассылки по значению октета более высокого разряда.</span><span class="sxs-lookup"><span data-stu-id="6cab7-138">The unicast addresses are distinguished from the multicast addresses by the value of the high-order octet.</span></span> <span data-ttu-id="6cab7-139">Октет старшего порядка адресов многоадресной рассылки имеет шестнадцатеричное значение FF.</span><span class="sxs-lookup"><span data-stu-id="6cab7-139">The multicast addresses' high-order octet has the hexadecimal value of FF.</span></span> <span data-ttu-id="6cab7-140">Любое другое значение для этого октета определяет адрес одноадресной рассылки.</span><span class="sxs-lookup"><span data-stu-id="6cab7-140">Any other value for this octet identifies a unicast address.</span></span> <span data-ttu-id="6cab7-141">Ниже приведены различные типы адресов одноадресной рассылки.</span><span class="sxs-lookup"><span data-stu-id="6cab7-141">The following are different types of unicast addresses:</span></span>  
  
    -   <span data-ttu-id="6cab7-142">**Адреса локального канала**.</span><span class="sxs-lookup"><span data-stu-id="6cab7-142">**Link-local addresses**.</span></span> <span data-ttu-id="6cab7-143">Эти адреса используются в одном канале и имеют следующий формат: FE80::*InterfaceID*.</span><span class="sxs-lookup"><span data-stu-id="6cab7-143">These addresses are used on a single link and have the following format: FE80::*InterfaceID*.</span></span> <span data-ttu-id="6cab7-144">Адреса локального канала используются между узлами в канале для автонастройки адресов, обнаружения окружения или при отсутствии маршрутизаторов.</span><span class="sxs-lookup"><span data-stu-id="6cab7-144">Link-local addresses are used between nodes on a link for auto-address configuration, neighbor discovery, or when no routers are present.</span></span> <span data-ttu-id="6cab7-145">Адрес локального канала используется главным образом во время запуска и в случае, когда система еще не получила адреса большей области.</span><span class="sxs-lookup"><span data-stu-id="6cab7-145">A link-local address is used primarily at startup and when the system has not yet acquired addresses of larger scope.</span></span>

-   <span data-ttu-id="6cab7-151">**Адрес многоадресной рассылки**.</span><span class="sxs-lookup"><span data-stu-id="6cab7-151">**Multicast address**.</span></span> <span data-ttu-id="6cab7-152">Идентификатор набора интерфейсов (обычно принадлежащих разным узлам).</span><span class="sxs-lookup"><span data-stu-id="6cab7-152">An identifier for a set of interfaces (typically belonging to different nodes).</span></span> <span data-ttu-id="6cab7-153">Пакет, отправленный на этот адрес, доставляется во все интерфейсы, определенные адресом.</span><span class="sxs-lookup"><span data-stu-id="6cab7-153">A packet sent to this address is delivered to all the interfaces identified by the address.</span></span> <span data-ttu-id="6cab7-154">Типы адресов многоадресной рассылки заменяют IPv4-адреса широковещательной рассылки.</span><span class="sxs-lookup"><span data-stu-id="6cab7-154">The multicast address types supersede the IPv4 broadcast addresses.</span></span>
