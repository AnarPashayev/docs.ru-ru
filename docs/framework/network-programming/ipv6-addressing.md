# <a name="ipv6-addressing"></a><span data-ttu-id="da732-103">Адресация IPv6</span><span class="sxs-lookup"><span data-stu-id="da732-103">IPv6 Addressing</span></span>

<span data-ttu-id="da732-104">Длина IPv6-адресов составляет 128 бит.</span><span class="sxs-lookup"><span data-stu-id="da732-104">In the Internet Protocol version 6 (IPv6), addresses are 128 bits long.</span></span> <span data-ttu-id="da732-105">Одна из причин такого большого адресного пространства заключается в том, чтобы разделить доступные адреса на иерархию доменов маршрутизации, отражающих топологию Интернета.</span><span class="sxs-lookup"><span data-stu-id="da732-105">One reason for such a large address space is to subdivide the available addresses into a hierarchy of routing domains that reflect the Internet's topology.</span></span> <span data-ttu-id="da732-106">Другая причина состоит в том, чтобы сопоставить адреса сетевых адаптеров (или интерфейсов), которые подключают устройства к сети.</span><span class="sxs-lookup"><span data-stu-id="da732-106">Another reason is to map the addresses of network adapters (or interfaces) that connect devices to the network.</span></span> <span data-ttu-id="da732-107">IPv6 обладает встроенной возможностью разрешать адреса на их самом низком уровне, который находится на уровне сетевого интерфейса, а также позволяет выполнять автоматическую настройку.</span><span class="sxs-lookup"><span data-stu-id="da732-107">IPv6 features an inherent capability to resolve addresses at their lowest level, which is at the network interface level, and also has auto-configuration capabilities.</span></span>

## <a name="text-representation"></a><span data-ttu-id="da732-108">Текстовое представление</span><span class="sxs-lookup"><span data-stu-id="da732-108">Text Representation</span></span>

<span data-ttu-id="da732-109">Ниже приведены три стандартные формы, которые используются для представления адресов IPv6 в виде текстовых строк:</span><span class="sxs-lookup"><span data-stu-id="da732-109">The following are the three conventional forms used to represent the IPv6 addresses as text strings:</span></span>

- <span data-ttu-id="da732-114">**Сжатая форма**.</span><span class="sxs-lookup"><span data-stu-id="da732-114">**Compressed form**.</span></span> <span data-ttu-id="da732-115">Из-за длины адреса часто используются адреса, содержащие длинную строку нулей.</span><span class="sxs-lookup"><span data-stu-id="da732-115">Due to the address length, it is common to have addresses containing a long string of символом двойного двоеточия (::).</span><span class="sxs-lookup"><span data-stu-id="da732-116">To simplify writing these addresses, use the compressed form, in which a single contiguous sequence of 0 data-stu-id="da732-117">This symbol can appear only once in an address.</span></span> <span data-ttu-id="da732-118">Например, адрес многоадресной рассылки `FFED:0:0:0:0:BA98:3210:4562` в сжатом виде выглядит как `FFED::BA98:3210:4562`.</span><span class="sxs-lookup"><span data-stu-id="da732-118">For example, the multicast address `FFED:0:0:0:0:BA98:3210:4562` in compressed form is `FFED::BA98:3210:4562`.</span></span> <span data-ttu-id="da732-119">Адрес одноадресной рассылки `3FFE:FFFF:0:0:8:800:20C4:0` в сжатом виде выглядит как `3FFE:FFFF::8:800:20C4:0`.</span><span class="sxs-lookup"><span data-stu-id="da732-119">The unicast address `3FFE:FFFF:0:0:8:800:20C4:0` in compressed form is `3FFE:FFFF::8:800:20C4:0`.</span></span> <span data-ttu-id="da732-120">Петлевой адрес `0:0:0:0:0:0:0:1` в сжатом виде выглядит как `::`1.</span><span class="sxs-lookup"><span data-stu-id="da732-120">The loopback address `0:0:0:0:0:0:0:1` in compressed form is `::`1.</span></span> <span data-ttu-id="da732-121">Незаданный адрес `0:0:0:0:0:0:0:0` в сжатом виде выглядит как `::`.</span><span class="sxs-lookup"><span data-stu-id="da732-121">The unspecified address `0:0:0:0:0:0:0:0` - <span data-ttu-id="da732-122">**Смешанная форма**.</span><span class="sxs-lookup"><span data-stu-id="da732-122">**Mixed form**.</span></span> <span data-ttu-id="da732-123">Эта форма объединяет адреса IPv4 и IPv6.</span><span class="sxs-lookup"><span data-stu-id="da732-123">This form combines IPv4 and IPv6 addresses.</span></span> <span data-ttu-id="da732-124">В этом случае адрес имеет формат n:n:n:n:n:n:d.d.d.d, где n соответствует шестнадцатеричным значениям шести элементов 16-разрядного IPv6-адреса старшего порядка, а d соответствует десятичному значению IPv4-адреса.</span><span class="sxs-lookup"><span data-stu-id="da732-124">In this case, the address format is n:n:n:n:n:n:d.d.d.d, where each n represents the hexadecimal values of the six IPv6 high-order 16-bit address elements, and each d represents the decimal value of an IPv4 address.</span></span>

## <a name="address-types"></a><span data-ttu-id="da732-125">Типы адресов</span><span class="sxs-lookup"><span data-stu-id="da732-125">Address Types</span></span>

<span data-ttu-id="da732-126">Начальные биты в адресе определяют конкретный тип адреса IPv6.</span><span class="sxs-lookup"><span data-stu-id="da732-126">The leading bits in the address define the specific IPv6 address type.</span></span> <span data-ttu-id="da732-127">Поле переменной длины, содержащее эти начальные биты, называется префиксом формата (FP).</span><span class="sxs-lookup"><span data-stu-id="da732-127">The variable-length field containing these leading bits is called a Format Prefix (FP).</span></span>

<span data-ttu-id="da732-128">IPv6-адрес одноадресной рассылки состоит из двух частей.</span><span class="sxs-lookup"><span data-stu-id="da732-128">An IPv6 unicast address is divided into two parts.</span></span> <span data-ttu-id="da732-129">Первая часть содержит префикс адреса, а вторая часть — идентификатор интерфейса.</span><span class="sxs-lookup"><span data-stu-id="da732-129">The first part contains the address prefix, and the second part contains the interface identifier.</span></span> <span data-ttu-id="da732-130">Краткий способ сочетания IPv6-адреса и префикса выглядит следующим образом: ipv6-address/prefix-length.</span><span class="sxs-lookup"><span data-stu-id="da732-130">A concise way to express an IPv6 address/prefix combination is as follows: ipv6-address/prefix-length.</span></span>

<span data-ttu-id="da732-131">Ниже приведен пример адреса с 64-разрядным префиксом.</span><span class="sxs-lookup"><span data-stu-id="da732-131">The following is an example of an address with a 64-bit prefix.</span></span>

<span data-ttu-id="da732-132">`3FFE:FFFF:0:CD30:0:0:0:0/64`.</span><span class="sxs-lookup"><span data-stu-id="da732-132">`3FFE:FFFF:0:CD30:0:0:0:0/64`.</span></span>

<span data-ttu-id="da732-133">В этом примере префикс — `3FFE:FFFF:0:CD30`.</span><span class="sxs-lookup"><span data-stu-id="da732-133">The prefix in this example is `3FFE:FFFF:0:CD30`.</span></span> <span data-ttu-id="da732-134">Адрес также может быть написан в сжатом виде, например `3FFE:FFFF:0:CD30::/64`.</span><span class="sxs-lookup"><span data-stu-id="da732-134">The address can also be written in a compressed form, as `3FFE:FFFF:0:CD30::/64`.</span></span>

<span data-ttu-id="da732-135">IPv6 определяет следующие типы адресов:</span><span class="sxs-lookup"><span data-stu-id="da732-135">IPv6 defines the following address types:</span></span>

- <span data-ttu-id="da732-136">**Адрес одноадресной рассылки**.</span><span class="sxs-lookup"><span data-stu-id="da732-136">**Unicast address**.</span></span> <span data-ttu-id="da732-137">Идентификатор для отдельного интерфейса.</span><span class="sxs-lookup"><span data-stu-id="da732-137">An identifier for a single interface.</span></span> <span data-ttu-id="da732-138">Пакет, отправленный на этот адрес, доставляется в определенный интерфейс.</span><span class="sxs-lookup"><span data-stu-id="da732-138">A packet sent to this address is delivered to the identified interface.</span></span> <span data-ttu-id="da732-139">Адреса одноадресной рассылки отличаются от адресов многоадресной рассылки по значению октета более высокого разряда.</span><span class="sxs-lookup"><span data-stu-id="da732-139">The unicast addresses are distinguished from the multicast addresses by the value of the high-order octet.</span></span> <span data-ttu-id="da732-140">Октет старшего порядка адресов многоадресной рассылки имеет шестнадцатеричное значение FF.</span><span class="sxs-lookup"><span data-stu-id="da732-140">The multicast addresses' high-order octet has the hexadecimal value of FF.</span></span> <span data-ttu-id="da732-141">Любое другое значение для этого октета определяет адрес одноадресной рассылки.</span><span class="sxs-lookup"><span data-stu-id="da732-141">Any other value for this octet identifies a unicast address.</span></span> <span data-ttu-id="da732-142">Ниже приведены различные типы адресов одноадресной рассылки.</span><span class="sxs-lookup"><span data-stu-id="da732-142">The following are different types of unicast addresses:</span></span>

  - <span data-ttu-id="da732-143">**Адреса локального канала**.</span><span class="sxs-lookup"><span data-stu-id="da732-143">**Link-local addresses**.</span></span> <span data-ttu-id="da732-144">Эти адреса используются в одном канале и имеют следующий формат: FE80::*InterfaceID*.</span><span class="sxs-lookup"><span data-stu-id="da732-144">These addresses are used on a single link and have the following format: FE80::*InterfaceID*.</span></span> <span data-ttu-id="da732-145">Адреса локального канала используются между узлами в канале для автонастройки адресов, обнаружения окружения или при отсутствии маршрутизаторов.</span><span class="sxs-lookup"><span data-stu-id="da732-145">Link-local addresses are used between nodes on a link for auto-address configuration, neighbor discovery, or when no routers are present.</span></span> <span data-ttu-id="da732-146">Адрес локального канала используется главным образом во время запуска и в случае, когда система еще не получила адреса большей области.</span><span class="sxs-lookup"><span data-stu-id="da732-146">A link-local address is used primarily at startup and when the system has not yet acquired addresses of larger scope.</span></span>

  - <span data-ttu-id="da732-147">**Адреса локальных узлов**.</span><span class="sxs-lookup"><span data-stu-id="da732-147">**Site-local addresses**.</span></span> <span data-ttu-id="da732-148">Эти адреса используются в одном узле и имеют следующий формат: FEC0::*SubnetID*:*InterfaceID*.</span><span class="sxs-lookup"><span data-stu-id="da732-148">These addresses are used on a single site and have the following format: FEC0::*SubnetID*:*InterfaceID*.</span></span> <span data-ttu-id="da732-149">Адреса локальных узлов используются для адресации внутри узла и не требуют глобального префикса.</span><span class="sxs-lookup"><span data-stu-id="da732-149">The site-local addresses are used for addressing inside a site without the need for a global prefix.</span></span>

  - <span data-ttu-id="da732-150">**Глобальные IPv6-адреса одноадресной рассылки**.</span><span class="sxs-lookup"><span data-stu-id="da732-150">**Global IPv6 unicast addresses**.</span></span> <span data-ttu-id="da732-151">Эти адреса могут использоваться в Интернете и имеют следующий формат: 010 (FP, 3 бита) TLA ID (13 бит) Reserved (8 бит) NLA ID (24 бита) SLA ID (16 бит) *InterfaceID* (64 бита).</span><span class="sxs-lookup"><span data-stu-id="da732-151">These addresses can be used across the Internet and have the following format: 010(FP, 3 bits) TLA ID (13 bits) Reserved (8 bits) NLA ID (24 bits) SLA ID (16 bits) *InterfaceID* (64 bits).</span></span>

- <span data-ttu-id="da732-152">**Адрес многоадресной рассылки**.</span><span class="sxs-lookup"><span data-stu-id="da732-152">**Multicast address**.</span></span> <span data-ttu-id="da732-153">Идентификатор набора интерфейсов (обычно принадлежащих разным узлам).</span><span class="sxs-lookup"><span data-stu-id="da732-153">An identifier for a set of interfaces (typically belonging to different nodes).</span></span> <span data-ttu-id="da732-154">Пакет, отправленный на этот адрес, доставляется во все интерфейсы, определенные адресом.</span><span class="sxs-lookup"><span data-stu-id="da732-154">A packet sent to this address is delivered to all the interfaces identified by the address.</span></span> <span data-ttu-id="da732-155">Типы адресов многоадресной рассылки заменяют IPv4-адреса широковещательной рассылки.</span><span class="sxs-lookup"><span data-stu-id="da732-155">The multicast address types supersede the IPv4 broadcast addresses.</span></span>

- <span data-ttu-id="da732-156">**Адреса произвольной рассылки**.</span><span class="sxs-lookup"><span data-stu-id="da732-156">**Anycast address**.</span></span> <span data-ttu-id="da732-157">Идентификатор набора интерфейсов (обычно принадлежащих разным узлам).</span><span class="sxs-lookup"><span data-stu-id="da732-157">An identifier for a set of interfaces (typically belonging to different nodes).</span></span> <span data-ttu-id="da732-158">Пакет, отправленный на этот адрес, доставляется только в один интерфейс, определенный адресом.</span><span class="sxs-lookup"><span data-stu-id="da732-158">A packet sent to this address is delivered to only one interface identified by the address.</span></span> <span data-ttu-id="da732-159">Это ближайший интерфейс, определенный метриками маршрутизации.</span><span class="sxs-lookup"><span data-stu-id="da732-159">This is the nearest interface as identified by routing metrics.</span></span> <span data-ttu-id="da732-160">Адреса произвольной рассылки берутся из пространства адресов одноадресной рассылки и синтаксически не отличаются.</span><span class="sxs-lookup"><span data-stu-id="da732-160">Anycast addresses are taken from the unicast address space and are not syntactically distinguishable.</span></span> <span data-ttu-id="da732-161">Адресный интерфейс выполняет отличие адресов одноадресной рассылки от адресов произвольной рассылки в качестве функции его конфигурации.</span><span class="sxs-lookup"><span data-stu-id="da732-161">The addressed interface performs the distinction between unicast and anycast addresses as a function of its configuration.</span></span>
