# <a name="socket-performance-enhancements-in-version-35"></a><span data-ttu-id="09191-102">Улучшения производительности сокетов в версии 3.5</span><span class="sxs-lookup"><span data-stu-id="09191-102">Socket Performance Enhancements in Version 3.5</span></span>
<span data-ttu-id="09191-103">Класс <xref:System.Net.Sockets.Socket?displayProperty=nameWithType> был оптимизирован в версии 3.5 для обеспечения максимальной производительности приложений, которые используют асинхронные операции ввода-вывода по сети.</span><span class="sxs-lookup"><span data-stu-id="09191-103">The <xref:System.Net.Sockets.Socket?displayProperty=nameWithType> class has been enhanced in Version 3.5 for use by applications that use asynchronous network I/O to achieve the highest performance.</span></span> <span data-ttu-id="09191-104">В рамках набора усовершенствований класса <xref:System.Net.Sockets.Socket>, предоставляющих альтернативный шаблон асинхронной модели, который может использоваться специализированными высокопроизводительными приложениями сокетов, был добавлен ряд новых классов.</span><span class="sxs-lookup"><span data-stu-id="09191-104">A series of new classes have been added as part of a set of enhancements to the <xref:System.Net.Sockets.Socket> class that provide an alternative asynchronous pattern that can be used by specialized high-performance socket applications.</span></span> <span data-ttu-id="09191-105">Эти усовершенствования были специально разработаны для сетевых серверных приложений, требующих высокой производительности.</span><span class="sxs-lookup"><span data-stu-id="09191-105">These enhancements were specifically designed for network server applications that require high performance.</span></span> <span data-ttu-id="09191-106">Приложение может быть основано исключительно на улучшенном шаблоне асинхронной модели или может использовать его только в определенных областях (например, при получении больших объемов данных).</span><span class="sxs-lookup"><span data-stu-id="09191-106">An application can use the enhanced asynchronous pattern exclusively, or only in targeted hot areas of their application (when receiving large amounts of data, for example).</span></span>  
  
## <a name="class-enhancements"></a><span data-ttu-id="09191-107">Улучшения класса</span><span class="sxs-lookup"><span data-stu-id="09191-107">Class Enhancements</span></span>  
 <span data-ttu-id="09191-108">Главное преимущество этих улучшений — устранение необходимости в повторном выделении и синхронизации объектов во время асинхронных операций ввода-вывода больших объемов данных на сокете.</span><span class="sxs-lookup"><span data-stu-id="09191-108">The main feature of these enhancements is the avoidance of the repeated allocation and synchronization of objects during high-volume asynchronous socket I/O.</span></span> <span data-ttu-id="09191-109">Конструктивный шаблон на основе методов Begin и End, в настоящее время реализуемый классом <xref:System.Net.Sockets.Socket> для асинхронных операций ввода-вывода на сокете, требует выделения объекта <xref:System.IAsyncResult?displayProperty=nameWithType> для каждой асинхронной операции.</span><span class="sxs-lookup"><span data-stu-id="09191-109">The Begin/End design pattern currently implemented by the <xref:System.Net.Sockets.Socket> class for asynchronous socket I/O requires a <xref:System.IAsyncResult?displayProperty=nameWithType> object be allocated for each asynchronous socket operation.</span></span>  
  
 <span data-ttu-id="09191-110">В новом классе <xref:System.Net.Sockets.Socket> асинхронные операции на сокете описываются повторно используемыми объектами класса <xref:System.Net.Sockets.SocketAsyncEventArgs?displayProperty=nameWithType>, которые выделяются и обслуживаются приложением.</span><span class="sxs-lookup"><span data-stu-id="09191-110">In the new <xref:System.Net.Sockets.Socket> class enhancements, asynchronous socket operations are described by reusable <xref:System.Net.Sockets.SocketAsyncEventArgs?displayProperty=nameWithType> class objects allocated and maintained by the application.</span></span> <span data-ttu-id="09191-111">Высокопроизводительному приложению сокета лучше всего известно, сколько перекрывающихся операций сокета необходимо поддерживать.</span><span class="sxs-lookup"><span data-stu-id="09191-111">High-performance socket applications know best the amount of overlapped socket operations that must be sustained.</span></span> <span data-ttu-id="09191-112">Приложение может создать столько объектов <xref:System.Net.Sockets.SocketAsyncEventArgs>, сколько ему требуется.</span><span class="sxs-lookup"><span data-stu-id="09191-112">The application can create as many of the <xref:System.Net.Sockets.SocketAsyncEventArgs> objects that it needs.</span></span> <span data-ttu-id="09191-113">Например, если для обеспечения постоянной готовности к приему входящих подключений клиентов серверному приложению требуется 15 операций на сокете, оно может выделить для этой цели 15 повторно используемых объектов <xref:System.Net.Sockets.SocketAsyncEventArgs>.</span><span class="sxs-lookup"><span data-stu-id="09191-113">For example, if a server application needs to have 15 socket accept operations outstanding at all times to support incoming client connection rates, it can allocate 15 reusable <xref:System.Net.Sockets.SocketAsyncEventArgs> objects in advance for that purpose.</span></span>  
  
 <span data-ttu-id="09191-114">Шаблон выполнения асинхронной операции на сокете с помощью этого класса состоит из перечисленных ниже действий.</span><span class="sxs-lookup"><span data-stu-id="09191-114">The pattern for performing an asynchronous socket operation with this class consists of the following steps:</span></span>  
  
1. <span data-ttu-id="09191-115">Выделите новый объект контекста <xref:System.Net.Sockets.SocketAsyncEventArgs> или получите свободный объект из пула приложений.</span><span class="sxs-lookup"><span data-stu-id="09191-115">Allocate a new <xref:System.Net.Sockets.SocketAsyncEventArgs> context object, or get a free one from an application pool.</span></span>  
  
2. <span data-ttu-id="09191-116">Задайте свойства объекта контекста в соответствии с операцией, которая должна быть выполнена (это могут быть, например, метод делегата обратного вызова и буфер данных).</span><span class="sxs-lookup"><span data-stu-id="09191-116">Set properties on the context object to the operation about to be performed (the callback delegate method and data buffer, for example).</span></span>  
  
3. <span data-ttu-id="09191-117">Вызовите соответствующий метод сокета (xxxAsync), чтобы инициировать асинхронную операцию.</span><span class="sxs-lookup"><span data-stu-id="09191-117">Call the appropriate socket method (xxxAsync) to initiate the asynchronous operation.</span></span>  
  
4. <span data-ttu-id="09191-118">Если асинхронный метод сокета (xxxAsync) возвращает значение true в результате обратного вызова, запросите состояние завершения из свойств контекста.</span><span class="sxs-lookup"><span data-stu-id="09191-118">If the asynchronous socket method (xxxAsync) returns true in the callback, query the context properties for completion status.</span></span>  
  
5. <span data-ttu-id="09191-119">Если асинхронный метод сокета (xxxAsync) возвращает значение false в результате обратного вызова, операция была выполнена синхронно.</span><span class="sxs-lookup"><span data-stu-id="09191-119">If the asynchronous socket method (xxxAsync) returns false in the callback, the operation completed synchronously.</span></span> <span data-ttu-id="09191-120">Результат операции можно запросить из свойств контекста.</span><span class="sxs-lookup"><span data-stu-id="09191-120">The context properties may be queried for the operation result.</span></span>  
  
6. <span data-ttu-id="09191-121">Используйте контекст повторно для другой операции, верните его в пул или удалите.</span><span class="sxs-lookup"><span data-stu-id="09191-121">Reuse the context for another operation, put it back in the pool, or discard it.</span></span>  
  
 <span data-ttu-id="09191-122">Время существования нового объекта контекста для асинхронной операции на сокете определяется ссылками в коде приложения и ссылками асинхронного ввода-вывода.</span><span class="sxs-lookup"><span data-stu-id="09191-122">The lifetime of the new asynchronous socket operation context object is determined by references in the application code and asynchronous I/O references.</span></span> <span data-ttu-id="09191-123">Приложению не требуется сохранять ссылку на объект контекста асинхронной операции на сокете после ее передачи в качестве параметра в один из методов асинхронной операции на сокете.</span><span class="sxs-lookup"><span data-stu-id="09191-123">It is not necessary for the application to retain a reference to an asynchronous socket operation context object after it is submitted as a parameter to one of the asynchronous socket operation methods.</span></span> <span data-ttu-id="09191-124">Ссылка сохраняется до тех пор, пока не будет возвращен обратный вызов завершения.</span><span class="sxs-lookup"><span data-stu-id="09191-124">It will remain referenced until the completion callback returns.</span></span> <span data-ttu-id="09191-125">Однако ссылку на объект контекста полезно сохранять, чтобы приложение могло использовать этот объект повторно для последующей асинхронной операции на сокете.</span><span class="sxs-lookup"><span data-stu-id="09191-125">However it is advantageous for the application to retain the reference to the context object so that it can be reused for a future asynchronous socket operation.</span></span>
