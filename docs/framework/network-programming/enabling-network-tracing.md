- <span data-ttu-id="b1e2b-105">Скомпилируйте код с включенной трассировкой.</span><span class="sxs-lookup"><span data-stu-id="b1e2b-105">Compile your code with tracing enabled.</span></span> <span data-ttu-id="b1e2b-106">См. практическое руководство по [ Условная компиляция с использованием атрибутов Trace и Debug](../../../docs/framework/debug-trace-profile/how-to-compile-conditionally-with-trace-and-debug.md).</span><span class="sxs-lookup"><span data-stu-id="b1e2b-106">See [How to: Compile Conditionally with Trace and Debug](../../../docs/framework/debug-trace-profile/how-to-compile-conditionally-with-trace-and-debug.md) for more information about the compiler switches required to enable tracing.</span></span>  
  
- <span data-ttu-id="b1e2b-107">Укажите назначение для выходных данных трассировки.</span><span class="sxs-lookup"><span data-stu-id="b1e2b-107">Specify a destination for tracing output.</span></span>  
  
- <span data-ttu-id="b1e2b-108">Настройте поведение трассировки сети.</span><span class="sxs-lookup"><span data-stu-id="b1e2b-108">Configure the behavior of network tracing.</span></span> <span data-ttu-id="b1e2b-109">См. практическое руководство по [ Настройка трассировки сети](../../../docs/framework/network-programming/how-to-configure-network-tracing.md).</span><span class="sxs-lookup"><span data-stu-id="b1e2b-109">See [How to: Configure Network Tracing](../../../docs/framework/network-programming/how-to-configure-network-tracing.md) for detailed information.</span></span>  
  
 <span data-ttu-id="b1e2b-110">Чаще всего в качестве назначений для трассировки сети (прослушивателей трассировки) выступают прослушиватели по умолчанию и файлы журнала.</span><span class="sxs-lookup"><span data-stu-id="b1e2b-110">The most common trace destinations, also referred to as trace listeners, are the default listener and the log file.</span></span>  
  
 <span data-ttu-id="b1e2b-111">Если прослушиватель трассировки не указан, используется прослушиватель по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="b1e2b-111">Tracing uses the default listener if you do not specify a trace listener.</span></span> <span data-ttu-id="b1e2b-112">Вы можете просматривать сообщения, отправляемые в прослушиватель по умолчанию при выполнении кода в отладчике с поддержкой управляемого кода, такого как отладчик среды CLR из состава пакета .NET Framework SDK или DBwin32.exe из пакета Windows SDK.</span><span class="sxs-lookup"><span data-stu-id="b1e2b-112">You can view messages sent to the default listener by executing your code in a managed code-enabled debugger such as the CLR Debugger shipped with the .NET Framework SDK, or DBwin32.exe shipped with the Windows SDK.</span></span> <span data-ttu-id="b1e2b-113">При использовании отладчика среды CLR сообщения трассировки появляются в окне **Выходные данные**.</span><span class="sxs-lookup"><span data-stu-id="b1e2b-113">Using the CLR Debugger, the trace messages appear in the **Output** window.</span></span>  
  
 <span data-ttu-id="b1e2b-114">Если вы хотите записывать результаты трассировки в файл, укажите файл журнала в параметрах конфигурации, как показано в следующем примере.</span><span class="sxs-lookup"><span data-stu-id="b1e2b-114">If you prefer to use a file to receive traces, you can specify a log file by using configuration settings, as shown in the following example.</span></span> <span data-ttu-id="b1e2b-115">(Общие сведения о файлах конфигурации см. в разделе [Файлы конфигурации](../../../docs/framework/configure-apps/index.md).)</span><span class="sxs-lookup"><span data-stu-id="b1e2b-115">(For a general discussion of configuration files, see [Configuration Files](../../../docs/framework/configure-apps/index.md).)</span></span>  
  
 <span data-ttu-id="b1e2b-116">Чтобы отправлять сообщения трассировки в файл журнала, добавьте следующий узел в узел `<system.diagnostics>` соответствующего файла конфигурации приложения или компьютера.</span><span class="sxs-lookup"><span data-stu-id="b1e2b-116">To send traces to a log file, add the following node to the `<system.diagnostics>` node of the appropriate configuration file (application or machine).</span></span> <span data-ttu-id="b1e2b-117">При необходимости имя файла (trace.log) можно изменить.</span><span class="sxs-lookup"><span data-stu-id="b1e2b-117">You can change the name of the file (trace.log) to suit your needs.</span></span>  
  
```xml  
<system.diagnostics>  
  <trace autoflush="true" indentsize="4">  
    <listeners>  
      <add name="file" type="System.Diagnostics.TextWriterTraceListener" initializeData="trace.log"/>  
    </listeners>   
  </trace>  
</system.diagnostics>  
```
