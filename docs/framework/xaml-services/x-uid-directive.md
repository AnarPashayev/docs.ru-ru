## <a name="remarks"></a><span data-ttu-id="16504-108">Заметки</span><span class="sxs-lookup"><span data-stu-id="16504-108">Remarks</span></span>  
 <span data-ttu-id="16504-109">В [MS-XAML] `x:Uid` определен как директива.</span><span class="sxs-lookup"><span data-stu-id="16504-109">In [MS-XAML], `x:Uid` is defined as a directive.</span></span> <span data-ttu-id="16504-110">Дополнительные сведения см. в [разделе\[MS-XAML\] 5.3.6](https://docs.microsoft.com/previous-versions/msp-n-p/ff650760(v=pandp.10)).</span><span class="sxs-lookup"><span data-stu-id="16504-110">For more information, see [\[MS-XAML\] Section 5.3.6](https://docs.microsoft.com/previous-versions/msp-n-p/ff650760(v=pandp.10)).</span></span>  
  
 <span data-ttu-id="16504-111">`x:Uid` является дискретным от `x:Name` обоих из-за указанного сценария локализации XAML, поэтому идентификаторы, используемые для локализации, не имеют зависимостей от влияния модели программирования на `x:Name`.</span><span class="sxs-lookup"><span data-stu-id="16504-111">`x:Uid` is discrete from `x:Name` both because of the stated XAML localization scenario and so that identifiers that are used for localization have no dependencies on the programming model implications of `x:Name`.</span></span> <span data-ttu-id="16504-112">Кроме того, `x:Name` регулируется областью имен XAML; Однако `x:Uid` не регулируется какой-либо из концепций, определенных языком XAML, для обеспечения уникальности.</span><span class="sxs-lookup"><span data-stu-id="16504-112">Also, `x:Name` is governed by the XAML namescope; however, `x:Uid` is not governed by any XAML language defined concept of uniqueness enforcement.</span></span> <span data-ttu-id="16504-113">Обработчики XAML в широком смысле (процессоры, не являющиеся частью процесса локализации) не должны обеспечивать уникальность `x:Uid` значений.</span><span class="sxs-lookup"><span data-stu-id="16504-113">XAML processors in a broad sense (processors that are not part of the localization process) are not expected to enforce uniqueness of `x:Uid` values.</span></span> <span data-ttu-id="16504-114">Эта ответственность концептуально полагается на источник значений.</span><span class="sxs-lookup"><span data-stu-id="16504-114">That responsibility is conceptually on the originator of the values.</span></span> <span data-ttu-id="16504-115">Ожидание уникальности значений `x:Uid` в одном источнике XAML целесообразно для потребителей значений, таких как выделенные процессы или средства глобализации.</span><span class="sxs-lookup"><span data-stu-id="16504-115">The expectation of uniqueness of `x:Uid` values within a single XAML source is reasonable for consumers of the values, such as dedicated globalization processes or tools.</span></span> <span data-ttu-id="16504-116">Типичная модель уникальности заключается в том, что `x:Uid` значения уникальны в XML-файле, который представляет XAML.</span><span class="sxs-lookup"><span data-stu-id="16504-116">The typical uniqueness model is that `x:Uid` values are unique within an XML-encoded file that represents XAML.</span></span>  
  
 <span data-ttu-id="16504-117">Средства, имеющие существенные знания о конкретной схеме XAML, могут применять `x:Uid` только для истинно локализуемых строк, а не во всех случаях, когда в разметке встречается текстовое значение строки.</span><span class="sxs-lookup"><span data-stu-id="16504-117">Tools that have significant knowledge of a particular XAML schema can choose to apply `x:Uid` only for true localizable strings, instead of for all cases where a text string value is encountered in markup.</span></span>  
  
 <span data-ttu-id="16504-118">Платформы могут указывать конкретное свойство в своей объектной модели, чтобы быть псевдонимом для `x:Uid`, применяя <xref:System.Windows.Markup.UidPropertyAttribute> атрибута к определяющему типу.</span><span class="sxs-lookup"><span data-stu-id="16504-118">Frameworks can specify a particular property in their object model to be an alias for `x:Uid` by applying the attribute <xref:System.Windows.Markup.UidPropertyAttribute> to the defining type.</span></span> <span data-ttu-id="16504-119">Если платформа определяет конкретное свойство, недопустимо указывать как `x:Uid`, так и член с псевдонимом для одного и того же объекта.</span><span class="sxs-lookup"><span data-stu-id="16504-119">If a framework specifies a particular property, it is not valid to specify both `x:Uid` and the aliased member on the same object.</span></span> <span data-ttu-id="16504-120">Если указаны и `x:Uid`, и член с псевдонимом, .NET Framework API служб XAML, как правило, выдает <xref:System.Xaml.XamlDuplicateMemberException> в этом случае.</span><span class="sxs-lookup"><span data-stu-id="16504-120">If both `x:Uid` and the aliased member are specified, the .NET Framework XAML Services API typically throws <xref:System.Xaml.XamlDuplicateMemberException> for this case.</span></span>
