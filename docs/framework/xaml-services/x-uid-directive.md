 <span data-ttu-id="3aa64-111">`x:Uid` является дискретным из `x:Name` оба из-за указанного сценария локализации XAML и таким образом, чтобы идентификаторы, используемые для локализации не имеют зависимостей от особенностей модели программирования `x:Name`.</span><span class="sxs-lookup"><span data-stu-id="3aa64-111">`x:Uid` is discrete from `x:Name` both because of the stated XAML localization scenario and so that identifiers that are used for localization have no dependencies on the programming model implications of `x:Name`.</span></span> <span data-ttu-id="3aa64-112">Кроме того `x:Name` регулируется области видимости имен XAML;  <span data-ttu-id="3aa64-118">Платформы можно указать конкретного свойства в модели объекта в качестве псевдонима для `x:Uid` путем применения атрибута <xref:System.Windows.Markup.UidPropertyAttribute> типу.</span><span class="sxs-lookup"><span data-stu-id="3aa64-118">Frameworks can specify a particular property in their object model to be an alias for `x:Uid` by applying the attribute <xref:System.Windows.Markup.UidPropertyAttribute> to the defining type.</span></span> <span data-ttu-id="3aa64-119">Если платформа задает определенное свойство, не допускается указывать оба `x:Uid` и член с псевдонимом в одном объекте.</span><span class="sxs-lookup"><span data-stu-id="3aa64-119">If a framework specifies a particular property, it is not valid to specify both `x:Uid` and the aliased member on the same object.</span></span> <span data-ttu-id="3aa64-120">Если оба `x:Uid` и указаны член с псевдонимом, обычно вызывает API служб XAML платформы .NET Framework <xref:System.Xaml.XamlDuplicateMemberException> для этого случая.</span><span class="sxs-lookup"><span data-stu-id="3aa64-120">If both `x:Uid` and the aliased member are specified, the .NET Framework XAML Services API typically throws <xref:System.Xaml.XamlDuplicateMemberException> for this case.</span></span>
