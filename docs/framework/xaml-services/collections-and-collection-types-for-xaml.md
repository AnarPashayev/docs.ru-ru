<span data-ttu-id="d9d0b-105">По сути, любая связь в XAML, где существует несколько дочерних элементов в области объекта XAML или элемента свойства XAML, должна быть реализована как коллекция.</span><span class="sxs-lookup"><span data-stu-id="d9d0b-105">Conceptually, any relationship in XAML where there are multiple child items within the scope of a XAML object element or XAML property element must be implemented as a collection.</span></span> <span data-ttu-id="d9d0b-106">Эта коллекция должна быть связана с определенным свойством XAML типа XAML, который является родительским в этой связи.</span><span class="sxs-lookup"><span data-stu-id="d9d0b-106">That collection must be associated with a particular XAML property of the XAML type that is the parent in that relationship.</span></span> <span data-ttu-id="d9d0b-107">Свойство должно быть коллекцией, так как обработчик XAML должен назначить каждый элемент в разметке только что добавленным элементом свойства резервной коллекции.</span><span class="sxs-lookup"><span data-stu-id="d9d0b-107">The property must be a collection because a XAML processor expects to assign each item in markup to be a newly added item of the backing collection property.</span></span>

## <a name="read-only-collections-and-initialization-logic"></a><span data-ttu-id="d9d0b-123">Коллекции только для чтения и логика инициализации</span><span class="sxs-lookup"><span data-stu-id="d9d0b-123">Read-Only Collections and Initialization Logic</span></span>

<span data-ttu-id="d9d0b-124">В .NET Framework программировании это распространенный шаблон проектирования, позволяющий сделать любое свойство, содержащее значение коллекции, в качестве коллекции только для чтения.</span><span class="sxs-lookup"><span data-stu-id="d9d0b-124">In .NET Framework programming, it is a common design pattern to make any property that holds a value of a collection as a read-only collection.</span></span> <span data-ttu-id="d9d0b-125">Этот шаблон позволяет экземпляру, владеющему свойством коллекции, лучше контролировать то, что происходит с коллекцией.</span><span class="sxs-lookup"><span data-stu-id="d9d0b-125">This pattern permits the instance that owns the collection property to better control what happens to the collection..</span></span> <span data-ttu-id="d9d0b-126">В частности, шаблон предотвращает случайную замену всей уже существующей коллекции, задавая свойство.</span><span class="sxs-lookup"><span data-stu-id="d9d0b-126">Specifically, the pattern prevents accidental replacement of the entire pre-existing collection by setting the property.</span></span> <span data-ttu-id="d9d0b-127">В этом шаблоне любой доступ к коллекции с помощью вызывающих объектов должен осуществляться вызовом методов или свойств, которые поддерживаются типом коллекции и (или) соответствующими интерфейсами коллекции, <xref:System.Collections.IList>такими как.</span><span class="sxs-lookup"><span data-stu-id="d9d0b-127">In this pattern, any access to the collection by callers should instead be made by calling methods or properties as supported by the collection type and/or the relevant collection interfaces such as <xref:System.Collections.IList>.</span></span>
