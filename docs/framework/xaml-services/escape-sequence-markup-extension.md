# <a name="-escape-sequence--markup-extension"></a>{}Escape-последовательность/расширение разметки
Предоставляет escape-последовательность XAML для значений атрибутов. Escape-последовательность позволяет интерпретировать последующие значения в атрибуте как литерал.  
  
## <a name="xaml-attribute-usage"></a>Использование атрибута XAML  
  
```xaml  
<object property="{} literalValue" .../>  
```  
  
## <a name="xaml-property-element-usage"></a>Использование элемента свойства XAML  
  
```xaml  
<object>  
  <object.property>  
    {} literalValue  
  </object.property>  
</object>  
```  
  
## <a name="xaml-values"></a>Значения XAML  
  
|||  
|-|-|  
|*литералвалуе*|Строка литерала, следующая за escape-последовательностью. Обычно эта строка содержит открывающую или закрывающую фигурную скобку ({или}).|  
  
## <a name="remarks"></a>Примечания  
 Escape-последовательность ({}) используется, чтобы открывающую фигурную скобку ({) можно было использовать в качестве литерального символа в XAML.  
  
 Средства чтения XAML обычно используют открывающую фигурную скобку ({) для обозначения точки входа расширения разметки; однако сначала проверяется следующий символ, чтобы определить, является ли он закрывающей фигурной скобкой (}). Только если две фигурные скобки{}() являются смежными, они считаются escape-последовательностью.  
  
 Если обнаружена escape-последовательность, средство чтения XAML должно обработать оставшуюся часть строки в виде строки. Однако если escape-последовательность применяется к элементу, имеющему преобразователь типов, строка может преобразовывать тип, когда она интерпретируется модулем записи XAML.  
  
 Escape-последовательность не является расширением разметки и не поддерживается классом. Однако это соглашение о том, что средства чтения XAML (включая пользовательские средства чтения XAML) должны учитывать.  
  
 Кавычки (") нельзя использовать в качестве escape-последовательности таким образом. Если необходимо задать кавычку как значение свойства для свойства, не использующего содержимое, используйте синтаксис элемента свойства и поместите кавычку как строку внутри элемента свойства или используйте сущность символов XML. Для свойства содержимого кавычки могут быть всего содержимого.  
  
 Escape-последовательность ({}) часто требуется при указании типа XML, который должен включать квалификатор пространства имен в расположение, где может отображаться расширение разметки XAML. Это включает начало значения атрибута XAML и в расширении разметки сразу после знака равенства (=).
