## <a name="wpf-usage-notes"></a><span data-ttu-id="130be-129">Примечания об использовании WPF</span><span class="sxs-lookup"><span data-stu-id="130be-129">WPF Usage Notes</span></span>  
 `x:Subclass` <span data-ttu-id="130be-130">может быть на корневой страницы либо в <xref:System.Windows.Application> корневой в определении приложения, которая уже содержит `x:Class`.</span><span class="sxs-lookup"><span data-stu-id="130be-130">can be on a page root or on the <xref:System.Windows.Application> root in the application definition, which already has `x:Class`.</span></span> <span data-ttu-id="130be-131">Объявление `x:Subclass` для любого элемента, отличного от корня страницы или приложения, и не указывать там, где нет `x:Class` существует, приводит к ошибке времени компиляции.</span><span class="sxs-lookup"><span data-stu-id="130be-131">Declaring `x:Subclass` on any element other than a page or application root, or specifying it where no `x:Class` exists, causes a compile-time error.</span></span>  
  
 <span data-ttu-id="130be-132">Создание производных классов, правильно работающих `x:Subclass` сценарий является довольно сложной задачей.</span><span class="sxs-lookup"><span data-stu-id="130be-132">Creating derived classes that work correctly for the `x:Subclass` scenario is fairly complex.</span></span> <span data-ttu-id="130be-133">Может потребоваться проверка промежуточных файлов (g-файлы в папке obj проекта путем компиляции разметки, с именами, включающими имена файлов).</span><span class="sxs-lookup"><span data-stu-id="130be-133">You might need to examine the intermediate files (the .g files produced in the obj folder of your project by markup compile, with names that incorporate the .xaml file names).</span></span> <span data-ttu-id="130be-134">Промежуточные файлы могут помочь определить происхождение отдельных программных конструкций в разделяемых классах в скомпилированном приложении.</span><span class="sxs-lookup"><span data-stu-id="130be-134">These intermediate files can help you determine the origin of certain programming constructs in the joined partial classes in the compiled application.</span></span>  
  
 <span data-ttu-id="130be-135">Обработчики событий в производном классе должны быть `internal override` (`Friend Overrides` в Microsoft Visual Basic) для переопределения заглушки для обработчиков, созданный в промежуточный класс во время компиляции.</span><span class="sxs-lookup"><span data-stu-id="130be-135">Event handlers in the derived class must be `internal override` (`Friend Overrides` in Microsoft Visual Basic) in order to override the stubs for the handlers as created in the intermediate class during compilation.</span></span> <span data-ttu-id="130be-136">В противном случае реализация производного класса скрыть (теневых) реализацию промежуточного класса и промежуточный класс обработчики не вызываются.</span><span class="sxs-lookup"><span data-stu-id="130be-136">Otherwise, the derived class implementations hide (shadow) the intermediate class implementation and the intermediate class handlers are not invoked.</span></span>  
  
 <span data-ttu-id="130be-137">При определении обоих `x:Class` и `x:Subclass`, необходимо использовать любую реализацию для класса, на который ссылается `x:Class`.</span><span class="sxs-lookup"><span data-stu-id="130be-137">When you define both `x:Class` and `x:Subclass`, you do not need to provide any implementation for the class that is referenced by `x:Class`.</span></span> <span data-ttu-id="130be-138">Необходимо присвоить ему имя с помощью `x:Class` таким образом, чтобы у компилятора есть некоторые рекомендации для класса, который он создает промежуточные файлы, (компилятор не выбирает имя по умолчанию в данном случае).</span><span class="sxs-lookup"><span data-stu-id="130be-138">You only need to give it a name via the `x:Class` attribute so that the compiler has some guidance for the class that it creates in the intermediate files (the compiler does not select a default name in this case).</span></span> <span data-ttu-id="130be-139">Можно предоставить `x:Class` реализации класса; Однако это не типичный сценарий использования обоих `x:Class` и `x:Subclass`.</span><span class="sxs-lookup"><span data-stu-id="130be-139">You can give the `x:Class` class an implementation; however, this is not the typical scenario for using both `x:Class` and `x:Subclass`.</span></span>
