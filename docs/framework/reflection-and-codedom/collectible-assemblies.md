# <a name="collectible-assemblies-for-dynamic-type-generation"></a><span data-ttu-id="55100-104">Забираемые сборки для динамической генерации типа</span><span class="sxs-lookup"><span data-stu-id="55100-104">Collectible assemblies for dynamic type generation</span></span>

<span data-ttu-id="55100-105">*Забираемые сборки* — это динамические сборки, которые можно выгрузить без выгрузки домена приложения, в котором они были созданы.</span><span class="sxs-lookup"><span data-stu-id="55100-105">*Collectible assemblies* are dynamic assemblies that can be unloaded without unloading the application domain in which they were created.</span></span> <span data-ttu-id="55100-106">Всю управляемую и неуправляемую память, используемую забираемой сборкой и содержащимися в ней типами, можно освободить.</span><span class="sxs-lookup"><span data-stu-id="55100-106">All managed and unmanaged memory used by a collectible assembly and the types it contains can be reclaimed.</span></span> <span data-ttu-id="55100-107">Сведения, такие как имя сборки, удаляются из внутренних таблиц.</span><span class="sxs-lookup"><span data-stu-id="55100-107">Information such as the assembly name is removed from internal tables.</span></span>

<span data-ttu-id="55100-108">Чтобы включить выгрузку, используйте флаг <xref:System.Reflection.Emit.AssemblyBuilderAccess.RunAndCollect?displayProperty=nameWithType> при создании динамической сборки.</span><span class="sxs-lookup"><span data-stu-id="55100-108">To enable unloading, use the <xref:System.Reflection.Emit.AssemblyBuilderAccess.RunAndCollect?displayProperty=nameWithType> flag when you create a dynamic assembly.</span></span> <span data-ttu-id="55100-109">Сборка является временной (то есть ее невозможно сохранить), а также на нее распространяются ограничения, описанные в разделе [Ограничения забираемых сборок](#restrictions-on-collectible-assemblies).</span><span class="sxs-lookup"><span data-stu-id="55100-109">The assembly is transient (that is, it cannot be saved) and is subject to limitations described in the [Restrictions on Collectible Assemblies](#restrictions-on-collectible-assemblies) section.</span></span> <span data-ttu-id="55100-110">Общеязыковая среда выполнения (CLR) автоматически выгружает забираемую сборку, когда вы освободите все связанные со сборкой объекты.</span><span class="sxs-lookup"><span data-stu-id="55100-110">The common language runtime (CLR) unloads a collectible assembly automatically when you release all objects associated with the assembly.</span></span> <span data-ttu-id="55100-111">Во всех прочих отношениях забираемые сборки создаются и используются так же, как и другие динамические сборки.</span><span class="sxs-lookup"><span data-stu-id="55100-111">In all other respects, collectible assemblies are created and used in the same way as other dynamic assemblies.</span></span>

## <a name="lifetime-of-collectible-assemblies"></a><span data-ttu-id="55100-112">Время существования забираемых сборок</span><span class="sxs-lookup"><span data-stu-id="55100-112">Lifetime of collectible assemblies</span></span>

<span data-ttu-id="55100-113">Время существования забираемой сборки определяется наличием ссылок на содержащиеся в ней типы и объекты, созданные из этих типов.</span><span class="sxs-lookup"><span data-stu-id="55100-113">The lifetime of a collectible assembly is controlled by the existence of references to the types it contains and the objects that are created from those types.</span></span> <span data-ttu-id="55100-114">Общеязыковая среда выполнения не выгружает сборку, пока существует хотя бы один из следующих элементов (`T` — любой тип, определенный в сборке):</span><span class="sxs-lookup"><span data-stu-id="55100-114">The common language runtime does not unload an assembly as long as one or more of the following exist (`T` is any type that is defined in the assembly):</span></span>

- <span data-ttu-id="55100-115">Экземпляр `T`.</span><span class="sxs-lookup"><span data-stu-id="55100-115">An instance of `T`.</span></span>

- <span data-ttu-id="55100-116">Экземпляр массива `T`.</span><span class="sxs-lookup"><span data-stu-id="55100-116">An instance of an array of `T`.</span></span>

- <span data-ttu-id="55100-117">Экземпляр универсального типа, имеющий `T` в качестве одного из аргументов типа.</span><span class="sxs-lookup"><span data-stu-id="55100-117">An instance of a generic type that has `T` as one of its type arguments.</span></span> <span data-ttu-id="55100-118">Сюда входят универсальные коллекции `T`, даже если такая коллекция пуста.</span><span class="sxs-lookup"><span data-stu-id="55100-118">This includes generic collections of `T`, even if that collection is empty.</span></span>

- <span data-ttu-id="55100-119">Экземпляр <xref:System.Type> или <xref:System.Reflection.Emit.TypeBuilder>, представляющий `T`.</span><span class="sxs-lookup"><span data-stu-id="55100-119">An instance of <xref:System.Type> or <xref:System.Reflection.Emit.TypeBuilder> that represents `T`.</span></span>

   > [!IMPORTANT]
   > <span data-ttu-id="55100-120">Нужно освободить все объекты, представляющие части сборки.</span><span class="sxs-lookup"><span data-stu-id="55100-120">You must release all objects that represent parts of the assembly.</span></span> <span data-ttu-id="55100-121"><xref:System.Reflection.Emit.ModuleBuilder>, определяющий `T`, хранит ссылку на <xref:System.Reflection.Emit.TypeBuilder>, а объект <xref:System.Reflection.Emit.AssemblyBuilder> хранит ссылку на <xref:System.Reflection.Emit.ModuleBuilder>, поэтому ссылки на эти объекты нужно освободить.</span><span class="sxs-lookup"><span data-stu-id="55100-121">The <xref:System.Reflection.Emit.ModuleBuilder> that defines `T` keeps a reference to the <xref:System.Reflection.Emit.TypeBuilder>, and the <xref:System.Reflection.Emit.AssemblyBuilder> object keeps a reference to the <xref:System.Reflection.Emit.ModuleBuilder>, so references to these objects must be released.</span></span> <span data-ttu-id="55100-122">Даже наличие <xref:System.Reflection.Emit.LocalBuilder> или <xref:System.Reflection.Emit.ILGenerator>, используемых при создании `T`, препятствует выгрузке.</span><span class="sxs-lookup"><span data-stu-id="55100-122">Even the existence of a <xref:System.Reflection.Emit.LocalBuilder> or an <xref:System.Reflection.Emit.ILGenerator> used in the construction of `T` prevents unloading.</span></span>

- <span data-ttu-id="55100-123">Статическая ссылка на `T` со стороны другого динамически определяемого типа `T1`, который все еще доступен выполняющемуся коду.</span><span class="sxs-lookup"><span data-stu-id="55100-123">A static reference to `T` by another dynamically defined type `T1` that is still reachable by executing code.</span></span> <span data-ttu-id="55100-124">Например, `T1` может наследовать от `T` или `T` может быть типом параметра в методе `T1`.</span><span class="sxs-lookup"><span data-stu-id="55100-124">For example, `T1` might derive from `T`, or `T` might be the type of a parameter in a method of `T1`.</span></span>

- <span data-ttu-id="55100-125">**ByRef** для статического поля, который относится к `T`.</span><span class="sxs-lookup"><span data-stu-id="55100-125">A **ByRef** to a static field that belongs to `T`.</span></span>

- <span data-ttu-id="55100-126"><xref:System.RuntimeTypeHandle>, <xref:System.RuntimeFieldHandle> или <xref:System.RuntimeMethodHandle>, ссылающийся на `T` или на компонент `T`.</span><span class="sxs-lookup"><span data-stu-id="55100-126">A <xref:System.RuntimeTypeHandle>, <xref:System.RuntimeFieldHandle>, or <xref:System.RuntimeMethodHandle> that refers to `T` or to a component of `T`.</span></span>

- <span data-ttu-id="55100-127">Экземпляр любого объекта отражения, который можно использовать косвенно или напрямую для обращения к объекту <xref:System.Type>, представляющему `T`.</span><span class="sxs-lookup"><span data-stu-id="55100-127">An instance of any reflection object that could be used indirectly or directly to access the <xref:System.Type> object that represents `T`.</span></span> <span data-ttu-id="55100-128">Например, объект <xref:System.Type> для `T` можно получить из типа массива с типом элементов `T` или из универсального типа, имеющего аргумент типа `T`.</span><span class="sxs-lookup"><span data-stu-id="55100-128">For example, the <xref:System.Type> object for `T` can be obtained from an array type whose element type is `T`, or from a generic type that has `T` as a type argument.</span></span>

- <span data-ttu-id="55100-129">Метод `M` в стеке вызовов любого потока, где `M` — это метод `T` или метод уровня модуля, который определен в сборке.</span><span class="sxs-lookup"><span data-stu-id="55100-129">A method `M` on the call stack of any thread, where `M` is a method of `T` or a module-level method that is defined in the assembly.</span></span>

- <span data-ttu-id="55100-130">Делегат для статического метода, который определен в модуле сборки.</span><span class="sxs-lookup"><span data-stu-id="55100-130">A delegate to a static method that is defined in a module of the assembly.</span></span>

<span data-ttu-id="55100-131">Если всего один элемент из этого списка существует только для одного типа или метода в сборке, среда выполнения не может выгрузить сборку.</span><span class="sxs-lookup"><span data-stu-id="55100-131">If only one item from this list exists for only one type or one method in the assembly, the runtime cannot unload the assembly.</span></span>

> [!NOTE]
> <span data-ttu-id="55100-132">Среда выполнения не выгружает сборку, пока не выполнены методы завершения для всех элементов в списке.</span><span class="sxs-lookup"><span data-stu-id="55100-132">The runtime does not actually unload the assembly until finalizers have run for all items in the list.</span></span>

<span data-ttu-id="55100-133">В целях отслеживания времени существования сконструированный универсальный тип, например `List<int>` (в C#) или `List(Of Integer)` (в Visual Basic), который создается и используется при генерации забираемой сборки, считается определенным либо в сборке, содержащей определение универсального типа или определение одного из его аргументов типа.</span><span class="sxs-lookup"><span data-stu-id="55100-133">For purposes of tracking lifetime, a constructed generic type such as `List<int>` (in C#) or `List(Of Integer)` (in Visual Basic) that is created and used in the generation of a collectible assembly is considered to have been defined either in the assembly that contains the generic type definition or in an assembly that contains the definition of one of its type arguments.</span></span> <span data-ttu-id="55100-134">Фактически используемая сборка зависит от реализации и может изменяться.</span><span class="sxs-lookup"><span data-stu-id="55100-134">The exact assembly that is used is an implementation detail and subject to change.</span></span>

## <a name="restrictions-on-collectible-assemblies"></a><span data-ttu-id="55100-135">Ограничения забираемых сборок</span><span class="sxs-lookup"><span data-stu-id="55100-135">Restrictions on collectible assemblies</span></span>

<span data-ttu-id="55100-136">К забираемым сборкам применяются следующие ограничения:</span><span class="sxs-lookup"><span data-stu-id="55100-136">The following restrictions apply to collectible assemblies:</span></span>

- <span data-ttu-id="55100-137">**Статические ссылки.** Типы в обычной динамической сборке не могут включать статические ссылки на типы, определенные в коллективной сборке.</span><span class="sxs-lookup"><span data-stu-id="55100-137">**Static references** Types in an ordinary dynamic assembly cannot have static references to types that are defined in a collectible assembly.</span></span> <span data-ttu-id="55100-138">Например, если вы определяете обычный тип, наследующий от типа в забираемой сборке, возникает исключение <xref:System.NotSupportedException>.</span><span class="sxs-lookup"><span data-stu-id="55100-138">For example, if you define an ordinary type that inherits from a type in a collectible assembly, a <xref:System.NotSupportedException> exception is thrown.</span></span> <span data-ttu-id="55100-139">Тип в забираемой сборке может иметь статические ссылки на тип в другой забираемой сборке, но при этом время существования сборки, на которую указывает ссылка, становится равным времени существования сборки с такой ссылкой.</span><span class="sxs-lookup"><span data-stu-id="55100-139">A type in a collectible assembly can have static references to a type in another collectible assembly, but this extends the lifetime of the referenced assembly to the lifetime of the referencing assembly.</span></span>

- <span data-ttu-id="55100-140">**Взаимодействие COM.** COM-интерфейсы нельзя задавать внутри забираемой сборки, и экземпляры типов внутри забираемой сборки нельзя преобразовать в COM-объекты.</span><span class="sxs-lookup"><span data-stu-id="55100-140">**COM interop** No COM interfaces can be defined within a collectible assembly, and no instances of types within a collectible assembly can be converted into COM objects.</span></span> <span data-ttu-id="55100-141">Тип в забираемой сборке не может служить в качестве вызываемой оболочки COM (CCW) или вызываемой оболочки времени выполнения (RCW).</span><span class="sxs-lookup"><span data-stu-id="55100-141">A type in a collectible assembly cannot serve as a COM callable wrapper (CCW) or runtime callable wrapper (RCW).</span></span> <span data-ttu-id="55100-142">Однако типы в забираемых сборках могут использовать объекты, реализующие COM-интерфейсы.</span><span class="sxs-lookup"><span data-stu-id="55100-142">However, types in collectible assemblies can use objects that implement COM interfaces.</span></span>

- <span data-ttu-id="55100-143">**Вызов неуправляемого кода.** Методы, имеющие атрибут <xref:System.Runtime.InteropServices.DllImportAttribute>, не будут компилироваться, если они объявлены в забираемой сборке.</span><span class="sxs-lookup"><span data-stu-id="55100-143">**Platform invoke** Methods that have the <xref:System.Runtime.InteropServices.DllImportAttribute> attribute will not compile when they are declared in a collectible assembly.</span></span> <span data-ttu-id="55100-144">Инструкция <xref:System.Reflection.Emit.OpCodes.Calli?displayProperty=nameWithType> не может использоваться в реализации типа в забираемой сборке, и такие типы невозможно маршалировать в неуправляемый код.</span><span class="sxs-lookup"><span data-stu-id="55100-144">The <xref:System.Reflection.Emit.OpCodes.Calli?displayProperty=nameWithType> instruction cannot be used in the implementation of a type in a collectible assembly, and such types cannot be marshaled to unmanaged code.</span></span> <span data-ttu-id="55100-145">Однако вы можете вызывать машинный код с помощью точки входа, объявленной в незабираемой сборке.</span><span class="sxs-lookup"><span data-stu-id="55100-145">However, you can call into native code by using an entry point that is declared in a non-collectible assembly.</span></span>

- <span data-ttu-id="55100-146">**Маршалинг.** Объекты (в частности, делегаты), определенные в забираемых сборках, невозможно маршалировать.</span><span class="sxs-lookup"><span data-stu-id="55100-146">**Marshaling** Objects (in particular, delegates) that are defined in collectible assemblies cannot be marshaled.</span></span> <span data-ttu-id="55100-147">Это ограничение действует для всех временных выдаваемых типов.</span><span class="sxs-lookup"><span data-stu-id="55100-147">This is a restriction on all transient emitted types.</span></span>

- <span data-ttu-id="55100-148">**Загрузка сборки.** Порождение отражения — это единственный поддерживаемый механизм для загрузки забираемых сборок.</span><span class="sxs-lookup"><span data-stu-id="55100-148">**Assembly loading** Reflection emit is the only mechanism that is supported for loading collectible assemblies.</span></span> <span data-ttu-id="55100-149">Сборки, загружаемые любым другим образом, выгрузить невозможно.</span><span class="sxs-lookup"><span data-stu-id="55100-149">Assemblies that are loaded by using any other form of assembly loading cannot be unloaded.</span></span>

- <span data-ttu-id="55100-150">**Контекстно-зависимые объекты.** Контекстно статические переменные не поддерживаются.</span><span class="sxs-lookup"><span data-stu-id="55100-150">**Context-bound objects** Context-static variables are not supported.</span></span> <span data-ttu-id="55100-151">Типы в забираемой сборке не могут расширять <xref:System.ContextBoundObject>.</span><span class="sxs-lookup"><span data-stu-id="55100-151">Types in a collectible assembly cannot extend <xref:System.ContextBoundObject>.</span></span> <span data-ttu-id="55100-152">Однако код в забираемых сборках может использовать контекстно привязанные объекты, определенные в другом месте.</span><span class="sxs-lookup"><span data-stu-id="55100-152">However, code in collectible assemblies can use context-bound objects that are defined elsewhere.</span></span>

- <span data-ttu-id="55100-153">**Потокостатические данные.** Потокостатические переменные не поддерживаются.</span><span class="sxs-lookup"><span data-stu-id="55100-153">**Thread-static data** Thread-static variables are not supported.</span></span>
