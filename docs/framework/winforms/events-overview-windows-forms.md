## <a name="delegates-and-their-role"></a><span data-ttu-id="05825-114">Делегаты и их роли</span><span class="sxs-lookup"><span data-stu-id="05825-114">Delegates and Their Role</span></span>
 <span data-ttu-id="05825-115">Делегаты — это классы, обычно используемые в .NET Framework для создания механизмов обработки событий.</span><span class="sxs-lookup"><span data-stu-id="05825-115">Delegates are classes commonly used within the .NET Framework to build event-handling mechanisms.</span></span> <span data-ttu-id="05825-116">Делегаты приблизительно соответствуют указателям на функции, обычно используемым C++ в визуальных и других объектно-ориентированных языках.</span><span class="sxs-lookup"><span data-stu-id="05825-116">Delegates roughly equate to function pointers, commonly used in Visual C++ and other object-oriented languages.</span></span> <span data-ttu-id="05825-117">В отличие от указателей функций делегаты объектно-ориентированы, типобезопасны и безопасны.</span><span class="sxs-lookup"><span data-stu-id="05825-117">Unlike function pointers however, delegates are object-oriented, type-safe, and secure.</span></span> <span data-ttu-id="05825-118">К тому же, если указатель функций содержит только ссылку на определенную функцию, то делегат содержит ссылку на объект и ссылки на один или несколько методов в этом объекте.</span><span class="sxs-lookup"><span data-stu-id="05825-118">In addition, where a function pointer contains only a reference to a particular function, a delegate consists of a reference to an object, and references to one or more methods within the object.</span></span>

 <span data-ttu-id="05825-119">Эта модель событий использует *делегаты* для привязки событий к методам, которые используются для их обработки.</span><span class="sxs-lookup"><span data-stu-id="05825-119">This event model uses *delegates* to bind events to the methods that are used to handle them.</span></span> <span data-ttu-id="05825-120">Делегаты позволяют другим классам записывать уведомление о событии, определяя метод обработки.</span><span class="sxs-lookup"><span data-stu-id="05825-120">The delegate enables other classes to register for event notification by specifying a handler method.</span></span> <span data-ttu-id="05825-121">При возникновении события делегат вызывает соответствующий метод.</span><span class="sxs-lookup"><span data-stu-id="05825-121">When the event occurs, the delegate calls the bound method.</span></span> <span data-ttu-id="05825-122">Дополнительные сведения об определении делегатов см. в разделе [события](../../standard/events/index.md).</span><span class="sxs-lookup"><span data-stu-id="05825-122">For more information about how to define delegates, see [Events](../../standard/events/index.md).</span></span>

<span data-ttu-id="05825-123">Делегаты можно связать с одним или несколькими методами, создав так называемую многоадресную рассылку.</span><span class="sxs-lookup"><span data-stu-id="05825-123">Delegates can be bound to a single method or to multiple methods, referred to as multicasting.</span></span> <span data-ttu-id="05825-124">При создании делегата для события обычно создается многоадресное событие (или Windows).</span><span class="sxs-lookup"><span data-stu-id="05825-124">When creating a delegate for an event, you (or the Windows) typically create a multicast event.</span></span> <span data-ttu-id="05825-125">Редким исключением является событие, вызывающее выполнение определенной процедуры (например, отображение диалогового окна), которая не будет логически повторяться несколько раз за событие.</span><span class="sxs-lookup"><span data-stu-id="05825-125">A rare exception might be an event that results in a specific procedure (such as displaying a dialog box) that would not logically repeat multiple times per event.</span></span> <span data-ttu-id="05825-126">Сведения о создании делегата многоадресной рассылки см. [в разделе Объединение делегатов (многоадресные делегаты)](../../csharp/programming-guide/delegates/how-to-combine-delegates-multicast-delegates.md).</span><span class="sxs-lookup"><span data-stu-id="05825-126">For information about how to create a multicast delegate, see [How to combine delegates (Multicast Delegates)](../../csharp/programming-guide/delegates/how-to-combine-delegates-multicast-delegates.md).</span></span>

 <span data-ttu-id="05825-127">Делегат многоадресной рассылки поддерживает список вызова методов, к которым он привязан.</span><span class="sxs-lookup"><span data-stu-id="05825-127">A multicast delegate maintains an invocation list of the methods it is bound to.</span></span> <span data-ttu-id="05825-128">Делегат многоадресной рассылки поддерживает метод <xref:System.Delegate.Combine%2A>, позволяющий добавить метод в список вызова, и метод <xref:System.Delegate.Remove%2A>, позволяющий его удалить.</span><span class="sxs-lookup"><span data-stu-id="05825-128">The multicast delegate supports a <xref:System.Delegate.Combine%2A> method to add a method to the invocation list and a <xref:System.Delegate.Remove%2A> method to remove it.</span></span>

 <span data-ttu-id="05825-129">Когда приложение регистрирует событие, элемент управления порождает это событие, вызывая для него делегат.</span><span class="sxs-lookup"><span data-stu-id="05825-129">When an event is recorded by the application, the control raises the event by invoking the delegate for that event.</span></span> <span data-ttu-id="05825-130">Делегат, в свою очередь, вызывает соответствующий метод.</span><span class="sxs-lookup"><span data-stu-id="05825-130">The delegate in turn calls the bound method.</span></span> <span data-ttu-id="05825-131">В самом распространенном случае (делегат многоадресной рассылки) делегат вызывает каждый метод связки из списка вызова по очереди, что обеспечивает уведомление один-ко-многим.</span><span class="sxs-lookup"><span data-stu-id="05825-131">In the most common case (a multicast delegate) the delegate calls each bound method in the invocation list in turn, which provides a one-to-many notification.</span></span> <span data-ttu-id="05825-132">Данная стратегия означает, что элементу управления не нужно вести список целевых объектов для уведомления о событии, поскольку записью и уведомлением занимается делегат.</span><span class="sxs-lookup"><span data-stu-id="05825-132">This strategy means that the control does not need to maintain a list of target objects for event notification—the delegate handles all registration and notification.</span></span>

 <span data-ttu-id="05825-133">Делегаты также позволяют связать с одним методом несколько событий, чтобы использовать уведомление по типу многие-к-одному.</span><span class="sxs-lookup"><span data-stu-id="05825-133">Delegates also enable multiple events to be bound to the same method, allowing a many-to-one notification.</span></span> <span data-ttu-id="05825-134">Например, событие нажатия на кнопку и событие выбора команды в меню вызывают один и тот же делегат, который вызывает один и тот же метод, обрабатывающий эти события одинаковым образом.</span><span class="sxs-lookup"><span data-stu-id="05825-134">For example, a button-click event and a menu-command–click event can both invoke the same delegate, which then calls a single method to handle these separate events the same way.</span></span>

 <span data-ttu-id="05825-135">В делегатах используется динамический механизм связки: во время выполнения делегат может быть связан с любым методом, подпись которого совпадает с подписью обработчика событий.</span><span class="sxs-lookup"><span data-stu-id="05825-135">The binding mechanism used with delegates is dynamic: a delegate can be bound at run time to any method whose signature matches that of the event handler.</span></span> <span data-ttu-id="05825-136">С помощью этой функции можно устанавливать или изменять метод связки в зависимости от условий и динамически привязывать обработчик событий к элементу управления.</span><span class="sxs-lookup"><span data-stu-id="05825-136">With this feature, you can set up or change the bound method depending on a condition and to dynamically attach an event handler to a control.</span></span>
