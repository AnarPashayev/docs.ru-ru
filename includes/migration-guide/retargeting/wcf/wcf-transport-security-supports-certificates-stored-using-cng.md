|<span data-ttu-id="97c43-102">Подробные сведения</span><span class="sxs-lookup"><span data-stu-id="97c43-102">Details</span></span>|<span data-ttu-id="97c43-103">Начиная с приложений, предназначенных для .NET Framework 4.6.2, защита транспорта WCF поддерживает сертификаты, сохраненные с использованием библиотеки шифрования Windows (CNG).</span><span class="sxs-lookup"><span data-stu-id="97c43-103">Starting with apps that target the .NET Framework 4.6.2, WCF transport security supports certificates stored using the Windows Cryptography Library (CNG).</span></span> <span data-ttu-id="97c43-104">Эта поддержка ограничивается сертификатами с открытым ключом, длина экспоненты которого не превышает 32 бита.</span><span class="sxs-lookup"><span data-stu-id="97c43-104">This support is limited to certificates with a public key that has an exponent no more than 32 bits in length.</span></span> <span data-ttu-id="97c43-105">Если приложение предназначено для .NET Framework 4.6.2, эта функция включена по умолчанию. |<span data-ttu-id="97c43-106">Предложение</span><span class="sxs-lookup"><span data-stu-id="97c43-106">Suggestion</span></span>|<span data-ttu-id="97c43-107">Для приложений, которые предназначены для .NET Framework 4.6.1 и более ранних версий, но работают в .NET Framework 4.6.2, можно включить поддержку сертификатов CNG, добавив следующую строку в раздел <code>&lt;runtime&gt;</code> файла app.config или web.config:</span><span class="sxs-lookup"><span data-stu-id="97c43-107">Apps that target the .NET Framework 4.6.1 and earlier but are running on the .NET Framework 4.6.2 can enable support for CNG certificates by adding the following line to the <code>&lt;runtime&gt;</code> section of the app.config or web.config file:</span></span><pre><code class="lang-xml">&lt;runtime&gt;&#13;&#10;&lt;AppContextSwitchOverrides value=&quot;Switch.System.ServiceModel.DisableCngCertificates=false&quot;
