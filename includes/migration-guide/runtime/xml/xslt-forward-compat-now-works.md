### <a name="xslt-forward-compat-now-works"></a>Теперь обеспечивается прямая совместимость с XSLT

#### <a name="details"></a>Подробнее

В .NET Framework 4 XSLT 1.0 имелись следующие проблемы с прямой совместимостью:<ul><li>Загрузка таблицы стилей завершалась сбоем, если ее версия была установлена равной 2.0, и средство синтаксического анализа обнаруживало нераспознанную конструкцию XSLT 1.0.</li><li>Конструкции <code>xsl:sort</code> не удавалось сортировать данные, если версия таблицы стилей была установлена равной 1.1.</li></ul>В .NET Framework 4.5 эти проблемы исправлены, и режим прямой совместимости с XSLT 1.0 работает нормально.

#### <a name="suggestion"></a>Предложение

Эта проблема не затрагивает большинство приложений, однако в некоторых случаях сортировка данных теперь будет выполняться по-другому с учетом конструкции xsl:sort. Если в таблицах стилей версии 1.1 используется конструкция <code>xsl:sort</code>, убедитесь, что приложения не зависят от порядка данных без сортировки. Если приложения зависят от параметров сортировки версии 4.0, удалите конструкцию <code>xsl:sort</code> из таблицы стилей.
